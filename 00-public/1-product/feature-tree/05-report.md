# 功能树 05：Report（报告场域）

**Owner：** 娄跃、陈波
**日期：** 2026-04-21
**版本：** v0.4.0 对齐版（L1+L2）
**对应 Spec：** `spec/design/learner/05-report-classroom.md`

---

## 一句话定位

学员查看个人学习报告 + 与 Neo 讨论解读。报告不是"宣判"，是"对话"，可质疑、提问、共同解读成长。

---

## 功能结构

```
A. Neo 在报告场域
   A1 Neo 身份（Tutor；可读全部课堂数据，以当前 Course 为主视角）
   A2 报告解读引导（渐进式呈现）
   A3 证据追溯（逐层钻取：结论→证据摘要→原始对话）
   A4 知识关联（报告讨论中关联课程知识点）
   A5 ★报告访问权限矩阵 [补漏，统一三重入口]
       A5.1 学员自查（Hall E2 课程目录进入 + Hall G1 Leo 推荐进入 + Course 结束后自动进入）
           - 只能看自己的报告
           - 看不到具体排名名次，只看分位区间
           - 分组标签不可见
       A5.2 HR 代查（Operation D1 个人报告）
           - 可看项目内所有学员的个人报告
           - 额外可见分组标签（高活跃 / 稳健 / 需关注）
           - 可见具体排名
       A5.3 运营代查（Operation D1 个人报告）
           - 权限同 HR，本期完全一致
       A5.4 API 级隔离（学员端 API 不返回排名字段和分组标签，详见 F4）
       A5.5 HR 体验模式（通过 Topbar 端口切换以学员身份看报告，报告按学员权限呈现）

B. 报告三部分结构 ★v0.4.0 简化（从 8 模块 → 3 部分）
   B1 第一部分：学了什么（内容回顾 — 关键知识点和方法论）
   B2 第二部分：怎么表现的（表现复盘 — 高光 + 待改进，具体案例支撑）
   B3 第三部分：接下来做什么（应用计划 — 可执行步骤）

C. 报告生成
   C1 触发与入口（Course 结束后固定生成，本期不做辅导随时进入）
   C2 渐进式加载（祝贺动画 + 后台并行生成 + 分段呈现）
   C3 碎片汇聚降级（超时 >30s / 失败三级降级：已得渲染 / 标"数据收集中" / 重试 3 次失败标"数据不足"）
   C4 异常恢复与断点恢复 [补漏明确]
       C4.1 重新进入（学员关闭场域后可从 Course 列表重新进入已生成报告）
       C4.2 生成中断恢复（已计算模块缓存复用，未完成模块重新计算）
       C4.3 讨论断点（E 节 Neo 对话历史保留，再次进入可接续讨论）
       → 规范沿用 Infra M 统一断点恢复机制
   C5 情感化文案（成长型语言 + 个性化引用具体行为 + 数据不足兜底）

D. BI 图表（通用组件库，跨端复用）
   D1 五种图表（雷达 / 热力 / 饼 / 折线 / 柱状）
   D2 图表 ↔ 部分映射（B1 纯文字 / B2 雷达 + 热力 + 柱 / B3 纯文字）
   D3 可交互（点击图表数据点 → Agent 解读）
   D4 数据可追溯（点击数据点查看原始记录）
   D5 联动事件协议 `[REF:chart=xx,dim=yy]`（组件库跨端复用，也供 Operation 消费）

E. 报告解析（双向讨论）
   E1 提问报告内容（Neo 引用行为证据回答）
   E2 质疑评估（不是宣判是对话，双向讨论）
   E3 深入解读（按需展开）
   E4 讨论不修改报告（反馈保存供模型优化）

F. 成长时间线 + 匿名分位
   F1 个人成长时间线（跨 Course 折线，维度 ID 匹配对齐非名称匹配）
   F2 单/双 Course 降级（单点用雷达快照 / 双 Course 加"趋势仅供参考"提示）
   F3 匿名分位区间（柔和灰色标尺，前 25% / 中 50% / 后 25%，不展示名次）
   F4 ★API 级隔离（学员端不返回排名字段，防前端泄漏）
   F5 成长幅度计算（消费 Evaluation 的结果）

G. 导出 PDF
   G1 学员导出（含图表静态截图 + 文字解读）
   G2 图表服务端静态化（渲染为图片 + AI 生成文字解读）
   G3 批量导出接口（本期不做，预留给 HR / 运营）

H. 报告数据消费（消费 Evaluation 产出）
   H1 学习特质碎片（Activity 级）
   H2 认知转化碎片（Quiz 级）
   H3 能力评估碎片（对练级）
   H4 被动信号碎片（会话级）
   H5 BEI 案例碎片（调研）
   H6 行为改变证据链汇聚（初始 → 过程 → 最终）

I. 布局（两栏 + 可折叠导航）
   I1 可折叠导航区（Project 目录）
   I2 左栏：报告内容（三部分 + BI 图表，渐进式呈现）
   I3 右栏：Neo 对话（质疑 / 追问 / 解读）
   → 笔记悬浮球常驻（见 Hall F）
```

## 跨模块依赖

| 对象模块 | 依赖关系 |
|---------|---------|
| Evaluation (09-A) | H 所有碎片来源 + F5 成长幅度计算 + 分位计算 |
| Infra (07) | 图表组件库 / 布局（两栏+可折叠导航，非三栏）/ PDF 服务端渲染 |
| Hall (01) | 报告入口来自 Leo 推荐 |
| Operation (06) | D 图表组件库跨端复用 / 批量导出接口（G3 预留） |
| Lecture / Practice / Inquiry | I 信号采集是 H 碎片来源的上游 |

---

## 与 v0.3.3 差异

- ★ **B1-B8 八模块 → 三部分**（最大简化）
- 原 B2 能力成长 / B3 认知转化 / B5 技能提升 / B6 成长表现 / B7 综合评估 并入新 B2（课堂表现复盘）
- 原 B8 行动建议 → 新 B3（下一步应用计划）
- 原 B1 高光时刻 → 新 B2 高光表现子节
- 报告场域 AI 从"辅导教练"正式改名 **Neo**
- 布局从三栏改为 **两栏 + 可折叠导航**（v0.4.0 spec 明确）
- E1 成长时间线保留，但作为 F 节独立组件（不在三部分内）
