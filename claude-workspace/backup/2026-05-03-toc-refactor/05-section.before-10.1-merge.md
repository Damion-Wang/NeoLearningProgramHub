## 10 · 管理端（待 DM 审）

> 本章描述管理端的整体形态 + 4 个核心模块。
> AI 角色 = Ora（详见 § 3.3）。
> 用户 = HR + 项目运营 + admin（详见 § 4.3）。
> 详细字段 / UI 细节 详见 [04-v0.5.0-backlog.md](04-v0.5.0-backlog.md)。

### 10.1 管理端总览

#### 10.1.1 业务定义

管理端是**HR / 项目运营 / admin 监控项目运行 + 产出可举证 ROI 报告 + 配置项目运行参数**的产品形态。AI 角色 = **Ora**（详见 § 3.3 / § 3.4.4 上下文范围）—— Ora 在管理端服务管理员，**给建议 + 解读 + 选项 / HR 拍板**。

**与学员端的关系**：学员端（§ 9）+ 管理端（§ 10）是**两个独立产品端**，通过 Database 间接协作（详见 § 3.1.3）—— Ora 不直接联系学员 / Neo 不读管理端数据。

**管理端的两个运行状态**：

| 状态 | 触发 | 主要使用者 | 主要行为 |
|------|------|----------|---------|
| **开营前 · 项目配置状态** | 签单后 admin 账号开通 → 确认开营前 | admin | 全量配置项目（项目信息 / 学员名单 / 角色 / 周期 / 催学规则 / 平台个性化）+ 创建管理员账号 + 锁定服务周期（详见 § 10.1.2 表 2 + § 6.4 / § 6.5）|
| **开营后 · 项目运营状态** | 确认开营触发后 → 服务周期内 | 管理员（HR + 项目运营）| 日常监控 / 询问 / 报告生成 / 材料导出 / 学员干预 / 配置零星变更（详见 § 10.1.2 表 1 + § 6.6）|

> **状态切换点**：**"确认开营"按钮触发**（详见 § 6.5）—— 触发连锁动作（账号开放 + 系统消息 + 状态切换），管理端从配置态切到运营态。

#### 10.1.2 管理员核心行为与对应模块

> 本节给出管理员在管理端的核心行为概览（行为类型 → 时机 → 对应功能板块）。具体的串联用户故事 / 详细操作流程 留 behaviour 文档 / 各模块章节展开。

**表 1 · 开营前 admin 行为**（项目配置准备 · 3 类）

| # | 行为类型 | 时机 | 对应功能板块 |
|---|---------|------|------------|
| 1 | **开营前全量配置** | 开营前一次 | 项目配置 Program Config |
| 2 | **创建管理员账号 + 角色分配** | 开营前一次 | 项目配置 Program Config（人员名单）|
| 3 | **锁定服务周期** | 确认开营触发 | 项目配置 Program Config |

**表 2 · 开营后管理员核心行为**（HR + 项目运营 合并 · 7 类）

| # | 行为类型 | 时机 | 对应功能板块 |
|---|---------|------|------------|
| 1 | **项目情况监控** | 学习期持续 | 首页 Home |
| 2 | **具体项目情况询问** | 按需 | 首页 Home（Ora 问答）|
| 3 | **定期总结报告** | 阶段性 / 里程碑触发 | 报告中心 Report Center |
| 4 | **项目材料汇总导出** | 阶段性 | 报告中心 Report Center |
| 5 | **学员进度干预** | 信号触发 / 按需 | 项目配置 Program Config + 消息中心 Message |
| 6 | **项目情况配置** | 学习期内零星 | 项目配置 Program Config |
| 7 | **企业个性化配置** | 开营前一次 / 后偶有 | 项目配置 Program Config |

#### 10.1.3 管理端模块清单

| 模块 | 类型 | 一句话 |
|------|------|--------|
| **首页 Home** | 监控 | 项目健康度看板 + Ora 问答 + 异常学员发现 |
| **报告中心 Report Center** ★ | 交付 | Ora 拉数据 + 多轮对话 + HITL 编辑 + 导出 + 历史报告归档（管理端核心交付能力）|
| **项目配置 Program Config** | 配置 | 项目信息 / 学员名单 / 角色分配 / 催学规则 / 平台个性化 |
| **消息中心 Message** | 沟通 | 手动定向发送富文本 + 发送记录 |

#### 10.1.4 入口与端口切换

- **默认入口**：管理员账号登录 → 默认进入首页 Home
- **模块切换**：管理端左侧导航栏 4 模块直达
- **端口切换**：一人多角色（如某 HR 同时是某 Course 学员）→ 顶部端口切换器（详见 § 7.5）
- **多管理员场景**：本期 3 类管理员（admin / HR / 项目运营）**共用同一管理端**，按账号区分各自的报告库 / 操作记录

#### 10.1.5 角色协同

| 角色 | 定位 | 主要行为 |
|------|------|---------|
| **admin** | 项目根账号 | 开营前配置全权 + 创建管理员账号 + 锁定服务周期 |
| **管理员**（HR + 项目运营 合并）| 日常运营 + 阶段性报告 | 看板巡视 / 异常学员处置 / 报告生成 / 配置变更 / 对外汇报 |

**HR vs 项目运营 当前版本合并处理的理由**：

- 当前版本 HR / 项目运营 在产品功能上**完全一致**（都用同一管理端 / 同一权限 / 同一交互）
- 实际执行中由**业务侧约定分工**（如 HR 偏战略报告 / 项目运营偏日常执行），不是产品强制
- 本期不做 HR / 项目运营 权限差异化，未来按客户反馈迭代

> **与 § 1.3 三方价值对应**：§ 1.3 中 HR + 项目运营 列为不同价值视角，但**产品形态合一**——属于 CLAUDE.md "三方价值实际是两方"（学员 + HR/管理员）的具体落地。

### 10.2 首页 Home

#### 10.2.1 业务定义

管理员日常监控项目运行健康度的场域。三维度监控 + 热力图 + Ora 双交互。

#### 10.2.2 三维度监控

| 维度 | 内容 |
|------|------|
| **项目维度** | 项目整体健康度 / 进度 / 完成率 |
| **人员维度** | 学员个体进度 / 异常学员 / 活跃度分布 |
| **内容维度** | 各 Course 学习情况 / 卡点分布 / Quiz 通关率 |

#### 10.2.3 热力图

学员行为热力可视化——每个学员每日活跃度色块 / 每个 Course 学员通过率色块。

#### 10.2.4 Ora 双交互模式
| 模式 | 触发 | 行为 |
|------|------|------|
| **看板被动呈现** | 默认呈现 | 三维度数据 + 热力图 + Ora 自动生成的项目健康度解读 |
| **Ora 问答** | 管理员主动提问 | "为什么 W4 脱训突增？" / "张三这周参与度怎样？" → Ora 用证据回答 |

> **本期不做**：看板点击 → Ora 实时解释——这是未来形态（详见 § 3.3.4）

#### 10.2.5 异常学员处置

按 § 6.6.2：

- **平台层**：Ora 看板 / 问答 中发现异常学员（脱训 / 进度落后 / Quiz 频繁失败）
- **线下层**：项目运营**线下跟进**该学员（电话 / 微信 / 1on1 沟通）
- **平台不强制干预**——干预靠人 / 渠道完成

#### 10.2.6 颜色条 / 排序 / 详情页

- **颜色条标识健康度**（绿 / 黄 / 红）
- **异常自动排顶**（异常项目 / 异常学员 自动排在顶部）
- **详情页"建议动作"** —— Ora 给出建议动作（详见 § 3.3.5）

### 10.3 报告中心 Report Center

> 这是 § 10 的**核心模块**——管理端的主要交付能力，承担中期反馈 / 半程 review / 结项报告 / 剩余服务期内的新报告 全部生成场景（详见 § 6.7 + § 6.8）+ 历史报告归档查看 + 每管理员独立报告库管理。

#### 10.3.1 业务定义

管理员产出对外可举证报告 + 历史报告归档查看 + 报告库管理 的场域——Ora 拉全程数据 + 多轮对话生成初稿 + 管理员 HITL 编辑 + 导出 PDF / Word + 历史报告查询。

#### 10.3.2 引擎核心 · 每管理员独立报告库
- **每个管理员**有**独立的报告库**——存放该管理员生成 / 编辑的全部 team report
- **里程碑触发时**给每个管理员一份**独立草稿副本**
- **本期不做协同分享 / 不做实时多人编辑** —— 需协同时由管理员**导出后线下同步**

> 这个设计简化了多人协同的复杂度——研发只需保证每个管理员的报告库独立，不需做实时协同编辑。

#### 10.3.3 prompt 模板由"关联动作 / 评估要点"驱动
- 管理员**不需要选模板，也不手写 prompt**
- 管理员在创建 team report（或里程碑配置）时**填"关联动作 / 评估要点"**（如"评估学员中期参与度 + 概念掌握度"）
- 系统据此**自动生成**对应的 prompt 模板

> 本期 v0.5.0 优先做 **"结项报告 prompt 模板"** —— 用作里程碑结营时的默认 prompt。其他模板（中期 / 半程 / 自定义）后续逐步加（详见 § 6.7.1）。

#### 10.3.4 多轮对话生成 + HITL 编辑

生成流程：

```
里程碑到期触发 / 管理员手动触发
   ↓
Ora 拉项目全程数据
   ↓
Ora 与管理员多轮对话（确认范围 / 学员维度 / 数据筛选 / 文本风格）
   ↓
Ora 生成初稿
   ↓
管理员 HITL 编辑（增删改 / 调整结构 / 改文案）
   ↓
导出 PDF / Word
```

#### 10.3.5 与里程碑的关联

详见 § 6.6.3 + § 6.7.1：

- 里程碑到期当天 → Ora 自动生成 team report 草稿（每管理员一份副本）
- 站内消息提前 3 天每日提醒管理员
- Ora 在草稿生成后即时推送通知
- 过期未处理 → 提醒保留在列表（不堆累 / 不重复推送 / 不升级渠道）

#### 10.3.6 模板含义澄清
按 § 6.7.1：

> 本节"模板"指的是 **prompt 约束的写作规则**（即给 Ora 的"写作 brief / 文章骨架 / 核心要素清单"），**不是固定文章模板或 HTML 模板**。
>
> Ora 基于模板的写作约束 + 项目实际数据 + 关联动作 / 评估要点 → 生成具体文本。

#### 10.3.7 本期不做

- ❌ **学员个人结营报告** —— 学员小结全部在阶段 5 学习期内通过 Course 个人小结 Activity 完成（详见 § 6.7）
- ❌ **多模板自由切换 UI** —— 本期仅"结项报告 prompt 模板"
- ❌ **协同分享 / 实时多人编辑** —— 见 § 10.3.2

### 10.5 消息中心 Message

#### 10.5.1 业务定义

管理员对学员定向沟通的场域——支持手动发送富文本消息 + 系统自动催学消息的统一查看与跟踪。

#### 10.5.2 GUI 与交互

| 子模块 | 功能 |
|-------|------|
| **手动定向发送** | 项目运营从学员名单中**手动选人** + 编辑富文本 + 发送 |
| **发送记录** | 手动 + 自动催学消息**统一查看**——含对象 / 时间 / 内容 / 投递日志 |

#### 10.5.3 与催学规则的关系

按 § 6.6.5：

- **全员催学** = 通过催学规则自动触发（系统层）
- **异常学员定向催** = 通过本模块**手动选人**发送（运营层）
- 两者的发送记录在本模块**统一呈现**

#### 10.5.4 投递日志

- 完整记录：谁收到 / 是否查看 / 是否点击链接
- 本期**不支持学员回应**（学员无法在催学消息上回复）

#### 10.5.5 本期不做

- ❌ **邮件 / IM / SMS 渠道**（仅站内）
- ❌ **消息已读率 dashboard**（投递日志后台可查，前端不展示）
- ❌ **消息模板库 UI**（文案模板配置在催学规则模块，详见 § 6.6.5）
