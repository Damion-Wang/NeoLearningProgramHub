## 0. 7 动作的操作化定义（与 spec 对齐前的统一口径）

为了把 spec 文本映射到 7 动作，先固定每个动作的操作化定义：

| 动作 | 操作化定义 | spec 中最接近的概念 |
|----|---------|------------------|
| **讲授** | Neo 主动输出知识结构、概念、方法论。Tutor 单向输出为主 | 授课口播 / ContentSkill / 主动讲解 PPT |
| **解惑** | 学员有具体疑问时 Neo 给出针对性答复。被动响应 | 答疑口播 / 学员插嘴提问 / 大厅 ASK 策略 |
| **讨论** | Neo 与学员就一个开放主题展开多轮交换观点。双向对等 | 大厅 5 种对话策略中 ASK+REVIEW+FRAMEWORK_GUIDED 的组合 / 报告解读对话 |
| **考研** | Neo 用题目/任务检测学员是否掌握。评估输出 | Quiz / ASSESSMENT_TAG / passConditions / 量表 / BEI |
| **练习** | Neo 给学员行为级任务进行实操，重复至熟练 | 对练 Roleplay / 学员主动要求出题 / 重练 |
| **追问** | Neo 不放过含糊回答，向"为什么/具体怎么"深挖 | 苏格拉底式追问 / DEEPENING / STAR 完整性追问 / CHALLENGER 模式 |
| **反思** | Neo 引导学员复盘行为—结果—认知，形成新认知或行动 | 复盘引导 / OBSERVER 模式 / 报告 Part2 解读 / 自评 vs 实际对比 |

## 2. 关键发现（10 条）

1. **大厅 Neo 的"讲授"明确度最低**：spec 把 Bloom 概念层的讲授责任甩给授课场域，大厅 Neo 只在"应用层/迁移层"做引导。但**线下 1v1 老师常在闲聊中即兴讲授**——spec 没有定义这个动作，会导致大厅 Neo 在学员问"什么是 PIN 法"时陷入两难（推回授课 vs 简短讲）。
2. **大厅"考研"完全缺失**：5 种对话策略和 8 种信号都是"识别"，没有"出题验证"。Neo 在大厅观察到学员说"我懂了"时，spec 没给 Neo 出一道题验证的能力——这是与线下老师"那我考你一下"动作的核心 gap。
3. **大厅"练习"通过推荐对练 Activity 间接实现**：spec 让 Neo 在大厅引导学员去对练场域（01-hall §ContentSkill 应用层策略：直接引导结合学员真实工作场景做应用指导）。但"应用指导"是讨论级，不是练习级——Neo 在大厅不会让学员做行为级演练。
4. **授课"讨论"被压缩为 SCO 间插入的单点互动**：02-lecture §交互 #7 明确"互动问题必须与当前内容教学相关，不能泛化"。这意味着授课场域的讨论始终绑定在当前 SCO 上，不能展开成开放式深度讨论——这与"线下老师在课堂上让大家聊 5 分钟"差距大。
5. **授课"追问"的边界没有 spec 化**：当 Quiz 答错或答含糊，Neo 是否追问、追问几轮、什么条件停下，spec 全部交给 AOM 脚本（"按 AOM 脚本中定义的反馈规则"）。这等于把追问能力外化给内容生产者——平台层没有兜底追问机制。
6. **对练阶段"讲授"被明令禁止**：03-practice §角色扮演阶段行为边界写"不能直接给答案"。这是正确的设计，但**意味着 Neo 在对练中遇到学员完全不会的情况无路可走**——只能给方向提示。spec 没说"如果学员真的卡住了 5 轮怎么办"。
7. **对练"考研"集成在剧本里，平台无独立机制**：剧本定义成功标准/技能阶梯/passConditions，平台是被动播放器。这导致同一对练 Activity 的考研严格度由剧本决定，Neo 没有横向比较能力（"你这次比上次怎么样"是基于行为数据，不是基于 Neo 重新评估）。
8. **调研"反思"被刻意削弱**：0424 决策 C-02 把第三段从"信息确认复盘"改为"成果澄清"，明确"复盘"留给对练。这是 spec 内部的重要边界——调研是采集，反思留给对练和报告——但意味着学员在调研中不会被"逼着想清楚"，调研对认知改变的贡献度有限。
9. **报告"练习/考研"完全留白**：报告是回顾 + 解读，Neo 不会在报告里"那我们再来一题验证一下"或"咱们再练一遍"。这与线下老师"看完成绩单后再让你做一道补救练习"的常见动作不同。
10. **7 动作在 5 场域的分布严重不均**：明确格仅在"对练"和"授课"集中（共 8 个明确格），大厅有 4 个明确 + 报告 4 个明确，调研只有 3 个明确。**未来打通的关键不是补每个空格，而是回答"哪些缺失是设计取舍、哪些是真 gap"**。

## 4. Leo 取消后的连锁影响（需要在调研中验证）

Leo 角色完全取消（2026-04-26 决策升级，requirements §4 + common/03-ai-brand §1.2）后，下列假设需要在调研中验证：

- **统一 Neo 形象的认知负担**：spec 假设"用户视角合并 + 后台合并"会让用户更轻松，但**反向风险是 Neo 在大厅闲聊和在场域严肃教学的语气切换**会被用户感知为"人格分裂"。需要在用户访谈中检验：用户能否接受同一形象的能力切换？
- **Neo 在大厅的 5 种策略 vs 在场域的 AOM 脚本**：原本 Coach（大厅）和 Tutor（场域）分两套 prompt，是因为侧重不同。合并后要求**一套 Neo prompt 覆盖所有场域** + 场域 prompt 库变体。开发实现上是否能保持 7 动作的一致性（例如追问风格在大厅是认知链提问，在对练是苏格拉底式，调研是 STAR 缺失元素——这些表面不同的策略本质都是"追问"，合并后是否要统一）？
- **HR 期待的"Neo 是教学督导"还是"Neo 是同伴陪练"**：原 Leo 偏督导（结构化教学），Neo 偏陪伴（关系建立）。合并后 HR 调研中需要确认：HR 在哪些场景希望 Neo 严肃，哪些场景希望 Neo 温和？这关系到 7 动作中"讲授/考研"和"讨论/反思"的优先级。
- **GROW/coaching-skills 框架的承载位置**：Coach 能力（5 种人格、8 种信号、6 种对话阶段）现在是 Neo 在大厅的能力。但**学员可能在场域中也需要被教练**（例如对练复盘时学员情绪低落）。spec 没说大厅 Coach 能力是否在场域局部启用，还是严格分场域——这是 Leo 取消后留下的真空。
- **课程包内容方对 Neo 的期待**：原本课程包/AOM 脚本面对 Tutor 角色设计。Leo 取消后内容方是否需要重新设计脚本——例如知道 Neo 在场域内不能完全脱离 AOM 脚本（无法像大厅一样自由发挥），内容方是否会过度依赖脚本写死所有反应？
- **Neo 单一身份下的 4 人格切换**：spec 说大厅 Neo 自动在 MENTOR/STRATEGIST/CHALLENGER/OBSERVER 间切换，学员无感。但**进入场域后 Neo 还是这 4 种人格吗**？或者场域里只有"Tutor 一种人格"？spec 未明确——人格 vs 能力 vs 模式三个词在 spec 中混用。
- **管理端 Ora 的画像数据**：Ora 读取"Neo 全场域对话原文"（management/01-operation §Ora 数据来源）。Leo 取消后所有原文都是 Neo 的——HR 看的是"Neo 在大厅说了什么"vs"Neo 在对练里说了什么"。需要 HR 调研验证 HR 是否能区分这两类对话的价值，还是会混为一谈。

## 6. 给后续调研的输入清单

### 6.1 coaching-skills 项目应该重点看哪些能力？

coaching-skills 项目（CoachPersona）作为 Neo 在大厅的能力实现（01-hall §技术原型）。结合 7 动作 gap，后续调研应重点抽取：

1. **大厅"考研"能力**：coaching-skills 是否有"反向出题验证"机制？如果没有，需要从其他来源补
2. **大厅"练习"能力**：coaching-skills 是否支持迷你 Roleplay？还是只做对话辅导？这决定了"大厅练习"是新增能力还是改造既有能力
3. **5 种对话策略的优先级冲突解决**：coaching-skills 实际跑起来时同时触发多策略如何裁决，spec 没说
4. **8 种信号识别的实战准确率**：①缺概念/②遗忘/③误用/④选错工具/⑥错误心智模型 5 种被甩给授课，但**它们在大厅辅导对话中也会出现**——coaching-skills 检测到后是推回授课还是大厅处理？
5. **跨课程记忆的引用粒度**：coaching-skills 如何定义"当前 vs 相关 vs 远期"？是按时间、按 Course 关联、还是按学员标签？
6. **沟通风格 6 条约束的落地方式**：直接朴实/事实先行/行为优先/不表演倾听/每次聚焦一个点/不编造经历——这些在 coaching-skills 里是 prompt 约束还是后处理过滤？

### 6.2 GROW 素材应该重点抽哪些动作的话术？

GROW 是辅导的经典框架（在 spec discovery-library §2.1 中作为对练剧本范例多次出现）。后续素材抽提应聚焦：

1. **G（Goal）= 大厅"讨论"开场**：GROW 第一步是定义目标，与 Neo 在大厅 OPENING 阶段重叠——抽 G 阶段的提问话术
2. **R（Reality）= 大厅"追问" + 对练"反思"**：探索现状是追问能力的核心——抽 R 阶段的"再说说""具体是什么"等递进追问模板
3. **O（Options）= 大厅"讨论"中段**：探索选项——抽 O 阶段的开放式提问 + 选项罗列话术
4. **W（Will）= 大厅"反思"收尾 + 报告"反思"**：行动承诺——抽 W 阶段的"3 秒 + 我想确认"式可执行行动话术
5. **GROW 与 PIN 法的差异化**：本期内容是基层管理者课包（PIN/POWER/推拉法等），GROW 不是教学内容而是 Neo 的辅导框架。需要明确：Neo 用 GROW 辅导学员学 PIN 法时，"GROW 是 Neo 的脚手架，PIN 是教学内容"——两者层级不同
6. **重点抽"追问"和"反思"的话术**：因为讲授/考研/练习有 AOM 和剧本支撑，**追问和反思是平台层兜底能力**，gap 最大

### 6.3 HR 访谈应该问哪些场域的具体期望？

HR 访谈需覆盖 5 场域 × 7 动作的 35 个格子，但抓重点：

| 场域 | HR 必问问题 |
|------|------------|
| 大厅 | "你期望 Neo 在大厅多频繁主动找学员？" / "学员说他懂了，Neo 要不要验证？" / "学员长时间不来，Neo 是温和还是要给压力？" |
| 授课 | "学员答错 Quiz，Neo 是直接给答案还是逼他想？逼到什么程度？" / "学员插嘴问到课程外的内容，Neo 怎么处理？" |
| 对练 | "学员卡住完全不会，Neo 救还是不救？" / "对练完成的标准是 Neo 判定还是剧本判定，HR 期望谁说了算？" |
| 调研 | "调研 P3 后实际上不一定做——HR 是否接受调研缺失？如果不调研，Neo 凭什么知道学员的真实场景？" |
| 报告 | "学员不认同报告评分，Neo 是辩论还是接受？" / "报告解读后 HR 要不要看 Neo 怎么解读的？" |

**跨场域问题**：

- "你的学员从来都不会在大厅主动找 Neo 聊，怎么办？" — 检验大厅 Neo 主动触发规则的覆盖度
- "学员说一套做一套（汇报积极学习但实际不学），HR 期望 Neo 识别并怎样处理？" — 检验信号⑦元认知偏差
- "Neo 跨场域的同一形象——学员会不会觉得'这个 Neo 在课上很严肃，在大厅却像朋友'有割裂感？" — 验证 Leo 取消的连锁假设
