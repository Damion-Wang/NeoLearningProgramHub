## 2. 轮 3 中 DM 的 3 题反馈（关键）

### 题 1 · 不给答案的轴是什么 ⭐ 核心修正
**DM 反馈**：按场景类型轴（辅导可给 / 教学不给），不是按学员阶段（W 周）

### 题 2 · 学员跨栏目叫 Neo
**DM 反馈**：双栏 GUI 物理隔离，这场景不存在 → 砍掉 Story 5.2

### 题 3 · 主产出形态 ⭐ 重大方向调整
**DM 反馈**：
> 我不需要你给我具体 behaviour，但是最好给我比较全面的 behaviour 设计维度和场景

**意义**：
- 我（PM 角色）方向错了——15 个具体 BDD 是 PM 拍脑袋
- DM 想要 **"维度 + 场景框架"**，让研发后续生产具体故事
- 这是**研发可拓展的资产** vs **PM 拍脑袋的清单**

## 3. 我作为 PM 的反思

### 哪里做对了
- 议题 1 跨场域 Memory 维度（覆盖核心）
- 议题 2 情绪感知（GROW 10 种映射）
- 议题 3 学员卡点 + 翻车（落地轮 2 议题 4）
- 议题 4 Voice 边界（不当舔狗具体场景）
- 议题 5 Agent 协同（清晰边界）

### 哪里没做对（被 DM 修正）
- ❌ 题 1：错把"不给答案"绑学员阶段轴——实际是场景类型轴
  - 反思：我（PM）和挑战者都没识别出这点。一直在 W1/W6/W12 的渐进比例里争论
  - 真正的轴 = 场景类型（辅导/教学/对练/调研/报告）
  - **教训**：当争论都聚焦"比例"时，问问自己是不是错了维度

- ❌ 题 3：写 15 个具体 BDD 故事方向错
  - 反思：我以为 BDD 三件套（write-story/write-behavior/group-stories）就要写具体故事
  - 实际 DM 想要**框架性产出**——给研发生产空间，不是 PM 替研发干活
  - **教训**：BDD skill 是工具，但 PM 输出的颗粒度要 DM 拍板

### 主持人裁定的反思
- 主持人在轮 3 也没识别出"按场景类型 vs 按 W 阶段"的轴错误
- 主持人没拒"15 个具体故事 vs 维度框架"的 PM 方向
- **教训**：主持人不是万能裁定者，DM 互动是最后兜底

## 4. 工作流学到的

- 轮 3 是 3 轮中最容易"陷入细节"的——主题大、容易堆砌具体故事
- DM 反馈"维度 + 场景"是个**抽象层** PM 容易跳过（直接进具体）
- **越是大主题越需要维度先行**

## 5. 14 个示例故事的去留

DM 没说要砍 14 个故事。它们作为 **"维度举例"** 保留在 04 阶段性结论里——研发参考，不是 PRD final。

实际 PRD 时：
- 每维度需要 **30-100 个**具体故事（DM 主导生产 + 研发实施）
- 14 个示例 = 验证维度框架是否能 cover 关键场景
