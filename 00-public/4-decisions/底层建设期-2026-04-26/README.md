## 50 题决策的关键产出（速查）

### Meta 决策
- **Neo 不是单一身份**——按 field 调用相应 skill 和倾向（不是简单的"老师" / "教练" / "治疗师"二选）
- **Soul 格式**：采用 aaronjmars/soul.md 5 文件结构（SOUL.md / STYLE.md / SKILL.md / MEMORY.md / soul.json）
- **不预设时间表**——研发能力决定，不锁周数

### 颠覆性原则
- **Teacher Credibility（Hattie d=0.90）作为 Neo Soul 顶级原则**（超过 V1-V5）
- **Neo 不当舔狗**：不过度奉承，独立观点，用户错也表态
- **遇事不决参照 1v1 真人老师**（设计判断器，不是 Neo Soul）
- **Soul 按场域差异化**——大厅、授课、对练、调研、报告 各场域 Neo 调用不同 skill 和倾向
- **教学法 + 记忆联动**——多维度加权（W 阶段 + skill mastery + 情绪 + 意图）

### 业务边界
- **HR 访谈本期取消**——研发完成后用天使客户试用调优
- **Neo 溯源靠 Agent 能力，不靠 RAG / 向量库**
- **不提供"被遗忘权"**（参照真人老师不能选择性遗忘）
- **不提供学员可定制人格**（参照真人老师人格固定）
- **Ora 可读 Neo 全部记忆**——Neo 对个体学员负责，Ora 对全部学员负责（互补不隔离）
- **0424 决策不重新评估**（含 C-06 红线保持原话直出 HR）

### 内容方向
- **Neo 设计「翻车 + 修复」场景**（增真实感）
- **GROW 10 种情绪识别**（不是 32 种也不是 4 种）
- **本期补中文调研**（樊登 / 得到 / B 站 等）
- **MathDial 反例作 Neo 测试集**（"过早给答案"是 Neo 红线）
- **HBR Coaching Real Leaders 全量爬取转写作学习素材**

### 本期范围
- **本期仅做 Neo + Ora Soul**，Actor / 导演 入 V2
- **研发交付形式**：PRD + 效果验收标准（关键对话场景 / 用户故事）+ GUI testcase + demo 原型

## 协作风格锁定（研发与 PM 沟通约定）

DM（PM）已锁定 4 项协作偏好：

1. **PM 角度 + 不要问技术实现**——技术选型 / 架构 / 验证机制 由研发决定，PM 只管效果与 Agent 表现
2. **遇事不决参照 1v1 真人老师**——设计冲突时回到此问题
3. **Soul 按场域差异化**——不是单一 Soul，是按场域调用差异化 skill / 倾向
4. **不决定时间，研发能力决定**——不预设周数 / 节点
