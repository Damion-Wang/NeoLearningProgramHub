# 管理端报告生成（Report-Gen）

## 模块定位

报告生成页是管理员的团队报告撰写与管理工具。每个管理员有自己的报告库，支持新建报告→Ora 生成初稿→HITL 编辑→下载导出。满足管理员在不同阶段、不同范围、不同目的下的汇报需求。

> 注意区分：本模块处理的是**团队报告**（team report），即管理员面向项目负责人/领导的汇报材料。**个人学习报告**在 learner 端的 report zone 中生成和查看，管理员可在首页 Operation 的人员维度中查看。

---

## 功能区域

| 区域 | 内容 |
|------|------|
| **常显导航** | 首页 / 报告（当前）/ 消息 / 配置 |
| **左栏** | 报告库首页（列表视图）/ 报告编辑器（编辑视图） |
| **右栏：Ora 对话** | 报告辅助生成 + 数据查询 + 内容修改指令 + 证据追溯 |

## Ora 在报告生成中的角色定位

### Ora 是谁

与首页 Operation 中的 Ora 是同一个角色，**共享记忆**——在首页讨论过的数据分析和学员洞察，切到报告页时 Ora 仍然记得，可以直接引用。

### Ora 在报告生成中的核心价值

1. **初稿生成**——根据管理员定义的参数（模板/学员范围/时间/Course），从统一 Database 拉取数据自动生成结构化报告初稿
2. **HITL 协作编辑**——管理员在右栏下达修改指令，Ora 直接修改左栏报告内容，像一个懂数据的写作助手
3. **数据补充**——编辑过程中管理员随时向 Ora 查询数据，Ora 回答后管理员决定是否纳入报告
4. **证据支撑**——报告中的每个结论 Ora 都能追溯到原始对话记录，确保报告有据可依

### Ora 在报告生成中的核心能力

| 能力 | 说明 |
|------|------|
| **初稿生成** | 根据模板和参数生成 markdown + 基础图表的报告初稿 |
| **内容修改** | 根据管理员指令直接修改左栏报告（增删改段落、替换案例、更新数据）。本期直接修改不确认；P3 做 diff 预览确认 + 划词修改 |
| **数据查询** | 编辑中随时回答数据问题，标注数据更新时间 |
| **证据追溯** | 从结论→证据摘要→原始对话，逐层钻取（本期不做脱敏，直接展示原话） |
| **记忆偏好** | 记忆管理员的模板偏好和编辑习惯，越用越精准 |
| **共享上下文** | 与首页 Operation 共享对话记忆，首页发现的洞察可直接带入报告编辑 |

### Ora 在报告生成中的行为边界（能做 vs 不能做）

| 能做 | 不能做 |
|------|--------|
| 根据参数和模板生成报告初稿 | 不经管理员确认自行发布或导出报告 |
| 根据指令修改报告内容（增删改） | 擅自修改管理员手动编辑的内容 |
| 引用具体学员的对话记录和行为数据作为证据 | 编造不存在的数据或虚构案例 |
| 给出数据趋势分析和可能原因 | 给出超出数据支撑的过度推断 |
| 记忆管理员偏好，下次生成更精准 | 将一个管理员的报告内容泄露给另一个管理员 |
| 提示报告中可能存在的数据不一致 | 替管理员做判断（"这个学员表现差，建议重点关注"） |

每个管理员账号有独立的报告库（按账号隔离），用于管理所有已创建的报告。

## 内容模板

新建报告时可选择内容模板，模板定义报告的默认结构和侧重点：

| 模板 | 适用场景 | 默认结构 |
|------|---------|---------|
| **中期报告** | 项目进行中的阶段性汇报 | 整体进度 + 活跃度趋势 + 重点学员表现 + 风险提示 |
| **结项报告** | 项目结束后的总结汇报 | 项目概览 + 学员成长总结 + 行为改变证据 + 成果亮点 + 改进建议 |
| **自定义** | 管理员自由定义 | 空白结构，由 Ora 根据参数生成 |

> 模板只定义内容结构，不做视觉风格模板。报告统一为 markdown + 图表。

## 新建报告流程

### 步骤一：填写基本信息

点击「新建报告」后弹出表单：

| 字段 | 说明 |
|------|------|
| **内容模板** | 中期报告 / 结项报告 / 自定义（模板决定报告的默认结构和侧重方向） |
| **学员范围** | 两种方式：①勾选具体学员 ②按筛选条件（如"完成了 Course 2 的学员""全部学员"） |
| **时间范围** | 报告覆盖的时间段（如"本月""本周""2026-04-01 ~ 2026-04-30"） |
| **Course 范围** | 报告聚焦的课程范围（如"Course 1-3""全部 Course"） |
| **补充说明（可选）** | 自由文本，管理员可补充额外要求（如"重点提及张磊和陈思""不要写具体姓名"） |

### 步骤二：Ora 生成初稿

管理员确认基本信息后，Ora 基于参数从统一 Database 拉取数据，生成报告初稿（markdown + 基础图表）。生成完毕后左栏展示初稿。

### 步骤三：HITL 编辑

左栏进入报告编辑器，右栏 Ora 对话。管理员可以通过三种方式编辑报告：

| 方式 | 说明 |
|------|------|
| **直接编辑** | 在左栏编辑器中直接修改文字、调整结构、增删章节 |
| **Ora 查询** | 在右栏问 Ora "帮我查一下张磊的对练表现"，Ora 回答后管理员自行决定是否纳入报告 |
| **Ora 修改** | 在右栏对 Ora 说"把第三段改成..."或"在高光部分加上陈思的案例"，Ora 直接修改左栏报告内容（本期直接修改不确认；P3 做 diff 预览确认 + 划词修改） |

编辑过程中自动保存，管理员可随时退出，下次进入从上次位置继续。

### 步骤四：下载导出

管理员预览确认后，点击「导出」选择格式（PDF / Word）下载。

## 用户故事：管理员王经理准备月度汇报

### 场景背景
王经理月底需要向项目负责人汇报培训进展。她刚在 Operation 首页审查了本月的学习数据。

### 完整旅程

**10:00 — 进入报告页**

王经理左侧点击「报告」，左栏展示报告库首页。已有 2 份之前的报告，她点击「新建报告」。

**10:02 — 填写基本信息**

弹出表单，王经理填写：
- 内容模板：中期报告
- 学员范围：全部学员（32人）
- 时间范围：2026-05-01 ~ 2026-05-31
- Course 范围：Course 1-3
- 补充说明：重点提及张磊和陈思的优秀表现

点击「生成报告」。

**10:03 — Ora 生成初稿**

Ora 拉取数据后生成初稿，左栏展示报告内容（markdown+基础图表）：
- 第一页：项目概览（整体完成率、活跃学员数、本月亮点数据）
- 第二页：学习进度详情（各 Course 完成率、趋势图）
- 第三页：学员典型案例（张磊的对练进步、陈思的调研表现）
- 第四页：需关注事项（3 名低活跃学员）
- 第五页：下月展望

**10:10 — HITL 编辑**

王经理在左栏直接修改：第四页不要写具体姓名，改为"有 3 名学员需要重点关注"。

在右栏问 Ora："张磊在对练中具体进步了多少？"
Ora 回答具体数据，王经理决定把这段加到第三页。

在右栏对 Ora 说："帮我在第三页张磊部分加上这段数据，用'对练评分从 65 分提升到 88 分'这个表述。"
Ora 直接在左栏第三页插入内容。

王经理继续补充一段自己的分析："本月重点推进了横向协作模块的对练环节，学员参与度较上月提升显著..."

**10:20 — 下载**

王经理预览确认后，点击「导出」选择 PDF 下载。

**下午 — 项目负责人追问**

负责人看了报告后追问张磊的证据。王经理重新进入报告页，在右栏问 Ora，Ora 支持钻取到原始对话记录。

## 用户故事二：编辑已有报告

### 场景背景
上个月王经理写了一份月度汇报报告，这个月需要在此基础上更新数据。

### 完整旅程

王经理进入报告库，看到上月的报告"5月月度汇报"。点击进入编辑器。

左栏展示上次的完整报告内容，右栏 Ora 对话区。

王经理在右栏说："帮我把报告中的数据更新到 6 月最新的，结构不变。"

Ora 识别报告中的数据引用，逐段更新为 6 月数据，左栏实时变化。

王经理检查后发现第三页的"学员典型案例"需要换人。在右栏说："第三页换成周明的案例，他这个月进步最大。"

Ora 替换案例内容。王经理补充一段自己的分析，导出新的 PDF。

## 用户故事三：结项报告

### 场景背景
项目结束，小李需要准备一份完整的结项报告。

### 完整旅程

小李新建报告，选择「结项报告」模板：
- 学员范围：全部学员
- 时间范围：项目全程

Ora 生成结项报告初稿，结构包含：项目概览 → 学员成长总结 → 行为改变证据 → 成果亮点 → 改进建议。

Ora 回答：

小李把这段加入报告，补充运营视角的总结，导出 Word 发给 HR。

## 已确认决策

- 报告页定位为报告撰写与管理工具（不只是查看）
- 每个管理员有独立报告库（按账号隔离）
- 新建报告通过表单定义参数（内容模板/学员范围/时间/Course/补充说明）
- Ora 生成初稿后，管理员可通过三种方式编辑（直接编辑/Ora查询/Ora修改）
- Ora HITL 编辑本期直接修改；P3 做 diff 预览确认 + 划词修改
- 只做团队报告（team report），个人报告在 learner 端
- 内容模板：中期报告/结项报告/自定义（不做视觉风格模板）
- 报告内容为 markdown + 基础图表（柱状图/折线图/进度条/表格）
- 导出格式：PDF / Word
- Ora 与首页共享记忆
- 个人学习报告查看在首页 Operation 的「个人报告汇总」Tab
- 自动保存，随时可退出继续编辑
