## 1 · 跨 spec 设计决议（影响多个 spec 文件）

### 1.1 默认密码机制（平台级）

睿学（NeoLearning）平台**后台统一管理**一个**全平台默认密码**（所有培训项目共用同一个 / 不按项目区分）。

**初始登录流程**：
1. 项目开营时 admin 添加人员（学员 / HR / 项目运营 等）
2. 所有新增人员的**初始密码 = 平台默认密码**（由后台静默写入）
3. admin / 管理员可在 program config 内**查看**默认密码（用于复制发给学员帮助首次登录）
4. 学员首次登录后**可自行修改密码**

**忘记密码流程**：
1. 学员联系管理员（HR / 项目运营）
2. 管理员在 program config 内**选择该学员** → 点击"**重置密码**"
3. 学员密码被重置为**平台默认密码**
4. 管理员把默认密码发给学员
5. 学员重新登录后**可再次修改密码**

**关键约束**：

| 维度 | 规则 |
|------|------|
| 默认密码归属 | **平台级**（睿学 NeoLearning 后台统一管理）/ 不是项目级 |
| 默认密码可修改性 | 在 program config 里**不可修改** —— 产品 UI **仅做"显示"** / 不暴露修改入口 |
| 管理员能力 | 查看默认密码 + 重置任意学员密码（重置目标 = 默认密码） |
| 学员能力 | 首次登录后自行修改 / 再次重置（通过管理员）后可再修改 |

**影响 spec 章节**：
- [06-management/04-program-config § 1.2.2]：加"默认密码显示 + 重置密码 SOP"小节 + **删除"更改"动作**（与本节"不可修改"冲突）
- [04-global/02-platform-global]：平台默认密码字段位置 / 后台管控声明
- [03-lifecycle/01-pre-learning § 1.5]：开营时 admin 添加人员的初始密码下发流程

**demo 偏差**：proconfig 板块 3 当前写「**默认密码（项目级）**」→ 应改为「**默认密码（平台级）**」或不带"项目级"修饰。

### 1.2 TTS 口播范围

**TTS 口播仅在 3 个学员端学习场域内提供**。其他位置（大厅 + 管理端）**无 TTS 口播**。

| 位置 | TTS 口播 | 说明 |
|------|:-------:|------|
| **学员端大厅 Neo（hub）** | ❌ 无 | 大厅 Coach 角色不口播 / **右上角 Neo 头像取消**（无口播形态没必要）|
| **学员端授课 lecture（Neo）** | ✅ 有 | 场域内 Neo Tutor 口播讲授 / 解惑 / 追问 |
| **学员端对练 practice（Neo）** | ✅ 有 | 场域内 Neo 口播复盘 / 反馈 |
| **学员端小结 recap（Neo）** | ✅ 有 | 场域内 Neo 口播 Walk-through / 反思引导 |
| **管理端 Ora** | ❌ 无 | Ora 全场无口播 / 仅文本交互 |

**新增规则（补 cross-spec）**：**AI 语速设置（朗读速度滑块）仅在有 TTS 的页面**（lecture / practice / recap）/ 其他 5 页（hall + 4 管理端）**应移除** AI 语速设置。

**影响 spec 章节**：
- [05-learner/02-hub]：删除大厅右上角 Neo 头像位置 + 加"大厅 Neo 无 TTS 口播 / Coach 角色仅文本交互"
- [05-learner/03-lecture] / [05-learner/04-practice] / [05-learner/05-recap]：加"场域内 Neo 提供 TTS 口播 + AI 语速设置位置"约定
- [06-management/01-overview]：加"Ora 全场无 TTS 口播 / 仅文本交互" + "管理端不放 AI 语速设置"

**demo 偏差**（4 项需修）：
1. **practice 未实装真 TTS** / 应实装
2. **recap 未实装真 TTS** / 应实装
3. **大厅 Neo 大头像未取消**（hall.html `.neo-hero`）/ 应取消
4. **AI 语速设置在每页都有**（学员 4 页 + 管理 4 页 = 8 页）/ 应仅在 lecture / practice / recap 三页

### 1.3 Neo 头像 GIF（本次实现 / 替换呼吸占位）

**决策**：lecture / practice / recap 三场域内 **Neo 头像本次实现 GIF 表示讲话动作**（不再是"未来设计"）。

| 范围 | GIF 动作 | 动作控制权 |
|------|:-------:|----------|
| 学员端 lecture Neo | ✅ 本次实现 | 由 lecture 场域内 Neo 自身管理 |
| 学员端 practice Neo | ✅ 本次实现 | 由 practice 场域内 Neo 自身管理 |
| 学员端 recap Neo | ✅ 本次实现 | 由 recap 场域内 Neo 自身管理 |
| 学员端大厅 Neo | ❌ 不在范围 | 头像位置已取消（详见 § 1.2）|
| 管理端 Ora | ❌ 不在范围 | — |

**关键原则**：
- **场域差异化**——不同场域的 Neo GIF 动作不同（与 CLAUDE.md 核心约束 #11 "Soul 按场域差异化"一致）
- **场域 Neo 自管理**——动作选择 / 切换 / 时机由该场域内的 Neo 决定 / **不集中由全局 Neo 控制**
- **GIF 表达"Neo 在讲话"动作** / 替换 demo 当前的呼吸动画占位

**demo 现状**：
- demo 当前用 **PNG 静态头像 + CSS `neoBreathe` 呼吸动画**（说话时 scale 1→1.025 + box-shadow 脉冲）作为占位
- 实际产品需替换为**真讲话 GIF**（lecture/practice/recap 各场域独立动作库）

**影响 spec 章节**：
- [05-learner/03-lecture] / [05-learner/04-practice] / [05-learner/05-recap]：加"Neo 头像 GIF（讲话动作）+ 场域 Neo 自管理"约定
