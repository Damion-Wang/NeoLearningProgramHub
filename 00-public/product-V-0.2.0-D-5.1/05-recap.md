## 1 recap（小结）

> **总览**：recap 是学员在 Course 学习中的**阶段性反思场域**——通过查看 Neo 主动产出的反思报告 + 与 Neo 对话挖掘洞察，把"做过什么"沉淀为"看见什么 + 接下来怎么用"。学员在前序 Activity 全部完成后解锁入口，**首次小结**事件触发后报告永久锁定；之后再进入是自由复习态。
>
> **结构（6 节）**：业务定位 + Activity 生命周期 + 报告全景 + 界面布局 + 状态详述 + 全局通用。

### 1.1 业务定位

#### 1.1.1 一句话定位 + 与其他场域的差异

> recap（小结）是学员在 Course 学习中的**阶段性反思场域** —— 通过查看 Neo 给出的反思报告 + 与 Neo 对话挖掘洞察，把"做过什么"沉淀为"看见什么 + 接下来怎么用"。

**与其他场域的两个差异维度**：

| 差异维度 | 与谁对比 | recap 的差异 |
|---------|---------|--------------|
| **跨 Activity 阶段性整合** | lecture 单次内容巩固 / practice 单次演练复盘 | recap 跨多个 Activity 做阶段性整合反思 |
| **Neo 主动产出** | 大厅辅导按需应用 | Neo 主动 Walk-through + 主动产出应用计划 |

#### 1.1.2 5 重价值（与 5 章报告一一对应）

报告 5 章承载 5 重价值：

| # | 价值 | 学员视角 | 项目视角 | 对应章 |
|---|------|---------|---------|------|
| 1 | **综合定调** | 一眼看到 Neo 对自己整体表现的判断 + 跨课程记忆呼应 | 让"老师记得你"成为可举证体感 | 第 1 章 综合评估 |
| 2 | **知识层面回顾** | 把 Course 内学过的关键概念 / 工具重新整合 | 让"学过"沉淀为"内化" | 第 2 章 这一段我学了什么 |
| 3 | **能力维度提升** | 看到 Neo 对自己内容掌握 + 能力变化的整体判断（自己看不到的视角）| 沉淀可举证的能力成长轨迹 | 第 3 章 这一段我学得怎么样 |
| 4 | **上课状态评估** | 看到 Neo 对自己学习过程（投入 / 模式 / 节奏 / 状态）的解读 | 沉淀过程画像（Ora 可调用）| 第 4 章 这一段我表现怎么样 |
| 5 | **行动闭环** | 从反思转向具体下一步行动（应用计划）| 让学习闭环到工作场景 | 第 5 章 我接下来该做什么 |

### 1.2 recap Activity 生命周期

> 本节集中讲清 recap Activity **从锁定到首次小结到自由复习的全生命周期** —— 包含状态总览 / 入口与解锁 / 首次小结判定 / 报告更新 / 切换信号 / 术语对应表 6 块。

#### 1.2.1 3 Activity 状态总览

| 状态 | 含义 | 报告呈现 | Neo 行为态 |
|------|------|---------|----------|
| **① 锁定态** | 关联 Activity 未全部完成 | 入口锁定 / 不展示报告 | — |
| **② 首次小结** | 关联 Activity 全部完成 + 学员首次进入（一次性事件 / 触发后转入自由复习态）| 首次小结报告 5 章完整生成 | **首聊态**：完整 Walk-through + 主动开场 + 应用计划产出 |
| **③ 自由复习态** | 首次小结事件触发后的所有再进入 | 首次小结报告（永久不变）+ 自由复习层（覆盖式更新）| **回访态**（距上次有变化）/ **复习态**（无变化）|

#### 1.2.2 进入路径

| 进入路径 | 含义 |
|---------|------|
| **从大厅内容目录主动进入** | 学员通过大厅内容目录点入 recap |
| **从其他 Activity 通过目录抽屉跳转** | 学员在 lecture / practice / 其他 recap 内通过左侧抽屉 Tab2 整个课程目录跳转进入 recap |
| **从上一个 Activity 自然跳转进入** | 学员完成 recap 前一个 Activity 后，通过"完成 Modal · 继续下一 Activity"按钮自然过渡进入 |

进入路径不影响**首次小结**判定（不论从哪条路径进入，只要满足"前序全完成 + 首次"就触发）。

#### 1.2.3 首次小结四件齐全

**首次小结**触发需四件齐全：

1. **完整态首次进入**（前提条件 / 关联 Activity 全部完成 + 学员首次进入）
2. **完整 Walk-through 走完**
3. **实质性反思**
4. **应用计划产出 + 任务卡片推送**

详细机制 / 触发瞬间发生什么 见 § 1.5.2 首次小结主场。

#### 1.2.4 报告增量更新机制

- 报告以"关联 Activity 完成"事件为单位**后台增量重新生成**；学员进入时直接看最新缓存
- 首次小结事件触发瞬间锁定**首次小结报告**，永久保留（不变）
- 首次小结事件后，新数据通过**自由复习层**覆盖式更新（详见 § 1.3.7）
- 学员侧零等待，后台静默完成；不提供"刷新"按钮（自动更新即可）

#### 1.2.5 行为态切换的数据信号

Neo 综合多个信号自主判断当前应该用哪种态（spec 不规定固定阈值）：

**关联 Activity 内信号**（限定 KGP 配的关联范围）：

- 距上次会话是否有新关联 Activity 完成
- 关联 Activity 内是否有新高光卡片

**全场景信号**（不限定来源）：

- 6 维画像是否有明显变化（含大厅 / 跨 Course 来源）
- 学员在大厅或其他场景是否有新对话
- 学员上次在 recap 内跟 Neo 聊到哪
- 上次 Walk-through 走到哪一章

### 1.3 小结报告 · 全景

> **核心交付物定位**：Course 个人报告是 recap 的核心交付物。**作用**：报告是 Neo 引导对话的"道具"——写到 70%，留 30% 给 Neo Chat 展开。
>
> **核心受众定位**：报告以**学员自己看为主，管理员附带**。学员是第一受众；管理端通过 team report 拉数据是次要场景。

#### 1.3.1 报告 6 章结构总览

报告共 **6 章** —— 前 5 章在首次小结事件触发瞬间锁定，永久保留；第 6 章自由复习是覆盖式更新区块。

**6 章结构**（前 5 章与 § 1.1.2 5 重价值一一对应，第 6 章为动态更新区）：

| # | 章 | 承载价值 | 形态 | 数据来源 | 可变性 |
|---|----|----|------|---------|------|
| **第 1 章** | **综合评估** | 综合定调 | 1-2 段 markdown / 含跨课程引用 | Neo 综合所有关联 Activity 数据 + 跨课程记忆 | 锁定后永久不变 |
| **第 2 章** | **这一段我学了什么** | 知识层面回顾 | markdown / 知识列表 + 跨知识点关联文字段 | Course AOM + 关联 Activity + 学员场景 + Neo 实时推导跨 Activity 知识网络 | 锁定后永久不变 |
| **第 3 章** | **这一段我学得怎么样** | 能力维度提升（**成果**）| markdown 文字叙事 + 关键对话片段引用块 | 内容掌握 + 能力变化的 Neo 论断 + 高光库子集 + 卡点 + 起点 vs 终点能力变化对比（认知 / 能力 / 行为 三维度）| 锁定后永久不变 |
| **第 4 章** | **这一段我表现怎么样** | 上课状态评估（**过程**）| markdown 文字叙事 + 6 维画像图 + 数据卡 | 全场域日志（投入/节奏/频率/完成）+ 6 维画像 + 学习模式画像 + 状态变化（情绪/投入度）| 锁定后永久不变 |
| **第 5 章** | **我接下来该做什么** | 行动闭环 | markdown 1-3 条便签 + Neo 任务卡片同步推送 | Neo 基于报告各章 + 跨场域记忆 + 学员场景 | 锁定后永久不变 |
| **第 6 章** | **自由复习** | 覆盖式更新区块 | markdown 综合内容 / 不分块 | 自由复习态对话 + 关联 Activity 重练新画像变化 + 新高光卡点 | **每次更新覆盖整章** |

**第 3 章 vs 第 4 章边界**：

- **第 3 章 学得 = 内容和能力维度的提升**（"学完后能力变化 / 知识掌握程度 / 高光卡点作为证据"）
- **第 4 章 表现 = 上课状态评估**（"上课时投入 / 节奏 / 模式 / 情绪起伏 / 6 维画像 / 数据卡"）

**报告全长**：前 5 章 1200-1800 字 / 学员阅读时长 4-6 分钟 / 完整解读体验（含 Neo Walk-through 对话）10-15 分钟；第 6 章自由复习视当下情况覆盖式更新。

#### 1.3.2 第 1 章 综合评估

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 用客观语气对学员在本 Course 的整体表现做的 **1-2 段总评**（150-250 字）+ 跨课程记忆引用 |
| **目的** | 给整份报告**定调** —— 让学员第一眼看到 Neo 的综合判断 + 感受到"老师记得我之前学过什么" |
| **形态** | 1-2 段纯 markdown / 不含图 |
| **数据来源** | Neo 综合本 Course 所有关联 Activity 数据（互动行为 + 高光卡点 + 6 维变化）+ 跨课程记忆 |
| **可变性** | 首次小结事件前每次刷新；事件触发时锁定；之后**永久不变** |

**示例骨架**（占位符）：

> **数据显示**，你在本 Course《{Course 主题}》上**{整体表现一句话}**——{N} 次 Activity 中累计互动 {M} 次，高光时刻集中在 {场景}。**自《{先修 Course 1}》以来**，你在 {能力维度} 上 {延续/突破描述}；本 Course 中你新打开了 {新能力点}，**这是与你在《{先修 Course 2}》中的 {对应表现} 相比的明显升级**。

#### 1.3.3 第 2 章 这一段我学了什么

| 维度 | 说明 |
|------|------|
| **是什么** | 跨 Activity 知识回顾 —— 学到了哪些知识 / 知识点之间的关联关系 / 还有疑问的 |
| **目的** | 让学员把 Course 内的知识点串起来——核心是**知识层面回顾** + **跨知识点关联** |
| **形态** | markdown / 含知识列表 + 关联关系文字段 / 不含图 |
| **数据来源** | Course AOM 元数据 + 关联 Activity 列表 + 学员场景描述 + Neo 实时推导跨 Activity 知识网络 |
| **可变性** | 首次小结事件前每次刷新；锁定后**永久不变** |

**3 个子部分**：

**A. 学了哪些知识**（核心 / 关键概念 + 工具 + 框架）

- 列出本 Course 学到的关键概念 / 工具 / 框架（"什么"）
- 每个配 1 句作用描述
- 数量不限 / 由 Course 内容决定

**B. 跨知识点的关联关系**（核心 / Neo 实时推导）

- 用一段文字讲清楚**知识点之间的关系**
- 关联关系**不限于一种** —— 可以是：
  - **应用场景关联**（if-then 推荐用的逻辑：在 X 场景用 A 工具 / 在 Y 场景用 B 工具）
  - **递进关联**（A 是 B 的前置 / 学完 A 再学 B 才更深）
  - **互补关联**（A 解决 X 问题 / B 解决 Y 问题 / 二者相补）
  - **对比关联**（A 与 B 在 Z 维度上的差异）
  - 其他 Neo 实时推导出的关系
- **不画流线图 / 不画树状图** —— 用文字段落表达
- **不展示推理链**（只说结论，不写"我之所以这么判断…"）
- **不延伸到未学但相关**的内容（严格贴现阶段已学）

**示例骨架**（应用场景关联 / 占位符）：

> 这门课里你学到的几个工具，**用法是分场景的**——**{场景 1}** 时用 **{工具 A}**（{一句作用}），{场景 2} 时用 **{工具 B}**（{一句作用}），**{场景 3}** 里用 **{工具 C}**（{一句作用}）。

**C. 还有疑问的**

- Neo 综合识别学员在关联 Activity 中**反复绕回**的概念 / 卡点（仍未充分理解）
- 列 1-2 条 / 每条 1 句话表述
- **诚实回看**："观察到你在 {概念 X} 上还有点不清楚 / 在自由复习态可以找我聊聊"
- 作为留白追问的钩子，让学员在自由复习态可主动深入

**状态自适应（含低数据兜底）**：

- 正常态：A + B + C 完整 3 子部分
- **低数据兜底**：A 简短列概念 + B 关联关系一句话；"还有疑问的" 数据不足时可不写

#### 1.3.4 第 3 章 这一段我学得怎么样

> **本章定位**：能力维度提升（学完后的能力变化）—— 与第 4 章"上课状态评估"区分。

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 对学员**学完后的内容和能力维度提升**的整体论断 + 具体维度的认知转化（学习前 vs 学习后）+ 高光卡点作为证据 |
| **目的** | 让学员看到**自己看不到的视角** —— Neo 对自己学得怎么样的核心判断 |
| **形态** | markdown 文字叙事 + 关键对话片段引用块 / 不含图 |
| **数据来源** | 高光库子集 + practice 复盘待改进点 + Neo 跨场域感知卡点模式 + Activity 集合起点 vs 终点能力对比 + Neo 对内容掌握程度的综合判断 |
| **可变性** | 首次小结事件前每次刷新；锁定后**永久不变** |

**4 个子部分**：

**A. 认知转化**（核心 / 按维度展开 / 学习前 vs 学习后对比）

> 这是第 3 章的**核心** —— Neo 对学员"学得怎么样"的具体维度对比。

- 形态：3 维度小节，每个小节 = 学习前 vs 学习后的对比表 + 1-2 句 Neo 洞察 + 对话片段举证（如有）
- 3 维度：
  - **维度 1 · 认知**（学员对内容的理解 / 思考方式的转变）
  - **维度 2 · 能力**（学员能做什么 / 工具掌握程度）
  - **维度 3 · 行为**（学员实际做了什么 / 动作上的转变）
- **不量化数字** —— 文字叙事为主 + 关键对话片段举证为辅
- **关键基线**：对比的是**关联 Activity 集合的起点 vs 终点** —— 不是开营基线、不是上次报告

**B. 高光行为**（1-2 条 / A 认知转化的证据）

- 形态：markdown 引用块（每条 = 标题 + 引用对话片段 + Neo 客观一句话）
- 限定：关联 Activity 内的高光库子集（高光卡片是 Neo 在各场域的通用能力，详见 [05-learner/02-hub § 1.1]）
- 数量不足时**宁可空着也不补关联外**
- 作为 A 认知转化的**佐证**（不是独立内容）

**C. 卡点行为**（1-2 条 / A 认知转化的证据）

- 形态：markdown 引用块
- 限定：关联 practice 复盘的待改进点 + Neo 跨场域感知到的反复出现的卡点模式
- **不依赖 Quiz 错题**（Quiz 不一定有对错，详见 [05-learner/03-lecture § 1.3.4]）
- 作为 A 认知转化的**佐证**

**D. 学习成长总评**（Neo 高级论断 / 可选）

- 形态：1-2 段 markdown
- 内容：
  - **特质画像**（这一段学习中 Neo 看到的学员 2-3 个特质 / 如"主动学习 / 工具内化能力 / 框架化思维"）
  - **关键转变**（这一段学习中 Neo 看到的 2-3 个明显转变 / 如"单点思考 → 系统思考"）
- **三态写法**（适用于 A 维度对比 + D 总评）：
  - **进步** → 直说能力提升 + 举证
  - **持平**（无明显变化）→ 写"**稳定保持**"+ 稳定点取在哪里
  - **退步** → 软化为"**近期波动**"+ 具体证据（不直接说"退步了"）

**状态自适应（含低数据兜底）**：

- 正常态：A + B + C + D 完整 4 子部分
- **低数据兜底**：A 认知转化按可观察到的维度缩短（不强行 3 维度）；B/C 数量不足时宁可空着；D 数据不足时不强写

#### 1.3.5 第 4 章 这一段我表现怎么样

> **本章定位**：上课状态评估（学习过程行为）—— 与第 3 章"能力维度提升"区分。

| 维度 | 说明 |
|------|------|
| **是什么** | 学员在本 Course 关联 Activity 内的**上课状态** —— 数据卡 + 6 维画像 + 学习模式画像 + 状态变化 |
| **目的** | 让学员看到自己在学习这件事上的**过程画像** —— 数据指标 + 6 维评估 + Neo 对学习风格的解读 |
| **形态** | markdown 文字叙事 + 数据卡 + 6 维画像图 |
| **数据来源** | 全场域日志（学习时长 / 互动次数 / 实战演练 / 主动探索 / 应用反思）+ 6 维画像变化数据 + 6 维当前值 + Neo 综合学习模式 + 状态变化（情绪 / 投入度）感知 |
| **可变性** | 首次小结事件前每次刷新；锁定后**永久不变** |

**4 个子部分**：

**A. 学习数据卡**（学习投入的核心数据）

- 形态：4 数据卡 / 表格呈现
- 4 维度（参考 mock 1）：
  - **互动次数**（含 Quiz / 答疑 / 演练对话）
  - **实战演练**（关联 Activity 完成情况 / 重练次数）
  - **主动探索**（学员主动延伸提问数）
  - **应用反思**（关联到自己真实工作场景的次数）
- **不画柱图**（数据卡形式呈现即可）
- **不区分类型计 token 消耗**（跨类型 Activity token 差异巨大）

**B. 6 维画像**（雷达图 + Neo 解读）

- 形态：雷达图 + markdown 1 段解读（80-150 字）
- 数据来源：6 维感知体系 / 学员在所有场域累积行为
- **始终显示**（6 维数据不会缺失 / 见下方"状态自适应"）
- **变化呈现规则**：每个变化点嵌入触发证据（具体到 Activity / 对话片段）
- Neo 解读：用人话讲清楚每个维度分数代表什么 + 为什么

**C. 学习模式画像**（Neo 用人话解读上课风格）

- 形态：1 段 markdown（100-150 字）
- 内容：学员在本 Course 中的学习风格特征（如"边练边问型 / 先理论后实战型 / 反复打磨型"）+ 学习节奏（深入 vs 浅尝 / 系统 vs 跳跃）
- 用人话描述："你属于那种 {X} 型学员 —— 表现是 {Y} / 优势在 {Z}"
- **不预设"哪种好哪种差"**（这是行为事实陈述，让学员自己反思）

**D. 状态变化**（情绪 / 投入度 / 心态起伏 / 可选）

- 形态：1 段 markdown（80-120 字 / 可选）
- 内容：学员在关联 Activity 内的**情绪 / 投入度 / 心态**起伏
- **必须有具体证据**（哪个对话 / 哪个时间点）
- **如无明显波动则不写本子部分**（避免硬编）

**状态自适应（含低数据兜底）**：

- 正常态：A+B+C 必有 / D 可选
- **6 维画像始终显示**：6 维数据来源已锁定（基于学员在所有场域累积行为），不会缺失。无变化时显示"当前快照"形态；有变化时显示"变化点 + 当前值"形态
- **低数据兜底**：A 数据卡仅显示有数据的维度；C 学习模式简短；D 数据不足时不强写

#### 1.3.6 第 5 章 我接下来该做什么

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 产出的 **1-3 条便签式行动建议**（应用计划）—— 报告内文字便签 + Neo 任务卡片同步推送学员选择 |
| **目的** | 把反思转化为可执行的小任务，闭环到学员真实工作场景。**Neo 是给建议，不是命令** |
| **形态** | markdown 列表（每条 = 1 句简洁可执行的小任务 + 时间表达）|
| **生成时机** | 首次小结走完前 4 章后生成一次；后续不再变 |

**5 个核心规范**：

**A. 便签化形态**

- 每条 = 一句话一个动作
- 形态像便签贴提醒，不是作业本
- **不带"问题→建议→时间→推卡片"四件套**

**B. 时间表达 = 近期 / 中期 / 长期 三档**

- 不用具体时间（24h / 本周 / 月内）
- 改为 **近期 / 中期 / 长期** 三档表达
- **语气是建议不是命令**（不用"立即"、"必须"等强硬话术）

**C. 不夹带学习内容推荐**

- 行动建议聚焦"行动"本身，不夹带"推荐看 X 课"等推卡片

**D. 数量限制**

- **强制最多 3 条**，Neo 内部排序取优
- 不展示折叠后的"还有 N 条"

**E. 与第 2 章不重复**

- 第 5 章建议**只写动作**（"在 {工作场景} 主动 {做 X}"）
- **不在括号里重述"在 X 场景用 Y 法"** —— 那是第 2 章的工作

**呈现示例**（占位符）：

```markdown
### 第 5 章 我接下来该做什么

基于这一段你学了什么 + 学得怎么样 + 表现，**给你几个建议**可以试一下：

1. **【近期】** 下一次 {工作场景 1}，**主动 {具体动作 1}**
2. **【中期】** 在 {工作场景 2} 里，**{具体动作 2}**
3. **【长期】** 选一个 {工作场景 3}，**完整 {具体动作 3}**

→ Neo 已经把这几件事做成任务卡片推到你面前 / 你可以选接收哪几条加到大厅待办
```

**应用计划全流程图**：

```
首次小结 Walk-through 走完前 4 章
  ↓
Neo 主动收尾 + 报告内列出 1-3 条文字便签（永久保留）
  ↓
Neo 通过任务卡片推送（场域对话流 / 沿用 [05-learner/06-cross-context § 4.4]）
  ↓
学员对每条任务卡片选择：接受 / 不接受
  ↓
首次小结事件触发（不依赖学员选择结果 / Neo 产出+推送即触发）
  ↓
接受的 → 加入大厅待办清单 → 学员主动点掉时 Neo 跟进问情况
不接受的 → 仅留在 recap 报告里（永久可见 / 不进大厅）
```

**状态自适应（含低数据兜底）**：

- 正常态：1-3 条便签 + 任务卡片推送
- **低数据兜底**：1 条核心 + 任务卡片可选推送

#### 1.3.7 第 6 章 自由复习

> **作用**：首次小结事件触发后，自由复习态产生的所有新内容统一收录在本章——与前 5 章并列、并位于其后，**每次更新覆盖整章**（区别于前 5 章的永久锁定）。

**位置**：在报告末尾（前 5 章之后），用 markdown `## 第 6 章 自由复习` 作分界。

**内容范围**：

| 内容类型 | 说明 |
|---------|------|
| Neo 在自由复习态对话中产生的**当下观察** | 学员重进 recap 跟 Neo 聊天时，Neo 基于截至当下所有自由复习态对话产生的综合观察 |
| **截至当下的画像变化点** | 首次小结事件后 Neo 在 lecture / practice 内新写入的画像变化 |
| **截至当下的新高光 / 卡点** | 首次小结事件后产生的新关联 Activity 内高光 + practice 重练带来的新卡点 |
| **新关联 Activity 总结**（如有）| KGP 后期追加新关联 Activity 时，Neo 对该新 Activity 内学员表现的综合回顾 |

**更新机制（★）**：

- **每次更新覆盖整章**：第 6 章不是按时间叠加，而是**每次更新触发时基于截至当下的所有数据重新生成一份当下最新综合，整体覆盖上次内容**
- **理由**：自由复习态的更新触发点不统一，按时间叠加会让内容割裂；用"覆盖式整体重生成"保证学员每次看到的是**当下最新一份完整综合**
- **不保留历史版本**：学员看不到"上一次的第 6 章"，也无法对比
- **前 5 章永远不变**：第 6 章的覆盖式更新不影响前 5 章

**学员侧呈现**：

```markdown
## 第 1 章 综合评估
...

## 第 2 章 这一段我学了什么
...

## 第 3 章 这一段我学得怎么样
...

## 第 4 章 这一段我表现怎么样
...

## 第 5 章 我接下来该做什么
...

## 第 6 章 自由复习（截至当下最新）

- Neo 综合观察：...
- 截至当下的画像变化点：...
- 截至当下的新高光 / 卡点：...
```

**Neo Chat 历史**：在 Activity 内不被清空——学员可翻 Chat 历史回看跟 Neo 聊过的具体对话内容（不在自由复习层重复呈现）。

**笔记复用**：学员"想留住某段感动"通过笔记功能记录（笔记悬浮球，沿用 [05-learner/02-hub § 1.3]）。

#### 1.3.8 报告产物属性（生成 / 修改 / 导出 / KGP 边界 / 受众）

| 维度 | 说明 |
|------|------|
| **触发（首次小结事件前）** | 关联 Activity 完成事件触发后台增量重新生成；新版本覆盖旧版本（不留旧快照）|

| **知识网络轻处理** | Neo **实时推导**跨 Activity 知识关联 / **不依赖 KGP 概念关系图** / **AOM 不加新字段** |
| **学员第一受众** | 报告以**学员自己看为主，管理员附带** —— 学员是第一受众；管理端拉数据是次要场景 |
| **管理端可见性** | **完成态（首次小结事件后）**的 recap 报告 → 管理端**可露出**（首页 [06-management/02-home § 1.2.2] (4) 个体详情面板 / Ora 拉数据生成 team report 均可调用）；**非完成态**（锁定 / 首次小结进行中）→ 管理端**不露出**（未定型内容不应进入对外举证链）|

### 1.5 3 状态详述

#### 1.5.1 ① 锁定态

前序 Activity 未全部完成时入口锁定 / 不进入对话场景。学员尝试点入时呈现：

```
┌───────────────────────────────────────────────────────┐
│ ⏳ 完成前序 X 个 Activity 后解锁                         │
│ 还需完成：[Activity Y] [Activity Z] [Activity W]       │
│                                                        │
│ [→ 跳转到 Activity Y]                                  │
└───────────────────────────────────────────────────────┘
```

不进入 recap 报告页面 / 不展示半成品报告 / 不计 Activity 完成事件 / 不消耗会话上下文。

#### 1.5.2 ② 首次小结（首次进入）★ 主场

**报告呈现**：前 5 章完整生成 / 第 6 章未启用（首次小结事件未触发）。

**Neo 行为（首聊态）**：完整态首次进入触发首聊态——Neo 主动逐章引导学员看报告。

> **语气说明**：以下 Walk-through 示例中 Neo 在右栏 Chat 说话用 **"我"**（场域层语气，详见 § 1.6.1）；当 Neo 引用报告时使用"我观察到 / 数据上显示"等组合表达；报告 markdown 文字本身用 **"数据显示 / 观察到"**（报告层语气）。

**Walk-through 节奏（仅首聊态 / 5 章版本）**：

```
1. Neo 第 1 句：定调 + 邀请反思
   "我们一起看看这份小结。我先一句话告诉你整体判断——
    {一句话综合 + 跨课程引用}。
    问你一件事：这句话听起来熟吗？跟你自己的感觉对得上吗？"

2. 学员回应 → Neo 引导到第 2 章 这一段我学了什么
   "我们看看你这一段学到了什么……
    特别是这几个工具的场景应用关联……"

3. 引导到第 3 章 这一段我学得怎么样
   "我们先看你学得怎么样——
    我观察到你在 [认知/能力/行为] 这几个维度上从 [学前状态] 转化到 [学后状态]……
    高光时刻在 [Activity X 的对话片段]……"

4. 引导到第 4 章 这一段我表现怎么样
   "再看看你这一段的上课状态——
    数据上看你互动了 N 次 / 6 维画像在 [维度] 上有变化……
    学习风格上你属于 [边练边问型 / 先理论后实战型]……"

5. 引导到第 5 章 我接下来该做什么
   "基于以上这些，我给你列了几件下一步可以试的事——
    我已经把它们做成任务卡片推到你面前 / 你可以选接收哪几条加到大厅待办。"
```

**关键点**：

- 首次小结是一次性事件——走完即触发，不重复
- 学员中途退出 walk-through → 重进时从中断点续 walk-through，不从头
- **没有"重新听 walk-through"机制** —— 首聊态完成后通过 Neo 自由问答覆盖学员需求

**留白式追问规范**（首聊态主场）：

Neo 追问的核心：**不替学员说总结，引发学员自陈**。

| 类型 | 示例 |
|------|------|
| 引发观察 | "你看到这条洞察吗？想到了什么？" |
| 引发反思 | "为什么你觉得这维度低？" / "这个模式是不是熟悉的？" |
| 引发联想 | "这跟你上周在大厅说的那件事有关系吗？" |
| 引发承诺 | "你觉得接下来一周能做的一件事是什么？"（首聊态收尾）|

**跨场域记忆调用规范**：详见 § 1.6.1 Neo 行为规范（集中说明 / 本节不重复）。

**必收尾约束**：Neo 追问之后**必须**给收尾——不能只追问留开放状态（沿用 practice 复盘 [05-learner/04-practice § 1.6.5]）。首聊态收尾 = Neo 给观察 + 留言 + 产出应用计划 + 任务卡片同步推送学员选择。

**学员动作**：Walk-through 跟随 / 主动追问 / 任务卡片选择是否接受 / 全程右栏 Chat 可问 Neo / 中途退出 walk-through → 重进时从中断点续。

**触发判定**：见 § 1.2.3 首次小结四件齐全。

**应用计划全流程**：见 § 1.3.8。

#### 1.5.3 ③ 自由复习态

> 本节主讲 **Neo 在自由复习态的解读与辅导行为**。

**Neo 在自由复习态的核心定位**：

- 不再走 Walk-through（首次小结已一次性走完）
- 学员主导对话节奏 / Neo 被动答疑 + 主动辅导
- 重点是**对报告内容的深入解读 + 对学员实际工作场景的针对性辅导**

**Neo 的 4 类辅导行为**：

| # | 辅导类型 | Neo 做什么 | 触发场景 |
|---|---------|------------|---------|
| 1 | **报告解读** | 解释报告中某章某句的含义 / 拆解 Neo 的判断依据 | 学员问"为什么你说我 XX" / "这一段什么意思" |
| 2 | **跨场域应用辅导** | 基于学员当前工作场景的实操建议 + 对学员应用工具尝试的反馈 | 学员说"我最近在 X 场景遇到 Y 问题"/"我试了 PIN 法但..." |
| 3 | **解锁还有疑问** | 针对第 2 章 C "还有疑问的"做深入讲解 / 苏格拉底式追问引导学员自己想通 | 学员问到第 2 章 C 中提到的疑问点 |
| 4 | **新增数据回访** | 关联 Activity 重练 / 大厅有新对话产生新画像变化时，Neo 主动指出变化点 + 引用上次对话 + 询问近况 | 自由复习层有新内容更新（Subbar "📊 报告有更新"红点）|

**Neo 行为约束**：

- **不重讲首次小结 Walk-through**（首次小结是一次性结构化讨论 / 学员"想再听一遍 walk-through"通过 Neo 自由问答覆盖）
- **不强行追问**（学员不主动来 Neo 不强行开场 / 但若进来时有报告更新 Neo 简短同步更新点）
- **必收尾**（一轮对话后给观察 + 留言 / 不留开放状态）

**与首次小结报告的关系**：

- 首次小结报告（前 5 章）**永久不变** —— Neo 解读时引用其内容，但不修改报告本身
- 自由复习层（第 6 章）**覆盖式更新** —— Neo 在自由复习态对话中产生的新观察、关联 Activity 重练的新画像变化、新高光新卡点等，全部进入第 6 章覆盖式更新（详见 § 1.3.7）

**学员侧动作**：

- 主动提问 Neo / 跨场域追问 / 对报告章节进行深入讨论
- 笔记悬浮球记录"想留住此刻"
- Neo Chat 历史可翻看（Activity 内不被清空）

### 1.6 全局通用

#### 1.6.1 Neo 行为规范（合并：反思设计原则 / 7 动作分工 / 双层语气 / 跨课程记忆 / 卡片产出）

**A. Neo 反思设计原则**

| 约束 | 说明 |
|------|------|
| **不替学员总结洞察** | Neo 引导学员说，不替学员说——防止报告 / 对话变成 Neo 的"评分书"。学员反思必须由学员**自陈**，Neo 用追问引发，不强行总结 |
| **不预设标准答案 / 不给"应该"** | 沿用 [02-foundation/02-methodology § 1.7] + [05-learner/03-lecture § 1.6.1] —— Neo 不告诉学员"你应该这样想 / 这样做"，引导学员自己得出 |
| **显性证据限定关联 Activity / 隐性背景不限定** | 报告内可被学员看到具体来源的证据类（高光 / 卡点 / 对练具体行为 / 6 维画像变化点）严格限定 KGP 配的关联 Activity 内；Neo 综合产物（自由复习层观察 / 应用计划依据 / 跨场域记忆）**不限定来源**（限定反而失真）。**不依赖 Quiz 错题**（Quiz 不一定有对错） |
| **不展示推理链** | Neo 在报告中**只说结论**，不写"我之所以这么判断…"|
| **不延伸到未学但相关的内容** | 严格贴现阶段已学 |

**B. Neo 在 recap 的定位 + 7 动作分工**

Neo 在 recap 与大厅 / lecture / practice 共用同一身份（同 voice / 方法论 / 记忆，详见 [02-foundation/01-personas § 1.2] + [05-learner/06-cross-context § 3.0]）。在 recap 中 Neo **引导反思为主**——不主导讲授（与 lecture 区别）/ 不旁观演练（与 practice 区别）。

核心姿态：**引导学员看见自己 + 不替学员说总结**。

| 动作 | 权重 | recap 中的体现 |
|------|------|---------------|
| **反思** | 主场 | 引导学员看见自己的行为模式与变化轨迹 |
| **追问** | 主场 | 苏格拉底式追问，挖出学员自己的洞察 |
| **讨论** | 主场 | 与学员就报告内容平等对话 |
| **解惑** | 辅助 | 学员对报告或概念有疑问时回答 |
| **考核** | 辅助 | 报告里的 6 维画像 + 高光 + 卡点 隐含评估反馈 |
| **讲授** | 轻量补漏 | 学员忘了某概念 Neo 可补，**不主动**讲新内容 |
| **练习** | — | 不出现（→ practice）|

**C. Neo 双层语气（核心约束）**

> 报告口吻 ≠ Neo 场域口吻。

| 层 | 用在哪 | 语气 | 人称 |
|----|------|------|------|
| **报告层（recap 报告 markdown 文字）** | 报告 6 章文字 | 客观正式 / 严肃教练 | 用 **"数据显示 / 观察到"** / **不用"我"** |
| **场域层（recap 内 Neo Chat 对话）** | recap 内 Neo Chat / 其他场域 lecture / practice 对话 | 像 1v1 老师 / 有立场 / 敢指出错位 | 用 **"我"** |

**两层之间的衔接**（重要）：

- Walk-through 中 Neo 在 Chat 里说话 → 用"我"
- 当 Neo 引用 / 转述报告内容时 → 用 **"我观察到 / 数据上显示 / 报告里写到"** 等组合表达
- 例："**我观察到** 你在 Activity X 上的高光是…" / "**报告里写到** 你这一段的稳定点在…，你怎么看？"

**当学员对"严肃教练"风格反弹**（希望 Neo 像朋友）时：
- 这指的是**报告口吻**，不是 Neo 本身的口吻
- Neo 在场域里仍可像 1v1 老师那样温和应对
- 报告口吻保持客观正式（不因学员反弹而切换）

**D. 跨课程记忆使用规范**

**密度**：报告**每章都引用**跨课程记忆。

| 章 | 引用方式 |
|---|---------|
| 第 1 章 综合评估 | 1-2 句跨课程引用作为定调 |
| 第 2 章 学了什么 | 在 if-then 应用关联中引用之前课程相关工具（如有连贯性）|
| 第 3 章 学得怎么样 | 认知转化按维度对比中引用之前课程的延续 / 突破（如"延续了你在《X》中的认知姿态"）|
| 第 4 章 表现怎么样 | 学习模式画像 + 6 维画像解读中引用跨课程一致性（如"6 维 X 维度延续了你在《Y》的高位"）|
| 第 5 章 接下来该做什么 | 行动建议中引用跨课程的应用机会 |

**引用形式**：

| 形式 | 示例 |
|------|------|
| 延续 | "**自《{先修课}》以来**，你在 {能力} 上持续保持高位" |
| 突破 | "**与你在《{先修课}》中的 {表现} 相比**，本课程你新打开了…" |
| 一致性 | "**这与你在《{课 A}》《{课 B}》中的特征是一致的**" |
| 应用机会 | "**结合你在《{先修课}》中练过的 {工具}**，可以试…" |

**引用准确性信任** [05-learner/06-cross-context § 4.1] 反幻觉机制。

**Walk-through 引用密度**：Neo 在场域 Chat 里**主动**引用学员跨场域 + 跨课程信息——让"老师记得你"成为体感。

**E. 卡片产出矩阵**

recap 3 状态卡片产出矩阵（5 类卡片定义见 [02-foundation/02-methodology § 1.6]）：

| 卡片类型 | ① 锁定 | ② 首次小结 | ③ 自由复习 | 备注 |
|---------|-------|------------|-----------|------|
| **askUserQuestion** | — | ✓ | ✓ | Neo Walk-through / 留白追问 / 回访态 全程可推 |
| **任务卡片** | — | ★ 主场 | — | 首次小结事件触发瞬间一次性产 1-3 条 + 同步推送学员选择是否接受加大厅待办（详见 § 1.3.8 / [05-learner/06-cross-context § 4.4]）|
| **高光卡片** | — | ✓ | ✓ | recap 不产出**新**高光（高光生于 lecture / practice 实际行为时刻）；recap 第 3 章**引用**关联范围内已有高光库子集；自由复习态若识别学员新顿悟可推 |
| **知识卡片** | — | ✓ | ✓ | Neo 反思引导中如学员表示某概念模糊 → 可补推延伸阅读形态知识卡片（轻量场景 / 不主动推）|
| **课程卡片** | — | — | — | 不出现（课程卡片是大厅 → 课程入口卡片 / 与场域内无关）|

> recap 与 lecture / practice 同口径：**课程卡片不进任何场域内**；高光 / 任务 / 知识 / askUserQuestion 按本场域定位差异化推送。

#### 1.6.2 学员行为 + 体验机制约束

**输入方式**：

| 方式 | 交互 |
|------|------|
| **右栏 Neo Chat 输入框** | 学员与 Neo 对话（打字 / 语音）—— recap 内学员只跟 Neo 对话，不写应用计划 |

**核心交互**（多数沿用 lecture 设计 [05-learner/03-lecture § 1.5.2]）：

| 交互 | 说明 |
|------|------|
| **解锁后进入** | recap Activity 入口在前序未完成时锁定；前序完成后**学员需重新点击进入**（不自动跳转）|
| **防挂机** | 默认 5 分钟无操作 → 系统弹提醒（沿用 lecture，可在 program config 调整阈值）|
| **学员主动提问 Neo** | 全程右栏 Chat 可问 Neo |
| **抽屉锚点跳转** | 学员点抽屉 Tab1 报告 5 章锚点 → 课件区滚动到对应章 |
| **Activity 间跳转** | 通过抽屉 Tab2 主动跳转其他 Activity（弹窗确认 + 触发断点续播保存）|
| **断点续播** | 中途退出 → 系统记录当前对话上下文 + Walk-through 进度；下次进入自动恢复 |
| **任务卡片选择** | Neo 推送任务卡片后，学员可选择"接受"（加入大厅待办）或"不接受"（仅留在报告里）|
| **笔记悬浮球** | 屏幕边缘常驻（沿用 [05-learner/02-hub § 1.3]）|

**体验机制约束**：

| 约束 | 说明 |
|------|------|
| **取消 SCO 锁定** | recap Activity 内部不存在 SCO 序列；与 lecture / practice 内部严格顺序锁定不同 |
| **入口前置锁定** | recap Activity 入口在前序未完成时锁定 / 不展示半成品报告 |
| **6 维画像可见性** | 学员侧 recap 报告露出（含变化点 / 当前快照）；管理端通过**完成态 recap 报告**间接可见（详见 § 1.3.9 / [02-foundation/01-personas § 1.4.4] Ora 上下文范围）；管理端独立 6 维画像 widget = [06-management/02-home § 1.2.2] (4) 个体详情面板内的 6 维画像快照（已落地决策 2026-05-04 / 复用 [02-foundation/02-methodology § 1.3.3] / [05-learner/02-hub § 1.1] B.2.B.4 雷达图组件）|

**通用沿用**：

- **防挂机**：默认 5 分钟无操作触发（沿用 [05-learner/03-lecture § 1.6.4]）
- **数据持久化**：推荐每个会话 / Walk-through 节点完成时落库（沿用 [05-learner/03-lecture § 1.6.6]）
- **隐私三层**：沿用 [04-global/01-user-global § 3.4.3]（学员之间默认完全隔离 / 跨学员引用需同意 / 管理端例外）

#### 1.6.3 完成 Modal

学员首次触发首次小结事件时弹出完成 Modal：

```
┌─────────────────────────────┐
│                        [X]  │
│         ✅                    │
│   Course 小结完成！            │
│  你已完成「{Course 小结名}」   │
│  Neo 已为你产出 {N} 条应用计划 │
│  其中 {M} 条已加到大厅待办      │
│                              │
│  [回到大厅]                   │
│  [自由复习]                   │
│  [继续下一 Activity]            │
└─────────────────────────────┘
```

**文案动态规则**：

- 学员接受 ≥1 条 → "Neo 已为你产出 {N} 条应用计划 / 其中 {M} 条已加到大厅待办"
- 学员接受 0 条 → "Neo 已为你产出 {N} 条应用计划 / 它们留在报告里 / 你可以稍后回头看"

按钮行为沿用 lecture / practice 3 选项设计（详见 [05-learner/03-lecture § 1.7.2] + [05-learner/04-practice § 1.8.7]）。

> **"自由复习"统一概念**：lecture / practice / recap 完成 Modal 里的"自由复习"按钮 + recap "自由复习态"状态，三者指向同一概念——**Activity 流程结束之后，学员允许回到该场域针对 Activity 内容和已经发生过的事情和 Neo 进行探讨**。

回到大厅：默认进大厅首页（项目总览区已含待办清单），不为 recap 跳转打破默认体验。

#### 1.6.4 跨场域衔接

> 全局跨场域机制详见 [05-learner/06-cross-context § 3.4] 跨场域协同（Truth Source）；本节仅从 recap 侧视角列入口与出口。

| 衔接 | 数据流 |
|------|--------|
| **关联 Activity（lecture / practice）→ recap** | 报告数据流：高光库子集 / 6 维画像变化 / 学员互动行为日志 / practice 复盘待改进点 / Neo 跨场域感知 全部进入报告生成（详见 § 1.3 各章数据来源）|
| **recap → 大厅** | 完成状态 + 报告进入大厅"我的报告"主题（[05-learner/02-hub § 1.2.3]）；应用计划经学员选择接受后进入大厅待办清单（详见 § 1.3.8 + [05-learner/02-hub § 1.2.1] 待办清单子条）|
| **recap → 管理端** | team report 拉**完成态**（首次小结事件后）recap 数据；首页 [06-management/02-home § 1.2.2] (4) 个体详情面板（行内展开含报告列表）展示**完成态**报告；数据共享但呈现完全分开（详见 [06-management/03-report-center § 1] / § 1.3.9 管理端可见性条）|

> **与大厅的边界**：项目进度类信息由大厅 [05-learner/02-hub § 1] 承担，recap 不承担也不重复（详见 § 1.3.9）。
