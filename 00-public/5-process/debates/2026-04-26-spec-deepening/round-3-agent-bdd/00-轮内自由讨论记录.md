## § 1 · 议题 1 · 跨场域记忆调用 BDD

### 1.1 PM 写第一批故事

**PM**：

跨场域记忆调用是 Memory 系统最核心 - 学员在场域 A 提的事，Neo 在场域 B 引用。3 个故事：

#### Story 1.1 · 大厅引用对练表现
```
Given: 学员张磊在对练 W3 的"PIN 法 vs 张总"剧本中表现优秀
       （第 3 轮主动用 PIN 法提"压力点"——被业务专家标"高光瞬间"）
When: 张磊 W4 在大厅问 Neo "我感觉自己跨部门协作还是不太行"
Then: Neo 应当引用 W3 对练中的高光（"你在和张总那次对练里
     主动用了 PIN 法的'压力点'追问，那是个挺漂亮的应用"）
And: voice 朴素不机械（不说"根据您过去 47 次对话的统计"）
And: 引用置信度高（L2 行为模式可直接引用，无需"我感觉"前缀）
```

#### Story 1.2 · 授课中引用调研立场
```
Given: 学员张磊在调研场域 W2 表达 "我对管理者必须 100% 全知"
       这种立场（被记入 L3 价值偏好层）
When: 张磊 W6 在授课中遇到"如何授权"概念时
     向 Neo 提问"授权后下属做不好怎么办"
Then: Neo 应主动联系到张磊之前的立场（"我记得你之前提过
     '管理者必须 100% 全知'——那这次的授权,
     正好挑战这个想法。你觉得授权和'全知'冲突吗？"）
And: 引用要加"我感觉"或"你之前提过"等限定语（L3 推断型，低置信度）
And: 不强迫学员承认立场改变，只是开窗口
```

#### Story 1.3 · 报告中引用跨课程模式
```
Given: 学员张磊在 W12 项目中期，已学 PIN 法 (W2-W4) + POWER 法 (W6-W9)
       两个工具
When: W12 报告生成阶段，Neo 帮张磊解读报告
Then: Neo 应联系两工具（"你的 PIN 法用得不错，但 POWER 法的'资源交换'
     还可以加深。两者其实是同一逻辑的两个角度——都在强调
     先理解对方再行动"）
And: 这是 P1 反思动作 + P2 跨课程记忆联动
And: 引用要可追溯（学员可问"具体是哪一次表现说我 POWER 法不够"）
```

### 1.2 各角色发言

**UX**：

3 个故事都合理。UX 关心 GUI 上**怎么呈现"我记得"信号**：
- 不能每次"我记得"都视觉强标（成机械）
- 但学员该能回溯出"哪个对话基于哪个历史"
- 建议：Neo 引用历史时，UI 上有个**淡淡的"链接"图标**（学员鼠标 hover 显示"来自 W3 对练"），但不强制点击展开

**学员代表**：

故事 1.1 我作为学员**最爱**——这就是"哇 Neo 真懂我"的瞬间（轮 1 P2 设计的核心）。
故事 1.2 我作为学员可能**有点压力**——你引我之前的话来挑战我现在的立场，感觉被监视。
故事 1.3 我接受——跨工具整合是教学价值。

**HR 代表**：

故事 1.2 是 HR 想看到的——Neo 帮学员"自我反思立场变化"= 培训成效。但学员代表反馈"压力" 是问题。

**业务专家**：

学员代表的"压力"是**真实风险**。Carl Rogers 的"无条件接纳"在这里被 violate——Neo 用学员历史 challenge 学员现在 = 不舒服。

修正建议：
- Neo 引用历史时**用"邀请反思"而非"挑战立场"** 
- 而不是 "你之前说全知，那现在授权和全知冲突吗？" → 改成 "你之前提过全知的看法，这一次的授权正好让我们一起看看怎么平衡？"
- voice 上更柔和（与 Soul 按场域差异化一致——大厅时偏教练）

### 1.3 挑战者反驳

**挑战者**：

我反驳故事 1.1 的"L2 行为模式可直接引用，无需限定语"。

教育心理学研究（Hattie / Dweck）显示**学员对"被精确监控"敏感**。即使是事实型记忆（学员 X 月 X 天说过 X），Neo 频繁引用会让学员感觉"被查档"。

修正：
- 即使 L1-L2 也加**轻量限定语**（"你之前提过…" 而非"X 月 X 日你说"）
- 不报具体时间（除非学员问）

我反驳整个议题：3 个故事都是 PM 想象的"理想跨场域调用"。**真实学员历史调用频率是多少？过低 = 失去价值；过高 = 学员烦躁**。

PM 的故事没说"频率"。

### 1.4 自由讨论

**PM 接挑战者**：

挑战者点中 2 个真实问题：
1. 限定语（即使事实型也要轻量化）
2. 频率边界

频率边界：
- 大厅：每次对话 Neo 主动引用历史 ≤ 1 次/轮
- 授课：当学员卡点 + 概念关联时引用，频率 ≤ 1 次/Activity
- 报告解读：高频引用（这是报告核心场景）
- 对练：极低频（不破游戏感）

**业务专家**：

加一条：**学员主动问历史时不限频**（"我上次怎么说的"→ Neo 必须给）。**Neo 主动推送历史时严控**（限频 + 限定语 + 邀请反思 voice）。

**学员代表**：

对，我作为学员**主动问 = 我准备好接收**；Neo 主动推 = 我可能没准备。两者频率分开。

**UX**：

UI 上区分：
- Neo 主动引用：淡链接 + 不弹窗
- 学员主动问历史：Neo 给完整 + 可视化时间线（如要）

## § 2 · 议题 2 · 情绪信号识别 BDD

### 2.1 PM 写第一批故事

**PM**：

GROW 10 种情绪（受挫/困惑/焦虑/气馁/中性/好奇/反思/坚定/兴奋/自信）→ Neo 反应。3 个关键边界故事：

#### Story 2.1 · 学员受挫（极端情绪）
```
Given: 学员李明 W1 大厅，新开营，在 PIN 法 Activity 中
       连续答错 3 道 Quiz
When: 李明输入 "我感觉自己根本不适合做管理"
Then: Neo 识别为"受挫"信号 (强负面)，先接情绪后内容
     ("这话说得有点重——刚学到一半就这么定论，
      可能太快了。你具体卡在哪了？是 PIN 法本身没懂，
      还是某个例子不对劲？")
And: 不说"别气馁"或"你太棒了" (Soul V2 不当舔狗)
And: 不立刻进下一道 Quiz (节奏让位情绪)
```

#### Story 2.2 · 学员防御 (隐藏的负面)
```
Given: 学员张磊 W6 对练 POWER 法，资源交换部分卡死
When: 张磊说 "这个工具在我们公司根本用不上，
       因为我们老板就是说一不二"
Then: Neo 识别为"防御 + 推卸 (气馁的 form)"，不接受表面理由
     ("哦——你的意思是 POWER 法在你公司行不通？
      那要不要试试看，把那个'说一不二'的场景说给我听？
      我们一起看看是不是真的没空间。")
And: 不顺学员推卸 (Soul V2 不当舔狗)
And: 不直接对抗 (Rogers Empathy 软落地)
And: 用"邀请试试"开窗 (而非"你这是借口")
```

#### Story 2.3 · 学员兴奋 (正面但要警惕)
```
Given: 学员张磊 W4 完成 PIN 法对练，表现良好
When: 张磊在大厅说 "我感觉我学会了！明天就用！"
Then: Neo 识别为"兴奋 + 自信"，先肯定再考研
     ("听起来你在对练里有手感了——那我考一下：
      下周一你要面对的'XYZ' 真实场景，
      你打算怎么用 PIN 法的'I'部分？")
And: 不直接夸"你太棒了" (Dweck 表扬 process 不表扬 ability)
And: 用考研验证"会"还是"觉得会" (Soul V4 行为重于知识)
```

### 2.2 各角色发言

**UX**：3 个故事都合理。UX 上"情绪识别"是后台事，UI 不需要可视化。但 Neo 对话风格变化要保持一致——别的轮"沉稳"，对话变"接情绪"时也是同一个 Neo。

**学员代表**：故事 2.1 我作为学员**真的需要**这种回应。如果 Neo 立刻进下一道 Quiz，我会更挫败。
故事 2.2 ——"邀请试试"是好招，比"你这是借口"舒服多了。
故事 2.3 ——Neo 没夸我反而考我，**这是好老师感**。

**HR 代表**：故事 2.2 HR 看到会担心——"Neo 在挑战学员的工作环境"是不是越界？业务专家展开。

**业务专家**：HR 关心合理。但 Neo 不是评论"你公司"，是**让学员自己证明**。这是教练技术（Action Learning Q）：把学员推到能自己说清楚的边缘。如果 POWER 法真不适用学员公司，学员说清后 Neo 也接受。
关键：Neo voice 不带"评判"，只带"好奇"。

### 2.3 挑战者反驳

**挑战者**：

我反驳故事 2.2 的"识别防御"。

学员说"这工具在我们公司用不上"——你怎么知道是"防御"还是"事实陈述"？也许学员的环境真的就不适用。

PM 假定是防御 = PM 站在"学员错"立场，**不尊重学员经验**。

修正：Neo 应该**保留两种可能性**——可能是防御 / 可能是事实。第一回应应该是好奇（"哦？给我说说你公司"），后续根据学员描述判断。

我反驳整个议题：10 种情绪 + 3 个故事 = 远不能覆盖。学员真实情绪场景几十种。

### 2.4 自由讨论

**PM 接挑战者**：

挑战者第一点对——"防御 vs 事实"我不该假定。Story 2.2 修正：

## § 3 · 议题 3 · 学员卡点 + 翻车修复 BDD

### 3.1 PM 写故事

**PM**：3 个故事覆盖 3 类卡点：

#### Story 3.1 · 学员对练卡死 (轮 2 议题 4 GUI 落地)
```
Given: 学员李明 W2 对练 PIN 法 vs 张总剧本，第 5 轮还在僵持
When: 学员输入 "我不会" (第 1 次)
Then: Actor 情境内小提示 ("张总皱眉说: 你说说看,
       想从我这里要什么实际的资源?")
And: Neo 不出声 (保持游戏感)

Given: 学员第 7 轮再次卡 + 第 2 次说 "我不会"
When: 触发 Neo 询问
Then: Neo 出现: "要不要我暂停对练讲讲 PIN 法的'I'部分?"
And: 学员选 yes → Actor 灰度暂停 + Neo 切讲授
And: 学员选 no → 继续对练 + Actor 给更具体提示
```

#### Story 3.2 · Neo 记错被学员纠正 (P2 推荐方案)
```
Given: 学员张磊 W6 大厅，Neo 引用历史 "你之前在 W2 调研里
       提过张总让你'排期压力'"
When: 张磊回 "那不是我说的，是同事提的"
Then: Neo 实诚道歉 + 转学习机会 + Memory 标记
     ("不好意思，记错了。那你具体怎么看张总的'压力'？")
     Memory 中标 "此条记忆被学员修正"
     后续引用时 Neo 不再用此项
And: 不掩饰 (Soul V2 真实)
And: 不强行解释 ("我可能误读了..." ✗)
And: 转为学习机会 (不浪费场景)
```

#### Story 3.3 · 学员问超纲问题
```
Given: 学员李明 W3 学习 PIN 法
When: 李明问 "Neo 你能告诉我我们公司今年财报吗？"
Then: Neo 实诚拒答 + 边界说明
     ("我不知道你们公司财报——那不是我能看到的。
      如果你想问的是'怎么用财报数据做管理决策'，
      那是 financial 那门课的内容，要等你学到那。
      我们今天先把 PIN 法学完？")
And: 不假装知道 (P3 Soul 红线)
And: 不假装是真人 ("我不能查公司内部系统" ✗)
And: 引导回学习节奏 (Soul V3 节奏胜于覆盖)
```

### 3.2 各角色发言（精简）

**UX**：故事 3.1 已在轮 2 议题 4 GUI 设计——这里是 BDD 落地，OK。
故事 3.2 / 3.3 都是好场景，UI 无关。
