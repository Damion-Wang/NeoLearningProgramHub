# AI老师 — MVP 定义文档

> **阶段**: Phase 6 — MVP Definition  
> **项目**: ai-teacher  
> **日期**: 2026-04-09  
> **MVP 范围**: 基层管理者·横向协同 培训项目（NeoLearning V1）  
> **交付周期**: 2个月（5月中内测 → 6月底 Online MVP）  
> **目标**: 可售卖给60家天使客户的完整培训项目  
> **置信度**: 中等（基于Phase 1-5研究综合 + 已有技术验证，成功标准数值均为估算）  
> **标签说明**: [Data] = 已验证数据, [Estimate] = 推算值, [Assumption] = 待验证假设, [Opinion] = 主观判断

## 1. 核心假设（Core Hypothesis）

> **AI原生自适应培训可以产出可验证的行为证据（Kirkpatrick L3），让HR用数据向业务部门证明培训ROI。**

### 假设拆解

| # | 子假设 | 验证方式 | 标签 |
|---|--------|----------|------|
| H1 | AI授课+AI对练的交互式学习效果优于视频课单向灌输 | 天使客户A/B对比：学员能力测评分数 vs 传统视频课 | [Assumption] |
| H2 | AI情境模拟和对练可产出**行为层数据**（能力提升曲线、错误模式变化），填补Kirkpatrick L2→L3断层 | MVP交付后检查：AI报告是否包含可被HR直接引用的行为证据 | [Assumption] |
| H3 | HR拿到行为证据报告后，能有效说服业务部门/管理层认可培训价值 | 天使客户回访：HR是否实际使用AI报告进行汇报及效果 | [Assumption] |
| H4 | "横向协同"主题足够通用，可覆盖60家天使客户的真实需求 | 60家中实际签约/启动项目的转化率 | [Assumption] |
| H5 | 自适应（L1+L2）能让不同水平学员获得差异化路径，避免"千人一面" | 学员端数据：不同学员的SCO路径分布是否呈现差异化 | [Assumption] |

### 为什么是这个假设

- 82%的企业仅做到满意度评估（Kirkpatrick L1），64%因"效果不可视"被砍预算 [Data]
- 仅30.7%的企业能执行到行为层评估（L3），核心原因是**缺乏技术手段采集行为数据** [Data]
- AI对话、情境模拟天然产出行为层数据——这是视频课根本无法提供的 [Opinion]
- HR的核心困境是政治生存问题：无法证明效果 → 预算被砍 → 资源更少 → 效果更差 [Data]

### 假设验证逻辑

```
AI交互式教学（授课/辅导/对练）
    ↓ 天然产出
行为层数据（对话深度、能力曲线、错误模式变化、情境表现）
    ↓ 结构化为
Kirkpatrick L3 行为举证报告
    ↓ 使得
HR可向管理层"用数据说话"，打破死亡螺旋
    ↓ 验证
"AI原生培训 ≠ 更好的视频课，而是全新的效果举证范式"
```

**如果假设成立**：AI老师填补了中国企业培训市场最大的空白——K-L2到K-L3断层。这是现有竞品的嫁接式AI架构结构性无法提供的。 [Data + Opinion]

**如果假设不成立**：AI老师退化为"更好的线上课程工具"，差异化仅限于交互体验，无法支撑1,600-1,800元/账号的溢价定价。 [Opinion]

## 2. Must-Have 功能：学员端

### 数据模型（全局）

```
Project（培训项目：基层管理者·横向协同）
  └── Course（课程）
       └── Activity（学习活动：授课/对练/辅导/测评/萃取）
            └── SCO（最小学习单元，支持多版本 + tag标记，用于L2自适应分支）
```

### 2.1 AI授课（交互式讲解）

| 维度 | 说明 |
|------|------|
| **描述** | 替代视频课的AI原生授课体验。AI以对话+多媒体方式讲解"横向协同"核心知识点，学员可随时提问、互动 |
| **当前状态** | Demo已有 [Data] |
| **MVP要求** | 产品化Demo，覆盖"横向协同"完整课程内容；支持L1自适应（LLM实时响应）+ L2自适应（SCO多版本分支） |
| **验收标准** | 学员可完整走完一个Course的所有授课Activity；交互率 > 传统视频课完成率 |
| **数据模型映射** | Activity(授课类) → SCO(讲解内容，多版本+tag) |

### 2.2 AI对练（模拟场景演练）

| 维度 | 说明 |
|------|------|
| **描述** | 基于真实工作情境的角色扮演对练。学员扮演基层管理者，AI扮演跨部门同事/上级，模拟横向协同场景 |
| **当前状态** | 模块已有 [Data] |
| **MVP要求** | 产品化现有对练模块；至少3个横向协同典型场景（如跨部门资源争夺、项目推进受阻、利益冲突调解）[Estimate] |
| **验收标准** | 对练过程产出可量化的行为数据（沟通策略选择、情绪管理、利益平衡等维度评分） |
| **核心价值** | **这是产出Kirkpatrick L3行为证据的主要来源** |

### 2.3 AI辅导（个性化指导）

| 维度 | 说明 |
|------|------|
| **描述** | 基于学员授课和对练表现，AI提供个性化改进建议和知识补充 |
| **当前状态** | 需新建 |
| **MVP要求** | 辅导内容基于学员实际表现数据生成（非通用建议）；可在对练后自动触发或学员主动发起 |
| **验收标准** | 辅导建议引用学员在对练中的具体表现；学员可基于辅导建议重新进入对练 |

### 2.4 AI测评（行为证据采集）

| 维度 | 说明 |
|------|------|
| **描述** | 非传统选择题考试。通过情境模拟+开放式对话评估学员的"横向协同"能力，产出行为层证据 |
| **当前状态** | 需新建（可复用对练模块能力） |
| **MVP要求** | 前测（项目开始时）+ 后测（项目结束时），产出可对比的能力变化数据；评估维度至少覆盖：沟通策略、利益协调、冲突处理 [Estimate] |
| **验收标准** | 前后测数据可自动生成能力变化曲线；评估结果可被HR报告直接引用 |
| **战略意义** | **填补Kirkpatrick L2→L3断层的关键功能，是整个MVP的核心差异化** |

### 2.5 AI报告（学习成果报告）

| 维度 | 说明 |
|------|------|
| **描述** | 面向学员个人的学习成果报告。汇总授课、对练、测评全过程数据，呈现个人能力变化 |
| **当前状态** | 需新建 |
| **MVP要求** | 自动生成；包含：能力雷达图、前后测对比、对练行为亮点和改进点、个性化发展建议 |
| **验收标准** | 学员可查看完整学习报告；报告内容基于真实行为数据而非模板 |

### 2.6 AI萃取（作业/知识提炼）

| 维度 | 说明 |
|------|------|
| **描述** | 引导学员将所学知识与自身工作场景结合，通过AI对话完成结构化作业/行动计划 |
| **当前状态** | 需新建 |
| **MVP要求** | 至少1个萃取活动（如制定个人"横向协同改进计划"）；AI引导结构化输出，非自由填写 |
| **验收标准** | 萃取产出物有统一结构，可被管理端汇总分析 |

### 2.7 自适应引擎（L1+L2 产品化）

| 维度 | 说明 |
|------|------|
| **描述** | 不同学员走不同学习路径。基础好的跳过基础内容，薄弱的补充强化 |
| **当前状态** | L1（LLM交互级）已实现 [Data]；L2（SCO级分支）已实现但hack实现 [Data] |
| **MVP要求** | L1维持现状；L2从hack升级为可配置的产品化机制——Quiz SCO → tag判定 → 分支SCO选择的通用流程 |
| **验收标准** | 运营人员可通过配置（非代码）设置分支规则；至少2个分支点在横向协同课程中实际运行 [Estimate] |

## 3. Must-Have 功能：管理运营端

### 3.1 运营工具

| 维度 | 说明 |
|------|------|
| **描述** | 培训运营人员的日常工作台：项目创建、学员分配、进度跟踪、催学提醒 |
| **MVP要求** | 项目创建与配置（选择课程、设置时间线）；学员导入/分配；进度概览（完成率、活跃度）；基础催学功能（手动/定时触发） |
| **核心用户** | 培训运营人员——日常使用者和最强内部推动者 [Data] |
| **关键约束** | 定位为"赋能运营人员"而非"替代运营人员"。21.13%的从业者担心AI替代培训岗位 [Data] |

### 3.2 报告解读Agent

| 维度 | 说明 |
|------|------|
| **描述** | HR/培训经理可用自然语言向Agent提问，获取项目数据洞察。如："这批学员在冲突处理方面表现如何？""哪些学员进步最大？" |
| **MVP要求** | 支持基于项目数据的自然语言问答；可生成可用于PPT汇报的结论性文字 |
| **战略意义** | 直接解决HR"用数据说话"的核心需求，降低数据分析门槛 |

### 3.3 Dashboard（数据看板）

| 维度 | 说明 |
|------|------|
| **描述** | 项目级和组织级的数据可视化看板 |
| **MVP要求** | 项目进度概览；学员能力分布；前后测对比趋势；对练行为数据汇总；完成率和活跃度统计 |
| **验收标准** | HR可在不借助外部工具的情况下，直接从Dashboard获取汇报所需的关键数据 |

### 3.4 Harness报告生成

| 维度 | 说明 |
|------|------|
| **描述** | 根据管理员需求，自动生成结构化的培训项目结项报告 |
| **MVP要求** | 一键生成标准结项报告（PDF/PPT格式）；报告包含：项目概览、学员参与数据、能力变化分析（Kirkpatrick L3证据）、典型案例摘要、改进建议 |
| **验收标准** | 报告可直接用于HR向管理层汇报，无需大量二次编辑 |
| **战略意义** | **HR"政治生存武器"的最终交付物——把行为证据包装成管理层看得懂的语言** |

## Flags

### Red Flags

- **2个月MVP范围仍然很大**：学员端7个功能模块 + 管理端4个功能模块，即使有Claude Code加速，交付风险依然存在。建议设置"最小可售卖子集"——如果时间不够，优先保证：AI授课 + AI对练 + AI测评 + AI报告 + Harness报告生成。这5个构成行为证据闭环的最小集合。 [Opinion]
- **L2自适应产品化是技术卡点**：从hack到可配置产品化，涉及数据模型、编排逻辑、运营工具三层改造，可能比预期耗时更长 [Data + Opinion]
- **测评维度的科学性**：如果行为测评维度不被HR认可，整个"L3行为证据"的叙事会崩塌。建议在开发前与3-5位天使客户HR共创测评框架 [Opinion]

### Yellow Flags

- **"横向协同"主题的普适性**：60家天使客户的行业和管理层级可能差异较大，"横向协同"是否足够通用需要验证 [Assumption]
- **企业微信集成推迟到v1.1**：企业微信是核心学习入口 [Data]。MVP阶段通过链接分发可行，但可能影响学员体验和完成率 [Opinion]
- **天使客户期望管理**：60家天使客户是"体验"还是"付费"？如果是免费体验，成功标准中的"付费转化率"指标可能不适用 [Assumption]
- **AI报告的"可信度"**：HR是否信任AI生成的行为评估报告？这是新认知，需在天使阶段密切关注 [Assumption]
