## 5. HR Agent对话

### 关键交互

| 交互 | 触发 | 响应 | 联动 |
|------|------|------|------|
| 数据查询 | "本周谁进度最快？" | Agent回答+数据截止时间 | 看板对应行高亮 |
| 个人查询 | "张三学习情况如何？" | Agent返回数据摘要 | 看板高亮张三行 |
| 报告生成 | "帮我准备CEO汇报" | Agent确认需求→选模板→生成→预览→修改→导出 | 看板切换为报告预览 |
| 能力边界 | 超出范围提问 | Agent坦诚告知+建议 | 无 |
| 看板联动 | HR先点看板再追问 | Agent关联看板上下文 | 自动承接 |

### 报告生成流程视觉

```
HR: 帮我准备给CEO的月度汇报报告

Agent: 好的，请确认:
(1) 模板——用上次的还是新模板？
(2) 重点——关注哪些方面？

HR: 用上次的，重点看进度

Agent: 收到，开始生成，预计30秒...

[看板区切换为报告预览视图]

Agent: 报告初稿已生成，请预览。

HR: 第三段结论太笼统，加入张三李四案例

Agent: 好的，修改中...
[看板区报告实时更新，修改部分高亮]

Agent: 修改完成，是否定稿？

HR: 可以了，导出PDF

Agent: 正在生成PDF...
[弹出下载确认]
```

**Agent约束:** 仅描述可验证数据，推测性分析标注"AI推测，请酌情参考"

## 11. 催学管理

### 催学规则合并执行逻辑（HR+运营共用）

1. 同一学员同一天不重复发送相同条件消息
2. **系统硬上限: 每人每日不超过3条（不可调高）**
3. 运营规则默认优先级高于HR规则
4. HR通过协商获得覆盖权后优先级提升至与运营平级

### 运营Agent催学快捷操作

```
运营: 帮我给所有逾期学员发一条催学消息

Agent: 检测到当前有5名逾期学员。
即将发送给: 张三、李四、王五、赵六、陈七
消息内容: "你有一个新的管理技能课程可以学习..."

确认发送？ [确认] [修改] [取消]

运营: 确认

Agent: 已向5名学员发送催学消息。可在催学管理中查看状态。
```

## 14. 组件需求

| 组件 | 规格 |
|------|------|
| 项目选择器 | 卡片列表，每卡片含名称+学员数+进度+进入按钮 |
| 进度表格 | 可排序+筛选+搜索，行可点击展开详情 |
| 半屏详情面板 | 右侧滑出，不完全遮挡列表，含返回按钮 |
| 环形进度图 | 可点击，hover显示数值 |
| 催学规则表格 | 含状态开关、创建者标识、操作按钮 |
| 效果漏斗图 | 发送→查看→登录→完成，每层可点击查看名单 |
| 五步向导 | 步骤条+表单+校验+上下步按钮 |
| 报告预览 | 看板区全宽预览，支持实时更新+修改高亮 |
| 审核队列表格 | 支持键盘快捷键(J/K/A/R/Esc) |
| 批量进度条 | 实数进度(24/30)，可中途离开 |
| 体验模式提示条 | 顶部常驻，醒目颜色，含返回按钮 |
| 数据高亮 | 黄色底色+图标标记，满足提醒条件自动触发 |
| Agent催学预览 | 名单+消息全文预览，确认/修改/取消三按钮 |
