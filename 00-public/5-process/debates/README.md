# debates/ -- 产品设计辩论

## 辩论机制

每次辩论由 **7个角色 + 1个主持人** 参与：
- 学员代言人、HR代言人、运营代言人
- 产品经理、授课老师、技术架构师、商业分析师
- 主持人（控制议程和深度）

**3轮深度递进**：
1. 第一轮：广泛识别问题
2. 创始人回应：确认/否认/补充
3. 第二轮：聚焦 P0+P1 问题细化 behaviour 方案
4. 创始人二次回应
5. （第三轮：可选，视需要启动）

创始人每轮结束后介入，提供真实业务上下文。
