### 5.6 数据反馈循环

> AOM 不是单向的"生产 → 演绎"——学员的真实行为数据会反馈到 KGP，迭代优化 AOM。

#### 5.6.1 采集什么

平台运行时采集学员的**半结构化行为数据**——产品视角关注 4 类信号：

| 信号类型 | 产品意义 |
|---------|---------|
| **交互行为** | 学员在 Quiz / 对话 / 角色扮演中说了什么、做了什么 |
| **能力标签** | 这次互动覆盖了哪些能力维度（如"反馈沟通" / "冲突处理"）|
| **行为观察** | AI 标记的具体行为表现（如"使用了共赢策略" / "首次主动倾听"）|
| **被动信号** | 停留时长 / 中途离开 / 回翻 / 点击工具卡片（参与度的辅助指标）|

#### 5.6.2 反馈到 KGP 的价值

- KGP 看到"学员在哪个 SCO 卡得最多" → 优化通关条件 / 重写演示素材
- KGP 看到"学员标签分布" → 调整版本数量与覆盖
- KGP 看到"哪些工具最常被学员引用" → 扩充知识工具库
- KGP 看到"哪个 Activity 整体效果差" → 重设计教学意图

#### 5.6.3 闭环图示

```
KGP 生产 AOM 内容包
   │
   ▼
平台演绎（Neo 按 AOM 教学）
   │
   ▼
学员行为数据采集
   │
   ▼
反馈到 KGP（数据看板 / 异常告警）
   │
   ▼
KGP 迭代 AOM（修订下一版内容包）
   │
   └──→ 循环
```
