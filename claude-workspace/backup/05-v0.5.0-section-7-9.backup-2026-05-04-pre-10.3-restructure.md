### 10.3 报告中心 Report Center

> 这是 § 10 的**核心模块**——管理端的主要交付能力 / 对外可举证报告产出场域。
>
> 双主体：**报告库**（不需要 Ora · 看历史 + 新建入口）+ **报告编辑**（需要 Report Ora · vibe 写作 + 划词修改 + ChatBI 图表生成）。
>
> Ora 实例：**Report Ora** = § 10.2 Home Ora 之外的第 2 个 Ora 实例 / 信息共享 / 各有独特能力（详见 § 10.1 待补 § 10.1.6 Ora 整体定位与双实例实现）。

#### 报告中心 = 2 主体 + 2 Tab + 6 图表

```
报告中心 Report Center
├── 报告库（不需要 Ora · § 10.3.2）
│   ├── Tab 1 管理员报告（当前管理员独立 / 全部可编辑 / 时间倒序）
│   └── Tab 2 学员报告（项目过程产生 / 完成态 recap + 对练 + 高光库 / 按学员分组）
└── 报告编辑（需要 Report Ora · § 10.3.3）
    ├── 编辑器（WYSIWYG Markdown + 插图 + 跨 Tab 引用）
    ├── 6 类图表（柱状 / 折线 / 表格 / 堆叠柱状 / 饼图 / 散点图）
    ├── 双触发（chat 直接 + 编辑器按钮多轮确认）
    └── Report Ora 协作（vibe 写作 + 划词修改 + 自动保存）
```

#### 入口与模块关系

- **入口**：管理端左侧导航第 2 项 ★ / 也可从 Home 跳入（沿用 § 10.2.4.3）
- **创建报告路径**（本期统一手动）：报告库 Tab 1 顶部"+ 新建报告"→ 弹表单 → Ora 生成初稿 → 进入编辑器 HITL 协作
- **与其他模块关系**：
  - § 10.2 首页 Home：Home Ora 上下文传 Report Ora（信息共享 / 沿用 § 10.2.4.3）
  - § 10.4 项目配置：本期不联动（未来 milestone 触发会读 config）
  - § 10.5 消息中心：本期不联动（未来 milestone 触发会推消息）
  - § 9.3.3 学员侧 recap 报告：完成态 recap raw 进 Tab 2 / 详见 § 10.3.2.3

#### 10.3.1 业务定义

管理员产出**对外可举证 ROI 报告** + 历史归档查看 + 报告库管理 的场域。Ora 拉全程数据 + 管理员描述意图 + Ora 生成初稿 + HITL 协作（vibe 写作 + 划词修改 + ChatBI 图表）+ 导出 PDF / Word。

**与学员端 § 9.3.3 recap 报告的边界**：

| 维度 | § 9.3.3 学员 recap 报告 | § 10.3 管理端 team report |
|------|----------------------|------------------------|
| 主体 | 学员侧个人反思 | 管理员侧团队汇报 |
| 粒度 | 个人 × Course | 团队 × 项目周期 |
| 触发 | milestone 完成 | 管理员手动新建（本期）|
| 编辑 | 学员不可改 | 管理员可改（HITL + Ora）|
| 数据共享 | recap 完成态 raw 进入管理端（沿用 § 9.3.3.2.10 管理端可见性）| Ora 6 维 + Home Ora 跨模块记忆 |

#### 10.3.2 报告库

##### 10.3.2.1 整体结构（2 Tab + 不需要 Ora）

报告库主页 = **2 个 Tab 横排切换** + 顶部"+ 新建报告"按钮（仅 Tab 1 显示）。

**报告库本身不需要 Ora 参与**——纯 GUI 列表 / 查询 / 筛选 / 进入编辑器入口。Ora 在管理员**进入某个具体报告编辑**时才接管（详见 § 10.3.3）。

##### 10.3.2.2 Tab 1 · 管理员报告

**包含范围**：当前管理员账号下的**全部报告**——
- 全部报告**统一为可编辑状态**——不区分草稿 / 完成 / 已导出
- 是否导出**由管理员决定**（导出不锁定 / 导出后仍可继续编辑）
- 沿用 § 10.3.5.2 每管理员独立报告库

**组织方式**：按**创建时间倒序**——最新的报告置顶。

**查询能力**：
- **报告名 + 关键词查询**（顶部搜索框 / 模糊匹配）
- **按时间范围筛选**（创建日期 起止 picker）
- 不含状态筛选（无草稿 / 完成区分）

**报告条目展示**（列表行）：
| 列 | 内容 |
|----|------|
| 报告名 | 文字（点击 → 进入编辑器）|
| 创建时间 | 相对时间（如"2 天前"）+ tooltip 绝对时间 |
| 时间周期 | "项目至今" / "W4-W8" / "2026-04-01 至 2026-05-01" 等 |
| 人员范围 | "全部学员"或"X 人选定" |
| 操作 | 导出 chevron（PDF / Word / docx）+ 删除按钮 |

##### 10.3.2.3 Tab 2 · 学员报告

**包含范围**：项目过程中学员端产生的报告 ★ 3 类——
1. **完成态 recap 报告**（仅 milestone 后 / 红线已校准 § 9.3.3.2.10 / § 3.4.4 一致）
2. **对练报告**（所有学员的所有 practice 演练报告）
3. **高光库**（项目级聚合 / 跨学员 + 跨 Activity 的高光卡集合）

**组织方式**：**按学员分组**——
```
张三（中期学员）
├── 📊 recap 报告
│   ├── Course 1 横向协作 · 2026-04-15 →
│   └── Course 2 辅导反馈 · 2026-05-02 →
├── 🎭 对练报告
│   ├── Activity 1 · 第 3 次 · 2026-04-10 →
│   └── ...
└── ✨ 高光卡（X 张）→

李四（新员工）
└── ...
```

**查询能力**：
- **学员姓名查询**（顶部搜索框 / 模糊匹配）
- **报告名 + 关键词查询**（在选定学员或全员范围内搜）

**不可编辑**——学员侧产出的报告管理端只读 / 可引用到 Tab 1 自己的报告中（详见 § 10.3.2.5）。

**与 § 10.2.2.2 (5) 学员个人报告库的差异**：

| 维度 | § 10.2.2.2 (5) 学员个人报告库 | § 10.3.2.3 Tab 2 学员报告 |
|------|------------------------------|--------------------------|
| 入口 | 首页人员维度 → 点学员 → 弹窗 | 报告中心 → Tab 2 |
| 范围 | 单学员的报告 | **项目内全部学员**的报告 |
| 用途 | 个体诊断 / 看具体学员产出 | 报告编辑时引用 / 项目级综合查阅 |
| 形态 | 弹窗 + 新开页查看 | 主页内分组列表 |

##### 10.3.2.4 报告创建入口（统一手动表单）

**唯一入口**：Tab 1 顶部"+ 新建报告"按钮（**本期不做 milestone 自动触发** / 详见 § 10.3.6）。

**点击 → 弹表单**（5 字段）：

| 字段 | 形态 | 必填 | 不选模板默认 | 选"综合汇报"模板 |
|------|------|:-:|------------|----------------|
| 报告名称 | 文字输入 | ✓ | 空 | 空（模板不预填名称）|
| 时间周期 | 起止日期 picker（含"项目至今"快捷选项）| ✓ | 项目至今 | 项目至今 |
| 人员范围 | 全部学员（默认）+ 多选学员名单（可选切换）| ✓ | 全部 | 全部 |
| 报告描述 | 多行文本框 / **给 Ora 的写作指令**（如"分析项目至今学员的学习进展 / 重点关注完成率 + 6 维画像 / Top + 风险学员各 Top 3"）| ✓ | 留空 / 管理员自己填 | "综合汇报" |
| 模板选择 | 下拉（"不选模板" + "综合汇报"）| 可选 | "不选" | "综合汇报" |

**关键差异**（选模板 vs 自定义）：
- 选"综合汇报"模板 → **3 个字段全预填**（时间 + 人员 + 描述）/ 名称仍需管理员填
- 不选模板 → 时间 + 人员预填，**报告描述留空**

**模板规则**：
- 本期 1 个系统预制 = "**综合汇报**"
- 不可改 / 不可新增 / 由 admin 后台预制
- 未来扩展（V0.6.0+）：admin 在 § 10.4 项目配置中预制更多模板 / 管理员仅选用

**提交流程**：
```
管理员点提交
   ↓
Ora 拉数据（按时间周期 + 人员范围筛选 / 数据范围 § 3.4.4.2 全 6 维 + Home Ora 跨模块记忆 § 10.2.4.3）
   ↓
Ora 根据"报告描述"生成完整初稿（含图表 / 文字 / 结构）
   ↓
进入报告编辑器（左主体 + 右 Report Ora chat）
   ↓
HITL 协作 + 自动保存（详见 § 10.3.3）
```

##### 10.3.2.5 跨 Tab 引用机制

**Tab 1 报告中可引用 Tab 2 学员报告**作为附录 / 证据来源——

**引用形态**：
- ✅ **内容直接拷入**（recap 报告全文 / 对练报告全文 / 高光卡内容 等）
- ✅ **标注来源文件名**（如"引自：张三 · Course 1 横向协作 recap · 2026-04-15"）
- ❌ **不做超链接跳转**（拷入后该副本独立 / 学员报告原文修改不回流到 team report）

**触发方式**：
- 编辑器内 chat："引用张三的 Course 1 recap" → Ora 从 Tab 2 找该报告 → 内容拷入光标位置 + 自动加来源标注
- 划词后追问："这部分加入张三 recap 的相关段落作为佐证" → Ora 拉对应内容拷入

#### 10.3.3 报告编辑

##### 10.3.3.1 编辑界面布局

沿用 § 10.2.5 GUI 布局原则——**左主体编辑器 + 右 Report Ora chat 侧栏**。

```
┌──────┬──────────────────────────────────────┬──────────────┐
│      │  报告编辑器（主体 / 占 ~70-75%）        │              │
│ 左侧 │ ┌────────────────────────────────────┐ │  Report Ora │
│ 导航 │ │ [工具栏]  + 加图 / 导出 / 撤销 ...    │ │  chat 侧栏  │
│      │ ├────────────────────────────────────┤ │ (~25-30%)   │
│ Home │ │                                    │ │             │
│ Rpt  │ │   报告标题（管理员可改）              │ │  对话历史    │
│ Cfg  │ │   报告正文（WYSIWYG Markdown）       │ │  +          │
│ Msg  │ │   含图表 + 插图 + 跨 Tab 引用         │ │  输入框      │
│      │ │                                    │ │             │
└──────┴──────────────────────────────────────┴──────────────┘
```

**工具栏**（编辑器顶部）：
- ➕ **加图按钮**（触发 6 类图表选择器 / 详见 § 10.3.3.4 触发 B）
- 🖼️ **插图按钮**（本地上传 / 等同拖拽 / Ctrl+V）
- ↶ **撤销 / 重做**（沿用 § 10.3.3.6 Ctrl+Z 50-100 步）
- 📥 **导出菜单**（PDF / Word · 沿用 § 10.3.4）

##### 10.3.3.2 编辑器能力

**Markdown 模式**：**WYSIWYG（所见即所得）**——
- 类 Notion / Typora 实时渲染
- 管理员**不需要看 md 源码** / 适合非技术 HR
- 表格 / 标题 / 列表 / 加粗 / 链接 / 等基础 md 能力

**插图能力**：
| 来源 | 触发 |
|------|------|
| 本地上传 | 工具栏图片按钮 → 文件选择器 |
| 拖拽 | 拖图片到编辑区 → 自动插入光标位置 |
| 粘贴板 | 复制图片后 Ctrl+V → 自动插入 |

**跨 Tab 引用** = 详见 § 10.3.2.5。

**不支持**：
- ❌ 视频嵌入（本期不做 / 静态报告）
- ❌ 音频嵌入
- ❌ 复杂自定义 HTML 嵌入
- ❌ 实时协作光标 / 多人在线状态（沿用 § 10.3.5.2）

##### 10.3.3.3 6 类图表组件（ChatBI 模式）

**6 类清单**（仅做"文字替代不了的信息表达"图表）：

| # | 图表 | 形态 | 典型用法 | 复用关系 |
|---|------|------|---------|---------|
| 1 | **柱状图**（Bar 竖/横）| 单维度对比 | "对比 Course 1-5 完成率" / "Top 10 学员" | ❌ 新建 |
| 2 | **折线图**（Line）| X 时间 / Y 数值 / 单或多线 | "至今完成率走势" / "活跃度周变化" | ❌ 新建 |
| 3 | **表格**（Table）| 列可变 + 行动态 + 列点击排序 | "学员名单" / "Top N" / "Course 详情" | ✅ 复用 § 10.2.2.2 (1) 学员列表组件 + § 10.2.2.3 (1) 目录表格组件 |
| 4 | **堆叠柱状图**（Stacked Bar）| 每根柱内分段 | "Course 内 Activity 时长占比" | ✅ 复用 § 10.2.2.3 (2) 学习时长分布组件 |
| 5 | **饼图**（Pie）| 单一维度占比 | "活跃 / 风险 / 沉默 三态占比" | ❌ 新建 |
| 6 | **散点图**（Scatter）| X/Y 双维度相关性 | "学员投入时长 vs 完成率" / "互动量 vs 学习深度" | ❌ 新建 |

**4 类可复用 / 2 类新建** —— 研发量减半。

**砍掉**（与"文字替代不了"原则一致）：
- ❌ KPI 数字卡（"完成率 65%" 一句话即可）
- ❌ 进度条（文字可替代）
- ❌ 环形图 / 圆环图（与饼图重复）
- ❌ 雷达图（管理员引用 6 维画像可截图 / 本期不做实时生成 / 留 V0.6.0+）
- ❌ 热力图 / 漏斗 / 桑基 / 树状（过于独特 / 文字 + 表格 + 堆叠柱状已能表达）
- ❌ 仪表盘 Gauge / 时间轴 / 3D 图

**数据范围与生成原则**：
- ✅ 数据来自 Ora 6 维（§ 3.4.4.2）+ Home Ora 跨模块记忆（§ 10.2.4.3）
- ✅ 数据缺失 → "暂无数据"占位 + Ora 提醒
- ❌ 不引入外部信息（沿用 § 10.2.3.C.3 "不臆造外部"原则）

**写入即固化（snapshot 模式）**：
- 图表生成时拉的数据 = snapshot 写入报告
- **不实时刷新**（与 Home § 10.2.4.1 准实时机制不同）
- 管理员需明确指令"刷新数据" → Ora 重新拉 + 重新填入

##### 10.3.3.4 双触发方式

**触发 A · chat 直接触发**（一步到位 / 适合简单需求）：

```
管理员在 Report Ora chat："加一个 Course 完成率对比图"
   ↓
Report Ora 识别意图：
   - 图表类型 = 柱状图（横轴 Course / 纵轴完成率%）
   - 数据维度 = 项目数据 + 个人学习成果（Ora 6 维）
   - 拉数据 + 填充 + 标准化样式
   ↓
直接插入到编辑器光标位置
   ↓
管理员可继续追问："换排序" / "改类型" / "加标题"
```

**触发 B · 编辑器按钮触发**（多轮确认 5 步 / 适合复杂需求 / 适合不知道怎么描述的管理员）：

```
1. 编辑器工具栏点 "+ 加图" 按钮
   ↓
2. 弹图表类型选择器（6 类列表 + 缩略图预览）
   ↓
3. 选定类型 → 编辑区生成占位（如灰色框 + "图表占位 · 待 Ora 填充" 提示）
   ↓
4. 弹出"用户指令"输入窗口（管理员描述需求 / 数据范围 / 标题等）
   ↓
5. 提交 → Report Ora 接管 chat 区
   ↓
6. Report Ora 在 chat 区与管理员确认细节：
   - askUserQuestion 卡（"是要 Course 1-5 还是仅完整 Course？" / "纵轴用完成率% 还是已完成数？"）
   - 数据维度确认 / 排序确认 / 标题确认
   ↓
7. 管理员在 chat 中确认 "开始生成" → Ora 拉数据填入占位 → 占位变实图
   ↓
8. 管理员可继续在 chat 中追问"换排序" / "改类型" 等
```

**两种触发的差异**：
- **A 一步到位** = 适合管理员清楚自己要什么 / 描述准确
- **B 多轮确认** = 适合管理员描述不清 / 想先选类型再细化

##### 10.3.3.5 Report Ora 协作能力

Report Ora 在编辑器内**接管 chat 区**——与管理员协作完成 HITL 编辑。

**核心能力 3 类**：

**(1) vibe 写作**（描述意图 → Ora 生成整段）

```
管理员在 chat："写一段关于 Course 3 学员推进受阻的分析"
   ↓
Report Ora 拉数据 + 生成整段（200-500 字）
   ↓
插入到编辑器当前光标位置（管理员可手动调整位置 / 改文字）
```

**(2) 划词修改**（选中编辑器中的文字 → Ora 改）

划词后弹出菜单，**3 类修改选项 + 1 类自定义**：

| 选项 | 效果 |
|------|------|
| **扩写** | Ora 充实内容（加证据 / 加数据 / 加例子）|
| **缩写** | Ora 浓缩 / 简化 / 保留重点 |
| **划词后 chat 区输入指令** | 管理员选中文字 → 在 Report Ora chat 直接输入"改成更专业的语气" / "加一句对比同期数据" 等 → Ora 据此改 |

**(3) 数据范围**

Report Ora 可拉的数据 = **沿用 Home Ora 6 维（§ 3.4.4.2）+ Home Ora 跨模块记忆（§ 10.2.4.3）**：

| 数据来源 | 说明 |
|---------|------|
| Ora 6 维上下文 | 学员画像 / 学员聊天记录 / AOM 数据 / 项目数据 / 个人学习成果（含完成态 recap raw / 对练报告 raw / 高光库）/ 管理端业务数据（沿用 § 3.4.4.2）|
| Home Ora 历史对话 | 当前管理员在 Home Ora 的历次对话 → Report Ora 起底（避免重复对话 / 沿用 § 10.2.4.3）|
| Tab 2 学员报告内容 | recap 报告 raw + 对练报告 raw + 高光卡（用于跨 Tab 引用 § 10.3.2.5）|

**Report Ora vs Home Ora 差异**：

| 维度 | Home Ora | Report Ora |
|------|---------|-----------|
| 第一性目标 | 信息呈现 + 证据解读（让管理员发现） | 协助管理员产出对外可举证报告 |
| 主导动作 | 呈现 / 解读 / 钻取 | vibe 写作 / 划词修改 / ChatBI 图表生成 |
| 输出形态 | chat 内回答 + 4 类卡片 | 编辑器内文本 + 6 类图表 + chat 内确认对话 |
| 数据 snapshot | 准实时（§ 10.2.4.1）| 写入即固化 / 报告内 snapshot |
| 状态 | 跨会话累积记忆 | 跨会话累积记忆 + 共享 Home Ora 上下文 |

**Ora 整体共性**（HR 拍板边界 / 转述 + memory_id 追溯 / 不直接联系学员 / 像人 > 懂 HR）→ 沿用整体 Ora 原则，详见 § 10.1（待补 § 10.1.6 Ora 整体定位与双实例实现）/ § 3.3 Ora Soul / § 3.4.4 Ora 上下文范围。

##### 10.3.3.6 保存与撤销

**保存机制**：**纯自动保存 / 无完成状态**——
- 每次编辑后自动保存到管理员个人报告库（沿用 § 10.3.5.2 每管理员独立）
- 不区分草稿 / 完成 / 已导出（沿用 § 10.3.2.2 全部可编辑）
- 退出编辑器再进入 → 看到上次离开时的状态

**撤销机制**：**Ctrl+Z 编辑历史**——
- 支持 50-100 步撤销（业界标准：Microsoft Word ~100 / Google Docs 几百 / Notion ~100 / VS Code 默认无限——本期取**保守值 50-100**，由研发决定具体上限）
- **跨会话持久**——退出编辑器再进入 / 仍可继续 Ctrl+Z 撤销之前的操作（Tab 关闭后 history 应保留至少最近一次会话的范围，由研发决定持久化策略）

#### 10.3.4 报告导出

**支持格式**：
- **PDF**（适合对外交付 / 版式固定 / 不可编辑）
- **Word / docx**（适合二次编辑 / 企业常用）

**双入口**：

| 入口 | 位置 | 适用 |
|------|------|------|
| 编辑器内 | 工具栏导出按钮 | 当前正在编辑的报告 |
| 报告库列表 | Tab 1 行末导出 chevron | 任意历史报告快速导出 |

**关键规则**：
- 导出**与"完成"无关**——导出后报告**仍可继续编辑**（沿用 § 10.3.2.2 全部可编辑）
- 导出后**不锁定** / 管理员可再次进入编辑器修改
- 历史导出版本**不保存**（每次导出生成新文件 / 不留版本树 / 报告本身留 Ctrl+Z 历史）

#### 10.3.5 系统约束

##### 10.3.5.1 数据更新机制（snapshot 模式）

报告中心的数据时效与 § 10.2 Home 不同——

| 维度 | § 10.2 Home | § 10.3 报告中心 |
|------|------------|---------------|
| 数据时效 | 准实时（按小时定时 + 手动 / § 10.2.4.1）| **snapshot 写入即固化** |
| 触发刷新 | 自动定时 + 手动按钮 | **管理员明确指令** "刷新数据" → Ora 重新拉 + 重新填入相关图表 / 段落 |
| 跨会话 | 看板每次进 = 当前 snapshot | 编辑器进入 = **上次保存的状态**（已写入的数据不会自动刷新）|

**为什么 snapshot**：
- 报告 = 对外可举证的固化产出 / 不应在管理员阅读时数据"漂移"
- 一份"W4 中期报告"的数据应锁定在 W4 的 snapshot / 不能 W5 再看时数据变了
- 管理员若需更新某段数据 → 主动指令 / 不被动跟随

##### 10.3.5.2 多管理员场景

**每管理员独立报告库**——
- 每个管理员账号下的 Tab 1 报告完全独立（A 看不到 B 的报告）
- 同一项目下的 Tab 2 学员报告**共享**（HR / 项目运营 都能看 / 都能引用）

**不支持多人同编**——
- 同一报告同一时间**只能一个管理员编辑**（无锁机制 / 也不支持实时协作）
- 协同需求 → 管理员**导出 Word/PDF 后线下同步**

**为什么**：
- 简化研发复杂度（无 OT / CRDT / WebSocket 同步）
- HR / 项目运营 各自独立汇报场景占多数
- 协同场景占少数 / 用导出工具兜底

##### 10.3.5.3 跨模块衔接

**从 Home 跳入**：
- Home Ora 当前对话上下文 + 管理员关注的数据切片 通过双 Agent 信息共享传给 Report Ora（沿用 § 10.2.4.3）
- Report Ora 接力时可直接引用管理员在 Home 提的问题 / 看的数据，避免重复对话

**与 § 10.4 项目配置 / § 10.5 消息中心**：
- 本期不联动（§ 10.4 / § 10.5 是纯 GUI 模块 / 无 Ora）
- 未来 milestone 自动触发实现时，会读 Config 模板 + 推 Message 通知

**与 § 9.3.3 学员侧 recap 报告**：
- 完成态 recap raw 进 Tab 2 学员报告（沿用 § 9.3.3.2.10 / § 3.4.4 管理端可见性条）
- 非完成态 recap **不进** Tab 2（红线一致）

### 10.5 消息中心 Message

#### 10.5.1 业务定义

#### 10.5.2 GUI 与交互

| 子模块 | 功能 |
|-------|------|
| **手动定向发送** | 项目运营从学员名单中**手动选人** + 编辑富文本 + 发送 |
