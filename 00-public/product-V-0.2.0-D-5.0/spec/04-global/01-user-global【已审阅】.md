## 7 · 用户级全局功能

> **本章定位**：跨学员端 / 管理端 / 整个平台共用的、与具体场域无关的功能模块。本章只描述需求（用户能做什么 / 系统提供什么），不描述 UI 实现 / 不卷布局细节。
>
> **基础**：本章基于 § 7.1 + § 7.2 共 100 题 PM 决策落地。


> 用户在产品中能做什么——账号、登录、设置、消息、帮助 等用户日常会用到的全局能力。

### 7.1 账号体系

##### 三类账号 · 同一套用户表 + 多角色加载

睿学采用**同一套用户表 + 多角色加载**模型——一个 user 可在不同 project 下被赋予不同 role：

```
user × project → role（学员 / 管理员 / admin · 多选）
```

**实体关系**：

- 同邮箱可在多个项目中作为学员（跨项目）
- 同邮箱可在项目 A 是学员、项目 B 是管理员（跨角色）
- 同一项目内同一人员可同时拥有学员 + 管理员双角色（详见 § 6.5）

##### 账号绑定层级（核心 → 可选）

| 层级 | 标识 | 必有 | 备注 |
|------|------|------|------|
| **核心** | 账号 + 密码 | ✅ 必有 | admin 账号本期就这一层 |
| **次级** | 邮箱（可绑定）| 推荐 | 学员推荐 |
| **顶层** | 手机（可绑定）| 可选 | 未来扩展 |

> **未来实现主要靠 GPB**（绚星统一账号体系）—— 本期研发期评估 GPB 支持情况。

##### 三类账号生命周期

| 账号 | 创建时机 | 生命周期 |
|------|---------|---------|
| **admin 账号** | 阶段 3 签单 / 产品运营端创建 | 项目周期 + 服务周期，归档时 disabled |
| **管理员账号** | 阶段 4 开营 / admin 创建 | 同 admin |
| **学员账号** | 阶段 4 开营 / admin 批量导入 | 同 admin |

详见 § 6.4 / § 6.5。

### 7.2 登录

##### 登录流程

- **同一项目统一登录页** —— 同一个项目内的学员 / 管理员 / admin 共用一个登录页（不分端口入口）；不同项目各有独立登录页
- **登录页元素可按项目个性化定制** —— Logo / 品牌色 / 欢迎语 等元素跟随该项目的 program config 平台个性化配置（详见 § 6.6.6）
- 登录字段：账号（邮箱 / 手机 / 自定义 ID）+ 密码
- **本期不做验证码**

##### 登录后路由

登录验证通过后，系统按以下流程决定路由 + 端口切换按钮可见性：

1. **查询该账号在本项目中的角色**（学员 / 管理员 / admin 中的任一 / 多个）
2. **路由决策**：
   - 仅学员角色 → 直接进学员端
   - 仅管理员角色 / admin 角色 → 直接进**管理端**（admin 走管理端，与管理员同 / 不进产品运营端）
   - 多角色（学员 + 管理员 / admin 等）→ 默认进**学员端**（学习端为主体，详见 § 7.5）
3. **端口切换按钮可见性**：
   - **仅一个角色** → 头像下拉的"切换端口"选项**置灰不可点**
   - **多角色** → "切换端口"可点

##### 首次登录

- **不强制改密**——admin 导入时可设置初始密码，学员可选改

##### 密码 / 锁定 / 找回

| 项目 | 决策 |
|------|------|
| 密码强度规则 | **按 GPB 规则** |
| 登录失败锁定 | **按 GPB 默认策略** |
| 找回密码 | **GPB 邮件重置链接** |

##### 记住我 / 多设备

- **支持"记住我"** / 默认不勾选
- **同账号多设备同时会话允许** / 不限制

##### 异常登录

- 异常登录提醒**本期不做** / 靠 GPB 补位
- 账号被 disabled 后登录显示 **"账密不匹配"** 提示（避免泄露状态信息）

### 7.3 个人设置

##### 两类子模块严格区分 ★

| 子模块 | 内容 | 是否可改 | **本期实现归属** |
|--------|------|---------|----------------|
| **个人设置** | 账号 / 密码 / 头像 / 邮箱手机绑定 | 学员可改密码 + 头像 + 绑定 / 显示名邮箱由 admin 决定 | **主要由 GPB 负责** ★（依赖绚星统一账号体系，详见 § 7.1 / § 7.2）|
| **个人偏好设置** | Neo 形象偏好（数字人 gif + TTS 音色）/ AI 语速 / 页面风格（未来）| 学员个人选 | NeoLearning 自研 |

##### 学员侧字段清单

**个人设置**：

- 头像（**支持上传** / 默认按姓氏生成，参考 Google 做法）
- 密码（可改）
- 邮箱绑定（可绑可解）
- 手机绑定（可绑可解）

**个人偏好**：

- **Neo 形象偏好**（参考豆包做法）—— 学员个人选 Neo 的视觉与音色组合：
  - **数字人形象 gif**（多个候选，由内容侧 / 设计侧提供候选库）
  - **TTS 音色**（多种音色候选，男 / 女 / 不同情感色彩等）
  - **仅影响 Neo 形象与音色，不影响 Neo 风格 / 内容**（Neo Persona 与 voice 一致性，详见 § 3.2.2）
- **AI 语速**（慢 / 中 / 快，默认中 / **同时影响 TTS 声音速度 + 文本流式速度**）
- **主题切换**（**明亮模式** / **黑夜模式**，默认明亮 / **个人级**——仅影响当前学员的所有端口页面 / 与 § 6.6.6 项目级"平台个性化"独立 / 切换后实时预览 / 与其他偏好同走"手动保存"规则）

##### 管理员侧字段清单

**个人设置**：头像 / 显示名 / 密码（可改）/ 邮箱（不可改）

**个人偏好**：

- **主题切换**（**明亮模式** / **黑夜模式**，默认明亮 / **个人级** / 切换后实时预览 / 与其他偏好同走"手动保存"规则）

> 说明：管理端仅 Ora / Ora 无形象 / 不用 TTS，故无 Neo 形象偏好与 AI 语速；本期管理端个人偏好仅含主题切换。

##### 设置生效与确认

- **手动保存**——需点击"保存"后生效（不立即生效）
- **仅账号设置需二次确认**（密码 / 解绑邮箱 / 解绑手机）/ 偏好设置不需

##### 设置入口

头像下拉 → "设置"

##### 本期不做

- ❌ **字体定制**（之后也不做）
- ❌ **通知偏好**（仅站内不需要）
- ❌ **语言切换**（本期仅中文）

### 7.4 站内消息

##### 消息入口

- **学员端**：大厅顶部 bell icon + 未读红点（详见 § 9.2）
- **管理端**：消息中心入口（详见 § 10.5）

##### 消息体系

- **同一体系按接收人区分** —— 一套 message 表，每条指定 recipient_id
- **消息保留时长**：项目服务周期内不清理

##### 消息分类（三大类，跨学员端 / 管理端 统一） ★

| 大类 | 来源 | 学员端示例 | 管理端示例 |
|------|------|-----------|-----------|
| **平台消息** | 平台层产品通知 | "新功能上线" / "维护通知" | "新版本上线" / "服务到期前 1 月提醒"（详见 § 6.8）|
| **系统消息** | 系统自动触发的业务事件 | 系统催学（详见 § 6.6.5）/ 里程碑提醒（学员侧产品化推送，如"项目过半啦"） | 里程碑提醒（前 3 天每日 + 当天 team report 草稿就绪，详见 § 6.6.3）/ 异常学员提醒（沿用催学规则 § 10.4.2.4）|
| **用户消息** | 用户主动下发 | 管理员向学员手动下发的消息 | **本期不做学员→管理员反向**（学员无法发消息给管理员）|

##### 关于学员端"系统催学" vs "里程碑提醒"

- 两者**学员视角无需区分**——学员看到的就是"系统给我推送的提醒"，统一归"系统消息"
- 系统催学 = 基于催学规则触发
- 里程碑提醒 = 基于里程碑到期触发
- 后台触发逻辑不同，但学员端展示形态一致

##### 关于管理端 Ora 与站内消息的关系

- **Ora 不直接推送站内消息** —— Ora 在管理端通过看板被动呈现 + 问答主动对话两种方式互动（详见 § 3.3.3 与管理员的交互场景；具体管理端表现待 § 10 落定）
- 里程碑当天 team report 草稿生成后，**由系统**（不是 Ora）以"系统消息"形式通知管理员"team report 草稿已生成"
- 管理员需要查看 team report 时，从系统消息跳转到 team report 页（Ora 在该页内部对话）

##### 关于"用户消息"本期 scope

- ✅ **管理员向学员下发**（手动选人 + 富文本）
- ❌ **学员 → 管理员反向**——学员无法发消息给管理员
- ❌ **学员之间消息**——学员之间默认隔离（详见 § 3.4.3）

##### 消息中心交互（需求层面）

- **不做搜索筛选** / 时间倒排 / 默认 20 条 / 折叠 + 加载更多 20 条
- **不做星标 / 收藏**
- **未读判定**：点击清空未读 或 点开具体消息（任一即标记已读）
- **不分 tab** / 一个列表中区分图标
- 各分类消息有对应跳转路径

##### 消息推送

- **WebSocket 实时**——是产品起点
- **未读状态服务端持久化** —— 关闭浏览器下次登录仍可见
- **本期渠道仅站内**——邮件 / IM / SMS 全部本期不做

##### 手动发送（运营侧）

- 支持**多人同发**（全选 / 多选名单）
- 支持**富文本**（图片 + 链接）
- **本期不做撤回**

##### 消息日志

- **完整记录**：谁收到 / 是否查看 / 是否点击链接
- **本期不支持学员回应**
- 查看权限：admin / HR / 项目运营 都能看

### 7.5 端口切换 / 退出登录

##### 端口切换

- 入口：头像下拉菜单 → "切换到学员端" / "切换到管理端"
- **账号只有一个角色时，切换端口选项置灰**
- **不需要重新登录**——一次登录管两端
- 多角色用户登录默认进 **学员端**（学习端为主体）
- 同浏览器同时一个端口活跃

##### 退出登录

- 头像下拉 → "退出"
- 退出后**清除登录状态 / 保留偏好**（AI 语速 / Hero 选择 / 账号记忆 等保留）
- 退出后**不返回上次页面** / 下次登录进默认首页
- 多浏览器同账号登录场景：**各浏览器独立退出**（不同步）

##### 会话 / 超时

- **不超时**——用户不主动退出不会被踢出
- 异常登录提醒本期不做（详见 § 7.2）

> **术语约定**：本文档不使用"session"一词（避免与 Agent 多对话混淆），统一用"会话"或"登录状态"。

### 7.6 帮助

##### 产品手册

- 学员手册 / 管理员手册 **各自独立两个页面**
- 头像下拉 → "帮助" → 打开新窗口
- **markdown 文档** / 内部维护更新 / 本期不做复杂托管平台
- 内容范围：**仅产品使用说明 + FAQ**

##### Neo 不负责帮助

- 学员**不能在产品内"问 Neo"**作为帮助渠道
- Neo 仅负责辅导（详见 § 2 / § 3.2）

##### 反馈

- **复用原 XUI 在 Agent 消息下的反馈点赞 / 点踩功能** —— 学员 / 管理员 在每条 Agent 消息（Neo / Ora / Actor 输出）下可点 👍 / 👎，反馈具体某条 AI 输出的质量
- 点赞 / 点踩数据回流用于**调教 Agent prompt**（仅限 NeoLearning）
- 不做独立的"反馈入口"或 in-app feedback widget
- 整体反馈渠道（产品 bug / 建议 / 客户层反馈）走外部：客户成功 / 销售反馈群

##### 紧急问题

- 本期不设产品内紧急通道
- 链路：admin → 客成 → 产品运营
