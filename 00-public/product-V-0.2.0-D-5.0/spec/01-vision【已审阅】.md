## 1 · 愿景

> 本章定义产品愿景——一句话定位 / 解决什么问题 / 三方价值 / 关键设计承诺 / 是什么不是什么 / 当前不做 / 内容示例约定。

> **术语速查**（本章高频术语 / 完整术语列表见 [02-foundation/04-data-model § 1.2.1]）：
> - **睿学 / NeoLearning**：产品名 / 面向企业的 AI-Native 培训交付平台
> - **睿学培训项目 / NeoLearningProgram**：本期交付项目
> - **Neo**：学员端唯一 AI（1v1 老师 AI 孪生）
> - **Ora**：管理端 AI（HR 的 BI 分析师 + 项目顾问）
> - **3 学习场域**：lecture（授课）/ practice（对练）/ recap（小结）—— 大厅是中枢，不算场域
> - **KGP**：Knowledge Generation Platform / 知识生产平台
> - **Course Pack**：KGP 在 Project 级提供的内容包（本期示例 = 基层管理者能力发展）

### 1.1 一句话定位

**睿学（NeoLearning）** 是面向**企业培训**的 **AI-Native 学习陪伴平台**。本期讨论交付的是**睿学培训项目（NeoLearningProgram）**——在 6 个月项目周期里，通过 1v1 老师 Neo 跨大厅 + 3 个学习场域持续陪伴学员，把学习从"出勤 + 完课"升级为"行为改变 + 可举证 ROI"。

### 1.2 解决什么问题

企业培训行业面对一个本质矛盾：**HR / 培训部门用"学了么"作为成功指标（出勤率 / 完课率 / 考试分数），但业务部门要的答案是"会了么"（能不能在真实工作中做到）**。

这两件事不等价：

- **学了** = 接收到信息、记得住、能选对——指向"知道"
- **会了** = 在真实工作场景中能做到、形成习惯、迁移应用——指向"行为改变"

传统培训的失败不只是"学完就忘"那么简单——而是把"学了"和"会了"混淆。以为完课率高就等于培训有效，但 6 个月后业务部门看到的常常是"培训完该不会还是不会"。

由此衍生出三层连锁问题：

1. **混淆"学了"vs"会了"** · 培训 KPI 停在出勤 / 完课 / 考试，看不到行为层
2. **无法举证 ROI** · HR 拿不出"学员行为真的改变了"的证据，向业务部门交代时只能凭主观感受
3. **1 对多无法个性化** · 讲师有限 / 学员千差万别，无法给每个学员匹配进度、节奏、案例

### 1.3 三方价值

| 角色 | 价值承诺 |
|------|---------|
| **学员** | 1v1 老师 Neo 在 6 个月项目周期里持续陪伴，跨大厅 + 3 个学习场域真正掌握课程内容，并迁移到真实工作行为（不止"学了"，而是"会了"）|
| **企业 / HR** | 基于全过程行为数据产出**可举证的行为改变报告**，把培训从"成本中心"升级为"价值证据中心" |
| **项目运营**（客户 / 服务侧）| 系统自动执行可配置的催学规则，Ora 解读数据，单项目运营人力 ≤ 传统项目 50% |

> **关于"三方价值"**：
> - **HR 与项目运营 = 同一组人**（共用管理端 / 协同工作）—— 上表保留三方独立的"价值视角"区分（HR 看战略价值 / 项目运营看运营效率）；实际人员组织上是一体，冲突时以 HR 视角为准
> - **项目运营 vs 产品运营**：项目运营 = 派到具体项目的运营角色（客户侧 / 与 HR 协作）；产品运营 = 睿学平台侧的产品级运营（供应商侧）/ 详见 [02-foundation/03-roles-and-ports § 1.4]

### 1.4 关键设计承诺

睿学不是 e-Learning + ChatBot 的简单组合——它由三个底层原则支撑（详见 [02-foundation/02-methodology § 1]、[02-foundation/01-personas § 1]）：

- **第一性原理** · Neo = 1v1 老师的 AI 孪生，不是聊天伙伴 / 不是百科全书 / 不是应声虫 → [02-foundation/02-methodology § 1]
- **懂你** · Neo 越用越懂学员，跨场域 / 跨课程 / 跨会话 都记得，不重置 → [02-foundation/01-personas § 1.4.1]
- **像人** · 每个 Agent 有自己的画像 + 立场 + voice，绝不无理由表扬，错时直接指出 → [02-foundation/01-personas § 1.4.2]

### 1.5 是什么 · 不是什么

| 维度 | 是 | 不是 |
|------|----|------|
| 形态 | Neo 1v1 陪伴 | 视频库 + Quiz 任务集合 |
| 成功指标 | "会了么"——行为可追溯 | "学了么"——出勤 / 完课率 |
| 周期 | 6 个月持续 + 跨场域记忆 | 单次集训 + 课后断联 |
| 个性化 | 按学员差异化进度与内容 | 统一进度一刀切 |
| 数据 | 可举证的行为改变报告 | 依赖讲师主观评价 |
| 范式 | AI-Native 重构 | e-Learning 工具升级 |

### 1.6 当前不做（边界声明）

- **不做考证 / 学历教育**——不是职业资格证书 / 学位培训平台
- **不做 K12 / 通用 To C**——只面向企业 B 端培训，不做学生 / 个人学习者
- **本期不做订阅 / 长期陪伴**——以项目服务周期为单位，最长 12 个月；是否扩展到长期陪伴形态、订阅制，由后续用户反馈决定

本文后续涉及具体内容主题的章节（如 [02-foundation/04-data-model § 1.5] Course 清单 / [03-lifecycle/01-pre-learning § 1.3] 试运行体验 / [03-lifecycle/01-pre-learning § 1.5]-[03-lifecycle/02-learning § 1] 学习期描述等），统一以**基层管理者能力发展**作为示例 Course Pack，便于上下文一致、读者建立具体场景感。

- **平台层面（睿学 NeoLearning）支持任意企业培训主题**——销售 / 客服 / 中层管理 / 技术能力 / 文化价值观 / 等
- Course Pack 清单与优先级由内容侧（KGP）另行规划，不属于本期产品 spec 范围
- 任何"基层管理者"出现的地方都是**内容层示例**，不是**产品层承诺**
