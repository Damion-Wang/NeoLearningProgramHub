## 用户级 vs 平台级 边界

| 维度 | 用户级（01-user-global） | 平台级（02-platform-global） |
|---|---|---|
| **关注点** | 用户能做什么 | 系统在什么条件下运行 |
| **示例** | 个人设置改密码 / 切端口 / 改主题 | 浏览器支持 / 时区 / 数据保留策略 |
| **变更影响** | 单用户 | 全平台 |
| **谁配置** | 学员/管理员个人 | NeoLearning 平台侧 |

## 推荐阅读顺序

1. **01-user-global** —— 先看用户怎么用（账号 + 登录 + 设置 + 消息 + 帮助）
2. **02-platform-global** —— 再看平台底线（合规 + 安全 + 数据策略）

## 重要约定

- **用户级**：个人设置 vs 个人偏好严格区分（前者 GPB 负责 / 后者 NeoLearning 自研）
- **平台级**：仅遵中国个人信息保护法 / AIGC 规范 / GDPR 等本期不考虑
- **会话/登录状态**术语约定：本文档不使用 "session" 一词（避免与 Agent 多对话混淆）
- **大模型安全声明**："以上内容由 NeoLearning AI 生成，仅供参考"——所有有 AI 生成内容的地方都要有
