# 管理端首页（Operation）

## 模块定位

首页是管理员对学员学习情况的统筹了解中心——涵盖整体概览、学员对比、个人详情、个人报告汇总和项目进度。Ora 在右栏提供数据解读。

---

## 功能区域

| 区域 | 内容 |
|------|------|
| **常显导航** | 首页（当前）/ 报告 / 消息 / 配置 |
| **左栏：看板区**（Tab/子导航切换） | 项目维度（默认）、人员维度、内容维度 |
| **右栏：Ora 对话** | 自然语言查询 + 数据解读 + 追溯钻取 |

---

## Ora 在首页的角色定位

### Ora 是谁

**前台名称**：Ora
**角色类型**：管理端 Agent，独立于学员端的 Neo
**数据来源**：统一 Database（全局视角，可读取所有学员的 Neo 全场域对话原文——大厅辅导+授课+对练+调研+报告）

### Ora 在首页的核心价值

1. **数据解读**——管理员点击任何数据，Ora 自动解读背后的原因和趋势，不需要管理员自己分析
2. **深度追问**——管理员可以追问任何维度的问题，Ora 支持逐层钻取到原始对话记录
3. **对比洞察**——对比学员之间、Course 之间、时间段之间的差异，找出规律和异常
4. **综合评估**——在管理员需要时给出项目整体健康度评估和风险提示

### Ora 在首页的核心能力

| 能力 | 说明 |
|------|------|
| **自然语言查询** | 管理员用自然语言问任何数据问题，Ora 回答并标注数据更新时间 |
| **图表联动解读（P3）** | 管理员点击左栏图表元素，Ora 自动解读该数据点背后的行为证据 |
| **对比分析** | 对比学员、对比 Course、对比时间段，发现差异和规律 |
| **异常深挖** | 发现数据异常时，从对话记录中分析可能原因 |
| **证据追溯** | 从结论→证据摘要→原始对话，逐层钻取 |
| **共享记忆** | 首页和报告页共享对话上下文，切到报告页时 Ora 仍然记得首页的讨论 |

### Ora 在首页的行为边界（能做 vs 不能做）

| 能做 | 不能做 |
|------|--------|
| 回答数据问题，标注数据来源和时间 | 编造不存在的数据或无依据的推测 |
| 解读趋势，给出可能原因（从对话记录中感知） | 对学员做价值判断（"这个学员不行"） |
| 追溯到原始对话记录（Neo 全场域对话原文——大厅+授课+对练+调研+报告，本期不做脱敏，直接展示原话） | — |
| 对比分析，发现规律和异常 | 自行决策或代替管理员做判断（"你应该开除这个人"） |
| 综合评估项目健康度和风险 | 给出超出数据支撑的过度推断 |
| 坦诚告知无法回答的问题 | 猜测或编造答案 |

## 首页子功能（三个维度）

首页从三个维度统计当前项目进行状态：项目维度、人员维度、内容维度。通过 Tab/子导航切换。

### 1. 项目维度（默认页）

从项目管理视角看整体状态。

| 数据项 | 内容 |
|--------|------|
| 项目进度 | 已进行天数 / 剩余天数 / 整体完成率 |
| 学员活跃度 | 上周平均活跃天数/人，环比变化 |
| 学员概览 | 学员总数、活跃人数、进度分布 |
| **日活热力图** | 以日为单位展示每天的活跃学员数，颜色深浅代表活跃程度，快速看出哪些天学员参与度高/低 |
| **学习投入度热力图** | 以日为单位展示每天的学习互动量（底层为 token 消耗），颜色深浅代表投入程度，反映整体学习互动强度 |

### 2. 人员维度

从学员个体视角看每个人的学习状态。

- **学员列表**：含进度百分比、最后登录时间、活跃状态
- **个人详情**：点击某学员 → 展开详情面板（Activity 级进度、学习时长）
- **个人报告汇总**：从详情可跳转查看该学员的个人学习报告（与学员端同一份报告），管理员可查看学员端不可见的排名信息
- 所有学员的个人报告在此统一查看和管理

### 3. 内容维度

从课程内容视角看各模块的完成情况。

- 各 Course 完成率柱状图
- 各 Activity 完成率明细表（Activity 级粒度）
- 活跃度趋势折线图（按周）
- 所有图表元素可点击 → 右栏 Ora 自动解读

> 被动信号数据（停留时长/离开率/回翻率）本期不在管理端展示，作为后台数据埋点用于产品迭代分析。

## 已确认决策

- 首页从三个维度统计项目状态：项目维度（默认）/ 人员维度 / 内容维度
- 项目维度含两个热力图：日活热力图 + 学习投入度热力图（底层为 token 消耗）
- 人员维度含个人报告汇总，管理员可查看排名信息（学员端不可见）
- 不含催学管理（催学规则在配置模块）
- Ora 负责数据解读，首页和报告页共享记忆
- 数据粒度为 Activity 级，不做 SCO 级
- 数据呈现为主，本期不做智能预警评价
- 30-40 人规模暂无学员分组/部门划分

---

## 本期不做的功能

- 智能预警评价——先呈现数据，管理员自行判断
- 学员分组分析——暂无分组需求

## 反向补充：学员状态定义标准

学员列表的"活跃/风险/沉默"状态判定标准：

| 状态 | 判定 | 视觉 | 含义 |
|------|------|------|------|
| 活跃 | 3 天内有登录 | emerald chip | 正常学习 |
| 风险 | 3-7 天未登录 | amber chip | 需要关注（可能发催学消息） |
| 沉默 | 超过 7 天未登录 | rose chip | 严重流失风险（需人工介入） |

**关键**：状态与**学习进度无关**，只看**登录活跃度**。理由：进度慢可以是"慢学"，不等于不学；登录活跃度更反映参与意愿。

**Hover 提示**：鼠标悬停状态 chip 显示判定标准 tooltip。

## 反向补充：管理员查看学员个人报告

人员维度学员列表中"[查看报告]"链接的跳转行为：

- 进入该学员的**个人学习报告只读视图**
- 内容与学员端完全一致（Part 1/2/3 + 高光 + 证据）
