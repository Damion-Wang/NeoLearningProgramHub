> **模块代号**：learner/07-discovery-library
> **所属层级**：学员端 · 大厅 · 四库之一
> **版本**：v0.4.0 初稿
> **最后更新**：2026-04-22

---

## 1. 模块定位

Discovery Library（发现库）是学员端的**AI 主动识别 + 学员确认收藏**机制，用于沉淀学员在学习旅程中被 Neo 捕捉到的「亮眼表现」。

它是学员端「四库体系」中唯一由 AI 主导发现、学员被动审核的库，区别于其他三库的**用户主动性**或**系统自动归档**逻辑。

### 1.1 与其他库的区别

| 库名 | 发起方 | 内容来源 | 用户动作 | 核心价值 |
|---|---|---|---|---|
| **Tool Library**（工具库） | 系统 | 固定方法论、框架、模板（如 GROW、SBI） | 查阅、引用 | 知识资产 |
| **Note Library**（笔记库） | 学员 | 学员主动记录的心得、片段 | 创建、编辑、标签 | 个人学习轨迹 |
| **Discovery Library**（发现库） | **Neo（AI）** | Neo 识别的学员高光行为 | **确认收藏 / 忽略** | 自我认知强化 |
| **Result Library**（成果库） | 系统 | 完成的 Activity 报告、评估结果 | 查阅、分享 | 成果归档 |

### 1.2 核心理念

> **"你做得比你以为的更好"** —— Discovery Library 的本质，是让 AI 帮学员发现自己看不到的闪光点。

- 学员常常**低估**自己的表现（尤其是过程性的细节能力）
- Neo 作为全程观察者，拥有跨 Activity 的全局视角
- 通过「发现 → 推送 → 确认 → 沉淀」的闭环，形成**学员自我画像的正向反馈**

## 2. 触发时机

Neo 在以下场景下启动「高光识别」流程：

### 2.1 对练场域（Practice Zone）

- **触发点**：Phase 3 反思复盘结束时
- **识别粒度**：单轮对练内的关键行为（如第 N 轮的策略切换）
- **典型高光**：
  - 在僵局中主动切换提问策略
  - 对下属情绪的准确捕捉与回应
  - 反思环节中的自我归因与改进意愿

### 2.2 授课场域（Lecture Zone）

- **触发点**：SCO 内实时检测（学员答题 / 提问 / 笔记行为）
- **识别粒度**：SCO 级别的理解深度
- **典型高光**：
  - 概念迁移（将当前 SCO 的知识点应用到自己工作场景）
  - 举一反三（主动提出未讲到的延伸问题）
  - 深度笔记（笔记中出现超出讲义的个人洞察）

### 2.3 报告场域（Report Zone）

- **触发点**：Neo 解读报告时引用历史高光作为证据
- **识别粒度**：跨 Activity 的表现归纳
- **典型高光**：
  - 在报告 Part 2「高光表现」区块中，每条证据都可点击跳转到 Discovery Library 的原始条目
  - Neo 主动将报告中的某个结论「回写」为一条 Discovery

### 2.4 跨 Activity 模式识别

- **触发点**：后台定期扫描（每日 / 每次 Activity 结束）
- **识别粒度**：跨 3+ Activity 的连续性行为
- **典型高光**：
  - 连续 3 次对练都使用了 GROW 框架的 R 步骤，说明已内化
  - 近 2 周笔记中反复出现「心理安全」关键词，说明关注点聚焦
  - 不同场域都展现出「倾听 → 复述 → 确认」的稳定习惯

## 3. 识别规则 · Neo 检测的高光类型

Neo 按以下 5 类维度识别高光行为。每类有明确的**触发条件**和**判定阈值**（由后台 LLM + 规则引擎协同实现）。

### 3.1 策略切换（Strategy Shift）

- **定义**：在困境 / 僵局 / 失败尝试后，主动切换到另一种方法
- **典型案例**：对练第 3 轮提问无效后，第 4 轮切换到「自我归因」路径，成功突破下属防御
- **判定条件**：
  - 同一 Phase 内，连续 2 轮同类尝试后出现方法变化
  - 切换后的效果指标（如下属开放度）显著提升

### 3.2 情绪感知（Emotional Awareness）

- **定义**：识别自己或他人的情绪变化，并作出恰当回应
- **典型案例**：注意到下属语气中的迟疑，暂停推进，转为共情倾听
- **判定条件**：
  - 对话中出现情绪相关词汇（"我感受到…""你现在是不是…"）
  - 情绪识别后伴随策略调整

### 3.3 概念迁移（Concept Transfer）

- **定义**：将学到的技巧 / 框架应用到新场景
- **典型案例**：授课中学到 SBI 反馈框架，在对练中主动使用
- **判定条件**：
  - Activity A 学习的概念，在 Activity B 中被正确调用
  - 调用时机与场景契合

### 3.4 反思深度（Reflection Depth）

- **定义**：主动复盘 + 提出具体改进方案
- **典型案例**：反思复盘时说「我意识到第 2 轮的问题在于过早给建议，下次应该多问一轮」
- **判定条件**：
  - 反思内容包含「原因分析 + 改进动作」两个要素
  - 改进动作具体可执行（非泛泛而谈）

### 3.5 细节捕捉（Detail Capture）

- **定义**：注意到对话 / 材料中的关键信号
- **典型案例**：在对练中捕捉到下属「加班太多」的一次性陈述，主动回应
- **判定条件**：
  - 对话中捕捉到非显性信号
  - 捕捉后的处理方式体现对信号重要性的判断

## 4. 卡片交互设计

### 4.1 推送入口：Chat 面板卡片

Neo 在对应场域的 Chat 面板中推送一条**Discovery 卡片**，视觉上与普通消息明显区分。

#### 视觉规范

| 元素 | 规范 |
|---|---|
| 背景色 | Amber 色调（与 Note 的 cyan、Tool 的 violet 区分） |
| 左侧 Icon | ✨（Sparkle 图标，表示"发现"） |
| 卡片标题 | Neo 生成的 8-16 字高光标题 |
| 描述文本 | Neo 生成的 30-60 字描述（包含场景 + 行为 + 效果） |
| 来源标签 | "来自：GROW 辅导对练 · 第 4 轮" |
| 日期 | 相对时间（如"刚刚""3 分钟前"） |
| 主按钮 | **[✓ 收藏到 Discovery 库]**（Amber 主色，实心） |
| 次按钮 | **[忽略]**（淡灰色文字按钮） |

#### 卡片示例

```
┌───────────────────────────────────────┐
│ ✨ 自我归因策略的灵活运用                │
│                                       │
│ 你在第 4 轮主动切换到自我归因提问，     │
│ 有效突破了下属的防御心理。             │
│                                       │
│ 来自：GROW 辅导对练 · 第 4 轮 · 刚刚   │
│                                       │
│ [✓ 收藏到 Discovery 库]   [忽略]       │
└───────────────────────────────────────┘
```

### 4.2 确认动作

#### 收藏（✓ 主按钮）

- 卡片平滑收起（300ms 动画）
- 右下角出现 toast：**"已收藏到 Discovery 库"**（含小链接"查看"，点击跳转到大厅 Discovery 库）
- 后台记录 `acceptedAt` 时间戳
- Chat 中保留一条精简记录：「✨ 已收藏：自我归因策略的灵活运用」

#### 忽略（次按钮）

- 卡片直接消失（无 toast）
- 后台记录该条 Discovery 被拒绝（用于优化识别算法，但**不在 UI 任何地方向学员暴露**）
- Neo 不会重复推送同一条 Discovery

### 4.3 补救动作

- 若学员**误点忽略**，可在大厅「最近忽略的发现」中找回（保留 7 天）
  - 该入口默认折叠，不打扰

## 5. 数据结构

```json
{
  "id": "d1714000000000",
  "title": "自我归因策略的灵活运用",
  "description": "你在第 4 轮切换到自我归因提问，有效突破下属防御心理",
  "source": {
    "activityId": "A4",
    "activityName": "GROW 辅导对练",
    "zone": "practice",
    "roundOrSCO": 4
  },
  "discoveredAt": "2026-04-22T10:15:00+08:00",
  "acceptedAt": "2026-04-22T10:16:32+08:00",
  "status": "accepted",
  "tags": ["策略切换", "心理安全", "GROW"],
  "evidence": {
    "dialogueSnippet": "学员：听你这么说，是不是最近你自己也觉得没达到心里的标准？\n下属：……其实是的，我最近有点迷茫。",
    "neoAnalysis": "学员敏锐识别到下属的防御姿态后，主动切换到自我归因提问路径，让对方有了自我表达的空间，这是 GROW 框架 R 阶段的经典应用。"
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | `d` + 时间戳毫秒 |
| `title` | string | Neo 生成，8-16 字 |
| `description` | string | Neo 生成，30-60 字 |
| `source.activityId` | string | 来源 Activity ID |
| `source.activityName` | string | Activity 可读名称 |
| `source.zone` | enum | `lecture` / `practice` / `inquiry` / `report` |
| `source.roundOrSCO` | number | 对练轮次或 SCO 编号 |
| `discoveredAt` | ISO timestamp | Neo 识别到的时间 |
| `acceptedAt` | ISO timestamp | 学员确认收藏的时间（忽略则为 null） |
| `status` | enum | `pending` / `accepted` / `ignored` |
| `tags` | string[] | 自动标签（Neo 生成）+ 学员可后续编辑 |
| `evidence.dialogueSnippet` | string | 对话原文（对练场景） |
| `evidence.neoAnalysis` | string | Neo 的分析解读 |

## 6. 大厅展示

### 6.1 入口位置

大厅「资源中心」区块中，四库以 **2×2 网格**展示：

```
┌──────────────┬──────────────┐
│ Tool Library │ Note Library │
│  工具库       │  笔记库       │
├──────────────┼──────────────┤
│ Discovery ✨ │ Result 库    │
│  发现库       │  成果库       │
└──────────────┴──────────────┘
```

Discovery Library 卡片上显示：
- 图标：✨（Sparkle，Amber 色）
- 标题：Discovery Library · 发现库
- 副标题：`12 条高光` / `3 条本周新增`（动态计数）

### 6.2 列表页

点击进入 Discovery Library 详情页：

#### 顶部区域
- 页面标题：Discovery Library · 我的发现
- 简介：「Neo 为你识别的亮眼表现，已收藏 12 条」
- 筛选器：
  - **全部** / **策略切换** / **情绪感知** / **概念迁移** / **反思深度** / **细节捕捉**（tag 筛选）
  - 时间范围筛选（近 7 天 / 近 30 天 / 全部）
  - 来源筛选（对练 / 授课 / 调研 / 报告）

#### 列表区域
- 卡片列表，时间倒序（`acceptedAt` 倒序）
- 每张卡片：标题 + 描述 + 标签 + 来源 + 日期
- 点击卡片 → 展开详情抽屉

#### 详情抽屉
- 完整 title + description
- **证据区**：
  - 对话原文（折叠，点击展开）
  - Neo 分析解读
- **来源链接**：「查看原始 Activity」→ 跳转到对应 Activity
- **标签管理**：学员可增删 tag
- **操作**：
  - 导出为单条 PDF
  - 从库中移除（二次确认）

### 6.3 导出功能

- 学员端：可选中多条导出为一个 PDF（含证据）
- 管理员端（HR）：在员工画像页可查看该员工的 Discovery Library 汇总（需授权）

## 7. 价值定位

### 7.1 对学员

- **自我认知强化**：「原来我做过这些事」的惊喜感与肯定感
- **学习动机维持**：持续的正向反馈 → 延长学习周期
- **个人成长档案**：跨越多个 Activity 的能力沉淀

### 7.2 对报告系统

- 报告 Part 2「高光表现」中的每条证据，**底层指向**某条 Discovery
- 报告与 Discovery Library 形成**双向引用**：
  - 报告中点击证据 → 跳转到 Discovery 详情
  - Discovery 详情中标注「已被 XX 月度报告引用」

### 7.3 对管理端（HR / 导师）

- HR 可在员工画像中查看 Discovery Library（识别员工亮点）
- 导师可引用 Discovery 作为辅导谈话素材
- 管理端批量导出用于团队发展分析

### 7.4 对长期画像

- Discovery Library 是学员能力画像的**可解释性证据库**
- 每一条能力标签背后都有具体的 Discovery 支撑
- 避免"算法黑箱"式的能力评估

## 8. 用户故事

### 故事 1 · 刚完成对练的学员

> **背景**：小李刚结束一次 GROW 辅导对练，进入反思复盘 Phase。
>
> **流程**：
> 1. 反思复盘结束，Neo 在 Chat 推送卡片：「✨ 自我归因策略的灵活运用」
> 2. 小李点开卡片，看到 Neo 描述他在第 4 轮的关键切换
> 3. 点击 [✓ 收藏到 Discovery 库]，右下角弹出 toast
> 4. 小李在当天晚上复盘时，进入 Discovery Library 查看这条记录
> 5. 展开证据区，看到对话原文，感觉被肯定和理解
>
> **价值**：学员获得具体的、有证据的正向反馈，而非空洞的"做得很好"。

### 故事 2 · 长期学员的成长回顾

> **背景**：小王完成第 3 个月度报告，进入报告场域。
>
> **流程**：
> 1. Neo 解读报告 Part 2「高光表现」，引用 4 条 Discovery 作为证据
> 2. 小王点击其中一条证据，跳转到 Discovery Library
> 3. 筛选「策略切换」tag，发现自己近 30 天被识别到 6 次策略切换
> 4. 小王意识到自己在「灵活应变」这个能力点上持续进步
>
> **价值**：Discovery Library 成为学员自我反思的"证据桥梁"。

### 故事 3 · HR 了解员工亮点

> **背景**：HR 负责人希望为优秀员工写推荐语。
>
> **流程**：
> 1. HR 进入员工画像，查看 Discovery Library（经员工授权）
> 2. 筛选近 3 个月的 Discovery，看到 15 条高光记录
> 3. 选中其中 5 条导出 PDF，作为推荐材料的素材
>
> **价值**：Discovery Library 为 HR 提供具体、可信的员工表现证据。

## 9. 未来迭代（P3）

### 9.1 社区化

- **点赞与评论**：同团队学员可为彼此的 Discovery 点赞、评论
- **学习榜样**：公司内部「Discovery 热榜」（每周 Top 10 亮眼表现）
- **匿名化选项**：学员可选择是否公开自己的 Discovery

### 9.2 分享机制

- 分享到团队工作群（一键生成卡片图）
- 分享到导师 / HR（定向分享）
- 嵌入到个人 LinkedIn / 企业内部展示

### 9.3 AI 生成周报 / 月报

- Neo 自动汇总每周 / 每月的 Discovery，生成 Markdown 格式的「我的本周亮点」
- 可选择推送到邮箱 / 企业微信 / 钉钉

### 9.4 Discovery 驱动的学习推荐

- 基于 Discovery 标签分布，推荐学员尚未高光但接近的能力点学习资源
- 如某学员"策略切换"频次高，但"情绪感知"记录少 → 推荐情绪管理相关 Activity

### 9.5 Discovery 生命周期

- 旧 Discovery 可「沉淀」为能力标签（如累计 10 次"策略切换"自动生成「灵活应变 · 熟练」标签）
- 能力标签成为个人画像的正式组成部分

## 10. 技术相关约束（非代码实现）

- Discovery 识别由后台 LLM 完成，学员端仅消费结果
- 推送频率控制：单个 Activity 结束后最多推送 2 条，避免打扰
- 识别准确率目标：学员接受率 ≥ 60%（低于此值需调优 prompt）
- 数据存储：学员端仅保留已接受的 Discovery，已忽略的仅后台保留 7 天
