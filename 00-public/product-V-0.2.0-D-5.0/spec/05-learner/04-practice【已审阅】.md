## 1 practice（对练）

> **总览**：practice 是学员在拟真业务场景中通过角色扮演把所学技能转化为可执行行为的场域 —— 由 **Director（剧本驱动后台编排）+ Actor（剧本驱动学员可见）+ Neo（场外 Coach 旁路答疑）** 三角色协作演绎，覆盖 **导入 / 演练 / 复盘 / 报告** 4 个阶段，目的是把"知道"变成"会做"，并产出可追溯的行为数据。
>
> 本节按 **业务定义 / 三角色协作总览 / 4 阶段总览 / ① 导入 / ② 演练 / ③ 复盘 / ④ 报告 / 跨阶段处理与衔接 / 本期不做** 9 个维度展开。

---

### 1.1 业务定义

> practice（对练）是学员在拟真业务场景中**动手做**的场域 —— 学员扮演自己的目标角色，与 AI 演员（Actor）多轮对话，在 Neo 的旁路答疑辅导下完成预设的行为目标，并在结束后通过复盘和报告沉淀可举证的行为数据。

practice 同时承担 **三重价值**：

| 价值层次 | 学员视角 | 项目视角 |
|---------|---------|---------|
| **行为训练** | 把课程里学到的方法 / 框架在拟真场景中"动手做"，重复练习 → 形成可迁移到工作中的行为习惯 | 关键交付能力 —— 让"知道"变成"会做" |
| **能力鉴定** | 通过 Actor 的真实反馈知道自己"练得怎么样" | 产出可对外证明的行为改变数据(HR / 业务部 ROI 关注的核心) |
| **体验探索** | 在不会因犯错被惩罚的安全环境里尝试自己平时不敢用的策略 | 鼓励学员"试错 → 反思 → 修正"循环，是学习的高质量发生场所 |

### 1.2 三角色协作总览

practice 是学员端唯一含 **三角色协作** 的场域 —— 学员可见的 **Actor + Neo** 在前台演绎，**Director** 在后台调度。

#### 1.2.1 三角色通用职责

| 角色 | 前台呈现 | 主要职责 | 驱动方式 |
|------|---------|---------|---------|
| **Director**（导演） | **不暴露给学员** —— 后台运行，学员无感 | 解读剧本 + 编排每轮对话的场景转换 / 情绪变化 / 行动指令 + 维护 Actor 状态矩阵 | 剧本驱动 + 自身判断 |
| **Actor**（演员） | 具体业务人物（学员可见、可对话） | 按剧本设定的 性格 / 目标 / 触发点 / 软化点 / 底线 / 情绪天花板 进行表演 | 剧本驱动 + 导演指令 |
| **Neo**（场外 Coach） | Neo（与大厅 / lecture / recap 同一身份）| ① 导入：铺垫引导 ② 演练：旁路答疑辅导 ③ 复盘：苏格拉底追问 ④ 报告：解读对话 | Neo 自身判断（剧本 + 导演指令为上下文） |

> **协作关系（一句话）**：Actor ↔ 学员 是主线对话；Director 在后台指挥 Actor；Neo 在场外（右栏 Chat）旁路提供答疑与辅导，**不打断**主线对话。详细协作流程见 § 1.2.2。

**Director（剧本驱动 / 后台 / 学员完全不可见）**：

| 维度 | 内容 |
|------|------|
| **学员可见性** | **完全隐藏** —— GUI 不出现"导演"字样 / 剧本不提 / 学员无感 |
| **编排职责** | 解读剧本 + 每轮分析学员发言 → 决定下一轮场景转换 + Actor 情绪变化 + 行动指令 + 状态矩阵更新 |
| **驱动方式** | 剧本驱动 + Director 自主判断（spec 不规定具体编排规则）|

**Actor（剧本驱动 / 学员可见 / 拟真演员）**：

| 维度 | 内容 |
|------|------|
| **行为模式** | 剧本驱动的递进式反应链（spec 不控制具体规则；KGP 在剧本中定义）|
| **状态矩阵实时位置** | 受 Director 控制，反映学员对话推进对 Actor 状态的影响 |
| **不剧透** | 绝不主动说出"学员该自己想到的方案" |
| **不参与逃生** | 学员卡死时 Actor 不破戏 —— Neo 切讲授态作为逃生通道 |
| **不主动结束** | Actor 不会因学员沉默而推进对话 —— 学员不发言 = 天然暂停 |

**Neo（场外 Coach / 学员可见 / 旁路答疑）**：

Neo 在 practice 与大厅 / lecture / recap 共用同一身份（同 voice / 方法论 / 记忆，详见 § 3.2 + § 2.5）。在 practice 场域中通过右栏 Chat 在场外提供答疑辅导，**不打断 Actor 主线对话**。

**4 阶段任务**：

| 阶段 | Neo 任务 |
|------|---------|
| **① 导入** | 铺垫引导 —— 说明场景 / 角色 / 任务，引导学员阅读重点 |
| **② 演练** | 答疑辅导 —— 主动卡点提示 + 被动回应学员提问，**不打断 Actor 对话** |
| **③ 复盘** | 苏格拉底追问 + 必给总结 |
| **④ 报告** | 解读对话 —— 学员可问"为什么这分低"/"举个例子"等 |

**介入策略**：

- **被动响应**：学员在右栏提问 → Neo 回答（任意阶段全程可用）
- **主动介入**（演练阶段）：Neo 自主判断卡点出手（spec 不规定固定触发规则）

**Neo 不能做 5 条**（practice 全场域约束）：

| # | 内容 |
|---|------|
| 1 | **不连续主动介入 ≥3 轮** —— 避免"教练三连问"打断学员节奏 |
| 2 | **不透露 Actor 剧本机密** —— 不告诉学员 Actor 的隐藏目标 / 软化点 / 底线 等 |
| 3 | **不给具体句子** —— 给方向提示（"试试追问对方的 Interest"），不给现成台词（"你应该说……"）|
| 4 | **不破坏剧本走向** —— 不引导学员违反剧本预设的关键转折 |
| 5 | **不预判未发生轮次** —— 不告诉学员"对方下一步会让步"等剧本未呈现的内容 |

#### 1.2.2 三角色协作流程图

```
                  ┌──────────────────────────┐
                  │     Director（后台）       │
                  │  观察学员发言 + Actor 对话  │
                  │  → 编排导演指令            │
                  │  学员完全不可见             │
                  └──────────┬───────────────┘
                             │ 导演指令（指挥 Actor 表演）
                             ▼
┌──────────────┐   主线对话   ┌──────────────┐
│              │◄────────────│              │
│    Actor     │             │    学员      │
│   （中栏）    │────────────►│              │
│  剧本驱动     │             │              │
│  + 导演指令   │             │              │
└──────────────┘             └──────┬───────┘
                                    │ 旁路提问 / 接收建议
                                    ▼
                            ┌──────────────┐
                            │     Neo      │
                            │  （右栏 Chat） │
                            │  场外答疑辅导  │
                            └──────────────┘
```

**核心机制**：

- **Actor ↔ 学员** 是主线对话（中栏）
- **Director → Actor**：Director 在后台观察学员发言 + Actor 对话历史，每轮编排导演指令并**指挥 Actor 表演**（学员看不到 Director 存在，只看到 Actor 表演结果）
- **Neo ← 学员**：Neo 在右栏旁路提供辅导，**不打断**主线对话
- 学员可双轨输入（中栏对 Actor / 右栏对 Neo），自主选择

#### 1.2.3 非演练阶段约定

> **非演练阶段（导入 / 复盘 / 报告）Director / Actor 默认不出现 / 退场** —— 仅 Neo 与学员交互。三角色协作完整出现仅在演练阶段（详见 § 1.5）。

### 1.3 4 阶段总览 + 成功标准

#### 1.3.1 4 阶段总览

practice Activity 在 AOM 中可编排以下 4 个阶段（**KGP 可任意组合 1-4 个**，不强制全部出现）：

| 阶段 | 主体 | 学员看到 | KGP 提供（AOM 内容包） |
|------|------|---------|----------------------|
| **① 导入** | Neo + 背景 markdown | 场景设定 / Actor 人设 / 任务说明 / 可用资源 / 工具 | 场景文案 / 人设描述 / 学员任务 / 知识点回顾清单 |
| **② 演练** | Actor + Neo 旁路 + Director 后台 | 与 Actor 多轮对话；Neo 在右栏旁观 + 适时介入 | 对练剧本（Actor 对白库 / 触发条件 / 软化点 / 底线 / 情绪天花板 / 成功标准 / 技能阶梯） |
| **③ 复盘** | Neo + 对练小结 markdown | 实时生成的对练小结 + Neo 苏格拉底追问 | 复盘追问框架（哪些行为该追问 / 怎么追问 / 引导指向行为承诺） |
| **④ 报告** | Neo + 报告 markdown | 评分报告 + 高光时刻 + 提升建议 + Neo 解读对话 | 评分维度（competencyGoals）+ 评分阶梯（skill_ladders L1-L5） |

> **AOM 阶段任意组合**：KGP 可根据对练目的灵活选择 1-4 阶段组合 —— 如轻量对练（导入 + 演练）/ 不带复盘（导入 + 演练 + 报告）/ 不带报告（导入 + 演练 + 复盘）/ 完整流程（导入 + 演练 + 复盘 + 报告）等。学员**走完 AOM 编排的所有阶段**即 Activity 完成（详见 § 1.8.7）。
>
> **关键设计**：对练 Activity 拥有**自己独立的 AOM 数据结构**——剧本（角色心理画像 / 行为模式 / 递进式反应链 / 成功标准 / 技能阶梯 等）是 practice 场域专属的内容产物，不与其他场域共用。AOM 数据骨架详见 § 5（注：详细字段属研发实现层）。

#### 1.3.2 4 阶段共用元素

practice 由 **4 个连续阶段** 组成，每个阶段对应**一种独立布局**：

| 阶段 | 布局形态 | 与 § 4.2 学员端"各场域统一布局"的关系 |
|------|---------|-----------------------------------|
| **① 导入** | **两栏**（课件区 + Neo Chat） + 左侧覆盖式抽屉 + 顶部 Subbar | 沿用统一布局 |
| **② 演练** | **三栏**（Actor 信息区 + Actor 对话区 + Neo Chat） + 左侧覆盖式抽屉 + 顶部 Subbar | ★ 例外布局（practice 独有）|
| **③ 复盘** | **两栏**（课件区 + Neo Chat） + 左侧覆盖式抽屉 + 顶部 Subbar | 沿用统一布局 |
| **④ 报告** | **两栏**（课件区 + Neo Chat） + 左侧覆盖式抽屉 + 顶部 Subbar | 沿用统一布局 |

> **关于"阶段"的实现表述**：本节"四个阶段"是产品视角的概念抽象，**对应 AOM 脚本层面的演绎编排单元**。在技术方案确定前产品 spec 用"阶段"统一描述；**具体是否在研发实现层落地为"SCO"数据结构，由研发根据系统需求与开发进度把控** —— 产品 spec 不强约束实现路径。

4 阶段共用以下元素（在各阶段布局中不再重复说明）：

**顶部 Subbar**：

| 阶段 | Subbar 右侧按钮 |
|------|----------------|
| 导入（首次） | **不显示任何按钮**（让学员专注于 Neo 解读 + markdown 阅读）|
| 第 1 次演练进行中 | **显示「重新演练 ▾」但禁用**（灰显，hover 提示"首次演练需走完剧本"，与 § 1.8.1 第 1 次强制完整一致）|
| 第 1 次演练完成后（演练 / 复盘 / 报告 / 自由复习）| **「重新演练 ▾」可用**——左侧主按钮触发新一轮重练；右侧下拉箭头展开历史对话切换菜单 |

> **「重新演练 ▾」按钮的出现与可用规则**：
> - 第 1 次走完导入 → 进入演练阶段时按钮**出现**但**禁用**（不允许第 1 次中途重启）
> - 第 1 次演练走完剧本后按钮**启用**，此后覆盖 演练 / 复盘 / 报告 / 自由复习 四阶段一直可用；学员重进 Activity 时也持续显示

**「重新演练 ▾」下拉菜单规格**：

- 下拉项格式：**「演练 N · {状态点}」**（如「演练 3 · ✅ 已完整」/「演练 2 · ⚠️ 未完成」/「演练 1 · ✅ 已完整」）
- 状态点取值：**✅ 已完整**（走完剧本）/ **⚠️ 未完成**（中途退出 / 切走）
- 当前所在的演练在项末加 **(当前)** 后缀
- 排序：最新在最上
- 主按钮（不展开下拉）行为：**新开一轮重练**（演练 N+1 · 进行中）
- 切换某历史项 → Actor 中栏对话 / Neo 右栏 Chat / 状态矩阵 / 报告视图 全部切换到该次（"未完成"项也能切回继续，详见 § 1.8.1）

**左侧覆盖式抽屉**（沿用 [05-learner/03-lecture § 1.2.2] lecture 抽屉设计）：含两 Tab，承载**整个课程目录 + 当前 Activity 目录**——

- **Tab1：当前 Activity 目录** —— 当前 Activity 内的阶段 / 轮次结构（按 AOM 实际编排的阶段呈现，可能 1-4 个）
- **Tab2：整个课程目录** —— 当前 Course 的 Activity 列表（可切换其他已解锁 Activity，弹窗确认 + 触发断点续播保存）

> 抽屉与 lecture 同口径——practice / lecture 都装"课程目录 + 当前 Activity 目录"，仅 Tab1 内部呈现的"当前 Activity 目录"内容因场域而异（lecture 是 SCO 序列、practice 是 4 阶段进度）。

**右侧 Neo Chat**：4 阶段全程常驻，承载 Neo 的 4 阶段任务（铺垫 / 答疑辅导 / 追问 / 解读，详见 § 1.2.1）。

**笔记悬浮球**：屏幕边缘常驻，跨场域形态一致（详见 [05-learner/02-hub § 1.3]，本节不重复）。

#### 1.3.3 状态指示

practice 4 阶段共用以下状态指示：

| 状态 | 触发 | 视觉提示 |
|------|------|---------|
| **学习计时** | 默认 | Topbar 右侧呈现本 Activity 累计时长（不要求精确，本期不写画像） |
| **进度计数** | 演练阶段 | Topbar 右侧呈现"进度 X/N 轮"（X = 当前轮次，N = 剧本预设上限） |
| **Actor 当前状态** | 演练阶段，每轮更新 | 左栏 Actor 信息区上窗口 —— 状态矩阵叠加在铺满图左上角（4 象限当前点反映情绪 + 配合意愿，详见 § 1.5.2 + § 1.5.1）|

> **不做的状态**：
> - **当前阶段标识** —— 学员在抽屉 Tab1 已能看到当前阶段
> - **当前情绪标签独立呈现** —— 与状态矩阵合并，由矩阵承担情绪表达
> - **暂停** —— Actor 对话与 Neo 对话是**两路并行的独立对话流**，没有统一暂停概念；学员任意时刻在哪栏说话由自己控制
> - **重练中** —— 重练只是新一轮对话起始，不需要单独的状态指示

#### 1.3.4 成功标准

practice 的"成功"定义采用 **达成型**——以剧本预设的"成功标准"为准。

**学员视角的"完成"**：

- 一次对练 Activity 由 KGP 在 AOM 中编排 **1-4 个阶段任意组合**（导入 / 演练 / 复盘 / 报告）—— 不一定 4 个阶段都有
- 常见组合：导入 + 演练（轻量对练） / 导入 + 演练 + 报告（不带复盘） / 导入 + 演练 + 复盘 + 报告（完整流程） 等
- **学员走完 AOM 设计的所有阶段 = Activity 完成**（与具体阶段数无关，弹完成 Modal）

**spec 内部视角的"达成"**：

- "达成"= 学员在演练阶段达成剧本预设的"成功标准"（剧本中定义的目标条件）
- 学员"未达成"也允许走完 AOM 所有阶段产出报告 —— **不强制学员"练到通过为止"**
- 报告会显示"目标未达成" + 原因 + 提升建议；学员可点击 Subbar「重新演练 ▾」再次尝试（详见 § 1.8.1）

### 1.4 ① 导入阶段（仅 Neo）

#### 1.4.1 阶段任务

**功能定位**：让学员在与 Actor 对话之前**完整理解**自己处在什么场景、扮演什么角色、要达成什么目标、手里有什么资源 / 工具可用。

**课件区背景 markdown 内容规范**（5 模块）：

| 模块 | 内容 |
|------|------|
| **场景** | 学员所处的业务情境（时间 / 地点 / 触发事件 / 双方处境）|
| **双方角色** | 学员扮演的角色身份 + Actor 扮演的角色身份（含双方背景 / 当前压力）|
| **任务目标** | 学员需要达成什么（主目标 + 分项目标）|
| **可用资源** | 学员扮演这个角色时可调动的资源 / 筹码（如团队人员 / 预算额度 / 时间窗口 / 上级背书 等）—— 让学员有"手里有什么牌"的认知 |
| **可用工具 / 知识点回顾** | 学员可调用的方法 / 框架 + 与本次对练相关的关键概念清单（轻量回顾，不展开讲解） |

> 5 模块均为 KGP 内容生产侧的产物。本期不规定字段必填（详见 § 1.5.6 剧本与多版本）。

#### 1.4.2 GUI 布局（两栏）

```
┌─────────────────────────────────────────────────────────────────────┐
│ Topbar（项目名 / Course / Activity / 学习计时）                       │
├─────────────────────────────────────────────────────────────────────┤
│ Subbar（路径面包屑 / 当前阶段标识）                                   │
│                              （首次导入阶段：无按钮）                  │
├──┬──────────────────────────────────────┬───────────────────────────┤
│抽│                                      │                           │
│屉│  课件区(左侧)                         │  Neo Chat(右侧)            │
│  │                                      │                           │
│  │  · 标题 + 简介                        │  · Neo 头像 + 名称         │
│  │  · 背景 markdown(学员可随时查)         │  · Neo 主动逐模块讲解：     │

**核心要点**：

- **markdown = 完整信息集**：课件区全幅展示背景 markdown（5 模块详见 § 1.4.1），学员可随时查阅
- **Neo = 补充讲解 + 引导进演练**：Neo 在 Chat 中**主动逐模块讲解**（场景 → 角色 → 任务 → 资源 → 工具），每模块间问"有问题吗"，学员可随时插嘴提问
- **不显示「开始对练」按钮**：进入演练靠 **Neo 主动询问 + 学员确认**——Neo 评估学员理解到位后主动询问"我们开始演练？"，学员确认即自动转入演练阶段
- 进入演练后 Subbar 出现「重新演练 ▾」按钮（第 1 次演练进行中禁用、走完后启用，含历史下拉菜单；详见 § 1.3.2）

#### 1.4.3 Agent 行为（Neo 主动讲解 5 模块）

**Neo 在导入阶段的行为节奏**：

- Neo 进入导入阶段后**主动逐模块讲解**：先讲场景 → 再讲双方角色 → 任务目标 → 可用资源 → 可用工具 / 知识点回顾
- 每模块讲解后，Neo 会主动问学员"有问题吗？"留出提问空间
- 学员可在任何时刻插嘴提问（不必等 Neo 讲完一个模块）
- markdown 同步常驻在课件区供学员随时查阅 —— **markdown 是完整信息集，Neo 是补充讲解 + 引导**

**Neo 铺垫口播范围**：

| 能做 | 不能做 |
|------|--------|
| ✅ 解释场景背景 / 角色关系 / 任务目标 | ❌ 透露剧本中"学员该自己想到的方案" |
| ✅ 提示可用工具和知识点 | ❌ 预判对练走向（如"对方一开始会很强硬"）|
| ✅ 回答学员关于规则和流程的疑问 | ❌ 直接告诉学员"正确答案"或最优策略 |
| ✅ 基于学员提问回顾之前学过的知识点 | |

#### 1.4.4 学员行为

- **输入方式**：导入阶段学员仅通过**右栏对 Neo 输入框**与 Neo 互动（打字 / 语音）；中栏在导入阶段不存在 Actor 对话框（双输入框完整说明详见 § 1.5.4 演练阶段）
- 学员可在任何时刻插嘴提问（不必等 Neo 讲完一个模块）
- 学员理解到位后，可主动说"我们开始吧"，Neo 接收后自动转阶段

#### 1.4.5 阶段约束 + 完成判定

**进入演练的触发**（无独立按钮）：

- Neo 在 5 模块讲解 + 答疑过程中**自主评估**学员理解到位
- Neo 主动询问 "我们开始演练？"
- 学员在 Neo Chat 输入框确认（如"可以开始了" / "OK"）
- → 自动转入演练阶段，整体布局切换为三栏形态

> **不显示「开始对练」按钮**：进入演练完全靠对话节奏推动，避免按钮抢戏；学员"觉得可以开始了"也可主动说"我们开始吧"，Neo 接收后自动转阶段。

### 1.5 ② 演练阶段（★ 三角色协作高峰）

#### 1.5.1 阶段任务

**功能定位**：对练的核心阶段 —— 学员与 Actor 多轮对话，Neo 在右栏旁路答疑辅导，Director 在后台编排剧本走向。

**演练启动**：从导入阶段 Neo 询问 + 学员确认 自动转入演练后（详见 § 1.4.5），**直接进入剧本驱动的角色扮演** —— 没有 Neo 旁白、没有过场。**默认 Actor 先说开场白**（贴合 1v1 业务对练真实情境——学员被动应对），剧本可在 opening_speaker 字段标注切换为"学员先说"。

**Actor 状态矩阵**（替代 v0.4 一维情绪曲线）：

二维矩阵 4 象限：

| 象限 | 状态 | 典型表现 |
|------|------|---------|
| **顺从**（横配合 + 纵漠然）| 温和配合 | "好的，那就按你说的办" |
| **投入**（横配合 + 纵激烈）| 主动协作 | "对！我也是这么想的，我们再深入一下" |
| **冷漠**（横抗拒 + 纵漠然）| 消极抵抗 | "嗯……行吧，你看着办" |
| **抵抗**（横抗拒 + 纵激烈）| 激烈对抗 | "凭什么是我让步？这事儿没法这么做" |

矩阵机制：

- **横轴**：配合意愿（抗拒 ← → 配合）
- **纵轴**：情绪（漠然 ← → 激烈）
- **更新**：每轮学员发言后 Director **在编排下一轮导演指令时**同步维护矩阵当前位置（与 § 1.2.1 Director 编排职责一致；spec 不规定具体维护规则）
- **呈现**：仅显示当前点，不画轨迹（避免视觉冗余）
- **位置**：左栏 Actor 信息区上窗口（详见 § 1.5.2）
- **可见性范围**：**仅本期学员侧露出**（演练 / 复盘 / 报告 / 自由复习 学员可见）；**管理端 D-5.0 不露出**（首页 Home / Ora 报告辅助 不调用矩阵 4 象限），是否在 § 10 后续设计中加入管理端可视化由 § 10 阶段补决策

**演练结束 3 类触发条件**（由剧本定义）：

| 触发条件 | 含义 |
|---------|------|
| **轮数上限** | 对话达到剧本预设的最大轮数 |
| **目标达成** | 学员的方案满足剧本"成功标准"，Director 判定 Actor 接受 |
| **死循环检测** | 学员连续 N 轮在表面僵持 / 重复同样策略，Director 判定无法推进 |

结束后 Actor 退出对话，自动进入复盘阶段。

> **学员"想退出"的处理详见 § 1.8.2**。

**核心要点**：

- **左栏 · Actor 信息区**（拆分为上下两个独立窗口）：
  - **上窗口（Actor 视觉主体 + 实时状态）**：
    · Actor **铺满图**：采用"**中性底图 + 表情覆盖层**"形态——中性身份铺满图占满上窗口作为视觉主体（不变）；表情通过覆盖层方式动态变化（默认随状态矩阵 4 象限切换，剧本可在特定轮次硬编码覆盖；详见 § 1.9 争取项）

- **右栏 · Neo Chat**：与其他阶段右栏同构，但承载**答疑辅导**；输入框标注 **"问 Neo……不影响左侧对练"** 强化"不打断 Actor 主线"
- **双独立输入框**：中栏对 Actor / 右栏对 Neo；学员自主选择何时在哪边发言

> **三栏布局的核心目的**：让 Actor 主线对话与 Neo 旁路辅导**同时可见**，避免学员在两路对话间频繁切换抽屉、丢失上下文。
>
> **铺满图 + 矩阵 + 名字三层叠加的目的**：让学员第一眼看到 Actor 人物像建立"对面是谁"的代入感；状态矩阵在左上角小尺寸常驻，每轮反映 Actor 当前状态；名字与身份在底部不抢戏。

#### 1.5.3 Agent 行为（Director / Actor / Neo 演练阶段）

**Director / Actor**：通用职责详见 § 1.2.1（剧本驱动 + 后台编排 / 拟真演员 / 不剧透 / 不参与逃生 等），演练阶段是 Director / Actor 的完整出场场景。

**Neo 在演练阶段的行为**：

- **被动响应**：学员在右栏提问 → Neo 回答
- **主动介入**：Neo 自主判断卡点出手（spec 不规定固定触发规则）
- **不打断 Actor 对话**（★ 核心约束，详见 § 1.5.5）
- **5 条不能做** 全程适用（详见 § 1.2.1）

**7 动作调用** —— practice 在 § 2.5 7 动作矩阵中的分工：

| 动作 | 权重 | practice 中的体现 |
|------|------|-----------------|
| **练习** | ★ 主场 | 学员与 Actor 拟真对话练习是核心 |
| **解惑** | ✓ 辅助 | Neo 答疑辅导（演练 + 报告阶段）|
| **追问** | ✓ 辅助 | 复盘苏格拉底追问 / 演练答疑追问 |
| **反思** | ✓ 辅助 | 复盘阶段反思引导 |
| **考核** | ✓ 辅助 | 报告阶段对学员行为评估反馈 |
| **讲授** | — / 逃生 | 学员卡死时 Neo 切讲授态作为逃生通道 |
| **讨论** | — | 不出现（→ 大厅 / recap）|

#### 1.5.4 学员行为（双输入）

| 方式 | 交互 |
|------|------|
| **中栏对 Actor 输入框** | 仅演练阶段可用 —— 学员对 Actor 回复（打字 / 语音） |
| **右栏对 Neo 输入框** | 4 阶段全程可用 —— 学员向 Neo 提问（打字 / 语音） |

两个输入框**独立**，学员自主选择何时在哪个输入框发言，平台不强制。

#### 1.5.5 阶段约束

**Actor 与 Neo 独立**：

- Actor **不知 Neo 存在** —— Actor 的对话历史不包含 Neo 的右栏发言
- Neo **可读 Actor 对话** —— Neo 答疑辅导以剧本 + 学员当前演练进度为上下文

**Actor 不剧透 / 不参与逃生**：详见 § 1.2.1 Actor 属性；逃生通道由 Neo 切讲授态承担（详见 § 1.5.3 7 动作表 / 讲授为逃生）。

**Neo 不打断 Actor 对话（★ 核心）**：

Neo 的所有发言只出现在右栏 Chat，**不暂停**中栏的 Actor 主线对话。学员自主选择是否停下来和 Neo 交流。

**Neo 不给答案 / 仅给方向**：

详见 [02-foundation/02-methodology § 1.7] + [05-learner/03-lecture § 1.6.1]（与 lecture 同口径）：

- **演练阶段（剧本成功标准约束）**：未达成剧本成功标准时仅给方向提示，不给具体台词
- **答疑口播（含报告阶段）**：优先引导学员主动思考；简单概念询问可直接答；应用 / 思辨 / 判断类以引导追问为主

**演练结束 3 类触发条件**（由剧本定义，详见 § 1.5.1）：轮数上限 / 目标达成 / 死循环检测。

#### 1.5.6 剧本与多版本

**剧本主导原则**：KGP 在剧本中**写死**角色 / 阶段 / 成功标准，AI 仅做演绎。这是 practice 与 lecture 在内容驱动力上的关键差异。

**剧本本期不规定字段必填**：

- 本期目标是"半结构化中间信息传递"——KGP 内容老师按当前剧本结构提炼信息即可
- 产品 spec **不控制剧本内容生产规则**（不规定字段必填 / 不规定 Actor 行为反应链 / 不规定矩阵触发标准）
- 具体剧本生产方法本期不在产品 spec 内讨论

**对练变体的实现路径**：

| 变体 | 本期处理 | 实现方式 |
|------|---------|---------|
| **基础 Roleplay** | ✅ 默认形态 | 学员扮演自己的目标角色，Actor 扮演对方 |
| **角色互换** | ✅ **本期通过剧本调整实现** | 不需新平台能力 —— KGP 在剧本中互换"用户角色"与"演员角色"的字段（Actor 扮演学员 / 学员扮演对方），平台演绎引擎不变。**运行时学员端 GUI 无"互换模式"概念** —— 左栏 Actor 信息区如实显示 Actor 当前所演角色（剧本编排什么、左栏就显示什么），不出现"角色互换中"提示横幅，不切换学员身份标签 |
| **费曼** | ❌ 本期不做 | 14 项硬编码冲突，需新 Schema 与演绎引擎，本期不支持（详见 § 1.9）|

> **角色互换的关键设计要点**：复盘阶段需引导学员"切回原始身份"反思（如"刚才你扮演客户时是怎么想的？现在站在你自己角色看……"）—— 这部分由 KGP 在复盘 Neo 追问框架中编排。

**多版本剧本 · practice 内的消费规则**：

> 多版本切换 + 学员打标的具体机制**不属于 practice 章节范畴**（详见跨场域 / KGP 内容生产相关章节）。本节仅说明 practice 4 阶段如何**遵循**打标结果：

- **打标时机**：在 practice Activity **4 阶段开始时触发一次**（每次进入 Activity 触发一次新打标）
- **导入 + 演练阶段**：遵循当次打标命中的剧本版本进行演绎
- **复盘 + 报告阶段**：**无版本切换概念** —— 复盘追问框架 + 报告评分逻辑跨版本统一

### 1.6 ③ 复盘阶段（仅 Neo）

#### 1.6.1 阶段任务

**功能定位**：对练结束后帮助学员**反思关键决策点**，把"刚才发生了什么"转化为"下次该怎么做"，是行为养成的关键收尾。

**对练小结 markdown**（实时生成，4 模块）：

| 模块 | 内容 |
|------|------|
| **目标达成情况** | ✅ 主目标达成 / ⚠️ 部分达成 / ❌ 未达成 |
| **高光表现** | 学员表现亮眼的关键决策点 |
| **待改进点** | 学员可优化的关键节点 |
| **对话回看** | 按轮次完整展示对练记录 + 关键节点标注 |

**必问点机制**：

- 复盘是 4 阶段中**质量优先**的阶段：Neo 必须问完所有"必问点"才提示学员可进入报告阶段
- "必问点"由 KGP 在剧本中定义（如关键转折点 / 待改进项 等）
- 学员**不能跳过**复盘 —— 沿用 lecture **阶段锁定**机制（详见 [05-learner/03-lecture § 1.6.4]）：必问点未完成 → 报告阶段处于锁定态，不可访问

- Neo Chat 做复盘追问 —— 苏格拉底式引导反思 + 必给总结
- **联动**：学员点击对话回看的某一轮 → 右栏 Neo **自动**对该轮做定向分析（无需学员手动输入）

#### 1.6.3 Agent 行为（Neo 苏格拉底追问 + 必问点）

**Neo 复盘追问规范**：

| 行为 | 规范 |
|------|------|
| **苏格拉底式追问** | 引导学员自己发现问题，不直接告诉学员"你哪里做得好/不好" |
| **必须给出总结** | 追问之后**必须**给出明确总结和待改进项陈述 —— **不能只追问留开放状态不收尾**（★ 核心约束）|
| **引用具体行为** | 引用具体对话轮次做定向分析，不泛泛而谈 |
| **对比多次练习** | 如学员有多次重练记录（详见 § 1.8.1），Neo 对比多次表现差异 |

**对话回看分析**：学员点击对话回看的某一轮 → 右栏 Neo **自动**对该轮做定向分析（无需手动输入提问）。

#### 1.6.4 学员行为

- **输入方式**：复盘阶段学员通过**右栏对 Neo 输入框**与 Neo 互动（打字 / 语音）；中栏不再有 Actor 对话，呈现对练小结 markdown + 对话回看
- 学员可点击对话回看的某一轮 → 触发 Neo 自动定向分析
- 学员可主动在右栏发起追问 / 反思

#### 1.6.5 阶段约束

**复盘必收尾**：

Neo 苏格拉底追问后**必须**给出明确总结和待改进项陈述 —— **不能只追问留开放状态不收尾**（与 v0.4 设计一致）。

**学员不能跳过复盘**：沿用 lecture **阶段锁定**机制（详见 [05-learner/03-lecture § 1.6.4]）—— 必问点未完成 → 报告阶段处于锁定态，不可访问。

### 1.7 ④ 报告阶段（仅 Neo / 高光识别主场）

#### 1.7.1 阶段任务

**功能定位**：学员 + 管理端共享的**可举证报告**，是 practice 的最终交付物。

**报告 markdown 内容**（3 部分）：

| 部分 | 内容 |
|------|------|
| **能力评分** | 5 级打分（1-5 颜色 / 雷达图）+ 每维度几句话文字描述。**评分维度数 + 维度名 由剧本 competencyGoals 定义**，不固定（注：此评分是 practice 单 Activity 的 competency 评分 / **不是 [02-foundation/02-methodology § 1.3.3] [05-learner/06-cross-context § 4.3] 6 维画像** —— 6 维画像是跨场域聚合的长期画像，两者不混用）|
| **高光时刻** | Neo 从对练记录中提取的学员表现亮点 |
| **提升建议** | 明示提供 prescriptive 建议（"下次试试 X"），帮助学员把反思落到行动 |

**异步分段生成**：

```
演练结束 → 后台启动报告生成 → 第 1 部分（评分）就绪
   ↓
复盘进行 → 后台继续生成 → 第 2 部分（高光 + 提升建议）就绪
   ↓
学员进入报告阶段 → 已生成的部分立即加载 → 未完成部分以 loading 占位
   ↓
全部就绪 → 报告完整呈现
```

> **学员看不到等待感**：异步分段生成让学员在复盘期间报告就开始准备，进入报告阶段时第 1 部分已就绪。

**报告分层**：**同一份报告，学员 + 管理端都看（透明）** —— 不预留管理端额外信号。

**多次重练时报告呈现**（与 § 1.8.1 重练机制一致）：

- 一次 Activity = 一份报告
- 学员在 Activity 内多次重练演练段（详见 § 1.8.1）→ **1 份报告拼接 N 次完整演练的趋势**（如高光 = N 次总高光、**competency 评分**各维度 = N 次趋势线）
- **状态矩阵每次重练清零**（从中性起步），**competency 评分每次独立计算**——不与其他次混算；趋势对比体现在报告中
- **未完成的演练不参与趋势拼接**（中途退出 / 切走的次不进入报告上下文）
- 复盘时 Neo 可对比多次完整演练的表现差异，作为追问素材

#### 1.7.2 GUI 布局（两栏）

```
┌─────────────────────────────────────────────────────────────────────┐
│ Topbar（项目名 / Course / Activity）                                 │
├─────────────────────────────────────────────────────────────────────┤
│ Subbar（路径面包屑）                                  [「重新演练 ▾」]│
├──┬──────────────────────────────────────┬───────────────────────────┤
│抽│  课件区(左侧)                         │  Neo Chat(右侧)            │
│屉│                                      │                           │
│  │  · 报告 markdown                      │  · 报告解读口播             │
│  │    - 能力评分(雷达图 + 文字描述)      │  · 学员问"为什么这分低"     │
│  │    - 高光时刻                         │    "举个例子吗"等           │
│  │    - 提升建议("下次试试 X")           │                           │
│  │                                      │                           │
│  │                                      │  [输入框：和 Neo 讨论报告……]│
└──┴──────────────────────────────────────┴───────────────────────────┘

**核心要点**：

- 课件区全幅展示报告 markdown（3 部分详见 § 1.7.1）
- Neo Chat 做报告解读 —— 学员可问任何与报告相关的问题
- 报告**异步分段生成**：演练结束生第 1 部分（评分） / 复盘结束生第 2 部分（高光 + 提升建议）；生成完即时加载（详见 § 1.7.1）

#### 1.7.3 Agent 行为（Neo 报告解读 + 高光识别）

**Neo 报告解读**：学员可问任何与报告相关的问题（如"为什么这分低"/"举个例子吗"）；Neo 以学员演练对话 + 评分上下文为依据回答。

**高光识别与高光卡片**：

practice 是高光卡片的 ★主场。**高光识别 → 推送高光卡片**是 Neo 的卡片产出能力之一（5 类卡片定义见 [02-foundation/02-methodology § 1.6] / 产品形态见 [05-learner/06-cross-context § 4.4]）。

Neo 在 **报告阶段** 识别学员的"顿悟瞬间" / "高光行为"，自动生成高光卡片，**实时推送**到大厅"我的高光"区（详见 [05-learner/06-cross-context § 4.4] + [05-learner/02-hub § 1.2.3]）。

> **触发示例**：学员在某业务对练中第 N 轮做出超出预期的关键决策（如主动从表层信息推进到深层动机层）→ Neo 判断为"行为亮点"→ 推送对应高光卡片到大厅"我的高光"区。

> 与 lecture [05-learner/03-lecture § 1.4.5] 完全同口径 —— 笔记由学员手动添加，Neo 只推送高光卡片。

**Neo 推送高光 → 大厅"我的高光"区**：报告生成的高光时刻通过 Neo 的高光卡片产出能力**同步推送到大厅"我的高光"区**（详见 [05-learner/06-cross-context § 4.4]，与 lecture [05-learner/03-lecture § 1.4.5] 同口径）。

**practice 4 阶段卡片产出矩阵**（5 类卡片定义见 [02-foundation/02-methodology § 1.6] / 与 lecture [05-learner/03-lecture § 1.4.5] 对位）：

| 卡片类型 | 导入 | 演练 | 复盘 | 报告 | 备注 |
|---------|-----|-----|-----|-----|------|
| **askUserQuestion** | ✓ | — | ✓ | — | 导入：Neo 询问"是否准备好开始演练"；复盘：选择追问方向 |
| **高光卡片** | — | — | — | ★ 主场 | 报告阶段触发 / 推送大厅"我的高光"区 |
| **任务卡片** | — | — | — | — | practice 不主动推（行动承诺以报告"提升建议"段呈现 / 不单独推卡）|
| **知识卡片** | — | — | ✓ | ✓ | 复盘追问 / 报告解读中学员对工具/概念有疑问 → 可推延伸阅读形态知识卡片（轻量 / 不主动打断 / 演练阶段沉浸式不推）|
| **课程卡片** | — | — | — | — | 不出现（课程卡片是大厅 → 课程入口卡，与 practice 内场域无关）|

> 各场域差异化：lecture 任务卡片 ✓（学员说"我下周打算 X"即推）/ practice 任务卡片 — （行动承诺以报告"提升建议"段呈现 / 不单独推卡）/ recap 任务卡片 ★（Activity 完成事件触发产 1-3 条入大厅）。课程卡片仅大厅推。详见 [05-learner/06-cross-context § 4.4.2] Truth Source 矩阵。

#### 1.7.4 学员行为

**学员对报告的操作**：

- ✅ 仅看报告 + 与 Neo 在右栏讨论
- ❌ 不做"分享给上级" / "导出 PDF" / "反驳评分" 等操作（演练原始对话记录管理端可见但不可导出，详见 § 1.7.5）

#### 1.7.5 阶段约束

- **演练原始对话不可导出**：管理端可见但**不可导出**（HR / 运营督学 / 质检需要可见性，但防反推产品功能与商业秘密泄露 —— 与 [04-global/02-platform-global § 1.4] 数据交付分层一致）
- **学员侧"分享 / 反驳"不做**（本期）：学员对报告仅可看 + 与 Neo 讨论

### 1.8 跨阶段处理 + 完成与跨场域衔接

#### 1.8.1 重练机制

**核心模型**：**多次演练 = 多次独立聊天记录**（不是一根连续对话）。每次"重新演练"会**新开一段独立的演练对话流**，而不是接续上一次。

**第 1 次 vs 第 2 次起的差异**：

| 维度 | 第 1 次演练 | 第 2 次起重练 |
|------|------------|-------------|
| **完整性** | ★ **强制走完剧本**（不允许提前结束） | 不强制完整；学员可中途退出 / 切走 / 重开新一轮 |
| **入口** | 导入阶段 Neo 询问 + 学员确认后自动进入 | Subbar「重新演练 ▾」按钮触发 |
| **能否点"重新演练"** | ❌ 第 1 次未走完前按钮禁用 | ✅ 一直可用 |
| **是否进入复盘 / 报告上下文** | 走完后自动进入 | 仅"完整走完"的进入；中途退出的不进入 |

**触发**：第 2 次起，学员在 演练 / 复盘 / 报告 / 自由复习 任意阶段点击 Subbar「重新演练 ▾」按钮（详见 § 1.3.2 Subbar 下拉规格）。

**重练时的状态重置规则**：

| 维度 | 重练时的处理 |
|------|------------|
| **状态矩阵** | **清零**——每次重练从矩阵中性起点开始 |
| **competency 评分** | **独立计算**——每次重练单独打分，不与其他次混算（维度数由剧本 competencyGoals 定义）|
| **Neo Chat（右栏）** | **每次重练独立 Chat 上下文**——上一次的 Neo 辅导记录保留可查（通过下拉切换），但新一次重练右栏 Chat 是新的对话起点 |
| **Actor 中栏对话** | **每次重练独立**——切换历史会切回该次完整对话 |
| **导入阶段** | 不重做（跳过导入）——直接从演练第 1 轮起始 |
| **报告** | **1 份报告，拼接 N 次趋势**（如各 competency 维度趋势线、高光对比；详见 § 1.7.1）|

**重练次数**：**不设上限**（学员可无限重练；后台资源限制由研发把控）。

**多次对话切换**：「重新演练 ▾」下拉菜单列出历史 N 次演练，详见 § 1.3.2 Subbar 规格。学员切换历史时，Actor 中栏对话 / Neo 右栏 Chat / Actor 状态矩阵 全部切换到该次的对应内容。

**未完成演练的处理**：第 2 次起的重练若中途退出 / 切走，该次演练标记"未完成"——**不进入复盘 / 报告上下文**（不参与小结、不参与趋势拼接、不参与评分）；下拉菜单中以"未完成"状态点呈现。学员可随时切回继续，也可重开新一轮。

#### 1.8.2 学员"想退出"处理

**Actor 一律以角色响应**（不区分学员动机）—— 学员在中栏对 Actor 说什么、出于什么意图想退出，Actor **不分辨**，一律以剧本设定的角色身份继续回应。区别处理由 Neo（右栏）和系统（环境层）承担：

| 场景 | 处理 |
|------|------|
| **学员对 Actor 说"我不想聊了"等退出意向语** | Actor **保持角色**——按角色身份回应（如继续推进场景、表达不解、按剧本反应等），不打破第四面墙 + Neo 在右栏可主动询问"是不是遇到什么困难"+ 系统不响应"退出"动作（不跳阶段、不结束 Activity）|
| **学员点"返回大厅" / 关浏览器 / 抽屉切其他 Activity** | 弹窗确认 + 触发断点续播保存（沿用 [05-learner/03-lecture § 1.5.2] ⑩ lecture 同口径） + 下次进入恢复到当前演练阶段与轮次 |
| **第 1 次演练中途想退出** | 拦截规则按 § 1.8.7 演练完整性约束 + § 1.8.1 重练机制处理 —— 第 1 次必须走完，不允许提前结束 |

#### 1.8.3 跨场域 Neo 记忆体现

Neo 在 practice 右栏辅导中**可主动引用大厅 / lecture 的跨场域记忆**——增强个性化辅导。

> **示例**：学员在 lecture 阶段反馈采集中说过自己工作中遇到过某类冲突场景。进入 practice 对练（场景与该冲突类型同构）时，Neo 在右栏可主动说："这个场景跟你之前提过的那次经历比较相似，可以试试我们课程里学过的某方法的某层做切入。"（具体引用内容由 Neo 基于跨场域记忆生成）

具体跨场域记忆机制详见 [05-learner/06-cross-context § 4]。

#### 1.8.4 阶段锁定 + 防挂机

- **阶段锁定**：4 阶段顺序进入，前阶段未完成不可跳后阶段（沿用 lecture SCO 锁定同口径 [05-learner/03-lecture § 1.6.4]）
- **防挂机**：默认 5 分钟无操作触发，可在 program config 调整（沿用 [05-learner/03-lecture § 1.6.5]）

#### 1.8.5 数据持久化与隐私

- **保存频率**：推荐每个阶段完成时落库（沿用 [05-learner/03-lecture § 1.6.6]）；具体实现由研发决定
- **演练原始对话记录**：管理端可见但**不可导出**（HR / 运营督学 / 质检需要可见性，但防反推产品功能与商业秘密泄露 —— 与 [04-global/02-platform-global § 1.4] 数据交付分层一致）
- **隐私三层**：沿用 § 3.4.3（学员之间默认完全隔离 / 跨学员引用需同意 / 管理端例外）

#### 1.8.6 感知体系（practice 独特之处）

> 感知体系 Truth Source 见 § 2.3。本节仅讲 practice 场域的独特之处：

- **感知主要从 Actor 对话流提取**（不是 Neo 自己提问）—— 学员跟 Actor 演练时 Neo 在右栏旁观读对话
- **高光识别在报告阶段触发**（与 § 1.7.3 高光识别与高光卡片 协同）
- **矩阵清零机制**：每次重练演练 状态矩阵清零（不累积），但 6 维画像仍按全场聚合
- **数据写入 6 维画像**：**Neo 自主判断**从对练全程（演练对话 / 复盘表现 / 报告评分）中提取哪些观察值得汇入 6 维（详见 § 2.3.3）

**核心交互一览** —— practice 多数交互沿用 lecture 设计（详见 [05-learner/03-lecture § 1.5.2]）：

| 交互 | 说明 |
|------|------|
| **阶段锁定** | 4 阶段顺序进入；前一阶段未完成不可跳到下一阶段 |
| **防挂机** | 默认 5 分钟无操作 → 系统弹提醒（沿用 lecture，可在 program config 调整阈值）|
| **学员主动提问 Neo** | 4 阶段全程右栏 Chat 可问 Neo |
| **主动回顾跳转** | 学员点抽屉 Tab1 已学完阶段 → Neo 引导回顾（沿用 lecture）|
| **Activity 间跳转** | 学员通过抽屉 Tab2 主动跳转其他 Activity（弹窗确认 + 触发断点续播）|
| **断点续播** | 中途退出 → 系统记录当前阶段 + 对话上下文 + Actor 状态；下次进入自动从断点恢复 |
| **笔记悬浮球** | 屏幕边缘常驻（沿用 [05-learner/02-hub § 1.3]）|

#### 1.8.7 Activity 完成判定

- **走完 AOM 编排的所有阶段** = Activity 完成（与阶段数量无关，1-4 阶段任意组合都适用）
- **演练阶段的完整性约束**：
  - **第 1 次演练**必须走完剧本（不允许提前结束 / 不允许中途切到下一阶段）
  - **第 2 次起的重练**不强制完整（学员可中途退出 / 切换到其他历史对话 / 重开新一轮）
  - **未完成的演练不进入复盘 / 报告上下文**（不参与小结、不参与趋势拼接、不参与评分）
  - 详见 § 1.8.1 重练机制
- 完成后弹出完成 Modal（详见下一子节）

**完成 Modal**：

```

│         ✅                    │

│  你已完成「{Activity 名}」      │
│                              │
│  [回到大厅]                   │
│  [自由复习]                   │  ← v0.5 新增

└─────────────────────────────┘
```

按钮行为（**3 选项**，沿用 lecture [05-learner/03-lecture § 1.7.2] 设计）：

| 按钮 | 视觉 | 跳转目标 |
|------|------|---------|
| 回到大厅 | 次要边框 | 大厅（Neo 接收 Activity 完成状态） |

#### 1.8.8 自由复习阶段

**进入路径**（两种）：

1. Activity 完成 Modal 中点击"自由复习"
2. 已完成的 Activity 在大厅 / 抽屉再次进入时**默认即自由复习模式**

**模式行为**：

- **4 阶段内容全部可回看**：学员可点抽屉 Tab1 进入任意阶段（导入 markdown / 演练对话 / 复盘小结 / 报告）
- **Neo 仅被动答疑**：不主动巡查 / 不主动发起话题；学员在右栏问什么 Neo 答什么
- **Subbar「重新演练 ▾」按钮持续可用**（含历史下拉菜单，详见 § 1.3.2）

**进入演练阶段查看时的呈现**：

- 学员在自由复习阶段点入"演练"阶段查看时，左栏 Actor 信息区状态矩阵**显示最后一轮的矩阵位置**（即该次演练结束时的最终状态点），不重置 / 不动画 / 不重新计算
- 通过「重新演练 ▾」下拉切换到其他历史演练时，矩阵切换到对应那次的最后一轮位置
- 中栏 Actor 对话区呈现该次演练的完整对话记录（只读）

**「重新演练 ▾」路径**（与 § 1.8.1 一致）：

- 主按钮 → 新开一轮重练（演练 N+1）
- 下拉切换 → 切到某次历史演练的完整上下文
- **不设次数上限**
- 形成多次重练趋势（在报告中体现）
