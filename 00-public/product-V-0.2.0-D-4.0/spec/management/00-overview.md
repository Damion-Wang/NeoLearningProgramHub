# 管理端总览

## 定位

管理端是管理员（HR/运营统一角色）的项目管理界面——以"首页 + 报告 + 消息 + 配置"四大功能区组织。本期只有一个管理端，HR 和运营使用同一个端口、同一套界面、同一个角色。

---

## 布局

管理端采用**常显导航 + 内容区**布局，内容区根据模块不同为双栏或单栏：

### 双栏布局（首页 / 报告）

```
┌──────────────────────────────────────────────────────────┐
│  ┌────────┐  ┌─────────────────────┐  ┌────────────────┐│
│  │ 常显   │  │ 左栏：主内容区        │  │ 右栏：Ora 对话 ││
│  │ 导航   │  │                     │  │                ││
│  │        │  │ 首页→学习情况看板    │  │ 自然语言查询   ││
│  │ · 首页◀ │  │ 报告→汇报撰写编辑   │  │ 数据解读       ││
│  │ · 报告  │  │                     │  │ 报告辅助       ││
│  │ · 消息  │  │                     │  │ 追溯钻取       ││
│  │ · 配置  │  │                     │  │                ││
│  └────────┘  └─────────────────────┘  └────────────────┘│
└──────────────────────────────────────────────────────────┘
```

### 单栏布局（消息 / 配置）

```
┌──────────────────────────────────────────────────────────┐
│  ┌────────┐  ┌─────────────────────────────────────────┐│
│  │ 常显   │  │ 全幅内容区（纯 GUI，无 Ora）             ││
│  │ 导航   │  │                                         ││
│  │        │  │ 消息→督学消息编辑/催学规则/发送记录      ││
│  │ · 首页  │  │ 配置→项目信息/名单/角色/个性化/统计     ││
│  │ · 报告  │  │                                         ││
│  │ · 消息◀ │  │                                         ││
│  │ · 配置  │  │                                         ││
│  └────────┘  └─────────────────────────────────────────┘│
└──────────────────────────────────────────────────────────┘
```

- **常显导航**：始终可见，不可折叠，用于切换功能模块
- **首页/报告**：双栏（左栏内容 + 右栏 Ora 对话），核心交互：左栏数据可点击 → 右栏 Ora 自动解读
- **消息/配置**：单栏全幅（纯 GUI，无 Ora）

## Ora — 管理端 Agent

**前台名称**：Ora
**角色类型**：管理端 Agent（独立于学员端的 Neo）
**数据来源**：统一 Database（全局视角）

| 维度 | Ora（管理端） | Neo（学员端） |
|------|-------------|-----------------|
| **服务对象** | 管理员 | 学员 |
| **数据视角** | 全局（所有学员） | 个人（单个学员） |
| **核心职能** | 数据查询、学员表现解读、报告辅助生成与 HITL 编辑 | 大厅展现 Coach 能力（陪伴/辅导/Bloom 引导/情绪感知）；进入场域切换为 Tutor 能力（结构化教学） |

### Ora 的核心能力

| 能力 | 说明 |
|------|------|
| **数据查询与解读** | 基于统一 Database 回答管理员的任何数据问题，标注数据更新时间 |
| **报告辅助生成** | 根据管理员定义的参数生成团队报告初稿 |
| **HITL 编辑** | 根据管理员指令直接修改报告内容（具体方案待调研，需考虑国内模型性能） |
| **证据追溯** | 从结论→证据摘要→原始对话，逐层钻取 |
| **共享记忆** | 首页和报告页共享对话上下文，切换页面不丢失记忆 |

### Ora 的行为边界（能做 vs 不能做）

| 能做 | 不能做 |
|------|--------|
| 基于数据回答问题，标注数据来源和时间 | 编造不存在的数据或推测没有依据的结论 |
| 解读数据趋势，给出可能原因分析 | 对学员做价值判断（"这个学员不行"） |
| 根据指令修改报告内容 | 未经管理员确认自行修改报告 |
| 追溯到原始对话记录（Neo 全场域完整对话——大厅+授课+对练+调研+报告，本期不做脱敏） | — |
| 坦诚告知无法回答的问题 | 猜测或编造答案 |

## 管理员的一天

### 场景一：日常巡检（首页）

**管理员小李每天上午的固定动作。**

1. 小李登录管理端，默认进入「首页」
2. 查看项目整体数据：完成率 28%、上周活跃 3.2 天/人、需关注学员 Top 3、项目进度（已进行42天/剩余138天）
3. 右栏问 Ora："上周进度最快的 3 个人是谁？"——Ora 给出排名和具体数据
4. 切换到「学员管理」Tab，点击低活跃学员查看详情：进度 8%、5 天未登录
5. 切换到「数据展示」Tab，查看各 Course 完成率分布——点击某个柱状条，Ora 自动解读原因
6. 点击某学员查看个人报告汇总

### 场景二：准备月度汇报（报告）

**管理员王经理月底需要向项目负责人汇报培训进展。**

1. 王经理点击「报告」，进入报告库，点击「新建报告」
2. 弹出表单：选择内容模板"中期报告"、全部学员、本月时间范围、Course 1-3、补充说明"重点提及张磊和陈思"
3. Ora 基于参数生成初稿（文本+基础图表），左栏展示
4. 王经理在编辑器中修改文字、调整结构、补充自己的分析
5. 在右栏问 Ora"张磊对练具体进步了多少"，Ora 回答后王经理将数据补充到报告
6. 预览确认后导出 PDF
7. 项目负责人追问时，王经理重新进入向 Ora 查询证据，支持钻取到原始对话

### 场景三：督学消息（消息）

**管理员小李需要对低活跃学员发送督学消息。**

1. 小李左侧点击「消息」，进入消息管理页面
2. 选择 3 名低活跃学员，编辑富文本消息（遵循价值导向原则）
3. 发送后查看发送记录和已读状态（手动消息和自动催学消息统一查看）

### 场景四：新项目开营（配置）

**管理员小李为新签约客户完成开营配置。**

1. 小李左侧点击「配置」，填写项目名称、确认课包绑定、设置项目周期
2. 上传学员名单 Excel，系统展示导入预览（成功/失败/重复）
3. 为特定账号分配管理员角色（默认全部为学员）
4. 配置自动催学规则（触发条件+消息模板+执行策略）
5. 上传客户 LOGO、选择 睿学形象和音色
6. 查看项目内容预览（只读），确认已装载内容正确
7. 点击「确认开营」→ 学员端开放登录，运营在平台外发送登录链接给学员

## 已确认决策

- 本期只有一个管理端（不分 HR 端/运营端），HR 和运营使用同一角色"管理员"
- 端口切换只有两项：学员端 / 管理端
- 角色只有两种：学员 / 管理员
- 管理端 Agent 命名为 Ora，首页和报告页共享记忆
- 布局为常显导航+左栏+右栏（与学员端大厅类似）
- 常显导航四项：首页 / 报告 / 消息 / 配置
- 管理端数据粒度为 Activity 级，不做 SCO 级
- 数据呈现为主，本期不做智能预警评价
- 排名维度：进度排名 + 互动排名，不做能力维度排名
- 学员端/管理端排名数据 API 级隔离
- 30-40 人规模暂无学员分组/部门划分
- 同时在学员名单中的管理员可切换到学员端正常学习；不在名单中的管理员无法进入学员端
- 配置和消息页面为纯 GUI，不需要 Ora

## 本期不做的功能

- 智能预警评价——先呈现数据，管理员自行判断
- 能力维度排名——可信度和可比性本期无法保证
- CEO 汇报 PPT 生成——先做好报告格式
- 学员分组分析——暂无分组需求
- 学员自注册——所有账号由管理员导入创建
- 报告版本管理/数据源选择——未来迭代
- 多语言支持——本期不支持

## 反向补充：drill-down 交互范式

管理端复杂列表页统一采用 **drill-down 范式**：

```
列表视图（默认）─────点击 + 新建───▶ 编辑视图
    ▲                                   │
    └──────左上角"← 返回 XX 列表"──────┘
```

### 适用场景
- 消息模块：发送记录列表 ↔ 新建消息编辑
- 报告模块：报告库列表 ↔ 报告编辑器

### 对比 Tab 切换
drill-down 优于 Tab 的原因：
- 编辑过程不被列表干扰
- 返回路径明确（左上角）
- 移动端 / 窄屏友好

## 反向补充：管理端页面 sticky 标题

管理端所有页面（除首页外）顶部为 sticky 区：

- 页面标题 + 英文副标
- 右上角主 CTA（+新建报告 / +新建消息 / 确认开营）
- 筛选器（如有）

向下滚动时此区保持可见。

**首页例外**：KPI 卡片就是第一屏内容，无单独标题区。
