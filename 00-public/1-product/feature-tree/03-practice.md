# 功能树 03：Practice（对练场域）

**Owner：** 寅恒、曾亚、志扬
**日期：** 2026-04-21
**版本：** v0.4.0 对齐版（L1+L2）
**对应 Spec：** `spec/design/learner/03-practice-classroom.md`

---

## 一句话定位

Actor 角色扮演 + Neo 三任务 + 后台导演。核心目的是能力评估和行为训练，不做知识传递。

---

## 功能结构

```
A. 三角色架构 ★v0.4.0 核心新架构
   A1 Actor（演员）— 前台角色扮演对象（"张总" / "客户王经理"等）
       A1.1 角色人设执行（性格 / 目标 / 触发点 / 软化点 / 底线）
       A1.2 情绪状态管理（受情绪天花板约束）
       A1.3 防剧透（不主动提出剧本中学员必须自己想到的方案）
       A1.4 表情切换（P1 争取，本期可能仅静态头像）
   A2 Neo（Tutor）— 三任务
       A2.1 铺垫引导（情景导入阶段）
       A2.2 答疑与辅导（角色扮演阶段，主动 + 被动）
       A2.3 复盘追问（反思复盘阶段）
   A3 导演（后台，不暴露学员）
       A3.1 分析学员每轮发言
       A3.2 决定 Actor 的场景走向和情绪变化
       A3.3 向 Neo 提供辅导上下文

B. 三阶段流程
   B1 阶段一：情景导入
       B1.1 信息卡片三组（背景 / 目标 / 技能）
       B1.2 Neo 铺垫引导（不透心理画像和底线）
       B1.3 开始对练按钮
   B2 阶段二：角色扮演 ★左右双栏对话
       B2.1 左栏 Actor 对话 / 右栏 Neo 辅导（双栏分离）
       B2.2 Actor 直接开场（无旁白）
       B2.3 10-15-20 轮对话
       B2.4 Neo 答疑与辅导（给方向不给答案）
       B2.5 情绪响应（缓和 / 升级，受天花板）
       B2.6 对练结束（Actor 结束语 + 过渡提示，非突然静默）
   B3 阶段三：反思复盘
       B3.1 Actor 退出，Neo 切换为复盘身份
       B3.2 正向反馈先行（肯定 1-2 个点）
       B3.3 苏格拉底式追问
       B3.4 对话回看面板（轮次导航 + 高亮联动 + 点击定向分析）
       B3.5 ★必须给总结（不留开放状态，Neo 行为边界）

C. 剧本控制元素（映射 AOM）
   C1 characterProfile（心理画像，待 KGP 字段定义）
   C2 behaviorChain（递进反应链）
   C3 constraints（防剧透指令）
   C4 successCriteria（分阶段通关条件）
   C5 emotionCeiling（情绪天花板）
   C6 rubric 复用 competencyGoals.rubric（L1-L5）

D. 对练多版本
   D1 Segment 版本管理（销售版 / 产品版 / 管理版 等）
   D2 导学阶段内置打标（因标签限 Activity 内，无法读取授课打标）
   D3 降级策略（打标失败 → defaultSegment）

E. 重练 + 断点续播
   E1 断点续播（轮次信息 + 导演状态恢复，确保 Actor 连贯）
   E2 重练仅重练 Roleplay（保留情景导入信息）
   E3 数据独立（新旧分开存储，不覆盖不追加）
   E4 报告取值规则（默认取最近一次完成）

F. 角色形象展示
   F1 文生图头像（左栏展示）
   F2 未来表情切换（P1 预留接口）

G. 能力评估碎片产出
   G1 对练完成后生成能力评估碎片
   G2 写入统一 Database
   → 碎片 schema 由 Evaluation 定义，Practice 产出

H. 对练场域布局
   H1 可折叠导航 + 左栏看板（按阶段变化）+ 右栏对话
   H2 演绎阶段左右双栏对话（区别于其他场域）

附录 Z. 已知限制（本期不做）
   Z1 恶意测试 AI 底线 — 不做主动防御，依赖情绪天花板兜底
   Z2 实时教练介入 — 引导职能放反思阶段
   Z3 多人模拟 — 本期仅 1v1
   Z4 费曼 / 角色互换 / 辩论 — 未来通过不同剧本实现
```

## 跨模块依赖

| 对象模块 | 依赖关系 |

| Evaluation (09-A) | 打标（D2）+ 能力评估碎片 schema（G） |

---

## 与 v0.3.3 差异

- ★ **A 节完全新增**：0.3.3 里没有"三角色"架构（Actor 与 Neo 分离、后台导演）
- ★ B2.1 左右双栏对话为 v0.4.0 新结构（0.3.3 是单栏右侧对话）
- ★ Neo 三任务边界（能做 / 不能做矩阵）0.3.3 完全没有
- 整体从"场域二（drill）"正式改名 **Practice Classroom**
