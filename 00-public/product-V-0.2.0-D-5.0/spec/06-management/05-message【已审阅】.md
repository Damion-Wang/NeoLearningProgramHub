## 1 · 消息中心 Message

> 这是管理端的**沟通工具**——管理员对学员定向发送消息 + 查看手动 / 自动催学消息的统一发送记录。
>
> **纯 GUI 模块**（无 Ora 参与 / 沿用 [06-management/01-overview § 1.1.6.1]）。
>
> **仅站内消息**（邮件 / IM / SMS 不做 / 沿用 [03-lifecycle/02-learning § 1.5]）。
>
> **结构与 [06-management/03-report-center § 1] 报告中心同构** —— 2 大模块：消息管理（看历史 + 新建入口）+ 消息编辑（新页全屏 + 发送）。

### 消息中心 = 2 大模块

```
消息中心 Message
├── 消息管理（详见 § 1.2）
│   └── 统一列表（手动 + 自动催学统一呈现 / 类型 chip 区分 / 同项目全部管理员可见）
└── 消息编辑（详见 § 1.3）
    └── 新页全屏编辑器（收件人 + 标题 + 内容 / 类 [06-management/03-report-center § 1.3] 报告编辑器）
```

### 入口与模块关系

- **入口**：管理端左侧导航第 4 项 / Home [06-management/02-home § 1.2.2 (4)] "该学员发消息" CTA 跳入（自动填收件人 → 直接进入消息编辑器）
- **与 [06-management/04-program-config § 1.2.4] 催学规则联动**：催学规则触发的自动消息流入本模块消息管理列表
- **与 [06-management/02-home § 1] / [06-management/03-report-center § 1]**：仅 GUI 入口联动（Home CTA → 编辑器预填）/ **不共享 Ora 上下文** / 不带跨模块对话状态（沿用 [06-management/01-overview § 1.1.6.1] 双 Ora 实例架构）
- **双态**：开营前**完全隐藏**（沿用 [06-management/04-program-config § 1.1] 双态切换 / admin 仅能进 [06-management/04-program-config § 1] Config）/ 开营后可用

### 1.1 业务定义

消息中心是**管理员对学员定向沟通的场域**——管理员手动选人发消息 + 自动催学消息统一呈现 + 投递状态查询。

**与催学规则（[06-management/04-program-config § 1.2.4]）的关系**：

| 维度 | 手动消息 | 自动催学消息 |
|------|---------|------------|
| **来源** | 管理员手动新建（§ 1.3 消息编辑）| 催学规则自动触发（[06-management/04-program-config § 1.2.4]）—— 触发类型含 **里程碑绑定 / 脱训信号 / 进度落后信号 / 手动触发**|
| **发送人** | 当前管理员账号 | 系统（标"自动催学"）|
| **统一查看** | ✅ 都进 § 1.2 消息管理列表 | ✅ 里程碑触发的催学消息也进入本列表 |

### 1.2 消息管理

#### 1.2.1 列表展示与动作

**整体结构**：统一列表（不分 Tab）+ 顶部 "+ 新建消息" 按钮 + 类型 chip 区分。

**列表展示**（每行 6 列）：

| 列 | 内容 |
|----|------|
| 时间 | 相对时间（如"2 小时前"）+ tooltip 绝对时间（草稿 = 最后保存时间 / 定时 = 计划发送时间 / 取消定时转草稿 = 仍保留之前的定时时刻 / 已发送 = 实际发送时间）|
| 类型 chip | **手动** / **自动催学** / **草稿** / **定时**（颜色区分）|
| 发送人 | 管理员账号（手动 / 草稿 / 定时）/ "系统"（自动催学）|
| 收件人 | 单收件人显示姓名 / 多收件人显示"X 人"+ 展开列表 |
| 标题 / 内容预览 | 标题（若有）/ 否则取内容首 30 字 |
| 投递状态 | 已发送 = 接收状态 + 已读统计（X/Y 已读 / 不监控点击）/ **发送失败** = "失败 X/Y" + 行末"重试"按钮 / 草稿 = "未发送" / 定时 = "待 YYYY-MM-DD HH:MM 发送" |

**动作**（仅增 + 查 / 无复制 / 已发送不可改）：

| 动作 | 适用类型 | 触发 | 说明 |
|------|---------|------|------|
| 新增 | — | 顶部 "+ 新建消息" 按钮 | 进入 § 1.3 消息编辑器 |
| 查看 | 已发送 | 点击行 | 行内展开消息详情（完整内容 + 收件人列表 + 已读时间戳）|
| 重进编辑 | 草稿 / 定时 | 点击行 | 重进编辑器 / 可修改 / 可发送 / 可删除 |
| 删除 | 草稿 / 定时 | 编辑器内 / 列表行末 | 二次确认 / 已发送消息不可删 |
| 取消定时 | 定时 | 列表行末 | 二次确认 → 转为草稿（保留内容 + 保留定时时刻为草稿"最后保存时间"）|
| 重试发送 | 发送失败 | 列表行末"重试"按钮 | 二次确认 → 重新加入发送队列（已成功的收件人不重发）|

#### 1.2.2 查询 + 排序 + 筛选

- **查询**：顶部搜索框 / 收件人姓名 + 消息内容关键词 模糊匹配 / 实时搜索
- **排序**：默认按发送时间倒序（最新置顶）/ 不可切换
- **筛选**：按消息类型筛选（手动 / 自动催学 / 全部）+ 按时间范围筛选
- **分页**：每页 50 条 / 底部页码切换

#### 1.2.3 多管理员可见性

按消息类型差异化：

| 类型 | 可见性 |
|------|------|
| 手动（已发送）| 同项目下**所有管理员可见** |
| 自动催学（已发送）| 同项目下**所有管理员可见** |
| 草稿 | **仅当前管理员可见**（草稿是个人未发送内容）|
| 定时（未到达）| **仅当前管理员可见**（定时到达后转手动 / 全管理员可见）|

> 设计原因：已发送消息是对学员的对外动作 / 各管理员需协同知晓；草稿 / 定时未发是个人编辑过程 / 不应曝光给其他管理员。

### 1.3 消息编辑

#### 1.3.1 编辑界面布局

**新页全屏编辑器**（不弹 modal / 不嵌入主页）——结构类似 [06-management/03-report-center § 1.3] 报告编辑器但更简洁，**无右侧 Ora chat 侧栏**（消息中心无 Ora）。

```
┌──────┬──────────────────────────────────────────────────┐
│      │ [< 返回]  新建消息                                  │
│ 左侧 │ ┌────────────────────────────────────────────────┐ │
│ 导航 │ │ 收件人 *  [选择学员 ▾] [已选: 3 人]                  │ │
│      │ ├────────────────────────────────────────────────┤ │
│ Home │ │ 标题      [_____________________________]        │ │
│ Rpt  │ ├────────────────────────────────────────────────┤ │
│ Cfg  │ │ [B I S | 🔗] ← 富文本工具栏                         │ │
│ ★Msg │ │ ┌────────────────────────────────────────────┐ │ │
│      │ │ │  消息内容（富文本编辑区）                      │ │ │
│      │ │ │                                            │ │ │
│      │ │ └────────────────────────────────────────────┘ │ │
│      │ ├────────────────────────────────────────────────┤ │
│      │ │      [💾 草稿保存]  [⏰ 定时发送]  [📤 立即发送]      │ │
│      │ └────────────────────────────────────────────────┘ │
└──────┴──────────────────────────────────────────────────┘
```

#### 1.3.2 表单字段

| 字段 | 形态 | 必填 | 说明 |
|------|------|:-:|------|
| **收件人** | 多选 picker（学员名单）| ✓ | 单选 / 多选 / 全选 / 模糊查询 / Home 跳入自动预填 |
| 标题 | 文字输入 | 选填 | 类报告名 / 帮助管理员一眼识别该消息主题 |
| **内容** | 富文本编辑区 | ✓ | 加粗 / 斜体 / 删除线 / 链接 / 列表 等基础 markdown / **上限 500 字** |

#### 1.3.3 GUI 操作

**收件人选择**：
- 点 "选择学员 ▾" → 弹出学员名单 picker（含搜索框 / 全选 checkbox / 单个 checkbox）
- 已选学员显示为 chip 列表（点 chip × 移除）
- **Home 跳入预填** ：[06-management/02-home § 1.2.2 (4)] "该学员发消息" CTA → 该学员预填进收件人 chip / 管理员**可继续添加其他学员 / 可点 chip × 移除预填学员**（完全自由）

**内容编辑**：
- 富文本工具栏（沿用 [06-management/03-report-center § 1.3.1] 文字格式 + 链接 / 但**不含图表 / 插图 / 加图**——消息中心仅文字）
- 不支持插入图片 / 视频 / 附件
- **字数上限 500 字**（编辑区右下角实时显示 "X / 500" / 超过则发送按钮 disabled）

**隐式自动保存**：
- 编辑器内输入 30 秒后自动保存为草稿（类型 chip = "草稿" / 仅当前管理员可见）
- 编辑器底部显示 "已自动保存" 提示
- 防止意外关闭浏览器丢失内容

**发送动作**（编辑器底部 3 个按钮）：

| 按钮 | 行为 |
|------|------|
| 💾 **草稿保存** | 主动保存（同自动保存效果 / 立即触发）|
| ⏰ **定时发送** | 弹日期 + 时间 picker（必须未来时间）→ 二次确认 → 存入消息管理列表（类型 chip = "定时" / 仅当前管理员可见）/ 到时间自动发送转 "手动" 类型 / 全管理员可见 |
| 📤 **立即发送** | 二次确认（"确认发送给 X 名学员？"）→ 立即发送 / 类型 chip = "手动" / 不可撤回 / 已发送不可改 |

### 1.4 系统约束

#### 1.4.1 仅站内消息

本期渠道仅站内消息——邮件 / IM / SMS 不做（沿用 [03-lifecycle/02-learning § 1.5]）。

#### 1.4.2 不支持学员回应

学员收到消息后**无回复入口**——消息是单向通知 / 不是双向沟通。学员有疑问 → 可通过 Neo 求助 / Neo 视情况推荐学员**线下联系管理员**（不自动转发 / Neo 不直接联系管理员，详见 [05-learner/02-hub § 1.1]）；或通过线下渠道反馈给管理员。

#### 1.4.3 已发送不可撤回 / 不可改

消息发送后**进入投递管道立即不可撤回 / 不可修改 / 不可删除**——避免学员侧产生"消息消失"或"内容变化"困惑。

#### 1.4.4 数据保留

消息发送记录 + 投递状态 **永久保留**（项目归档后仍可查询）。

#### 1.4.5 学员侧消息呈现

学员侧消息入口在**学员端大厅**——大厅顶部 bell icon + 未读数红点 + 点击弹出消息列表（详见 [05-learner/02-hub § 1] 大厅 / 本节不重复）。

### 1.5 本期不做

| 项 | 说明 |
|----|------|
| 邮件 / IM / SMS 渠道 | 本期仅站内 / 沿用 [03-lifecycle/02-learning § 1.5] |
| 消息已读率 dashboard | 已读统计在列表内呈现 / 前端不做汇总图表 |
| 消息模板库 UI | 模板配置在 [06-management/04-program-config § 1.2.4] 催学规则中 / 本节不重复 |
| 学员回应 / 双向沟通 | 沿用 § 1.4.2 |
| 消息撤回 / 编辑已发送 | 沿用 § 1.4.3 |
| 富文本图片 / 视频 / 附件 | 本期仅文字 + 链接 / 不支持媒体嵌入 |
| 群组 / 标签发送 | 本期仅按学员名单逐个选 / 不做学员分组（沿用 [03-lifecycle/01-pre-learning § 1.5]）|
| 监控点击状态 | 仅监控接收 + 已读 / 不监控点击 |
| 批量操作 | 列表中草稿 / 定时 仅逐行操作 / 不做批量选中删除 / 批量取消定时 |
| 发送频率上限 | 不主动制上限 / 沿用 [03-lifecycle/02-learning § 1.5] "运营自负责" 原则 |
| 链接安全检查 | 内容含外部链接 / 本期不做白名单 / 警告 / 安全扫描 |
| 消息内容长度上限 > 500 字 | 本期硬上限 500 字 / 未来视实际场景调整 |
