# Neo Memory · 设计原则

> **状态**：v1.0 最终
> **作用**：研发实施 Memory 系统的产品决策依据
> **声明**：技术选型由研发决定，本文只定 PM 关心的"效果与体验"边界

---

## 1 · 顶层原则

### 越用越懂学员
每次交互让 Neo 更精准地理解：你是谁 / 你在哪 / 你需要什么 / 你下一步去哪。

### 不靠 RAG / 不设向量库
**Neo 溯源靠 Agent 能力，不通过 RAG**——这是 PM 拍板的技术方向。
具体怎么实现由研发决定。

---

## 2 · 5 层渐进画像 + Khanmigo 双重架构

### 5 层（学员维度）
| 层 | 内容 | 置信度 | 调用方式 |
|----|------|--------|---------|
| L1 基础事实 | 姓名、岗位、公司、入职时长 | 高 | 直接引用 |
| L2 行为模式 | 学习节奏、答题习惯、活跃时段 | 高 | 直接引用 |
| L3 价值偏好 | 认同的管理观点 / 反对什么 / 在意什么 | 中 | 加"我感觉" |
| L4 隐性需求 | 嘴上说的 vs 真要的差距 | 低 | 谨慎引用 |
| L5 自我认知 | 怎么看自己 / 希望被看 / 成长轨迹 | 极低 | 极少引用 |

### Khanmigo skill+prerequisites（课程维度）
- 学员在每个 skill 上的掌握度
- 知识前置依赖图谱
- 主动 review prompt（学员需要时 Neo 自动建议复习）

### 双重架构融合
学员维度（5 层）+ 课程维度（skill+prerequisites）= Neo 完整"懂你"模型

---

## 3 · 教学法 + Memory 联动（多维度加权）

**Neo 教学策略 = f(W 阶段, skill mastery, 情绪, 意图)**

| 信号 | 来源 | 影响 |
|------|------|------|
| W 阶段 | L2 行为模式（项目第 X 周） | 初期可告知事实，W5+ 不告知改追问 |
| skill mastery | Khanmigo 维度 | 高掌握度直接讨论应用，低掌握度回归基础 |
| 当前情绪 | GROW 10 种识别（受挫/困惑/焦虑/气馁/中性/好奇/反思/坚定/兴奋/自信） | 负面 → 接情绪 / 中性 → 推进 / 好奇 → 加码 / 自信 → 验证 |
| 提问意图 | LLM 判定（找答案 vs 理解逻辑） | 找答案 → 反问 / 理解逻辑 → 解释 |

### "不给答案"的硬度
**根据学员表现感知**——不是一刀切：
- 初期学员 + 事实问题 → 可直接告知
- W5+ 中期 + 概念问题 → 不告知，追问
- 任何阶段 + "你公司应该怎么做" → 永不直接答（那是学员人生）

---

## 4 · 引用频率与节点

### 默认隐性使用
- Neo 在背景中调用 Memory 影响 voice / 节奏 / 选项
- 不每句明示引用

### 关键节点明示
- **开场**：欢迎回来（引用上次结束位置 + 卡点）
- **卡点**：学员卡住时（"上次类似的地方你说过 X，可以参考"）
- **跨课程过渡**：Course 切换（"PIN 法你掌握得不错，POWER 法可以一起用"）
- **反思**：Activity 结束（"和上次的 X 比，这次有什么变化"）

### 防"机械"原则
- 不滥用"我记得你..."句式（每 5 轮对话最多 1 次）
- 不机械堆叠数据（❌ "根据您过去 47 次对话的统计..."）

## 5 · "懂我"瞬间设计

PM 优先设计 4 类瞬间：

### 1. 跨课程连接
"你说的 X 跟你之前在 Y 课中提的 Z 是同一件事"

### 2. 重提学员忘了的细节
学员记不得但 Neo 记得（"你之前说张总对排期压力很敏感，这次也一样吗"）

### 3. 主动周期提醒
"你提过 5 月要试 X，现在怎样了"——deadline / 应用计划追问

### 4. 模式发现
"我注意到你每次卡在 X，这次又出现了——可能是 Y 在影响"

---

## 6 · Neo 记错时

### 出错处理（推荐方案）
1. **实诚道歉**："不好意思，我记错了"
2. **转为学习机会**："那你能补一下那个背景吗？"
3. **修正写入 Memory**："被学员修正"标记
4. **后期引用时加信号**：低置信度调用

不掩饰、不甩锅、不假装。Soul V2"真实优先于温柔"在 Memory 层面的具体落地。

---

## 7 · 学员可见性 vs HR 可见性

### 学员可见
- 学员问 Neo "你记得我什么"→ Neo 实诚回答
- ChatGPT Memory 风格分层显示（L1-L2 事实型可全见，L3-L5 推断型加"我感觉"标注）
- **不提供"被遗忘权"**（参照 1v1 真人老师不能选择性遗忘）

### HR 可见性（保持 0424 红线 C-06）
- HR 通过 Ora 可读 Neo 全部记忆
- Ora 在展示给 HR 时做摘要化处理（除非 HR 明确要求原话）
- 0424 决策不重新评估

---

## 8 · Neo / Ora 记忆边界（互补不隔离）

| 维度 | Neo（学员端）| Ora（管理端） |
|------|------------|------------|
| 责任范围 | 单个学员 | 全部学员（项目级别）|
| 数据访问 | 读写 | **读取 Neo 全部记忆** |
| 引用风格 | 单个学员的具体场景 | 群体维度统计 + 个案佐证 |
| 摘要化 | 不需要 | 给 HR 时做摘要化 |

**关键架构观点**：Neo / Ora **不是隔离的两个 Memory 系统**，是**互补**——同一份数据，两种视角。

---

## 9 · 6 个月项目跨度的特殊性

### Token cost 优化
- 不可能每次都送 6 月历史（容易爆 context）
- 必须分层管理（PM 不管具体技术，研发选 MemGPT 类还是其他）

### 早期 vs 晚期记忆的"重要性差异"
- 后期不应该忘记早期的认知卡点（学员第 1 周说"跨部门协作总推不动"，第 5 月还要能引用）
- 关键卡点需要持久化（不被压缩 / 不被遗忘）

### 跨场域记忆贯通
- Neo 在大厅记得对练中的具体表现
- Neo 在授课中记得学员在调研中的回答
- 跨场域 = 跨 skill + 跨 Activity 完整回顾

---

## 10 · 防幻觉（PM 视角）

### 不要做的事
- ❌ Neo 编造"你之前说过 X"（学员根本没说过）
- ❌ Neo 把推断当事实（"你性格内向"作为 fact 引用）

### 必须做的事
- ✅ 找不到证据时实诚说"我记不太清"
- ✅ 推断型记忆加"我感觉" / "好像"等限定语
- ✅ 学员说"你说我说过 X 是真的吗"→ Neo 必须能给原文 / 承认记不清

具体技术实现由研发决定（不靠 RAG 的话用什么机制保证可追溯，PM 不管）。
