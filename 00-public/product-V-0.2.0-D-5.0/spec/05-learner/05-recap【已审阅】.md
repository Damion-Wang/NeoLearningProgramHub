## 1 recap（小结）

> **30 秒理解**：recap 是学员在 Course 学习中的**阶段性反思场域**。它产出**两件东西**——
> ① **报告**（系统产出 / 6 章 / 关联 Activity 全完成时后台自动生成并锁定 / 学员不可改），
> ② **Neo 的对话陪伴**（1v1 老师在场域中按学员体验状态展现不同行为：锁定 / 首聊 / 自由复习）。
> 报告与 Neo 是两个不同主体——报告是文本/书面/客观/系统产出（不用"我"），Neo 是 1v1 老师角色（场域中第一人称）。
>
> **结构（5 节）**：定位 + 报告 + Neo + 界面布局 + 全局通用。

### 1.1 定位

#### 1.1.1 一句话定位 + 与其他场域的差异

> recap（小结）是学员在 Course 学习中的**阶段性反思场域** —— 通过查看系统自动产出的反思报告 + 与 Neo 对话挖掘洞察，把"做过什么"沉淀为"看见什么 + 接下来怎么用"。

**与其他场域的两个差异维度**：

| 差异维度 | 与谁对比 | recap 的差异 |
|---------|---------|--------------|
| **跨 Activity 阶段性整合** | lecture 单次内容巩固 / practice 单次演练复盘 | recap 跨多个 Activity 做阶段性整合反思 |
| **系统主动产出 + Neo 主动引导** | 大厅辅导按需应用 | 报告由系统自动产出 / Neo 在首聊态主动 Walk-through |

#### 1.1.2 关联 Activity 集合

**定义**：本 recap 在 Course 内的"管辖范围"——recap Activity 在 KGP 编排上之前的**内容产出类 Activity 集合**（lecture / practice 等），作为本 recap 报告生成、解锁判定、报告增量更新的数据边界。

**关键约束**：关联 Activity 集合**仅含内容产出类 Activity（lecture / practice 等）/ 不含任何 recap Activity**（包含本 recap 之前的其他 recap）。recap 是反思场域、不产出新内容；其他 recap 的产物（报告 / 对话）不作为本 recap 的数据来源。

**配置来源**：KGP 编排时定义 / Course 内 Activity 序列上 recap 之前的内容产出类 Activity 默认全部纳入 / KGP 可手动收窄或扩展（如跨 Course 关联前置 Course 的内容产出类 Activity）。

**字段位置**：详见 [02-foundation/04-data-model § 1.3.4]（小结演绎 · placement + 关联 Activity 范围）。

**边界备注（PM 视角）**：

> 本概念可能存在过度设计。"recap 之前的所有 Activity"在 KGP 编排顺序中本身已经隐含，不一定需要单独的"集合"字段配置。具体是否要做成显式集合（带可空 / 中途变更 / 字段 schema）由研发对接时简化决定。spec 此处先保留概念，备注 PM 视角认为可以简化。

#### 1.1.3 5 重价值

报告前 5 章承载 5 重价值；第 6 章自由复习是首聊后的轨迹延续，不另起新价值。

| # | 价值 | 学员视角 | 项目视角 | 对应章 |
|---|------|---------|---------|------|
| 1 | **综合定调** | 一眼看到 Neo 对自己整体表现的判断 + 跨课程记忆呼应 | 让"老师记得你"成为可举证体感 | 第 1 章 综合评估 |
| 2 | **知识层面回顾** | 把 Course 内学过的关键概念 / 工具重新整合 | 让"学过"沉淀为"内化" | 第 2 章 这一段我学了什么 |
| 3 | **能力维度提升** | 看到 Neo 对自己内容掌握 + 能力变化的整体判断（自己看不到的视角）| 沉淀可举证的能力成长轨迹 | 第 3 章 这一段我学得怎么样 |
| 4 | **上课状态评估** | 看到 Neo 对自己学习过程（投入 / 模式 / 节奏 / 状态）的解读 | 沉淀过程画像（管理端可调用）| 第 4 章 这一段我表现怎么样 |
| 5 | **行动闭环** | 从反思转向具体下一步行动（应用计划）| 让学习闭环到工作场景 | 第 5 章 我接下来该做什么 |

### 1.2 报告（系统产出）

> 本节集中讲清**报告**——它是什么时候由系统产出的、长什么样、各章写什么、谁能看到。报告与学员行为解耦，Neo 在场域中的行为见 § 1.3。

#### 1.2.1 报告生命周期（系统层）

报告生命周期是**系统层事件**，与学员是否进入 recap 无关。

**2 个报告状态**：

| 报告状态 | 含义 | 触发 |
|---------|------|------|
| **未生成** | 关联 Activity 集合未全部完成 | — |
| **已生成已锁定** | 关联 Activity 全部完成 → 后台立即生成前 5 章并永久锁定 / 报告之后持续处于此状态 | Activity 完成事件 |

**第 6 章覆盖式重生成事件**（在"已生成已锁定"状态下持续可能发生 / 不是独立状态）：

- **触发**：学员重新进入关联 Activity 复习 / 产生新学习行为（重练 / 新对话 / 新高光 / 新卡点等）
- **行为**：基于截至当下的所有数据**重新生成一份当下最新第 6 章**，整体覆盖上次第 6 章
- **状态影响**：报告依然处于"已生成已锁定"状态——前 5 章永久不变 / 第 6 章是覆盖式区块 / 不切换状态

**关键规则**：

- **关联 Activity 全完成 → 后台立即生成报告并锁定** —— 不等学员进入、不等学员 Walk-through、不等"实质性反思"判定（详见 § 1.3.4 软销机制）
- **前 5 章一次锁定后永久不变** —— 学员不可改、Neo 不可改（Neo 在场域里可解释 / 可举证 / 但不修改报告本身，详见 § 1.3.7）
- **第 6 章是覆盖式更新区块** —— 每次更新触发时基于截至当下的所有数据**重新生成一份当下最新综合**，整体覆盖上次内容；不保留历史版本
- **学员侧零等待** —— 后台静默完成；不提供"刷新"按钮

#### 1.2.2 6 章结构总览

报告共 **6 章** —— 前 5 章在 Activity 完成事件触发瞬间锁定永久保留；第 6 章自由复习是**条件性存在**的覆盖式更新区块。

**第 6 章条件性存在的含义**：

- **关联 Activity 全完成时仅生成前 5 章** —— 此时第 6 章不存在（学员第一次进入 recap 看到的就是前 5 章）
- **当后续触发增量更新条件**（学员重新进入关联 Activity 复习 / 产生新学习行为 / 详见 § 1.2.8）**时第 6 章才出现** —— 学员再进入时报告末尾增加第 6 章
- **若学员一直没有触发增量条件**则报告永远只有前 5 章（这是合理状态，不是缺章）

**6 章结构表**：

| # | 章 | 承载价值 | 形态 | 数据来源 | 可变性 |
|---|----|----|------|---------|------|
| **第 1 章** | **综合评估** | 综合定调 | 1-2 段 markdown | Neo 综合所有关联 Activity 数据 + 跨课程记忆（按需） | 锁定后永久不变 |
| **第 2 章** | **这一段我学了什么** | 知识层面回顾 | markdown / 知识列表 + 跨知识点关联文字段 | Course AOM + 关联 Activity + 学员场景 + Neo 实时推导跨 Activity 知识网络 | 锁定后永久不变 |
| **第 3 章** | **这一段我学得怎么样** | 能力维度提升（**成果**）| markdown 文字叙事 + 关键对话片段引用块 | 内容掌握 + 能力变化的 Neo 论断 + 高光库子集 + 卡点 + 起点 vs 终点能力变化对比 | 锁定后永久不变 |
| **第 4 章** | **这一段我表现怎么样** | 上课状态评估（**过程**）| markdown 文字叙事 + 6 维画像图 + 数据卡 | 全场域日志 + 6 维画像（T1 锁定快照）+ 学习模式画像 + 状态变化 | 锁定后永久不变 |
| **第 5 章** | **我接下来该做什么** | 行动闭环 | markdown 2-3 条便签 + Neo 任务卡片同步推送 | Neo 基于报告各章 + 跨场域记忆 + 学员场景 | 锁定后永久不变 |
| **第 6 章** | **自由复习** | 轨迹延续（条件性存在）| markdown 综合内容 | 关联 Activity 内的新学习行为（重练 / 新对话 / 新高光 / 新卡点 等）+ 自由复习态对话 | **每次更新覆盖整章** |

**第 3 章 vs 第 4 章边界**：

- **第 3 章 学得 = 内容和能力维度的提升**（"学完后能力变化 / 知识掌握程度 / 高光卡点作为证据"）
- **第 4 章 表现 = 上课状态评估**（"上课时投入 / 节奏 / 模式 / 情绪起伏 / 6 维画像 T1 快照 / 数据卡"）

**报告全长**：前 5 章 1200-1800 字 / 学员阅读时长 4-6 分钟 / 含 Neo Walk-through 完整解读体验 10-15 分钟。这个体量是有意为之——首聊是**仪式感** + **首份持重报告**。第 6 章自由复习视当下情况覆盖式更新，长度不限。

#### 1.2.3 第 1 章 综合评估

| 维度 | 说明 |
|------|------|
| **是什么** | 对学员在本 Course 的整体表现做的 **1-2 段总评**（150-250 字）+ 跨课程记忆引用（按需） |
| **目的** | 给整份报告**定调** —— 让学员第一眼看到综合判断 + 必要时感受到"老师记得我之前学过什么" |
| **形态** | 1-2 段纯 markdown / 不含图 / **不用"我"**（报告层语气，详见 § 1.3.1）|
| **数据来源** | 综合本 Course 所有关联 Activity 数据（互动行为 + 高光卡点 + 6 维 T1 快照）+ 跨课程记忆（如有 / 按需调用，详见 § 1.2.9）|
| **可变性** | Activity 完成事件触发时锁定，之后**永久不变** |

**示例骨架**（占位符）：

> **数据显示**，学员在本 Course《{Course 主题}》上**{整体表现一句话}**——{N} 次 Activity 中累计互动 {M} 次，高光时刻集中在 {场景}。{若有跨课程记忆可引用：**自《{先修 Course}》以来**，{能力维度} 上 {延续/突破描述}。}

#### 1.2.4 第 2 章 这一段我学了什么

| 维度 | 说明 |
|------|------|
| **是什么** | 跨 Activity 知识回顾 —— 学到了哪些知识 / 知识点之间的关联关系 / 还有疑问的 |
| **目的** | 让学员把 Course 内的知识点串起来——核心是**知识层面回顾** + **跨知识点关联** |
| **形态** | markdown / 含知识列表 + 关联关系文字段 / **不画流线图 / 不画树状图** |
| **数据来源** | Course AOM 元数据 + 关联 Activity 列表 + 学员场景描述 + Neo 实时推导跨 Activity 知识网络 |
| **可变性** | 锁定后**永久不变** |

**3 个子部分**：

**A. 学了哪些知识**（关键概念 + 工具 + 框架）

- 列出本 Course 学到的关键概念 / 工具 / 框架
- 每个配 1 句作用描述
- 数量不限 / 由 Course 内容决定

**B. 跨知识点的关联关系**（Neo 实时推导）

- 用一段文字讲清楚**知识点之间的关系**
- 关联关系**不限于一种**——可以是：
  - **应用场景关联**（if-then：在 X 场景用 A 工具 / 在 Y 场景用 B 工具）
  - **递进关联**（A 是 B 的前置 / 学完 A 再学 B 才更深）
  - **互补关联**（A 解决 X 问题 / B 解决 Y 问题 / 二者相补）
  - **对比关联**（A 与 B 在 Z 维度上的差异）
  - 其他实时推导出的关系
- **不画流线图 / 不画树状图** —— 用文字段落表达
- **不展示推理链**（只说结论）
- **不延伸到未学但相关**的内容

**示例骨架**（应用场景关联 / 占位符）：

> 这门课里学到的几个工具，**用法是分场景的**——**{场景 1}** 时用 **{工具 A}**（{一句作用}），{场景 2} 时用 **{工具 B}**（{一句作用}），**{场景 3}** 里用 **{工具 C}**（{一句作用}）。

**C. 还有疑问的**

- 综合识别学员在关联 Activity 中**反复绕回**的概念 / 卡点
- 列 1-2 条 / 每条 1 句话表述
- 作为留白追问的钩子，让学员在自由复习态可主动深入

**状态自适应（含低数据兜底）**：

- 正常态：A + B + C 完整 3 子部分
- **低数据兜底**：A 简短列概念 + B 关联关系一句话；C 数据不足时可不写

#### 1.2.5 第 3 章 这一段我学得怎么样

> **本章定位**：能力维度提升（学完后的能力变化）—— 与第 4 章"上课状态评估"区分。

| 维度 | 说明 |
|------|------|
| **是什么** | 对学员**学完后的内容和能力维度提升**的整体论断 + 具体维度的认知转化（学习前 vs 学习后）+ 高光卡点作为证据 |
| **目的** | 让学员看到**自己看不到的视角** —— 对自己学得怎么样的核心判断 |
| **形态** | markdown 文字叙事 + 关键对话片段引用块 / 不含图 |
| **数据来源** | 高光库子集 + practice 复盘待改进点 + 跨场域感知卡点模式 + 关联 Activity 集合起点 vs 终点能力对比 + 内容掌握程度的综合判断 |
| **可变性** | 锁定后**永久不变** |

**4 个子部分**：

**A. 认知转化**（核心 / 按维度展开 / 学习前 vs 学习后对比）

- 形态：3 维度小节，每个小节 = 学习前 vs 学习后的对比表 + 1-2 句洞察 + 对话片段举证（如有）
- 3 维度：
  - **维度 1 · 认知**（学员对内容的理解 / 思考方式的转变）
  - **维度 2 · 能力**（学员能做什么 / 工具掌握程度）
  - **维度 3 · 行为**（学员实际做了什么 / 动作上的转变）
- **不量化数字** —— 文字叙事为主 + 关键对话片段举证为辅

**关键基线（时间锚点）**：

| 锚点 | 含义 |
|------|------|
| **T0**（起点） | 学员**首次进入关联 Activity 集合中第一个 Activity 之前**的快照状态。若学员有先修 Course，T0 = 先修 Course 结束时的累积状态（含先修 6 维画像 + 行为日志）；若无先修，T0 数据近空白（兜底见下） |
| **T1**（终点） | **关联 Activity 集合全部完成事件触发瞬间**的快照状态。与第 4 章 6 维画像 T1 锁定快照**同一时刻** |

> **不是**开营基线 / **不是**上次 recap 报告 / **是**关联 Activity 集合的 T0 → T1 区间。

**各维度数据源**：

| 维度 | 主要数据源 |
|------|-----------|
| **维度 1 · 认知** | 关联 Activity 集合内所有 **lecture Quiz 答案 + lecture 对话**（含追问 / 解惑 / 反思 表达）**+ practice 中的话术理解信号**——全部按**时间戳从早到晚分布**到 Neo 上下文 |
| **维度 2 · 能力** | 关联 Activity 集合内所有 **practice 演练表现**（策略选择 / 工具应用 / 应对效果）**+ 第 4 章 6 维画像"能力维度" T1 值 vs T0 值**（详见 § 1.2.6 B）——全部按**时间戳从早到晚分布** |
| **维度 3 · 行为** | 关联 Activity 集合内所有 **跨场域行为日志**（互动节奏 / 提问主动性 / 回答风格 / 反思深度）**+ 第 4 章 6 维画像"行为维度" T1 值 vs T0 值**——全部按**时间戳从早到晚分布** |

**Neo 早期 vs 后期判断**：spec **不规定固定时间窗** —— Neo 基于关联 Activity 集合内全数据按时间戳分布，**自主判断**学员在该维度上"早期" vs "后期"的表现差异，输出对比叙事。研发实现时只需提供**带时间戳的全量数据流** / 早晚切分逻辑由 Neo 处理。

**T0 空白兜底**：

- **学员无先修 Course**（首次接触本平台 / T0 近空白）→ A 子部分**软化为"建立"叙事**（不写"转化对比"，写"在该维度上从无到有建立了…"）
- **学员有先修 Course** → T0 数据复用先修 Course 结束时的累积状态（先修 6 维画像 + 历史日志）→ 写"转化"叙事
- T0 兜底场景下若对比弱 → A 子部分可只用文字叙事 + 举证 / 不强制对比表

**关联 Activity 集合的起点 vs 终点 不限制其他维度**：高光库子集（B）/ 卡点（C）/ 总评（D） 仍按各自数据来源（详见各子部分）/ 此处 T0/T1 仅约束 A 子部分对比基线。

**B. 高光行为**（1-2 条 / A 认知转化的证据）

- 形态：markdown 引用块（每条 = 标题 + 引用对话片段 + 客观一句话评注）
- 限定：关联 Activity 内的高光库子集
- 数量不足时**宁可空着也不补关联外**

**C. 卡点行为**（1-2 条 / A 认知转化的证据）

- 形态：markdown 引用块
- 限定：关联 practice 复盘的待改进点 + 跨场域感知到的反复出现的卡点模式
- **不依赖 Quiz 错题**（Quiz 不一定有对错，详见 [05-learner/03-lecture § 1.3.4]）

**D. 学习成长总评**（高级论断 / 可选）

- 形态：1-2 段 markdown
- 内容：
  - **特质画像**（这一段学习中观察到的学员 2-3 个特质）
  - **关键转变**（这一段学习中明显的 2-3 个转变）
- **三态写法**（适用于 A 维度对比 + D 总评）：
  - **进步** → 直说能力提升 + 举证
  - **持平** → 写"**稳定保持**" + 稳定点
  - **退步** → 软化为"**近期波动**" + 具体证据（不直接说"退步"）

**状态自适应（含低数据兜底）**：

- 正常态：A + B + C + D 完整 4 子部分
- **低数据兜底**：A 按可观察到的维度缩短；B/C 数量不足时宁可空着；D 数据不足时不强写

#### 1.2.6 第 4 章 这一段我表现怎么样

> **本章定位**：上课状态评估（学习过程行为）—— 与第 3 章"能力维度提升"区分。本章 6 维画像是 Activity 完成事件触发瞬间的 **T1 锁定快照**，永久不变。

| 维度 | 说明 |
|------|------|
| **是什么** | 学员在本 Course 关联 Activity 内的**上课状态** —— 数据卡 + 6 维画像（T1 快照）+ 学习模式画像 + 状态变化 |
| **目的** | 让学员看到自己在学习这件事上的**过程画像** —— 数据指标 + 6 维评估 + 学习风格解读 |
| **形态** | markdown 文字叙事 + 数据卡 + 6 维画像图 |
| **数据来源** | 全场域日志（学习时长 / 互动次数 / 实战演练 / 主动探索 / 应用反思）+ 6 维画像 T1 值 + 综合学习模式 + 状态变化（情绪 / 投入度）感知 |
| **可变性** | 锁定后**永久不变**（6 维画像 T1 快照永久不变 / **6 维画像仅在本章呈现** / 第 6 章自由复习不再呈现 6 维任何形式） |

**4 个子部分**：

**A. 学习数据卡**

- 形态：4 数据卡 / 表格呈现
- 4 维度：
  - **互动次数**（含 Quiz / 答疑 / 演练对话）
  - **实战演练**（关联 Activity 完成情况 / 重练次数）
  - **主动探索**（学员主动延伸提问数）
  - **应用反思**（关联到自己真实工作场景的次数）
- **不画柱图**（数据卡形式呈现即可）

**B. 6 维画像（T1 锁定快照）**

- 形态：雷达图 + markdown 1 段解读（80-150 字）
- 数据来源：6 维感知体系 / 学员在所有场域累积行为 / **取 Activity 完成事件触发瞬间的快照**
- **永久锁定**：本章 6 维画像 = T1 快照，永久不变；**6 维画像仅在本章呈现** / 第 6 章自由复习不再呈现 6 维（理由：6 维变化触发难判断）
- 解读：用人话讲清楚每个维度分数代表什么 + 为什么

**C. 学习模式画像**

- 形态：1 段 markdown（100-150 字）
- 内容：学员在本 Course 中的学习风格特征（如"边练边问型 / 先理论后实战型 / 反复打磨型"）+ 学习节奏（深入 vs 浅尝 / 系统 vs 跳跃）
- 用人话描述："学员属于那种 {X} 型——表现是 {Y} / 优势在 {Z}"
- **不预设"哪种好哪种差"**

**D. 状态变化**（情绪 / 投入度 / 心态起伏 / 可选）

- 形态：1 段 markdown（80-120 字 / 可选）
- 内容：学员在关联 Activity 内的**情绪 / 投入度 / 心态**起伏
- **必须有具体证据**（哪个对话 / 哪个时间点）
- **如无明显波动则不写本子部分**

**状态自适应（含低数据兜底）**：

- 正常态：A + B + C 必有 / D 可选
- **6 维画像始终显示**：6 维数据来源已锁定（基于学员在所有场域累积行为），不会缺失
- **低数据兜底**：A 数据卡仅显示有数据的维度；C 学习模式简短；D 数据不足时不强写

#### 1.2.7 第 5 章 我接下来该做什么

| 维度 | 说明 |
|------|------|
| **是什么** | **2-3 条便签式行动建议**（应用计划）—— 报告内文字便签 + Neo 任务卡片同步推送学员选择 |
| **目的** | 把反思转化为可执行的小任务，闭环到学员真实工作场景。**给的是建议，不是命令** |
| **形态** | markdown 列表（每条 = 1 句简洁可执行的小任务 + 时间档表达）|
| **生成时机** | Activity 完成事件触发时一次性生成 2-3 条；锁定后永久不变 |

**核心规范**：

**A. 便签化形态**

- 每条 = 一句话一个动作
- 形态像便签贴提醒，不是作业本

**B. 时间档表达 = 近期 / 中期 / 长期**

- 不用具体时间（24h / 本周 / 月内）
- 改为 **近期 / 中期 / 长期** 三档表达
- **语气是建议不是命令**（不用"立即"、"必须"等强硬话术）

**C. 不夹带学习内容推荐**

- 行动建议聚焦"行动"本身，不夹带"推荐看 X 课"等推卡片

**D. 数量限制**

- **2-3 条**（不强排序 / 不挑选 / 直接生成）
- 不展示折叠后的"还有 N 条"

**E. 与第 2 章不重复**

- 第 5 章建议**只写动作**（"在 {工作场景} 主动 {做 X}"）
- **不在括号里重述"在 X 场景用 Y 法"** —— 那是第 2 章的工作

**呈现示例**（占位符）：

```markdown
### 第 5 章 我接下来该做什么

基于这一段你学了什么 + 学得怎么样 + 表现，**给你几个建议**可以试一下：

1. **【近期】** 下一次 {工作场景 1}，**主动 {具体动作 1}**
2. **【中期】** 在 {工作场景 2} 里，**{具体动作 2}**
3. **【长期】** 选一个 {工作场景 3}，**完整 {具体动作 3}**

→ 已经把这几件事做成任务卡片推到面前 / 可以选接收哪几条加到大厅待办
```

任务卡片推送时序详见 § 1.3.6（Walk-through 内逐条推 + 学员逐条选择）。

#### 1.2.8 第 6 章 自由复习（条件性存在）

> **作用**：Activity 完成事件触发后，自由复习态产生的所有新内容统一收录在本章——位于前 5 章之后，**每次更新覆盖整章**（区别于前 5 章的永久锁定）。

**条件性存在**：

- 仅在触发增量更新条件时才出现
- 学员第一次进入 recap 看到的报告通常**只有前 5 章**（无第 6 章）
- 后续学员再进入时若已触发增量更新则报告末尾增加第 6 章

**触发条件（统一为"关联 Activity 内的新学习行为"）**：

第 6 章更新触发判断节点 = **学员是否重新进入关联 Activity 复习 + 是否产生新的学习行为**。具体场景包含：

- 学员重练 practice Activity（产生新对话 / 新高光 / 新卡点）
- 学员重听 lecture Activity（产生新追问 / 新理解）
- KGP 后期追加新关联 Activity / 学员完成该新 Activity（产生新关联 Activity 总结）
- 学员在自由复习态本场域内与 Neo 产生新对话观察

**不作为触发条件的项**（已删除）：

- ~~6 维画像有变化~~（理由：6 维变化触发难判断 / 6 维画像仅在第 4 章 T1 快照呈现 / 第 6 章不再呈现 6 维）

**位置**：在报告末尾（前 5 章之后），用 markdown `## 第 6 章 自由复习（截至当下最新）` 作分界。

**内容范围**：

| 内容类型 | 说明 |
|---------|------|
| **当下综合观察** | 学员重进 recap 跟 Neo 聊天时产生的截至当下综合观察 |
| **截至当下的新高光 / 卡点** | Activity 完成事件后产生的新关联 Activity 内高光 + practice 重练带来的新卡点 |
| **新关联 Activity 总结** | KGP 后期追加新关联 Activity 时，对该新 Activity 内学员表现的综合回顾 |
| **学员复习行为洞察** | 学员重练 / 重听关联 Activity 产生的新尝试、新理解、新疑问 |

**与第 4 章 6 维画像的边界**：

- **第 4 章** = 6 维画像 T1 锁定快照（雷达图 + 解读 / 永久不变）
- **第 6 章** = **不呈现 6 维任何形式**（理由：6 维变化触发难判断 / 学员看 6 维只在第 4 章看 T1 快照）
- 6 维画像在 recap 报告中**仅有第 4 章的 T1 一份**

**更新机制**：

- **每次更新覆盖整章**：第 6 章不是按时间叠加，而是每次更新触发时基于截至当下的所有数据**重新生成一份当下最新综合**，整体覆盖上次内容
- **理由**：自由复习态的更新触发点不统一，按时间叠加会让内容割裂；用"覆盖式整体重生成"保证学员每次看到的是**当下最新一份完整综合**
- **不保留历史版本**：学员看不到"上一次的第 6 章"
- **前 5 章永远不变**：第 6 章的覆盖式更新不影响前 5 章

**学员侧呈现**：

```markdown
## 第 1 章 综合评估
...
## 第 2 章 这一段我学了什么
...
## 第 3 章 这一段我学得怎么样
...
## 第 4 章 这一段我表现怎么样
...
## 第 5 章 我接下来该做什么
...

## 第 6 章 自由复习（截至当下最新）

- 当下综合观察：...
- 截至当下的新高光 / 卡点：...
- 学员复习行为洞察（重练 {Activity X} 时的新尝试 / 新理解 / 新疑问）：...
```

**Neo Chat 历史**：在 Activity 内不被清空——学员可翻 Chat 历史回看跟 Neo 聊过的具体对话内容（不在第 6 章重复呈现）。

**笔记复用**：学员"想留住某段感动"通过笔记功能记录（笔记悬浮球，沿用 [05-learner/02-hub § 1.3]）。

#### 1.2.9 跨课程记忆调用

**调用层级**（按优先级）：

| 层级 | 范围 | 优先级 |
|------|------|--------|
| **L1** | 本 Activity（本 recap 自身上下文）| 最高 |
| **L2** | 关联 Activity（本 recap 的关联范围内）| 次高 |
| **L3** | 其他 Activity 和大厅辅导教练（本 Course 其他 Activity / 大厅 Coach 对话 / 跨 Course 内容）| 兜底 |

**调用规则**：

- **按需调用 / 不再硬性"每章必引用"** —— 每章评估哪一层提供最相关材料；若 L1 / L2 已经能撑起本章内容则不必扩到 L3；若需要跨课程记忆来增强本章价值（如第 1 章定调时呼应"老师记得你"）则按需引用 L3
- **首课程学员场景** —— 学员第一次进入 recap 时 L3 层为空 / 此时所有章节按 L1 + L2 撑 / 不强行写"自《X》以来"等跨课程引用句式
- **引用准确性**：信任 [05-learner/06-cross-context § 4.1] 反幻觉机制 / 不引用不确定的事实

**引用形式**（仅 L3 时才用）：

| 形式 | 示例 |
|------|------|
| 延续 | "**自《{先修课}》以来**，在 {能力} 上持续保持高位" |
| 突破 | "**与在《{先修课}》中的 {表现} 相比**，本课程新打开了…" |
| 一致性 | "**这与在《{课 A}》《{课 B}》中的特征是一致的**" |
| 应用机会 | "**结合在《{先修课}》中练过的 {工具}**，可以试…" |

**Neo 在场域 Chat 中的引用密度**：Neo 在场域里**主动**引用学员跨场域 + 跨课程信息——让"老师记得你"成为体感。报告层与场域层是不同主体（详见 § 1.3.1），密度规则也不同。

#### 1.2.10 报告产物属性

| 维度 | 说明 |
|------|------|
| **生成主体** | 系统后台 / 与学员行为解耦 |
| **触发（前 5 章）** | 关联 Activity 全完成 → 后台立即生成并锁定 |
| **触发（第 6 章）** | 已锁定状态下 / 新增量驱动覆盖式重生成 |
| **修改性** | 学员侧**不可修改** / Neo 在场域中**也不可修改报告本身**（学员反驳时 Neo 用举证回应而不是改报告，详见 § 1.3.7）|
| **导出性** | 场域内不可导出（沿用 [05-learner/06-cross-context § 3.4]）；大厅"我的报告"主题可导出 |
| **KGP 控制权** | KGP 控制 recap Activity 在 AOM 中的位置 + 关联 Activity 集合配置（详见 § 1.1.2 + [02-foundation/04-data-model § 1.3.4]）|
| **知识网络轻处理** | **实时推导**跨 Activity 知识关联 / **不依赖 KGP 概念关系图** / **AOM 不加新字段** |
| **学员第一受众** | 报告以**学员自己看为主，管理员附带** |
| **管理端可见性** | **已生成已锁定状态**的 recap 报告 → 管理端**可露出**（首页 [06-management/02-home § 1.2.2] (4) 个体详情面板 / Ora 拉数据生成 team report 均可调用 / 第 6 章如已触发增量重生成则一并露出）；**未生成状态** → 管理端**不露出**。具体字段需求见 [06-management/03-report-center § 1.2.2] |
| **不重新触发** | 已锁定的前 5 章不可重新触发 / 学员状态不佳时踩中首聊也不重生 / 这是设计意图（沉淀可举证轨迹）|

### 1.3 Neo（场域行为）

> 本节集中讲清 Neo 在 recap 场域中**说什么 / 怎么说**——按学员体验状态（锁定 / 首聊 / 自由复习）展开。Neo 与报告是两个不同主体（详见 § 1.3.1）。

#### 1.3.1 Neo 在 recap 的定位

**Neo 与报告 = 两个不同主体**：

| 主体 | 来源 | 语气 / 人称 | 修改性 |
|------|------|------------|--------|
| **报告**（系统产出 / 文本） | 后台自动生成 | 客观正式 / 用 **"数据显示 / 观察到"** / **不用"我"** | 学员不可改 / Neo 不可改 |
| **Neo**（1v1 老师角色） | 场域中第一人称表达 | 像真人老师 / 有立场 / 敢指出错位 / 用 **"我"** | — |

> 这**不是 Neo 的双层语气切换**，而是两个不同主体的并列。学员同屏看到左栏报告（系统文字）和右栏 Neo Chat（1v1 老师）时，可以理解为"系统给了一份报告，老师 Neo 跟我聊报告"。

**Walk-through 中 Neo 转述报告的方式**：

- Neo 在 Chat 里说话 → 用"我"（第一人称 / 1v1 老师）
- 当 Neo 引用 / 转述报告内容时 → 用 **"报告里讲到 / 我看到你为什么这样是因为 / 数据上显示"** 等组合表达
- 例：
  - "**报告里讲到** 你在 Activity X 上的高光是…，**我看到** 这个是因为你那次在 …"
  - "**数据上显示** 你这一段稳定保持，**你怎么看？**"

**Neo 与大厅 / lecture / practice 共用同一身份**（同 voice / 方法论 / 记忆，详见 [02-foundation/01-personas § 1.2] + [05-learner/06-cross-context § 3.0]）。在 recap 中 Neo **引导反思为主**——不主导讲授（与 lecture 区别）/ 不旁观演练（与 practice 区别）。

**核心姿态**：**引导学员看见自己 + 不替学员说总结**。

**7 动作分工**：

| 动作 | 权重 | recap 中的体现 |
|------|------|---------------|
| **反思** | 主场 | 引导学员看见自己的行为模式与变化轨迹 |
| **追问** | 主场 | 苏格拉底式追问，挖出学员自己的洞察 |
| **讨论** | 主场 | 与学员就报告内容平等对话 |
| **解惑** | 辅助 | 学员对报告或概念有疑问时回答 |
| **考核** | 辅助 | 报告里的 6 维画像 + 高光 + 卡点 隐含评估反馈 |
| **讲授** | 轻量补漏 | 学员忘了某概念可补，**不主动**讲新内容 |
| **练习** | — | 不出现（→ practice）|

**Neo 反思设计原则**：

| 约束 | 说明 |
|------|------|
| **不替学员总结洞察** | Neo 引导学员说，不替学员说 |
| **不预设标准答案 / 不给"应该"** | 沿用 [02-foundation/02-methodology § 1.7] + [05-learner/03-lecture § 1.6.1] |
| **显性证据限定关联 Activity / 隐性背景不限定** | 报告内可被学员看到具体来源的证据类（高光 / 卡点 / 对练具体行为）严格限定 KGP 配的关联 Activity 内；Neo 综合产物（自由复习态观察 / 应用计划依据 / 跨课程记忆）**不限定来源** |
| **不展示推理链** | 报告中**只说结论**，不写"我之所以这么判断…"|
| **不延伸到未学但相关的内容** | 严格贴现阶段已学 |
| **必收尾** | Neo 任何一轮追问 / 对话之后**必须**给收尾 —— 给观察 + 留言 + 不留开放状态（沿用 [05-learner/04-practice § 1.6.5]）|

#### 1.3.2 学员体验生命周期（UI 层）

学员体验生命周期是**学员视角**的状态——与 § 1.2.1 报告生命周期独立、可能错位：

> 报告锁定与学员是否进入解耦——学员可能在关联 Activity 全完成后很久才进入 recap；也可能 Activity 全完成后立刻进入触发首聊。两条时序线**独立**且顺序不一定靠近。

**3 个学员体验状态**：

| 学员体验状态 | 含义 | 触发 | Neo 行为 |
|------------|------|------|---------|
| **① 锁定态** | 关联 Activity 未全部完成 / recap 入口锁定 | — | 不出现（recap 内）/ 大厅入口承担解锁引导 |
| **② 首聊态** | 关联 Activity 全完成 + 学员**首次进入** recap（一次性事件 / 走完即转入自由复习态）| 学员首次进入 | 主动 Walk-through + 主动开场 + 任务卡片推送 |
| **③ 自由复习态** | 学员**首次进入之后**的所有再进入（含复习态 + 回访态二级切换）| 学员再次进入 | 学员主导问答 / Neo 被动响应（复习态）或主动指出新增（回访态）|

**自由复习态二级切换**：

| 二级状态 | 切换信号 | UI 表现 |
|---------|---------|--------|
| **复习态** | 报告自上次离开后**无新增内容** | 无红点 |
| **回访态** | 报告自上次离开后**有新增内容**（第 6 章已增量更新）| Subbar **📊 报告有更新** 红点 |

**红点机制为系统判定 / Neo 不主观判断**：

- 系统检测自学员上次离开 recap 至当前 / 第 6 章是否有覆盖式更新发生
- 有 → 标记回访态 + Subbar 红点
- 无 → 标记复习态 + Subbar 无红点
- Neo 直接读这个状态决定行为，**不需要自己判断**

**进入路径**（不影响首聊态判定 / 只要满足"前序全完成 + 首次"就触发首聊）：

| 进入路径 | 含义 |
|---------|------|
| **从大厅内容目录主动进入** | 学员通过大厅内容目录点入 recap |
| **从其他 Activity 通过抽屉跳转** | 学员在 lecture / practice / 其他 recap 内通过左侧抽屉 Tab2 整个课程目录跳转进入 recap |
| **从上一个 Activity 自然跳转进入** | 学员完成 recap 前一个 Activity 后，通过"完成 Modal · 继续下一 Activity"按钮自然过渡进入 |

#### 1.3.3 锁定态 Neo 行为

**recap 内**：不出现（学员尝试点入时不进入对话场景）。

**锁定页呈现**（沿用大厅设计 / 不在 recap 内加额外引导）：

```
┌───────────────────────────────────────────────────────┐
│ ⏳ 完成前序 X 个 Activity 后解锁                         │
│ 还需完成：[Activity Y] [Activity Z] [Activity W]       │
│                                                        │
│ [→ 跳转到 Activity Y]                                  │
└───────────────────────────────────────────────────────┘
```

**关键规则**：

- **不展示半成品报告** / 不计 Activity 完成事件 / 不消耗会话上下文
- **沿用大厅解锁设计**（详见 [05-learner/02-hub § 1.2.2]）—— 大厅 Course/Activity 序列**解锁全自由 / 不锁顺序** / AOM 给推荐顺序但学员可以乱序学
- **不加"建议从 X 开始"引导** —— 与大厅设计一致 / 学员自主排序 / 跳转按钮列出剩余 Activity 即可

#### 1.3.4 首聊态 Walk-through

> 首聊态是 recap 的主场——一次性事件 / 走完即转入自由复习态 / 不可重新触发。

**触发判定**：学员**首次进入** recap 即触发首聊态。**与"实质性反思"软销机制配合**：

- 旧 spec 把"实质性反思"作为四件齐全之一（首次小结锁定门槛）/ 现已**软销**
- **实质性反思不再作为锁定门槛** —— 学员瞎答 / 应付式跳过也不会阻塞首聊态走完
- **Neo 内部记录反思质量** —— 反思质量影响**报告深度自适应**（低质量 → 报告精简 / 高质量 → 报告丰富 / 详见各章"状态自适应"子条）
- 首聊态一次性走完 / 即使学员状态不佳也不会重新触发（保留永久锁定的设计意图）

**Walk-through 节奏（5 步对接前 5 章 / 第 6 章首聊态时不参与）**：

```
1. Neo 第 1 句：定调 + 邀请反思
   "我们一起看看这份小结。报告里讲到整体判断是——
    {一句话综合 + 必要时跨课程引用}。
    问你一件事：这句话听起来熟吗？跟你自己的感觉对得上吗？"

2. 学员回应 → Neo 引导到第 2 章 这一段我学了什么
   "报告里讲到你这一段学到了什么……
    特别是这几个工具的场景应用关联……"

3. 引导到第 3 章 这一段我学得怎么样
   "我们先看你学得怎么样——
    报告里讲到你在 [认知/能力/行为] 这几个维度上从 [学前状态] 转化到 [学后状态]……
    高光时刻在 [Activity X 的对话片段]……"

4. 引导到第 4 章 这一段我表现怎么样
   "再看看你这一段的上课状态——
    数据上看你互动了 N 次 / 6 维画像在 [维度] 上 [突出 / 偏低]……
    学习风格上你属于 [边练边问型 / 先理论后实战型]……"

5. 引导到第 5 章 我接下来该做什么 + 任务卡片逐条推
   "基于以上这些，我给你列了 2-3 件下一步可以试的事——
    （第 1 条便签朗读）→ 推第 1 张任务卡片 → 学员选择接受/不接受
    （第 2 条便签朗读）→ 推第 2 张任务卡片 → 学员选择接受/不接受
    ...
    我把这几件事做成任务卡片推到你面前，你已经选好接收哪几条了。"
```

**关键点**：

- 首聊态是**一次性事件** —— 走完即转入自由复习态，不重复
- 学员中途退出 walk-through → 重进时**从中断点续** walk-through（不从头）
- **没有"重听 walk-through"机制** —— 首聊态完成后通过 Neo 自由问答覆盖学员需求 / 这是设计意图

**留白式追问规范**：

Neo 追问的核心：**不替学员说总结，引发学员自陈**。

| 类型 | 示例 |
|------|------|
| 引发观察 | "你看到这条洞察吗？想到了什么？" |
| 引发反思 | "为什么你觉得这维度低？" / "这个模式是不是熟悉的？" |
| 引发联想 | "这跟你上周在大厅说的那件事有关系吗？" |
| 引发承诺 | "你觉得接下来一周能做的一件事是什么？"（首聊态收尾）|

**必收尾约束**：Neo 追问之后**必须**给收尾——不能只追问留开放状态（沿用 practice 复盘 [05-learner/04-practice § 1.6.5]）。首聊态收尾 = 给观察 + 留言 + 引导学员看完第 5 章便签 + 任务卡片同步推送学员选择。

**学员动作**：Walk-through 跟随 / 主动追问 / 任务卡片逐条选择是否接受 / 全程右栏 Chat 可问 Neo / 中途退出 walk-through → 重进时从中断点续。

#### 1.3.5 自由复习态 Neo 行为

> **本节主讲 Neo 在自由复习态的解读与辅导行为**。自由复习态包含复习态 + 回访态二级切换（详见 § 1.3.2）。

**Neo 在自由复习态的核心定位**：

- 不再走 Walk-through（首聊态已一次性走完）
- 学员主导对话节奏 / Neo 被动答疑 + 主动辅导
- 重点是**对报告内容的深入解读 + 对学员实际工作场景的针对性辅导**

**复习态 vs 回访态 Neo 行为差异**：

| 状态 | 触发 | Neo 主动开场 | Neo 行为侧重 |
|------|------|------------|------------|
| **复习态**（无更新）| 学员再次进入 / 系统判定无新增内容（无红点）| 不主动开场 | 学员主导问答 / Neo 被动响应 |
| **回访态**（有更新）| 学员再次进入 / 系统判定有新增（红点）| **主动指出新增内容** + 引用上次对话 + 询问近况 | 主动同步更新点 + 学员问答 |

**Neo 的 4 类辅导行为**（复习态 + 回访态共用）：

| # | 辅导类型 | Neo 做什么 | 触发场景 |
|---|---------|------------|---------|
| 1 | **报告解读** | 解释报告中某章某句的含义 / 拆解判断依据 | 学员问"为什么这么说" / "这一段什么意思" |
| 2 | **跨场域应用辅导** | 基于学员当前工作场景的实操建议 + 对学员应用工具尝试的反馈 | 学员说"我最近在 X 场景遇到 Y 问题"/"我试了 PIN 法但..." |
| 3 | **解锁还有疑问** | 针对第 2 章 C "还有疑问的"做深入讲解 / 苏格拉底式追问引导学员自己想通 | 学员问到第 2 章 C 中提到的疑问点 |
| 4 | **新增数据回访** | 学员重新进入关联 Activity 复习产生新学习行为时主动指出变化点（新尝试 / 新理解 / 新高光卡点）+ 引用上次对话 + 询问近况 | 回访态触发（有红点）|

**回访态主动开场示例**（占位符）：

```
（学员第二次进入 recap / 系统判定为回访态 / 红点亮起）

Neo: "好久没见！上次我们聊到 {上次焦点话题}，你后来 {上次结论 / 任务卡片接受情况}。
     最近我看到你重新进了 {Activity X} 练了一次——这次有几个新东西我想跟你说一下：
     ① ...
     ② ...
     你想先聊哪个？"
```

**Neo 行为约束**（**总约束**详见 § 1.3.1 / 本节仅列自由复习态**特定约束**）：

- **不重讲首聊态 Walk-through**（首聊是一次性结构化讨论 / 学员"想再听一遍"通过 Neo 自由问答覆盖）
- **不强行追问**（学员不主动来 Neo 不强行开场 / 复习态完全被动 / 仅回访态主动同步更新点）

**与首聊态报告的关系**：

- 首聊态生成的前 5 章**永久不变** —— Neo 解读时引用其内容，但不修改报告本身
- 第 6 章**覆盖式更新** —— Neo 在自由复习态对话中产生的新观察、学员重新进入关联 Activity 复习产生的新学习行为、新高光新卡点等，全部进入第 6 章覆盖式更新（详见 § 1.2.8）

**学员侧动作**：

- 主动提问 Neo / 跨场域追问 / 对报告章节进行深入讨论
- 笔记悬浮球记录"想留住此刻"
- Neo Chat 历史可翻看（Activity 内不被清空）

#### 1.3.6 任务卡片推送

> 任务卡片推送是首聊态的核心动作之一 —— Walk-through 第 5 步走到第 5 章时**逐条推 + 学员逐条选择**。

**推送规则**：

| 维度 | 说明 |
|------|------|
| **推送时机** | Walk-through 走到第 5 章时 / **第 5 章逐条推** |
| **推送形式** | Neo Chat 中以任务卡片形态出现 / 沿用 [05-learner/06-cross-context § 4.4] 任务卡片机制 |
| **学员决策粒度** | **逐条选择** —— 每张任务卡片单独决策"接受 / 不接受" |
| **决策完成时机** | **Walk-through 内完成** —— 不放到完成 Modal / 不批量推 |
| **数量** | 2-3 条（与第 5 章便签一一对应）|

**推送流程**：

```
Walk-through 走到第 5 步（第 5 章 我接下来该做什么）
  ↓
Neo 朗读第 1 条便签 + 推第 1 张任务卡片
  ↓
学员选择：接受 / 不接受
  ↓
Neo 朗读第 2 条便签 + 推第 2 张任务卡片
  ↓
学员选择：接受 / 不接受
  ↓
（如有第 3 条）继续
  ↓
Walk-through 第 5 步收尾 / Activity 完成事件触发（不依赖学员选择结果）
  ↓
接受的 → 加入大厅待办清单 → 学员主动点掉时 Neo 跟进问情况
不接受的 → 仅留在 recap 报告里（永久可见 / 不进大厅 / 无"再考虑"出口）
```

**学员"不接受"后的边界**：

- 不接受的任务卡片**仅留在报告里**（学员日后可在自由复习态打开报告看到第 5 章便签）
- **不提供"再考虑"中间态** —— 学员在 Walk-through 当下做即时决策（接受 / 不接受 二选一）
- 这是设计意图（学员需在当下做决策 / 学员需适应）

**便签 vs 任务卡片 状态独立**：

| 维度 | 第 5 章便签 | 任务卡片 |
|------|------------|---------|
| 性质 | 报告内文字 / Activity 完成事件触发瞬间锁定 | 派生自第 5 章便签的大厅待办对象 |
| 可变性 | **永久不变**（不论卡片接受与否）| **状态独立流转**（接受 / 不接受 / 已点掉 / 已完成 等）|
| 影响关系 | 卡片状态变化**不影响**便签锁定 | 便签锁定**不影响**卡片在大厅的流转 |

#### 1.3.7 学员反驳报告 Neo 回应

> 学员在自由复习态可能反驳报告内容（如"我不同意第 3 章对我能力的判断"）—— Neo 必须坚持立场 + 举证回应 / 不当应声虫 / 不修改报告。

**回应规则**：

| 规则 | 说明 |
|------|------|
| **不改报告** | Neo 不修改报告内容 / 报告是已锁定的快照 |
| **不当应声虫** | Neo 不因学员反驳就改口 / "你说得对，那我们就改"是错误回应 |
| **必须举证当时状况** | Neo 引用具体证据（**哪段对话 / 哪个时间点 / 哪个行为**）说明判断依据 |
| **说明基于当时情况评判** | Neo 强调"这是基于当时的状况评判 / 你现在可以用行动证明改变" |
| **学员反驳进入下次 recap 数据池** | 学员的反驳被记录 / 下次 recap 报告（如有 / 在第 6 章覆盖式更新中）会反映这点（"上次反驳了 X / 这次看到 Y 变化"）|

**回应示例**（占位符）：

> 学员："我不觉得我'反复打磨型'，那次只是因为时间不够才反复改。"
>
> Neo："**报告里这么写是基于** 你在 Activity X 那次对话——你重复了 3 次同一个工具的应用 / 在 {时间点 Y} 跟我说'我再来一次试试'。**当时的情况看起来是反复打磨**。当然这只是**当时**的判断，你可以用接下来的 Activity 给出新的样子——下一次 recap 我们再看。"

**与"像人 > 懂你"原则的对接**：

- 当学员希望 Neo 像朋友（"你别那么严"）时——Neo 在场域里仍是 1v1 老师 / 有立场 / 不为讨好学员而软化
- "像人"原则要求 Neo 保持立场（详见 [02-foundation/01-personas § 1.2]）

#### 1.3.8 进度记录 + 结营前必须完成

> recap 完成是**进度组件**，与学员体验完成解耦——但项目结营前必须完成。

**进度记录规则**：

| 规则 | 说明 |
|------|------|
| **报告生成 = 系统进度** | 关联 Activity 全完成 / 报告后台生成并锁定后 / 系统进度即记录"recap 报告已生成" |
| **学员体验完成 ≠ 系统进度** | 学员是否进入 recap / 是否走完 Walk-through 是**学员体验**事件 / 但学员体验完成度也独立记录到进度组件 |
| **管理端可读完整报告** | Ora / 管理员可在管理端读已生成已锁定的报告（详见 § 1.2.10）/ 不依赖学员是否进入 |
| **结营前必须完成** | 项目结营前学员必须完成 recap（即首聊态走完） / 大厅在结营前会催促学员完成未走 Walk-through 的 recap |
| **大厅催促规则** | 详见大厅催学机制（[03-lifecycle/02-learning § 1.5]）/ recap 不重复定义 |

### 1.4 界面布局

recap 沿用学员端"各场域统一布局" —— Topbar + 全幅课件区 + 右侧 Neo Chat + 左侧覆盖式抽屉 + 顶部 Subbar。

#### 1.4.1 单一两栏布局

```
┌─────────────────────────────────────────────────────────────────────┐

│抽│  课件区（左侧）                       │  Neo Chat（右侧）          │

│  │  · 嵌入 6 维画像（在第 4 章 / T1 快照）│  · Neo 主动开场或被动应答  │
│  │                                       │                           │
│  │                                       │  [输入框：和 Neo 聊聊……]   │
└──┴──────────────────────────────────────┴───────────────────────────┘
```

**与 lecture / practice 布局的差异**：

- **不分阶段切布局**（与 practice 4 阶段不同布局相反）—— 全程单一两栏
- **课件区承载报告而非学习内容** —— 报告是 Neo 引导对话的"道具"

#### 1.4.2 共用元素

**左侧覆盖式抽屉**（沿用 [05-learner/03-lecture § 1.2.2] lecture 抽屉设计）：含两 Tab——

- **Tab1：当前 Activity 目录** —— recap 没有 SCO 序列 / Tab1 改为呈现"报告 6 章骨架"作为锚点跳转（**含第 6 章 / 标注"条件性存在"** —— 第 6 章不存在时锚点**灰显 + 提示"暂无（待自由复习触发）"**）
- **Tab2：整个课程目录** —— 当前 Course 的 Activity 列表（沿用 lecture / practice）

**右侧 Neo Chat**：常驻，承载 Neo 的反思引导对话——首聊态 Walk-through / 自由复习态留白追问 / 跨场域记忆调用。

**Subbar 红点**（**📊 报告有更新**）：

- 位置：Subbar 内 recap 标识附近
- 触发：自由复习态 / 系统判定第 6 章自上次学员离开后已覆盖式更新（即回访态）
- 表现：红点 + 文案"报告有更新"
- 点击：定位课件区到第 6 章
- 学员进入并离开后系统重置该红点（下次再有更新再触发）

**笔记悬浮球**：屏幕边缘常驻，跨场域形态一致（详见 [05-learner/02-hub § 1.3]）。

### 1.5 全局通用

#### 1.5.1 学员行为 + 体验机制约束

**输入方式**：

| 方式 | 交互 |
|------|------|
| **右栏 Neo Chat 输入框** | 学员与 Neo 对话（打字 / 语音）—— recap 内学员主要通过对话表达 |

**核心交互**（多数沿用 lecture 设计 [05-learner/03-lecture § 1.5.2]）：

| 交互 | 说明 |
|------|------|
| **解锁后进入** | recap Activity 入口在前序未完成时锁定 / 前序完成后**学员需重新点击进入**（不自动跳转）|
| **防挂机** | 默认 5 分钟无操作 → 系统弹提醒（沿用 lecture / 可在 program config 调整阈值）|
| **学员主动提问 Neo** | 全程右栏 Chat 可问 Neo |
| **抽屉锚点跳转** | 学员点抽屉 Tab1 报告 6 章锚点 → 课件区滚动到对应章（第 6 章不存在时锚点灰显）|
| **Activity 间跳转** | 通过抽屉 Tab2 主动跳转其他 Activity（弹窗确认 + 触发断点续播保存）|
| **断点续播** | 中途退出 → 系统记录当前对话上下文 + Walk-through 进度；下次进入自动恢复（首聊态从中断点续 / 复习态恢复对话历史）|
| **任务卡片选择** | Neo 推送任务卡片后逐条决策 / 接受 → 加入大厅待办 / 不接受 → 仅留在报告里 |
| **笔记悬浮球** | 屏幕边缘常驻（沿用 [05-learner/02-hub § 1.3]）|

**体验机制约束**：

| 约束 | 说明 |
|------|------|
| **取消 SCO 锁定** | recap Activity 内部不存在 SCO 序列；与 lecture / practice 内部严格顺序锁定不同 |
| **入口前置锁定** | recap Activity 入口在前序未完成时锁定 / 不展示半成品报告 |
| **6 维画像可见性** | 学员侧 recap 报告露出**仅第 4 章 T1 锁定快照**（第 6 章自由复习不再呈现 6 维任何形式 / 理由：变化触发难判断）；管理端通过**已生成已锁定**的报告间接可见 / 管理端独立 6 维画像 widget = [06-management/02-home § 1.2.2] (4) 个体详情面板内的 6 维画像快照 |
| **永久锁定** | 已锁定的前 5 章不可重新触发 / 学员不能"重新触发首次小结" / 这是设计意图 |

**通用沿用**：

- **防挂机**：默认 5 分钟无操作触发（沿用 [05-learner/03-lecture § 1.6.4]）
- **数据持久化**：推荐每个会话 / Walk-through 节点完成时落库（沿用 [05-learner/03-lecture § 1.6.6]）
- **隐私三层**：沿用 [04-global/01-user-global § 3.4.3]（学员之间默认完全隔离 / 跨学员引用需同意 / 管理端例外）

**变化叙事三态写法**（适用于报告内涉及变化 / 对比 / 评价的所有章节）：

| 变化方向 | 写法 |
|---------|------|
| **进步** | 直说能力提升 + 举证 |
| **持平** | 写"**稳定保持**" + 稳定点（不写"无变化"、"维持原状"等冷感表达）|
| **退步** | 软化为"**近期波动**" + 具体证据（**不直接说"退步"**）|

适用范围：第 3 章 A 维度对比 / 第 3 章 D 学习成长总评 / 第 4 章 D 状态变化 / 第 6 章学员复习行为洞察 / Neo 反驳举证回应 等所有涉及变化叙事的位置。

#### 1.5.2 完成 Modal

学员首次走完 Walk-through 触发首聊态完成时弹出完成 Modal：

```
┌─────────────────────────────┐
│                        [X]  │
│         ✅                   │
│   Course 小结完成！           │
│  你已完成「{Course 小结名}」  │
│  Neo 已为你产出 {N} 条应用计划│
│  其中 {M} 条已加到大厅待办    │
│                              │
│  [回到大厅]                   │
│  [自由复习]                   │
│  [继续下一 Activity]           │
└─────────────────────────────┘
```

**文案动态规则**：

- 学员接受 ≥1 条 → "Neo 已为你产出 {N} 条应用计划 / 其中 {M} 条已加到大厅待办"
- 学员接受 0 条 → "Neo 已为你产出 {N} 条应用计划 / 它们留在报告里 / 你可以稍后回头看"

按钮行为沿用 lecture / practice 3 选项设计（详见 [05-learner/03-lecture § 1.7.2] + [05-learner/04-practice § 1.8.7]）。

> **"自由复习"统一概念**：lecture / practice / recap 完成 Modal 里的"自由复习"按钮 + recap "自由复习态"状态，三者指向同一概念——**Activity 流程结束之后，学员允许回到该场域针对 Activity 内容和已经发生过的事情和 Neo 进行探讨**。

回到大厅：默认进大厅首页（项目总览区已含待办清单），不为 recap 跳转打破默认体验。

#### 1.5.3 跨场域衔接

> 全局跨场域机制详见 [05-learner/06-cross-context § 3.4] 跨场域协同（Truth Source）；本节仅从 recap 侧视角列入口与出口。

| 衔接 | 数据流 |
|------|--------|
| **关联 Activity（lecture / practice）→ recap** | 报告数据流：高光库子集 / 6 维画像 T1 快照 / 学员互动行为日志 / practice 复盘待改进点 / 跨场域感知 全部进入报告生成（详见 § 1.2 各章数据来源）|
| **recap → 大厅** | 完成状态 + 报告进入大厅"我的报告"主题（[05-learner/02-hub § 1.2.3]）；应用计划经学员选择接受后进入大厅待办清单（详见 § 1.3.6 + [05-learner/02-hub § 1.2.1] 待办清单子条）|
| **recap → 管理端** | team report 拉**已生成已锁定**的 recap 数据；首页 [06-management/02-home § 1.2.2] (4) 个体详情面板（行内展开含报告列表）展示已锁定报告；数据共享但呈现完全分开（详见 [06-management/03-report-center § 1] / § 1.2.10 管理端可见性条）|

> **与大厅的边界**：项目进度类信息由大厅 [05-learner/02-hub § 1] 承担，recap 不承担也不重复（详见 § 1.2.10）。
