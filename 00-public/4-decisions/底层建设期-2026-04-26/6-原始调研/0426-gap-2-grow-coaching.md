## 7 动作 × GROW 素材范例表

### 一、讲授

GROW 素材的**主要供给**。BP/SC 全文都是讲授脚本，密度高、隐喻强。

- **范例 1（隐喻型讲授）**：M1_SC Block 1.1 用"司机 vs GPS"开场——"传统管理就像是你在开车...而高效辅导式管理，应当像 GPS 导航，你提供地图和方向，但把驾驶权交给员工。"
- **范例 2（对比型讲授）**：M1_SC Block 2.2 "指责是锁住过去的枷锁，辅导是开启未来的钥匙"——天平视觉 + 金句口播，结构化对比"过去/未来"两种焦点。
- **范例 3（结构化讲授）**：M1_SC Block 5.2-5.5 GROW 四步原子化解构，每步都有"定义 + 关键原则 + 反例 + 话术工具"四件套。
- **范例 4（流程型讲授）**：M2_SC Block 6.3 询问许可"四步阶梯"——确认枯竭 → 询问许可 → 提供资源 → 归还决策，可直接作为 Neo 教学时的 step-by-step 模板。
- **范例 5（心理学讲授）**：M3_SC Block 4.1 "吊桥隐喻"——"吊桥的控制权完全在内部，你无法从外部强攻"——用单一隐喻把"心理防御"复杂概念讲透。

### 二、解惑

GROW 素材几乎不直接做"解惑"——因为是单向脚本，不是真实对话。但有 **2 类内嵌解惑**：

- **范例 1（误区破除）**：M1_BP Segment 5.3 反复强调"主观臆断 vs 客观事实"，并解释"为什么管理者会犯这个错"——这是对学员潜在疑问的预先解惑。
- **范例 2（Quiz 后解析）**：M1_BP Segment 6.1 Quiz 答案解析——"虽然提到了'目标'，但问题的核心在于盘点当前的障碍和资源，即理清'我们在哪里'"——这是对学员可能误判的针对性解惑。
- **范例 3（潜台词回应）**：M2_BP Segment 6.3 解释"为什么必须先询问许可而不是直接给建议"——本质上是回答学员心里的"我都懂了为啥还要敲门"的疑问。

**缺口**：GROW 素材**没有真实"学员提问 → 老师答疑"的多回合对话**。所有解惑都是讲师预先设想再单方面输出。

### 三、讨论

**最薄弱的动作**。GROW 单向脚本几乎无讨论。仅在 Quiz 设计中有"软讨论"信号：

- **范例 1**：M1_BP Segment 3.1 Quiz Type A 设计意图——"不设置标准选项，而是通过直击灵魂的二选一反思"，意图引发"心里的对话"，但**不期待学员真讨论**。
- **范例 2**：M2_BP Segment 9.1 综合应用挑战 Quiz 设计——"要求学员能按顺序识别正确步骤或纠正错误辅导行为"，预设了"标准答案"，本质仍是单向考核而非讨论。
- **范例 3（潜在讨论种子）**：M1_BP Segment 5.4 "Options 阶段不断追问'还有吗？'"——这条原则若移植到 Neo×学员对话场景，可催生真讨论；但素材中只是讲给管理者听的方法论。

### 四、考研（考核 / 验证）

Quiz 设计较系统，分 Type A（思考型）和 Type B（验证型），是 GROW 素材里**质量最高的"考研"动作设计**。

- **范例 1（Type A 思辨题）**：M1_BP Segment 3.1 "面对下属项目延期，你脱口而出的第一句话通常接近哪种意图？" —— 不评判对错，逼学员自照。
- **范例 2（Type B 概念识别题）**：M1_BP Segment 6.1 "为了实现这个目标，你觉得有哪些障碍？" 属 GROW 的哪一环？ —— 检验对 Reality 与 Options 的混淆。
- **范例 3（情境判断题）**：M2_BP Segment 3.1 "下属小李说'想试着追回来但不知从哪下手'，是否启动 GROW？" —— 验证"双高诊断"信号识别。
- **范例 4（行为纠错题）**：M2_BP Segment 7.1 "员工方案有盲点你心很急，应该怎么做？" —— 验证"询问许可"机制是否内化。
- **范例 5（找漏题）**：M2_BP Segment 9.1 "员工列出三个方案你说'就按这三个去办'——遗漏了哪个最致命环节？" —— 验证流程完整性。

### 五、练习

**真练习几乎没有**。SC 标注"互动练习环节，无讲师口播"——意味着练习交给平台执行，BP/SC 不提供真练习剧本。仅 Case 视频分镜可视为"观察式练习"。

- **范例 1（观察式练习）**：M1_BP Segment 4.1 视频分镜——王经理两次处理同一问题（咆哮 vs 引导）。学员看完视频"应该能识别 GROW 各环节"，是被动练习。
- **范例 2（情境分镜）**：M3_VS Segment 5.1 "赵抗拒的防线崩塌"——3 段分镜（防御期 / 转折期 / 融合期）配合心理学标注，**最接近"对练剧本"**，可改造为 Neo 与学员的角色扮演训练。
- **范例 3（话术示范）**：M3_BP Segment 4.2 提供完整自我归因话术——"老实说，最近在这个项目的推进上，我自己也感到非常困扰..."——可作为 Neo 让学员"复述/改写"的练习材料。

**结构性缺口**：GROW 素材没有"学员说一句 → 老师反馈一句 → 学员再改"的闭环对练设计。

### 六、追问

**GROW 教练素材的最强项**，三个 MANAGER 都把"追问"做成了核心方法论。

- **范例 1（What Else 追问循环）**：M2_BP Segment 6.1 "挤牙膏战术"——"机械式重复'还有吗？'，直到员工明确表示'没有了'"。配视觉隐喻"最有价值的方案，往往在最后一次用力时才会出来"。
- **范例 2（事实追问）**：M1_BP Segment 5.3 "请暂停一下'我觉得'。如果摄像机在现场，它拍到的是什么？" —— 用追问把主观臆断转客观事实。
- **范例 3（验收追问）**：M1_VS Segment 5.2 SMART 话术工具——"如果我们要为此付钱给第三方，你觉得怎样才算交付合格？验收标准是什么？" —— 用第三方视角追问目标清晰度。
- **范例 4（深度反思追问）**：M2_BP Segment 4.1 "钻头追问"——从外部环境（"配合度低"）向内部归因（"沟通机制缺失"）的钻取追问。
- **范例 5（信心追问）**：M2_BP Segment 8.1 "0-10 分你有几分把握？" —— 把意愿强度量化的追问，且设定"低于 7 分必须回炉"的硬规则。
- **范例 6（障碍追问）**：M3_BP Segment 6.3 "你的阻碍是什么？" + 区分"心理性阻抗 vs 资源性瓶颈"——从笼统答应深挖到具体障碍。
- **范例 7（认知失调式追问）**：M3_BP Segment 6.2 三步追问——"如果我没理解错的话..."（陈述事实）→ "这会导致下个季度..."（推演影响）→ "你觉得我们该怎么做？"（触发反思）。

**这是 GROW 素材最值钱的部分。** Neo 应该把"追问句库"全量提取并按场景分类。

### 七、反思

GROW 素材有**理论上的反思设计**，但落地依然靠学员"自我对照"。

- **范例 1（自我对照式反思）**：M1_BP Segment 3.1 "你脱口而出的第一句话通常接近以下哪种意图？" —— 强迫学员自照本能反应。
- **范例 2（认知失调反思）**：M3_BP Segment 6.1 "认知失调诱导"——通过"主观信念"与"客观事实"对撞，迫使员工修正信念。这是设计反思的标准心理学路径。
- **范例 3（角色身份重塑）**：M1_BP Segment 7.1 "管理者的价值：别做天花板，要做梯子" 与 M2_BP Segment 10.1 "你的新角色：思维催化剂" —— 用结尾金句触发身份层反思（Identity Shift）。
- **范例 4（隐性反思埋点）**：M3_BP Segment 9.1 结课金句"抗拒往往不是因为'不想做'，而是因为'不安全'" —— 把整门课的洞察压成一句钩子，反复回响。

## 关键发现（10 条）

1. **GROW 素材是教案不是真实对话**——BP（教学设计）和 SC（讲师口播）本质都是"讲师独白"。SC 中明确标注"互动练习环节，无讲师口播"——说明真实交互被外包给平台。**Neo 训练数据中"真实多回合对话"在此处稀缺**。

2. **Case 视频分镜是最接近"真实 1v1 对话"的部分**——M1_BP Segment 4.1（王经理 vs 李萌新）、M3_VS Segment 5.1（王经理 vs 赵抗拒）、Segment 7.1（认知调频）三段分镜含完整对话流转，可直接改造为 Neo 训练剧本。

3. **3 个 MANAGER 是难度递进的同心圆**：M1（认知/骨架）→ M2（流程/技能）→ M3（心理/破障）。这种"先学骨架，再练标准动作，最后破例外"的设计逻辑可直接借鉴到睿学课程结构。

4. **追问是 GROW 教得最好的动作**——What Else 循环、SMART 第三方追问、信心打分、障碍深挖、认知失调三步法——全是高质量句库。Neo 应建"追问场景库"。

5. **讨论和真练习是 GROW 教得最弱的动作**——单向脚本本质决定了无法生成讨论；练习被外包给"互动环节"占位符。**这恰是 Neo 相对线下教练的差异化优势所在**。

6. **GROW 没出现一段"教练翻车"的真对话**——所有 Case 都是"理想剧本"：示弱→员工放下防御→达成共识。**真实 1v1 老师必然会遇到学员防御不破、追问被怼回、认知失调反弹的场景**——这是 Neo 训练数据的最大缺口，需要专门补充"反面剧本"。

7. **隐喻系统是高质量"讲授知识胶囊"**——司机/GPS、天平、梯子、挤牙膏、吊桥、化学催化剂、起跑器——每个隐喻都是单 token 可激活的认知锚点，Neo 可直接复用作为讲授武器库。

8. **Quiz 设计有清晰的 A/B 分类**——Type A 思辨型（无标准答案，引发思考）、Type B 验证型（有标准答案，识别概念）。这套二分法可直接搬到 Neo 的"考研"动作设计。

9. **"询问许可"机制是 1v1 老师的核心心法**——M2_BP Segment 6.3 的"敲门四步"防止了"老师抢答"。**Neo 同样会面临"忍不住给答案"的诱惑，这套机制是 Neo 主体责任感设计的核心**。

10. **"先心情，后事情"是被动型员工辅导的元法则**——M3_SC 反复强调"在防御消除前任何逻辑说服都无效"。Neo 在面对学员焦虑/抵触时也应优先处理情绪，而非直接讲方法。

## 给睿学 Neo 的可复用清单

### 直接复用（可作为 Neo 训练数据）

- **追问句库**（最高价值）：M2_BP/SC 6.1-6.3 + M1_BP/SC 5.3-5.5 + M3_BP/SC 6.1-6.3 中所有"话术工具"——共约 30+ 条标准追问句，可直接作为 Neo 输出 token。
- **Case 5/6 对话剧本**：M3_VS Segment 5.1 + 7.1 的三段分镜对话——示弱 → 放下防御 → 价值认可 → 认知失调 → 障碍挖掘 → 锁定行动——是完整的"真实 1v1 对话"训练样本。
- **Quiz Type A/B 设计模式**：M1_BP Segment 3.1（思辨型）+ M2_BP Segment 9.1（找漏型）的设计意图段，可直接作为 Neo "考研"动作 prompt 模板。
- **隐喻视觉资源**：12+15+14 张 URL 中的关键隐喻图（司机/GPS、梯子、挤牙膏、吊桥、化学反应）—— 可 base64 内嵌到原型当 Neo "讲授"时的辅助卡片。

### 改造复用（需要改写为 Neo 风格）

- **王经理"咆哮 vs 引导"对比剧本**（M1_BP 4.1）—— 把"王经理 → 李萌新"改造成"Neo 自评 vs Neo 引导"的两种应答示例，让学员看到 Neo 应该长什么样。
- **吊桥隐喻话术**（M3_SC 4.1）—— 改为 Neo 在察觉学员防御时的开场话术："我注意到你今天回应比较短，是不是哪里我没解释清楚？"——把"管理者面对员工"语境转为"Neo 面对学员"。
- **询问许可四步**（M2_BP 6.3）—— 直接改写为 Neo 给"补充资源/参考资料"前的协议："关于这点我有个补充视角，方便分享吗？"——保护学员主体责任感。
- **0-10 信心打分**（M2_BP 8.1）—— 改造为 Neo 在每节课结尾的"理解度打分"，低于 7 触发自动回炉。

### 启发设计（不直接用但提供设计灵感）

- **"双高诊断"模型**（M2_BP 2.1）—— 责任意愿 × 问题意识四象限，可启发睿学"学员状态诊断"模型（如学员的"投入度 × 掌握度"四象限分流不同 Neo 策略）。
- **"认知失调诱导"3 步法**（M3_BP 6.2）—— 启发 Neo 在学员说"我都懂了"但实际未掌握时的破局路径——用客观数据/quiz 错题制造失调，触发反思。
- **"心理性 vs 资源性"障碍二分**（M3_BP 6.3）—— 启发 Neo 处理学员"完不成作业"时区分"懒/怕/焦虑"（心理）还是"没时间/没工具"（资源）。
- **递进 3 课设计**（M1→M2→M3）—— 启发"骨架 → 标准动作 → 破例外"的睿学课程开发节奏。

## 缺失清单（GROW 没覆盖但 1v1 老师需要的）

1. **学员真实多回合对话**——GROW 全是讲师独白 + 理想 Case 视频。**真实 1v1 必然 5-15 轮你来我往**，需补充。
2. **教练翻车 / 学员防御不破**的反面剧本——GROW 里赵抗拒"刚好被一句示弱破防"是理想化设计；真实场景需要 Neo 试 2-3 次才能破防的训练数据。
3. **学员主动提问 / 偏题 / 反问老师**的处理路径——GROW 全部场景都是老师主动追问员工，**学员主动发问的反向场景没覆盖**。Neo 必须能优雅处理"老师这题我不会，能直接告诉我答案吗？"
4. **知识层面的 1v1 教学**——GROW 是"管理对话"，不是"知识传递"。讲一个具体业务知识点（比如"什么是用户分群"）的 1v1 对话教 GROW 没教。
5. **学员无意愿但又不能放弃**的场景——GROW M3 假设的是"资深老员工有抵触但可以被认可击穿"。**新员工/低能力员工的"我不想学也不想被认可"场景没覆盖**。
6. **正向反馈机制**——GROW 整门课讲"不要评判"，但反过来"如何给到位的鼓励"没设计。Neo 在学员答对/进步时的反馈话术需自建。
7. **教学进度的暂停/重启/跳过决策**——GROW 是 1 小时单向授课，没有"学员中途说我现在不想学"的处理。Neo 必须能感知并处理学习节奏。
8. **跨场域上下文衔接**——睿学有授课/对练/调研/报告 4 场域，GROW 全在"授课"场域单点设计，跨场域知识连续性需自研。
9. **识别"学员假装会了"的能力**——GROW 假设员工说"我懂了"就是真懂。Neo 必须能用追问验证真假。M3 的"认知失调诱导"是雏形，但需要更轻量的日常版本。
10. **学员情绪兜底**——M3 教了"自我归因示弱"破防御，但没教"学员真的崩溃了/哭了/愤怒了"的兜底。Neo 需要类似 ChatGPT 安全机制的情绪兜底设计。

---

**总结**：GROW 教练课对 Neo 的最大价值在 **追问 + 隐喻 + 询问许可机制 + 心理穿透话术**这 4 块；最大缺口在 **真实多回合对话、翻车剧本、知识传递场景、学员主动发问**这 4 块。建议下一步从 M3_VS 的 Case 5/6 分镜入手做 Neo 对话训练样本的初版改写。
