## 一、GPB 架构全貌

GPB（Global Platform Base）是绚星的统一账户基础设施，采用六层实体模型：

```
自然人(User) — 全局唯一标识
    │
    ├── 账户(Account) — 身份凭证（手机号/邮箱/三方OAuth）
    │
    ├── 组织/租户(Tenant) — 数据隔离、计费基本单元
    │   ├── 成员(Member) — 用户加入组织后的身份
    │   ├── 角色(Role) — RBAC 角色
    │   └── 权限(Permission) — 具体操作许可
    │
    └── 权益(Entitlement) — 订阅产品获得的功能和配额
        ├── 功能(Feature)
        └── 配额(Limit)
```

**核心关系**：一个 User 可持有多个 Account（手机号+邮箱），可加入多个 Tenant，每个 Tenant 内有一个 Member 身份并绑定 Role。

## 二、已有能力清单

| 能力 | 状态 | 说明 |
|------|------|------|
| 手机号注册/登录 | **已实现** | 支持国际号段，验证码登录，国内版主力方式 |
| 邮箱登录 | **已实现** | 验证码方式 |
| 企业 SSO | **已实现** | 输入企业域名跳转企业认证 |
| Google/Apple/Microsoft | **已实现（仅海外）** | 国内版不支持 |
| 多租户 | **已实现** | 一个用户可关联多个租户，登录后选择进入哪个 |
| 自动创建租户 | **已实现** | 注册后自动为用户创建一个默认租户 |
| 成员邀请 | **已实现** | 支持邀请加入组织 |
| RBAC 角色权限 | **已实现** | 支持角色-权限映射 |
| 产品隔离 | **已实现** | 同一租户下不同产品可有不同权限 |
| 统一登录域名 | **已实现** | `login.radnova.com/产品名` |
| 个人管理中心 | **已实现** | 统一管理账号信息 |

**GPB 注册流程（国内版）**：

```
输入手机号 → 验证码 → 检查关联组织
  ├── 无组织 → 自动创建一个组织 → 进入
  ├── 1个组织 → 直接进入
  └── 多个组织 → 选择要进入的组织 → 进入
```

## 三、AI 老师对接方案

### 3.1 概念映射

| AI 老师概念 | GPB 概念 | 说明 |
|------------|---------|------|
| 培训项目(Project) | 租户(Tenant) | 每个项目 = 一个租户，数据完全隔离 |
| 学员 | Member + 学员角色(Role) | 项目内的普通成员 |
| HR | Member + HR角色(Role) | 有管理看板权限 |
| 运营 | Member + 运营角色(Role) | 有运营工具权限 |
| 权限控制 | RBAC | 一个角色 = 一种权限集合，不做细粒度 |

### 3.2 推荐对接流程

**项目创建流程**：
```
运营创建项目 → GPB 创建新租户 → 导入学员名单(Excel)
→ GPB 为每个学员创建账号 + 发送邀请 → 学员通过手机号验证码登录
```

**登录流程**：
```
学员收到链接 → 打开 login.radnova.com/ai-teacher
→ 输入手机号 + 验证码 → GPB 验证
→ 1个项目(租户) → 直接进入
→ 多个项目(租户) → 选择进入哪个
→ 进入 AI 老师学员端
```

**角色分配**（项目创建时由运营指定）：
- **学员角色** → 看到学员端（大厅 + 教室）
- **HR 角色** → 看到 HR 管理端
- **运营角色** → 看到运营管理端

### 3.3 三个角色的权限集合

| 权限 | 学员 | HR | 运营 |
|------|------|-----|------|
| 学员端大厅（辅导） | Y | Y（可切换） | N |
| 学员端教室（授课/对练/测评/报告） | Y | Y（可切换） | N |
| HR 管理端（报告/数据看板） | N | Y | N |
| 运营端（催学/任务下发/Dashboard） | N | N | Y |
| 项目创建/账号管理 | N | N | Y |

## 四、差距清单

| # | 差距 | GPB 现状 | 解决方向 | 优先级 |
|---|------|---------|---------|--------|
| 1 | **租户归档/冻结** | 租户无"归档"状态 | 需要 GPB 新增租户归档/冻结能力，项目结束后冻结租户保留数据 | **需要新建** |
| 2 | **批量导入创建账号** | 支持单个邀请加入 | 需确认是否支持批量导入 Excel 创建账号+自动发送邀请 | **需要确认** |
| 3 | **新项目新账号矛盾** | 同一手机号 = 同一 User，自动关联多租户 | 创始人暂定"新项目新账号"，但 GPB 架构下同一手机号不可能新建另一个 User。**建议复用账号 + 多租户选择** | **需要决策** |
| 4 | **组织架构裁剪** | GPB 有完整多层级部门架构 | AI 老师不使用组织架构功能，只用角色权限。需确认 GPB 能否关闭/隐藏组织架构模块 | **已实现（不使用即可）** |
| 5 | **权益体系简化** | GPB 支持订阅制/按功能付费 | AI 老师是项目制（买一个项目 = 全部功能），权益体系需简化为"项目级授权" | **需要适配** |

### GPB vs AI 老师的关注点差异

| 维度 | GPB 设计目标 | AI 老师需求 |
|------|------------|-----------|
| 租户 | 企业长期使用 | 项目短期使用（约6个月） |
| 账号 | 用户长期保留 | 项目结束后可归档 |
| 组织架构 | 多层级部门 | 不需要，扁平角色即可 |
| 权益 | 订阅制/按功能付费 | 项目制（买一个项目=全部功能） |
| 多产品 | 同一租户下多产品 | 只有 AI 老师一个产品 |

## 五、与产品需求的映射表

| 产品需求 | GPB 支持情况 | 对接方式 |
|---------|-------------|---------|
| B1 登录 | **已实现** | 接 GPB 统一登录（手机号验证码为主） |
| B2 角色权限（3角色） | **已实现** | 对齐 GPB RBAC，定义学员/HR/运营三个角色 |
| B3 角色切换（HR→学员） | **已实现** | 通过 GPB 的角色层实现 |
| B4 项目上下文 | **已实现** | 项目=租户，多项目=多租户选择 |
| 项目创建+批量导入 | **部分实现** | 创建租户已支持，批量导入需确认 |
| 项目归档 | **需要新建** | GPB 需新增租户归档/冻结能力 |
| 数据交付（文件包） | **无关** | 属于 AI 老师业务层功能 |
