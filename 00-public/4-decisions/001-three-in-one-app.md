# ADR-001: 三端合一 Web 应用

**状态：** 已决定
**日期：** 2026-04-10
**决策者：** 产品负责人

## 背景

AI 老师产品有三类用户（学员/HR/运营），需要决定是做三个独立应用还是一个统一应用。

## 决策

学员端、HR 端、运营端是同一个 Web 应用，通过权限控制不同角色看到的入口和功能。

## 理由

- HR 需要能切换为学员身份体验课程
- 减少开发和维护成本
- 统一技术栈和数据层
- 三个角色共享同一个统一 Database

## 后果

- 正面：开发效率高，权限体系统一
- 负面：需要设计好权限和视图切换逻辑，避免角色混淆
