## 一、需要拆分的用户故事（3条 → 8条）

### 拆分1：US-1.1 学员核心体验

**原故事**：作为学员，我希望有一个 AI 教练了解我的真实工作困境，能随时回答我的问题，推荐最适合我的学习内容，以便我在实际工作中做出改变

**→ 拆分为**：
- **US-1.1a**：作为学员，我希望 AI 教练了解我的真实工作困境，能基于我的具体情境给出建议，以便我获得个性化的辅导
  - AC: Given 学员描述工作困境，When AI 回复，Then 引用学员描述的具体细节（人名/场景/情绪），不使用泛化回答
- **US-1.1b**：作为学员，我希望 AI 教练能根据我的学习进度和能力画像推荐最适合我的下一个学习任务，以便我的学习路径是个性化的
  - AC: Given 学员回到大厅，When 教练推荐任务，Then 推荐基于学员当前进度和未完成任务
- **US-1.1c**：作为学员，我希望在学习结束后能感受到自己在工作中的行为有了改变，以便培训不仅是知识传递
  - AC: Given 学员完成一个 Course，When 查看报告，Then 至少展示3个具体行为改变证据

### 拆分2：US-1.2 HR核心需求

**原故事**：作为 HR，我希望随时了解培训进度和效果，并能快速生成向 CEO 汇报的材料，以便证明培训投入的价值

**→ 拆分为**：
- **US-1.2a**：作为 HR，我希望随时了解培训项目的整体进度和每个学员的个人进度
  - AC: Given HR 登录，When 打开进度看板，Then 显示整体完成率 + 每个学员进度百分比 + 最后登录时间
- **US-1.2b**：作为 HR，我希望通过 Agent 一句话就能生成向 CEO 汇报的 PDF 报告
  - AC: Given HR 说"帮我生成本月报告"，When Agent 完成，Then 产出包含进度/效果/需关注学员的 PDF

### 拆分3：US-4.3 大厅三栏交互

**原故事**：作为学员，我希望在大厅左侧课程目录中快速找到课程，中间看板看到进度，右侧对话区随时问教练

**→ 拆分为**：
- **US-4.3a**：作为学员，我希望在大厅左侧目录区通过多层折叠导航快速找到要学的课程
  - AC: Given 学员在大厅，When 展开目录，Then 显示三层折叠导航（Course→Activity→SCO），每个 Activity 显示完成状态
- **US-4.3b**：作为学员，我希望在大厅中间看板看到进度、待办、笔记和工具卡片
  - AC: Given 学员在大厅，When 查看看板，Then 至少显示进度（百分比/环形图）+ 推荐任务卡片 + 待办列表
- **US-4.3c**：作为学员，我希望在大厅右侧对话区随时向辅导教练提问
  - AC: Given 学员在大厅任何状态，When 输入问题，Then 3秒内开始回复，基于完整上下文

## 三、矛盾的用户故事（4组）

### 矛盾1：对话区宽度调整 vs 场景预设比例

- **涉及**：US-4.4 / US-9.4（可拖拽调整）vs "本期不做个性化配置保存"
- **问题**：用户每次切换场景都被重置，可能挫败
- **建议**：同一 Session 内保持手动调整，切换场景时恢复默认值
- **补充故事**：NEW-FIX-01（Session 内保持 + 场景切换重置）

### 矛盾2：辅导教练"必然引用" vs "不打扰发呆的学员"

- **涉及**：US-9.6（必然引用）vs §9.4（不打扰发呆学员）
- **问题**：学员沉默时教练是否发起引用？
- **建议**：回大厅后等待 5-10 秒发出引用（作为开场白）；"不打扰"指之后不再追问

### 矛盾3：学员端"不展示排名" vs HR端排名

- **涉及**：§13.4（学员不看排名名次）vs US-14.6（HR 排名可排序）
- **说明**：不是真正矛盾，而是有意设计，但需确保 API 级隔离
- **建议**：学员端报告 API 不返回排名字段，仅返回分位区间标识

### 矛盾4：所有 Activity 断点续播 vs 部分 SCO 不适合断点

- **涉及**：§5.5（所有 Activity 断点续播）vs ASSESSMENT_TAG 纯后台SCO
- **问题**：后台 SCO 无前台交互，断点无意义
- **建议**：断点为 SCO 级，后台 SCO 自动跳过
- **补充故事**：NEW-FIX-02（断点从上次有交互的 SCO 之后继续）

## 六、R3 遗留问题（建议后续讨论）

1. **笔记系统设计**：AI 自动生成笔记的格式、编辑能力、导出格式需细化
2. **工具卡片体系**：工具卡片的类型、生成时机、存储和管理机制需进一步定义
3. **消息中心设计**：消息分类（催学/系统/运营通知）、已读标记、消息保留策略需定义
4. **报告渐进式呈现的具体交互**：是自动播放还是手动翻页？动画效果如何？
5. **网络异常处理机制**：离线缓存、断线重连、数据同步策略需技术评审
6. **多浏览器/多设备支持**：学员在手机和电脑间切换时断点如何同步
