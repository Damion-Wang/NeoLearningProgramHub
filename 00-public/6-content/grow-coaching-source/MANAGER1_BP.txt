# Activity： Activity 01: 从指责到赋能：GROW模型结构化重构

## Block 1: [Type: H] - [Hook: 痛点唤醒与隐喻]
* **Composition**: Pure PPT
* **Block Discription**: 本模块不直接讲理论，而是通过“保姆式管理”的痛点场景切入，利用“司机与GPS”的隐喻，直击管理者“累死自己、废了员工”的现状，引发情感共鸣与学习动机。
* **Block Intent**：作为课程的开篇（Hook），旨在打破学员的心理防御，建立“我需要改变”的紧迫感。通过强烈的反差对比，让学员意识到传统指令式管理的局限性，为后续引入教练式管理做好心理铺垫。

### Segment 1.1: [Type: PPT] - [职场巨婴与保姆经理的困境]
* **Layout Strategy**: Focus + Canvas
* **Context Prompt (内容上下文)**: 
    *   **核心痛点**：你是否觉得下属像“职场巨婴”，推一下动一下？你是否每天忙于救火，而员工却准点下班？
    *   **隐喻植入**：别做握着方向盘的司机（累死且剥夺他人掌控感），要做提供地图的GPS（指引方向但交还控制权）。
    *   **金句**：你是在培养独当一面的下属，还是在喂养一群嗷嗷待哺的“职场巨婴”？
* **Design Intent（设置意图）**：利用高唤醒度的金句和隐喻，瞬间抓住学员注意力，将“管理累”的归因从“员工不行”引导至“管理方式不对”。
* **Visual Intent (视觉与图例)**: 
    *   屏幕背景是一张具有电影质感的暗调图片，左侧展示一个满头大汗、手忙脚乱在修补漏洞的管理者背影，右侧是一群在旁边无所事事、等待喂食的巨婴形象（隐喻化处理）。
    *   画面中央用醒目的亮色大字展示金句：“你是在培养独当一面的下属，还是在喂养一群嗷嗷待哺的‘职场巨婴’？”
    *   底部用小字号展示“司机 vs GPS”的对比图标，司机图标压暗，GPS图标高亮发光。

## Block 2: [Type: E] - [Explain: 管理角色认知跃迁]
* **Composition**: Pure PPT
* **Block Discription**: 本模块详细阐述管理者需要完成的两个核心转型：从“指令下达”到“效能教练”，以及从“归咎责任”到“解决问题”。
* **Block Intent**：在引入具体工具前，先夯实“元认知”基础（Explain）。如果心智模式不改变，工具只会被滥用。本模块旨在确立辅导对话的底层逻辑。

### Segment 2.1: [Type: PPT] - [从指令下达向效能教练转型]
* **Layout Strategy**: Overview + Contrast
* **Context Prompt (内容上下文)**: 
    *   **对比维度**：左侧是“指令模式”（单向输出、扼杀思考、依赖管理者全知全能）；右侧是“教练模式”（双向启发、激发潜能、构建员工自主解决机制）。
    *   **核心动作**：摒弃“直接给答案”，确立“启发式提问”。
    *   **价值主张**：授人以鱼不如授人以渔。
* **Design Intent（设置意图）**：清晰界定两种管理模式的边界，帮助学员识别自己当前的定位，并明确转型的方向。
* **Visual Intent (视觉与图例)**: 
    *   屏幕物理中轴显示一条发光的分割线。
    *   左侧区域色调偏冷，标题为“指令下达者”，图标为一个单向的大喇叭对着耳朵喊，下方列出关键词“单向输出”、“扼杀思考”、“依赖性强”。
    *   右侧区域色调偏暖且高亮显示，标题为“效能教练”，图标为两个咬合的齿轮或灯泡被点亮，下方列出关键词“双向启发”、“激发潜能”、“自主性强”。
    *   底部有一行文字：“从‘我告诉你怎么做’到‘你觉得可以怎么做’”。

### Segment 2.2: [Type: PPT] - [从归咎责任向解决问题转型]
* **Layout Strategy**: Overview + Contrast
* **Context Prompt (内容上下文)**: 
    *   **情绪阻断**：剥离“你怎么搞的”等个人指责情绪，消除员工防御心理。
    *   **焦点转移**：从“是谁的错”（过去/沉没成本）转向“如何解决”（未来/改进路径）。
    *   **金句**：指责是锁住过去的枷锁，辅导是开启未来的钥匙。
* **Design Intent（设置意图）**：解决辅导中常见的情绪对抗问题，引导管理者将能量聚焦于建设性的未来方案，而非无意义的追责。
* **Visual Intent (视觉与图例)**: 
    *   屏幕中央展示一个天平。
    *   左侧托盘放着标有“过去”和“指责”的沉重砝码（如石头），颜色灰暗。
    *   右侧托盘放着标有“未来”和“方案”的羽毛或光球，但天平倾向于右侧（寓意价值更重），并散发金色光芒。
    *   屏幕上方展示金句：“指责是锁住过去的枷锁，辅导是开启未来的钥匙。”

## Block 3: [Type: O] - [Observe: 认知自检]
* **Composition**: Pure Quiz
* **Block Discription**: 通过一个简短的场景反思，检测学员是否真正理解了“教练”与“警察”的区别。
* **Block Intent**：作为中间的连接点（Observe），在进入具体案例前，先让学员通过自测来验证刚才学到的“心智转型”是否已内化，防止带着旧认知看新案例。

### Segment 3.1: [Type: Quiz] - [Type A: 辅导焦点的识别]
* **Type**: Type A (引发思考型)
* **Question**: 面对下属项目延期，你脱口而出的第一句话通常接近以下哪种意图？（请在心里默默对照）：是“为什么没按时完成？谁的责任？”还是“现在的进度和目标差多少？我们需要什么支持来赶上？”
* **Explaination**: 如果你的本能反应是追责，那么你仍处于“指令与控制”的旧模式。真正的效能教练会立刻聚焦于现状（Reality）与未来方案（Options）的差距分析。
* **Design Intent**：不设置标准选项，而是通过直击灵魂的二选一反思，唤醒学员对自己日常行为的觉察，为引入GROW模型制造“技能缺口”。

## Block 4: [Type: C] - [Case: 王经理的蜕变]
* **Composition**: Pure Video
* **Block Discription**: 演绎王经理在面对同一问题（会议组织混乱）时的两种截然不同的处理方式：先是失败的“指令轰炸”，后是成功的“GROW引导”。
* **Block Intent**：通过正反案例的强烈对比（Case），将抽象的“转型”概念具象化。反面案例展示“指令式”的破坏力，正面案例展示“GROW”的实操样貌，为后续拆解GROW模型提供素材。

### Segment 4.1: [Type: Video] - [从咆哮到引导]
* **Script Prompt (分镜剧本)**:
    *   **场景1 (负向)**: 办公室嘈杂，王经理满脸通红，拍着桌子对李萌新咆哮：“你怎么搞的！连个会都组织不好！”李萌新低头，试图解释：“可是邮件...”王经理粗暴打断：“别找借口！下次必须一个个确认，听到没有？”李萌新眼神黯淡，机械地点头，内心独白：“反正你说了算，我不动脑子了。”
    *   **剧情描述**: 镜头特写王经理的愤怒和李萌新的委屈，色调压抑，强调“指令与指责”带来的对抗与消极。
    *   **场景2 (正向)**: 时间倒流或切换场景。王经理深吸一口气，压下火气，平和地问：“萌新，这次会议出勤率不高，你注意到了什么事实？”（Reality）。李萌新思考后回答：“大家好像没看清邮件。”王经理点头：“那为了确保下次全员到齐（Goal），你觉得除了发邮件，还能做什么？”（Options）。李萌新眼睛一亮：“我可以会前发个弹窗提醒，或者设置日历邀请！”
    *   **剧情描述**: 镜头展示王经理的克制与引导，以及李萌新从紧张到主动思考的微表情变化，色调明亮，强调“赋能”带来的活力。
* **Design Intent（设置意图）**：直观展示两种管理行为导致的不同结果。重点在于展示王经理如何通过“提问”而非“命令”改变了对话的走向，为后续讲解GROW的四个步骤埋下伏笔。

## Block 5: [Type: E] - [Explain: GROW模型维度原子化解构]
* **Composition**: Pure PPT
* **Block Discription**: 系统性拆解GROW模型的四个维度（Goal, Reality, Options, Will），并深入讲解每个维度的关键执行标准（如SMART原则、事实萃取、头脑风暴规则等）。
* **Block Intent**：这是课程的核心干货（Explain）。在建立了认知和看到了案例后，学员需要掌握具体的“操作说明书”，确保能将GROW模型标准化地应用到实际工作中。

### Segment 5.1: [Type: PPT] - [GROW模型全景图]
* **Layout Strategy**: Overview + Flow
* **Context Prompt (内容上下文)**: 
    *   **模型概览**：GROW模型是一个从目标到行动的闭环逻辑。
    *   **流程节点**：Goal（目标锚定） -> Reality（现状评估） -> Options（路径生成） -> Will（行动锁定）。
    *   **逻辑流向**：从“去哪里”到“在哪里”，再到“怎么去”和“何时去”。
* **Design Intent（设置意图）**：建立GROW模型的整体框架认知，帮助学员记忆四个步骤的逻辑顺序。
* **Visual Intent (视觉与图例)**: 
    *   屏幕展示四个卡片按顺时针或线性排列，中间有箭头连接。
    *   第一张卡片“G (Goal)”显示旗帜图标；第二张“R (Reality)”显示放大镜图标；第三张“O (Options)”显示多条路径的分岔路口图标；第四张“W (Will)”显示脚印或打钩的清单图标。
    *   每个卡片下方标注对应的中文含义（目标、现状、方案、意愿）。

### Segment 5.2: [Type: PPT] - [Goal：目标锚定与SMART审计]
* **Layout Strategy**: Overview + List
* **Context Prompt (内容上下文)**: 
    *   **核心定义**：明确“我们想做的事情”，对齐短期任务与长期意图。
    *   **SMART原则**：目标必须经过合规性审计。
        *   Specific（清晰）：消除歧义。
        *   Measurable（可衡量）：建立量化标准（如“会议缺席率0%”）。
        *   Achievable（可实现）：跳一跳够得着。
        *   Relevant（相关）：与核心战略高度对齐。         
        *   Time-bound（时限）：设定刚性截止日期（严守DDL）。
* **Design Intent（设置意图）**：教会学员如何设定一个合格的辅导目标，避免目标模糊导致后续辅导失效。
* **Visual Intent (视觉与图例)**: 
    *   屏幕上方展示标题“Goal：目标锚定”。
    *   下方平铺展示三个卡片，分别代表SMART中的关键要素。
    *   卡片1“Specific”：图标为聚焦的靶心，文字说明“拒绝模糊，具体明确”。
    *   卡片2“Measurable”：图标为尺子或仪表盘，文字说明“量化指标，数据说话”。
    *   卡片3“Achievable”：图标为阶梯，文字说明“挑战适度，切实可行”。
    *   卡片4“Relevant”：图标为指南针与地图，文字说明“对齐战略，确保价值”。     
    *   卡片5“Time-bound”：图标为带有倒计时的沙漏或秒表，文字说明“设定节点，严守时限”。

### Segment 5.3: [Type: PPT] - [Reality：现状评估与事实萃取]
* **Layout Strategy**: Overview + Contrast
* **Context Prompt (内容上下文)**: 
    *   **核心动作**：客观事实萃取，剥离主观臆断。
    *   **话术工具**：“请暂停一下‘我觉得’。如果摄像机在现场，它拍到的是什么？”
    *   **对比**：左侧“主观臆断”（经常、总是、态度差）；右侧“客观事实”（每周迟到3次、未回复邮件）。
* **Design Intent（设置意图）**：重点训练学员区分“事实”与“观点”的能力，这是现状评估中最容易犯错的地方。
* **Visual Intent (视觉与图例)**: 
    *   屏幕物理中轴有“VS”图标。
    *   左侧区域压暗，标题“主观臆断”，展示气泡框文字“他总是迟到”、“态度不好”，打上红色的叉。
    *   右侧区域高亮，标题“客观事实”，展示数据图表或清单文字“本周迟到3次”、“会议中未发言”，打上绿色的勾。
    *   底部展示摄像机图标，提示“摄像机视角”。

### Segment 5.4: [Type: PPT] - [Options：路径生成与头脑风暴]
* **Layout Strategy**: Overview + Tree
* **Context Prompt (内容上下文)**: 
    *   **核心机制**：构建从现状到目标的战略路径。
    *   **激活协议**：数量优先，批判暂停。
    *   **操作**：不断追问“还有吗？”，激发发散性思维，构建方案库。
* **Design Intent（设置意图）**：强调在方案阶段的“开放性”，防止管理者过早打断或评判，确保方案的多样性。
* **Visual Intent (视觉与图例)**: 
    *   屏幕左侧是一个标有“现状”的圆点，右侧是一个标有“目标”的圆点。
    *   从左侧圆点发散出多条曲线连接到右侧，形成树状或网络状的路径，代表多种可能性。
    *   每条路径上标注不同的简短方案（如“流程优化”、“工具升级”、“人员调整”）。
    *   画面一角有“禁止评判”的图标（如一个禁止符号盖在法官锤上），提示“批判暂停”。

### Segment 5.5: [Type: PPT] - [Will：行动锁定与决心评估]
* **Layout Strategy**: Overview + Flow
* **Context Prompt (内容上下文)**: 
    *   **核心要素**：将认知转化为行动契约。
    *   **三定原则**：定人（执行主体）、定时（截止时间）、定心（决心量化）。
    *   **话术**：“你有几分把握完成？需要什么支持？”
* **Design Intent（设置意图）**：确保辅导对话落地，不让方案停留在口头上，强调责任归属和时间节点。
* **Visual Intent (视觉与图例)**: 
    *   屏幕展示三个串联的步骤卡片。
    *   步骤1“Who”：图标为单人剪影，文字“责任到人”。
    *   步骤2“When”：图标为日历或时钟，文字“明确Deadline”。
    *   步骤3“Will”：图标为电池电量格或心形，文字“决心量化（1-10分）”。
    *   箭头从左至右贯穿，最终指向一个“Action”的印章。

## Block 6: [Type: O] - [Observe: 知识迁移检验]
* **Composition**: Pure Quiz
* **Block Discription**: 通过一个具体的对话片段，检验学员能否准确识别出该片段属于GROW模型中的哪一个环节，并判断其有效性。
* **Block Intent**：作为课程尾声的检测（Observe），确认学员不仅听懂了概念，还能在实际对话中识别出GROW的结构，完成知识的初步迁移。

### Segment 6.1: [Type: Quiz] - [Type B: GROW环节识别]
* **Type**: Type B (知识验证型)
* **Question**: 在辅导对话中，管理者问员工：“为了实现这个目标，你觉得有哪些障碍？目前手里有哪些资源可用？”请问这属于GROW模型中的哪一个环节？
* **Explaination**: 这属于 **Reality（现状评估）** 环节。虽然提到了“目标”，但问题的核心在于盘点当前的障碍和资源，即理清“我们在哪里”，为后续制定方案提供现实依据。
* **Design Intent**：纠正学员容易将“谈论障碍”误认为是“制定方案”的错觉，强化对Reality环节“盘点事实与环境”这一定义的理解。

## Block 7: [Type: R] - [Review: 总结与升华]
* **Composition**: Pure PPT
* **Block Discription**: 总结GROW模型的四步逻辑，并再次强调从指责到赋能的核心价值观，以金句收尾。
* **Block Intent**：收束全课（Review），巩固记忆，并升华课程主题，让学员带着“赋能者”的使命感离开课堂。

### Segment 7.1: [Type: PPT] - [从指责到赋能的闭环]
* **Layout Strategy**: Focus + Canvas
* **Context Prompt (内容上下文)**: 
    *   **回顾**：GROW四步（G目标-R现状-O方案-W意愿）是辅导的标准物理模型。
    *   **升华**：管理者的使命不是展示自己有多强，而是激发员工变得更强。
    *   **金句**：别做天花板，要做梯子。
* **Design Intent（设置意图）**：用极简的视觉和有力的语言，完成最后的情感升华和知识固化。
* **Visual Intent (视觉与图例)**: 
    *   背景为攀登高峰的场景。
    *   画面中央展示一个发光的梯子（隐喻GROW模型），连接着地面（现状）和山顶（目标）。
    *   梯子的每一级横杆分别标注G、R、O、W。
    *   屏幕上方展示大字号金句：“管理者的价值：别做限制员工的天花板，要做助其向上的梯子。”
