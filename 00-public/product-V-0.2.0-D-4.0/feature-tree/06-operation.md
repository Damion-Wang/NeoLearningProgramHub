# 功能树 06：Operation（管理端）

**Owner：** 鼎明、显华
**日期：** 2026-04-22
**版本：** v0.4.0 对齐版（L1+L2）
**对应 Spec：** `spec/design/management/00-overview.md` + `01-dashboard.md` + `02-report.md` + `03-project-config.md`

---

## 一句话定位

管理员（HR + 运营职能合一）的统一管理端——首页 Operation / 报告生成 Report-Gen / 消息 Message / 配置 Config 4 个一级入口，由 Ora（管理端 AI）驱动数据解读与报告辅助生成。

---

## 功能结构

```
A. 定位与端口
   A1 管理员角色（HR 职能 + 运营职能合一，一个角色一个端口）
   A2 通过 Topbar 端口切换进入（学员端 ↔ 管理端，见 07.D/C3）
   A3 ★开营前后系统状态 [spec §3.2]
       - 开营前：仅「配置」模块可用，其他三项锁定
       - 开营后：四模块全部解锁；重要配置本期允许编辑（安全验证放未来迭代）
   A4 项目生命周期（签约→商品下发创建项目+admin→交付内容预览→开营流程→学习期→结项）

B. 顶级导航（★v0.4.0 精简为 4 项）
   B1 首页 Operation
   B2 报告生成 Report-Gen
   B3 消息 Message
   B4 配置 Config

C. 首页 Operation — 三维度统筹（★v0.4.0 重组）
   （spec §7：项目维度 + 人员维度 + 内容维度，通过内部 Tab 切换呈现）
   C1 总览 Tab（项目整体完成率 + 需关注学员 Top 3 + 项目热力图）
   C2 学员管理 Tab（列表 / 状态 / 进度 · 人员维度）
   C3 进度看板 Tab（按 Course / 按学员，Activity 级 · 项目+内容维度）
   C4 数据展示 Tab（内容维度）
       C4.1 学习进度 / 学习效果（P2）/ 活跃度 / 完课率
       C4.2 被动信号聚合（停留 / 中途离开率 / 回翻率）
       C4.3 数据更新时间标注
       C4.4 ★不展示能力维度排名（本期限制）
   C5 催学管理 Tab（人员维度）
       C5.1 规则配置（触发条件 + 消息模板 + 执行策略）
       C5.2 自动执行 + 渠道（产品内 + Email 必做；企微 / 钉钉可选）
       C5.3 重要人工确认（升级规则触发时管理员审核）
       C5.4 效果漏斗（本期站内：登录→查看→完成；未来站外：发送→打开→登录→完成）

D. 报告生成 Report-Gen
   D1 个人学习报告（同学员版 + 额外可见排名 + 分组标签）
   D2 项目整体报告（汇总分析 + 各 Course 完成率分布）
   D3 进度排名（Activity 完成数，可排序可筛选）
   D4 互动排名（token 量，含计算说明）
   D5 分组标签（基于阈值规则，非智能评价，学员不可见）
   D6 报告工厂
       D6.1 选择类型 / 范围 / 模板（统一模板库，Ora 与工厂共享）
       D6.2 批量生成（后台并行，进度展示）
       D6.3 审核队列（逐份 + 批量操作 + 失败重试）
       D6.4 批量导出 PDF / Word

E. 消息 Message
   E1 发送记录列表（手动发送 + 自动催学统一查看）
   E2 手动定向发送（管理员编辑富文本 + 图片 + 链接，发送给指定学员 / 全员）
   E3 与 Topbar 铃铛的关系：铃铛 = 管理员收到的消息 / 消息页 = 管理员下发能力
   E4 ★多渠道配置 [P1，req §9]
       E4.1 渠道启用开关（企微 / 钉钉 / 短信，按客户 IT 能力决定）
       E4.2 OAuth 鉴权配置
       E4.3 消息格式适配（每渠道模板 + 长度限制 + 富文本支持）
       E4.4 渠道优先级与降级（主渠道失败回退 Email / 产品内）
       E4.5 分渠道效果追踪（接入 C5.4 催学漏斗）
   E5 本期渠道范围（仅站内；未来根据 GPB 能力扩展站外）

F. 配置 Config（纯 GUI 无 Ora）
   F1 项目信息（名称 / 周期 / 课包绑定）
   F2 学员名单导入（Excel/CSV + 预览 + 通知触发，见 07.B4/B5）
   F3 角色分配 GUI（学员 / 管理员）
   F4 催学规则初始配置（与 C5.1 共享规则库）
   F5 ★平台个性化（客户企业 Logo / 睿学形象 / 音色）
   F6 Token 统计查看（P3，管理员口径聚合）
   F7 内容包预览（开营前只读，交付人员确认内容正确）
   F8 确认开营（关键动作：解锁学员端登录 + 其他三模块解锁）
   F9 内容包完整性校验入口（跳 08.B）

G. 管理端 Agent（★Ora）
   （spec §4.2：Ora 核心价值 = 数据解读 + 对比洞察 + 报告辅助生成与 HITL 编辑 + 证据追溯 + 隐性知识发现）
   G1 自然语言查询（项目 / 学员 / 数据，含时间戳标注 + 能力边界坦诚）
   G2 生成汇报报告（HITL 流程：需求确认 → Ora 生成初稿 → 管理员编辑 → 定稿）
       G2.1 模板手动复用（本期不做跨会话自动记忆）
       G2.2 在线预览 + 修改迭代
       G2.3 PDF / Word 导出
   G3 解读数据（三层钻取：结论 → 证据摘要 → 原始对话）
   G4 辅助报告审核（与 D6.3 同一能力）
   G5 隐性知识发现（从对话数据中挖掘常见问题 / 最佳实践）
   G6 ★跨页面长上下文（首页 ↔ 报告 ↔ 消息共享记忆，跨会话保持）
   G7 配置页无 Ora（仅首页 / 报告 / 消息有）
   G8 全周期可用（学习期随时查 + 结项期生成汇报）

H. 数据粒度与隔离
   H1 ★数据粒度分层（底层存储 SCO 级，UI 展示 Activity 级，见 spec §4.4）
   H2 数据时间标注（看板 + Ora 回答）
   H3 学员端 / 管理端排名 API 级隔离（学员端不返回排名字段，见 05.F4）

附录 Z. 本期不做
   Z1 HR / 运营分化（本期角色合一）
   Z2 智能预警评价（先呈现数据，管理员自行判断）
   Z3 能力维度排名（可信度 / 可比性本期无法保证）
   Z4 CEO 汇报 PPT 生成（先做好报告格式）
   Z5 学员分组分析（30-40 人无分组需求）
   Z6 临时任务下发（管理员通过企微等渠道完成）
   Z7 学员自注册（所有账号管理员导入创建）
   Z8 多语言
```

## 术语速查

- **Ora** = 管理端 AI Agent 前台名称（spec §4.2）；跨页面共享记忆
- **HITL**（Human In The Loop）= Ora 生成初稿 → 管理员人工编辑 → 定稿的报告生成流程
- **三维度统筹** = 首页 Operation 按项目 / 人员 / 内容三视角组织数据展示
- **开营前后状态** = 开营前仅配置可用 / 开营后四模块解锁（spec §3.2）
