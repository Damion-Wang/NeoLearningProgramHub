## 5. HR Agent对话

### 关键交互

| 交互 | 触发 | 响应 | 联动 |
|------|------|------|------|
| 数据查询 | "本周谁进度最快？" | Agent回答+数据截止时间 | 看板对应行高亮 |
| 个人查询 | "张三学习情况如何？" | Agent返回数据摘要 | 看板高亮张三行 |
| 报告生成 | "帮我准备CEO汇报" | Agent确认需求->选模板->生成->预览->修改->导出 | 看板切换为报告预览 |
| 能力边界 | 超出范围提问 | Agent坦诚告知+建议 | 无 |
| 看板联动 | HR先点看板再追问 | Agent关联看板上下文 | 自动承接 |

### 报告生成流程视觉

```
HR: 帮我准备给CEO的月度汇报报告

Agent: 好的，请确认:
(1) 模板——用上次的还是新模板？
(2) 重点——关注哪些方面？

HR: 用上次的，重点看进度

Agent: 收到，开始生成，预计30秒...

[看板区切换为报告预览视图]

Agent: 报告初稿已生成，请预览。

HR: 第三段结论太笼统，加入张三李四案例

Agent: 好的，修改中...
[看板区报告更新完成后一次性高亮修改部分]

Agent: 修改完成，是否定稿？

HR: 可以了，导出PDF

Agent: 正在生成PDF...
[弹出下载确认]
```

> **[D10修订]** 报告预览修改高亮: Agent完成修改后一次性diff高亮（非逐字实时高亮），减少性能消耗和视觉干扰。

> **[B6修订]** 报告模板管理: 运营预设模板 + HR可自建模板。HR端在Agent对话中可说"保存为模板"，或在设置中进入"模板管理"子菜单。

**Agent约束:** 仅描述可验证数据，推测性分析标注"AI推测，请酌情参考"

## 6. 体验课程（身份切换）

### 切换流程视觉

```
HR点击"体验课程"
    |
确认弹窗: "即将切换到学员视图体验课程。
你的体验数据不会计入学员统计。确认切换？"
    |
[确认] -> 服务端保存HR端状态快照 + sessionStorage缓存
    |
视图切换为学员端大厅
    |
顶部常驻提示条: [体验模式: 你的数据不会计入统计 | 返回管理端]
对话区顶部提示: "体验模式"
    |
HR正常体验学习
    |
点击"返回管理端"
    |
优先从sessionStorage恢复HR端状态（导航位置+滚动位置+筛选条件）
数据部分重新加载（不依赖缓存）
```

> **[U10/A8修订]** 状态保存改为服务端快照为主+sessionStorage为快速恢复缓存。仅保存轻量导航状态（路径+滚动位置+筛选条件），数据本身重新加载，避免session过大。

### 协商机制

> **[U11/X7修订] 完整协商流程:**

```
1. HR保存规则时检测到与运营规则冲突
   -> 提示"类似规则已存在（运营创建）"
   -> 选项: [覆盖申请] [修改我的规则] [取消]

2. HR选择"覆盖申请"
   -> 填写覆盖理由
   -> 提交后HR端该规则状态变为"协商中"

3. 运营端催学管理收到"待审批协商"通知
   -> 查看HR规则 vs 运营规则对比
   -> [批准] 或 [拒绝(填写理由)]

4. 结果通知
   -> 批准: HR规则优先级提升至与运营平级，HR端状态变为"已批准"
   -> 拒绝: HR收到通知+拒绝理由，HR端状态变为"已拒绝"
```

## 11. 催学管理

### 催学规则合并执行逻辑（HR+运营共用）

1. 同一学员同一天不重复发送相同条件消息
2. **系统硬上限: 每人每日不超过3条（不可调高）**
3. 运营规则默认优先级高于HR规则
4. HR通过协商获得覆盖权后优先级提升至与运营平级

### 运营Agent催学快捷操作

```
运营: 帮我给所有逾期学员发一条催学消息

Agent: 检测到当前有5名逾期学员。
即将发送给: 张三、李四、王五、赵六、陈七
消息内容: "你有一个新的管理技能课程可以学习..."

确认发送？ [确认] [修改] [取消]

运营: 确认

Agent: 已向5名学员发送催学消息。可在催学管理中查看状态。
```

### 协商待审批入口

> **[X7修订]** 运营端催学管理页增加"待审批协商"入口（角标显示待审批数量），点击展开HR提交的覆盖申请列表。

## 14. 降级/异常方案

> **[X14修订]** 管理端降级方案:

| 异常场景 | 处理方式 |
|---------|---------|
| 数据加载失败 | 看板区显示"数据加载失败，请刷新"按钮+上次成功数据的截止时间 |
| Agent不可用 | 对话区显示"助手暂时不可用，请稍后重试"，看板功能不受影响 |
| 报告生成失败 | 审核队列标注"生成失败"+重新生成按钮，不阻塞其他报告 |
| 导出超时 | toast提示"导出超时，请缩小范围后重试" |
