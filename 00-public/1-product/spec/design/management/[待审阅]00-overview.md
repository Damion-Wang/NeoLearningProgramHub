# 管理端总览

## 定位

管理端是 HR 和运营共用的项目管理界面——以"看板 + 报告 + 配置"三大功能区组织。看板是日常监控中心，报告是数据输出窗口，配置是项目启动和运维的操作面板。本期 HR 端和运营端功能完全一致，两个独立端口入口、同一套界面。

## 管理员的一天

### 场景一：运营日常巡检（看板页）

**运营小李每天上午的固定动作。**

1. 小李登录运营端，默认进入「首页」，中间看板区展示总览
2. 查看项目整体数据：完成率 28%、上周活跃 3.2 天/人、需关注学员 Top 3
3. 右侧问 Agent："上周进度最快的 3 个人是谁？"——Agent 给出排名和具体数据
4. 在中间看板区切换到「学员管理」Tab，点击低活跃学员查看详情：进度 8%、5 天未登录
5. 切换到「催学管理」Tab，查看自动催学规则执行情况，对触发升级规则的学员发送个性化消息
6. 切换到「数据展示」Tab，查看各 Course 完成率分布——点击某个柱状条，Agent 自动解读原因
7. 查看催学漏斗（发送→查看→登录→完成），追问 Agent 转化瓶颈在哪

### 场景二：HR 准备月度汇报（报告页）

**HR 王经理月底需要向 CEO 汇报培训进展。**

1. 王经理登录 HR 端，左侧点击「报告」，中间展示报告页，查看「项目报告」整体数据
2. 切换「进度排名」Tab 查看学员排名表，再切「互动排名」看参与深度
3. 点击某学员名字，查看个人学习报告（含 HR 专属的排名和分组标签信息）
4. 向 Agent 说"帮我准备一份给 CEO 的月度汇报报告"
5. Agent 确认模板/重点/风格后生成初稿，王经理预览修改后下载 PDF
6. CEO 追问时，王经理再次登录向 Agent 查询具体数据证据，支持钻取到原始对话记录

### 场景三：新项目开营（项目配置）

**运营小李为新签约客户完成开营配置。**

1. 小李左侧点击「配置」，中间展示项目配置表单，填写项目名称、确认课包绑定、设置项目周期
2. 上传 HR 提供的学员名单 Excel，系统展示导入预览（成功/失败/重复）
3. 为特定账号分配 HR/运营角色（默认全部为学员）
4. 确认后系统批量创建账号并发送 Email 通知
5. 配置自动催学规则（2天/5天/7天阈值）
6. 上传客户 LOGO、选择 AI 老师形象和音色
7. 校验内容包完整性，确认后正式开营

## 三栏布局说明

管理端统一采用三栏布局：

| 栏位 | 位置 | 功能 |
|------|------|------|
| **左侧导航** | 可折叠侧边栏 | 4 个一级入口：首页 / 报告 / 消息 / 配置 |
| **中间主区域** | 页面中心 | 根据左侧导航展示对应页面；「首页」内通过 Tab/子导航切换总览、学员管理、进度看板、数据展示、催学管理等子功能 |
| **右侧 Agent** | 对话面板 | 自然语言查询 + 数据解读 + 报告生成（配置页无 Agent） |

核心交互模式：中间看板的任何数据元素可点击 → 右侧 Agent 自动给出解读。

## 管理端 Agent 与学员端 Leo/Neo 的关系

管理端右侧的 Agent 和学员端的 Leo（Coach）/ Neo（Tutor）是**完全独立的角色系统**：

| 维度 | 管理端 Agent | 学员端 Leo/Neo |
|------|-------------|---------------|
| **服务对象** | HR / 运营 | 学员 |
| **数据来源** | 统一 Database（全局视角） | 统一 Database（个人视角） |
| **核心职能** | 数据查询、报告生成、催学辅助 | Leo: 陪伴辅导；Neo: 结构化教学 |
| **角色身份** | 本期未命名，统一称"Agent" | Leo（Coach）和 Neo（Tutor） |

三者共享同一个统一 Database，但各自按权限和视角读取数据。管理端 Agent 的命名和独立化在未来迭代中考虑。

## 跨模块共享能力

| 能力 | 说明 | 涉及模块 |
|------|------|---------|
| **统一 Agent** | HR/运营 Agent 本期完全一致，基于统一 Database 回答 | 看板+报告 |
| **数据追溯** | 从结论→证据摘要→原始对话记录，逐层钻取 | 看板+报告 |
| **Activity 级粒度** | 所有数据以 Activity 为最小展示单位，不做 SCO 级 | 全部 |
| **数据时间标注** | Agent 回答标注数据更新时间 | 看板+报告 |
| **HR 体验隔离** | HR 体验学员课程时数据单独记录，不污染学员统计 | 通过端口切换进入学员端体验 |

## 已确认决策

- HR 端和运营端本期界面与功能完全一致，两个独立端口入口
- Agent 两端一致，不做独立运营 Agent
- 管理端数据粒度为 Activity 级，不做 SCO 级
- 数据呈现为主，本期不做智能预警评价
- 排名维度：进度排名 + 互动排名，不做能力维度排名
- 学员端/HR 端排名数据 API 级隔离
- 30-40 人规模暂无学员分组/部门划分
- 左侧导航精简为 4 项：首页 / 报告 / 消息 / 配置
- 首页内的子功能（总览/学员/进度/数据/催学）在中间看板区通过 Tab/子导航切换
- 管理端无独立体验课程入口——HR 通过通用栏端口切换进入学员端体验
- 内容包由内部团队预配置，管理员只做绑定确认
- 配置页面为纯 GUI，不需要 Agent

---

## 本期不做的功能

- 独立运营 Agent——本期与 HR Agent 一致
- 智能预警评价——先呈现数据，运营/HR 自行判断
- 能力维度排名——可信度和可比性本期无法保证
- CEO 汇报 PPT 生成——先做好报告格式
- 学员分组分析——暂无分组需求
- 临时任务下发——运营可通过企业微信等渠道完成
- 学员自注册——所有账号由运营导入创建
- 多语言支持——本期不支持
