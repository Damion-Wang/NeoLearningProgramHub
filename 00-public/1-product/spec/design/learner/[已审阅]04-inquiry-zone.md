## 模块定位

> **优先级：P3**——本模块整体为六月版本 P3项（非 P0 必做），视开发资源决定是否纳入首版交付。

调研场域是 Neo（Tutor）通过对话或 GUI 量表采集学员真实信息并进行评估的空间——调研不是"考试"，而是"深度访谈"或"标准化量表"。

**未来扩展**：Inquiry 模块未来可承接萃取（经验提炼）、考试等信息采集类行为，不局限于访谈和量表。

## Neo（Tutor）— 调研场域中的角色定位

### Neo 是谁

**前台名称**：Neo
**角色类型**：Tutor（老师）
**知识范围**：Activity 内 + 已完成 Activity + 学员画像（可读取全部课堂数据，有选择地引导回当前 Activity 主题）

### Neo 在调研场域的核心价值

1. **结构化信息采集**——按调研剧本引导对话或展示量表，确保采集到有效的学员行为数据和认知信息
2. **温和专业的访谈**——让学员感到"被倾听"而非"被审讯"，鼓励真实分享
3. **信息确认与修正**——采集完毕后提炼核心信息点，与学员确认准确性

### Neo 在调研场域的核心能力

| 能力 | 说明 |
|------|------|
| **结构化访谈/量表** | 按调研剧本引导对话（BEI/画像/困惑收集/萃取）或展示 GUI 量表 |
| **EvaluationSkill** | 信息采集 + STAR 完整性检查 + 评估 |
| **InsightCaptureSkill（轻量）** | 捕捉 BEI 案例证据、学员画像碎片，写入统一 Database |

### Neo 各阶段的行为边界（能做 vs 不能做）

**调研说明阶段**

| 能做 | 不能做 |
|------|--------|
| 说明调研目的、流程、预计时长 | 透露具体的评估标准和能力维度 |
| 消除学员焦虑，强调"没有标准答案" | 暗示"正确"的回答方向 |

**核心调研阶段**

| 能做 | 不能做 |
|------|--------|
| 温和追问，引导学员补充细节 | 判断学员说的是真是假（AI 不做判官） |
| 检测 STAR 完整性，追问缺失元素 | 纰缠回避行为（连续 2 次模糊即切换新案例） |
| 营造安全的对话氛围 | 让学员感到被审讯或被评判 |
| 基于学员提问回顾之前学过的知识点 | 在调研过程中做教学（调研是采集，不是教学） |

**信息确认阶段**

| 能做 | 不能做 |
|------|--------|
| 提炼 3-5 个核心信息点逐一确认 | 下结论性评价（"你的能力很差""你这方面有问题"） |
| 接受学员的修正并更新记录 | 否定学员的修正（"不对，你刚才明明说的是..."） |
| 追问是否有补充信息 | 强迫学员接受 Neo 的总结 |

## 功能区域

> 全局布局为两栏 + 可折叠导航区。调研场域三个阶段的栏数不同：说明双栏、核心单栏沉浸、确认双栏。

### 阶段一：调研说明（双栏）

| 区域 | 内容 |
|------|------|
| **可折叠导航区** | Project 目录 |
| **左栏：调研说明** | 调研说明卡片（markdown）：调研主题、预计时长、形式、"没有标准答案" |
| **右栏：Neo 对话** | Neo 温和开场引导，回答学员关于流程的疑问 |

### 阶段二：核心调研（单栏沉浸）

| 区域 | 内容 |
|------|------|
| **可折叠导航区** | Project 目录 |
| **全屏单栏** | 对话式调研 → 对话区（学员 vs Neo 的访谈对话）；量表式调研 → GUI 量表界面（逐题呈现）。主打沉浸，不分左右栏 |

### 阶段三：信息确认（双栏）

| 区域 | 内容 |
|------|------|
| **可折叠导航区** | Project 目录 |
| **左栏：调研总结** | 实时生成的调研总结——提炼的核心信息点列表 |
| **右栏：Neo 确认对话** | Neo 逐一确认信息点是否准确，追问是否有补充。学员可修正不准确的总结 |

## 三阶段结构

### 阶段一：调研说明（场景铺垫）

Neo 告诉学员本次调研的目的、要收集什么信息、大概流程和时长。类似对练的情景导入——让学员知道接下来会发生什么，消除焦虑。

### 阶段二：核心调研（执行）

根据类型分为两种：
- **对话式调研**：Neo 通过自然对话引导学员讲述真实经历。适用于 BEI 行为事件访谈、学员画像采集、困惑收集、高光萃取
- **GUI 量表调研**：类似 MBTI 的标准化量表，学员通过 GUI 界面选择。适用于结构化能力评估、态度量表

### 阶段三：信息确认复盘

Neo 提炼本次调研中收集到的 3-5 个核心信息点，逐一向学员确认是否准确。这与对练复盘有本质区别：
- **对练复盘** = 苏格拉底式深度思考追问 → 总结
- **调研复盘** = 信息确认（"我总结的对不对？你觉得有需要修正的吗？"）+ 追问是否有补充

### 结束信号

调研何时结束由两个条件共同决定：
1. **剧本定义的最低采集量**——例如"至少采集 2 个完整案例"或"量表全部题目作答完成"
2. **学员确认无补充**——信息确认阶段，学员明确表示"没有补充了"

两个条件都满足后，显示"下一个任务"卡片 + "返回大厅"链接（与其他场域衔接方式一致）。

## 用户故事：张磊完成一次沟通风格调研

### 场景背景
张磊在开始 Course 2「横向协作」之前，需要完成一次沟通风格调研。这是一次对话式 BEI 访谈，目的是采集张磊在横向协作中的真实行为模式。

### 完整旅程

**阶段一：调研说明（约 1 分钟）— 双栏**

左栏展示调研说明卡片：
- 调研主题：横向协作沟通风格
- 预计时长：10 分钟
- 形式：对话式访谈
- "没有标准答案，根据实际情况聊就好"

右栏 Neo 温和开场：
> "接下来我们做一个简单的交流，大约 10 分钟。我想了解一下你在跨部门协作中的一些真实经历。没有标准答案，你根据实际情况来聊就好。准备好了吗？"

张磊确认准备好后，进入核心调研阶段。

**阶段二：核心调研（约 8 分钟）— 单栏沉浸**

界面切换为全屏对话区，Neo 按调研剧本引导，围绕 STAR 结构追问：

**第 1 轮 — 开放式提问**：
> "请描述一个你最近在跨部门协作中印象深刻的经历——无论是成功的还是有挑战的。"

张磊："上个月市场部要做一个大型客户答谢会，需要我们销售部配合邀请重点客户。但时间太赶了，我们自己还有季度冲刺任务。"

**第 2 轮 — Situation 追问**：
> "当时具体是什么情况？市场部是怎么找到你的？时间有多紧？"

**第 3 轮 — Task 追问**：
> "在这个情况下，你需要承担什么具体任务？你的角色是什么？"

**第 4 轮 — Action 追问**（Neo 后台检测到 Action 信息不完整）：
> "你说'后来协调了一下'——能具体说说你做了什么吗？比如你找了谁、说了什么、用了什么方式？"

张磊回答模糊，Neo 温和追问一次。连续 2 次模糊/转移话题，Neo 不再纰缠，切换到新案例。

**第 5 轮 — 第二个案例**：
> "再聊一个其他的经历吧。有没有一次跨部门合作特别顺利的案例？"

同一案例最多追问 3 轮；不完整的案例记录完整度评分——案例不完整本身就是评估信息。

剧本定义最低采集量达到后（如 2 个案例），Neo 过渡到信息确认阶段。

**阶段三：信息确认（约 2 分钟）— 双栏**

界面切换回双栏：左栏显示实时生成的调研总结，右栏 Neo 逐一确认。

左栏调研总结：
> 1. 在跨部门协作中倾向于先理解对方的诉求再回应，而不是直接拒绝
> 2. 当时间紧迫时，更倾向于自己想解决方案而不是拉着对方一起讨论
> 3. 面对不合理的请求时，倾向于先妥协再找补救办法

右栏 Neo 确认对话：
> "好的，根据我们刚才的交流，左边列出了几个要点。逐一看一下——第一点准确吗？"

张磊确认第一和第三点准确，但修正了第二点："不是不愿意讨论，是觉得讨论太费时间，不如自己先想好方案再跟对方确认。"

Neo 更新左栏总结并确认：
> "明白了，已修正。还有什么想补充的吗？"

张磊："没有了。"

调研结束——最低采集量已达 + 学员确认无补充。显示"下一个任务"卡片。

## 量表调研场景补充

### GUI 量表交互流程

1. 调研说明阶段（双栏）：左栏展示量表说明，右栏 Neo 引导
2. 核心调研阶段（单栏沉浸）：全屏展示量表界面——逐题呈现，每题有选项（单选/多选/滑块）
3. 学员通过 GUI 组件作答，完成后自动计分
4. 信息确认阶段（双栏）：左栏展示量表结果总结，右栏 Neo 解读并确认

### 量表结果处理

- 量表题目、选项、计分逻辑全部由调研剧本定义
- 结果写入统一 Database，供报告和 Leo 引用

## 核心交互清单

### 1. 调研说明展示

| 项目 | 内容 |
|------|------|
| **触发** | 学员进入调研 Activity |
| **响应** | 双栏：左栏调研说明卡片，右栏 Neo 温和引导 |
| **联动** | 学员确认准备好后进入核心调研阶段 |

### 2. 对话式调研执行

| 项目 | 内容 |
|------|------|
| **触发** | 进入核心调研阶段 |
| **响应** | 单栏沉浸：全屏对话区，Neo 按调研剧本引导对话，后台自动检查 STAR 完整性 |
| **联动** | 同一案例最多追问 3 轮；连续 2 次模糊则切换新案例。不完整案例记录完整度评分 |

### 3. GUI 量表执行

| 项目 | 内容 |
|------|------|
| **触发** | 进入量表类核心调研阶段 |
| **响应** | 单栏沉浸：全屏展示量表 GUI 界面，学员逐题作答 |
| **联动** | 完成后自动计分，进入信息确认阶段 |

### 4. 信息确认

| 项目 | 内容 |
|------|------|
| **触发** | 剧本定义的最低采集量达成 |
| **响应** | 双栏：左栏实时生成调研总结，右栏 Neo 逐一确认信息点。学员可修正不准确的总结 |
| **联动** | 修正后左栏总结实时更新。Neo 追问是否有补充。确认/修正后的信息写入记忆和统一 Database |

### 5. 回避行为识别

| 项目 | 内容 |
|------|------|
| **触发** | 对话式调研中学员连续 2 次回答模糊或转移话题 |
| **响应** | Neo 不再纰缠当前案例，温和过渡到新案例 |
| **联动** | 回避行为本身被记录为评估信息 |

### 6. 断点续播

| 项目 | 内容 |
|------|------|
| **触发** | 学员中途退出调研场域 |
| **响应** | 系统记录当前位置（对话进度或量表题号）。再次进入时从断点继续 |
| **联动** | 大厅课程目录中该 Activity 显示为 ▶ 学习中状态 |

### 7. 重新开始

| 项目 | 内容 |
|------|------|
| **触发** | 学员选择"重新开始" |
| **响应** | 从调研说明阶段重新开始，原数据保留存档 |
| **联动** | 新旧调研数据独立，不覆盖 |

### 8. 调研完成衔接

| 项目 | 内容 |
|------|------|
| **触发** | 最低采集量达成 + 学员确认无补充 |
| **响应** | 显示"下一个任务：[名称]" + "开始"按钮 + "返回大厅"链接 |
| **联动** | 与其他场域衔接方式一致 |

## 已确认决策

- [ADR-004] 调研行为由调研剧本控制
- 调研三阶段：说明（双栏）→ 执行（单栏沉浸）→ 信息确认（双栏）
- 调研复盘 = 信息确认 + 追问补充（不是苏格拉底式深度思考）
- 两种调研类型：对话式 + GUI 量表，由剧本选择
- 结束信号：剧本最低采集量 + 学员确认无补充
- BEI 只是一种调研方法，不是唯一的
- BEI 编造识别不考虑真假——不做"判官"
- 调研中 Neo 追问路径围绕 KGP 定义的能力维度
- 断点续播：对话和量表都从断点继续
- 可重新开始，原数据保留存档
- Inquiry 模块未来可扩展承接萃取、考试等信息采集类行为

---

## 本期不做的功能

- BEI 编造识别——设计哲学决定不做
- 调研结果自动影响学习路径——本期供报告和 Leo 引用，不自动调整路径（属于 L3 自适应）
- 萃取、考试等扩展调研类型——未来迭代
