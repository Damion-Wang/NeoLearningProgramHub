### 10.3 报告中心 Report Center

> 这是 § 10 的**核心模块**——管理端的主要交付能力 / 对外可举证报告产出场域。
>
> 双主体：**报告库**（不需要 Ora · 看历史 + 新建入口）+ **报告编辑**（需要 Report Ora · 协同编辑 + ChatBI 图表生成）。
>
> Ora 实例：**Report Ora** = § 10.2 Home Ora 之外的第 2 个 Ora 实例 / 信息共享 / 各有独特能力（详见 § 10.3.4 与 Home Ora 差异）。

#### 报告中心 = 2 主体 + 2 Tab + 6 图表

```
报告中心 Report Center
├── 报告库（不需要 Ora · § 10.3.2）
│   ├── Tab 1 管理员报告（当前管理员独立 / 全部可编辑 / 时间倒序 / 增删改查复制）
│   └── Tab 2 学员报告（项目过程产生 / 完成态 recap + 对练 + 高光库 / 按学员分组）
└── 报告编辑（需要 Report Ora · § 10.3.3）
    ├── 编辑器（WYSIWYG Markdown + 标准工具栏 + 插图）
    ├── 6 类图表（柱状 / 折线 / 表格 / 堆叠柱状 / 饼图 / 散点图）
    ├── 双触发（chat 直接 + 编辑器按钮多轮确认）
    └── 协同编辑（3 种方式：直接输入 / vibe 写作 / 划词修改）
```

#### 入口与模块关系

- **入口**：管理端左侧导航第 2 项 ★ / 也可从 Home 跳入（沿用 § 10.2.4.3）
- **创建报告路径**（本期统一手动）：报告库 Tab 1 顶部"+ 新建报告"→ 弹表单 → Ora 生成初稿 → 进入编辑器协同编辑
- **与其他模块关系**：
  - § 10.2 首页 Home：Home Ora 上下文传 Report Ora（信息共享 / 沿用 § 10.2.4.3）
  - § 10.4 项目配置：本期不联动
  - § 10.5 消息中心：本期不联动
  - § 9.3.3 学员侧 recap 报告：完成态 recap raw 进 Tab 2 / 详见 § 10.3.2.3

#### 10.3.1 业务定义

管理员产出**对外可举证 ROI 报告** + 历史归档查看 + 报告库管理 的场域。Ora 拉全程数据 + 管理员描述意图 + Ora 生成初稿 + 协同编辑 + 导出 PDF / Word。

**与学员端 § 9.3.3 recap 报告的边界**：

| 维度 | § 9.3.3 学员 recap 报告 | § 10.3 管理端 team report |
|------|----------------------|------------------------|
| 主体 | 学员侧个人反思 | 管理员侧团队汇报 |
| 粒度 | 个人 × Course | 团队 × 项目周期 |
| 触发 | Activity 完成事件 | 管理员手动新建（本期）|
| 编辑 | 学员不可改 | 管理员可改（协同编辑）|
| 数据共享 | recap 完成态 raw 进入管理端（沿用 § 9.3.3.2.10）| Ora 6 维 + Home Ora 跨模块记忆 |

#### 10.3.2 报告库

##### 10.3.2.1 整体结构

报告库主页 = **2 个 Tab 横排切换** + 顶部"+ 新建报告"按钮（仅 Tab 1 显示）。

报告库本身**不需要 Ora 参与**——纯 GUI 列表 / 查询 / 筛选 / 进入编辑器入口。Ora 在管理员**进入某个具体报告编辑**时才接管（详见 § 10.3.3）。

##### 10.3.2.2 Tab 1 · 管理员报告

**包含范围**：当前管理员账号下的全部报告——
- 全部报告统一为可编辑状态（不区分草稿 / 完成 / 已导出）
- 是否导出由管理员决定（导出不锁定 / 导出后仍可继续编辑）
- 沿用 § 10.3.6.1 每管理员独立报告库

###### 10.3.2.2.1 列表展示与动作

**列表展示**（每行 5 列）：

| 列 | 内容 |
|----|------|
| 报告名 | 文字（点击 → 进入编辑器修改 / 查看）|
| 创建时间 | 相对时间（如"2 天前"）+ tooltip 绝对时间 |
| 时间周期 | "项目至今" / "W4-W8" / "2026-04-01 至 2026-05-01" 等 |
| 人员范围 | "全部学员"或"X 人选定" |
| 操作 | 行末操作菜单（删除 / 复制 / 导出）|

**5 类动作**：

| 动作 | 触发 | 说明 |
|------|------|------|
| 新增 | Tab 1 顶部"+ 新建报告"按钮 | 弹表单 → 详见 § 10.3.2.2.2 |
| 删除 | 行末操作菜单 → "删除" | 二次确认 modal（"确认删除报告 X？此操作不可恢复"）|
| 修改 / 查看 | 点击报告名 → 进入编辑器 | 编辑态即查看态（沿用全部可编辑原则）/ 协同编辑详见 § 10.3.3 |
| 复制 | 行末操作菜单 → "复制" | 生成新报告 / 名称自动加"副本"后缀 / 内容完全复制 / 进入编辑器可改 |
| 导出 | 行末操作菜单 → "导出" | 详见 § 10.3.5 |

**查询 / 排序 / 筛选**：

- **查询**：顶部搜索框 / 报告名 + 关键词模糊匹配（同时搜报告名和正文）/ 实时搜索（输入即过滤）
- **排序**：默认按创建时间倒序（最新置顶）/ 可切换按报告名字母序 / 按时间周期起始日 / 按修改时间倒序
- **筛选**：时间范围（创建日期 起止 picker）/ 不含状态筛选（全部可编辑无状态区分）/ 不含模板筛选（本期仅 1 模板）
- **分页**：不分页（项目内报告数预计 ≤ 50 / 单页可承载）

###### 10.3.2.2.2 创建入口（统一手动表单）

**唯一入口**：Tab 1 顶部"+ 新建报告"按钮（本期不做项目里程碑（milestone）自动触发 / 详见 § 10.3.7）。

**点击 → 弹表单**（5 字段）：

| 字段 | 形态 | 必填 | 不选模板默认 | 选"综合报告"模板 |
|------|------|:-:|------------|----------------|
| 报告名称 | 文字输入 | ✓ | 空 | 空（模板不预填名称）|
| 模板选择 | 下拉（"不选模板" + "综合报告"）| 可选 | "不选" | "综合报告" |
| 时间周期 | 起止日期 picker（含"项目至今"快捷选项）| ✓ | 项目至今 | 项目至今 |
| 人员范围 | 全部学员（默认）+ 多选学员名单（可选切换）| ✓ | 全部 | 全部 |
| 报告描述 | 多行文本框 / **给 Ora 的写作指令** | ✓ | 留空 | "综合报告" |

**模板规则**：
- 本期 1 个系统预制 = "**综合报告**"（骨架详见 § 10.3.2.2.4 模板生成样例）
- 不可改 / 不可新增 / 由 admin 后台预制
- 未来扩展（V0.6.0+）：admin 在 § 10.4 项目配置中预制更多模板

**提交流程**：
```
管理员点提交
   ↓
Ora 拉数据（按时间周期 + 人员范围筛选）
   - 数据范围：§ 3.4.4.2 全 6 维 + Home Ora 跨模块记忆 § 10.2.4.3
   - 人员范围 在生成时锁快照——之后项目增删学员不影响本报告 / 报告内的"X 人选定"指向当时的学员名单
   ↓
Ora 根据"报告描述"生成完整初稿（含图表 + 文字 + 结构）
   ↓
进入报告编辑器（左主体 + 右 Report Ora chat）
   ↓
协同编辑（详见 § 10.3.3）
```

###### 10.3.2.2.3 跨 Tab 引用机制（按需摘取）

**Tab 1 报告中可引用 Tab 2 学员报告**作为附录 / 证据来源——

**引用形态**：
- ✅ 按需摘取原文片段（Ora 根据上下文从学员报告中摘相关段落 / 不全文拷入）
- ✅ 标注来源文件名（如"引自：张三 · Course 1 横向协作 · Course 个人报告 · 2026-04-15"）
- ❌ 不全文拷入（recap / 对练 / 高光卡 全文都不直接整段塞入 / 避免冗余）
- ❌ 不做超链接跳转（摘取后该副本独立 / 学员报告原文修改不回流）

**触发方式**：
- 编辑器内 chat："引用张三的 Course 1 recap 中关于团队协作的部分" → Ora 从 Tab 2 找该报告 → 摘取相关段落拷入光标位置 + 自动加来源标注
- 划词后追问："这部分加入张三 recap 的相关佐证" → Ora 摘取对应段落

###### 10.3.2.2.4 模板生成样例与内容规划（"综合报告"）

本期唯一模板"综合报告"的报告骨架——5 块结构 + 内容规划。Ora 据此生成初稿。

**报告 5 块结构**：

| # | 块 | 是什么 | 数据来源 | 形态 |
|---|----|------|---------|-----|
| ① | **项目总体表现** | 项目周期 / 总学员数 / 整体完成率 / 关键节奏 | 项目数据（§ 3.4.4.2 第 4 维）| markdown 段 + 进度数字 + 完成率柱状图 |
| ② | **学员分布与动态** | 登录态 + 学习进度态 + 三态分布 + 关键学员（领先 / 落后 各 Top 3）| 学员画像 + 项目数据 | markdown 段 + 饼图（三态占比）+ 表格（关键学员）|
| ③ | **内容推进情况** | Course / Activity 完成情况 + 卡点 + 投入时长分布 | 项目数据 + AOM 数据 | markdown 段 + 堆叠柱状图（学习时长分布）+ 表格（卡点 Activity）|
| ④ | **6 维画像聚合** | 项目均值 + 显著变化（项目级前后对比）+ 个体亮点 | 学员画像（6 维）| markdown 段 + 柱状图（6 维项目均值）|
| ⑤ | **关键发现与建议** | 结论性发现 + 后续动作建议 | Ora 综合上述 4 块 | markdown 段 / 列表式呈现 |

**每块内容规划**：

```
① 项目总体表现
   - 项目周期：YYYY-MM-DD 至 YYYY-MM-DD（共 N 周）
   - 总学员数：N 人 / 角色分布（HR + 运营 + 中期学员 + 新员工 等）
   - 整体完成率：X%（人均完成 X/Y Activity）
   - 关键节奏：标杆完成进度 + 中位数 + 最慢
   [图表：完成率柱状图 / Course 维度对比]

② 学员分布与动态
   - 登录态分布：活跃 X / 风险 Y / 沉默 Z（沿用 § 10.2.2.1 (5) 定义）
   - 学习进度态分布：领先 X / 跟上 Y / 落后 Z（沿用 § 10.2.2.2 (2) 定义）
   - 关键学员：领先 Top 3 / 落后 Top 3（含简短描述）
   [图表：饼图三态占比 + 表格关键学员]

③ 内容推进情况
   - Course 完成情况：各 Course 完成率
   - 卡点 Activity：完成率最低的 N 个 Activity（含描述：多少人卡 / 平均尝试次数）
   - 投入时长分布：各 Course 总时长占比
   [图表：堆叠柱状图（时长分布）+ 表格（卡点 Activity）]

④ 6 维画像聚合
   - 项目 6 维均值（项目至今）
   - 显著变化点：项目至今显著上升 / 下降的维度
   - 个体亮点：6 维某维度突出的学员
   [图表：柱状图 6 维项目均值]

⑤ 关键发现与建议
   - 1-3 条结论性发现（基于上述 4 块的综合判断）
   - 1-3 条后续动作建议（HR 可考虑的方向 / 不下结论由 HR 拍板）
```

**内容规划约束**：

- 不臆造外部信息（沿用 § 10.2.3.C.3 / 不引入项目外的数据 / 不预设公司日程冲突等臆测）
- 数据缺失 → "暂无数据"占位（沿用 § 10.3.3.3 数据原则）
- 第 ⑤ 块"建议"= 给 HR 的方向性参考（不出具体方案 / 不替决策 / 沿用 § 10.2.3.C.4）
- 整篇报告字数由 Ora 根据数据丰满度自主判断 / spec 不锁字数

> **管理员自定义不选模板时**：Ora 根据"报告描述"自由生成结构 / 不强制 5 块骨架 / 但需遵守上述内容约束

##### 10.3.2.3 Tab 2 · 学员报告

**包含范围**：项目过程中学员端产生的报告 ★ 3 类——
1. **完成态 recap 报告**（仅 Activity 完成事件后 / 红线已校准 § 9.3.3.2.10 / § 3.4.4 一致）
2. **对练报告**（所有学员的所有 practice 演练报告）
3. **高光卡**（每学员各自产出的高光卡集合 / Activity 级聚合入口在 § 10.2.2.3 (1) Activity 高光库）

**组织方式**：**按学员分组**——每个学员下展示其本人产出的 recap 报告 / 对练报告 / 高光卡：
```
张三（中期学员）
├── 📊 recap 报告
│   ├── Course 1 横向协作 · 2026-04-15 →
│   └── Course 2 辅导反馈 · 2026-05-02 →
├── 🎭 对练报告
│   ├── Activity 1 · 第 3 次 · 2026-04-10 →
│   └── ...
└── ✨ 高光卡（X 张 · 该学员本人产出）→

李四（新员工）
└── ...
```

**查询能力**：
- 学员姓名查询（顶部搜索框 / 模糊匹配）
- 报告名 + 关键词查询（在选定学员或全员范围内搜）

**不可编辑**——学员侧产出的报告管理端只读 / 可被 Tab 1 报告按需摘取（详见 § 10.3.2.2.3）。

**与 § 10.2.2.2 (5) 学员个人报告库的差异**：

| 维度 | § 10.2.2.2 (5) 学员个人报告库 | § 10.3.2.3 Tab 2 学员报告 |
|------|------------------------------|--------------------------|
| 入口 | 首页人员维度 → 点学员 → 弹窗 | 报告中心 → Tab 2 |
| 范围 | 单学员的报告 | 项目内全部学员的报告 |
| 用途 | 个体诊断 / 看具体学员产出 | 报告编辑时摘取引用 / 项目级综合查阅 |
| 形态 | 弹窗 + 新开页查看 | 主页内分组列表 |

#### 10.3.3 报告编辑

##### 10.3.3.1 编辑界面布局

沿用 § 10.2.5 GUI 布局原则——**左主体编辑器 + 右 Report Ora chat 侧栏**（约 70-75% / 25-30%）。

**编辑器顶部工具栏**（Markdown 标准功能 + 报告专属功能）：

```
[B 加粗] [I 斜体] [S 删除线]  | [H1] [H2] [H3]  | [• 无序] [1. 有序]  | [🔗 链接] [` 代码]  | [➕ 加图] [🖼️ 插图]  | [↶ 撤销] [↷ 重做]  | [📥 导出 ▾]
```

| 区段 | 按钮 | 说明 |
|------|------|------|
| 文字格式 | 加粗 / 斜体 / 删除线 | Markdown 标准 |
| 标题 | H1 / H2 / H3 | 多级标题 |
| 列表 | 无序 / 有序 | Markdown 标准 |
| 内联 | 链接 / 内联代码 | Markdown 标准 |
| 报告专属 | + 加图（6 类图表）/ 插图 | 详见 § 10.3.3.3 / § 10.3.3.2 |
| 历史 | 撤销 / 重做 | 沿用 § 10.3.3.4 撤销机制 |
| 导出 | PDF / Word 菜单 | 沿用 § 10.3.5 |

##### 10.3.3.2 编辑器能力

**Markdown 模式**：**WYSIWYG（所见即所得）**——类 Notion / Typora 实时渲染 / 管理员不需要看 md 源码。

**插图能力**：

| 来源 | 触发 |
|------|------|
| 本地上传 | 工具栏插图按钮 → 文件选择器 |
| 拖拽 | 拖图片到编辑区 → 自动插入光标位置 |
| 粘贴板 | 复制图片后 Ctrl+V → 自动插入 |

**不支持**：
- ❌ 视频嵌入
- ❌ 音频嵌入
- ❌ 复杂自定义 HTML 嵌入
- ❌ 实时协作光标 / 多人在线状态（沿用 § 10.3.6.2）

##### 10.3.3.3 6 类图表组件（ChatBI 模式）

**6 类清单**（仅做"文字替代不了的信息表达"图表）：

| # | 图表 | 形态 | 典型用法 | 复用关系 |
|---|------|------|---------|---------|
| 1 | **柱状图**（Bar 竖/横）| 单维度对比 | "对比 Course 1-5 完成率" / "Top 10 学员" | ❌ 新建 |
| 2 | **折线图**（Line）| X 时间 / Y 数值 / 单或多线 | "至今完成率走势" / "活跃度周变化" | ❌ 新建 |
| 3 | **表格**（Table）| 列可变 + 行动态 + 列点击排序 | "学员名单" / "Top N" / "Course 详情" | ✅ 复用 § 10.2.2.2 (1) + § 10.2.2.3 (1) |
| 4 | **堆叠柱状图**（Stacked Bar）| 每根柱内分段 | "Course 内 Activity 时长占比" | ✅ 复用 § 10.2.2.3 (2) |
| 5 | **饼图**（Pie）| 单一维度占比 | "活跃 / 风险 / 沉默 三态占比" | ❌ 新建 |
| 6 | **散点图**（Scatter）| X/Y 双维度相关性 | "学员投入时长 vs 完成率" | ❌ 新建 |

**数据原则**：
- 数据来自 Ora 6 维（§ 3.4.4.2）+ Home Ora 跨模块记忆（§ 10.2.4.3）
- 数据缺失 → "暂无数据"占位 + Ora 提醒
- 不引入外部信息（沿用 § 10.2.3.C.3）
- **写入即固化**——图表生成时拉的数据写入报告 / 此后不变 / 报告内不存在"刷新数据"动作

###### 10.3.3.3.1 双触发方式

**触发 A · chat 直接触发**（一步到位）：

```
管理员在 Report Ora chat："加一个 Course 完成率对比图"
   ↓
Report Ora 识别意图 + 选图表类型 + 拉数据 + 填充
   ↓
直接插入到编辑器光标位置
```

**触发 B · 编辑器按钮触发**（多轮确认）：

```
1. 工具栏点 "+ 加图" 按钮
   ↓
2. 弹图表类型选择器（6 类列表 + 缩略图）
   ↓
3. 选定类型 → 编辑区生成占位（灰色框 + "图表占位 · 待 Ora 填充"）
   ↓
4. 弹"用户指令"输入窗口（管理员描述需求）
   ↓
5. 提交 → Report Ora 接管 chat 区
   ↓
6. Ora 在 chat 与管理员确认细节（askUserQuestion / 数据维度 / 排序 / 标题）
   ↓
7. 管理员确认 "开始生成" → Ora 拉数据填入占位 → 占位变实图
```

##### 10.3.3.4 协同编辑

**3 种编辑方式**——管理员可在同一报告内任意混用：

###### 10.3.3.4.1 直接在编辑器中输入

管理员在编辑器主体直接打字 / 删字 / 调整结构——
- WYSIWYG 模式 / 即写即见
- 不依赖 Ora / 完全由管理员主导
- 支持 Markdown 标准快捷键（Ctrl+B 加粗 / Ctrl+I 斜体 等）
- 工具栏按钮辅助（详见 § 10.3.3.1）

###### 10.3.3.4.2 vibe 写作（描述意图 → Ora 生成整段）

```
管理员在 chat："写一段关于 Course 3 学员推进受阻的分析"
   ↓
Report Ora 拉数据 + 生成整段（段落长度由 Ora 根据描述自主判断）
   ↓
插入到编辑器当前光标位置
   ↓
管理员可手动调整位置 / 改文字
```

###### 10.3.3.4.3 划词修改

选中编辑器中的文字 → 弹出菜单，**3 类修改选项**：

| 选项 | 效果 |
|------|------|
| **扩写** | Ora 充实内容（加证据 / 加数据 / 加例子）|
| **缩写** | Ora 浓缩 / 简化 / 保留重点 |
| **划词后 chat 区输入指令** | 管理员选中文字 → 在 chat 直接输入"改成更专业的语气" / "加一句对比同期数据" 等 → Ora 据此改 |

###### 10.3.3.4.4 数据范围

Report Ora 可拉的数据：

| 数据来源 | 说明 |
|---------|------|
| Ora 6 维上下文 | 沿用 § 3.4.4.2 全部 6 维（含完成态 recap raw + 对练报告 raw + 高光库）|
| Home Ora 历史对话 | 当前管理员在 Home Ora 的历次对话 → Report Ora 起底（沿用 § 10.2.4.3）|
| Tab 2 学员报告内容 | 用于跨 Tab 摘取引用（详见 § 10.3.2.2.3）|

**正文边界**：报告正文仅写入**结论性内容**——Ora 的推理过程 / 数据筛选过程 / 多轮讨论 停留在 chat 区，不进入正文。

###### 10.3.3.4.5 保存机制

**纯自动保存 / 无完成状态**——
- 每次编辑后自动保存到管理员个人报告库
- 不区分草稿 / 完成 / 已导出
- 退出编辑器再进入 → 看到上次离开时的状态

###### 10.3.3.4.6 撤销机制

**Ctrl+Z 编辑历史**——
- 支持 50-100 步撤销（业界标准范围 / 由研发决定具体上限）
- 跨会话持久——退出再进入仍可继续撤销之前的操作
- Tab 关闭后 history 至少保留最近一次会话范围（持久化策略由研发决定）

#### 10.3.4 Report Ora 与 Home Ora 差异

Report Ora 是 § 10.2 Home Ora 之外的**第 2 个 Ora 实例**——信息共享 / 各有独特能力。

**核心差异表**：

| 维度 | Home Ora（§ 10.2.3）| Report Ora（§ 10.3.3）|
|------|--------------------|-----------------------|
| 第一性目标 | 信息呈现 + 证据解读（让管理员发现） | 协助管理员产出对外可举证报告 |
| 主导动作 | 呈现 / 解读 / 钻取 | vibe 写作 / 划词修改 / ChatBI 图表生成 |
| 输出形态 | chat 内回答 + 4 类卡片 | 编辑器内文本 + 6 类图表 + chat 内确认对话 |
| 数据 snapshot | 准实时（§ 10.2.4.1）| 写入即固化（§ 10.3.3.3）|
| 状态 | 跨会话累积记忆 | 跨会话累积记忆 + 共享 Home Ora 上下文 |
| 是否产出长文 | ❌ 不在 Home 出长篇产出（§ 10.2.3.A）| ✅ 主战场是长篇报告 |
| 推送形态 | 4 类卡片（askUserQuestion / 证据卡 / 报告引用卡 / 学员快照卡）| 编辑器内插入（文本 + 图表 + 引用）|
| 决策边界 | 不出方案 / 不替决策（§ 10.2.3.C.4）| 不替管理员判断 / 仅辅助 HITL |

**信息共享**（双 Ora 实例间）：
- Home Ora 与管理员的对话历史 → Report Ora 可读
- Tab 2 学员报告内容 → 双 Ora 都可读
- **持久化记忆 = 管理员账号 × 项目维度**（仅同项目内 / 不跨项目 / 与 § 10.2.4.4 跨项目对比本期不做 一致 / 沿用 § 10.2.4.3）

**Ora 整体共性**（HR 拍板边界 / 转述 + memory_id 追溯 / 不直接联系学员 / 像人 > 懂 HR）→ 沿用整体 Ora 原则，详见 § 10.1.6 / § 3.3 Ora Soul / § 3.4.4 Ora 上下文范围。

#### 10.3.5 报告导出

**支持格式**：
- **PDF**（适合对外交付 / 版式固定）
- **Word / docx**（适合二次编辑 / 企业常用）

**双入口**：

| 入口 | 位置 | 适用 |
|------|------|------|
| 编辑器内 | 工具栏导出按钮 | 当前正在编辑的报告 |
| 报告库列表 | Tab 1 行末操作菜单 → "导出" | 任意历史报告快速导出 |

**关键规则**：
- 导出与"完成"无关——导出后报告仍可继续编辑（沿用 § 10.3.2.2 全部可编辑）
- 导出后不锁定 / 管理员可再次进入修改
- 历史导出版本不保存（每次导出生成新文件 / 报告本身留 Ctrl+Z 历史）

#### 10.3.6 系统约束

##### 10.3.6.1 多管理员场景

- **每管理员独立报告库**：A 看不到 B 的报告 / 同项目 Tab 2 学员报告共享

##### 10.3.6.2 跨模块衔接

- **从 Home 跳入**：Home Ora 上下文 + 关注的数据切片 通过双 Agent 信息共享传给 Report Ora（沿用 § 10.2.4.3）
- **与 § 10.4 / § 10.5**：本期不联动（§ 10.4 / § 10.5 是纯 GUI 模块）
- **与 § 9.3.3 学员侧 recap 报告**：完成态 recap raw 进 Tab 2 / 非完成态不进（沿用 § 9.3.3.2.10）

#### 10.3.7 本期不做

| 项 | 说明 |
|----|------|
| 项目里程碑（milestone）自动触发创建 | 本期仅手动新建 / 未来 V0.6.0+ |
| 多模板 | 本期仅 1 个"综合报告" / 未来由 admin 在 § 10.4 预制更多 |
| 多人同编 / 实时协作 | 协同需导出后线下 |
| 雷达图实时生成 | 截图替代 / 留 V0.6.0+ |
| 跨项目报告引用 | 本期单项目内引用（沿用 § 10.2.4.4）|
| 报告评论 / 反馈 | 不支持读者对报告评论 |
| 报告分享链接 | 不支持生成可分享链接 |
| 历史导出版本管理 | 每次导出生成新文件 / 不留版本树 |
| 视频 / 音频 / 复杂 HTML 嵌入 | 沿用 § 10.3.3.2 不支持清单 |
