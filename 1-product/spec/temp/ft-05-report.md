## A. 报告生成

```
报告生成
├── A1 触发与入口
│   ├── A1.1 Course结束后固定生成
│   │   └── A1.1.1 自动触发 — 学员完成一个Course的所有Activity后 — 系统提示"生成个人报告"，进入报告教室
│   └── A1.2 未来入口预留
│       └── A1.2.1 辅导随时进入（本期不做） — 架构预留 — 未来可从辅导模块随时进入查看历史报告
│
├── A2 渐进式加载
│   ├── A2.1 祝贺动画
│   │   └── A2.1.1 开场动画 — 进入报告教室 — 播放祝贺动画和预告文案
│   ├── A2.2 后台并行生成
│   │   └── A2.2.1 碎片汇聚 — 报告生成中 — 后台并行拉取全部数据碎片（多模块数据汇聚），进行计算和分析
│   └── A2.3 分段渐进呈现

│
└── A3 情感化文案
    ├── A3.1 成长型语言
    │   └── A3.1.1 文案风格 — 报告全文 — 使用成长型语言，强调进步而非排位
    └── A3.2 个性化描述
        └── A3.2.1 引用具体行为 — 报告内容 — 引用学员在学习中的具体行为（如"你在第三次对练中首次主动使用了共赢策略"），评估结论有据可依
```

## D. 报告解析（学员与AI讨论报告）

```
报告解析
├── D1 讨论报告
│   ├── D1.1 提问报告内容
│   │   └── D1.1.1 学员提问 — 学员在右侧对话区提问（如"你为什么给我这个维度这么低的分？"） — AI引用具体的行为证据解释评估结论
│   ├── D1.2 质疑评估
│   │   └── D1.2.1 双向讨论 — 学员对报告中不认同的点提出反馈 — AI回应并解释，不是单向的"宣判"而是双向的"对话"
│   └── D1.3 深入解读
│       └── D1.3.1 按需展开 — 学员点击某个报告模块 — 可展开详情，AI在对话区同步提供该模块的深入解读
│
└── D2 图表联动Agent
    ├── D2.1 图表点击解读
    │   └── D2.1.1 自动解读 — 学员点击报告中任意图表的数据点 — 右侧Agent自动解读该数据背后的行为证据
    └── D2.2 对话引用图表
        └── D2.2.1 高亮跳转 — AI对话中提到某个维度或数据 — 看板自动高亮对应图表区域
```

    ├── E3.1 项目内排名

## G. 数据采集机制（碎片汇聚）

```
数据采集机制
├── G1 碎片生产规则
│   ├── G1.1 学习特质碎片
│   │   └── G1.1.1 Activity级汇总 — 每个Activity完成后 — 生成一个汇总碎片（从"每SCO一个"优化为"每Activity一个"，减少AI调用成本）
│   ├── G1.2 认知转化碎片
│   │   └── G1.2.1 Quiz级生产 — 每个Quiz答题闭环后 — 生成认知转化碎片
│   └── G1.3 能力评估碎片
│       └── G1.3.1 对练级生产 — 每次对练完成后 — 生成能力评估碎片
│
├── G2 碎片来源优先级
│   ├── G2.1 最高权重
│   │   └── G2.1.1 Quiz互动数据 — 报告数据权重 — Quiz互动数据权重最高
│   ├── G2.2 次高权重
│   │   └── G2.2.1 对练行为数据 — 报告数据权重 — 对练行为数据次之
│   ├── G2.3 中等权重
│   │   └── G2.3.1 测评数据 — 报告数据权重 — 测评数据中等权重
│   ├── G2.4 较低权重
│   │   └── G2.4.1 辅导对话数据 — 报告数据权重 — 辅导对话数据较低权重
│   └── G2.5 最低权重
│       └── G2.5.1 被动信号数据 — 报告数据权重 — 停留时长、回翻等被动信号权重最低
│
└── G3 行为改变证据链
    └── G3.1 全过程证据
        └── G3.1.1 证据链汇聚 — 报告生成时 — 按能力维度汇聚全模块证据：初始表现（课前测评/学员画像）→学习过程（授课互动/对练行为/辅导求助）→最终表现（课后测评/对练改善）
```
