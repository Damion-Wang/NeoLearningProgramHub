## 来源2: AOM 结构分析（横向协作样例）

详细报告：temp/aom-analysis.md

### 需要更新到需求文档的内容

1. **数据模型精确化**：五层结构 Project → Module → Activity → SCO(scoFlow) → Segment
   - 需求文档中的 "Course" 在 AOM 中实际叫 "Module"
   - SCO 下面还有 Segment 层（PPT 页面级别）
   - 无 segmentList 的 SCO 使用 `sco` 对象（quiz 等交互类型）

2. **SCO 类型补充**：
   - 目前 AOM 只有两种 interactionType：NARRATION（自动播放）和 CHAT（多轮对话）
   - scoType 只出现了 quiz，其余为隐含的讲授类型（通过 segmentList 直接嵌入）
   - 未来需要增加 drill、assessment 等 scoType

3. **自适应机制的技术实现**：
   - `separateTag`：定义分流标签（如防御拖延型/利益博弈型/情绪对抗型）
   - `groupKey`：标识同一槽位的不同版本
   - `tutorMode=1`：标记需要分流的 SCO
   - 课前 CHAT 的 passConditions 输出标签 → 后续 SCO 的 separateTag 消费
   - **这就是 L2 自适应的具体实现方式**

4. **interactionPoint 结构**（每个 SCO 都有）：
   - blueprint：教学设计意图（给 AI 读的）
   - rule：交互结束条件
   - description：模块描述
   - aiTutorDialogue：AI 教师对话预设
   - interactionType：NARRATION / CHAT

5. **内容驱动的直接证据**：
   - passConditions（通关条件）以自然语言写在数据中
   - solution（AI行为准则）以自然语言写在数据中
   - blueprint（教学设计意图）以自然语言写在数据中
   - 平台不硬编码业务逻辑，全部由 LLM 解释执行

## 来源5: 对练 AOM + 剧本 + SIDE 项目（已完成）

详细报告：temp/drill-analysis.md

### 需要更新到需求文档的内容

1. **对练AOM结构差异**：
   - 对练AOM的directScoContent为null，不走授课的"段落驱动"逻辑
   - 通过resourceId/artifactId/competencyArtifactId三个外部引用关联剧本
   - 授课的blueprint/rule/solution三元组在对练中被剧本的behavioral_patterns/success_criteria/decision_points替代

2. **对练剧本是深度结构化的**：
   - 角色心理画像（70+字段）
   - 递进式行为反应规则（如"用户施压则翻旧账"）
   - 防剧透指令
   - 4阶段能力触发图谱
   - L1-L5 五级评估标准

3. **对练四阶段技术实现已落地**：
   - 导学（guide）：三步走，从剧本提取信息发送卡片
   - 演绎（deduction）：**导演-演员分离模式**（Director编排→Actor表演→Narrator旁白）
   - 复盘（review）：苏格拉底追问，基于演绎对话记录
   - 报告（report）：三次并发LLM（评分+高光+提升），上报6种指标

4. **能力评估**：
   - 实时评估（进度百分比+目标达成）+ 事后评估（L1-L5维度评分）并存
   - 评分完全依赖LLM判断，无硬编码规则
   - competencyGoals有4个能力维度，每个有5级rubric

5. **当前只实现Roleplay一种玩法**——角色互换/费曼/辩论/多人均未实现

6. **excludeReport: 1**——对练AOM中明确排除了报告生成，与创始人说的"对练报告不是必须的"一致
