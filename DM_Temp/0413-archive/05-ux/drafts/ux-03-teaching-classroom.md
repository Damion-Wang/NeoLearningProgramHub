## 2. PPT讲解场景 (NARRATION)

### 三栏内容定义
- **左侧目录区**: Tab1 SCO缩略图列表（当前页高亮边框），Tab2 Project完整目录
- **中间看板区**: PPT图片全宽渲染，一个NARRATION SCO = 一张PPT页
- **右侧对话区**: AI基于blueprint自主讲解，流式逐字输出
- **默认比例**: 10:55:35

### 关键交互

| 交互 | 触发 | 响应 | 联动 |
|------|------|------|------|
| AI讲解 | SCO加载完成 | AI基于PPT+blueprint开始讲解，流式输出 | PPT图片在看板区展示 |
| 互动问题 | AI讲到KGP预设互动点 | AI在对话区提出关联知识点的问题 | 无 |
| 学员沉默 | 互动问题后10-15秒无回应 | AI自然过渡继续讲解，不追问 | 记录被动信号 |
| 插嘴提问 | 学员在输入框打字发送 | AI暂停讲解→简短回答→自然衔接回原位 | PPT保持不动 |
| 回顾请求 | 学员说"再讲一下..." | 看板切换到对应PPT页→AI精简重讲→回到当前页 | 看板切换+恢复 |
| 引用槽触发 | AI讲到必选引用位置 | AI必须引用学员FEEDBACK_COLLECT场景 | 无 |
| 翻页 | 当前SCO讲解完成 | 调度引擎推进→看板加载下一页PPT | 缩略图列表高亮移动 |
| 查看已完成页 | 点击缩略图列表中的已完成页 | 看板切换到该页（只读回顾） | 不影响当前讲解进度 |

### 状态变化

| SCO类型设置 | 未设计互动 | 有预设互动 |
|------------|-----------|-----------|
| AI行为 | 纯讲解，不自主插入互动 | 在预设点提出互动问题 |
| 学员选择 | 安静听完 | 回答互动 或 沉默跳过 |

### 组件需求

| 组件 | 规格 |
|------|------|
| PPT渲染器 | 支持HTML或图片格式，全宽自适应，保持比例 |
| SCO缩略图列表 | 纵向排列，当前页蓝色边框高亮，已完成页可点击 |
| Tab组件 | 两个Tab，激活指示线 |
| 流式消息 | AI消息逐字显示，打字机效果 |

## 3. 视频播放场景 (VIDEO)

### 三栏内容定义
- **左侧目录区**: 同PPT场景
- **中间看板区**: HTML5视频播放器 + 自定义控制栏
- **右侧对话区**: 视频播放中AI待命；播放后AI总结+引导思考
- **默认比例**: 10:65:25

### 关键交互

| 交互 | 触发 | 响应 | 联动 |
|------|------|------|------|
| 播放视频 | 学员点击大播放按钮 | 视频开始播放（不自动播放） | 无 |
| 暂停/播放 | 点击控制栏按钮 | 视频暂停/恢复 | 无 |
| 进度拖动 | 拖动进度条 | 跳转到指定位置 | 记录跳过区间 |
| 倍速切换 | 点击倍速按钮 | 0.5x/1x/1.5x/2x | 无 |
| 插嘴提问 | 视频播放中学员输入问题 | 自动暂停视频→AI回答→提示"视频已暂停，点击继续" | 视频暂停 |
| 播放完毕 | 进度达100% | 视频停在最后帧→AI发送总结+思考引导 | 对话区激活 |

## 4. Quiz互动场景 (QUIZ)

### 三栏内容定义
- **左侧目录区**: 同PPT场景
- **中间看板区**: 题干 + 辅助文件 + GUI交互组件
- **右侧对话区**: 苏格拉底式追问（问答题）/ 提交后反馈（选择/判断/连线）
- **默认比例**: 10:40:50

### 四种题型的看板+对话配合

#### 问答题（苏格拉底式）
```
看板区                              对话区
┌────────────────────┐    ┌────────────────────┐
│ 题干:               │    │ AI: 如果下属上周     │
│ 用BIC法的B应该      │    │ 迟到3次，用BIC法     │
│ 怎么描述？          │    │ 的B怎么描述？        │
│                    │    │                    │
│ [辅助图片/表格]     │    │ 学员: 你上周迟到3次  │
│                    │    │                    │
│                    │    │ AI: 不错，但"迟到"   │
│                    │    │ 是否有主观成分？     │
│                    │    │ (追问第1/最多5轮)    │
└────────────────────┘    └────────────────────┘
```

**AI判断逻辑:** 当学员回答覆盖核心要素（脚本定义的关键点列表）→ 提前结束追问

#### 选择题
```
看板区                              对话区
┌────────────────────┐    ┌────────────────────┐
│ BIC法的正确顺序？   │    │                    │
│ ○ A. 行为-结果-影响 │    │ (等待学员作答)      │
│ ○ B. 行为-影响-后果 │    │                    │
│ ● C. 行为-影响-期望 │    │ AI: 没错！BIC的     │
│                    │    │ 正确顺序是...       │
│     [提交]         │    │ (提交后即时反馈)     │
└────────────────────┘    └────────────────────┘
```

#### 判断题
看板: 陈述 + 对/错按钮 → 提交后对话区AI反馈

#### 连线题
看板: 左右两列概念 + 拖拽连线 → 提交后对话区AI逐对反馈

### 提示机制
- 学员说"不知道"/"给我提示" → AI给引导性提示（最多2-3次）→ 仍不会则给出答案+解释

## 5. 闭环反馈场景

### 课前采集 (FEEDBACK_COLLECT)

**看板区:** 空白或简要主题说明
**对话区:** AI以自然对话引导学员分享工作场景

```
AI: 在开始今天的学习之前，我想先了解一下你在工作中的真实情况。
    你最近在团队沟通方面有什么让你头疼的事吗？

学员: 我的下属小王最近绩效不好，不知道怎么跟他谈

AI: 能具体说说吗？小王绩效不好体现在哪些方面？

学员: 上个月项目完成率只有60%...

AI: 谢谢你的分享。接下来我们一起学习一些方法，看看能不能帮到你。
```

**退出条件:** 至少1个场景（含人物+事件+处理方式）或5轮上限

### 课后复盘 (FEEDBACK_REVIEW)

**数据为空降级:** AI引导现场分享简单场景 → 采集1轮后进入正常REVIEW

## 6. 自适应教学视觉

- **L1自适应（每次交互）:** 语言风格+深度+举例实时调整，学员无感知
- **L2版本切换:** AI自然过渡讲解（"让我换个角度来看"），看板内容无缝切换
- **风格底线:** 始终专业友善，不使用不当用语
