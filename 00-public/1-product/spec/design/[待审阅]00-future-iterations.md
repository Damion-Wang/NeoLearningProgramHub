# 未来迭代计划

> 收纳所有"本期不做但未来要做"的功能和设计方向。各模块 spec 中的"本期不做"部分仅列出与该模块直接相关的内容，跨模块和全局性的未来计划统一在此管理。

---

## 学员端

### 大厅
- **看板区与 Leo 联动**：点击看板数据可触发 Leo 解读（v3.1 设计哲学"看板区与对话区双向联动"）
- **社区功能建设**：从 Constructing 占位发展为完整社区——价值交换机制（先贡献才能看到别人想法）、同位置同学发现推送、高频问题互动、组织者下发推荐内容
- **左侧导航扩展**：课程目录、学习进度、消息中心等独立导航入口
- **从 Leo 随时进入报告**：Leo 对话中可随时查看历史报告（本期报告仅 Course 结束后固定生成）

### 授课教室
- **HTML 替代 PPT**：PPT 转 HTML 后实现交互式暂停、弹出提问、内嵌练习
- **"电影解说"式旁白介入**：授课中暂停并提问"你的隐含假设是什么？这是事实还是观点还是情绪对抗？"
- **动态新增 Quiz SCO**：学员要求"再出一道题"时在 AOM 结构中动态插入新 SCO 节点
- **标准 MP4 视频格式**：替代本期的伪视频格式

### 对练教室
- **老手模式**：Neo 默默观察、练完再谈，不打扰沉浸感（本期做新手模式）
- **情绪/耐心值显示**：Actor 角色有可见的情绪状态指示（如耐心值变化），让学员感知对方状态并调整策略
- **多人对练**：多个 Actor 角色 + 多个学员的团队模拟

### 报告教室
- **BI 图表组件库**：雷达图、热力图、饼图、折线图、柱状图，通用图表组件库
- **组织洞察维度**：个体/群体画像、人岗映射、群体共性发现

## 管理端

- **管理端 Agent 命名与独立化**：给 HR/运营端 Agent 独立命名，根据运营特殊需求独立化
- **看板区与 Agent 联动深化**：更丰富的双向联动模式
- **智能预警评价**：在数据呈现基础上加入智能判断
- **临时任务下发**："单向群聊"模式
- **CEO 汇报 PPT 生成**：从报告格式升级到 PPT
- **学员分组分析**：规模扩大后支持部门/分组维度

## 通用/全局

### 角色体验
- **Leo/Neo 转场 UX 设计**：从大厅进入教室时 Leo→Neo 的视觉区分和过渡体验设计
- **Leo 风格按课程调整**：通过 KGP 生产不同 Context 包，不同课程的 Leo 有不同的 Coach 风格

### 行为数据采集体系
- 设计大量可观察行为点（接受/拒绝建议、修改、备注、收藏等）
- 区分"用户主动"和"系统主动"
- 行为数据是报告分析的基础

### 自适应（高级）
- **L3**：Activity 顺序调整——根据学员能力跳过/重排学习活动
- **L4**：动态内容生成——实时生成新 Quiz/新模块/临时文件

### 移动端
- PC 和移动端完整覆盖功能，用功能区分场景
- 移动端可能以全语音/全对话模式为主，适合复习和紧急求助场景

### 复习模式（待讨论）
- 学员复习已学内容的机制——具体方案（在 Leo 对话中做 / 独立复习入口 / 其他形式）待讨论

### 催学渠道与效果追踪
- 根据 GPB 能力增加站外提醒渠道（微信/邮箱/短信等）
- 站外渠道支持完整漏斗追踪：发送 → 打开 → 登录 → 完成活动
- 站内消息的活动完成检测（登录→查看→完成的后半段）

### 其他
- 企业画像接入（获取方式待定）
- 数据双向回流（管理端分析结果反向优化学员端体验）
- 新模块类型：沙盘推演、研讨、播客、案例拆解工坊
