## 1 lecture（授课）

> **总览**：lecture 是学员端 3 个学习场域之一。Neo 在此场域调用以 **讲授 + 解惑 + 追问** 为主的教学动作（详见 § 2.5 矩阵），按 AOM 脚本编排的 SCO 序列逐步带学员 **吸收概念 / 实时检测教学信号 / 闭环采集学员场景** —— 是"概念教学 + 信号检测 + 闭环采集"的核心场域。
>
> 本节给出 lecture 的 **业务定义** / **形态** / **内容** / **Neo 行为** / **学员交互** / **约束** / **协同** / **边界** 8 个维度的需求描述。


### 1.1 业务定义

#### 1.1.1 一句话定位

> lecture（授课）是 Neo 按 AOM 脚本演绎结构化教学的场域——通过 **讲授概念 / 播放案例 / 问题检测 / 疑惑答疑 / 闭环采集** 等动作，结合学员状态感知及时调整授课方法，确保学员"吸收"而不只是"听过"。

#### 1.1.2 跨场域 Neo 身份对照

跨场域 Neo 是同一个人（同 voice / 方法论 / 记忆，详见 § 2.5 + § 3.2），但 lecture 调用的 skill 组合以"讲授"为主导。**学员侧 GUI 不显式呈现"Tutor"标签**——通过场域转场和动作差异让学员感知到"Neo 的不同侧面"。

spec 内部场域差异化对照（不在产品界面呈现）：

| 场域 | Neo 主导能力 |
|------|-----------|
| 大厅 | 陪伴 / Bloom 引导 / 情绪感知 |
| **lecture** | **结构化教学（概念演绎 + 信号检测 + 闭环采集）** |
| practice | 旁观练习（卡死介入 / 复盘） |
| recap | 解读反问（不替学员总结） |

### 1.2 界面布局形态

#### 1.2.1 三区视觉总图

lecture 沿用 § 4.2 学员端 UI 设计统一规范——Topbar + Subbar + 全幅课件区 + 右侧 Neo Chat + **左侧覆盖式抽屉**。

```
┌──────────────────────────────────────────────────────────────────┐
│ Topbar（项目名 + 周数 · bell · avatar）                            │
├──────────────────────────────────────────────────────────────────┤
│ Subbar（抽屉 btn · Activity 名 · 进度条 · 计时器）                 │
├──────────────────────────────────────────────┬───────────────────┤
│                                              │                   │
│ 抽屉 ←覆盖弹    课件区（全幅）                │ Neo Chat          │
│                                              │ （右侧固定）       │
│ · Project 目录   · 当前 SCO 演绎              │                   │
│ · Activity 目录  · SLICE / VIDEO / QUIZ      │  · 授课口播       │
│                  · FEEDBACK 题干 + 提示在 Chat 作答 │  · 答疑口播  │
│                                              │  · FEEDBACK 答题区 │
└──────────────────────────────────────────────┴───────────────────┘

   笔记悬浮球（屏幕边缘常驻）
```

抽屉**从屏幕左侧覆盖弹出**（点击 Subbar 抽屉按钮触发），含 Project 目录 + Activity 目录 两 Tab；关闭后课件区回到全幅。

#### 1.2.2 覆盖式抽屉 · 两 Tab

抽屉内两 Tab 并列：

| Tab | 用途 | 状态符号 |
|-----|------|---------|
| **Tab1: Project 目录** | 全项目 Course × Activity 浏览；**可切换其他已解锁 Activity**——弹出弹窗确认是否离开当前 Activity，确认后跳转 + 触发断点续播保存 | ✅ 已完成 / ▶ 学习中 / 🔒 锁定 |
| **Tab2: Activity 目录** | 当前 Activity 的 SCO 序列；**可回跳已学过的 SCO**（详见 § 1.5.2 ⑧ 主动回顾跳转） | ✅ 已学完 / ▶ 学习中 / 🔒 锁定 |

#### 1.2.3 右侧 Neo Chat（两态）

Neo Chat 抽屉**固定常驻**右侧，承载 Neo 的两种口播：

| 口播 | 触发 | 内容 |
|------|------|------|
| **授课口播**（主动） | Neo 按 SCO 序列驱动 | 概念讲解 / Quiz 引导 / 采集引导 |
| **答疑口播**（被动） | 学员插嘴提问 | 即时回应学员问题，回答完恢复授课口播 |

切换规则详见 § 1.4.4。

**学员反馈入口**：每一条 Neo 消息下都附 **点赞 / 点踩反馈入口**（功能尽量复用 XUI 原有组件），学员对 Neo 讲解 / 答疑质量的反馈通过此入口收集。

**会话历史持久化**：同 Activity 内多次进入 Chat 历史**接续保留**；跨 Activity 时 Chat 历史**重置**（Neo 自身记忆通过 [05-learner/06-cross-context § 4] 跨场域记忆机制保留，与 UI Chat 历史是两件事）。

#### 1.2.4 Subbar（学员端 3 学习场域共用 · 顶部 Topbar 之下）

lecture 在 Topbar 之下 + 课件区之上设 **Subbar**（lecture / practice / recap 3 学习场域共用 / 大厅不出现）：

| 位置 | 元素 | 说明 |
|------|------|------|
| **左** | 抽屉按钮 | 点击触发覆盖式抽屉（Project 目录 / Activity 目录 两 Tab）|
| **左** | Activity 名 | 当前 Activity 标题 + 类型标 (· 授课) |
| **中** | 进度条 | 当前 Activity 内 SCO 进度（已完成 / 总数）|
| **右** | 学习计时器 | 本 Activity 累计学习时长（不精确 / 不写入画像 / 后续基线积累后再决定是否纳入）|

**暂停状态**（学员触发"随时打断" § 1.5.3 ⑦ 时）显示在 **课件区中央覆盖**——"已暂停 / 点击继续"——不在 Subbar 呈现。

**主动回顾跳转**（学员点击已学完 SCO，详见 § 1.5.2 ⑧）期间不做特殊视觉提示——状态等同于 ⑦ 静默暂停 + 答疑口播。

### 1.3 课件区内容模型

#### 1.3.1 SCO 4 类总览

lecture 中课件区按 SCO 类型呈现内容。SCO 类型沿用 § 5.2.4 已定 4 大类：

| SCO 类型 | lecture 中的呈现 | 答题位置 | 是否有授课口播 |
|---------|------------------|---------|--------------|
| **SLICE**（PPT/HTML 知识切片）| 图片 / HTML 渲染 | — | ✅ 有 |
| **VIDEO**（视频）| 视频播放器 | — | ❌ 无（视频自身承载） |
| **QUIZ**（测验）| 题干 + 答题区（7 类题型） | 课件区（问答题除外，详见 § 1.3.4）| ✅ 有 |
| **FEEDBACK_COLLECT + FEEDBACK_REVIEW**（**双头配对**）| 课件区**仅显示题干 + 提示** | **右侧 Chat 区**（学员打字 / 语音输入）| ✅ 有 |

> **FEEDBACK 双头配对**：collect（课前 / 章前采集）在前 / review（课后 / 章后复盘）在后 / 同一 memory_id 串联（详见 § 1.3.5）。

#### 1.3.2 SLICE 呈现规范

- **图片**：渲染于课件区主区，自适应屏幕比例
- **HTML**：用于动画 / 互动图示 / 数据演示
- Neo 主动做授课口播——**不照念 PPT**，按教学提示和学员状态自主发挥（详见 § 1.4.3）

**授课口播支持暂停 + 打断**（两种独立场景）：

| 场景 | 触发 | 行为 | 详见 |
|------|------|------|------|
| **静默暂停** | 学员在课件区点击"暂停"按钮 / 按空格键 | Neo 停止口播 → 学员手动"继续"恢复 | § 1.5.3 ⑦ 随时打断 |
| **插嘴提问** | 学员在 Chat 区发送一条消息 | Neo 暂停口播切答疑口播 → 答完自动恢复 | § 1.5.3 ⑦ 随时打断 |

#### 1.3.3 VIDEO 呈现规范

- **标准播放器**：暂停 / 进度条 / **倍速允许**
- **首次观看不允许向后拖动进度条**（防跳过）—— 已经向前播放过的位置允许往前拖再回到最新播放断点；尚未播放过的位置禁止前拖
- **Neo 不做授课口播**（视频自身承载内容）
- 学员可在 Chat 区随时插嘴提问触发答疑口播
- 视频后如需总结 / 复盘，由紧接的 SLICE SCO 承担授课口播
- **未看完退出，下次进入同 Activity 从上次播放断点继续**（首次观看不允许向后拖动的限制依然生效）
- 本期使用伪视频格式（详见 § 1.8 ②）

#### 1.3.4 QUIZ 呈现规范 + 7 题型

Quiz 在 lecture 中的 7 题型 + 输入区分 + HTML 交互支持：

| 题型 | 输入位置 | 交互 | HTML 交互 |
|------|---------|------|---------|
| **单选题** | 课件区 | 点选答题 | ✅ |
| **多选题** | 课件区 | 多选答题 | ✅ |
| **判断题** | 课件区 | 点选 √/× | ✅ |
| **填空题** | 课件区 | 输入框作答 | ✅ |
| **连线题** | 课件区 | 拖拽连线 | ✅ |
| **排序题** | 课件区 | 拖拽排序 | ✅ |
| **问答题** | Chat 区 | 打字 / 语音输入 | ❌ |

> **HTML 交互支持**：除问答题外其他 6 题型都允许 HTML 交互（点选 / 拖拽 / 连线 等）；问答题为纯文本 / 语音输入。

题干可附图片 / 表格辅助（点击放大）。Neo 答题反馈遵循 AOM 脚本规则：是否允许提示 / 是否透露答案 / 反馈深度（详见 § 1.5.2 ④）。

> **Quiz 用途多样（★）**：Quiz 是 lecture 中的**互动手段**，不只是"考核"。KGP 在 AOM 中通过 **passCondition（通关条件）+ solution（引导话术）** 两个字段的灵活运用，可让同一种 Quiz 题型承担多种用途——
>
> - **考核类**：有标准答案 / 验证学员是否掌握（passCondition = 答对某选项 / 输出含某关键词）
> - **思考引发类**：无标准答案 / 引发学员主动思考（passCondition = 完成作答即可，不判对错）
> - **反思激起类**：用学员的回答作后续 FEEDBACK 素材或对话锚点（passCondition = 给出非空回答）
>
> 因此本 spec 中 Quiz 的"完成判定"统一以 **passCondition 满足**为准，**不假定"答对"**（详见 § 1.6.1 + § 1.7.1）。任何"Quiz 命中率 / 通关率 / 错题模式"等指标在产品层都不作为通用统计——具体 Quiz 类型决定能否做这类统计，由 KGP 内容设计层面把控。

#### 1.3.5 FEEDBACK 拆 COLLECT 与 REVIEW

FEEDBACK 类型在 lecture 内部拆为两个子型：

| 子型 | 时机 | 内容 |
|------|------|------|
| **FEEDBACK_COLLECT**（课前 / 章前采集）| 教学开始前 | 采集学员真实场景 / 现状 / 困惑 |
| **FEEDBACK_REVIEW**（课后 / 章后复盘）| 教学结束后 | 引导学员用学完的概念回看自己的真实场景 |

**作答位置**：FEEDBACK 课件区**仅显示题干 + "请在右侧 Chat 区作答"提示**；学员实际作答在右侧 Neo Chat 区（打字 / 语音输入），不用担心写得长，越具体越好。

**FEEDBACK_REVIEW 中 Neo 综合调用以下 3 类信息**：

| 信息源 | 内容 | 用途 |
|--------|------|------|
| **COLLECT 上下文** | 学员在 COLLECT 阶段自述的真实场景（场景描述 / 困惑 / 情绪 / 关键词等） | Neo 用学员自己说过的场景作 REVIEW 锚点，不讲教材通用例 |
| **当前 Activity 知识点** | 本 Activity 教学覆盖的概念 / 工具 / 框架 | Neo 把学完的内容映射到学员真实场景，做"概念 → 应用"对照 |
| **Neo 自身记忆** | 跨 Activity 的学员画像 / 历史表现 / 跨场域行为模式 | Neo 引用更长程的学员状态做 REVIEW 反思引导（详见 [05-learner/06-cross-context § 4] 跨场域记忆） |

> **配对约束**：每个 COLLECT SCO 必须有对应 REVIEW SCO 配对（KGP 编排强制，详见 § 1.6.3）。具体存储 schema + 召回机制属研发实现层，本 spec 不定义。

#### 1.3.6 多版本切换 + 隐性打标

**多版本机制**（详见 § 5.3.2）：同一个 SCO 可挂多个 asset 版本，按学员标签（separateTag）切换。

实际效果（学员视角）：

- 学员甲（销售标签）→ SCO 5 看到"销售场景案例"asset
- 学员乙（建筑标签）→ SCO 5 看到"建筑场景案例"asset
- **教学逻辑相同 / 内容呈现不同**

**学员标签（separateTag）的获取方式**：

| 阶段 | 打标方式 |
|------|---------|
| **早期（v0.5 本期）** | KGP 内容老师在 AOM 编排时**预设打标点位**——通常在 FEEDBACK_COLLECT / 早期 QUIZ 后触发，规则由内容老师写入 |
| **后期（演进方向）** | Neo 自主决定何时打标 / 打什么标 / 置信度判定 |

**频次与有效期**：

- **一个 Activity 内可多次打标**——后一次打标覆盖前一次（同维度）
- **separateTag 仅 Activity 内有效**——跨 Activity 不传递；下一 Activity 如需个性化版本切换需在自己的导学阶段重新打标

> **不做独立 ASSESSMENT_TAG SCO 类型**（v0.4 设计取消）——打标融入互动 SCO 的通关条件 / passConditions，学员无感。
>
> **本节不写实现层细节** —— 切换机制 / 打标具体实现 等属 Prosona 演绎架构，由研发设计。

#### 1.3.7 AOM 前端演示事件

lecture 课件区演示规则（高亮 / 动画 / 进度提示 等）由 AOM 内容包按场域演绎规则定义。详见 § 5.3.2。

### 1.4 Neo 在 lecture 的行为

#### 1.4.1 7 动作调用

lecture 在 [02-foundation/02-methodology § 1.5] 7 动作矩阵中的分工（Neo 跨场域同身份 / 详见 § 1.1.2）：

| 动作 | 权重 | lecture 中的体现 |
|------|------|-----------------|
| **讲授** | ★ 主场 | SLICE 概念演绎 / 视频复盘 / 综合升华 |
| **解惑** | ✓ 辅助 | 学员插嘴提问 → 答疑口播 |
| **讨论** | ✓ 辅助 | SCO 间互动 / Neo 自主插互动问题 |
| **考核** | ✓ 辅助 | 即时验证 Quiz（嵌入 SCO，作用较轻） |
| **练习** | ✓ 辅助 | QUIZ SCO 答题 / 学员主动出题 |
| **追问** | ✓ 辅助 | Quiz 错答时苏格拉底式引导 / 问答题深入追问 |
| **反思** | — 不出现 | lecture 不主导反思（→ recap） |

#### 1.4.2 感知（lecture 侧重点）

> 感知体系 Truth Source 见 § 2.3。本节仅讲 lecture 场域的侧重点：

- **5 教学信号是 lecture 主场**（场域处理）：①缺概念 / ②忘了 / ③用错 / ④选错工具 / ⑤错误心智模型 —— lecture 是教学反馈的核心场域
- **Quiz 反馈即时感知**：Quiz SCO 答题 + FEEDBACK 提交时是高密度感知窗口
- **6 维画像写入主要数据源**：lecture 学员行为是 6 维聚合的关键来源（详见 [02-foundation/02-methodology § 1.3.3]）；6 维呈现位置见 [05-learner/02-hub § 1.1] B.2.B.4 + [05-learner/02-hub § 1.2.1] 项目总览区雷达图

#### 1.4.3 自主发挥边界

Neo 在 lecture 的自由度：

| Neo 可以 | Neo 不可以 |
|---------|-----------|
| ✅ 按学员认知负荷调整讲解节奏 | ❌ 跳过任何 SCO |
| ✅ 根据学员行业 / 现状适配举例 | ❌ 改变 AOM 序列顺序 |
| ✅ 在 SCO 之间插入互动问题（不新增 SCO） | ❌ 主动透露后续 SCO 内容（不剧透） |
| ✅ 根据感知决定追问深度 | ❌ 透露 Quiz passConditions 下的答案（详见 § 1.6.1） |
| ✅ 调整答疑回应的详略 | ❌ 主动跳到学员"看起来已掌握"的下一段 |

> **答疑口播知识范围**：
> - **范围内**：当前 Activity 内的教学知识 + 已完成 Activity 的知识 + 学员画像数据（记忆工程）
> - **范围外（未解锁 Activity）**：防剧透——Neo 告知"后面会讲到"+ 写入记忆
> - **完全超出 Course Pack 范围**（如医学 / 法律 / 投资）：按 § 2.8 教学边界**实诚拒答 + 引回主线**——"这个超出本期项目内容，我们先回到 X"

#### 1.4.4 两种口播 + 切换原则

**切换原则**：学员在 Chat 区发送消息（文字 / 语音）打断 Neo → Neo 暂停**授课口播** → 切答疑口播 → 答完自动恢复授课口播（不重复中间内容）。

**SCO 类型差异**：
- **SLICE / QUIZ / FEEDBACK SCO**：暂停 Neo 授课口播切答疑口播
- **VIDEO SCO**：视频本身**不被暂停**（学员控制播放器）；VIDEO 没有授课口播，学员插嘴直接进入答疑口播

详细 3 子场景（静默暂停 / 插嘴提问 / 主动回顾跳转）见 § 1.5.3。

#### 1.4.5 lecture 卡片产出（高光 ★ + 任务 ✓）

lecture 在 Neo Chat 区可推送 2 类卡片（5 类卡片完整定义见 [02-foundation/02-methodology § 1.6] + [05-learner/06-cross-context § 4.4] 矩阵）。

**lecture 推送卡片清单**：

| 卡片 | 频率 | 触发 | 落地 |
|------|------|------|------|
| **高光卡片** | ★ 主场 | Neo 实时识别学员"顿悟瞬间" / "高光应用" | 实时推送 chat → 学员收藏 → 沉淀大厅"我的高光"区 |
| **任务卡片** | ✓ 辅助 | Neo 检测到学员可执行下一步行动（如学员说"我下周打算 X"）| 推送 chat → 学员加入待办 → 沉淀大厅待办清单 |

**lecture 不推送的 3 类卡片**（仅大厅推 / 详见 [05-learner/06-cross-context § 4.4] 矩阵）：

- **askUserQuestion 卡片**：不打断学习流推选项
- **课程卡片**：学员已在 Activity 内 / 不再推荐切换
- **知识卡片**：不打断学习流推工具卡

**高光识别详情（★ 主场）**：

- **实时识别**：Neo 实时感知学员"顿悟瞬间" / "高光应用"
- **实时推送**：识别后立即在 Chat 区推送高光卡片
- **学员收藏**：学员在卡片上点"收藏" → 进入大厅"我的高光"区（详见 [05-learner/06-cross-context § 4.4] + [05-learner/02-hub § 1.2.3]）；忽略不收藏的卡片不沉淀

> **高光触发示例**：学员说"原来 Pressure 是表面，Need 是深层" → Neo 立即判断为顿悟瞬间 → 在 Chat 区推"PIN 法 · 学员高光"卡片 → 学员点收藏 → 沉淀到大厅"我的高光"。

> **任务触发示例**：学员说"我下周打算和老李正式聊一次反馈" → Neo 推送任务卡 → 学员加入大厅待办清单。

> **本期不做笔记推荐** —— 笔记由学员手动通过笔记悬浮球添加（详见 [05-learner/02-hub § 1.3]）；Neo 只推送高光 + 任务卡片 / 不推笔记 / 不入笔记库。

### 1.5 10 项核心互动（串联 Agent 和课件区）

> **章节定位**：本节是 Agent 和课件区之间的交互机制——10 项交互按性质分组，让"哪些交互连接 Agent / 哪些连接课件区 / 哪些跨区联动"一目了然。

#### 1.5.1 输入方式

| 方式 | 交互 |
|------|------|
| **打字** | Chat 区输入框直接打字发送 |
| **语音** | Chat 区点击语音按钮 → 说话 → 语音转文字 → 点击发送 |

#### 1.5.2 10 项交互一览（按性质 4 类分组）

**🤖 Agent 触发（Neo 主动 / 2 项）**

| # | 交互 | 说明 |
|---|------|------|
| ① | SCO 间互动 | Neo 讲完知识点后**自主判断**；内容偏抽象理论时在 Chat 区插互动问题（结合学员场景）；不新增 SCO。**判断逻辑不够清晰时优先无互动**——本项优先级较低，宁缺勿滥 |
| ② | 主动出题 | 学员说"再来一道题" / "我想练习" → Neo 即时生成同知识点练习题；通过 ⑦ 随时打断的答疑口播实现，不新增 SCO |

**📚 课件区操作（学员 / Neo 在课件区交互 / 4 项）**

| # | 交互 | 说明 |
|---|------|------|
| ③ | SCO 锁定 | 未播放到的 SCO 锁定（灰 + 🔒）不可点击；已学完可回看 |
| ④ | Quiz 反馈 | 按 AOM 脚本规则（是否允许提示 / 是否透露答案 / 反馈深度）；学员说"不知道"时按规则给提示或引导 |
| ⑤ | Activity 间跳转 | **学员通过左侧抽屉 Tab1 Project 目录主动跳转**（不是 Activity 结束后的自动跳转）→ 弹窗"确定离开当前？进度已保存" → 确认跳转，触发断点续播保存 |
| ⑥ | 断点续播 | 中途退出 lecture（点"返回大厅" / 关浏览器）→ **弹窗确认 + 触发断点续播保存** → 系统记录当前 SCO 位置 + 对话上下文 + VIDEO 播放断点；下次进入同 Activity **自动从断点恢复**（含 VIDEO 上次播放位置） + Neo 简要回顾"上次学到 X，继续吗" |

**🔀 跨区联动（Chat ↔ 课件区 / 2 项）**

| # | 交互 | 说明 |
|---|------|------|
| ⑦ | 随时打断 | 学员主动让 Neo "停一下"——含 3 子场景：静默暂停 / 插嘴提问 / 主动回顾跳转（与 ⑧ 联动）；详见 § 1.5.3 |
| ⑧ | 主动回顾跳转（正式学习中） | **仅抽屉 Tab2 Activity 目录点击已学完 SCO**（课件区无左右箭头 / 左右箭头仅自由复习模式有） → Neo 停下当前口播询问"这一页还有什么不理解" → 进入答疑口播等待学员提问 → Neo 评估学员问题已得到解答 → 询问"是否跳回原进度" → 学员确认后跳回。**机制上属 ⑦ 随时打断的特殊子场景（带跳转）** |

**⚙️ 平台机制（系统级 / 2 项）**

| # | 交互 | 说明 |
|---|------|------|
| ⑨ | 防挂机 | 默认 5 分钟无操作（无键盘 / 鼠标 / 滚动）→ 系统弹提醒并暂停课程；暂停期间不计学习时长。**阈值可在 program config 调整**（详见 [06-management/04-program-config § 1] / 字段定义在 [06-management/04-program-config § 1] 阶段补）|
| ⑩ | 完成判定 | 所有 SCO 完成 = Activity 完成；Quiz SCO 完成标准 = 满足 passConditions（不要求答对，要求完成判定流程）。完成后弹窗 3 选项：回到大厅 / 自由复习 / 继续下一 Activity（详见 § 1.7.2） |

#### 1.5.3 ⑦ 随时打断 详细

学员可在任何时刻打断 Neo 授课口播，分 3 种子场景：

| 子场景 | 触发 | Neo 响应 | 恢复方式 |
|--------|------|---------|---------|
| **静默暂停** | 课件区点击"暂停"按钮 / 按空格键 | 立即停止授课口播；学员可静默看课件 / 翻 SCO 进度 | 学员手动点击"继续" → Neo 接续讲解（不重复已讲内容） |
| **插嘴提问** | Chat 区输入文字 / 语音消息 | 暂停授课口播 → 切换答疑口播 → 回应学员问题 | 答完自动恢复授课口播（不重复中间内容） |
| **主动回顾跳转**（带跳转的打断）| 学员点击抽屉 Tab2 Activity 目录 已学完 SCO | 停下当前口播 → 跳到该 SCO → 询问"这一页还有什么不理解" → 进入答疑口播等学员提问 | Neo 评估学员问题已解答 → 询问"是否跳回原进度" → 学员确认 → 跳回原进度恢复授课 |

**通用约束**：
- 静默暂停 / 插嘴提问期间**课件区中央覆盖**"已暂停 / 点击继续"（不在 Topbar，详见 § 1.2.4）；暂停时长不计入挂机判定
- 问题涉及未解锁内容时 Neo 告知"后面会讲到"+ 写入记忆
- VIDEO 阶段插嘴**不暂停视频本身**，只切答疑口播；视频是否暂停由学员自己用播放器控制（详见 § 1.4.4）
- ⑦ 主动出题（学员说"再来一道题"）通过本机制中的"答疑口播"实现
- 主动回顾跳转 单独列为 § 1.5.2 ⑧ 项，便于研发关注其特殊跳转流程

### 1.6 关键约束

**教学约束**（如何教 / 不能教 / 必须采集）：

#### 1.6.1 不给答案（分两种场景）

"不给答案"不是绝对——按场景区分：

| 场景 | Neo 行为 |
|------|---------|
| **AOM 规定的 Quiz 答题中** | 按 AOM 要求演绎；**未满足 passConditions 时仅根据 solution 引导**——不允许透露答案；学员答错或说"不知道"时用苏格拉底式追问帮助学员自己想出来 |
| **答疑口播过程中** | **优先引导学员主动思考**；Neo 自主判断是否直接给答案——简单的概念询问（如"PIN 法的 P 是什么"）可直接解答；涉及应用 / 思辨 / 判断类问题以引导追问为主 |

详见 § 2.7 lecture 场域级约束。

#### 1.6.2 不剧透

详见 § 2.7。Neo 不主动透露：

- 后续 SCO 的具体内容
- 后续 Course / Activity 的结论
- 学习路径终点 / 评估结果

学员问及未解锁内容时，Neo 回应"后面会讲到，先记下你的疑问"。

#### 1.6.3 闭环反馈强约束

每个 FEEDBACK_COLLECT SCO **必须**有对应 FEEDBACK_REVIEW SCO 配对（KGP 编排强制，缺失配对的 AOM 不通过审核）。

**运行约束**（系统行为限制）：

#### 1.6.4 SCO 锁定与已学解锁

| 状态 | 触发 | 学员可做 |
|------|------|---------|
| **未播到锁定** | 默认 | 灰 + 🔒 不可点击 |
| **当前学习中** | 当前播放 SCO | ▶ 高亮 |
| **已学完可回看** | 完成的 SCO | ✅ 可点击触发 ⑧ 主动回顾跳转 |
| **完成全解锁**（自由复习模式） | Activity 完成弹窗选"自由复习" 或 已完成的 Activity 再次进入 | 全部 SCO 解锁；**课件区左右箭头 + 抽屉 Tab2 双路切页**；Neo 进入答疑模式（不主动反应）|

#### 1.6.5 防挂机

**默认 5 分钟**无操作（无键盘 / 鼠标 / 滚动 / 语音 / 文字）触发，**阈值可在 program config 调整**（详见 [06-management/04-program-config § 1] / 字段定义在 [06-management/04-program-config § 1] 阶段补）：

- 系统弹提醒：要继续吗？
- 学员未响应 → 暂停课程，停止计时
- 学员确认继续 → 恢复授课

#### 1.6.6 数据持久化与隐私

- **保存频率**：推荐**每个 SCO 完成时落库**（学习进度 / Chat 上下文 / Quiz 答题状态 / VIDEO 播放断点）；具体实现由研发决定
- **隐私边界**：沿用 § 3.4.3 隐私三层（学员之间默认完全隔离 / 跨学员引用需同意 / 管理端例外）；学员在 FEEDBACK_COLLECT 自述的真实场景默认仅 Neo 可读 + 写入学员个人记忆，不跨学员共享
- **后续演进方向**：**敏感信息安全管控**作为整体功能后期立项（识别个人 / 商业敏感信息 + 脱敏 + 学员授权机制）—— 本期不实现

### 1.7 完成与跨场域衔接

#### 1.7.1 Activity 完成判定

- 所有 SCO 完成（按各 SCO 类型自身完成判据）
- Quiz SCO 满足 passConditions（不要求答对，要求完成判定流程）

#### 1.7.2 完成 Modal

```
┌─────────────────────────────┐
│                        [X]  │
│         ✅                    │
│    Activity 完成！             │
│  你已完成「PIN 法三步走」       │
│                              │
│  [回到大厅]                   │
│  [自由复习]                   │  ← v0.5 新增
│  [继续下一 Activity]            │  ← 主 CTA
└─────────────────────────────┘
```

按钮行为（**3 选项**，文案待定）：

| 按钮（顺序） | 视觉 | 跳转目标 |
|------|------|---------|
| 回到大厅 | 次要边框 | 大厅（Neo 接收 Activity 完成状态） |
| **自由复习**（v0.5 新增） | 次要边框 | 进入本 Activity 自由复习模式（全 SCO 解锁，Neo 答疑模式，详见 § 1.7.3） |
| 继续下一 Activity | 主 CTA（cyan→violet 渐变） | AOM 下一节点（practice / 下一 Activity） |

> **变更**：
> - 取消 v0.4 设想的"重新学习"按钮（自由复习已能覆盖大部分回看场景；如需完整重学可在自由复习模式下手动从第一个 SCO 开始）
> - 文案最终措辞待 GUI 设计阶段确认

#### 1.7.3 自由复习模式

**进入路径**（两种）：

1. Activity 完成弹窗（§ 1.7.2）选"自由复习"
2. 已完成的 Activity 在大厅 / 抽屉再次进入时**默认即自由复习模式**

**模式行为**：

- 所有 SCO 全解锁
- **切页方式**（**双路**）：
  - **课件区左右箭头**（自由复习模式专属 / 正式学习中无）
  - 抽屉 Tab2 Activity 目录 点击 SCO 跳转
- **Neo 不反应**：切页时**不打断 / 不询问 / 不主动辅导**；仅当学员主动提问时被动答疑
- 不主动巡查（如不会主动说"你上次这题错了，要不要重看"）

#### 1.7.4 跨场域衔接

依赖 [05-learner/06-cross-context § 4] 跨场域机制。lecture 与外部场域的衔接：

| 衔接 | 数据流 |
|------|--------|
| **lecture → practice** | lecture 采集场景 / 隐性打标 / 学员行为模式 → practice 作为 hint / 剧本冷启动；具体机制详见 [05-learner/06-cross-context § 4.2] |
| **lecture → recap** | lecture 全程行为数据 → recap 反思素材；详见 [05-learner/06-cross-context § 4.2] |
| **大厅 → lecture** | 学员从大厅"进入"→ 自动恢复断点续播（如有）；详见 § 1.5.2 ⑥ 断点续播 |
| **lecture → 大厅** | 学员中途退出或 Activity 完成 → 返回大厅，Neo 接收完成 / 进度状态 |
