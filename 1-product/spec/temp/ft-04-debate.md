## 角色一：学员张三

> 视角：体验流畅性、认知负荷、情感安全

### 对练模块

| 严重度 | 节点 | 问题 | 说明 |
|--------|------|------|------|
| **高** | A2.5 | 对练结束的学员感知缺失 | 功能树只写"系统提示对练结束"，但学员视角需要知道：结束提示长什么样？是突然切断对话还是有过渡？AI角色最后一句话是什么？突然中断会打破沉浸感 |
| **高** | A3.2 | 反思阶段的心理安全未保障 | 苏格拉底追问可能让表现不佳的学员感到被"审判"。功能树缺少"先肯定再反思"的正向反馈机制，或至少应在A3.1身份切换时有一个情感过渡（如"你刚才的表现中有几个很好的点…"） |
| **中** | A1.1 | 三组信息卡片的阅读负荷 | 背景+目标+技能三张卡片信息量大，学员是否可以在演绎阶段随时回看？功能树未提及"导入信息在演绎阶段是否可回看" |
| **中** | C1.2 | 断点续播的选择焦虑 | "从上次位置继续"或"重新练习"——学员不知道上次到了哪里，选择缺乏依据。应提示"你上次完成了第X轮对话" |
| **低** | A2.2 | 对话轮数不确定性 | "约10-15-20轮"——学员不知道还要聊多久，长时间对练可能产生疲劳。是否需要进度暗示？ |

### 测评模块

| 严重度 | 问题 | 说明 |
|--------|------|------|
| **高** | F1.1 BEI中"被追问"的压迫感 | 功能树写了"同一案例最多追问3轮"和"回避行为处理"，但学员视角是"AI一直在追问我"。缺少追问语气的柔化机制——功能树层面应有"追问语气由剧本控制"的节点 |
| **中** | G2 评阶段的期望管理 | 评阶段是"可选"的，学员怎么知道有没有评？如果期待评估结果却没有，会失望；如果不期待却突然出现，会意外 |

# Round 2：修改建议

基于R1走查结果，按优先级给出具体修改建议。

## 修改建议 1（严重）：统一阶段数，A4降级为"后处理"

**问题：** 功能树定义"四阶段"，需求文档明确"三阶段"，AOM样本中excludeReport=1暗示报告非必选。

**建议：**
- 将功能树A节标题改为"三阶段流程 + 后处理"
- A4从"阶段四"降级为"对练后处理（可选）"，明确它不是流程中的一个阶段，而是完成后的数据产出环节
- 树结构调整为：

```
对练流程
├── A1 阶段一：情景导入
├── A2 阶段二：Roleplay演绎
├── A3 阶段三：反思复盘
└── A-POST 后处理（可选）
    ├── A-POST.1 excludeReport机制
    └── A-POST.2 能力评估碎片生产
```

## 修改建议 2（严重）：明确量表测评阶段右侧对话区的内容

**问题：** 量表测评的核心交互在中间看板（GUI），但功能树写"所有测评类型右侧承载AI对话"，量表阶段右侧内容空白。

**建议：**
- 在I3下区分两种场景：

```
I3 右侧对话区
├── I3.1 对话式测评
│   └── I3.1.1 AI测评对话 — 测评全程 — 右侧承载AI访谈对话
├── I3.2 量表测评-测阶段
│   └── I3.2.1 引导说明 — 量表作答中 — 右侧展示AI简要引导（如"请根据你的第一反应选择"），可选空状态
└── I3.3 量表测评-评阶段
    └── I3.3.1 AI解读对话 — 量表完成后 — 右侧承载AI对量表结果的解读和讨论
```

## 修改建议 3（严重）：新增"已知限制"占位节点

**问题：** 需求文档明确列出"本期不做"的功能（学员测试AI底线、实时教练介入等），功能树完全没有体现。

**建议：**
- 在对练模块末尾新增 Z 节"已知限制与本期不做"：

```
Z. 已知限制（本期不做）
├── Z1 学员恶意测试AI底线 — 不做主动防御，依赖剧本的情绪天花板自然兜底
├── Z2 对练实时教练介入 — 引导职能放在反思阶段
├── Z3 多人模拟 — 本期仅1v1
└── Z4 角色互换/费曼/辩论 — 未来通过不同剧本实现
```

## 修改建议 4（高）：补充对练结束的过渡体验

**问题：** A2.5只写"系统提示对练结束"，缺少从演绎到复盘的过渡体验。

**建议：**
- A2.5下新增过渡节点：

```
A2.5 对练结束
├── A2.5.1 剧本控制结束 — 达到结束条件 — 系统提示"对练结束"
├── A2.5.2 AI角色结束语 — 结束提示后 — AI角色说出最后一句符合角色的结束语（由剧本定义），而非突然静默
└── A2.5.3 阶段过渡提示 — 结束语后 — 系统展示过渡提示"接下来进入复盘环节"，给学员心理准备
```

## 修改建议 5（高）：补充反思阶段的正向开场

**问题：** A3直接进入苏格拉底追问，缺少心理安全缓冲。

**建议：**
- 在A3.2前插入正向反馈节点：

```
A3.2 反思开场
├── A3.2.0 正向反馈先行 — 进入反思 — AI教练先肯定学员做得好的1-2个点（由导演在演绎阶段标记），再进入追问
├── A3.2.1 引导反思 — 正向反馈后 — 苏格拉底式追问
└── A3.2.2 对照成功标准 — 反思对话中 — 对照剧本成功标准
```

## 修改建议 7（高）：剧本控制元素与AOM结构的映射注释

**问题：** A2.3的6项控制元素在AOM JSON中找不到对应字段。

**建议：**
- 在A2.3后新增说明性注释（非功能节点）：

```
注：A2.3中的控制元素在AOM中的映射关系：
- 角色心理画像 → roleplay SCO内的characterProfile字段（待KGP定义）
- 行为模式 → roleplay SCO内的behaviorChain字段（待KGP定义）
- 防剧透指令 → roleplay SCO内的constraints字段（待KGP定义）
- 成功标准 → roleplay SCO内的successCriteria字段（待KGP定义）
- 技能阶梯 → 复用competencyGoals.rubric（已有）
- 情绪天花板 → roleplay SCO内的emotionCeiling字段（待KGP定义）
```

同时明确：技能阶梯(A2.3.5)与AOM中rubric是同一数据源的两种表述，功能树中应统一为"复用KGP rubric"。

## 修改建议 8（高）：回避行为规则应由剧本控制

**问题：** F1.1.4写死"连续2次模糊→切换案例"，与ADR-004"平台不硬编码"矛盾。

**建议：**
- 修改F1.1.4：

```
F1.1.4 回避行为处理 — 学员连续模糊/转移话题 — 由访谈剧本定义回避检测阈值和处理策略（如"2次后切换"或"3次后切换"），平台不硬编码次数
```

与H3.1保持一致：学员拒绝分享的处理方式由脚本控制。

## 修改建议 9（高）：对话回看组件补充交互规格

**问题：** A3.3的回看组件缺少关键交互细节。

**建议：**
- 补充A3.3子节点：

```
A3.3 对话回看组件（中间看板）
├── A3.3.1 回看面板展示 — 进入反思阶段 — 按轮次展示完整对话，标注角色名
├── A3.3.2 点击定向分析 — 学员点击某轮 — 右侧Agent自动分析该轮
```
