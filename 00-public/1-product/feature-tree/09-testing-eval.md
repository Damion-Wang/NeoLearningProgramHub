## 一句话定位

**双重能力模块** —
- **9-A Evaluation**：产品内运行时评估引擎（打标 / 碎片 / 评分 / 成长幅度），跑在项目里
- **9-B Simulation**：独立于项目代码之外的 UX 测试 Agent，黑盒模拟真人体验

两者架构性解耦，仅在 team owner 层面归并。

## 功能结构

### 【9-A】Evaluation — 产品内评估引擎

```
A. 打标引擎（ASSESSMENT_TAG）[ADR-008]
   A1 后台执行（学员无感知）
   A2 信号汇总（前序互动 SCO / Quiz 回答 / 对话 / FEEDBACK_COLLECT）
   A3 概率模型输出标签
   A4 五字段 schema
       A4.1 tag_dimension（标签维度）
       A4.2 assigned_tag（判定标签）
       A4.3 confidence（置信度 0-1）
       A4.4 evidence_summary（判定依据摘要）
       A4.5 effective_until（有效截止位置）
   A5 影响范围限 Activity 内（标签严格不跨 Activity）
   A6 低置信度降级（<0.6 使用默认 Segment）
   A7 多打标点支持（Activity 内多段自适应）
   A8 ★AI 双轨运行约束 [req §7.6，与 Lecture B6 对称]
       A8.1 打标耗时不影响前台教学（异步执行 + 默认版本兜底）
       A8.2 采集信号缓存机制（失败重试，不回传学员）
       A8.3 打标结果写入 DB 后不直接通知前台（由 Lecture 下一轮调度时读取）

B. 碎片生产规则
   B1 学习特质碎片（Activity 级汇总，从"每 SCO 一个"优化为"每 Activity 一个"）
   B2 认知转化碎片（Quiz 答题闭环后生产）
   B3 能力评估碎片（对练完成后生产）
   B4 被动信号碎片（会话级汇总停留 / 回翻 / 点击等）
   B5 BEI 案例碎片（调研 BEI 完成后生产）
   B6 碎片 schema 治理（与 Infra N4 协同）
   B7 ★学习时长定义（横切概念权威归属）[补漏]
       B7.1 字段定义（active_time 活跃时长 / total_time 总在线 / session_gap 断点间隔）
       B7.2 计算口径（累计活跃时长，不含挂机 / 长停顿 / tab 失焦）
       B7.3 采集源（Infra N session 数据 + 各场域 dwell_time 被动信号）
       B7.4 异常处理（离线重连 / tab 切换 / 挂机 5min 后截断）
       B7.5 消费点（Hall E1 / Report F / Operation C4，见 manifest §8 横切映射）

C. 碎片权重与裁决
   C1 五级权重（Quiz > 对练 > 测评 > 辅导 > 被动信号）
   C2 评分加权规则（具体比例由内容包配置）
   C3 证据排序（高权重来源优先展示）
   C4 矛盾裁决（学员端简化综合结论 / Agent 追问时展示矛盾细节）

D. 成长幅度计算
   D1 BEI 基线优先（课前 BEI vs 课后评分）
   D2 备选方法（前 1/3 均值 vs 后 1/3 均值 + z-score 标准化）
   D3 最小样本要求（6 个数据点以上）
   D4 语义标签输出（显著 / 中等 / 轻微，避免伪精确数字）

E. 行为改变证据链
   E1 三段汇聚（初始表现 / 学习过程 / 最终表现）
   E2 按能力维度组织
   E3 全模块证据整合（授课 + 对练 + 调研 + 辅导）

F. BEI 评分
   F1 STAR 完整性评分
   F2 能力维度打分（基于 KGP 定义的 rubric）
   F3 完整度评分（案例不完整本身就是信息）

G. 分位计算
   G1 项目内排名（30-40 人为基准）
   G2 分位区间输出（前 25% / 中 50% / 后 25%）
   G3 样本不足降级（<20 人不展示分位区间）
   G4 API 级隔离控制（学员端不返回排名字段）

H. 质控与可解释性
   H1 打标结果留痕（evidence_summary 必填）
   H2 碎片原始记录可追溯
   H3 技术调试后门（基础版，不做复杂四层架构）
   H4 AI 质量数据不流向学员端 / HR 端
   H5 ★后台查找溯源链 [req §7.5]
       H5.1 按学员 / 按 Activity / 按碎片 ID 检索
       H5.2 逐层钻取（结论 → 证据摘要 → 原始对话 / Quiz 答题 / 被动信号）
       H5.3 调试交互（查看 prompt / 模型响应 / 打标中间结果）
       H5.4 权限隔离（仅研发 + 指定运营可访问，学员 / HR 不可见）
       H5.5 操作审计（查询记录留痕）
```

### 【9-B】Simulation — 独立 UX 测试 Agent

```
I. Agent 本体（独立部署）
   I1 基础 LLM + 行为规划能力
   I2 ★零项目代码依赖（不 import 任何模块、不调内部 API）
   I3 独立账号（以真用户身份登录被测环境）
   I4 人格驱动（从人设反推行为倾向 / 知识背景 / 耐心阈值）

J. 输入契约
   J1 ★人设卡（必填） — 画像 + 角色 + 耐心 / IT 熟练度 / 行业背景
   J2 用户手册（可选） — 不提供时 Agent 按真新用户"没人教过"自探索
   J3 任务目标（可选） — 例如"走完 Course 1 全流程"或"不给目标自由逛"
   J4 约束条件 — 不许查代码 / 遇到卡点像真人一样求助或放弃

K. UI 交互层
   K1 浏览器驱动（Playwright / Puppeteer，纯黑盒）
   K2 视觉理解（截图 + DOM 识别控件）
   K3 基础操作（点击 / 输入 / 滚动 / 等待）
   K4 与 Neo / Actor 对话时像真人回复（有人设风格 / 不暴露 AI 身份）
   K5 错误 / 卡顿 / 超时的感知（不容忍、不重试到成功）

L. 体验感知
   L1 流畅度（加载等待 / 过渡动画 / 响应速度）
   L2 路径认知（"我想 X 但找不到入口" / "点错几次才对"）
   L3 文案理解（"这个按钮叫 Segment 我不知道是什么"）
   L4 情绪模拟（烦躁 / 好奇 / 满意 / 困惑，基于人设阈值触发）
   L5 截图留证（关键步骤 + 问题点）

M. 反馈输出
   M1 试用感受报告（自然语言，分段：第一印象 / 核心任务 / 整体评价）
   M2 卡点清单（按严重度：阻断 / 不便 / 小瑕疵）
   M3 改进建议（像真用户说"如果这里有返回按钮就好了"）
   M4 量化指标（任务完成用时 / 出错次数 / 求助次数 / 放弃节点）
   M5 截图附件包

N. 任务管理
   N1 任务创建（选人设 + 可选目标 + 可选手册）
   N2 批量 persona 并发（多画像同时跑）
   N3 进度监控
   N4 结果汇总（N 个 Agent 跨人设的共识卡点）

O. 数据隔离
   O1 独立被测环境（不污染生产 / 真实学员数据）
   O2 Agent 账号可识别（报告侧可过滤）
   O3 用完即毁 / 可回滚

P. 典型使用场景
   P1 发版前自动冒烟（核心旅程走一遍）
   P2 新功能上线前 UX 验证
   P3 竞品对比（同一 Agent + 同一人设跑两个产品）
   P4 客户演示前预检（demo 脚本各环节通畅）
   P5 KGP 新内容包体验验证（Simulation 作为第一批"学员"）
```

## 与 v0.3.3 差异

- 0.3.3 无独立 Evaluation 功能树；打标 / 碎片 / 成长幅度散在 03-teaching / 04-drill-assessment / 05-report 各处
- 0.3.3 无 Simulation 概念
- v0.4.0 把两者合并为"测试评估"模块，统一 owner，解决跨模块评估逻辑难以治理的问题
