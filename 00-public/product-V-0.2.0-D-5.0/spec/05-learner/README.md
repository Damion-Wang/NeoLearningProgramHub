## 推荐阅读顺序

### 完整顺序

1. **01-overview** —— 学员端全景（旅程图 + 模块清单 + 入口路径）
2. **02-hub** —— 大厅（学员的"主页" / 一切交互的起点）
3. **06-cross-context** —— 跨场域机制（笔记 + 4 库 + 5 卡片矩阵 / 这是横切的）
4. **03-lecture** / **04-practice** / **05-recap** —— 3 学习场域逐个深入

### 快速路径

- 看 Neo 在大厅做什么 → `02-hub` § 1.2 Neo 辅导
- 看 lecture 的 SCO 类型 → `03-lecture` § 1.3 课件区内容模型
- 看 practice 的三角色协作 → `04-practice` § 1.2 业务定义 + § 1.3 形态
- 看 recap 的 Activity 完成事件 → `05-recap` § 1.1.3 + § 1.2 报告
- 看 5 卡片矩阵 → `06-cross-context` § 4.4

- **02-foundation/02-methodology**：教学方法论（5 大理论 / 7 动作 / 4 维感知 / 6 维画像）—— Neo 在 3 场域的所有教学行为基础

- **06-management/03-report-center** Tab 2：管理端学员报告库（学员侧产生的报告进入管理端 Tab 2）

## 重要约定

- **大厅（中枢层 / 不算场域）**：明确边界 / 大厅是 Neo 与学员的常驻陪伴层 / 不属于学习场域之列
- **3 学习场域**：lecture / practice / recap 是当前 scope 的所有学习场域 / 未来场域（inquiry / assignment / sandbox）列未来扩展
- **5 类卡片**：askUserQuestion / 课程卡片 / 高光卡片 / 任务卡片 / 知识卡片（统一带"卡片"后缀 / askUserQuestion 例外）
- **4 库**：我的报告 / 我的高光 / 我的工具 / 我的笔记（议题 4 决议）
