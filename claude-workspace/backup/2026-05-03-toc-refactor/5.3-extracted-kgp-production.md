### 5.3 怎么生产 · KGP 内容生产规则

#### 5.3.1 KGP 团队的角色

KGP（Knowledge Generation Platform / 内容生产平台）团队**不是写 JSON 的**——他们是**用自然语言为 AI 编写演绎脚本**。

类比：传统培训设计师写一份讲师手册（"先问 X 问题，学员答 Y 时这样回应，答 Z 时那样回应"）；KGP 做的是同样的事，但产物是 AI 能直接执行的 AOM 内容包。

#### 5.3.2 KGP 生产的内容产物 · 按 5 层结构组织

KGP 生产的内容产物按照 AOM 5 层结构归位——每一层有明确的产物清单：

**5.3.2.1 Project 层产物（项目级）**

- 项目背景 / 学员画像假设 / 能力目标体系 / 行业热词
- **大厅辅导知识体系** ★（关键产物，未来可插拔）：
  - 领域知识工具库（基层管理者主题：冰山模型 / GROW 教练法 / RACI 矩阵 / SMART 目标 / ...）
  - 辅导场景库（学员可能遇到的工作场景 + 对应辅导策略）
  - 案例库（按能力维度组织的真实场景案例）
  - 关键词 → 工具 / 场景 / 案例 的检索映射

> **未来可插拔愿景**：不同 Project（基层管理 vs 销售 vs 客服）使用完全不同的知识体系——大厅辅导的 Neo 切换 Project 即切换"领域大脑"，平台无需为每个领域定制代码。

**5.3.2.2 Module 层产物（课程主题级）**

- Module 主题 / 关键概念图 / 教学目标
- Module 级辅导原则（用于小结时 Neo 引导反思的角度）

**5.3.2.3 Activity 层产物（按 Activity 类型差异化）**

不同 activityType 需要的内容产物**不同**：

| Activity 类型 | KGP 需要生产的产物 |
|--------------|-------------------|
| **lecture**（授课）| 流程脚本（SCO 序列）+ 答疑风格规则 + 前端演示事件（高亮 / 动画 / 进度提示）|
| **practice**（对练）| 前情设定 + 对练剧本（Actor 人设 / 对白库 / 触发条件 / 预设走向）+ 复盘追问原则 |
| **recap**（小结）| 内容回顾框架 + 表现复盘维度 + 应用计划引导话术 |

**5.3.2.4 SCO 层产物（按 SCO 类型差异化）**

不同 SCO 类型需要的内容产物**不同**：

| SCO 类型 | KGP 需要生产的产物 |
|---------|-------------------|
| **SLICE**（PPT 讲解）| PPT 图片素材 + Neo 讲解话术 |
| **VIDEO**（动画视频）| 场景背景 + 角色对白 + 音频字幕 + 动画帧 |
| **QUIZ**（测验）| 题目库 + 通关条件（自然语言规则）+ 答错引导话术 |
| **FEEDBACK**（采集 / 复盘）| 采集问题 + 复盘追问框架 |

**5.3.2.5 SCO 内的 AI 演绎指令（每个 SCO 都有，跨类型）**

不论 SCO 类型，每个 SCO 都需要 KGP 写以下 3 类自然语言指令：

| 指令类型 | 给 AI 解释什么 | 例 |
|---------|---------------|-----|
| **教学设计意图** | "**为什么**这么设计这个环节" | "通过'轻负担'对话获取特征，实现'初阶个性化'课程推送" |
| **AI 行为准则** | "**具体怎么做**"（对话策略 / 引导步骤）| "第一步探场景 → 第二步挖动机 → 第三步看风格；保持好奇、不评判" |
| **收尾话术规范** | "通关 / 失败时**怎么收尾**" | "满足通关条件时按 X 结构生成；触发兜底时按 Y 结构生成" |

**5.3.2.6 asset 层产物（同 SCO 多版本时）**

当 SCO 需要按学员标签切换 asset 版本时（如 § 5.5.1 PIN 法 SCO 5 的 3 版本案例视频）：

- 不同 separateTag 的 asset 内容（如 3 种学员标签的不同案例视频）
- 每个 asset 的元数据（适用条件 / 标签 / 切换规则）
