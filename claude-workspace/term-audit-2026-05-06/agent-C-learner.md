## 5 · 备注 / 不确定项

### 5.1 「5 场域」用法澄清

glossary § 4 标记"~~5 场域~~ 旧口径 / 不再使用 / 含未来 inquiry/assignment"。但 spec 06-cross-context § 9.4.4.2 用了 **"5 场域 × 5 卡片矩阵"** —— 这里的"5 场域" = 大厅 + lecture + practice + recap = 4 个，但叫"5 场域 × 5 卡片"。

> 实际看 06-cross-context:158 的矩阵表头是 4 行（大厅 / lecture / practice / recap），章节名称却是"§ 9.4.4 Neo 卡片产出（5 类 × 5 场域）"。

**判断**：这里"5 场域"指的应该是 **5 类卡片** × **5 卡片矩阵**（同一字段名）的笔误，**或**"5 场域"含未来场域留口（§ 9.4.4.2 备注："**未来学习场域**：本期不实现 / 未列入矩阵"）。建议 T1.4 全量审查时确认：

- 选项 A：把 § 9.4.4 章节名改为"4 场域 × 5 卡片矩阵"或"大厅 + 3 学习场域 × 5 卡片矩阵"
- 选项 B：保留"5 场域"语义 = 大厅 + 3 学习 + 未来场域留口（已在备注里说明），但需要 glossary 校准 / 加一节"5 场域含未来场域"

我倾向选项 A —— 当前矩阵实际只有 4 行，文字表述不应预留未来。

### 5.2 「Course 完成事件」与「Activity 完成事件」的命名歧义

glossary 决议：取消"Course 完成事件"，统一为"Activity 完成事件"。但 spec 05-recap:59 解释 "**特指 recap Activity 自身完成**——即首聊态首次完整 Walk-through 触发的一次性事件 / **区别于 lecture / practice Activity 的常规完成**"。

**注意**：glossary 表述 "特指 recap Activity 自身完成" 可能让人误以为它是"recap Activity 完成 = 任何 recap Activity 完成"，但 spec 实际严格限定为 "**完整态首次** + 走完 Walk-through + 实质性反思 + 应用计划产出"四件齐全。

**建议**：glossary 加 inline 注释 "recap Activity 完整态首次进入 + 完整 Walk-through 走完时一次性触发 / 不是任意 recap Activity 完成"。

### 5.3 「自由复习」三处统一概念的边界

spec 05-recap:787 明示 "自由复习"统一概念跨 lecture / practice / recap 三处指向同一概念："Activity 流程结束之后，学员允许回到该场域针对 Activity 内容和已经发生过的事情和 Neo 进行探讨"。

但 practice 完成 Modal 里写的是"**自由探讨**"（04-practice:775），lecture / recap 完成 Modal 里写的是"**自由复习**"（03-lecture:441 / 05-recap:780）。

**判断**：这是有意区分（practice 用"自由探讨"是因为 practice 不存在"复习"概念，是探讨已演练的对话）—— 但 spec 同时声明"统一概念"。

**建议**：glossary 加新增条目"自由复习 / 自由探讨"或在 § 14 状态机制里加一行 "lecture/recap 用'自由复习' / practice 用'自由探讨' / 三者指向同一概念（详见 § 9.3.3.9.6）"。

### 5.4 「报告」相关多重命名
