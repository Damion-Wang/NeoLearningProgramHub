## 1 · 三轮主要产出

### 轮 1 · 项目全生命周期 → **8 阶段重写**

#### 核心新增（DM 引入二元模型）
**服务周期 vs 项目周期**：
- 服务周期：学员账号能正常使用平台的总时长（最长 1 年）
- 项目周期：服务周期内的子周期（典型 6 个月，可延期）
- **项目结束后到服务周期结束之间**：学员**正常使用平台**（不是只读延续期）
- 服务周期到期 → 账号关闭

#### 8 阶段
1. 商机识别 → 2. 客户意愿确认（含 KGP 内容包 review）→ 3. 签单（不做订阅）→ 4. 开营组织（含 Pilot 子阶段）→ 5. 学习期（项目周期主体，可延期）→ 6. 结营 → 7. **服务期延续**（学员正常使用）→ 8. 服务截至（续约/退出/归档）

#### 6 议题共识
- 周期分离 → 服务/项目二元
- 试用期 → "Pilot"作为开营子阶段（5-15 人 / 2-4 周）
- 结项后状态 → 服务期内正常使用（推翻 S-12 原决议）
- 7 阶段 → 不增阶段，子动作扩
- CSM → 70% Ora 自动化降级为"仅看板"
- RACI → 5 角色简化表通过

### 轮 2 · GUI spec → **6 模块深化**

#### 核心新增
- **服务/项目周期 UI 表达**：学员端温度（项目第 X 周 / 项目已完成）+ 管理端标准术语
- **学员端实诚不糖衣化**（不包装成"复习期"）
- **GUI 设计原则 5 条**

#### 5 议题共识
1. 服务/项目周期 UI → 学员端温度 + 管理端标准
2. **4 库 GUI ⭐**：4 tab 水平排列 + 数量显示（**推翻 0424 "2×2 网格"**）
3. 7 题型 GUI → 5 种本期 + 连线/排序看 KGP 数据 + drag+click 双轨
4. 对练逃生通道 → 1 次 Actor 提示 + 2 次 Neo 询问切讲授
5. Ora 看板 → 颜色+排序 OK / 自动日报砍

### 轮 3 · Agent BDD → **5 大维度框架**

#### 核心新增（**根据 DM 反馈大调整**）
- **不要 PM 拍脑袋具体 BDD 故事**
- **要 behavior 设计维度框架**（给研发可扩展资产）
- 5 大维度 × 子分类 × 关键场景 × 关键约束

#### 5 大维度

##### 维度 1 · Memory 调用
- 子分类：L1-L5 渐进画像
- 关键场景：跨场域引用 / 跨课程关联 / 周期提醒 / 学员主动问 / Neo 主动推送
- 关键约束：L1-L2 加轻量限定 / L3-L5 加"我感觉" / 大厅每轮 ≤ 1 次 / voice 朴素不机械

##### 维度 2 · Voice 边界
- 子分类：不当舔狗 / 真实优先 / 场域差异化 / 情绪感知
- 关键场景：表扬具体 / 不接受表面同意 / 接情绪 / 不假定防御
- 关键约束：表扬具体 / 不贴心理标签 / 不假装真人 / 不假装记得

##### 维度 3 · 教学策略 ⭐ DM 修正
**按场景类型轴**（不是按 W 阶段）：
- 大厅辅导：可给答案 / 引出轻量思考
- 授课：不给答案 / 引导方向
- 对练：不给答案 / 引导方向
- 调研：不剧透 / 接纳采集
- 报告：解读 + 反问

##### 维度 4 · 卡点处理 + 翻车修复
- 子分类：学员卡点 / Neo 翻车 / 超纲拒答
- 关键场景：对练逃生通道 / Neo 记错修正 / 超纲拒答
- 关键约束：翻车 + 修复是 Neo 特点 / 拒答必说明"未来能学"

##### 维度 5 · 多 Agent 协同
- 子分类：Neo/Ora 共享 / Neo/Actor 边界 / 导演/Actor / Neo/学员
- 关键约束：4 Agent 各自独立 / Memory 共享但人格独立 / 学员对话不出现"导演"

作为 5 维度举例（**不是 PRD final**），研发参考。实际 PRD 需 50-100 个故事，DM 后续主导生产。
