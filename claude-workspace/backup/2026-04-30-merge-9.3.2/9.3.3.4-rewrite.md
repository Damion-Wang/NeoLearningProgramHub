#### 9.3.3.4 Course 个人报告 · 数据与生成（★ 核心节）

> **定位原则**：报告是 Neo 引导对话的"道具" —— 写到 70%，留 30% 给 Neo Chat 展开。报告**内容维度**记录学员的"学过什么 + 学得怎么样 + 接下来怎么用"，不承担项目进度类信息（详见 § 9.3.3.1.2 边界）。

##### 9.3.3.4.0 报告全景图

**5 块结构总览**：

| # | 块 | 一句话内容 | 形态 | 数据来源（大类）| 可变性 |
|---|----|----------|------|--------------|------|
| ① | 内容锚点 | 学员目标 + 本 Course 主题 + 两者关联 | 纯 markdown | 开营反馈采集 + Course AOM 元数据 | 每次刷新 |
| ② | 走过的轨迹 | 学员在不同关联 Activity 的互动行为画像 | markdown + 1 张图（互动行为对比柱图）| 全场域日志（互动次数 + token 消耗）| 每次刷新 |
| ③ | 6 维画像 · 本 Course 变化点 | Neo 在关联 Activity 内对学员 6 维度的改变 + 关键证据 | markdown + 条件性 1 张图（仅高亮变化维度）| Neo 跨 Activity 写入的画像变化数据（后台累积）| 每次刷新 |
| ④ | 关键证据 | 高光 + 卡点（具体行为片段引用）| 纯 markdown 引用块 | 关联 Activity 内：高光库 + practice 复盘待改进点 + Neo 跨场域感知卡点模式 | 每次刷新 |
| ⑤ | Neo 留言 | 师长口吻的"班主任谈话" | 纯 markdown 引用块 | 全数据 + 跨场域记忆（不限作用域）| 每次刷新 |

**报告全长 / 阅读时长**：500-800 字 + 1-2 张图 / 阅读时长 2-3 分钟。

**总览原则**：

- **图表准入条件（三条全过才放图）**：① 对学员有内容反思价值 ② 范式稳定（可套模板）③ 数据来源清晰
- **状态自适应**：每块在"空白 / 部分 / 完整 / 重练"4 状态下的呈现规则在该块专节内说明（不再做集中矩阵表）
- **每次更新整体覆盖**：报告 5 块每次增量更新都重新生成（与"应用计划仅首聊态生成不变"是两套逻辑）—— 报告反映**当下时刻** Neo 对学员的判断
- **不保留历史快照**：每次更新覆盖旧版本

##### 9.3.3.4.1 ① 内容锚点

| 维度 | 说明 |
|------|------|
| **是什么** | 一段 30-50 字的 markdown：把学员开营时说过的目标 + 本 Course 主题 + 两者关联用一句话表达 |
| **目的** | 让学员开篇一秒回到"我为什么学这门课"的语境——对内容（不是对项目时间表）的关注 |
| **形态** | 纯 markdown，无图表 |
| **数据来源** | 开营反馈采集（学员目标，需 KGP 在开营 Activity 设结构化字段，PM 决策点 D4）+ Course AOM 元数据（Course 主题）|
| **4 状态呈现** | 4 状态下都显示（数据来源是开营 + AOM，不依赖前序 Activity 完成度）|
| **可变性** | 每次报告更新都重新生成——内容稳定但 Neo 写法可能微调 |

**示例**（学员 / 主题 / 关联均为占位，KGP 内容生产侧产物）：

> 你开营时说想"学会带跨部门项目"。本 Course 是《冲突管理》——这门课跟你的目标的关系是：跨部门带项目里最大的难点不是分工不是预算，而是不同部门的目标和顾虑会撞车。这门课就是练你怎么不躲、不退，把撞车解决掉。

**降级**（学员开营无目标数据时）：

> 本 Course 是《Course 主题》。我们一起看看你这门课走得怎么样。

##### 9.3.3.4.2 ② 走过的轨迹

| 维度 | 说明 |
|------|------|
| **是什么** | 学员在本 Course 关联 Activity 中的**学习行为画像**——哪些 Activity 互动最多、哪些最少、整体投入度（token 消耗作为投入度代理）|
| **目的** | 让学员看到自己在内容上的**投入分布**——在哪里花了时间、在哪里走过场。**这是内容反思的素材**（"为什么我在 Activity A 问了 30 次，在 Activity B 一句话不问？"），不是项目进度仪表盘 |
| **形态** | markdown 一段（50-100 字解读）+ **1 张图**：每个关联 Activity 的"学员发送消息数 + Neo 回应轮次 + token 消耗（投入度代理）"对比柱图 |
| **数据来源** | 全场域日志（每 Activity 学员发送消息数 + Neo 回应轮次 + token 累计）—— 全部直接可用 |
| **4 状态呈现** | 空白态：不显示这块（学员没数据可看）/ 部分态：仅显示已完成的 Activity 数据 + 标注"仅含已完成的 X/Y" / 完整态：完整对比图 / 重练态：含某 practice 多次重练的累计 |
| **可变性** | 每次报告更新都重新生成 |

**图表设计原则**：
- 关联 Activity ≥ 2 个时显示柱图；单一 Activity 时退化为一句话描述
- 不显示完成度 / 不显示百分比 / 不显示项目时间（这些是项目进度，不是行为画像）

**markdown 解读要点**：
- 指出投入差异最大的 1-2 个对比（"你在 Activity A 互动最多 / 在 Activity B 几乎没问问题"）
- 不预设"哪种好哪种差"——这是行为事实陈述，让学员自己反思

##### 9.3.3.4.3 ③ 6 维画像 · 本 Course 变化点（★ 灵魂）

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 在本 Course 关联 Activity 过程中对学员 6 维度（详见 § 9.2.1 B.2.B）的**判断改变**——哪几维改变了、改变方向、关键证据指向哪个 Activity 哪一刻 |
| **目的** | 让学员看到 Neo 在本 Course 中**为什么改变了对自己的判断**——这是学员看不到的视角，是 1v1 老师的真功夫。**不展示**全部 6 维当前快照（避免与项目初期对比的歧义），**只展示发生了变化的维度** |
| **形态** | markdown 解读 + **条件性 1 张图**（仅高亮变化维度的精简横向进度条 + 改变方向箭头 + 改变前后值 + 触发改变的 Activity 标注）—— **本 Course 内无变化时不出图**，只用一句话说"本 Course 中 Neo 对你的画像没有显著调整" |
| **数据来源** | Neo 在 lecture / practice 中**主动写入**的画像改变记录（后台数据，不暴露为前端日志）。具体存储 / 持久化机制由研发实现层自决（产品 spec 不规定数据形态）—— 产品层**只描述呈现内容**：维度 / 改变方向 / 改变前后值 / 触发改变的 Activity + 关键证据片段 |
| **4 状态呈现** | 空白态：不显示 / 部分态：基于已完成 Activity 内的改变点（可能为空，写"目前没看到明显变化"）/ 完整态：所有变化点清单 / 重练态：含重练带来的变化叠加 |
| **可变性** | 每次报告更新都重新生成（基于后台累积到当前时刻的变化点）|

**与 § 9.2.1 6 维画像的衔接**：

- 6 维画像本身（持续感知 / 当前值）属于 § 9.2.1 B.2.B 已有定义
- recap ③ 块依赖一个**事件层数据**：Neo 在场域内观察到学员某维度有实质性变化时主动写入"改变点"记录（含维度 / 方向 / 前后值 / 触发证据）
- 该事件数据的产生时机由 § 9.3.1 lecture / § 9.3.2 practice 内 Neo 行为承担——具体技术实现层是研发自决（**不强制立"变化点日志"作为产品功能**）
- recap 报告 ③ 块只**读取并呈现**这部分数据，不规定写入机制

**图表设计原则**：
- 仅高亮变化维度（如本 Course 改变了 2 维 → 显示 2 个进度条；其他 4 维不显示）
- 每条进度条标注：维度名 + 改变方向（↑ / ↓）+ 前→后值 + 触发改变的 Activity 标识
- 无变化时不出图——避免硬塞

**markdown 解读要点**：
- 用一两句话引出每个变化（"我注意到你在 X Activity 之后，Y 维度上来了——你的 Z 行为是关键"）
- 不替学员解读为什么——留白让 Neo Chat 引导

##### 9.3.3.4.4 ④ 关键证据

| 维度 | 说明 |
|------|------|
| **是什么** | 把抽象洞察落到**具体行为片段**——高光 1-2 条（学员表现亮眼的具体瞬间）+ 卡点 1-2 条（学员反复栽的地方 / 模式）|
| **目的** | 让学员**认得出**自己——洞察必须基于学员能识别的具体场景，不是抽象总结 |
| **形态** | 纯 markdown 引用块（每条 = 标题 + 引用片段 + Neo 一句话留白）—— **无图表**，因为是叙事性证据不是统计 |
| **数据来源** | **严格限定关联 Activity 内**（"显性证据"作用域）。高光 = 关联 Activity 内高光库；卡点 = 关联 practice 复盘的待改进点 + Neo 跨场域感知到的反复出现的卡点模式。**不依赖 Quiz 错题**（Quiz 不一定有对错，详见 § 9.3.1.3.4）|
| **4 状态呈现** | 空白态：不显示 / 部分态：基于已完成 Activity 内的证据；数量不足时**宁可空着也不补关联外** / 完整态：完整 / 重练态：含跨次对比的卡点（"你这个卡点在演练 1 / 2 都出现了"）|
| **可变性** | 每次报告更新都重新生成（增加新高光 / 新卡点）|

**「我以为 vs 实际」偏差子块** —— 本期不做（V2），原因：依赖倒逼 KGP 在所有 lecture FEEDBACK_COLLECT 设结构化"自评"字段，本期 KGP 内容侧工作量过大；待 V2 视诉求强度回头评估。

**呈现示例**（占位）：

> **🌟 高光｜在 Activity A 你做对了一件事**
>
> > 你在演练第 N 轮主动追问对方的真实顾虑，没有被表面的拒绝挡住。这是这门课希望你练出来的关键动作。
>
> （Neo 不替学员解读"你是怎么想到的"——留 Chat 里展开）

> **⚠️ 卡点｜你反复栽在同一个地方**
>
> > 在 Activity B 第 X 轮 / Activity C 第 Y 轮，你都是先认同对方再说自己的观点，但最后**忘记把自己的立场说清楚**。
>
> （Neo 留白：这个模式是不是熟悉的？）

##### 9.3.3.4.5 ⑤ Neo 留言

| 维度 | 说明 |
|------|------|
| **是什么** | 一段 80-150 字的师长口吻"班主任谈话"——表达观察 / 担心 / 期待 |
| **目的** | 不是评分总结，是**关系性话语** —— 为后续 Neo Chat 对话提供锚点（学员读完想接话） |
| **形态** | 纯 markdown 引用块——无图表 |
| **数据来源** | "隐性背景"——基于全数据 + 跨场域记忆 + 学员开营目标的 Neo 综合判断；**不限定数据来源**（含大厅 / 跨 Course）|
| **4 状态呈现** | 空白态：轻量留言"我们等你迈出第一步" / 部分态：基于现有数据的轻量留言 / 完整态：完整班主任谈话 / 重练态：含跨次对比的留言（"你重练了 N 次，我看到 X 在变化"）|
| **可变性** | 每次报告更新都重新生成——Neo 留言反映"当下时刻 Neo 对学员的判断"，数据变了判断也该变 |

**呈现示例**（占位）：

> **Neo 想跟你说**
>
> > 我注意到你开营时说想"学会带跨部门项目"，但这门课里你 3 次演练都先妥协再补救。我担心你不是不会沟通——是不敢冲突。下次我们聊聊这事儿好吗。

**生成原则**：
- 师长口吻——"我注意到 / 我担心 / 我希望"，不用"建议你 / 应该你"
- 引用学员**自己说过的话**或**自己做过的事**（跨场域记忆）
- 不替学员说总结——表达观察 + 留问题
- 一次性生成（每次报告更新覆盖），不可改

##### 9.3.3.4.6 生成 / 更新 / 降级 通用规则

| 维度 | 规则 |
|------|------|
| **触发** | 每次"前序 Activity 完成"事件触发后台增量重新生成 |
| **缓存** | 学员进入 recap 时直接看最新缓存 |
| **不保留历史快照** | 每次更新覆盖旧版本；学员不能"看上次的版本" |
| **学员侧零等待** | 后台静默完成，学员无感 |
| **不提供"刷新"按钮** | 自动更新即可，按钮反而暴露机制 |
| **修改性** | 学员侧**不可修改**（整体一次性生成） |
| **导出性** | 场域内不可导出（沿用 § 9.3.5.6 跨场域统一规则）；大厅"我的报告"主题可导出 |
| **图表准入** | 三条全过才放图：① 对学员有内容反思价值 ② 范式稳定（可套模板） ③ 数据来源清晰。本期最多 2 张（② 互动行为对比 + ③ 6 维画像变化条件性）|
| **数据降级** | 数据不足时：不硬塞 / 不编造 / 写占位说明（保护"老师真实"感）。每块在专节内有降级规则 |
| **报告 diff（后台）** | 每次报告更新时记录 vs 上版的变化点（哪些数据是新的 / 哪些块有变化）—— 给 Neo 判断回访态用 + Subbar 红点提示用。学员侧**不需看到详细 diff** |
