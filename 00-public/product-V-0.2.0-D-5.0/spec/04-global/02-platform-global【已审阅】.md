## 1 · 平台级全局策略

> 系统在哪些条件下运行——支持范围、合规、安全、数据策略 等平台级约束。

### 1.1 设备与浏览器支持

##### 本期产品形态

> **本期产品 = web 端**（浏览器跑），不专做 PC 客户端 / 移动 App / 平板 App / PWA。

##### 浏览器支持

- **Chrome / Edge / Safari 最近 2 个版本**
- Firefox / Opera 不官方支持但不拒（不主动调试）
- 不支持 IE / 老 Safari

##### 设备适配

- 手机浏览器打开 PC 页面：**不限制 / 不调整**（本期不做手机适配）
- 平板（iPad）：同手机 / 不专门做体验

##### 性能与加载

- **性能目标 / 大文件加载策略 交研发**——产品仅要求"加载体验不脱节"
- **弱网 / 断网**：**自动重连 / 进度不丢 / 不支持离线学习**

### 1.2 国际化策略

##### 语言策略

- **本期 UI / AOM 纯中文（简体）**
- **预留多语言架构**（i18n 能力）—— 未来仅扩展英文 / 繁体中文 / 日语，不考虑小语种
- 客户企业的英文姓名 / 英文邮箱：**原始显示**（admin 填什么展什么）

##### 时区策略

- **服务端统一 UTC+8（中国时区）**
- 项目周期 / 里程碑 / 脱训判定 都按 UTC+8

##### 格式策略

- **日期 / 时间使用 ISO 8601**（如 `2026-04-28 14:30`）—— 国际化格式

##### 货币 / 计费

- **本期不涉及**——客户付款走合同 / 商品平台，**不进产品**

### 1.3 安全与合规底线

##### 大模型安全声明 ★

| 项 | 决策 |
|---|------|
| **文案** | **"以上内容由 NeoLearning AI 生成，仅供参考"** |
| **位置** | **所有有 AI 生成内容的地方都要有**（国内 AIGC 规范要求）—— 学员端 Topbar 下、Ora 对话区、team report 生成区 等 |

##### powered by NeoLearning

- **登录页有就行** / 其他位置不强制

##### 服务条款 / 隐私同意

- **由 GPB 负责** / 本期不实现具体内容
- 设计上**预留勾选动作**（首次登录弹窗）—— 勾选什么内容待定

##### 学员数据用途 ★

- **不用于训练模型**
- **可能用于调教 Agent prompt**——**仅限 NeoLearning** / 不外发不外用

##### Agent 安全（已锁定）

详见 § 3.4.3。

##### Agent 数据隔离（已锁定）

详见 § 3.1.3 数据空间架构 3 层。

##### 跨租户隔离

- **不同客户数据完全隔离**
- **本期产品侧不要求实现租户概念**——每个项目就是一个独立数据包
- 未来再考虑租户概念

##### 内容审核

- 学员对话**敏感信息（公司机密 / 个人隐私）不检测 / 学员自负 / 产品不介入**
- 录音 / 录屏 本期不做

##### 管理端权限

- **本期 admin / HR / 项目运营 同权限** —— 仅 admin 与其他区分（admin 创建管理员 + 锁定服务周期 等高权限动作）
- HR / 项目运营 仅可见**本项目学员**（不能跨项目）

##### 合规适用

- **仅遵中国个人信息保护法 / AIGC 规定**
- GDPR 等本期不考虑

##### 交研发的安全项

| 项 | 本期处理 |
|----|---------|
| API 限流 / 防爬 | 交研发 |
| 备份 / 灾备 | 交研发 / 运维 |
| 账号防滥用 | 交研发 |
| 第三方 API 合规 | 交研发 |
| 数据加密策略 | 交研发 |
| 操作审计日志 | **本期不做** |

### 1.4 数据交付与隔离

##### 学员侧数据可见性

- **学员之间完全看不到对方** —— 不知道班级有 N 名同学 / 看不到名单
- 社区功能本期 constructing（作为未来钩子，吸引未来扩展）

##### 学员私有数据可见性

- **管理员可见学员全部数据**（笔记 / 报告 / 反思 / 对话 等）
- 这是**产品设计核心场景**（详见 § 3.4.3 / 50 题 Q42）

##### 管理员报告库可见性

- **每个管理员独立报告库 + 独立草稿副本**
- **完全私有**——admin 也不能看其他管理员的草稿
- 协作靠**导出后线下同步**

##### 服务到期前提醒

- 服务到期前 1 个月：**站内消息提醒管理员**
- 邮件等其他渠道：靠**线下实现**（产品本期不集成邮件）

##### 数据导出

| 项 | 本期 | 备注 |
|----|------|------|
| **team report 导出** | ✅ PDF / Word | 详见 [06-management/03-report-center § 1.3.5] |
| **学员 4 库下载**（我的报告 / 我的高光 / 我的工具 / 我的笔记）| ✅ 内容下载或批量下载 | 学员个人数据自主下载范围 |
| 能力曲线 Excel / 项目运营记录 / 原始对话 等 | ❌ 本期不做 | 后台拉数据生成 / 下一版逐步加 |

> 导出包详细内容（含 / 不含清单）+ 服务到期后产营导出流程 详见 [03-lifecycle/03-completion § 3.1]。

##### 数据生命周期 ★（global 4 个关键点）

产品仅锁定 **4 个关键点**：

1. **项目隔离**：每个项目独立数据包 / 跨项目数据不复用
2. **服务周期内不清理**：消息 / 学习记录 / 报告 等服务期内全部保留
3. **到期前 1 月提醒**：自动推送给管理员
4. **归档永久原始保留**：项目归档后原始学习记录永久保留（用于产品优化 / Agent prompt 调教 仅限 NeoLearning）

##### 数据销毁

- 客户主动要求"立即销毁"：**靠产营后台手动 + 出具销毁证明**
- 产品**不提供"一键销毁"按钮**

##### 应急访问

- 客户出问题：**产营后台直接访问**
- 产品**不设"应急按钮"**

##### 中途换运营

- 取决于**接管账号 vs 重新注册账号**——账号之间是隔离的
- **接管账号** = 看到原账号所有报告 / 草稿
- **重新注册** = 完全隔离 / 看不到原账号内容
- 选择由 admin 决定

##### 跨项目数据复用

- **本期不做** / 同客户不同项目数据完全隔离
- 未来扩展看需求
