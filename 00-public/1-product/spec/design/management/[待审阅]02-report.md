# 管理端报告页

## 模块定位

报告页是 HR/运营查看、生成、定制培训报告的空间——从个人学习报告到项目整体报告到 CEO 汇报报告，全部通过 Agent 辅助完成。

---

## 功能区域

| 区域 | 内容 |
|------|------|
| **左侧：目录区** | 报告类型导航（个人报告/项目报告/汇报报告/报告工厂） |
| **中间：报告展示** | 报告内容、排名表格、图表、数据详情 |
| **右侧：Agent 对话** | 报告查询、报告定制生成、数据解读、数据追溯 |

---

## 用户故事：HR 王经理准备月度汇报

### 场景背景
王经理是某企业的 HR 总监，负责"基层管理者培养项目"。本月底需要向 CEO 汇报培训进展。今天是月底前三天。

### 完整旅程

**10:00 — 查看项目整体报告**

王经理登录 HR 端，点击左侧「项目报告」。中间看板展示项目整体数据：
- 整体完成率：35%（上月 18%，本月 +17%）
- 各 Course 完成率分布柱状图：Course 1（72%）、Course 2（45%）、Course 3（22%）、Course 4-8（未开始）
- 平均学习时长：每人 8.5 小时/月
- 本月新完成 Activity 数：总计 186 个

**10:10 — 查看排名数据**

王经理点击「进度排名」Tab：

| 排名 | 学员 | 已完成 Activity | 总 Activity | 完成率 | 最后学习时间 |
|------|------|----------------|------------|--------|------------|
| 1 | 张磊 | 12 | 32 | 38% | 今天 08:30 |
| 2 | 陈思 | 11 | 32 | 34% | 昨天 20:15 |
| 3 | 周明 | 10 | 32 | 31% | 昨天 14:00 |
| ... | ... | ... | ... | ... | ... |
| 30 | 王芳 | 2 | 32 | 6% | 12天前 |

排名表支持按各列排序和筛选。

王经理切换到「互动排名」Tab——基于互动 token 量排序：

| 排名 | 学员 | 互动 Token 量 | 对话轮次 |
|------|------|-------------|---------|
| 1 | 李华 | 45,200 | 328 |
| 2 | 张磊 | 42,100 | 305 |
| ... | ... | ... | ... |

旁边有说明："互动排名基于学员在授课、对练、测评、辅导中产生的互动 token 总量，反映学员的参与深度。"

**10:20 — 查看个人报告**

王经理点击学员"张磊"的名字 → 进入张磊的个人学习报告（与张磊自己看到的同一份报告）。

与学员端不同的是，HR 可以看到：
- 具体排名（进度排名第 1，互动排名第 2）
- 分组标签（"高活跃"——学员端不可见此标签）

**10:30 — 用 Agent 生成汇报报告**

王经理在右侧对话区说：
> "帮我准备一份给 CEO 的月度汇报报告，重点突出培训进展和效果。"

Agent 确认：
> "好的，我来为你生成月度汇报报告。几个问题确认一下：
> 1. 重点维度：整体进度 + 学员参与度 + 典型案例，对吗？
> 2. 需要特别提及哪些学员的表现吗？
> 3. 上次你用的是简洁风格（3-5 页），这次也一样吗？"

王经理："对，重点说进度和参与度。提一下张磊和陈思的优秀表现。简洁风格就行。"

Agent 生成初稿（约 30 秒）→ 在中间看板展示报告预览：
- 第一页：项目概览（整体完成率、活跃学员数、本月亮点数据）
- 第二页：学习进度详情（各 Course 完成率、趋势图）
- 第三页：学员典型案例（张磊的对练进步、陈思的测评表现）
- 第四页：需关注事项（王芳等 3 名低活跃学员）
- 第五页：下月展望

王经理预览后标注修改："第四页不要写具体姓名，写'有 3 名学员需要重点关注'就行。"

Agent 修改后 → 王经理确认 → 下载 PDF。

**10:40 — CEO 追问数据**

下午 CEO 看了报告后追问："你说张磊进步最大，依据是什么？"

王经理重新登录系统，问 Agent：
> "张磊进步最大的具体证据是什么？"

Agent 回答：
> "张磊在 PIN 法对练中的表现变化是最直接的证据：
> - 第一次对练：花了 5 轮才开始探索对方的 Interest
> - 第二次对练（重练）：第 1 轮就主动追问深层需求
> - 对练评分：从第一次 65 分提升到第二次 88 分
> - 课后复盘中，他用 PIN 法重新分析了自己的真实工作场景（市场部张总借人事件），给出了具体可行的替代方案
>
> 数据截至 2026-05-30 08:00"

王经理点击"查看原始对话" → 系统展示对练的原始对话记录。

## 核心交互清单

### 1. 项目整体报告

| 项目 | 内容 |
|------|------|
| **触发** | HR/运营点击「项目报告」 |
| **响应** | 中间看板展示整体完成率、各 Course 完成率分布、活跃度趋势、聚合统计 |
| **联动** | 所有图表可点击触发 Agent 解读 |

### 2. 进度排名

| 项目 | 内容 |
|------|------|
| **触发** | 点击「进度排名」Tab |
| **响应** | 展示所有学员的进度排名表（已完成 Activity 数/总数、完成率、最后学习时间） |
| **联动** | 可排序（按完成率/最后学习时间等）、可筛选 |

### 3. 互动排名

| 项目 | 内容 |
|------|------|
| **触发** | 点击「互动排名」Tab |
| **响应** | 展示所有学员的互动排名表（互动 token 量、对话轮次） |
| **联动** | 界面展示互动排名计算说明（基于互动 token 量） |

### 4. 汇报报告生成

| 项目 | 内容 |
|------|------|
| **触发** | HR 向 Agent 描述报告需求 |
| **响应** | Agent 确认模板/重点/风格（记忆 HR 历史偏好）→ 生成初稿 → 预览 → 修改 → 确认 → 下载 PDF |
| **联动** | 报告中每个数据都可追溯到原始记录 |

### 5. 数据追溯

| 项目 | 内容 |
|------|------|
| **触发** | HR 追问报告中某个数据的来源 |
| **响应** | Agent 给出数据来源和计算逻辑，支持钻取到原始对话记录 |
| **联动** | 逐层钻取：结论 → 证据摘要 → 原始对话 |

### 6. 需关注学员展示

| 项目 | 内容 |
|------|------|
| **触发** | HR 打开看板 |
| **响应** | 显著位置展示进度最低 3 人（含姓名、进度百分比、最后登录时间） |
| **联动** | 点击可查看个人详情 |

## 已确认决策

- HR 端排名不看能力——先实现进度排名 + 互动排名
- 不做能力维度排名（样本量小、评分标准化未完成）
- 学员端/HR 端排名数据 API 级隔离（学员端不返回排名字段）
- HR 可见分组标签（高活跃/稳健/需关注），学员不可见
- 本期仅生成报告格式，不生成 PPT
- 报告模板：项目整体报告、个人学习报告、HR 汇报报告
- Agent 回答标注数据更新时间
- Agent 无法回答时坦诚告知
