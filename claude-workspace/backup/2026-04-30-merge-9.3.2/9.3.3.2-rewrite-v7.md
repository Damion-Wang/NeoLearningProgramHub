#### 9.3.3.2 Course 个人报告（核心交付物）

> **作用**：报告是 Neo 引导对话的"道具"——写到 70%，留 30% 给 Neo Chat 展开。报告内容承载 recap 的三层价值（知识回顾 + 表现回顾 + 下一步建议）。

##### 9.3.3.2.0 报告全景

**首聊层 5 块结构**（milestone 触发瞬间锁定，永久保留）：

| # | 块 | 承载层面 | 内容 | 形态 | 数据来源 |
|---|----|--------|----|------|---------|
| ① | **内容锚点** | 知识层面回顾 | Course 主题 + 关键概念 / 框架 + 学员在场域内提出的真实场景串联 + 学员表现一句概括 | 纯 markdown | Course AOM 元数据 + 关联 Activity 概览 + 学员场景描述（来自 lecture FEEDBACK_COLLECT 自由文本 / Neo Chat 对话 / practice 演练上下文）|
| ② | **走过的轨迹** | 表现层面回顾 | 学员在不同关联 Activity 的互动行为画像 | markdown + 1 张图 | 全场域日志：每关联 Activity 的全类型互动总数（含 Quiz 答题 / practice 中栏 / Neo Chat）|
| ③ | **6 维画像** | 表现层面回顾 | 有变化点 → 变化点列表 + 嵌触发证据 / 无变化 → 6 维当前快照 | markdown + 条件性 1 张图 | Neo 在关联 Activity 内记录的画像变化数据（详见 § 9.2.1 + § 9.3.1.4.6 / § 9.3.2.4.6）+ 6 维当前值 |
| ④ | **关键证据** | 表现层面回顾 | 高光 1-2 条 + 卡点 1-2 条（具体行为片段引用）| 纯 markdown 引用块 | 关联 Activity 内：高光库子集 + practice 复盘待改进点 + Neo 跨场域感知卡点模式 |
| ⑤ | **待办任务** | 下一步建议 | Neo 产出的 1-3 条具体行动建议（应用计划），列在报告里同时自动入大厅待办清单 | 纯 markdown 列表 | Neo 基于报告全数据 + 跨场域记忆 + 学员场景的高度个性化生成 |

**报告全长**：首聊层 700-1000 字 + 嵌入图表 1-2 张 / 学员阅读时长 2-3 分钟 / 完整解读体验（含 Neo Walk-through 对话）8-10 分钟。

##### 9.3.3.2.1 ① 内容锚点（首聊层）

| 维度 | 说明 |
|------|------|
| **是什么** | 一段 80-150 字的 markdown：Course 主题 + 关键概念 / 框架 + 学员在场域内提出的真实场景串联 + 学员表现一句概括 |
| **目的** | 让学员一秒回到"这门课在讲什么 + 我跟它的连接"——核心是**知识层面回顾**，结合学员自己的场景做内化锚点 |
| **形态** | 纯 markdown，无图 |
| **数据来源** | Course AOM 元数据（主题 / 关键概念）+ 关联 Activity 列表 + 学员在场域内主动提出的场景描述（由 Neo 跨场域记忆调取）|
| **状态自适应** | 空白态：仅 Course 主题一句 / 部分态：Course 主题 + 已学概念 + 已提场景串联 / 完整态：完整知识回顾 + 学员场景串联 + 表现一句话 |
| **可变性** | milestone 前每次刷新；milestone 触发时锁定为首聊层；之后**永久不变**（学员场景已在首聊时整合） |

**示例骨架**（占位）：

> 这门课讲的是 **{Course 主题}**，关键概念是 **{概念 1 / 概念 2 / 概念 3}**。你在学习中提到过 **{学员真实场景 1，如"我跟同事在 KPI 拆解上有分歧"}**，这正好是本课要解决的问题——{概念 N} 在这种场景下能 {落地用法}。从 {Activity X / Activity Y} 走过来，你整体上 {表现节奏一句概括}。

##### 9.3.3.2.2 ② 走过的轨迹（首聊层）

| 维度 | 说明 |
|------|------|
| **是什么** | 学员在本 Course 关联 Activity 中的**学习行为画像**——哪些 Activity 互动最多 / 哪些最少 |
| **目的** | 让学员看到自己在内容上的**投入分布**——在哪里花了时间 / 在哪里走过场。这是**内容反思素材**，不是项目进度仪表盘 |
| **形态** | markdown 一段（50-100 字解读）+ 图 1（互动行为对比柱图，详见 § 9.3.3.2.0 图表清单）|
| **数据来源** | 全场域日志：每关联 Activity 内学员的全类型互动数（不区分类型，避免跨类型比较失真）。**不记 token 消耗**（跨类型 Activity token 差异巨大会误导）|
| **状态自适应** | 空白态：不显示 / 部分态：仅显示已完成 Activity 的对比 + 标注"仅含 X/Y" / 完整态：完整对比图 |
| **可变性** | milestone 前每次刷新；milestone 触发时锁定为首聊层 |

**markdown 解读要点**：
- 指出投入差异最大的 1-2 个对比（"你在 Activity A 互动最多 / 在 Activity B 几乎没问问题"）
- 不预设"哪种好哪种差"——这是行为事实陈述，让学员自己反思

##### 9.3.3.2.3 ③ 6 维画像（首聊层）

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 对学员 6 维度（详见 § 9.2.1 B.2.B）的判断：**有变化点 → 显示变化点列表 + 嵌触发证据**；**无变化点 → 显示 6 维当前快照** |
| **目的** | 让学员看到 Neo 在本 Course 中**对自己的判断**——这是学员看不到的视角，是 1v1 老师的真功夫 |
| **形态** | markdown 解读 + 图 2（6 维画像图，变化点 / 快照二选一，详见 § 9.3.3.2.0 图表清单）|
| **数据来源** | Neo 在 lecture / practice 关联 Activity 内观察到学员某维度有实质变化时记录的"画像变化数据"（详见 § 9.2.1 + § 9.3.1.4.6 / § 9.3.2.4.6）+ 6 维当前值 |
| **状态自适应** | 空白态：不显示 / 部分态：基于已完成 Activity 的变化点（可能为空 → 显示当前快照）/ 完整态：变化点列表 / 当前快照 二选一 |
| **可变性** | milestone 前每次刷新；milestone 触发时锁定为首聊层 |

**变化点呈现规则（每个变化点自含触发证据）**：

```
> Y 维度（如"主动追问能力"）↑ 3→4
> 证据：在 Activity {X} 第 {N} 轮，你 {具体行为片段}——这是这门课希望你练出来的关键动作
```

每个变化点都嵌入触发证据（具体到 Activity / 对话片段），让学员看到"凭什么这么说"。

**无变化点的兜底**：

```
> 本 Course 中 Neo 对你的画像没有触发实质变化。当前 6 维快照：
> [6 维当前值进度条]
```

让学员看到"我现在的状态"，避免"白学了"的失望感。

##### 9.3.3.2.4 ④ 关键证据（首聊层）

| 维度 | 说明 |
|------|------|
| **是什么** | 把抽象洞察落到**具体行为片段**——高光 1-2 条 + 卡点 1-2 条 |
| **目的** | 让学员**认得出**自己——洞察必须基于学员能识别的具体场景 |
| **形态** | 纯 markdown 引用块（每条 = 标题 + 引用片段 + Neo 一句话留白）—— 无图 |
| **数据来源** | **严格限定关联 Activity 内**。高光 = 关联 Activity 内高光库子集（高光卡片是 Neo 在各场域的通用能力，详见 § 9.2.1，recap 取与本次关联范围相符的子集）；卡点 = 关联 practice 复盘的待改进点 + Neo 跨场域感知到的反复出现的卡点模式。**不依赖 Quiz 错题**（Quiz 不一定有对错） |
| **状态自适应** | 空白态：不显示 / 部分态：基于已完成 Activity 内的证据；数量不足时**宁可空着也不补关联外** / 完整态：完整高光 + 卡点 |
| **可变性** | milestone 前每次刷新；milestone 触发时锁定为首聊层 |

**与大厅高光区的关系（透明性对齐）**：

- 高光卡片本身是 Neo 在各场域（lecture / practice / 大厅）的通用能力产物（详见 § 9.2.1）
- recap ④ 块呈现的高光是**大厅高光区的子集** —— 取关联 Activity 范围内的高光
- 学员在大厅高光区能看到完整集合，recap 仅按本次关联范围筛选

**呈现示例**（占位）：

> **🌟 高光｜在 Activity A 你做对了一件事**
>
> > 你在第 N 轮主动追问对方的真实顾虑，没有被表面的拒绝挡住。这是这门课希望你练出来的关键动作。

> **⚠️ 卡点｜你反复栽在同一个地方**
>
> > 在 Activity B 第 X 轮 / Activity C 第 Y 轮，你都是先认同对方再说自己的观点，但最后**忘记把自己的立场说清楚**。

##### 9.3.3.2.5 ⑤ 待办任务（首聊层）

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 产出的 **1-3 条具体行动建议（应用计划）**——直接列在报告里同时自动入大厅待办清单 |
| **目的** | recap 的**下一步建议**层落地——把反思转化为可执行的小任务，闭环到学员真实工作场景 |
| **形态** | 纯 markdown 列表（每条 = 1 句简洁可执行的小任务）—— 无图 |
| **数据来源** | Neo 基于报告 ①-④ 全数据 + 跨场域记忆 + 学员场景的**高度个性化生成**——同 Course 不同学员产出的待办任务差异明显 |
| **生成时机** | **仅在首聊态走完 Walk-through 后生成一次**；后续 milestone 不再变 |
| **流向** | 同步加入大厅"待办清单"（沿用 § 9.2.2.1 (1) E + § 9.2.1 C.3.4 大厅待办清单机制）；后续追踪 / 学员勾选 / Neo 跟进 全部在大厅待办流程内 |
| **学员参与** | 不参与撰写、不需确认——Neo 直接产出，直接入待办 |
| **可变性** | milestone 触发瞬间生成 + 锁定首聊层；**永久不变**（即使后续报告增量更新、自由复习态再进入也不再生成新待办）|

**呈现示例**（占位）：

```markdown
### ⑤ 待办任务

基于这门课，我建议你接下来一周做这几件事——已加入大厅待办清单：

1. {具体行动 1}（如"下周一跟 X 部门同事的会上，试一次先表达自己立场再求共识"）
2. {具体行动 2}
3. {具体行动 3}

→ [跳转大厅待办查看]
```

**后续追踪**（与大厅协同，详见 § 9.3.3.8.4）：

- 学员可在大厅待办主动点掉
- 学员点掉时 Neo 跟上来问情况
- Neo 评估后再决定是否允许点掉
- **完成不反映到 recap 报告**（闭环在大厅）

##### 9.3.3.2.6 自由复习层（独立区块，持续追加）

> **作用**：milestone 触发后，自由复习态产生的所有新内容统一收录在本区块——独立于首聊层 5 块结构，**不打散到 5 块里**，**不断变化追加**。

**位置**：在报告末尾（首聊层 ①②③④⑤ 之后），用 markdown 标题 `## 自由复习追加` 作分界。

**内容**：

| 内容类型 | 说明 |
|---------|------|
| Neo 在自由复习态对话中产生的**新观察 / 新留言** | 学员重进 recap 跟 Neo 聊天时，Neo 主动表达的观察 / 担心 / 期待 |
| **新画像变化点** | milestone 后 Neo 在 lecture / practice 内新写入的画像变化（如学员重练 practice 后某维度有变化）|
| **新高光 / 卡点** | milestone 后产生的新关联 Activity 内高光 + practice 重练带来的新卡点 |
| **新关联 Activity 总结**（如有）| KGP 后期追加新关联 Activity 时，Neo 对该新 Activity 内学员表现的简短回顾 |

**呈现形态**：

- 自由复习层是**连续的追加内容流**，不分块（区别于首聊层的 5 块结构）
- markdown 形态自由——可含小标题 / 引用块 / 列表 / 图表（如有新画像变化点可附图）
- **按时间顺序倒序排列**：最新追加的在最上，旧追加的在下

**合并规则**：

- 多次自由复习态进入产生的内容可合并——**第 N 次自由复习的内容可合并 / 覆盖第 N-1 次**，保持复习层始终是当下最新综合
- 但首聊层永久不变

**学员侧呈现**：

```markdown
## 首聊回顾（永久保留）

### ① 内容锚点
...

### ② 走过的轨迹
...

### ③ 6 维画像
...

### ④ 关键证据
...

### ⑤ 待办任务
...

---

## 自由复习追加（持续变化）

### {最新追加日期}
- Neo 新观察：...
- 新画像变化点：...
- 新高光：...

### {上次追加日期}
...
```

**Neo Chat 历史**：在 Activity 内不被清空——学员可翻 Chat 历史回看跟 Neo 聊过的具体对话内容（不在自由复习层重复呈现）。

**笔记复用**：学员"想留住某段感动"通过笔记功能记录（笔记悬浮球，沿用 § 9.2.3）。

##### 9.3.3.2.7 生成 / 更新 通用规则

| 维度 | 规则 |
|------|------|
| **触发（milestone 前）** | 关联 Activity 完成事件触发后台增量重新生成；新版本覆盖旧版本（不留旧快照）|
| **触发（milestone 触发瞬间）** | 当下报告内容固化为"首聊层"，永久独立保留 |
| **触发（milestone 后）** | 新数据进入"自由复习层"独立区块（详见 § 9.3.3.2.6）；多次自由复习可合并复习层；首聊层永远不变 |
| **缓存** | 学员进入 recap 时直接看最新缓存（首聊层 + 自由复习层叠加视图）|
| **学员侧零等待** | 后台静默完成，学员无感 |
| **不提供"刷新"按钮** | 自动更新即可，按钮反而暴露机制 |
| **修改性** | 学员侧**不可修改**（整体一次性生成）|
| **导出性** | 场域内不可导出（沿用 § 9.3.5 跨场域统一规则）；大厅"我的报告"主题可导出 |
| **图表准入** | 三条全过才放图：① 对学员有内容反思价值 ② 范式稳定（可套模板）③ 数据来源清晰。本期 ≤ 2 张（图 1 互动行为对比 / 图 2 6 维画像）|
