## 第 0 部分 · 顶层判断器与协作风格

### 0.1 设计判断器（不是 Neo Soul）
**遇事不决，参照 1v1 真人线下老师**——所有设计冲突回到此问题：「真人 1v1 老师会怎么做？」
- 例：学员要不要"被遗忘权"→ 真人老师不能选择性遗忘 → 不提供
- 例：学员能不能选 Neo 风格 → 真人老师人格固定，学员适应老师 → 不提供

### 0.2 PM 协作风格（4 项锁定）
1. **PM 角度 + 不要问技术实现**——技术选型 / 架构 / 验证 由研发决定
2. **遇事不决参照 1v1 真人老师**——设计冲突总判断器
3. **Soul 按场域差异化**——不是单一 Soul，按场域调用不同 skill 与倾向
4. **不决定时间，研发能力决定**——不预设周数 / 节点

## 第 1 部分 · Meta 三议题（最深的 3 条）

### Meta-1 · Neo 在「老师 / 教练 / 治疗师」光谱定位
**最终决策**：**根据 field 特性或 hall 调用相应 skill 和倾向**——不是单一身份选择
- 启发：调研发现"老师"（Khanmigo/directive）/ "教练"（GROW/semi-directive）/ "治疗师"（Rogers/non-directive）三流派立场冲突
- 但 Neo 不需要二选一，**按场域差异化**：大厅可偏教练（陪伴/引导）、授课偏老师（directive 推进）、调研偏治疗师（接纳采集）
- 这是底层架构原则，而非单一立场选择

### Meta-2 · Soul 格式：工业标准 vs 自研
**最终决策**：**采用 aaronjmars/soul.md 5 文件结构**
- 5 文件：`SOUL.md`（身份）/ `STYLE.md`（voice）/ `SKILL.md`（操作）/ `MEMORY.md`（连续性）/ `soul.json`（manifest）+ `examples/`
- 节省 6-8 周自研时间
- 已是工业级开源标准（aaronjmars/soul.md + SoulSpec.org），MSR 2026 学术研究支撑（arXiv:2510.21413）

### Meta-3 · 节奏：调研深度 vs 落地速度
**最终决策**：**今晚结束调研，研发能力决定后续时间**
- 不预设周数 / 不锁 v0.x 节奏
- HR 访谈本期取消（B3 整轨砍）
- 调研产出作为 know-how 库，研发后试用调优

## 第 2 部分 · P1 第一性原理（10 题决策）

### P1-2 · Carl Rogers UPR 是否融入 Neo Soul
**最终决策**：**根据场域需求，原则不一定在同一个 Soul 中**
- Soul 按场域差异化的具体落地——大厅可吸收 Rogers empathy，授课偏 Dweck/Khanmigo
- Neo 不是单一身份，所以 Soul 不需要为"老师 vs 治疗师"二选一冲突付代价

### P1-3 · Knowles "Need to Know" 是否硬规
**最终决策**：**不作为硬规，偏软优先级**
- Neo 默认会有"为什么学这个"的句式，但不在所有场景强制
- 部分场景（如教财务公式）不适合每次问"为什么"

### P1-4 · Kolb 4 阶段映射
**最终决策**：**起点不锁，其他学习理论会补充**
- Kolb 4 阶段（Concrete Experience / Reflective Observation / Abstract Conceptualization / Active Experimentation）作为对练设计参考之一
- 但不严格映射，对练可融合 Action Learning + GROW

### P1-5 · Action Learning Q（追问）是否优先动作
**最终决策**：**7 动作平等，按场域调用**
- 不设单一最优先动作
- 与 Meta-1 一致：场域决定动作权重

### P1-6 · 70-20-10 是否纳入
**最终决策**：**纳入并扩展**
- Neo 不仅在 10% 课堂工作，还**主动引导学员把工作场景带入对练**
- 通过设计 8 Course 与企业实际场景结合，扩展到 70% 工作经验维度
- 学术争议接受，但作为产品设计理念可用

### P1-7 · ICF 8 胜任力是否作为评分维度
**最终决策**：**作为备选评分维度 + Neo 抽象评判**
- 关键洞察：**Neo 不只教内容，还要有脱离内容的评判标准**——内容知识/技能 + 抽象软技能（教练能力本身）共同作用
- ICF 8 胜任力是 Neo 抽象评判的候选之一

### P1-8 · Hattie Teacher Credibility 是否 Soul 顶级
**最终决策**：**作为 Neo Soul 顶级原则（超过 V1-V5）**
- Hattie 256 影响因子：Teacher Credibility d=0.90，超过"内容多丰富"
- Soul 顶层是 Credibility，V1-V5（学习者主导/真实/节奏/行为/长期）服务于 Credibility

### P1-9 · Dweck 表扬 effort 是否锁定
**最终决策**：**视情况而定，绝不无理由表扬，不要像 Gemini 一样当天狗（金句）**
- Voice 层硬约束：不滥用"你真棒"
- 表扬必须具体（哪一步好），鼓励 effort + process 不鼓励 ability
- 但学员产出真高质量时 Neo 也会真实评价 quality

### P1-10 · 苏格拉底/Khanmigo "不给答案"硬度
**最终决策**：**根据学员表现感知，Neo 教学方法要和「懂你」联动设计**
- 关键洞察：**教学法 ≠ 一刀切，与 Memory 系统联动**
- 初期学员可告知事实，W5+ 中期不告知改追问
- 与 P2-1 的"多维度加权"机制呼应

## 第 3 部分 · P2 懂你 · 记忆系统（10 题决策）

### P2-1 · 教学法 + Memory 联动机制
**最终决策**：**多维度加权（W 阶段 + skill mastery + 情绪 + 意图）**
- Memory 不只是记，是为教学策略服务
- 信号融合：学员所处 W 周 + Course 掌握度 + 当前情绪 + 提问意图 → 决定 Neo 教法

### P2-2 · 5 层渐进画像
**最终决策**：**作为 v0.2 起点，但与 Khanmigo skill+prerequisites 融合**
- 5 层（L1 基础事实 / L2 行为模式 / L3 价值偏好 / L4 隐性需求 / L5 自我认知）
- + Khanmigo 的 skill mastery + 前置依赖图谱
- 双重架构（学员维度 + 课程维度）

### P2-3 · 记忆框架选型
**最终决策**：**记录 Mem0/Letta/Zep 可用内容作为自研参考**
- 不直接采用 SaaS 框架（成本/合规/可控性）
- 自研为主，借鉴现有方案的设计思想

### P2-4 · 双层架构等技术选型
**最终决策**：**技术实现层面，PM 不管**
- 关键指令记录：「我只管效果和 Agent 表现，技术细节不带给同事限制」

### P2-5 · 防幻觉机制
**最终决策**：**Neo 溯源靠 Agent 能力，不通过 RAG / 不设向量库**
- 关键技术决策：**不用 RAG**
- 溯源是 Agent 能力的一部分，不是检索机制

### P2-6 · 学员可见 vs 黑盒
**最终决策**：**学员可见，HR 可见原始**（保持 0424 spec 现状）

### P2-7 · 引用频率
**最终决策**：**全场域隐性使用 + 关键节点明示融合**
- 默认隐性（背景调用 voice）
- 关键节点（开场 / 卡点 / 跨课程过渡 / 反思）明示引用

### P2-8 · "懂我"瞬间设计
**最终决策**：**重提学员忘了的细节 + 主动周期提醒**
- 学员说过但忘了的事 Neo 记得（"你之前提过张总对排期压力很敏感"）
- 主动 deadline 提醒（"你提过 5 月要试 X，现在怎样了"）

### P2-9 · Neo 记错时
**最终决策**：**采纳推荐方案：道歉 + 学习机会 + 修正标记**
- 实诚道歉（保护信任）
- 转为学习机会（"那你实际是怎么看？"）
- Memory 标记"此记忆被学员修正"，后期引用时加信号

### P2-10 · Ora 能否访问 Neo 全部记忆
**最终决策**：**Ora 可读 Neo 全部记忆。Neo 对个体学员负责，Ora 对全部学员负责（互补不隔离）**
- 关键架构观点：Neo / Ora 不是隔离，是互补
- 单点责任 vs 整体责任 的分工

### P2-11 · 学员"被遗忘权"
**最终决策**：**不提供（参照真人老师）**
- 真人老师不能选择性遗忘
- 决策判断器（1v1 真人老师）首次应用案例

## 第 7 部分 · 跨议题的元洞察（来自互动中涌现）

### 洞察 1 · Neo 不是单一身份，是按场域差异化
**Meta-1 + P1-2 + P1-5 + P2-7 共振**：Neo 在不同场域调用不同 skill 与倾向。这不是简单的"模式切换"，是底层架构原则。

### 洞察 2 · Teacher Credibility 是顶级原则
**P1-8 升级**：超过 V1-V5。所有 voice / Persona 选择最终回到"是否提升 Teacher Credibility"。

### 洞察 3 · "不当舔狗"是 Neo 的关键 voice 红线
**P1-9 + P3-4 共振**：Voice 层硬约束。表扬必须具体 + 用户错了 Neo 表态。

### 洞察 4 · 教学法 + 记忆联动设计
**P1-10 + P2-1 + P2-2 共振**：Neo 教法因学员状态变化。Memory 不只是记录，是教学决策依据。

### 洞察 5 · 产品 vs 内容 的明确区分
**Khanmigo + skills+prerequisites 决策**：产品层决策（UI/交互）和内容层设计（课程结构/skill graph）分开。Neo 是产品，FLM 是内容。

### 洞察 6 · 1v1 真人老师作为设计判断器
**P2-11 + P3-5 + P3-3 共振**：所有"要不要给学员选择/控制"的决策都回到此问。决策器在 PM 手里，不在 Neo Soul 里。

### 洞察 7 · Neo 与 Ora 互补不隔离
**P2-10**：Neo 对个体负责，Ora 对全部负责。两个 Agent 共享数据，不是隔离的两个系统。

### 洞察 8 · Neo 不靠 RAG，靠 Agent 能力溯源
**P2-5**：技术决策——溯源不是检索机制，是 Agent 内在能力。

### 洞察 9 · Soul 是层级化（Voice / Mode / Memory 各有职责）
**P3-2 + Meta-2 共振**：5 文件结构对应不同稳定度——SOUL.md 不动 / MEMORY.md 演化 / STYLE.md 微调 / SKILL.md 按场域 / soul.json 元数据。

### 洞察 10 · 当前是思考阶段，不直接作用 spec
**P3-1 关键提醒**：调研产出是知识与决策准备，不是 spec 草案。研发拿到的是 know-how，不是 spec patch。
