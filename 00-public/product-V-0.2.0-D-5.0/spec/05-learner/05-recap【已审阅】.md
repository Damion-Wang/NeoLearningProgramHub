## 1 recap（小结）

> **总览**：recap 是学员在 Course 学习中**随时可入**的内容反思场域 —— 对 Course 内容学习情况的小结，目的是**增强学习效果**（不是评价）。内容由三个层面组成：**知识层面回顾 + 表现层面回顾 + 下一步建议**。学员首次完整态进入触发 Activity 完成事件并锁定完成事件层永久保留；之后产生的新数据通过自由复习层覆盖式更新（每次覆盖整层为当下最新综合）。
>
> **结构（10 节）**：业务定义 + 报告全景 + 界面布局 + 4 学员状态总览 → 4 状态依次（① 空白 / ② 部分 / ③ 完整 ★ / ④ 自由复习）→ 全局统一（跨状态通用）→ 本期不做。

### 1.1 业务定义

#### 1.1.1 一句话定位

> recap（小结）是学员在 Course 学习中**随时可入**的反思场域 —— 通过查看 Neo 给出的当前反思报告 + 与 Neo 对话挖掘洞察，把"做过什么"沉淀为"看见什么 + 接下来怎么用"。

recap 同时承担 **三重价值**：

| 价值层次 | 学员视角 | 项目视角 |
|---------|---------|---------|
| **知识层面回顾** | 把 Course 内学过的关键概念 / 框架重新整合，结合自己学习时提出的真实场景做内化 | 让"学过"沉淀为"内化"——核心交付能力 |
| **表现层面回顾** | 看到 Neo 对自己学习行为的整体判断（互动行为 + 6 维画像变化 + 高光与卡点）—— 自己看不到的视角 | 沉淀可举证的学员成长轨迹 |
| **下一步建议** | 在 Neo 的引导下，从反思转向具体下一步行动（应用计划自动入大厅待办）| 让学习闭环到工作场景，避免学完即忘 |

#### 1.1.2 进入路径 + 增量更新机制

recap Activity 出现在**项目整体内容目录中**（与 lecture / practice 平级，三场域 icon 差异化区分）。学员有**两种进入路径**：

| 进入路径 | 含义 |
|---------|------|
| **从大厅内容目录随时进入** | 学员在 Course 学习的**任意时刻**通过大厅内容目录 / 课程目录抽屉点入 recap —— 不要求前序完成度，不强制顺序 |
| **从上一个 Activity 自然跳转进入** | 学员完成 recap 前一个 Activity 后，通过"完成 Modal · 继续下一 Activity"按钮自然过渡进入 |

**首次进入的双路并存** —— 默认是 Activity 完成 Modal 自然过渡，但学员主动从大厅内容目录提前进入也允许（按数据状态自适应）。

**完整态首次进入的判定**：

- **关联 Activity 全部完成** + **学员首次进入 recap** = 完整态首次（触发 Neo 首聊态行为：完整 Walk-through + 产出应用计划入大厅待办）
- 进入路径不影响判定（不论从大厅内容目录点入还是从上一个 Activity 跳转进入，只要满足"前序全完成 + 首次"就触发 Activity 完成事件）

**报告增量更新机制**：

- 报告以"关联 Activity 完成"事件为单位后台增量重新生成；学员进入时直接看最新缓存
- Activity 完成事件触发瞬间锁定完成事件层，永久保留
- Activity 完成事件后，新数据通过"自由复习层"覆盖式更新（详见 § 1.2.7）
- 学员侧零等待，后台静默完成

#### 1.1.3 Activity 完成事件定义

recap 的"完成"采用 **Activity 完成事件**概念（特指 **recap Activity 自身完成**——即首聊态首次完整 Walk-through 触发的一次性事件 / 区别于 lecture / practice Activity 的常规完成）：

| 事件状态 | 含义 | 学员看到 |
|---------|------|---------|
| **未发生** | 学员还没在"完整态首次"下走完 Walk-through —— 含两种情况：① 关联 Activity 未全部完成（部分态 / 空白态进入即使完整对话也不算 Activity 完成事件）② 关联 Activity 已全完成但学员尚未首次进入 | recap Activity 抽屉显示"未完成"；下次"完整态首次"进入触发 Activity 完成事件 |
| **已发生** | 完整态首次进入触发首聊态完整 Walk-through + Neo 评估有实质性反思 + 应用计划已产出并入大厅待办；锁定为完成事件层永久保留 | recap Activity 抽屉标"已完成"；下次进入按行为四态切换（自由复习态 → 回访 / 复习）|
| **已发生但报告有更新** | Activity 完成事件发生后报告增量更新 | 不退回未发生；Subbar 显示"📊 报告有更新"；下次进入触发回访态 |

**Activity 完成事件触发要点**：

- **触发** = **完整态首次进入** → Neo 首聊态完整 Walk-through → 对话有实质性反思 → Neo 一次性产出应用计划入大厅待办
- 仅"完整态首次进入"才能触发 Activity 完成事件（部分态 / 空白态进入即使完整对话也不算）
- 触发瞬间锁定完成事件层（永久独立保留）
- 触发后状态永久转入"自由复习态"——后续再进入按 Neo 行为态切换（回访态 / 复习态）
- **报告增量更新不退回未发生态** —— Subbar 显示"📊 报告有更新"小红点提示

学员部分态 / 空白态下进入也允许阅览当下报告 / 与 Neo 闲聊（Neo 预览态行为，详见 § 1.4），但不计 Activity 完成事件。

**完成判定的精化定义（四件齐全）** —— Activity 完成事件触发需四件齐全：① **完整态首次**（前提条件 / 关联 Activity 全部完成 + 学员首次进入）+ ② **完整 Walk-through 走完** + ③ **实质性反思**（Neo 评估对话不是敷衍）+ ④ **应用计划产出**（1-3 条入大厅待办）。Neo 自主判断后三项达成度（不规定固定轮数门槛）；前提项① 由数据状态客观判定。

### 1.2 Course 个人报告 · 全景

> **核心交付物定位**：Course 个人报告是 recap 的核心交付物。**作用**：报告是 Neo 引导对话的"道具"——写到 70%，留 30% 给 Neo Chat 展开。报告内容承载 recap 的三层价值（知识回顾 + 表现回顾 + 下一步建议）。

#### 1.2.1 报告 2 层结构总览

报告整体分**两层**：

- **完成事件层**（Activity 完成事件触发瞬间锁定，永久保留）—— 由 ① 内容锚点 / ② 走过的轨迹 / ③ 6 维画像 / ④ 关键证据 / ⑤ 待办任务 共 5 块构成
- **自由复习层**（独立区块，每次覆盖式更新）—— Activity 完成事件后新数据进入此层

**完成事件层 5 块结构**（Activity 完成事件触发瞬间锁定，永久保留）：

| # | 块 | 承载层面 | 内容 | 形态 | 数据来源 |
|---|----|--------|----|------|---------|
| ① | **内容锚点** | 知识层面回顾 | Course 主题 + 关键概念 / 框架 + 学员在场域内提出的真实场景串联 + 学员表现一句概括 | 纯 markdown | Course AOM 元数据 + 关联 Activity 概览 + 学员场景描述（来自 lecture FEEDBACK_COLLECT 自由文本 / Neo Chat 对话 / practice 演练上下文）|
| ② | **走过的轨迹** | 表现层面回顾 | 学员在不同关联 Activity 的互动行为画像 | markdown + 1 张图 | 全场域日志：每关联 Activity 的全类型互动总数（含 Quiz 答题 / practice 中栏 / Neo Chat）|
| ③ | **6 维画像** | 表现层面回顾 | 有变化点 → 变化点列表 + 嵌触发证据 / 无变化 → 6 维当前快照 | markdown + 条件性 1 张图 | Neo 在关联 Activity 内记录的画像变化数据（详见 [05-learner/02-hub § 1.1] + [05-learner/03-lecture § 1.4.2] / [05-learner/04-practice § 1.8.6]）+ 6 维当前值 |
| ④ | **关键证据** | 表现层面回顾 | 高光 1-2 条 + 卡点 1-2 条（具体行为片段引用）| 纯 markdown 引用块 | 关联 Activity 内：高光库子集 + practice 复盘待改进点 + Neo 跨场域感知卡点模式 |
| ⑤ | **待办任务** | 下一步建议 | Neo 产出的 1-3 条具体行动建议（应用计划），列在报告里同时自动入大厅待办清单 | 纯 markdown 列表 | Neo 基于报告全数据 + 跨场域记忆 + 学员场景的高度个性化生成 |

**完成事件层之外**：Activity 完成事件触发后，自由复习态产生的内容统一收录在 **9.3.3.2.7 自由复习层**（独立区块，**每次覆盖式更新整层**），**不打散到 5 块里**。

**报告全长**：完成事件层 700-1000 字 + 嵌入图表 1-2 张 / 学员阅读时长 2-3 分钟 / 完整解读体验（含 Neo Walk-through 对话）8-10 分钟。

#### 1.2.2 ① 内容锚点（完成事件层）

| 维度 | 说明 |
|------|------|
| **是什么** | 一段 80-150 字的 markdown：Course 主题 + 关键概念 / 框架 + 学员在场域内提出的真实场景串联 + 学员表现一句概括 |
| **目的** | 让学员一秒回到"这门课在讲什么 + 我跟它的连接"——核心是**知识层面回顾**，结合学员自己的场景做内化锚点 |
| **形态** | 纯 markdown，无图 |
| **数据来源** | Course AOM 元数据（主题 / 关键概念）+ 关联 Activity 列表 + 学员在场域内主动提出的场景描述（由 Neo 跨场域记忆调取）|
| **状态自适应** | 空白态：仅 Course 主题一句 / 部分态：Course 主题 + 已学概念 + 已提场景串联 / 完整态：完整知识回顾 + 学员场景串联 + 表现一句话 |
| **可变性** | Activity 完成事件前每次刷新；Activity 完成事件触发时锁定为完成事件层；之后**永久不变**（学员场景已在首聊时整合） |

**示例骨架**（占位）：

> 这门课讲的是 **{Course 主题}**，关键概念是 **{概念 1 / 概念 2 / 概念 3}**。你在学习中提到过 **{学员真实场景 1，如"我跟同事在 KPI 拆解上有分歧"}**，这正好是本课要解决的问题——{概念 N} 在这种场景下能 {落地用法}。从 {Activity X / Activity Y} 走过来，你整体上 {表现节奏一句概括}。

#### 1.2.3 ② 走过的轨迹（完成事件层）

| 维度 | 说明 |
|------|------|
| **是什么** | 学员在本 Course 关联 Activity 中的**学习行为画像**——哪些 Activity 互动最多 / 哪些最少 |
| **目的** | 让学员看到自己在内容上的**投入分布**——在哪里花了时间 / 在哪里走过场。这是**内容反思素材**，不是项目进度仪表盘 |
| **形态** | markdown 一段（50-100 字解读）+ 图 1（互动行为对比柱图，详见 § 1.2.1 图表清单）|
| **数据来源** | 全场域日志：每关联 Activity 内学员的全类型互动数（不区分类型，避免跨类型比较失真）。**不记 token 消耗**（跨类型 Activity token 差异巨大会误导）|
| **状态自适应** | 空白态：不显示 / 部分态：仅显示已完成 Activity 的对比 + 标注"仅含 X/Y" / 完整态：完整对比图 |
| **可变性** | Activity 完成事件前每次刷新；Activity 完成事件触发时锁定为完成事件层 |

**markdown 解读要点**：
- 指出投入差异最大的 1-2 个对比（"你在 Activity A 互动最多 / 在 Activity B 几乎没问问题"）
- 不预设"哪种好哪种差"——这是行为事实陈述，让学员自己反思

#### 1.2.4 ③ 6 维画像（完成事件层）

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 对学员 6 维度（详见 § 2.3.3）的判断：**有变化点 → 显示变化点列表 + 嵌触发证据**；**无变化点 → 显示 6 维当前快照** |
| **目的** | 让学员看到 Neo 在本 Course 中**对自己的判断**——这是学员看不到的视角，是 1v1 老师的真功夫 |
| **形态** | markdown 解读 + 图 2（6 维画像图，变化点 / 快照二选一，详见 § 1.2.1 图表清单）|
| **数据来源** | Neo 在 lecture / practice 关联 Activity 内观察到学员某维度有实质变化时记录的"画像变化数据"（详见 [05-learner/02-hub § 1.1] + [05-learner/03-lecture § 1.4.2] / [05-learner/04-practice § 1.8.6]）+ 6 维当前值 |
| **状态自适应** | 空白态：不显示 / 部分态：基于已完成 Activity 的变化点（可能为空 → 显示当前快照）/ 完整态：变化点列表 / 当前快照 二选一 |
| **可变性** | Activity 完成事件前每次刷新；Activity 完成事件触发时锁定为完成事件层 |

**变化点呈现规则（每个变化点自含触发证据）**：

```
> Y 维度（如"主动追问能力"）↑ 3→4
> 证据：在 Activity {X} 第 {N} 轮，你 {具体行为片段}——这是这门课希望你练出来的关键动作
```

每个变化点都嵌入触发证据（具体到 Activity / 对话片段），让学员看到"凭什么这么说"。

**无变化点的兜底**：

```
> 本 Course 中 Neo 对你的画像没有触发实质变化。当前 6 维快照：
> [6 维当前值进度条]
```

让学员看到"我现在的状态"，避免"白学了"的失望感。

#### 1.2.5 ④ 关键证据（完成事件层）

| 维度 | 说明 |
|------|------|
| **是什么** | 把抽象洞察落到**具体行为片段**——高光 1-2 条 + 卡点 1-2 条 |
| **目的** | 让学员**认得出**自己——洞察必须基于学员能识别的具体场景 |
| **形态** | 纯 markdown 引用块（每条 = 标题 + 引用片段 + Neo 一句话留白）—— 无图 |
| **数据来源** | **严格限定关联 Activity 内**。高光 = 关联 Activity 内高光库子集（高光卡片是 Neo 在各场域的通用能力，详见 [05-learner/02-hub § 1.1]，recap 取与本次关联范围相符的子集）；卡点 = 关联 practice 复盘的待改进点 + Neo 跨场域感知到的反复出现的卡点模式。**不依赖 Quiz 错题**（Quiz 不一定有对错） |
| **状态自适应** | 空白态：不显示 / 部分态：基于已完成 Activity 内的证据；数量不足时**宁可空着也不补关联外** / 完整态：完整高光 + 卡点 |
| **可变性** | Activity 完成事件前每次刷新；Activity 完成事件触发时锁定为完成事件层 |

**与大厅高光区的关系（透明性对齐）**：

- 高光卡片本身是 Neo 在各场域（lecture / practice / 大厅）的通用能力产物（详见 [05-learner/02-hub § 1.1]）
- recap ④ 块呈现的高光是**大厅高光区的子集** —— 取关联 Activity 范围内的高光
- 学员在大厅高光区能看到完整集合，recap 仅按本次关联范围筛选

**呈现示例**（占位）：

> **🌟 高光｜在 Activity A 你做对了一件事**
>
> > 你在第 N 轮主动追问对方的真实顾虑，没有被表面的拒绝挡住。这是这门课希望你练出来的关键动作。

> **⚠️ 卡点｜你反复栽在同一个地方**
>
> > 在 Activity B 第 X 轮 / Activity C 第 Y 轮，你都是先认同对方再说自己的观点，但最后**忘记把自己的立场说清楚**。

#### 1.2.6 ⑤ 待办任务（完成事件层）

| 维度 | 说明 |
|------|------|
| **是什么** | Neo 产出的 **1-3 条具体行动建议（应用计划）**——直接列在报告里同时自动入大厅待办清单 |
| **目的** | recap 的**下一步建议**层落地——把反思转化为可执行的小任务，闭环到学员真实工作场景 |
| **形态** | 纯 markdown 列表（每条 = 1 句简洁可执行的小任务）—— 无图 |
| **数据来源** | Neo 基于报告 ①-④ 全数据 + 跨场域记忆 + 学员场景的**高度个性化生成**——同 Course 不同学员产出的待办任务差异明显 |
| **生成时机** | **仅在首聊态走完 Walk-through 后生成一次**；后续 Activity 完成事件触发后不再变 |
| **流向** | 同步加入大厅"待办清单"（沿用 [05-learner/02-hub § 1.2.1] (1) E + [05-learner/06-cross-context § 4.4] 任务卡片产品形态）；后续追踪 / 学员勾选 / Neo 跟进 全部在大厅待办流程内 |
| **学员参与** | 不参与撰写、不需确认——Neo 直接产出，直接入待办 |
| **可变性** | Activity 完成事件触发瞬间生成 + 锁定完成事件层；**永久不变**（即使后续报告增量更新、自由复习态再进入也不再生成新待办）|

**呈现示例**（占位）：

```markdown
### ⑤ 待办任务

基于这门课，我建议你接下来一周做这几件事——已加入大厅待办清单：

1. {具体行动 1}（如"下周一跟 X 部门同事的会上，试一次先表达自己立场再求共识"）
2. {具体行动 2}
3. {具体行动 3}

→ [跳转大厅待办查看]
```

**后续追踪**（与大厅协同，详见 § 1.9）：

- 学员可在大厅待办主动点掉
- 学员点掉时 Neo 跟上来问情况
- Neo 评估后再决定是否允许点掉
- **完成不反映到 recap 报告**（闭环在大厅）

#### 1.2.7 自由复习层（独立区块，覆盖式更新）

> **作用**：Activity 完成事件触发后，自由复习态产生的所有新内容统一收录在本区块——独立于完成事件层 5 块结构，**不打散到 5 块里**，**每次更新覆盖整层**。

**位置**：在报告末尾（完成事件层 ①②③④⑤ 之后），用 markdown 标题 `## 自由复习追加` 作分界。

**内容范围**：

| 内容类型 | 说明 |
|---------|------|
| Neo 在自由复习态对话中产生的**当下观察** | 学员重进 recap 跟 Neo 聊天时，Neo 基于截至当下所有自由复习态对话产生的综合观察 |
| **截至当下的画像变化点** | Activity 完成事件后 Neo 在 lecture / practice 内新写入的画像变化（如学员重练 practice 后某维度变化）|
| **截至当下的新高光 / 卡点** | Activity 完成事件后产生的新关联 Activity 内高光 + practice 重练带来的新卡点 |
| **新关联 Activity 总结**（如有）| KGP 后期追加新关联 Activity 时，Neo 对该新 Activity 内学员表现的综合回顾 |

**更新机制（★）**：

- **每次更新覆盖整层**：自由复习层不是按时间叠加，而是**每次更新触发时基于截至当下的所有数据重新生成一份当下最新综合，整体覆盖上次内容**
- **理由**：自由复习态的更新触发点不统一（学员可能 Activity 1 聊两句、Activity 2 聊半小时，产生两次更新），按时间叠加会让内容割裂；用"覆盖式整体重生成"保证学员每次看到的是**当下最新一份完整综合**
- **不保留历史版本**：学员看不到"上一次的自由复习层"，也无法对比
- **完成事件层永远不变**：自由复习层的覆盖式更新不影响完成事件层（完成事件层永久独立保留）

**呈现形态**：

- 自由复习层是**一份当下最新的综合内容**，不分块（区别于完成事件层的 5 块结构）
- markdown 形态自由——可含小标题 / 引用块 / 列表 / 图表（如有新画像变化点可附图）
- 不显示更新时间轴 / 不显示"上次追加 vs 这次追加"分割

**学员侧呈现**：

```markdown
## 首次回顾（永久保留）

### ① 内容锚点
...

### ② 走过的轨迹
...

### ③ 6 维画像
...

### ④ 关键证据
...

### ⑤ 待办任务
...

---

## 自由复习追加（截至当下最新）

- Neo 综合观察：...
- 截至当下的画像变化点：...
- 截至当下的新高光 / 卡点：...
```

**Neo Chat 历史**：在 Activity 内不被清空——学员可翻 Chat 历史回看跟 Neo 聊过的具体对话内容（不在自由复习层重复呈现）。

**笔记复用**：学员"想留住某段感动"通过笔记功能记录（笔记悬浮球，沿用 [05-learner/02-hub § 1.3]）。

#### 1.2.8 生成 / 更新 通用规则

| 维度 | 规则 |
|------|------|
| **触发（Activity 完成事件前）** | 关联 Activity 完成事件触发后台增量重新生成；新版本覆盖旧版本（不留旧快照）|
| **触发（Activity 完成事件触发瞬间）** | 当下报告内容固化为"完成事件层"，永久独立保留 |
| **触发（Activity 完成事件后）** | 自由复习层每次更新触发时整层覆盖式重生成（基于截至当下所有数据综合）；详见 § 1.2.7；完成事件层永远不变 |
| **缓存** | 学员进入 recap 时直接看最新缓存（完成事件层 + 自由复习层叠加视图）|
| **学员侧零等待** | 后台静默完成，学员无感 |
| **不提供"刷新"按钮** | 自动更新即可，按钮反而暴露机制 |
| **修改性** | 学员侧**不可修改**（整体一次性生成）|
| **导出性** | 场域内不可导出（沿用 [05-learner/06-cross-context § 3.4] 跨场域统一规则）；大厅"我的报告"主题可导出 |
| **图表准入** | 三条全过才放图：① 对学员有内容反思价值 ② 范式稳定（可套模板）③ 数据来源清晰。本期 ≤ 2 张（图 1 互动行为对比 / 图 2 6 维画像）|

#### 1.2.9 KGP 在 recap 上的能力边界

KGP 只控制两件事：

| KGP 控制项 | 含义 |
|----------|------|
| ① **位置** | 把 recap Activity 摆放在 AOM 的什么位置（Course 内某点 / Course 末尾 / 项目末尾，详见 [03-lifecycle/03-completion § 1]）|
| ② **关联范围** | 决定 recap 关联**哪些前序 Activity** 作为数据源 —— 关联范围由 KGP 自由控制，**不强制与 Course 绑定**。本期典型场景是关联同 Course 内全部前序 Activity，但**功能上不限制 KGP 跨 Course 关联或仅关联部分**（为未来课程编排玩法留口）|

recap 内部行为（Walk-through 节奏、反思引导、应用计划产出、完成判定标准、报告生成规则）= 产品级统一规则，**KGP 不干预**。

#### 1.2.10 报告产物约束

| 约束 | 说明 |
|------|------|
| **不可修改 / 场域内不可导出** | 整体一次性生成 / 学员侧不可改；Activity 完成事件前每次覆盖更新 / Activity 完成事件触发后完成事件层永久保留 + 自由复习层每次覆盖式更新。场域内不导出（沿用 [05-learner/06-cross-context § 3.4]）；大厅"我的报告"主题可导出 |
| **不承担项目进度信息** | 项目进度类信息（时间锚 / 完成度 / 项目周数 / 阶段 KPI 等）由大厅 [05-learner/02-hub § 1] 承担，recap 不承担也不重复 |
| **与 team report 边界** | recap = 学员侧反思（个人 × Course / Course 末更新 / 学员不可改）；team report = 管理端阶段性对外报告（团队 × 项目周期 / 月度或项目末 / HITL 编辑）。数据可能共享（如 6 维画像），呈现完全分开（详见 [06-management/03-report-center § 1]）|
| **管理端可见性** | **完成态（Activity 完成事件后）**的 recap 报告 → 管理端**可露出**（首页 [06-management/02-home § 1.2.2] (4) 个体详情面板（行内展开含报告列表） / Ora 拉数据生成 team report 均可调用）；**非完成态**（空白态 / 部分态 / 完整态首聊中）→ 管理端**不露出**（未定型内容不应进入对外举证链）|

#### 1.3.3 状态指示

| 状态 | 触发 | 视觉提示 |
|------|------|---------|
| **学习计时** | 默认 | Topbar 右侧呈现本 Activity 累计时长 |
| **报告最后更新时间** | 默认 | Subbar 显示"📊 X 分钟前更新" |
| **报告有更新红点** | 距上次进入有增量更新 | Subbar 红点；进入后 Neo 完成回访态开场后清除 |
| **关联范围徽章** | 默认 | 报告头部显示"本次关联 Activity：[X / Y / Z]"（学员侧明示，详见 § 1.2.1）|
| **部分态徽章** | 学员处于部分态 | 报告头部显示"⚠️ 仅基于已完成的 X/Y 个前序 Activity" |

> - **当前 SCO / 阶段进度** —— recap 不存在 SCO 序列
> - **完成度百分比** —— Activity 完成事件本身是布尔值

#### 1.3.4 部分态特殊呈现

部分态进入时报告头部明确标注：

```

│ 完成更多前序 Activity 后报告会自动更新                  │
└───────────────────────────────────────────────────────┘
```

避免学员误以为是完整评估。

### 1.4 4 学员状态总览

> **命名统一说明**：本节及以下子节用一组 **"4 学员状态"**（① 空白 / ② 部分 / ③ 完整 / ④ 自由复习）作为主线命名。Neo 行为态（预览 / 首聊 / 回访 / 复习）作为内嵌字段呈现于各状态的"Neo 行为"段，不再独立成组。

#### 1.4.1 4 状态定义

| 状态 | 含义 | 报告呈现 |
|------|------|---------|
| **① 空白态** | 关联 Activity 内 0 个完成 | 不生成报告，引导"先做点什么再回来" |
| **② 部分态** | 部分关联 Activity 完成（Activity 完成事件未触发）| 仅基于已完成数据生成；明确标注"仅基于已完成的 X/Y 个前序 Activity" |
| **③ 完整态** | 所有关联 Activity ≥ 1 次完成 + Activity 完成事件未触发的窗口（首次进入触发 Activity 完成事件）| 完整报告——这是即将触发 Activity 完成事件的状态，等学员完整走完首聊 Walk-through 即触发 |
| **④ 自由复习态** | Activity 完成事件已触发后的所有再进入 | **报告分层呈现**：完成事件层（永久独立）+ 自由复习层（每次覆盖式更新）|

#### 1.4.2 4 状态切换图 + Neo 行为态对应

学员可能多次进入同一个 recap Activity（随时打开机制）。Neo 根据**进入时的数据状态 + 距上次发生了什么**自动切换行为：

| 学员状态 | Neo 行为态 | Walk-through | 主动开场 | 应用计划产出 | Activity 完成事件 | Neo 行为细节 |
|---------|----------|-------------|--------|------------|-----------|----------|
| ① 空白态 | **预览态** | ❌ 不做 | 应答 + 提醒"部分状态" | ❌ | ❌ | Neo 不主动 walk-through；应答学员提问 + 提醒"这份报告是部分状态，多做几个再过来会看得更准" |
| ② 部分态 | **预览态** | ❌ 不做 | 应答 + 提醒"部分状态" | ❌ | ❌ | 同空白态：不主动 walk-through；提醒"先去学完，多做几个再过来会看得更准" |
| ③ 完整态首次 | **首聊态** | ✅ 完整 5 块 | ✅ | ✅ 走完后产出入大厅待办 | ✅ 触发（一次性事件，触发后状态转入自由复习态）| 完整逐块引导学员看报告，详见 § 1.7 完整态 |
| ④ 自由复习态 + 距上次有"实质性变化" | **回访态** | ❌ 不重讲全程 | ✅ 主动指出变化点 | ❌（已在首聊态产出）| 已完成 | "自然温和 · 先建连接再深入"——主动指出变化点 + 引用上次对话 + 询问近况 |
| ④ 自由复习态 + 无实质变化 | **复习态** | ❌ 不主动 | ⚠️ Neo 自主判断（可问可不问）| ❌ | 已完成 | 主体被动答疑 —— 学员问什么 Neo 答什么 |

> **关键设计**：Activity 完成事件是一次性事件——完整态首次进入触发后状态永久转入"自由复习态"。锁定为"完成事件层 5 块"永久保留；自由复习态产生的新内容统一进入独立的"自由复习层"区块（**每次更新覆盖整层为当下最新综合**——因为更新触发点不统一，无法按时间叠加）。详见 § 1.2.7 自由复习层。

#### 1.4.3 行为态切换的数据信号

Neo 综合多个信号自主判断当前应该用哪种态（spec 不规定固定阈值）：

**关联 Activity 内信号**（限定 KGP 配的关联范围）：

- 距上次会话是否有新关联 Activity 完成
- 关联 Activity 内是否有新高光卡片

**全场景信号**（不限定来源）：

- 6 维画像是否有明显变化（含大厅 / 跨 Course 来源）
- 学员在大厅或其他场景是否有新对话
- 学员上次在 recap 内跟 Neo 聊到哪
- 上次 Walk-through 走到哪一块

#### 1.4.4 对话场景结构

recap 是融合的对话场景（看报告 + 与 Neo 聊 + Neo 给行动建议三件事在同一屏幕同时进行），不是分阶段的内容序列。recap 内部行为是产品级统一规则，KGP 不干预。

单一连续对话流：

```
学员进入 recap Activity
  ↓
Neo 按当前数据状态判断行为态（预览 / 首聊 / 回访 / 复习）
  ↓
学员 ↔ Neo 自由对话（融合看报告 + 反思挖掘 + 接收行动建议）
  ↓
首聊态走完 Walk-through 后 Neo 自动产出应用计划 → 入大厅待办清单
  ↓
完成 Activity 完成事件（Neo 引导收尾 + 应用计划产出 → 触发 / 详见 § 1.7.4）
  或退出（断点续播保存）
```

### 1.5 ① 空白态（无前序 Activity 完成）

#### 1.5.1 报告呈现

- ① 内容锚点：仅 Course 主题一句
- ② 走过的轨迹：不显示
- ③ 6 维画像：不显示
- ④ 关键证据：不显示

- **不主动 walk-through**

#### 1.5.3 学员动作

- 进入 → 看提示 → 退出
- 全程右栏 Chat 可问 Neo（学员主动提问 Neo）
- 不计 Activity 完成事件

### 1.6 ② 部分态（部分前序 Activity 完成）

#### 1.6.1 报告呈现

- 已完成块生成 + 顶部 ⚠️ "仅基于已完成的 X/Y" + 占位块灰显

- 不计 Activity 完成事件

### 1.7 ③ 完整态（首次进入）★ 主场

#### 1.7.1 报告呈现

- 完成事件层 5 块完整生成
- 自由复习层未启用（Activity 完成事件未触发）
- 完整 ① 内容锚点 + ② 走过的轨迹 + ③ 6 维画像 + ④ 关键证据 + ⑤ 待办任务

#### 1.7.2 Neo 行为（首聊态）

完整态首次进入触发首聊态——Neo 主动逐块引导学员看报告。

**Walk-through 节奏（仅首聊态）**：

Neo 在首聊态主动逐块引导学员看报告：

```
1. Neo 第 1 句：定位锚点
   "我们一起看看这份小结。先问你一件事：
    你看完第 ① 节内容回顾，最让你想起的场景是哪个？"

2. 学员回应 → Neo 引导到第 ② 节走过的轨迹
   "那我们看看你在不同环节的投入分布……"

3. 引导到第 ③ 节 6 维画像
   "我注意到你在 [Activity X] 之后 [Y 维度] 上来了，
    具体证据是 [嵌入证据片段]……你认这个评价吗？"

4. 引导到第 ④ 节高光 + 卡点
   "再看几个关键瞬间……"

5. 引导到第 ⑤ 节待办任务（=应用计划）
   "基于以上这些，我给你列了几件下一步可以试的事——
    我已经把它们加到你的大厅待办清单里了。"
```

**关键点**：
- 首聊态是一次性事件——走完即触发 Activity 完成事件，不重复
- 学员中途退出 walk-through → 重进时从中断点续 walk-through，不从头
- **没有"重新听 walk-through"机制** —— 首聊态完成后通过 Neo 自由问答覆盖学员需求

**留白式追问规范（首聊态主场）**：

Neo 追问的核心：**不替学员说总结，引发学员自陈**。

| 类型 | 示例 |
|------|------|
| 引发观察 | "你看到这条洞察吗？想到了什么？" |
| 引发反思 | "为什么你觉得这维度低？" / "这个模式是不是熟悉的？" |
| 引发联想 | "这跟你上周在大厅说的那件事有关系吗？" |
| 引发承诺 | "你觉得接下来一周能做的一件事是什么？"（首聊态收尾）|

**跨场域记忆调用**：

Neo 在首聊态**主动**引用学员的跨场域信息——让"老师记得你"成为体感：

| 引用对象 | 来源 |
|---------|------|
| 学员在 lecture 中提到的真实场景 | lecture FEEDBACK_COLLECT / Neo Chat 对话 |
| 学员在 practice 演练里的具体行为 | 关联 practice 演练对话 |
| 学员在大厅 Neo Chat 里说过的话 | 跨场域 Neo 记忆系统 |

引用准确性信任 [05-learner/06-cross-context § 4.1] 反幻觉机制。

**必收尾约束**：

Neo 追问之后**必须**给收尾——不能只追问留开放状态（沿用 practice 复盘 [05-learner/04-practice § 1.6.5]）。

- 首聊态收尾 = Neo 给观察 + 留言 + 产出应用计划入大厅待办

#### 1.7.3 学员动作

- Walk-through 跟随 / 主动追问 / 待办任务确认（不需手动确认 — Neo 直接产出，直接入待办）
- 全程右栏 Chat 可问 Neo
- 中途退出 walk-through → 重进时从中断点续 walk-through

#### 1.7.4 Activity 完成事件触发条件

- 完整态首次进入 → Neo 首聊态完整 Walk-through → 对话有实质性反思 → Neo 一次性产出应用计划入大厅待办
- 触发条件：完整 Walk-through + 实质性反思 + 应用计划产出（Neo 自主判断，不规定固定轮数门槛）
- Activity 完成事件触发瞬间锁定完成事件层（永久独立保留）；状态永久转入"自由复习态"

#### 1.7.5 待办任务产出 → 大厅待办清单

待办任务（应用计划）的产出形态、生成时机、内容规则详见 § 1.2.6（完成事件层 ⑤ 待办任务）。本节仅说明**对话场景中的产出节奏**与**与大厅协同的后续追踪**。

**对话场景节奏**：

- 首聊态 Walk-through 走完 ①-④ 后，Neo 主动收尾 → 直接列出 1-3 条待办任务 → 同步加入大厅待办清单 → Activity 完成事件触发
- 学员侧不需手动确认；Neo 产出即定型

**与大厅协同的后续追踪**：

- 学员可在大厅待办清单**主动点掉**待办项（永远不限制总量）
- 学员点掉时 **Neo 跟上来问情况**（"做完了吗？怎么做的？"）
- Neo 评估学员回应后**再决定是否允许点掉**（即点掉需 Neo 确认）
- **完成不反映到 recap 报告** —— 闭环在大厅待办区完成（详见 § 1.9.7 应用计划后续追踪）

### 1.8 ④ 自由复习态（Activity 完成事件后再进入 / 不再细分）

#### 1.8.1 报告呈现

- **完成事件层永久保留**（① 内容锚点 / ② 走过的轨迹 / ③ 6 维画像 / ④ 关键证据 / ⑤ 待办任务，5 块永久不变）
- **自由复习层动态更新**（覆盖式 —— 详见 § 1.2.7）
- 当报告有增量更新时，Subbar 显示"📊 报告有更新"小红点

#### 1.8.2 Neo 行为（回访态 + 复习态合并描述）

自由复习态不分子态，Neo 行为统一描述：

- **被动答疑为主 / 学员主导自由探索**
- **进入时若报告有更新 → 主动简短同步更新点**（即"回访态"行为：自然温和 · 先建连接再深入；主动指出变化点 + 引用上次对话 + 询问近况）
- **报告无更新时**（即"复习态"）：Neo 自主判断是否开场（可问可不问），主体被动答疑——学员问什么 Neo 答什么后给收尾

不重讲全程 walk-through。

学员"想再听一遍 walk-through"的需求**不通过独立按钮覆盖**——首聊态是一次性结构化讨论，剩下通过 Neo 自由问答覆盖需求。

#### 1.8.3 学员动作

- 自由探索 / 跨场域追问 / 自由复习层增量更新（覆盖式）
- 全程右栏 Chat 可问 Neo
- 笔记悬浮球记录"想留住此刻"
- Neo Chat 历史可翻看（Activity 内不被清空）

#### 1.8.4 重进逻辑（按报告是否有更新红点）

Activity 完成事件发生后再进入 recap → Neo 按行为态判断：

| 进入场景 | Neo 行为 | 视觉提示 |
|---------|---------|---------|
| 报告无更新 | 复习态——Neo 自主判断是否开场，主体被动答疑 | Subbar 无红点 |
| 报告有更新（Subbar 红点提示）| 回访态——Neo 主动指出变化点 | Subbar "📊 报告有更新" |

报告随关联 Activity 完成自动更新（增量），学员侧无感。

Activity 完成事件未发生前再进入：

- 仍处于空白 / 部分 / 完整态 → 对应预览态 / 首聊态行为
- 中途退出后重进 → 从断点续 walk-through（不从头）

### 1.9 全局统一 · 跨状态通用

> 本节归集跨学员状态都适用的通用规则——Neo 设计原则 / 7 动作分工 / 学员行为通用 / 体验机制 / 通用沿用 / 完成 Modal / 应用计划后续追踪 / 跨场域衔接。

#### 1.9.1 Neo 反思设计原则

| 约束 | 说明 |
|------|------|
| **不替学员总结洞察** | Neo 引导学员说，不替学员说——防止报告 / 对话变成 Neo 的"评分书"。学员反思必须由学员**自陈**，Neo 用追问引发，不强行总结 |
| **不预设标准答案 / 不给"应该"** | 沿用 [02-foundation/02-methodology § 1.7] + [05-learner/03-lecture § 1.6.1] —— Neo 不告诉学员"你应该这样想 / 这样做"，引导学员自己得出 |
| **显性证据限定关联 Activity / 隐性背景不限定** | 报告内可被学员看到具体来源的证据类（高光 / 卡点 / 对练具体行为 / 6 维画像变化点）严格限定 KGP 配的关联 Activity 内；Neo 综合产物（自由复习层观察 / 待办任务依据 / 跨场域记忆）**不限定来源**（限定反而失真）。**不依赖 Quiz 错题**（Quiz 不一定有对错，详见 [05-learner/03-lecture § 1.3.4]） |

#### 1.9.2 Neo 在 recap 的定位 + 7 动作分工

Neo 在 recap 与大厅 / lecture / practice 共用同一身份（同 voice / 方法论 / 记忆，详见 [02-foundation/01-personas § 1.2] + [05-learner/06-cross-context § 3.0]）。在 recap 中 Neo 引导反思为主——不主导讲授（与 lecture 区别）/ 不旁观演练（与 practice 区别）。

核心姿态：**引导学员看见自己 + 不替学员说总结**。

| 动作 | 权重 | recap 中的体现 |
|------|------|---------------|
| **反思** | 主场 | 引导学员看见自己的行为模式与变化轨迹 |
| **追问** | 主场 | 苏格拉底式追问，挖出学员自己的洞察 |
| **讨论** | 主场 | 与学员就报告内容平等对话 |
| **解惑** | 辅助 | 学员对报告或概念有疑问时回答 |
| **考核** | 辅助 | 报告里的 6 维画像 + 高光 + 卡点 隐含评估反馈 |
| **讲授** | 轻量补漏 | 学员忘了某概念 Neo 可补，**不主动**讲新内容 |
| **练习** | — | 不出现（→ practice）|

**recap 4 状态卡片产出矩阵**（5 类卡片定义见 [02-foundation/02-methodology § 1.6] / 与 lecture [05-learner/03-lecture § 1.4.5] / practice [05-learner/04-practice § 1.7.3] 对位）：

| 卡片类型 | ① 空白 | ② 部分 | ③ 完整 / 首聊 | ④ 自由复习 | 备注 |
|---------|-------|-------|------------|-----------|------|
| **askUserQuestion** | ✓ | ✓ | ✓ | ✓ | Neo Walk-through / 留白追问 / 回访态 全程可推 |
| **任务卡片** | — | — | ★ 主场 | — | 完成事件触发瞬间一次性产 1-3 条入大厅待办（详见 § 1.2.6 / [05-learner/06-cross-context § 4.4]）|
| **高光卡片** | — | — | ✓ | ✓ | recap 不产出**新**高光（高光生于 lecture / practice 实际行为时刻）；recap ④ 关键证据块**引用**关联范围内已有高光库子集；自由复习态若识别学员新顿悟可推（与各场域同口径，详见 [05-learner/06-cross-context § 4.4]）|
| **知识卡片** | — | — | ✓ | ✓ | Neo 反思引导中如学员表示某概念模糊 → 可补推延伸阅读形态知识卡片（轻量场景 / 不主动推）|
| **课程卡片** | — | — | — | — | 不出现（课程卡片是大厅 → 课程入口卡片 / 与场域内无关）|

> recap 与 lecture / practice 同口径：**课程卡片不进任何场域内**；高光 / 任务 / 知识 / askUserQuestion 按本场域定位差异化推送。

#### 1.9.3 学员行为通用

**输入方式**：

| 方式 | 交互 |
|------|------|
| **右栏 Neo Chat 输入框** | 学员与 Neo 对话（打字 / 语音）—— recap 内学员只跟 Neo 对话，不写应用计划 |

**核心交互一览**：

recap 多数交互沿用 lecture 设计（详见 [05-learner/03-lecture § 1.5.2]）：

| 交互 | 说明 |
|------|------|
| **随时进入** | recap 整个 Activity 入口对前序完成度无锁——可随时进；4 状态自适应 |
| **防挂机** | 默认 5 分钟无操作 → 系统弹提醒（沿用 lecture，可在 program config 调整阈值）|
| **学员主动提问 Neo** | 全程右栏 Chat 可问 Neo |
| **抽屉锚点跳转** | 学员点抽屉 Tab1 报告 5 块锚点 → 课件区滚动到对应块 |
| **Activity 间跳转** | 通过抽屉 Tab2 主动跳转其他 Activity（弹窗确认 + 触发断点续播保存）|
| **断点续播** | 中途退出 → 系统记录当前对话上下文 + Walk-through 进度；下次进入自动恢复 |
| **笔记悬浮球** | 屏幕边缘常驻（沿用 [05-learner/02-hub § 1.3]）—— 学员"想留住此刻"通过笔记功能记录 |

#### 1.9.4 体验机制约束

| 约束 | 说明 |
|------|------|
| **取消 SCO 锁定** | recap 整个 Activity 入口可随时进；不存在 SCO 序列，与 lecture / practice 严格顺序锁定不同 |
| **6 维画像可见性** | 学员侧 recap 报告露出（含变化点 / 当前快照）；管理端通过**完成态 recap 报告**间接可见（详见 § 1.2.10 报告产物约束 / [02-foundation/01-personas § 1.4.4] Ora 上下文范围）；**管理端独立 6 维画像 widget = [06-management/02-home § 1.2.2] (4) 个体详情面板内的 6 维画像快照**（已落地决策 2026-05-04 / 复用 [02-foundation/02-methodology § 1.3.3] / [05-learner/02-hub § 1.1] B.2.B.4 雷达图组件）|

#### 1.9.5 通用沿用

- **防挂机**：默认 5 分钟无操作触发（沿用 [05-learner/03-lecture § 1.6.4]）
- **数据持久化**：推荐每个会话 / Walk-through 节点完成时落库（沿用 [05-learner/03-lecture § 1.6.6]）
- **隐私三层**：沿用 § 3.4.3（学员之间默认完全隔离 / 跨学员引用需同意 / 管理端例外）

#### 1.9.6 完成 Modal

学员首次触发 Activity 完成事件时弹完成 Modal：

```
┌─────────────────────────────┐
│                        [X]  │
│         ✅                    │
│   Course 小结完成！            │
│  你已完成「{Course 小结名}」   │
│  应用计划已入大厅待办（3 项）   │
│                              │
│  [回到大厅]                   │
│  [自由复习]                   │
│  [继续下一 Activity]            │
└─────────────────────────────┘
```

按钮行为沿用 lecture / practice 3 选项设计（详见 [05-learner/03-lecture § 1.7.2] + [05-learner/04-practice § 1.8.7]）。

> **"自由复习"统一概念**：lecture / practice / recap 完成 Modal 里的"自由复习"按钮 + recap "自由复习态"状态，三者指向同一概念——**Activity 流程结束之后，学员允许回到该场域针对 Activity 内容和已经发生过的事情和 Neo 进行探讨**。

回到大厅：默认进大厅首页（项目总览区已含待办清单），不为 recap 跳转打破默认体验。

#### 1.9.7 应用计划后续追踪

应用计划入大厅待办后，在大厅待办流程内闭环（不在 recap 内回访）：

```
学员在大厅待办主动点掉（永远不限制总量）
   ↓
Neo 跟上来问情况（"做完了吗？怎么做的？"）
   ↓
学员回复
   ↓
Neo 评估后再决定是否允许点掉
```

剩余服务期内同样适用。

> **应用计划完成不反映到 recap 报告** —— 闭环在大厅待办区完成；不污染 recap 报告分层结构。

#### 1.9.8 跨场域衔接

> 全局跨场域机制详见 [05-learner/06-cross-context § 3.4] 跨场域协同（Truth Source）；本节仅从 recap 侧视角列入口与出口。

| 衔接 | 数据流 |
|------|--------|
| **关联 Activity（lecture / practice）→ recap** | 报告数据流：高光库子集 / 6 维画像变化 / 学员互动行为日志 / practice 复盘待改进点 / Neo 跨场域感知 全部进入报告生成（详见 § 1.2 各块数据来源）|
| **recap → 大厅** | 完成状态 + 报告进入大厅"我的报告"主题（[05-learner/02-hub § 1.2.3]）；应用计划进入大厅待办清单（详见 [05-learner/02-hub § 1.2.1] 待办清单子条）|
| **recap → 管理端** | team report 拉**完成态**（Activity 完成事件后）recap 数据；首页 [06-management/02-home § 1.2.2] (4) 个体详情面板（行内展开含报告列表）展示**完成态**报告；数据共享但呈现完全分开（详见 [06-management/03-report-center § 1] / § 1.2.10 管理端可见性条）|

> **与大厅的边界**：项目进度类信息由大厅 [05-learner/02-hub § 1] 承担，recap 不承担也不重复（详见 § 1.2.10）。
