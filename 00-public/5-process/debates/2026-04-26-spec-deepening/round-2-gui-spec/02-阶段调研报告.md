## 1. mobile / chat UI 上拖拽题型（连线 / 排序）实践

### 关键发现
- ✅ 拖拽题型在教育领域是**经典互动**——drag-and-drop matching / sequencing 大量使用（Moodle / Agilix / IQBoard 等都有支持）
- ⚠️ **不应是唯一方式**：mobile / chat UI 上拖拽必须有"简单交互"作为 fallback
- 业内最佳实践：**drag + click 双轨**（学员可拖拽，也可逐项点击对应）
- 拖拽视觉反馈：浮起 / 阴影 / 半透明 / 目标高亮

### 引用
- [LogRocket · Drag and Drop UI Best Practices](https://blog.logrocket.com/ux-design/drag-and-drop-ui-examples/)
- [Pencil & Paper · Drag and Drop UX](https://www.pencilandpaper.io/articles/ux-pattern-drag-and-drop)
- [IQBoard · Drag-and-Drop Quiz](https://iqboard.net/how-to-create-a-drag-and-drop-quiz-for-interactive-learning/)
- [Moodle · Drag-and-Drop Matching plugin](https://moodle.org/plugins/qtype_ddmatch)

### 对轮 2 议题 3 决议的影响
- ✅ 维持决议："连线/排序在看板区拖拽"
- ➕ 新增约束：**必须有 click fallback**（连线 = 也可逐项选择 / 排序 = 也可逐题选第 N 步）
- 教育实践证明 drag-drop 增加学员 engagement，**值得本期 P0/P1 做**（即使 KGP 抽样 < 5%）
- mobile 上要测试可用性

## 2. 教育产品"学员资产库"导航对标

### LinkedIn Learning · "My Learning" 模块
- **tab 结构**：In Progress / Saved / Collections / Learning History / Assigned by Org / Recommended by Org
- 每 tab 显示**数量**（如 In Progress (5)）
- "Browse" tab 按 Business / Creative / Technology 等大类
- 顶部 banner：个人化推荐（基于历史 + 角色 + 兴趣）

### 对睿学 4 库 GUI 设计的启示
- ✅ 多 tab + 每 tab 显示数量：直接借用
- ✅ 默认按大类（库）分而非时间：4 库就是 4 个大类，类似 LinkedIn 分类
- ⚠️ "Recommended"——LinkedIn 是组织推荐，睿学是 AI 推送（发现库的本质）→ 可参考组织推荐 UI 思路
- 个人化 banner（"基于你最近学的 PIN 法，推荐你看 POWER 法资源"）→ 可作为 V2 增强

### 引用
- [LinkedIn Learning · Use the Learning Library](https://www.linkedin.com/help/learning/answer/a704863/using-the-linkedin-learning-library)
- [Wooster · Navigate LinkedIn Learning Interface](https://inside.wooster.edu/technology/knowledge-base/navigate-the-linkedin-learning-interface/)

### 对轮 2 议题 2 决议的影响
- ✅ 维持决议：4 个 tab + 时间线 + Course 筛选
- ➕ 新增约束：**每 tab 显示数量**（如"成果库 (8)"）—— LinkedIn 工业实践
- ➕ V2 加"个人化推荐 banner"
