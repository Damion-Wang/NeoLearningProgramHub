# 项目配置模块

## 模块定位

管理端导航中的项目定制化页面，用于开营配置和项目管理。管理员在常显导航点击「配置」进入，左栏展示配置表单。纯 GUI 交互，不需要 Ora。

---

## 功能区域

| 区域 | 内容 |
|------|------|
| **常显导航** | 首页 / 报告 / 消息 / 配置（当前） |
| **单栏：配置表单** | 项目信息 / 学员导入 / 角色分配 / 平台个性化 / Token 统计 / 项目内容预览（纯 GUI，无 Ora） |

---

## 功能清单

### 1. 项目基本信息

- 项目名称（文本输入）
- 课包绑定确认（内容包由内部团队预先导入，管理员确认绑定即可）

### 2. 项目周期配置

- 日期选择器设置起止时间
- 范围：1 个月 ~ 1 年
- 管理员自行配置，非后端管控

### 3. 学员名单导入

- Excel 文件上传
- 导入预览（成功/失败/重复数量，失败行标注原因）
- 确认后自动创建账号并发送登录通知（Email）

### 4. 角色分配

- 导入后默认角色：学员
- 手动调整：选中单个账号 → 设置为管理员
- 批量操作：勾选多个账号 → 批量设置角色
- 可选角色：学员 / 管理员（本期两种）

### 5. 自动催学规则配置

管理员配置自动催学触发条件和消息模板，系统按规则自动执行。自动消息的发送记录在「消息」模块中统一查看。

| 项目 | 内容 |
|------|------|
| **触发条件** | 未登录天数 / 进度落后比例 / 未完成活动天数等，可自定义 |
| **消息模板** | 支持变量插入（学员姓名、课程名称、进度信息等） |
| **执行策略** | 自动执行频率、每日上限 |
| **升级规则** | 满足升级条件时标记为需人工确认 |
| **消息原则** | 本期优先固定模板；争取实现大模型根据学员实际情况生成个性化消息 |

### 6. 平台个性化

- **LOGO 上传**：上传客户企业的 LOGO，替换系统默认 LOGO
- **老师形象选择**：从预设形象库中选择 AI 老师的头像/形象（本期固定选项）
- **老师音色选择**：从预设音色库中选择 AI 老师的语音音色（本期固定选项）

### 7. 角色管理入口

- 查看当前角色列表和权限
- 当前两种角色：学员 / 管理员
- 入口保留，未来可新增组长等角色

### 8. Token 消耗统计（只读，P3）

- 总 Token 消耗量
- 本月消耗
- 按日趋势折线图
- 按周趋势柱状图

> 项目进度统计已移至首页总览（[01-operation.md](01-operation.md)）

## 用户故事：管理员小李的开营配置完整流程

### 场景背景
某消费品企业签约了"基层管理者培养项目"，管理员 已提供 32 名参训学员的 Excel 名单。管理员小李需要完成开营配置。

### 完整旅程

**前置条件 — 内容包已由内部团队配置完成**

在管理员操作之前，内部技术团队已完成 AOM 内容包的导入和配置。管理员拿到的是一个"已装好课程的项目"。

**步骤 1 — 项目基础配置**

小李在管理端常显导航点击「配置」，进入项目配置页面，填写基本信息：
- 项目名称：XX 企业基层管理者培养项目（2026 上半年）
- 课程包：基层管理者课包（8 Course）——已由内部团队预先导入，管理员确认绑定
- 项目周期：通过日期选择器设置起止时间（范围：1个月~1年）

**步骤 2 — 导入学员名单**

小李点击「学员管理」→「导入名单」，上传 管理员 提供的 Excel 文件。

系统解析后展示导入预览：
- 成功：30 条（自动创建账号）
- 失败：1 条（缺少手机号字段）— 标注失败原因
- 重复：1 条（手机号已存在）— 标注重复信息

小李修正失败的 1 条数据后重新导入，确认全部 32 条成功。

**步骤 3 — 设置角色**

导入完成后，小李在学员列表中为每个账号分配角色：
- 默认角色：学员（导入时自动分配）
- 手动调整：选中李经理 → 设置为"管理员"角色；选中自己 → 确认"管理员"角色
- 批量操作：勾选多个账号 → 批量设置角色
- 可选角色：学员 / 管理员（本期两种，未来可能新增组长等）

**步骤 4 — 确认并通知**

小李点击「确认并发送通知」→ 系统自动：
1. 为 32 名学员创建账号（角色已分配）
2. 通过 Email 发送登录通知（含登录地址和临时密码）
3. 不同角色登录后看到不同界面（学员→大厅，管理员→管理端）

**步骤 5 — 配置自动提醒**

> 催学规则配置已移至消息模块，小李切换到「消息」导航进行配置。

**步骤 6 — 平台个性化配置**

小李在项目配置页面点击「平台个性化」，完成品牌定制：
- 上传客户企业的 LOGO
- 从预设形象库中选择 AI 老师的头像
- 从预设音色库中选择 AI 老师的语音音色

**步骤 7 — 内容包完整性校验**

小李在项目配置中点击「项目内容预览」→ 预览已装载的课程内容：
- 查看 Course/Activity/SCO 完整结构
- 预览各 Activity 的教学内容
- 确认内容与项目需求匹配

> 项目内容预览为只读，本质上是给研发团队一个预览窗口，确保装载的项目内容正确。

小李预览确认无误后，点击「确认开营」。系统触发：
1. 学员端开放登录（开营前学员无法进入系统）
2. 运营在平台外发送登录链接给学员（本期不做系统自动群发 Email）

## 已确认决策

- 内容包由内部团队预配置，管理员只做绑定确认
- 项目周期由管理员通过日期选择器配置（1个月~1年范围），非后端管控
- 角色在导入名单后分配——默认为学员，可手动/批量调整为管理员
- 学员首次登录后无需复杂配置，直接进入大厅
- 30-40 人规模暂无学员组/部门分组划分
- 自动提醒规则可配置（触发天数、消息模板、执行策略）
- 平台个性化配置：LOGO + 老师形象 + 老师音色
- 角色管理入口保留，当前只有两种角色（学员/管理员），未来可新增
- Token 消耗统计：总量 + 按日/周趋势，只读
- 项目内容预览：只读，用于确认装载内容正确
- 确认开营触发：学员端开放登录，运营在平台外发送登录链接（本期不做系统自动群发）

---

## 本期不做的功能

- 学员自注册——所有账号由管理员导入创建
- 学员分组/部门划分——30-40 人规模暂不需要
- 企业级权限体系——本期聚焦单项目交付
- AI 互动频率调整——学员不能自己调整 Quiz 追问次数等
- 多语言支持——本期不支持

## 反向补充：催学规则与消息模块独立

- 催学规则在**本配置页**设置：
  - 触发条件（未登录 X 天 / 进度落后 Y%）
  - 消息模板（富文本+变量）
  - 频率（每 X 天 / 每日上限）
  - 启用/暂停开关
- 消息记录在**消息页**查看，自动消息带"来自催学规则"标识

**不做双向联动**（避免产品复杂度）：
- 在配置页不显示"此规则已触发多少次"等实时统计
- 在消息页不显示"点击修改规则"等反向入口
- 两者各自独立，用标识连接

## 反向补充：「确认开营」二次确认弹窗

点击 [确认开营] 按钮后：

```
┌────────────────────────────┐
│  ⚠ 确认开营                  │
│                             │
│  开营后配置将锁定，不可撤回。 │
│                             │
│  项目摘要：                  │
│   - 学员：32 人              │
│   - 课程：4 门               │
│   - 周期：2026-03-15 ~ 06-30 │
│                             │
│     [取消]  [确认开营]       │
└────────────────────────────┘
```

### 确认后效果

- 所有表单字段置灰不可编辑
- [确认开营] 按钮变为 [已开营 · YYYY-MM-DD] 灰态
- Topbar chip 由"开营前 · PRE-LAUNCH" → "运营中 · RUNNING"
- 系统自动：
  - 学员端开放登录
  - 运营在平台外发送登录链接（本期不做站内推送）
  - 催学规则自动激活

## 反向补充：双身份账号支持（可选配置）

**背景**：某些场景下一个账号需要同时承担"学员"和"管理员"身份，例如：
- 初创公司 HR 兼学员
- 部门经理自己学 + 管理团队

### 设计

- 配置页"角色分配"列，允许勾选"学员+管理员"双身份
- 双身份账号的端口切换免登录直接跳
- 单身份账号端口切换时返回登录页（切账号）

### demo 的应用
- 张磊（student01）：双身份（中期学员 + 管理员），能来回切
- 李明（newbie01）：双身份（初期学员 + 开营新运营），同上
