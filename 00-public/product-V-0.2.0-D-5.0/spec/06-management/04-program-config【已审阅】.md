## 1 · 项目配置 Program Config

> 这是管理端的**配置中枢**——开营前 admin 全量配置项目 / 开营后管理员日常变更。
>
> **纯 GUI 模块**（无 Ora 参与 / 沿用 [06-management/01-overview § 1.1.6.1]）。
>
> 多管理员场景：admin / 管理员（HR + 项目运营）**都可改 Config**（详见 § 1.3.2）。

### 项目配置 = 双态 + 6 配置板块

```
项目配置 Program Config
├── 双态（详见 § 1.1）
└── 6 配置板块（详见 § 1.2）
    ├── 1. 项目信息
    ├── 2. 人员名单与角色
    ├── 3. 里程碑配置
    ├── 4. 催学规则
    ├── 5. 平台个性化
    └── 6. 内容预览
```

### 入口与模块关系

- **入口**：管理端左侧导航第 3 项（开营前是**唯一**可访问模块 / 开营后与其他 3 模块并列）
- **双态下其他模块可见性**：
  - 开营前：Home / Report / Message **完全隐藏**（管理员账号未开放 / admin 仅能进 Config）
  - 开营后：4 模块全部开放（沿用 [06-management/01-overview § 1.1.4]）
- **与 [03-lifecycle/01-pre-learning § 1] 项目生命周期引用**：详细字段定义见 [03-lifecycle/01-pre-learning § 1.4] / [03-lifecycle/01-pre-learning § 1.5] / [03-lifecycle/02-learning § 1]

### 1.1 业务定义与双态切换

项目配置是**项目准备期与变更期的设置场域**——管理项目信息 / 学员名单 / 角色分配 / 里程碑 / 催学规则 / 平台个性化 / 内容预览，是项目从启动到收尾的"控制台"。

**双态**：开营前 admin 配置态（详见 [03-lifecycle/01-pre-learning § 1.5]）→ "确认开营"触发 → 开营后管理员运营态（详见 [03-lifecycle/02-learning § 1]）。

| 维度 | 开营前态 | 开营后态 |
|------|---------|---------|
| **触发** | 签单后 admin 账号开通 → 确认开营前 | admin 点 "确认开营" 按钮后 |
| **主要使用者** | admin（持有根账号）| 管理员（HR + 项目运营 合并）+ admin 仍可访问 |
| **全局模块状态** | **仅 Config 入口开放** / Home + Report + Message 完全隐藏 | 4 模块全部开放 |
| **字段可改性** | **全可改**（含服务周期 + 人员名单）| **服务周期 锁定**（仅后台改）/ **人员名单总数锁定 + 部分可换人**（未首次登录账号可换 / 已首次登录账号不可换 / 详见 [03-lifecycle/02-learning § 1.5] SOP + § 1.2.2）/ 其他全可改 |
| **必填校验** | 必填字段未填则 "确认开营" 按钮禁用 | 不存在必填校验（已开营）|

**"确认开营"按钮**（开营前唯一在 Config 主页底部）：
- 位置：Config 主页底部 / 必填字段全部填完后才可点击（否则 disabled + tooltip 提示哪些字段未填）
- 点击触发连锁动作（沿用 [03-lifecycle/01-pre-learning § 1.5]）：
  1. 所有账号（学员 + 管理员）统一开放登录
  2. 统一发送系统消息通知所有学员 + 管理员
  3. Config 进入开营后态 / Home + Report + Message 入口开放
- **不可逆**：点击后无法回到开营前态

**服务周期 锁定**：开营后该字段灰置 + tooltip 提示"已开营 / 需联系睿学后台修改"。

**人员名单总数锁定 + 部分可换人**（沿用 [03-lifecycle/02-learning § 1.5] 学员名单变更 SOP）：
- 总账号数:合同签单时锁定 / 学习期内**不可增加**(GUI 体现:"+ 添加人员"按钮灰置)
- 未首次登录账号:✅ 可换人（替换离职 / 新员工等场景）
- 已首次登录账号:❌ 不可换人（保留学员学习数据）
- 节点判定:该账号是否曾首次登录平台

### 1.2 配置板块

#### 1.2.1 项目信息

**包含字段**：

| 字段 | 形态 | 必填 | 默认值 | 双态 |
|------|------|:-:|------|------|
| 项目名 | 文字输入 | 选填 | **AOM 课包名称** | 全可改 |
| 服务周期 | 起止日期 picker | 选填 | 默认 1 年 | 开营前可改 / 开营后锁定（仅后台改）|
| 项目周期 | 起止日期 picker | 选填 | 默认 = 服务周期 - 1 周 | 全可改（开营后修改弹严格确认）|
| 项目描述 | 多行文本框 | 选填 | 空 | 全可改 |
| 客户名 | 文字输入 | 选填 | 空 / 仅供运营标识 | 全可改 |
| 公司画像 context | **本期不做** · "Coming Soon" 占位 | — | — | — |

**GUI 操作**：
- 单页表单 / 字段从上至下排列
- 修改后需点击 "保存" 按钮才生效（详见 § 1.3.1）
- 项目周期修改时弹出二次确认（涉及合同 / 沿用 [03-lifecycle/02-learning § 1.4]）
- 公司画像 context 字段在界面上**显示为 "Coming Soon" 灰置占位**——作为产品钩子 / 不可输入 / V0.6.0+ 实现

**影响其他模块**：
- 项目名 → [06-management/02-home § 1] 看板顶部 + [06-management/03-report-center § 1] 报告头部 + 全平台项目名展示
- 项目周期 → [06-management/02-home § 1] 看板项目进度卡（已用 X / 总 Y 天）+ 学员端大厅倒计时

#### 1.2.2 人员名单与角色

**人员表格列**：

| 列 | 形态 | 必填 | 说明 |
|----|------|:-:|------|
| 姓名 | 文字输入 | ✓ | — |
| **账号** | 文字输入（自动生成 + 可改）| ✓ | 默认 = **姓氏全拼 + 名字首字母**（如"王小明" → "wangxm"）/ 重复时自动加括号备注（如 "wangxm(2)"）|
| 邮箱 | 文字输入 | 选填 | — |
| 联系方式 | 文字输入 | 选填 | — |
| 角色 | 多选 checkbox（学员 / 管理员）| ✓ | 默认仅学员 / 同一人可同时勾两个 |
| 行末操作 | 按钮组 | — | **重置密码** + **删除** |

**GUI 操作**：
- 表格主体 + 顶部 **"+ 添加人员"** / **"批量导入"** 按钮
- **批量导入**：点 "批量导入" → 选 CSV / Excel → 上传 → 预览（含校验提示 / 账号自动生成 / 重复检测）→ 确认导入
- **手动添加**：点 "+ 添加人员" → 弹表单（姓名必填 / 账号自动生成可改 / 邮箱+联系方式选填）→ 默认勾学员角色 → 确认
- **角色分配**：行内 2 个 checkbox（学员 / 管理员）/ 多选
- **重置密码**：行末 "重置密码" 按钮 → 二次确认 → 该用户密码重置为当前默认密码（详见下方"默认密码设置"）
- **删除**：行末删除按钮 + 二次确认（开营后离职场景需严格确认）
- 修改后需点 "保存" 按钮生效

**默认密码设置**（人员配置最后一行 / 临时功能）：

管理员可查看 / 重新设置项目级默认密码——所有新导入用户与"重置密码"动作均使用此默认密码。

| 动作 | 触发 | 说明 |
|------|------|------|
| **查看** | 点击"查看"按钮 → 弹密码可见框 | 显示当前默认密码（明文 / 仅本会话可见）|
| **更改** | 点击"更改"按钮 → 弹输入框 | 输入新密码 → 二次确认 → 替换默认密码（已存在用户密码不自动同步）|
| **复制** | 点击"复制"按钮 | 默认密码复制到剪贴板（便于线下分发给学员）|

**双态可改性**：
- 开营前：全可改（增删人员 / 改角色 / 改账号 / 改默认密码）
- 开营后（沿用 [03-lifecycle/02-learning § 1.5] 学员名单变更 SOP）：
  - **总账号数锁定**:合同签单时锁定 / 学习期内不可增加（"+ 添加人员"按钮灰置）
  - **未首次登录账号 ✅ 可换人**:可删除 + 增加同等数量替换（替换离职 / 新员工等场景）
  - **已首次登录账号 ❌ 不可换**:保留学员学习数据 / 删除按钮禁用
  - 默认密码 + 重置密码 仍可用

**影响其他模块**：
- 学员名单 → [06-management/02-home § 1.2.2] 学员维度看板 + [06-management/03-report-center § 1.2.3] Tab 2 学员报告 + [06-management/05-message § 1] 消息中心收件人范围
- 管理员角色 → 谁能登录管理端 / 看谁的报告库（沿用 [06-management/03-report-center § 1.6.1] 每管理员独立报告库）
- 学员 + 管理员同一人 → 顶部端口切换器（沿用 [04-global/01-user-global § 1.5]）
- 账号 + 默认密码 → [04-global/01-user-global § 1] 登录态 / 全平台账号体系

#### 1.2.3 里程碑配置

**包含设置项**（沿用 [03-lifecycle/02-learning § 1.3]）：

| 类型 | 数量约束 | 配置项 |
|------|--------|------|
| **阶段里程碑** | 仅 2 种 | 名称 + 时间 / **结营**（必选 / 不可删 / 可改名 + 调时间）+ **中期**（可选）|
| **自定义里程碑** | **上限 10 个** | 名称 + 时间 + **关联内容进度（可选）** |

**关联内容进度**（自定义里程碑专属配置）：

- 管理员可从项目内 Activity 列表中勾选**该里程碑之前必须完成的 Activity 子集**——表示"到达此里程碑前，学员应完成这些 Activity"
- **可选配置**：不勾 = 仅时间触发 / 不与内容关联
- 勾选效果：里程碑到期时 Ora / 看板可参考"未完成关联内容学员"作为风险信号

**GUI 操作**：
- 列表主体 + 顶部 "+ 添加里程碑" 按钮
- 阶段里程碑：默认存在 2 行 / 点击编辑名称 + 时间
- 自定义里程碑：点 "+ 添加里程碑" → 输入名称 + 时间 → 可选 "关联内容进度" 弹出 Activity 树（按 Course 分组 / 多选 checkbox / 默认未勾）→ 提交（达上限后按钮 disabled）
- 编辑：点击行 → 进入编辑态 / 可改名 + 时间 + 关联内容
- 删除：行末删除按钮（结营不可删）
- 修改后需点 "保存" 按钮生效

**约束**：里程碑只能在项目周期内 / 同一天可有多个 / 不设禁用

**双态可改性**：开营前后均可改

**影响其他模块**：
- 学员端：大厅倒计时 + 项目进度区里程碑标记 + 到期产品化推送（沿用 [03-lifecycle/02-learning § 1.3]）
- [06-management/03-report-center § 1] 报告中心：未来 milestone 自动触发 team report 草稿生成（V0.6.0+ / 本期不做 / 沿用 [06-management/03-report-center § 1.7]）
- § 1.2.4 催学规则：可被催学规则"里程碑绑定"触发（详细联动规则详见 § 1.2.4）
- 关联内容进度 → 里程碑到期时的"未完成关联内容学员"作为 [06-management/02-home § 1] 看板风险信号 / Ora 钻取上下文

#### 1.2.4 催学规则

**包含设置项**（沿用 [03-lifecycle/02-learning § 1.5]）：

| 设置项 | 形态 | 默认值 |
|------|------|------|
| **脱训阈值** | 数字输入（天）| 连续 **7 天**未登录 |
| **进度落后阈值** | 数字输入（M × 倍数）| **M × 0.80**（即落后群体均值 20%）|
| **进度领先阈值** | 数字输入（M × 倍数）| **M × 1.20**（即领先群体均值 20% · 不触发催学 / 仅用于 [06-management/02-home § 1] 首页学员状态显示分类）|
| **触发类型** | 多选 checkbox | 里程碑绑定 / 脱训信号 / 进度落后信号 |
| **渠道** | 锁定 | 本期仅站内消息（邮件 / IM / SMS 后续逐步加）|
| **文案模板** | 富文本输入框 + 变量占位符 | 默认基础文案 / 支持变量（学员名 / 进度 / 等）|

**GUI 操作**：
- 单页表单 / 各设置项分区呈现
- 阈值输入：脱训"X 天" / 进度落后 + 进度领先"M × X 倍"
- 触发类型：多选 checkbox
- 文案模板：富文本编辑器 + 变量插入按钮（点击插入 ${学员名} / ${进度} 等占位符）
- 修改后需点 "保存" 按钮生效

**双态可改性**：开营前后均可改

**影响其他模块**（**配置即源 / 多处使用**）：
- [06-management/02-home § 1.2.1 (3)] 登录状态分布卡：3 chip（活跃 / 风险 / 沉默）依本节"脱训阈值"配置
- [06-management/02-home § 1.2.2 (2)] 学习进度状态：3 chip（领先 / 跟上 / 落后）依本节"进度落后 + 进度领先阈值"配置
- [06-management/05-message § 1] 消息中心：催学规则触发的消息进入 [06-management/05-message § 1] 消息中心投递日志

**与里程碑联动**：触发类型勾"里程碑绑定" → 里程碑到期当天自动发催学消息

#### 1.2.5 平台个性化

**包含设置项**（沿用 [03-lifecycle/02-learning § 1.6]）：

| 设置项 | 形态 | 默认值 | 影响 |
|------|------|------|------|
| Logo | 上传图片（PNG / SVG）| 默认睿学品牌 Logo | 全平台头部 + 登录页 + 报告导出头部 |
| 欢迎语 | 多行文本框 | 默认通用欢迎语 | 学员端大厅首屏欢迎语 + 管理端登录后欢迎提示 |
| Neo 名字 | 文字输入 | 默认 "Neo" | 学员端大厅 + 3 学习场域 Neo Chat 显示名 + Neo 自我称谓 |
| Ora 名字 | 文字输入 | 默认 "Ora" | 管理端 [06-management/02-home § 1] Home Ora chat + [06-management/03-report-center § 1] Report Ora chat 显示名 + Ora 自我称谓 |

**GUI 操作**：
- 单页表单 / 各设置项分区呈现
- Logo：点击上传 → 文件选择器 → 预览缩略图
- 欢迎语 / Neo·Ora 名字：文字输入框
- 修改后需点 "保存" 按钮生效

**不可改 / 合规底线**（沿用 [03-lifecycle/02-learning § 1.6]）：
- "powered by NeoLearning" 品牌声明（页脚 / 关键位置保留）
- 大模型安全声明（如"内容由 NeoLearning AI 提供，使用前请核对"）
- **品牌色**：本期统一使用睿学品牌色 / 不开放配置
- **项目名称**：在 § 1.2.1 项目信息配置 / 本节不重复

**双态可改性**：开营前后均可改

#### 1.2.6 内容预览

**仅查看 / 不算配置项**——浏览本项目绑定的 Course / Activity 内容。

**GUI 操作**：
- 单页流式分区内嵌入**可折叠展开的树状目录**（按 Course → Activity 层级 / 沿用 [05-learner/02-hub § 1.2.2] 内容目录区视觉同源）
- 点击 Activity → 弹出 modal 预览内容文本（仅查看 / 不进入独立页）
- **不支持编辑**（内容由 KGP 在 AOM 中编排 / Config 仅查看）

**双态可改性**：开营前后均仅查看

**影响其他模块**：无（只读）

### 1.3 系统约束

#### 1.3.1 保存生效

- 所有配置板块的修改 → **点击 "保存" 按钮才生效**
- 非自动保存（区别于 [06-management/03-report-center § 1.3] 报告编辑的纯自动保存机制）
- 未保存时**在本页跳锚点 / 离开本页** → 弹出二次确认（"有未保存的修改，确定离开？"）

#### 1.3.2 多管理员场景

- admin / 管理员（HR + 项目运营）**都可改 Config**（不做权限差异化 / 本期）
- 不支持多管理员同时编辑同一字段（沿用 [06-management/03-report-center § 1.6.1] 不做实时协同思路）
- 同一字段被多管理员先后修改 → 后保存的覆盖前保存的（无版本冲突提示）

### 1.4 本期不做

| 项 | 说明 |
|----|------|
| 字段差异化权限（HR vs 运营）| 本期 admin / 管理员（HR + 项目运营）同权限 / 未来按客户反馈迭代（沿用 [06-management/01-overview § 1.1.5]）|
| 配置版本 / 回滚 / 操作历史 UI | 沿用 [03-lifecycle/02-learning § 1.4] 变更操作日志本期不做 |
| 学员分组 / 学员标签 / 入组 | 沿用 [03-lifecycle/01-pre-learning § 1.5] |
| 自定义域名 / 字体 / 证书 / 徽章 / Neo 形象企业定制 | 沿用 [03-lifecycle/02-learning § 1.6] |
| admin 在 Config 预制更多报告模板 | V0.6.0+ / 沿用 [06-management/03-report-center § 1.7] |
| 邮件 / IM / SMS 催学渠道 | 本期仅站内消息 / 沿用 [03-lifecycle/02-learning § 1.5] |
| 多管理员实时同编 / 字段冲突提示 | 详见 § 1.3.2 |
| Token 用量统计 UI 开关 | 后台默认开启 / 不做 UI / 沿用 [03-lifecycle/01-pre-learning § 1.5] |
| **6 项 UI 细节待详细设计阶段决策** | 里程碑独立 tab vs 嵌入"项目信息" / 里程碑日历视图 vs 列表视图 / 自定义里程碑表单复杂度 / 模板启用是否自动批量插入 / 默认结营改名改日期 UI / 里程碑编辑二次确认 |

### 1.5 GUI 布局

**Config 主页**（顶部锚点 + 单页流式 / 不做二级导航）：

```

│ ★Cfg │ │ # 项目信息                                      │ │
│ Home │ │   [字段表单]                                    │ │
```

**布局规则**：
- 单页流式（不分 tab / 不分二级页）/ 6 配置项自上而下顺序展开
- 顶部横向**锚点条**（粘性 sticky）/ 点击锚点 → 平滑滚动到对应配置项分区
- 每个配置项分区独立 "保存" 按钮（保存仅作用于本分区 / 互不影响）
- 配置项分区之间用分隔线视觉区分

**双态视觉差异**：
- 开营前：左侧导航**仅 Config 高亮 / 其他 3 模块隐藏** + 底部 "确认开营" 按钮
- 开营后：左侧导航 4 模块**全部显示**（Home / Cfg / Rpt / Msg）+ 项目信息中**服务周期**字段灰置 + 人员名单中"+ 添加人员"按钮灰置（总数锁定）+ 删除按钮对**已首次登录账号禁用** / 仅未登录账号可删除并增加同数替换（详见 § 1.2.2 + [03-lifecycle/02-learning § 1.5]）+ 默认密码 + 重置密码仍可用 + 无"确认开营"按钮
