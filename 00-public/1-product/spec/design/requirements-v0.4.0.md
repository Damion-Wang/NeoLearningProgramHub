**版本：** 0.4.0
**日期：** 2026-04-14
**状态：** 基于 v0.3.3 + 0414 创始人会议（15 项更新）整合。本版本拆分为统领文档（本文件）+ 13 个模块子文档，统领文档仅做总纲，模块详细需求见各子文档。

## 1. 产品定义与愿景

AI 老师是面向企业培训场景的 AI 原生学习陪伴系统。通过"辅导大厅 + 多个场域"的空间模型，让 30-40 名基层管理者在 6 个月周期内，获得 1v1 AI 教练的持续陪伴、结构化课程的自适应学习、沉浸式对练的行为训练，最终产出可举证的行为改变报告。

**核心比喻**：平台是钢琴，内容是乐谱，AI 是演奏者。更换乐谱（AOM 内容包），就能交付完全不同主题的培训项目。

**核心价值**：
- 对学员：有一个懂你的教练随时在身边，陪伴式辅导 + 个性化学习
- 对 HR：基于全过程数据的行为改变证据报告，可直接向 CEO 汇报
- 对运营：系统自动执行可配置的运营规则，一键生产报告

## 2. 角色与端口

| 角色 | 归属 | 核心诉求 |
|------|------|---------|
| 学员 | 客户企业 | 解决真实工作问题，完成培训任务 |
| HR | 客户企业 | 向 CEO 汇报培训成果，验证投入产出 |
| 运营 | 供应商 | 高效管理项目全程，保证交付质量 |
| KGP | 供应商 | 生产高质量课程内容包（不是软件端口） |

**v0.4.0 变更**：运营端与 HR 端本期界面与功能完全一致，两个独立端口但功能权限相同，未来再区分。

## 3. 项目生命周期

```
线下售前 → POC → 签单 → [产品覆盖开始]
  → 开营选人 → 任务下发 → 线上学习（6个月持续）
  → 实时线上辅导 → HR 随时查报告 → 线下集训（人工）
  → 运营写成果 → 结项报告 → [产品覆盖结束]
```

- 单项目学员数：30-40 人 | HR：1-2 个 | 运营：1-2 个
- 第一个内容主题：基层管理者（8 个 Course）
- **v0.4.0 变更**：内容包由内部团队预先配置，管理员配置项目周期（日期选择，1个月~1年）、导入名单、分配角色（学员/HR/运营）

## 4. 三端架构总览

### 4.1 三端合一 Web 应用 [ADR-001]

学员端、HR 端、运营端是同一个 Web 应用，通过权限控制不同角色看到的入口和功能。

### 4.2 顶部通用栏

所有页面顶部有一个通用栏，三端统一存在，包含两个可见元素：

| 元素 | 功能 |
|------|------|
| **提醒按钮🔔** | 点击展开消息列表面板（未读角标 + 消息列表 + 点击跳转） |
| **账号头像👤** | 点击展开下拉菜单：个人信息 / 切换端口（学员↔HR↔运营） / 设置 / 帮助 / 反馈（本期为空） / 退出登录 |

通用栏是纯 GUI 交互，不涉及 Agent。通用栏只提供个人级操作入口；学员管理（名单导入）、催学规则配置等运营操作仍在管理端看板页中。

### 4.3 三栏统一布局 [ADR-007]

三端统一布局框架（目录区 + 看板区 + 对话区），具体比例与交互规范见 UI 设计文档。

**看板与对话双向联动** [ADR-010]：看板元素点击触发 Agent 解读；Agent 提到的实体在看板中高亮/跳转。

## 5. 数据模型（AOM 五层）

```
Project（培训项目）
  └── Course（课程/模块，共 8 个）
       └── Activity（活动：授课/对练/调研/报告，15-25分钟）
            └── SCO（最小学习内容单元）
                 └── Segment（SCO 的版本，用于自适应切换）
```

**SCO 类型**：`NARRATION`（PPT讲解）、`VIDEO`（案例视频）、`QUIZ`（互动测验）、`QA`（小解答）、`FEEDBACK_COLLECT`（闭环-课前采集）、`FEEDBACK_REVIEW`（闭环-课后复盘）、`ASSESSMENT_TAG`（打标判定，纯后台）

---

## 5.5 角色架构：Leo（Coach）与 Neo（Tutor）

学员端有两个独立的 AI 角色：

| 角色 | 前台名称 | 类型 | 作用位置 | 知识范围 |
|------|---------|------|---------|---------|
| **Leo** | Leo | Coach（教练） | 大厅（课堂外） | Program 全量知识 |
| **Neo** | Neo | Tutor（老师） | 场域（课堂内） | Activity 内 + 已完成 Activity + 学员画像 |

**核心区分**：Leo 做全局的、长期的、跨课程的陪伴辅导；Neo 做即时的、Activity 内的、教学驱动的结构化教学。两者记忆打通（共享统一 Database），但引用时 Leo 不提 Neo，Neo 不提 Leo。

**对练场域特殊架构**：对练场域中还有独立的演员 Actor（由剧本驱动的角色扮演对象，如"张总"），与 Neo 分开对话窗口（左栏模拟对话+右栏 Neo 辅导）。Neo 在对练中承担情景导入、新手辅导、反思复盘三个任务。

**管理端 Agent**：管理端右侧的 Agent 与 Leo/Neo 是完全独立的角色系统，本期未命名。

## 7. 跨模块系统

### 7.1 内容驱动 [ADR-004]

平台是引擎，内容是燃料。所有教学行为（AI 何时出题、如何评判、情绪天花板、结束条件）由 AOM 脚本 / 对练剧本 / 调研剧本控制，平台不硬编码。

**v0.4.0 变更**：SCO 间互动由 AI 根据内容逻辑和学员状态自主决定是否插入——不与时间绑定（"每5分钟一次"太机械），而与内容逻辑绑定（"讲完一个完整知识点模块后，如果偏抽象理论，就插互动"）。AI 自主出题场景放在答疑模块，不新增 SCO。

### 7.2 记忆工程 [ADR-005]

三级记忆管道，本期全做：
1. **课堂内采集**：授课/对练/调研中的所有对话 + Quiz 回答 + 行为观察 + 被动信号（停留/回翻/点击）
2. **大厅辅导引用**：Leo（Coach）读取课堂数据，跨 Course 记忆打通
3. **管理端汇聚**：报告智能体拉取全链路数据生成证据报告

统一 Database，三端按权限读取。半结构化记录（含 `passive_signals` 字段）。

### 7.3 自适应体系

| 层级 | 机制 | 本期 |
|------|------|------|
| L1 | 每次对话交互，LLM 实时调整语言/深度/举例 | 做 |
| L2 | `ASSESSMENT_TAG` 打标 + Segment 版本切换 | 做 |
| L3 | Activity 顺序调整 | 未来 |
| L4 | 动态内容生成 | 未来 |

### 7.4 闭环反馈

`FEEDBACK_COLLECT`（课前采集）→ 授课中 `referenceSlots` 引用 → `FEEDBACK_REVIEW`（课后复盘）→ 辅导引用 → 报告呈现。通过 `memory_id` 关联，数据链路完整可追溯。

### 7.5 AI 质量控制

本期做基础版本：后台查找溯源 + 技术调试后门。不做复杂四层架构。AI 质量数据不放学员端/HR端。

### 7.6 举证机制

路径个性化，评估维度标准化。AI 双轨运行（前台教学不受后台采集干扰）。报告按能力维度汇聚全模块证据，生成行为改变证据链。

## 8. 核心设计哲学

1. **AI 不做"判官"**——所有学员输入都有价值，不判断真假
2. **内容驱动行为**——平台不硬编码教学逻辑，由脚本控制
3. **看板与对话联动**——GUI 和 LUI 双向协作
4. **LUI 优先于 GUI**——优先用对话式交互
5. **SCO 锁定机制**（v0.4.0 新增）——未播放到的 SCO 为锁定状态，学员只能回顾已学完内容
6. **防挂机**（v0.4.0 新增）——每 5 分钟进行一次提醒或暂停

## 9. 六月版本 Scope

### P0 必做

- 学员端：大厅（三栏+辅导+热力图+工具库逐步解锁）、授课（含 SCO 锁定+防挂机）、对练（三种变体+重练）、报告（三部分）、笔记（悬浮窗）
- 管理端：HR 端 + 运营端（本期完全一致的两个端口）、进度/互动排名、催学规则、报告工厂
- 通用：账号管理、消息提醒、系统设置（AI语速+提醒+个性化+Token统计）
- 跨模块：记忆三级管道、L1+L2 自适应、闭环反馈、举证机制、AI 质量基础版
- 内容：基层管理者课包（8 Course）

### P1 可选

- 调研模块（三阶段，含BEI——由内容决定是否使用）、企业微信/钉钉催学渠道

### V2+ 不做

- PPT 生成、临时任务下发、独立运营 Agent、L3/L4 自适应、多人对练、新模块类型（沙盘/研讨/播客）、企业知识库、动态新增 Quiz、辅导风格按课程调整
