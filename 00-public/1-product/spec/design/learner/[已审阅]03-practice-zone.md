## 模块定位

对练场域是基于场景的角色扮演训练空间——演员（Actor）模拟真实人物与学员进行沉浸式对话练习，Neo（Tutor）在旁提供答疑与辅导。核心目的是能力评估和行为训练而非知识传递。

---

## 对练场域中的三个角色

| 角色 | 前台呈现 | 职责 | 驱动方式 |
|------|---------|------|---------|
| **演员（Actor）** | 具体人物（如"张总"） | 角色扮演对话——按剧本的性格/目标/触发点/底线进行表演 | 剧本驱动 |
| **Neo（Tutor）** | Neo | 三个任务：①铺垫引导 ②演练答疑与辅导 ③复盘追问 | Neo 自身判断（以剧本和导演指令为上下文） |
| **导演（后台）** | 不暴露给学员 | 分析学员每轮发言，决定 Actor 的场景走向和情绪变化 | 剧本驱动 |

### Neo 在对练场域的角色定位

**前台名称**：Neo
**角色类型**：Tutor（老师），在对练场域承担三个任务
**知识范围**：Activity 内 + 已完成 Activity + 学员画像

| 任务 | 阶段 | 说明 |
|------|------|------|
| **铺垫引导** | 情景导入 | 引导学员了解场景、角色、任务和可用工具 |
| **答疑与辅导** | 角色扮演 | 主动发现知识应用卡点给出建议 + 被动回应学员提问 |
| **复盘追问** | 反思复盘 | 苏格拉底式追问帮助学员反思关键决策点 |

### Neo 各阶段的行为边界（能做什么 vs 不能做什么）

**情景导入阶段**

| 能做 | 不能做 |
|------|--------|
| 解释场景背景、角色关系、任务目标 | 透露剧本中不允许透露的内容（Actor 的心理画像、软化点、底线、隐藏目标等） |
| 提示可用的工具和知识点 | 告诉学员"正确答案"或最优策略 |
| 回答学员关于规则和流程的疑问 | 预判对练走向（"对方一开始会很强硬"） |
| 基于学员提问回顾之前学过的知识点 | |

**角色扮演阶段**

| 能做 | 不能做 |
|------|--------|
| 给方向提示（"试试追问对方的 Interest"） | 直接给答案（"你应该说..."） |
| 推测后果（"这样说可能会让对方更防御"） | 直接评价表现（"你这句说得不好"） |
| 站在 Tutor 角度推测对方可能的反应（"如果你换一种方式，对方可能会..."） | 透露 Actor 将要做什么（"对方下一步会让步"） |
| 回应学员主动提问 | 替学员思考或做决定 |
| 基于学员提问回顾之前学过的知识点 | |

**反思复盘阶段**

| 能做 | 不能做 |
|------|--------|
| 苏格拉底式追问引导学员自己发现问题 | 只追问不总结——必须在追问后给出明确总结 |
| 给出明确的总结和待改进项陈述 | 只说"你觉得呢"然后留在开放状态不收尾 |
| 引用具体对话轮次做定向分析 | 泛泛而谈不引用具体行为（"你整体还行"） |
| 对比多次练习的表现差异 | 只看最后一次，忽略之前的练习数据 |

### Neo 在对练场域的核心能力

| 能力 | 说明 |
|------|------|
| **答疑与辅导** | 演练中主动+被动辅导，发现知识应用卡点给出方向提示（以剧本和导演指令为上下文） |
| **EvaluationSkill** | 对练行为评估、能力维度标注、生成对练小结 |
| **反思复盘引导** | Actor 退出后苏格拉底式追问，对照剧本成功标准帮助学员发现改进空间 |
| **InsightCaptureSkill（轻量）** | 捕捉对练中的行为证据碎片和高光瞬间，写入统一 Database |

### 演员（Actor）角色说明

演员是独立于 Neo 的角色，学员在对练中看到的对话对象（如"张总""客户王经理"等）就是演员。

| 能力 | 说明 |
|------|------|
| **角色人设执行** | 按剧本定义的性格、目标、触发点、软化点、底线进行表演 |
| **情绪状态管理** | 学员沟通有效则缓和、粗暴则升级，受情绪天花板约束（天花板由剧本定义） |
| **防剧透** | 绝不主动提出剧本中标记为学员必须自己想到的方案 |
| **表情变化（P3）** | 左栏头像随情绪状态切换表情，本期可能只有静态头像 |

> **导演**由剧本驱动——剧本定义角色心理画像、行为模式、递进式反应链、成功标准、技能阶梯等控制元素，导演据此分析学员发言并指挥演员表演。学员不知道导演的存在。

| **右栏：Neo 复盘对话** | Neo 苏格拉底式追问，引导学员反思关键决策点 |

## 三种对练变体

三种变体通过不同剧本实现，平台演绎功能不变（UI/交互/架构不区分变体）。Neo 的答疑与辅导适用于所有变体：

| 变体 | 说明 | Neo 答疑与辅导侧重 |
|------|------|-------------------|
| **基础 Roleplay** | 学员扮演目标角色，Actor 扮演对象。最常用形态 | 辅导学员识别和应用目标技能 |
| **角色互换** | Actor 扮演学员角色，学员扮演对方。复盘时切回原身份反思 | 辅导学员理解对方状态，给出方向建议 |
| **费曼** | Actor 扮演什么都不知道的学生，学员当老师讲解。倒逼深度理解 | 辅导学员成为老师讲解知识，提示遗漏 |

## Neo 答疑与辅导机制

演练过程中，Neo 在右栏独立对话窗口提供辅导，**不打断左栏的 Actor 对话**。

### 工作方式

```
Actor 角色扮演在左栏模拟对话区持续进行
        ↓
Neo 在右栏实时观察学员每轮发言
        ↓
发现知识应用卡点时，Neo 在右栏给出辅导建议：
  "你注意到对方刚才说了'排期压力大'吗？
   这背后可能藏着他的 Interest。试试追问一下？"
        ↓
学员自主选择：
  (A) 暂时不理 Neo，继续在左栏和 Actor 对话
  (B) 在右栏和 Neo 聊一下，消化后再回到左栏继续
        ↓
学员也可以随时主动在右栏向 Neo 提问
```

### 辅导原则

| 原则 | 说明 |
|------|------|
| **不打断 Actor 对话** | Neo 的建议出现在右栏，不暂停左栏的模拟对话 |
| **学员自主** | 学员自己决定是否停下来和 Neo 聊，Neo 不强制中断 |
| **卡点驱动（主动）** | 发现知识应用卡点时主动给建议，不做无信号的干预 |
| **随时可问（被动）** | 学员随时可以在右栏向 Neo 提问 |
| **简短精准** | 每次建议 1-2 句话，点到即止，不展开讲解 |
| **不替代思考** | 给方向提示，不给具体答案（"试试追问"而非"你应该说..."） |
| **剧本为上下文** | Neo 的判断以剧本和导演给 Actor 的指令为上下文，不脱离教学目标 |

## 用户故事：张磊完成一次 PIN 法对练

### 场景背景
张磊刚学完 PIN 法概念讲解，进入「PIN 法跨部门协商对练」。Actor 将扮演市场部的"张总"——一个有自己压力和利益的跨部门同事。

### 完整旅程

**阶段一：情景导入（约 2 分钟）**

左栏展示铺垫内容（markdown 格式）：

> **场景**：你是产品部主管李明（手下 8 人，Q2 有 3 个重点项目），市场部总监张总（手下 12 人，筹备年中大促）找你借 3 个销售支援活动。
>
> **你的任务**：运用 PIN 法探索张总的深层需求，达成双赢方案。
> - 分项目标：识别张总的 P/I/N、提出替代方案、保持合作关系
>
> **可用工具**：PIN 法三步走（Pressure → Interest → Need）
>
> **知识点回顾**：PIN 法冰山模型——P 是表面诉求，I 是利益关注，N 是深层需求

右栏 Neo 铺垫引导：
> "接下来你会和市场部的张总进行一次对话练习。先看看左边的背景信息，了解你们各自的处境。准备好了就在左边的对话区点击'开始对练'。有任何疑问随时在这边问我。"

张磊阅读完背景信息后，点击「开始对练」。

**阶段二：角色扮演 + Neo 答疑与辅导（约 12 分钟）**

左栏切换为三区域布局：上方显示张总的头像（静态），中部是刚才的背景信息缩略（可展开回顾），下方是模拟对话区。

Actor（张总）开场白出现在左栏对话区：
> "李明，你有空吗？我有个事得跟你商量一下。下个月的年中大促你知道吧？我这边人手严重不足，想找你借 3 个销售顶一周。你看行不行？"

**第 1 轮** — 张磊在左栏对话区回复："张总，我这边 Q2 指标压力也很大，3 个人走了一周我项目就拖了。"

Actor（张总，语气略带不满）：
> "我知道你忙，谁不忙啊。但这是公司级的大促活动，老板都点了名的。你看能不能想想办法？"

**第 2 轮** — 张磊继续："这个确实不好办，我回去想想吧。"

**右栏 Neo 主动推送辅导建议**（检测到学员连续 2 轮在表面僵持）：
> "前两轮你们都在'冰山表面碰撞'。张总刚才说了'老板点名'，这背后可能有她的 Interest。试试追问'具体缺什么环节的人'？"

张磊看到 Neo 的建议，决定调整策略。

**第 3 轮** — 张磊在左栏对话区回复："张总，我理解这个活动很重要。我想了解一下，你说人手不足，具体缺的是什么环节的人？是现场导购还是后端支持？"

Actor（张总，语气稍缓和——学员开始探索 Interest）：
> "主要是现场。活动当天需要有经验的人在柜台接待 VIP 客户，我自己的团队都是做线上的，没有线下接待经验。"

**第 5 轮** — 张磊在右栏主动问 Neo："她说的'有经验的人在现场'，这算 Interest 还是 Need？"

Neo 回应：
> "这更接近 Need——她的深层需求不是'3个人'，而是'活动现场有线下经验的人坐镇'。你可以试试基于这个 Need 提一个替代方案。"

**第 8 轮** — 张磊在左栏提出方案："我可以派我们最有经验的小王去现场坐镇一天，其他时间远程培训你的团队怎么接待 VIP。"

Actor（张总，满意）：
> "这个方案可以啊！小王确实经验丰富，有他坐镇我就放心了。"

对练结束——剧本判定目标达成。

**阶段三：反思复盘（约 5 分钟）**

Actor 退出，左栏切换为对练小结（实时生成）：

> **目标达成**：✅ 主目标达成
> **高光表现**：第 3 轮开始探索 Interest，从表面僵持转为深入探询
> **待改进**：前 2 轮停留在表面对抗，未主动运用 PIN 法
> **对话回看**：（按轮次展示，第 3 轮标注为"关键转折点"）

右栏 Neo 开始复盘追问：
> "这次对练表现不错。来看看左边的小结——你在第 3 轮改变了策略。你觉得是什么让你在那个时候突然换了一种方式？"

张磊反思回答后，Neo 继续追问关键节点，帮助张磊深化理解。

### 重练场景

张磊对前两轮不满意，点击「重新练习」。

- 跳过情景导入，直接从 Actor 开场白开始
- 右栏 Neo 保留上一轮的辅导记录，新一轮继续累加
- 上一次对练数据完整保留，不被覆盖
- 复盘时 Neo 对比两次练习："有意思——你第一次花了 3 轮才开始探索 Interest，第二次第 1 轮就主动追问了。"

## 核心交互清单

### 1. 情景导入

| 项目 | 内容 |
|------|------|
| **触发** | 学员进入对练 Activity |
| **响应** | 左栏展示铺垫 markdown（背景/角色/任务/工具/知识点），右栏 Neo 引导说明，底部显示「开始对练」按钮 |
| **联动** | 如果对练需要个性化版本匹配，导学阶段内置 ASSESSMENT_TAG SCO 先完成打标（与授课场域的 separateTag/groupKey 是同一套 L2 自适应标签机制） |

### 2. 角色扮演开始

| 项目 | 内容 |
|------|------|
| **触发** | 学员点击「开始对练」 |
| **响应** | 左栏切换为三区域（演员头像+背景缩略+模拟对话），Actor 直接说出开场白（无旁白），右栏 Neo 进入答疑与辅导状态 |
| **联动** | Actor 行为由剧本+导演控制；两栏各有独立输入框 |

### 3. Actor 情绪变化

| 项目 | 内容 |
|------|------|
| **触发** | 学员的沟通方式影响 Actor 情绪 |
| **响应** | 有效沟通→情绪缓和；忽视/粗暴→情绪升级。情绪天花板由剧本定义 |
| **联动** | P3：头像表情随情绪切换 |

### 4. Neo 答疑与辅导

| 项目 | 内容 |
|------|------|
| **触发** | 主动：Neo 检测到知识应用卡点；被动：学员在右栏向 Neo 提问 |
| **响应** | Neo 在右栏给出辅导建议或回答问题，**不打断左栏 Actor 对话** |
| **联动** | 学员自主选择是否停下 Actor 对话来和 Neo 交流。辅导记录写入评估数据 |

### 5. 对练结束 → 复盘

| 项目 | 内容 |
|------|------|
| **触发** | Roleplay 部分由剧本定义的结束条件触发（轮数上限/目标达成/死循环检测）；整个 Activity 结束 = 复盘阶段学员确认完毕。具体结束条件待开发阶段参照 SIDE 项目确认 |
| **响应** | Actor 退出，左栏切换为对练小结（实时生成），右栏 Neo 开始复盘追问 |
| **联动** | 对练小结包含：目标达成+高光+待改进+对话回看+关键节点标注 |

### 6. 对话回看分析

| 项目 | 内容 |
|------|------|
| **触发** | 复盘阶段，学员点击左栏对练小结中的某一轮对话 |
| **响应** | 右栏 Neo 自动对该轮进行定向分析（无需手动输入） |
| **联动** | Neo 分析中引用剧本的成功标准和技能阶梯 |

### 7. 重新练习

| 项目 | 内容 |
|------|------|
| **触发** | 学员点击「重新练习」按钮 |
| **响应** | 仅重练角色扮演阶段，跳过情景导入，从 Actor 开场白重新开始 |
| **联动** | 原练习数据完整保留不覆盖；右栏 Neo 辅导记录保留，新一轮继续累加。复盘时可对比多次练习的表现差异 |

### 8. 角色互换复盘切换

| 项目 | 内容 |
|------|------|
| **触发** | 角色互换变体中，演绎结束进入复盘 |
| **响应** | Neo 引导学员切回原始身份进行反思："你刚才扮演客户时是怎么想的？" |
| **联动** | 对练小结展示互换后的对话记录 |

### 9. 断点续播

| 项目 | 内容 |
|------|------|
| **触发** | 学员中途退出对练场域 |
| **响应** | 系统记录左栏 Actor 对话位置和右栏 Neo 辅导上下文。再次进入时左右两栏均从断点恢复 |
| **联动** | 大厅课程目录中该 Activity 显示为 ▶ 学习中状态 |

## 已确认决策

- [ADR-004] 演员（Actor）行为完全由剧本控制（情绪天花板由剧本定义）；Neo 答疑与辅导基于自身判断（以剧本和导演指令为上下文）
- 对练场域三个角色：演员 Actor（角色扮演）+ Neo（引导/辅导/复盘）+ 导演（后台，剧本驱动）
