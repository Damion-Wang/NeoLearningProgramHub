## 一句话定位

HR 端 + 运营端本期界面与功能完全一致，2 个独立端口入口同 1 套界面。看板 + 报告 + 消息 + 配置 4 个一级入口。

---

## 功能结构

```
A. 定位与端口
   A1 HR / 运营本期完全一致（2 端口入口，同 1 套界面）
   A2 通过 Infra 的端口切换进入（学员 ↔ HR ↔ 运营）
   A3 未来再分化 HR / 运营差异化功能

B. 顶级导航 ★v0.4.0 精简为 4 项
   B1 首页（看板）
   B2 报告
   B3 消息
   B4 配置

C. 首页 — 内部 Tab 切换 ★v0.4.0 新结构
   C1 总览 Tab（项目整体完成率 + 需关注学员 Top 3）
   C2 学员管理 Tab（列表 / 状态 / 进度 GUI）
   C3 进度看板 Tab（按 Course / 按学员维度，Activity 级粒度）
   C4 数据展示 Tab
       C4.1 学习进度 / 学习效果（P2）/ 活跃度 / 完课率
       C4.2 被动信号聚合（停留 / 中途离开率 / 回翻率）
       C4.3 数据更新时间标注
       C4.4 ★不展示能力维度排名（本期限制）
   C5 催学管理 Tab
       C5.1 规则配置（触发条件 + 消息模板 + 执行策略）
       C5.2 自动执行 + 渠道（产品内消息 + Email 必做；企微 / 钉钉可选）
       C5.3 重要人工确认（升级规则触发时运营审核）
       C5.4 效果漏斗（发送 → 查看 → 登录 → 完成）

D. 报告页
   D1 个人学习报告（同学员版 + HR 可见分组标签）
   D2 项目整体报告（汇总分析 + 各 Course 完成率分布）
   D3 进度排名（Activity 完成数，可排序可筛选）
   D4 互动排名（token 量，含计算说明）
   D5 C5 分组标签（基于阈值规则，非智能评价，学员不可见）
   D6 报告工厂
       D6.1 选择类型 / 范围 / 模板（统一模板库，HR Agent 与工厂共享）
       D6.2 批量生成（后台并行，进度展示）
       D6.3 审核队列（逐份 + 批量操作 + 失败重试）
       D6.4 批量导出 PDF

E. 消息页 ★v0.4.0 独立一级入口
   E1 消息列表（平台 / 系统催学 / 运营群发三类）
   E2 下发与群发能力（运营 / HR 给学员发消息）
   E3 与 Topbar 铃铛的关系：铃铛 = 我收到的 / 消息页 = 我要发的（待确认）
   E4 ★多渠道配置 [P1，req §9] [补漏]
       E4.1 渠道启用开关（企微 / 钉钉 / 短信，按客户企业 IT 能力决定）
       E4.2 OAuth 鉴权配置（企业应用接入）
       E4.3 消息格式适配（每渠道模板 + 长度限制 + 富文本支持）
       E4.4 渠道优先级与降级（主渠道失败时回退 Email / 产品内）
       E4.5 分渠道效果追踪（接入 C5.4 催学漏斗）

F. 配置页（纯 GUI 无 Agent）
   F1 项目信息（名称 / 周期 / 课包绑定）
   F2 学员导入 GUI（上传 Excel/CSV + 预览 + Email 通知）
       → 调用 Infra 账号批量创建
   F3 角色分配 GUI（学员 / HR / 运营）
       → 调用 Infra 角色授权
   F4 催学规则初始配置（与 C5.1 共享规则库）
   F5 平台个性化（客户企业 Logo / AI 老师形象 / 音色）
   F6 内容包完整性校验入口 → 跳 ContentMgt

G. 管理端 Agent（HR / 运营共享，本期同 1 套）
   G1 自然语言查询（项目 / 学员 / 数据）
       G1.1 时间戳标注（"数据截至 YYYY-MM-DD HH:MM"）
       G1.2 能力边界（无法回答时坦诚告知）
   G2 生成汇报报告（HR 侧核心场景）
       G2.1 需求确认（模板 / 重点维度 / 风格）
       G2.2 模板手动复用（本期不做跨会话自动记忆）
       G2.3 在线预览 + 修改迭代
       G2.4 PDF 导出
   G3 解读数据（三层钻取：结论 → 证据摘要 → 原始对话）
   G4 辅助报告审核（与 D6.3 同一能力）
   G5 全周期可用（学习期随时查 + 结项期生成汇报）
   G6 配置页无 Agent（仅首页 / 报告 / 消息有）

H. 数据粒度与隔离
   H1 Activity 级粒度（不做 SCO 级）
   H2 数据时间标注（看板 + Agent 回答）
   H3 学员端 / HR 端排名 API 级隔离（见 Report F4）

附录 Z. 本期不做
   Z1 独立运营 Agent（与 HR Agent 共享）
   Z2 智能预警评价（先呈现数据，HR / 运营自行判断）
   Z3 能力维度排名（可信度 / 可比性本期无法保证）
   Z4 CEO 汇报 PPT 生成（先做好报告格式）
   Z5 学员分组分析（30-40 人无分组需求）
   Z6 临时任务下发（运营通过企微等渠道完成）
   Z7 学员自注册（所有账号运营导入创建）
   Z8 多语言
```

## 与 v0.3.3 差异

- ★ **HR + 运营合并**（0.3.3 分上下两部分，v0.4.0 完全一致）
- ★ **顶级导航从 6-7 项精简为 4 项**（首页 / 报告 / 消息 / 配置）
- 学员管理 / 进度 / 数据 / 催学 **下沉为首页内 Tab**（0.3.3 是顶级入口）
- 体验课程入口删除 → 改走 Infra 通用栏端口切换
- 平台个性化（Logo / 形象 / 音色）是 v0.4.0 新增
- 消息独立为一级入口（0.3.3 在首页看板内）
