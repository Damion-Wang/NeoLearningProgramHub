## 一、新增模块（P0 必补）

### 1.1 Discovery Library（发现库）

**定位**：AI 自动识别学员的「亮眼表现」，推送卡片让学员确认收藏。区别于笔记（学员主动记录）和工具（固定资源）。

**补充到 spec 位置**：`learner/00-overview.md` 的"模块总览" + 新建 `learner/07-discovery-library.md`

**关键设计**：
- 触发方式：Neo 在对练结束/授课完成时检测学员的高光行为，推送卡片到 Chat
- 交互：卡片含「接受收藏」/「忽略」两按钮
- 存储结构：`{id, title, source(ActivityID·type), date, accepted}`
- 渲染位置：大厅 4 库 2×2 网格之一
- 价值定位：作为学员自我认知、报告证据来源

### 1.2 Result Library（统一报告库）

**定位**：学员端的统一报告归档，包括学习报告、调研成果报告、结项报告等所有学员可见的报告。

**补充位置**：`learner/00-overview.md` + `learner/05-report-zone.md` 补充「历次报告在大厅 Result 库可回看」

**关键设计**：
- 区分于管理端的团队报告（management/report-gen）
- 结构：按生成时间倒序；每条带标题+日期+类型 chip
- 与调研场域联动：完成调研后成果汇总入库
- 点击进入对应报告页（A5 报告或调研结果页）

## 二、新增跨场域机制（P0 必补）

### 2.1 场域交互范式统一

**补充位置**：`common/00-overview.md` 新增章节"场域层布局范式"

**统一结构**：
```
Topbar（h-14 固定）
  ↓
Sub-bar（h-11 固定）：面包屑 + 右侧状态
  ↓
左侧覆盖式抽屉（默认收起，点击toggle打开，z-index>内容，点击外关闭）
  ↓
中间全幅内容区（根据场域特性设计固定/滚动）
  ↓
右侧 AI Chat 栏（固定宽度约 420px）
```

**抽屉规则**：
- top:56px（不覆盖 Topbar）
- 包含两个 Tab：课程大纲 / 当前场域目录
- 报告场域只有"课程大纲"Tab（报告没有场域目录）

### 2.2 固定 vs 滚动布局规范

**补充位置**：`common/01-general.md` 新增"区域滚动行为"

| 页面 | 主内容区 | 说明 |
|------|---------|------|
| 登录 | 固定全屏 | 不滚动 |
| 大厅 | 可滚动 | Dashboard→课程→4库→社区纵向堆叠 |
| 授课 | 固定沉浸式 | PPT 填充视口，不滚动 |
| 对练 P1 导入 | 整体固定 | 内容自适配 |
| 对练 P2 角色扮演 | 外框固定、内部三列独立滚动 | Actor 背景/对话/Neo 辅导各自滚动 |
| 对练 P3 复盘 | 外框固定、内部滚动 | 小结列表可长 |
| 学员报告 | 可滚动 | Markdown 长文档 |
| 调研 P1 说明 | 固定 | 内容短 |
| 调研 P2 沉浸 | 可滚动 | 对话随消息增加 |
| 调研 P3 确认 | 外框固定、内部滚动 | |
| 管理端页面 | 标题 sticky + 下方滚动 | 页面标题/CTA 固定顶部 |

### 2.3 场域 sub-bar 统一格式

**补充位置**：`common/00-overview.md`

```
[☰ 抽屉 toggle]  大厅 > 课程 > Activity  ............  右侧 状态/进度
```

- "大厅"永远可点击，链到 hall-daily
- 右侧状态：
  - Lecture: `SCO N/12 · PPT`
  - Practice: `进度 3/5 轮 · 进行 MM:SS`
  - Report: 空
  - Inquiry: `阶段 1/3 · 说明`

### 2.4 Activity 完成弹窗统一

**补充位置**：`learner/02-lecture-zone.md`

Activity 完成时触发 modal，含两个按钮：
- 主 CTA [开始下一个任务]（cyan→violet 渐变）→ 根据 AOM 下一节点跳转（对练/报告/下一 Activity）
- 次 CTA [回到辅导] → 大厅

点击任一按钮关闭 modal 并跳转。Modal 可 Esc 关闭。

## 三、UI 范式确立（P1 建议补）

### 3.1 drill-down 范式

**适用场景**：管理端消息模块、管理端报告模块

**规则**：
- 默认显示列表视图（消息记录/报告列表）
- 点击「+新建」按钮 → drill down 到编辑视图
- 编辑视图左上角「← 返回 XX 列表」
- 编辑完提交 → 自动返回列表

### 3.2 管理端页面 sticky 标题

**规则**：管理端所有页面（除首页外）顶部标题区 sticky：
- 页面标题 + 英文副标
- 右上角主 CTA（+新建报告 / +新建消息 / 确认开营）
- 筛选器（如有）

向下滚动时标题保持可见。

### 3.3 AI Chat 面板简化

**补充位置**：`common/00-overview.md` "AI Chat 面板规范"

**Header 结构**：只显示 AI 名字（Leo/Neo/Ora），不显示副标题（如 AI TUTOR · Activity 内 · AOM 脚本 等内部术语）

**消息样式**：Chat 消息不加任何标签（如 MENTOR·OPENING、授课·SCO 06、报告解读），只显示头像+气泡+时间戳

### 3.4 品牌名统一

**补充位置**：`common/00-overview.md` 产品标识

- 主品牌名：**睿学**
- 英文副名：AI TUTOR
- Topbar 显示 "睿学" + 版本号
- 删除所有 "AI 老师" 字样（仅作为早期工作名保留）

## 五、数据状态规范（P1 建议补）

### 5.1 学员状态定义（管理端）

**补充位置**：`management/01-operation.md` "学员列表状态"

| 状态 | 判定标准 | 视觉 |
|------|---------|------|
| 活跃 | 3 天内有登录 | emerald chip |
| 风险 | 3-7 天未登录 | amber chip |
| 沉默 | 超过 7 天未登录 | rose chip |

**关键**：状态判定与进度完全无关，只看登录活跃度。

### 5.2 团队报告类型

**补充位置**：`management/02-report-gen.md`

4 个内置报告类型（chip 显示）：
- **中期报告**（violet）：项目中段生成，学习过程分析
- **阶段报告**（cyan）：每月/每阶段末，进度汇总
- **结项报告**（emerald）：项目结束总结
- **自定义**（gray）：管理员自由命名

### 5.3 催学规则与消息模块独立

**补充位置**：`management/03-project-config.md` 和 `management/04-message.md`

- 催学规则在**配置页**设置（触发条件/模板/频率）
- 消息记录在**消息页**查看（自动消息显示"来自催学规则"标识）
- 两者**不做实时双向联动**（避免产品复杂度）
- 消息页的自动消息带来源标识即可

## 六、新增交互（P1 建议补）

### 6.1 笔记悬浮球跨页同步

**补充位置**：`learner/06-notes.md`

- 场域中记笔记 → 通过 localStorage 写入
- 大厅笔记库自动读取渲染（无需刷新）
- 每条笔记带 `source: Activity·type` 字段
- 按时间倒序

### 6.2 管理员查看学员个人报告

**补充位置**：`management/01-operation.md`

- 人员维度学员列表点"查看报告" → 进入学员个人报告**只读视图**
- 学员端和管理端看到的内容一致（含高光/待改进/行动计划）
- 管理员不可编辑学员报告
- 右上角有"导出 PDF"等管理员专属操作

### 6.3 配置「确认开营」二次确认

**补充位置**：`management/03-project-config.md`

- 点击 [确认开营] → 弹窗「⚠ 开营后配置将锁定·不可撤回」
- 显示项目摘要（学员数/课程数/周期）
- [取消] [确认开营] 两按钮
- 确认后所有表单字段置灰，按钮变「已开营 · YYYY-MM-DD」
