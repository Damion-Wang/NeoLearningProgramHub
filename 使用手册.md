## 常见任务怎么做

### 查阅 D-4.0 已审阅 spec（冻结基线）

```
路径：00-public/product-V-0.2.0-D-4.0/spec/
- learner/XX-name.md             学员端模块（8 个含 07-discovery）
- management/XX-name.md          管理端模块（5 个）
- common/XX-name.md              通用规则（3 个）
- requirements-v0.4.0.md         总纲
- 00-future-iterations.md        迭代计划
```

约定：
- 文件名**已去 `[已审阅]` 前缀**（路径 `product-V-0.2.0-D-4.0/` 即表达已审阅状态）
- 各文件 frontmatter 含 `version: V-0.2.0-D-4.0` `status: reviewed`

### 查阅 D-5.0 在写 spec

```
路径：00-public/product-V-0.2.0-D-5.0/plan/
- 03-v0.5.0-full.md       ★ 主体文档（§1-§9 完成）
- 04-v0.5.0-backlog.md    实施 backlog
- 02-v0.5.0-outline.md    1500 字大纲
- 01-spec-update-plan-2026-04-27.md  元规划
- 05-v0.5.0-section-7-9.md           §9 草稿（§8 已合并到 03-full）
```

### 修改 spec

- **D-4.0 已冻结**，不要直接改 D-4.0 文件
- 新设计走 D-5.0：直接编辑 `product-V-0.2.0-D-5.0/plan/03-v0.5.0-full.md` 对应章节
- 同步更新 `04-v0.5.0-backlog.md` 对应任务

### 看历史决策（"为什么这么设计"）

按时间顺序：
1. `00-public/5-process/journal/` 工作日志（按日期）
2. `00-public/5-process/debates/` 设计辩论（按议题轮次）
3. `00-public/4-decisions/` ADR（架构决策记录，含上下文+方案对比+结论）
4. `00-public/4-decisions/底层建设期-2026-04-26/` 50 题决策清单

### 让 Claude 帮你做事

新对话开始时 Claude 会自动触发 `session-opener` 显示项目状态。

常见指令：
- "**做个 demo**" → Claude 启动原型构建工作流
- "**改 spec 里 XX**" → Claude 直接编辑对应文件
- "**5 角色辩论一下 XXX**" → 触发 `multi-round-debate`
- "**审查 / 4 维审查**" → 触发 `long-doc-review`（长文本审查）
- "**完成了**" → Claude 触发 `completion-gate` 反问 4 件套
- "**推 git**" → Claude 触发 `git-push-guard` 三联检查

约定：
- 复杂任务先让 Claude 写规划 md，确认后再执行
- 大规模修改用并行 Agent
- UI 改动要求浏览器截图证据

## 项目内重要约定

### 命名约定

| 前缀/后缀 | 含义 |
|---------|------|
| `product-V-X.Y.Z-D-N.M/` | 产品文档版本归集目录（产品 V-X.Y.Z 时期的第 N.M 版文档） |
| `XX-name.md` | 模块文件，按数字编号排序 |
| `00-`/`01-`/`02-`... | 一级目录用 `0X-` 前缀排序 |
| `02-temp/` | DM 临时区 |
| `claude-workspace/` | Claude 临时区 |
| `.[name]/` | 隐藏目录（.git/.claude） |

### 路径约定

- `00-public/` 是「正式交付」，进入这里 = 已审阅或当前在编 spec
- `02-temp/` 是 DM 短期暂存（季度归档）
- `claude-workspace/` 是 Claude 行动产出区（行动规划/gap 报告/审查/备份）
- `01-ref/` 是「永久参考」，不删不改不归档

### 版本约定

- **产品版本**：V-0.2.0（用到 6 月交付，未指示前不动）
- **文档版本**：D-4.0（已冻结）/ D-5.0（在写）
- **目录归集**：`00-public/product-V-X.Y.Z-D-N.M/`
- **文件 frontmatter**：`version` `product-version` `doc-version` `status` `last-updated`
- **目录 VERSION.md**：每个版本目录顶层一份
- **历史版本**：不删除，旧版仍在 `00-public/4-decisions/历史/` 或对应版本目录

### 品牌约定

- ✅ 用 **睿学**（中文）/ **NeoLearning**（英文）
- ✅ 本期交付 = **睿学培训项目** / **NeoLearningProgram**
- ❌ 禁用「AI 老师」/「AI TUTOR」/「AI tutor」
- AI 角色：学员端只有 **Neo**（2026-04-26 起 Leo 完全取消）；Ora 保留为管理端独立身份

### 临时区职责约定（2026-04-29 立）

| 区 | 拥有者 | 内容性质 |
|----|--------|---------|
| `02-temp/` | DM | 自己的临时改动、会议笔记、产品思考、未定型笔记 |
| `claude-workspace/` | Claude | 行动规划、gap 报告、备份、审查产出、执行 log |

DM 不需要往 `claude-workspace/` 改内容（但可以读）；Claude 不应往 `02-temp/` 写新文件。

### 新建文件

- 放对位置（参考目录速查表）
- 按命名约定（如新 spec 加 `XX-name.md`）
- 同目录加/更新 `README.md`（说明本目录用途）
- 加 frontmatter（version/status/last-updated）

### 审阅 spec

- D-4.0 已冻结，不再走"加前缀"流程
- 新审阅工作针对 D-5.0：
  - 用 `long-doc-review` skill 跑 4 维审查
  - 审查产出存 `claude-workspace/review/<file>/`
  - PM 决议后由 Claude 自动修复

### 归档已完成的工作

- 重要决策落 `00-public/4-decisions/` ADR

- 季度末 → `02-temp/` 整体归档为 `02-temp/archived-YYYY-Q*/`

## 协作规范（Claude Code）

### 触发 Skill 的关键词

| 你说什么 | Claude 触发什么 |
|---------|---------------|
| 新 session 第一句话 | session-opener（自动） |
| "完成了"/"搞定"/"done" | completion-gate（4 件套反问） |
| "推 git"/"commit"/"提交" | git-push-guard（三联检查） |
| "5 角色辩论"/"做个辩论"/方案分歧 | multi-round-debate |
| "审查 / 4 维审查 / long-doc-review" | long-doc-review |

### 跳过 Skill

- "**先跳过**"/"**emergency**" → 绕过本次强制检查
- 但 Claude 会在下次相同时机再次提醒

### 多人协作

- DM 改 D-5.0 / 02-temp；Claude 改 claude-workspace 与按指令的 spec
- 共同 review：在 `02-temp/` 写 review.md
- 重要决策落 `00-public/4-decisions/` ADR

## 常见问题 FAQ

### Q1：spec 和原型不一致以哪个为准？

按时间：晚的为准。原型构建过程涌现的设计**反向补充**到 spec（章节标 `反向补充：XXX`）。当前 D-5.0 写完后会重新对齐。

### Q2：D-4.0 和 D-5.0 怎么区分？

- D-4.0 = v0.4.0 已审阅 spec + 原型 v1.1，已**冻结**，作为基线
- D-5.0 = v0.5.0 全文重写，**在写**
- D-5.0 完成后 D-4.0 中"未被推翻"的内容继承，"被推翻"部分以 D-5.0 为准
- 推翻清单见 `product-V-0.2.0-D-5.0/plan/03-v0.5.0-full.md` §11

### Q3：D-4.0 已审阅文件可以再改吗？

**不能直接改**。新设计走 D-5.0。如果 D-4.0 有错需要勘误，做最小修复并在 frontmatter 标注 `last-updated`，但不要做新设计。

### Q4：原型怎么跑起来？

```bash
cd 00-public/product-V-0.2.0-D-4.0/prototype && python -m http.server 8765
# 浏览器打开 http://localhost:8765/build/01-login.html
```

### Q5：找不到某个文件？

试试这些：
1. `find . -name "*关键词*"`
2. 看 `00-public/` 各 README.md
3. 问 Claude："找一下关于 XXX 的文件"

### Q6：要存的临时文件放哪？

- DM 自己的笔记 → `02-temp/`
- Claude 的行动规划 / 报告 → `claude-workspace/`
- 不要放项目根目录

### Q7：怎么清理项目？

让 Claude 触发 `git-push-guard`（推 git 前自动跑），或显式说"清理一下项目"。
