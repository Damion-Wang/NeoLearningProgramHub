## 1. 三栏通用框架

### 1.1 信息架构

```
+---------------------------------------------------------------------+
| [顶部导航条] 教室模式: <- 返回辅导 | 当前任务名称 | 进度 3/8        |
|             体验模式: 当前为体验模式 | [返回管理端]                   |
+----------+------------------------------+--------------------------+
| 左侧     | 中间看板区                    | 右侧对话区                |
| 目录区   |                              |                          |
|          |                              |                          |
| 角色+    | 场景动态内容                   | Agent Chat UI            |
| 场景     |                              |                          |
| 驱动     |                              | 消息列表                  |
| 加载     |                              | + 输入框                  |
|          |                              |                          |
+----------+                              +--------------------------+
| 折叠图标  |                              | 拖拽手柄                  |
+----------+------------------------------+--------------------------+
```

**层级关系:**
- 第一层: 顶部导航条（教室模式下可见，大厅模式下隐藏）
- 第二层: 三栏并列布局
  - 左侧目录区: 导航+状态指示
  - 中间看板区: 核心内容展示
  - 右侧对话区: Agent交互

### 1.3 关键交互

#### 折叠/展开
- 触发: 点击折叠图标 或 Ctrl+B / Cmd+B
- 响应: 0.3秒动画，目录区宽度过渡
- 联动: 中+右按当前比例重分配空间
- 约束: 动画中忽略重复操作

#### 宽度拖拽
- 触发: 鼠标移至对话区左边缘，光标变col-resize
- 响应: 实时调整宽度，看板区反向联动
- 约束: 最小宽度保护（任一栏不小于最小值）
- 持久化: session级别保存，跨session重置

#### 看板->对话联动
- 触发: 点击看板中可交互元素
- 响应: 元素短暂高亮 -> Agent 3秒内开始生成解读（限2-3句话）
- 约束: 不可交互区域无响应
- **[F2-1修订] 防抖规则:** 500ms debounce，同一时间窗口内重复点击合并为最后一次；Agent正在生成解读时收到新请求，取消当前生成并启动新请求（后到取消前）

#### 对话->看板联动
- 触发: Agent消息携带实体metadata
- 响应: 看板对应元素高亮5秒后淡出
- 约束: 同一时间最多1个高亮（新高亮触发时旧高亮立即淡出）

## 2. 场景切换动效规范

### 2.1 大厅->教室切换

```
T=0ms     用户点击任务卡片
T=0-300ms 三栏内容同步淡出（opacity 1->0）
          同时: 并行加载目标教室数据
T=300ms   如数据已就绪: 三栏内容淡入（opacity 0->1），比例平滑过渡到目标值
          如数据未就绪: 显示骨架屏，数据到达后替换
T=600ms   过渡完成，顶部悬浮导航条淡入
```

**缓动函数:** ease-in-out
**失败回退:** 数据加载失败时，300ms动画反向回退到大厅状态

### 2.2 教室->大厅切换

```
T=0ms     用户点击"返回辅导"
          系统保存教室进度
T=0-300ms 教室三栏内容淡出 + 顶部导航条淡出
T=300ms   大厅三栏内容淡入，比例恢复15:50:35
          辅导教练恢复 + 必然引用消息
T=600ms   过渡完成
```

### 2.3 教室间直跳

```
T=0ms     用户点击目标Activity / 衔接卡片
          系统保存当前教室断点
T=0-300ms 当前教室内容淡出
T=300ms   目标教室内容淡入
          如教室类型不同: 比例平滑过渡到新默认值
          如类型相同: 保持当前比例
T=600ms   过渡完成
```

> **[F4-2修订] 断点保存策略:** 使用navigator.sendBeacon()保证可靠性，淡出动画期间并行保存不阻塞切换。保存失败时下次登录仍可从上次成功断点恢复（最终一致性），不因保存失败中断切换。

### 2.4 教室内比例切换（对练三阶段）

```
导入(10:55:35) --> 演绎(10:35:55) --> 复盘(10:50:40)

过渡时长: 300ms
缓动: ease-in-out
触发: 阶段切换时自动执行
```

> **[F1-1修订] 比例重置规则:**
> 1. **同类型教室切换**（如PPT->PPT）: 保持用户拖拽比例
> 2. **不同类型/阶段切换**（如导入->演绎，或教室间切换到不同类型）: 重置为目标类型默认比例
> 3. **全局快捷操作**: 提供"恢复默认布局"选项（右键菜单或设置面板中），一键恢复当前场景的系统预设比例

### 2.5 小屏幕适配

| 窗口宽度 | 行为 |
|----------|------|
| >= 1024px | 正常三栏布局 |
| 960-1024px | 非阻断提示条"建议横屏或更大屏幕"，可关闭，不阻断操作 |
| < 960px | 全屏遮罩"请使用电脑访问"，阻断所有操作 |

## 3. 断点恢复弹窗

### 3.1 弹窗设计

```
+-------------------------------------------+
|                                           |
|   上次你在 [教室名 - SCO名] 离开          |
|   离开时间: 2026-04-13 15:30              |
|                                           |
|   +---------------+  +----------------+  |
|   |  继续学习      |  |  回到大厅       |  |
|   |  (主按钮)      |  |  (次按钮)       |  |
|   +---------------+  +----------------+  |
|                                           |
|   量表测评附加: "未完成的测评下次可继续"    |
+-------------------------------------------+
```

### 3.2 时序规则

> **[F5-4修订] 时间阈值:**
> - 离开 < 5分钟: 直接恢复到断点位置，不弹窗
> - 离开 5分钟 ~ 4小时: 弹出恢复弹窗
> - 离开 > 4小时: 弹出恢复弹窗（已覆盖）

1. 登录 -> 读取断点
2. 有教室断点 + 离开>=5分钟 -> 弹出恢复弹窗（不发欢迎消息）
3. 有教室断点 + 离开<5分钟 -> 直接恢复到断点（不弹窗，不发欢迎消息）
4. 用户选择"回到大厅" -> 进大厅 -> **[F4-1修订]** 教练发送引用消息变体："你之前在[教室名]学到了XXX，现在回到大厅，看看接下来要做什么"（非欢迎消息，因为本次session已有交互）
5. 用户选择"继续" -> 进教室 -> 不触发欢迎消息
6. 无断点 -> 直接进大厅 -> 触发欢迎消息

## 4. Agent上下文切换视觉

### 4.1 切换动效

当Agent角色发生切换时（如对练角色->教练）:
- 头像: 0.5秒渐变动画，从角色形象过渡到教练图标
- 气泡颜色: 从角色色渐变到教练色
- 消息标注: 从"[角色名]"变为"[教练]"

### 4.2 系统消息样式

- 系统消息（如"----对练结束----"）: 居中显示，灰色文字，无气泡
- 过渡提示: 居中显示，带浅色背景条，与对话气泡视觉区分
