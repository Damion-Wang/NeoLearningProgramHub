# A3 主线 2 · 企业培训项目全生命周期 SOP v0.1

> **背景**：本期睿学要替代企业的真人讲师 / 1v1 老师 / 线上 SCORM 课程。要做完整的线上培训项目交付，必须把"项目全生命周期"理清楚。
> **范围**：从客户意愿 → 签单 → 开营组织 → 学习 → 结营 → 成果汇报 → 服务截至 的 7 个阶段。
> **日期**：2026-04-26
> **状态**：v0.1 骨架，待 B3 HR 访谈完成后填话术 / 工具 / 异常处理

---

## 总览：7 阶段与 5 角色

### 5 个参与角色

| 代号 | 角色 | 主要责任 |
|------|------|--------|
| **CL** | 客户决策方 | HR 总监 / L&D 负责人 / 用人部门高管 |
| **HR** | 客户日常对接 | HR 培训经理 / L&D Lead（落地执行）|
| **SE** | 睿学销售 | 商机识别 / 方案沟通 / 合同 |
| **DE** | 睿学交付 | 项目配置 / 内容映射 / 上线运维 |
| **NE** | Neo（产品）| 课程交付 / 学员陪伴 / 数据采集 |
| **OR** | Ora（产品）| 看板解读 / 报告生成 / 异常预警 |

### 7 阶段总览

```
[0. 商机识别]
    ↓
[1. 客户意愿确认 + 方案沟通]
    ↓
[2. 签单（合同+计费）]
    ↓
[3. 开营组织（项目配置 + 学员名单 + 启动会）]
    ↓
[4. 学习期（运行中：学员学/管理员监控/Neo 教学/Ora 解读）]
    ↓
[5. 结营（结项报告 + 成果汇报会）]
    ↓
[6. 服务截至（账号到期 / 续约 / 退课）]
```

## 阶段 1 · 客户意愿确认 + 方案沟通

| 维度 | 内容 |
|------|------|
| **触发** | 销售团队识别商机（来自母公司 1700+ 客户库 / 60 家天使客户名单）|
| **输入** | 客户当前培训痛点 / 预算 / 时间窗 / 学员规模 |
| **输出** | 已签 LOI 或 PoC 合同；或 No-Go |
| **Owner** | SE（主）+ CL/HR（客户方）|
| **周期** | 2-8 周（视客户决策链路）|

### 关键动作（待 B3 访谈填实）
- 销售首访：识别痛点（真人讲师贵 / SCORM 完播率低 / 1v1 老师不可规模化）
- 产品 Demo：用现有 prototype/build/ 11 页演示
- 客户痛点 → 睿学能力映射文档（销售工具）
- ROI 估算（替代真人讲师的成本对比）
- 试点方案设计（30-40 人 / 6 个月 / 8 Course）

### 角色 × 动作

| 动作 | SE | DE | CL | HR | NE/OR |
|------|----|----|----|----|----|
| 首次需求挖掘 | 主 | — | 接见 | 接见 | — |
| Demo 演示 | 主 | 协助 | 评估 | 评估 | — |
| 痛点映射 | 主 | 协助 | 反馈 | 主 | — |
| ROI 估算 | 主 | — | 决策 | 评审 | — |
| LOI 签署 | 主 | — | 签字 | 协办 | — |

### 异常处理（待 B3 补）
- 客户问"和真人讲师比效果如何"→ 用对照案例 + ROI 数据回应
- 客户担心"AI 老师会不会很冷冰冰"→ Demo 现场让客户当学员体验 5 分钟
- 客户预算不够 → 推单 Course PoC（1 个月）

## 阶段 2 · 签单（合同+计费）

| 维度 | 内容 |
|------|------|
| **触发** | 客户决策通过 |
| **输入** | LOI / 配置确认书 / 学员名单初稿 |
| **输出** | 正式合同 / 商品平台下单 / 项目实例创建 |
| **Owner** | SE（合同）+ DE（产品配置）|
| **周期** | 1-3 周 |

### 关键动作
- 合同条款：服务期 / 学员数量 / 内容范围 / 数据归属 / 数据脱敏（待 0424 决策 C-06 红线落实）

- **周期分离**（0424 议题 N-01 待讨论）：项目周期 vs 账号有效期是否分离
- 商品平台下单 → 自动下发 AOM 内容包 → 创建项目实例 → 创建 admin 账号

## 阶段 3 · 开营组织（最关键 SOP 阶段）

| 维度 | 内容 |
|------|------|
| **触发** | 合同生效 |
| **输入** | 项目实例 / admin 账号 / 内容包 |
| **输出** | 开营完成 / 学员可登录 |
| **Owner** | DE（主）+ HR（学员沟通）|
| **周期** | 2-4 周 |

### 子阶段 3.1 · 项目内容预览（DE 视角）
- DE 用 admin 登录管理端 → 项目内容预览（只读）→ 确认 8 个 Course 与客户需求匹配
- 如有不匹配：和 KGP 团队协调内容调整（V2 才能动态配置，本期是预制菜）
- 输出：内容确认书

### 子阶段 3.2 · 项目配置（HR + DE 协作）
- 项目基本信息（名称 / 周期 / 阶段划分）
- 学员名单导入（Excel）
- 角色分配（学员 / 双角色管理员）
- 催学规则配置
- 平台个性化（LOGO / 主题色）

### 子阶段 3.3 · 启动会（HR 主导）
- HR 在客户公司线下/线上召开启动会
- DE 提供启动会工具包（PPT / 学员登录指引 / FAQ）
- 学员现场注册账号 / 体验 demo

### 子阶段 3.4 · 确认开营
- HR 在管理端点击"确认开营"
- 系统自动发送学员登录邮件（实际由运营在平台外发）
- 学员端开放登录

### 异常处理
- 学员名单错误 → DE 协助批量修正
- 客户 LOGO 资源未到位 → 用默认形象
- 学员注册失败 → DE/HR 双线兜底

## 阶段 4 · 学习期（运行中，6 个月主体期）

| 维度 | 内容 |
|------|------|
| **触发** | 开营完成 |
| **输入** | 已开营项目 / 学员开始登录 |
| **输出** | 学员每周/每月行为数据 / Ora 看板 / 周报 |
| **Owner** | NE（教学，主）+ OR（监控）+ HR（督学）+ DE（运维）|
| **周期** | 项目周期主体（默认 6 个月）|

### 4 类常态动作

#### 4.1 学员侧（Neo 主导）
- 每周登录学习（大厅 → 场域 → 大厅）
- Neo 持续陪伴：跨场域记忆 / Bloom 引导 / 情绪感知
- 笔记 / 工具下载 / 报告查看
- 个人产品报告（Activity 级 / 阶段级）

#### 4.2 管理员侧（Ora 主导）
- 三维度看板（项目 / 人员 / 内容）
- 热力图 / 行为数据
- 团队报告生成（HITL 编辑）
- 异常预警（脱训 / 进度落后）

#### 4.3 运营侧（HR 协同 DE）
- 每周看板 review
- 每月催学动作（系统自动 + 人工补漏）
- 中期反馈（半程 review）
- 异常学员单独跟进

#### 4.4 平台侧（DE 监控）
- 系统稳定性 / 数据质量 / Token 消耗
- 客户反馈采集
- bug / 体验问题响应

### 关键检查点
- W2：首次脱训预警（哪些学员还没登录？）
- W4：第一次中期反馈
- W12：半程 review（项目结项前 50% 时点）
- W20：进入结营准备

### 异常处理（待 B3 + 实际项目运行经验补）
- 学员脱训 ≥ 1 周 → 自动催学 + HR 介入
- 学员投诉 Neo 答非所问 → 客服记录 + Tech 调优
- 看板数据异常 → DE 排查
- HR 投诉学员进度看不到 → Ora 看板使用培训

## 阶段 5 · 结营

| 维度 | 内容 |
|------|------|
| **触发** | 项目周期到期 / 全部 Course 完成 |
| **输入** | 全程学员行为数据 / 报告 |
| **输出** | 结项报告 / 成果汇报会 / 续约意向 |
| **Owner** | OR（报告）+ HR（汇报会）+ SE（续约）|
| **周期** | 2-4 周 |

### 子阶段 5.1 · 结项报告生成
- Ora 拉全过程数据生成"结项报告"模板
- HITL 编辑（HR 视角调整 / 添加客户专属内容）
- 导出 PDF/Word
- 三类汇报场景的差异化模板（0424 决策 S-14）：
  - 中期报告
  - 结项报告
  - 持续价值追踪报告

### 子阶段 5.2 · 成果汇报会
- HR 在客户公司召开汇报会
- 内容：项目数据 / 高光案例 / 改进建议 / ROI 复盘
- DE 提供汇报会工具包

### 子阶段 5.3 · 学员收尾
- 学员收到个人结项报告
- 学员账号进入"结项后状态"（0424 决策 S-12：只读不互动）
- 个人资产下载（笔记 / 报告 / 工具卡）

### 关键决策（待用户拍板）
- [ ] 结项后账号状态（议题 S-12）：只读？延续期？
- [ ] 持续价值追踪报告的范围和频次（议题 S-14）

## 阶段 6 · 服务截至

| 维度 | 内容 |
|------|------|
| **触发** | 项目结项后或账号有效期到期 |
| **输入** | 结项后状态项目 / 续约意向 |
| **输出** | 续约 / 退课 / 数据归档 |
| **Owner** | SE（续约）+ DE（数据处理）|
| **周期** | 1-2 周 |

### 子阶段 6.1 · 续约判定
- SE 拉数据：本项目 ROI / 学员反馈 / HR 评价
- 客户决策（续期 / 升级 / 退出）
- 续约：进入新一轮阶段 1（新项目实例）
- 不续约：进入数据归档

### 子阶段 6.2 · 数据处理
- 学员账号状态变更（结项后 → 失效）
- 客户数据导出（合规要求）
- 平台侧数据归档 / 删除（按合同条款）
