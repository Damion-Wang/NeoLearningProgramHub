## 3 · 端口与角色（DM 议题 2 决议 · 维度区分）

> **关键原则（议题 2）**：管理员（角色权限名）与 admin（账号类型）**不在同一维度**，互不包含，**没有"广义/狭义"问题**。

### 3.1 端口（3 个）

| 术语 | 含义 |
|---|---|
| 学员端 | 3 个端口之一 / 产品化 |
| 管理端 | 3 个端口之一 / 产品化 |
| 产品运营端 | 3 个端口之一 / 短期非产品化 / 后台动作 |
| 端口架构 | 端口视角 vs 角色视角 / 一个 user 多角色加载 |
| 跨端关系 | 学员端↔管理端 / 管理端↔产品运营端 |

### 3.2 角色（4 类 · 角色权限维度）

| 术语 | 含义 |
|---|---|
| 学员 | 4 类角色之一 |
| HR | 4 类角色之一 / 付费方 / 与项目运营是同一组人 / 共用管理端 |
| 项目运营 | 4 类角色之一 / = HR 同一组人 / 不分账号类型 / 共用管理端 |
| 产品运营 | 4 类角色之一 / 平台侧团队 |
| **管理员** | **角色权限名** / = HR + 项目运营（管理端用户的统称） / **角色维度** |
| 角色↔端口映射 | 学员→学员端 / HR+项目运营→管理端共用 / 产品运营→产品运营端 |

### 3.3 账号（3 类 · 账号类型维度）

| 术语 | 含义 |
|---|---|
| **admin** | **项目根账号** / 1 个 / 开营前可改 / 开营后锁定 / **账号维度** |
| 三类账号 | admin / 管理员账号 / 学员账号 |
| 多角色加载 | 同一套用户表 / 一个 user 可同时持学员+管理员角色 |

### 3.4 维度交叉关系（议题 2 决议核心）

| 关系 | 说明 |
|---|---|
| admin 账号 默认权限 | 默认有"管理员"角色权限 + 项目根级 admin 权限 |
| HR / 项目运营 账号 默认权限 | 只有"管理员"角色权限（无 admin 权限） |
| **不能称 admin 是"广义管理员"** | 维度不同 / admin = 账号类型 / 管理员 = 角色权限 |

## 4 · 学员端场域（议题 1 决议 · 3 场域 + 大厅）

> **关键原则（议题 1）**：**大厅不算场域**——它是中枢层。**学习核心层 3 场域 = lecture / practice / recap**。未来场域 inquiry/sandbox/assignment 仅作概念预留。

### 4.1 现状口径（V-0.2.0）

| 术语 | 含义 | 备注 |
|---|---|---|
| **学员端 = 大厅 + 3 学习场域** | 现状统一口径 | ★ 唯一统一表述 |
| **大厅 (Hall)** | 中枢层 / **不算场域** / 默认主页 | 与 3 场域并列描述 |
| **学习核心层 3 场域** | lecture + practice + recap | 学员端学习实质所在 |
| **lecture / 授课 / Lecture** | 3 场域之一 | 中英对照 |
| **practice / 对练 / Practice** | 3 场域之一 | 中英对照 |
| **recap / 小结 / Recap** | 3 场域之一 / **小结 = recap 中文名** | 中英对照 / 议题 1 决议 |
| **跨场域机制** | 笔记悬浮球 + 4 库 + Neo 跨场域记忆三件套 | 核心层 + 中枢层贯通 |

### 4.2 三场域共享框架与差异

| 术语 | 含义 |
|---|---|
| 三场域统一框架 | lecture/practice/recap = AOM Activity 容器 |
| 三场域差异化 | 主任务差异（学/练/复） |
| 三场域共享设计 | 4 项共享 + 7 项约束（详见 [05-learner/06-cross-context § 4]） |

### 4.3 大厅核心结构

| 术语 | 含义 |
|---|---|
| 大厅 3 个顶层模块 | Neo 辅导 / 看板 / 笔记悬浮球 |
| 大厅 Neo 3 角色 | 辅导 + 督学 + 课程推荐 / 优先级：辅导 > 课程推荐 > 督学 |
| 应用咨询 | 大厅辅导子类 / 含讲授 + 解惑 |
| 看板 4 区域 | 项目总览区 / 内容目录区 / 学习成果区 / 社区窗口 |
| 项目总览区 5 大组件 | 热力图 / 项目时间轴 / 数据卡片 3 张 / 感知图谱呈现位 / 待办清单 |
| 数据卡片 3 张 | 进度卡 / 学习足迹卡 / 学习收获卡 |
| 类型 3 zone | Activity 类型 3 zone（lecture/practice/recap）|
| 学员旅程图 | 完整学习项目周期图（中枢+学习核心+跨场域机制全景）|

### 4.4 未来场域（仅概念预留）

| 术语 | 含义 | 备注 |
|---|---|---|
| 未来场域 | inquiry / sandbox / assignment 等 | 概念预留 / 未实现 |
| 加入未来场域后 = 4 场域 + 大厅 | inquiry 落地后 | 未来口径，本期不用 |

### 4.5 统一表述清单

- ✅ "学员端 = 大厅 + 3 学习场域"
- ✅ "学习核心层 3 场域（lecture / practice / recap）"
- ✅ "大厅（中枢层）+ 3 学习场域（学习核心层）"
- ✅ "5 类 × 3 场域 + 大厅"（卡片矩阵实际口径）
- ❌ "学员端 4 场域"
- ❌ "5 场域 × 7 动作矩阵"
- ❌ "5 类 × 5 场域"（spec [05-learner/06-cross-context § 4.4] 标题需 T1.4 修正）
