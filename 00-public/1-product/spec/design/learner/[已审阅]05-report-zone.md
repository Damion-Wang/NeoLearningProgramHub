## 模块定位

报告场域是学员查看个人学习报告并与 Neo 讨论的空间——报告不是"宣判"，而是"对话"，学员可以质疑、提问、与 Neo 共同解读自己的成长。

---

## Neo（Tutor）— 报告场域中的角色定位

**前台名称**：Neo
**角色类型**：Tutor（老师）
**知识范围**：可读取全部课堂数据和学员画像，以当前 Course 为主要视角引导

### Neo 在报告场域的核心价值

1. **报告解读引导**——渐进式呈现报告内容，引导学员理解评估结论背后的行为证据
2. **证据追溯**——学员质疑评分时，Neo 引用具体行为记录回答，每个结论都可追溯到原始数据
3. **知识关联**——报告讨论中关联课程知识点，帮助学员理解改进方向

### Neo 在报告场域的核心能力

| 能力 | 说明 |
|------|------|
| **报告解读** | 渐进式呈现报告三部分，引导学员理解内容 |
| **EvaluationSkill** | 引用行为证据回答质疑，支持逐层钻取（结论→证据摘要→原始对话） |
| **ContentSkill（轻量）** | 报告讨论中关联知识点解释，帮助学员理解改进方向 |

### Neo 在报告场域的行为边界（能做 vs 不能做）

| 能做 | 不能做 |
|------|--------|
| 渐进式呈现报告，引导学员逐步理解 | 一次性倾倒所有数据让学员自己消化 |
| 引用具体行为证据回答质疑（"第 3 题你选了X，正确答案是Y"） | 泛化结论不引用具体行为（"你整体还行"） |
| 关联课程知识点解释改进方向 | 对学员做价值判断（"你这个人不行""你能力很差"） |
| 接受学员对评估的不认同，记录反馈 | 透露其他学员的具体数据或排名 |
| 鼓励学员制定改进计划 | 强迫学员接受 Neo 的评估结论 |

## 功能区域

> 全局布局为两栏 + 可折叠导航区。报告生成在后台进行，展示和解读阶段为双栏。

## 报告三部分结构（v0.4.0 变更：从 8 模块简化为 3 部分）

### 第一部分：课程核心内容回顾——学了什么

提炼本课程的关键知识点和方法论，帮学员快速回顾学习重点。例如："本课程重点讲解了 PIN 法三步走——识别 Pressure、探索 Interest、挖掘 Need，以及横向协作四象限应对策略。"

### 第二部分：个人课堂表现复盘——怎么表现的

整合授课、对练、调研中的具体行为数据：

- **高光表现**（用具体案例支撑）：例如"在 PIN 法对练第二次尝试中，你在第 1 轮就主动追问对方的 Interest，比第一次提前了 2 轮"
- **待改进点**（用具体案例支撑）：例如"在四象限案例分析 Quiz 中，你将'资源争夺型'误判为'冷漠回避型'，混淆了主动施压和被动回避的区别"

### 第三部分：下一步应用计划——接下来做什么

结合课程内容和个人表现，给出可在实际工作中应用的具体步骤：

1. 本周内，在与市场部张总的下次沟通中，刻意使用 PIN 法的第一步——先追问对方的具体需求而非直接回应表面请求
2. 针对四象限判断的薄弱环节，重新完成案例分析 Quiz
3. 下载「PIN 法三步走对照表」和「横向协作四象限指南」，在下次跨部门会议前复习

## 用户故事：张磊查看 Course 2 个人报告

### 场景背景
张磊完成了 Course 2「横向协作」的全部 Activity（授课+对练+调研），进入报告场域。

### 完整旅程

**报告生成阶段（后台）**

张磊完成最后一个 Activity 后，系统提示"正在生成你的个人报告..."

系统播放简短的祝贺动画和预告文案：
> "恭喜你完成了横向协作课程！我正在整理你的学习报告，马上就好。"

报告在后台并行生成（拉取全部数据碎片汇总分析），约 10 秒后进入双栏展示。

**报告展示阶段——渐进式呈现（双栏）**

**第一部分先出现**：课程核心内容回顾

左栏展示精炼的知识点总结，配以结构化卡片和关联工具库链接。张磊快速浏览，确认知识点覆盖完整。

**第二部分渐入**：个人课堂表现复盘

- 个人成长时间线（折线图）：X 轴为 Course 内的各 Activity，Y 轴为沟通效能评分，展示张磊从授课→对练→调研的表现变化趋势
- 高光时刻卡片："在 PIN 法对练中，你第二次尝试比第一次提前 2 轮就开始探索 Interest——这是本课程你最大的进步"
- 待改进卡片："四象限判断准确率 60%（3/5），建议重点区分'资源争夺型'和'冷漠回避型'"
- 匿名分位区间标尺（柔和灰色）：张磊的总体表现处于"前 25%"区间

张磊点击高光时刻卡片上的"查看证据"链接 → 弹出对练第二次尝试的原始对话记录片段。

张磊点击折线图上的某个数据点 → 右栏 Neo 自动解读该数据背后的行为证据。

**第三部分渐入**：下一步应用计划

展示 3 条可落地的行动步骤，每条关联具体的工具或课程内容。

**报告解读阶段（双栏）**

张磊在右栏对话区提问：
> "你说我四象限判断准确率只有 60%，具体哪道题我判错了？"

Neo 引用具体 Quiz 记录：
> "第 3 题和第 5 题。第 3 题的场景是'同事不回消息不参加会议'，你选了'资源争夺型'，但正确答案是'冷漠回避型'。区别在于：资源争夺型会主动要资源但不配合，冷漠回避型是完全不参与。这个点在四象限 PPT 的第二页有详细区分。"

张磊继续问：
> "匿名分位区间前 25% 是什么意思？是跟谁比？"

Neo 解释：
> "是你在这个培训项目中和其他学员的相对位置。前 25% 意味着你的总体表现排在项目里前四分之一。具体排名我看不到也不会告诉你——这只是帮你了解大致位置。"

**报告导出**

张磊点击「导出 PDF」按钮，将报告保存到本地。

## 核心交互清单

### 1. 报告生成

| 项目 | 内容 |
|------|------|
| **触发** | 学员完成一个 Course 的全部 Activity |
| **响应** | 系统播放祝贺动画，后台并行生成报告（拉取全部数据碎片汇总） |
| **联动** | 生成完毕后进入双栏展示，报告分段渐进式呈现 |

### 2. 证据追溯

| 项目 | 内容 |
|------|------|
| **触发** | 学员点击报告中任何评估结论的"查看证据"链接 |
| **响应** | 弹出原始行为记录片段（对话记录/Quiz 回答/对练表现） |
| **联动** | 证据可追溯到半结构化记录的原始数据 |

### 3. 图表交互解读

| 项目 | 内容 |
|------|------|
| **触发** | 学员点击报告中 BI 图表的数据点 |
| **响应** | 右栏 Neo 自动解读该数据点背后的行为证据 |
| **联动** | 学员可以继续追问 |

### 4. 与 Neo 讨论报告

| 项目 | 内容 |
|------|------|
| **触发** | 学员在右栏对话区提出关于报告的任何问题 |
| **响应** | Neo 引用具体行为证据回答，不使用泛化结论 |
| **联动** | 学员可以质疑评分、要求重新解释、对不认同的点提出反馈 |

### 5. 报告导出

| 项目 | 内容 |
|------|------|
| **触发** | 学员点击「导出 PDF」按钮 |
| **响应** | 生成完整 PDF 报告并下载 |
| **联动** | PDF 包含报告三部分完整内容 + BI 图表 |

### 6. 断点续播

| 项目 | 内容 |
|------|------|
| **触发** | 学员查看报告中途退出 |
| **响应** | 系统记录当前浏览位置和 Neo 对话上下文。再次进入时从断点继续 |
| **联动** | 大厅课程目录中该 Activity 显示为 ▶ 学习中状态 |

## 已确认决策

- 报告简化为三部分：课程核心内容回顾 + 个人课堂表现复盘 + 下一步应用计划
- 报告去掉"基础信息"模块（姓名/时长等），直接聚焦分析维度
- 布局为两栏+可折叠导航：报告生成在后台，展示和解读阶段为双栏（左报告+右Neo）
- 报告是独立的教学模块，学员可与 Neo 讨论
- 学员端展示个人成长时间线 + 匿名分位区间（前25%/中间50%/后25%），不展示具体排名名次
- 排名基于项目内（30-40 人），不跨项目比较
- 学员端报告 API 不返回排名字段（API 级隔离）
- 碎片生产优化：学习特质碎片每 Activity 一个汇总
- 碎片来源权重：Quiz > 对练 > 调研 > 辅导 > 被动信号
- 渐进式呈现（不一次性全部展示）
- 断点续播：从断点继续
