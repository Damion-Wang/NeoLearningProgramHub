## 1 · 阶段 5 · 学习期 ★ 主体

**业务定义**：项目周期内学员在 3 学习场域 + 大厅学习 + 管理员监控与跟进。这是产品的"主战场"。

**时间范围**：项目周期主体（典型 4-6 个月）。

**涉及角色**：

- **学员**——主体使用者（在 3 学习场域 + 大厅与 Neo 互动）
- **HR + 项目运营**（同一组人 / 共用管理端）—— 看板监控 + 报告生成 + 变更管理 + 阶段评估
- **Neo**（学员端）/ **Ora**（管理端）—— AI 角色

### 1.1 学员视角

- Neo 在大厅持续陪伴（辅导 / 课程推荐 / 督学）
- 按 Course 序列学习 → 进入 lecture / practice / **recap（小结）** Activity
- **观看个人学习情况**——大厅看板查看自己的进度 / 能力变化 / 已学内容
- **复习与回顾**——重看历史 / 重练对练 + **4 库回顾**：
  - **我的报告**——回顾自己的报告（含 recap + practice 报告）
  - **我的高光**——回顾 AI 推送的高光时刻
  - **我的工具**——回顾已用过的方法论工具卡片
  - **我的笔记**——回顾自己的随手笔记
- 笔记记录（悬浮球贯穿大厅 + 学习场域）

> **关于小结 Activity**：小结 Activity 属于**学习期**（不是结营动作）——用于 Course 层级内容回顾，回顾学习动作 / 夯实学习效果。**小结的具体出现时机由 AOM 编排决定**——KGP 在 Course（Module）内的 Activity 序列中安排小结 Activity 的位置 / 数量（典型：每个 Course（Module）末尾配一个小结）。

### 1.2 管理员视角

管理员通过两种方式与 Ora 协同：

#### A · Ora 看板管理（被动呈现 · 静态查看）

- 三维度（项目 / 人员 / 内容）+ 热力图
- 静态数据 + Ora 自动生成的项目健康度解读（注：**本期不做"看板点击 → Ora 实时解释"**——这是未来形态，详见 [02-foundation/01-personas § 1.3.4]）
- 用途：日常巡检 / 整体状态把握

#### B · Ora 问答式管理（主动对话 · 即时响应）

- 管理员主动向 Ora 提问（"为什么 W4 脱训突增？" / "张三这周参与度怎样？"）
- Ora 用证据回答（基于学员聚合数据）
- 用途：定向数据探查 / 异常解释 / 报告前的数据准备

#### C · 异常学员跟进（平台发现 → 线下跟进）

- **平台层**：Ora 看板 / 问答 中发现异常学员（脱训 / 进度落后 / Quiz 频繁失败）
- **线下层**：**项目运营**（HR 同一组人 / 业务角色）**线下跟进**该学员（电话 / 微信 / 1on1 沟通）
- 平台不做学员个体的强制干预动作；干预靠人 / 渠道完成

#### D · 变更管理（详见 § 1.4）

### 1.3 项目里程碑（Milestone）

> **概念区隔**：本节"项目里程碑"是**项目级运营节奏锚点**（结营 / 中期 / 自定义上限 10 个）；与 [05-learner/05-recap § 1] recap 的 **Activity 完成事件**（Activity 级 / 学员个人 recap 报告完成）是两个独立概念。

学习期由**项目里程碑**驱动管理员的运营节奏——每个里程碑到期会自动触发 team report 草稿生成 + 站内消息预提醒（前 3 天每日）+ Ora 当天提醒，每个管理员有**独立报告库与草稿副本**。学员侧轻量可见（大厅倒计时 + 项目进度区里程碑标记 + 到期产品化推送）。

**阶段性评估机制**：通过 **team report 自定义功能**生成中期反馈 / 半程 review / 等——由阶段里程碑（中期）/ 自定义里程碑到期触发 team report 生成入口 + 多管理员独立草稿副本。

里程碑分 **2 类**：

- **阶段里程碑** —— 本期仅 2 种：**结营**（必选 / 不可删 / 可改名 + 调时间）+ **中期**（可选）
- **自定义里程碑** —— 自由插入，**上限 10 个**

约束：里程碑必须在项目周期内 / 不可超出结营时间 / 同一天可有多个 / 不设禁用 / 节奏由运营掌握。

> 详细配置规则、关联动作清单、阶段里程碑细则、催学规则与里程碑的关联机制 → [06-management/04-program-config § 1.2.3] 里程碑配置 + § 1.5 催学规则。

### 1.4 变更管理子流程 ★

学习期间允许的变更：

| 变更类型 | 是否允许 | 触发场景 | 审批流（本期） |
|---------|---------|---------|--------------|
| **学员名单变更** | ✅ 部分允许 | 替换未登录账号 / 不可增加总数 | 项目运营提议 → HR / admin 确认（详见下方 SOP）|
| **项目周期调整** | ✅ 允许（严格确认）| 学员整体进度慢需延长 / 项目提前结束 | 项目运营提议 → HR 确认 → 销售签字（涉及合同）|
| **催学规则调整** | ✅ 允许 | 规则详见 § 1.5 | 项目运营自决 |
| **平台个性化调整** | ✅ 允许（部分锁定）| 见 § 1.6 平台个性化 | admin / HR 自决，部分字段锁定 |
| **内容范围调整（Course Pack 变更）** | ❌ **本期不允许** | —— | 内容范围在阶段 3 签单时锁定（沿用 [03-lifecycle/01-pre-learning § 1.4]）|

**学员名单变更 SOP**：

- **总账号数**：合同签单时锁定 / 学习期内**不可增加**
- **未首次登录账号**：✅ 可换人（如新员工替换离职员工）
- **已首次登录账号**：❌ 不可换人（保留学员学习数据）
- **节点判定**：账号是否曾经首次登录平台

**变更管理 SOP**：

- 涉及合同字段的变更（项目周期）需要严格确认（弹窗 + 二次确认）
- **服务周期 / 内容范围 / 学员账号总数**任何情况都不可改
- **变更操作日志本期不做**（详见 [06-management/04-program-config § 1.4]）

**program config 功能映射**：

- 看板 / 数据问答（详见 [02-foundation/03-roles-and-ports § 1.3] + [04-global/02-platform-global § 1]）
- 学员名单变更（增减 + 状态修改）
- 项目周期变更（含严格确认流程）
- team report 生成入口（每个管理员独立报告库 + 独立草稿副本）
- 里程碑配置（详见 § 1.3）
- 催学规则 UI（详见 § 1.5）
- 平台个性化（详见 § 1.6）

### 1.5 催学规则

学习期内项目运营对学员的催学动作。本期 scope：

- **对象**：全员催学（异常学员系统标识但不自动催，由运营**手动选人**发站内消息）
- **异常学员判定信号**：脱训（连续 X 天未登录）+ 进度落后（落后项目平均进度 X%，按 Activity 粒度）+ Quiz 频繁失败 —— **具体阈值详见 [06-management/04-program-config § 1.2.4] / 本节仅描述业务定义 / 完整 3 类信号见 § 1.2 C 节**
- **渠道**：仅站内消息（邮件 / IM / SMS 后续逐步加）
- **触发类型**：里程碑绑定 / 脱训信号 / 进度落后信号 / 手动触发
- **调性**：仅提醒 / 警示（鼓励 / 表扬由 Neo 在大厅自然表达）
- **频率上限**：不设上限，运营自负责
- **投递日志**：完整记录（谁收到 / 是否查看）；**本期不支持学员回应**（不监控点击 / 详见 [06-management/05-message § 1]）

> 详细的文案模板变量清单、判定阈值默认值、渠道集成方案 → [06-management/04-program-config § 1.2.4] 催学规则（管理端配置）+ [06-management/05-message § 1] 消息中心（投递查看）。

### 1.6 平台个性化

学习期内 admin / HR 可对平台个性化字段做调整。本期 scope：

**学习期可改动项**：平台 Logo / 欢迎语 / 项目名称 / Neo / Ora 名字。

**本期不做**：详见 [06-management/04-program-config § 1.4] / [06-management/04-program-config § 1.2.5]（品牌色 / 自定义域名 / 字体 / 证书 / 徽章 / Neo 形象企业定制 等 / 本期统一使用睿学品牌色）。

**不可替换 · 合规底线**：

- "powered by NeoLearning" 品牌声明（**登录页强制保留** / 其他位置不强制；统一表述见 04-global/02-platform-global § 1.3）
- **大模型安全声明**（如"内容由 NeoLearning AI 提供，使用前请核对"）不可替换 / 不可隐藏

> 详细可改动字段清单 / UI 影响范围 → [06-management/04-program-config § 1.2.5] 平台个性化。
