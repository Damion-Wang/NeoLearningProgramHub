# CHANGELOG · D-4.0 → D-5.0 内容变更说明

> 本文档说明 D-4.0（v0.4.0 已审阅 spec）到 D-5.0（已交付 spec）的 **内容维度变更**。
>
> **不含**：文件拆分（17 → 22）/ 章节编号方式（全局 § → 每文件 § 1 起）/ frontmatter / 跨文件引用格式 等格式与结构层面的变更。
>
> **按功能模块对比**（不按章节）—— D-4.0 章节体系（learner / management / common 3 大区）与 D-5.0 章节体系（vision / foundation / lifecycle / global / learner / management 6 板块）已完全不同，所以不能逐章对比；本文档按"功能模块"维度对齐两版。
>
> 变更性质 4 类：➕ 新增 / ❌ 推翻 / 🔁 重构 / 🔧 校准。

## 2 · 按功能模块详述

### 模块 1 · 愿景与产品定位

**D-4.0 怎么说**：
- "面向企业培训场景的 AI 原生学习陪伴系统"
- "大厅 + 四间场域"空间模型
- 3 方价值（学员 / 管理员 / 运营）—— 但运营是单列角色

**D-5.0 怎么说**：
- "面向企业培训的 **AI-Native 学习陪伴平台**"——明确 AI-Native 定位
- "大厅（中枢层）+ 3 学习场域"空间模型
- "1v1 老师 Neo 跨场域陪伴"——加入 **1v1 老师 AI 孪生** 表述
- "学了 vs 会了"辨析 / "是 vs 不是 6 维对比"
- 3 方价值实际是两方（**学员 + HR/管理员**）/ "项目运营 = HR 同一组人" / 冲突时 HR > 学员

**主要变更**：
- ➕ 新增：1v1 老师 AI 孪生 / 学了 vs 会了 / 是 vs 不是 6 维 / "三方价值实际是两方"校准
- 🔁 重构：4 场域 → 3 学习场域 + 大厅
- 🔧 校准：AI 原生学习陪伴系统 → AI-Native 学习陪伴平台

### 模块 2 · AI 角色架构

**D-4.0 怎么说**：
- 4 个 AI 角色：**Neo**（学员端 / 大厅 Coach + 场域 Tutor）+ **Actor**（演员 / 对练）+ **导演**（后台 / 对练）+ **Ora**（管理端单实例）
- Leo 已在 2026-04-26 完全取消（D-4 反向补充加入）
- Neo 跨场域统一身份 / Database 共享
- Actor / 导演 不访问 Database / 仅遵循剧本

**D-5.0 怎么说**：
- 4 个 AI 角色：**Neo / Actor / Director / Ora**（导演英文名 Director 显式化）
- Neo 第一性目标 = **教学效果**（不是教学体验）/ 可为效果适度损失体验
- Ora 第一性目标 = **HR 可举证 ROI 报告** / HR 搭档 / **HR 拍板** / 不直接联系学员 / 不自动调度
- Ora 双实例：**Home Ora**（首页 / 看板被动呈现 + 问答）+ **Report Ora**（报告中心 / HITL 编辑 + ChatBI）
- **数据空间 3 层架构**（公共 / 学员独立 / Agent 内部）
- **三角色协作**（practice 中 Director + Actor + Neo）+ 第四面墙 + 逃生通道
- Neo 不能做 5 条（practice 行为约束清单）

**主要变更**：
- ➕ 新增：Neo 教学效果 vs 体验目标分层 / Ora HR 搭档定位 / Ora 双实例 / 数据空间 3 层 / Neo 不能做 5 条 / Director 英文名
- 🔁 重构：单 Ora → Home Ora + Report Ora 双实例

### 模块 3 · 教学方法论（D-5.0 全新立柱）

**D-4.0 怎么说**：
- "Neo 自主判断 / 以剧本和导演指令为上下文"
- "Bloom 引导"（仅大厅 Neo / 概念层引导进入授课场域）
- 设计哲学 8 条（AI 不做判官 / 内容驱动 / Neo 跨场域能力切换 / LUI 优先 GUI / 需求感知推荐 / SCO 锁定 / 防挂机 / 价值导向催学）
- **没有显式的教学方法论体系**

**D-5.0 怎么说**：
- **5 大理论底座**：CLT（认知负荷）/ ZPD（最近发展区）/ Bloom（认知层级 6 层 R/U/A/An/E/C）/ Dreyfus（5 级胜任）/ 苏格拉底+费曼
- **7 教学动作**：讲授 / 解惑 / 讨论 / 考核 / 练习 / 追问 / 反思
- **4 维感知**（实时雷达）：情绪 10 状态 / 认知负荷 3 类 / 参与度 3 态 / 学习状态（Dreyfus + mastery）
- **8 学习信号**：5 教学（缺概念 / 忘了 / 用错 / 选错工具 / 错误心智模型）+ 3 教练（知行差距 / 元认知偏差 / 语境错配）
- **6 维 × 5 等级画像**：思考与反思 / 学习投入 / 场景讨论 / 表达开放 / 挑战接纳 / 学习自驱意愿
- **3 学习场域 × 7 动作矩阵**（大厅 + 3 场域差异化调用 7 动作）
- **4 层行为约束**（情绪优先 / 教学边界 / 主动触发 / 被动响应）
- **教学边界**：范围内 / 范围外 / 完全超出 3 类
- **不剧透原则 / 不主动给答案 / 仅给方向**
- **必给总结 / 必收尾 / 苏格拉底式追问**

**主要变更**：
- ➕ 新增：5 大理论底座 / 7 教学动作 / 4 维感知 / 8 学习信号 / 6 维 × 5 等级 / 7 动作矩阵 / 4 层行为约束 / 不剧透 / 必收尾 等全部为 D-5.0 全新引入

### 模块 5 · 数据模型 AOM

- **数据反馈循环**：AOM 学习实例 → KGP → 优化下一版 AOM 模板（KGP 内容生产侧规则 / 不在 spec 详述）
- **AOM 双重身份**：数据（流程控制 / 平台代码）+ 提示词（教学设计意图 / 大模型解释）
- **自适应 L1-L4** 完整：L1 ✅ / L2 ✅（仅 lecture）/ L3 🟡 部分（大厅推荐 Activity）/ L4 ⏳ 未来

### 模块 6 · 生命周期与周期

**D-4.0 怎么说**：
- 全流程：企业签约 → 后台下发 → 项目实例 → admin 创建 → 内容预览 → 开营配置 → 开营 → 学习期 → 结项
- 项目周期 = 1 个月 ~ 1 年（管理员配置）
- 开营前后系统状态：开营前仅"配置"可用 / 开营后 4 模块全解锁
- 开营后所有配置项可编辑

- **学员数据用途**：不用于训练模型 / 可能用于调教 Agent prompt（仅限 NeoLearning / 不外发不外用）
- **Agent 安全 / Agent 数据隔离**（已锁定 / 详见 § 3.4.3 / § 3.1.3）
