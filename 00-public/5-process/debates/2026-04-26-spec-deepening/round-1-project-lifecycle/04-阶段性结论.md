## 0 · 重大概念新增（DM 引入）

### 服务周期 vs 项目周期 · 二元模型

```
   服务周期（最长 1 年）= 学员账号能正常使用平台的总时长
   ┌────────────────────────────────────────────────────┐
   │                                                    │
   │   项目周期（典型 6 个月）                          │
   │   ┌──────────────────┐                             │
   │   │ 学习目标完成时间锚 │   项目结束后剩余服务期    │
   │   │ 督促学员           │   学员仍可与 Neo 对话      │
   │   │ 可延期             │   仍可进入新 Activity      │
   │   └──────────────────┘                             │
   │                                                    │
   └────────────────────────────────────────────────────┘
                                                    ↑
                                               服务周期到期
                                                = 账号关闭
```

**核心规则**：
- 服务周期 ≥ 项目周期（服务期 ≥ 项目期）
- 服务周期最长 1 年（商业边界）
- 项目周期默认值 6 个月（教学边界，督促学员）
- 项目周期可延期（不强制关闭）
- **项目结束后到服务周期结束之间**：学员**仍可与 Neo 正常对话 + 进入新 Activity + 继续学习**（不是只读延续期）
- 服务周期到期 → 账号关闭（强制）

## 1 · 项目全生命周期 · 7 阶段（重写版）

### 阶段 1 · 商机识别（Lead）
- **触发**：销售在 60 家天使客户 / 母公司 1700+ 客户库 中识别 lead
- **输入**：客户当前培训痛点 + 预算 + 学员规模
- **输出**：合格 lead → 进入下一阶段
- **角色**：SE 主导
- **周期**：1-4 周

### 阶段 2 · 客户意愿确认（Discovery + Demo）
- **触发**：合格 lead 表达初步意向
- **输入**：客户痛点细化 + 现有培训资源（真人讲师 / SCORM / 1v1）
- **输出**：LOI 或 PoC 合同
- **关键子动作**：
  - 销售首访（识别痛点）
  - **内容包 review** —— KGP 让销售给客户看 8 Course 内容样本（轮 1 议题 4 共识）
  - Demo 演示（用 prototype/build/ 11 页）
  - ROI 估算
- **角色**：SE 主导 + KGP 配合
- **周期**：2-8 周

### 阶段 3 · 签单（Contract）
- **触发**：客户决策通过
- **输入**：LOI / 配置确认书 / 学员名单初稿
- **输出**：正式合同 / 商品平台下单 / 项目实例创建
- **合同关键条款**：
  - **服务周期**（最长 1 年）
  - **项目周期**（典型 6 个月，可配 1 个月 - 1 年内）
  - 学员数量（30-40 人典型）
  - 内容范围（8 Course 通用 / 行业定制 V2）
  - 数据归属与脱敏（0424 C-06 红线：原话直出 HR）
  - **是否启用 Pilot 子阶段**（轮 1 议题 2 共识）
- **角色**：SE（合同）+ DE（产品配置）
- **周期**：1-3 周
- **取消**：年包 / 订阅模式（DM 互动确认）

### 阶段 4 · 开营组织（Onboarding）
- **触发**：合同生效
- **输入**：项目实例 / admin 账号 / 内容包 / 学员名单
- **输出**：开营完成 / 学员可登录
- **子阶段**：
  - **4.0 · Pilot（可选）** — 5-15 人 / 2-4 周（**调研建议从 1-2 周延长**）/ 2-3 Course（**调研建议从 1 Course 延长**） / 不影响合同价格 / 通过后正式开营
  - 4.1 · 项目内容预览（DE 用 admin 登录）
  - 4.2 · 项目配置（HR + DE 协作 - 项目信息 / 学员名单 / 角色 / 催学规则 / 个性化）
  - 4.3 · 启动会（HR 主导）
  - 4.4 · 确认开营
- **角色**：DE 主导 + HR 客户对接
- **周期**：2-4 周

### 阶段 5 · 学习期（Operating）
- **触发**：开营完成
- **输入**：已开营项目 / 学员开始登录
- **输出**：学员每周 / 每月行为数据 / Ora 看板 / 持续运行
- **关键子流程**：
  - 学员侧：Neo 持续陪伴 + 跨场域记忆 + Bloom 引导 + 情绪感知
  - 管理员侧：**Ora 看板被 SE/HR 主动 review**（DM 互动确认：本期不主动推送）
  - 运营侧：每周看板 + 每月催学 + 中期反馈 + 异常学员跟进
  - **变更管理子流程**（轮 1 议题 4 共识）：管理端 Config 模块允许中途调名单 / 调课程 / 调催学规则
- **关键检查点**：W2 首次脱训预警 / W4 中期反馈 / W12 半程 review / W20 进入结营准备
- **周期**：项目周期主体（默认 6 个月，**可延期**）
- **角色**：Neo + Ora（产品）+ HR（客户对接）+ DE（运维）+ SE（关系维护）

### 阶段 6 · 结营（Closure）
- **触发**：项目周期结束 / 全部 Course 完成
- **输入**：全程学员行为数据 + 报告
- **输出**：结项报告 / 成果汇报会 / 后续动作
- **子阶段**：
  - 6.1 · 结项报告生成（Ora 拉全过程数据 + HITL 编辑 + 4 种模板：中期 / 结项 / **持续价值追踪** / 自定义，对接 0424 S-14）
  - 6.2 · 成果汇报会（HR 主导线下/线上）
  - 6.3 · 学员收尾（**重要修订**）
- **角色**：Ora（报告）+ HR（汇报会）+ SE（续约准备）

### 阶段 7 · 服务期延续（项目结束 → 服务周期到期）⭐ DM 引入新阶段
- **触发**：项目周期结束 + 服务周期未结束
- **状态**：**学员账号正常使用**——这不是"只读延续期"，是 DM 强调的"平台正常使用"
- **学员能力**：
  - ✅ 与 Neo 正常对话（应用追问、继续学习、复习历史）
  - ✅ 进入已学 Activity / 复习
  - ✅ 进入未学 Activity（如有）
  - ✅ 下载个人资产
  - ✅ 接收 Ora 持续价值追踪报告（**N 次，不限 1 次** - DM 互动确认修正）
- **管理员能力**：
  - Ora 仍可监控
  - HR 仍可生成"持续价值追踪报告"
  - 不自动催学（项目周期已过，不强督促）
- **持续时长**：直到服务周期到期
- **角色**：Neo（持续陪伴）+ Ora（监控 / 报告）

### 阶段 8 · 服务截至（Account Closure）
- **触发**：服务周期到期（最长 1 年）
- **输入**：到期项目 / 学员历史数据
- **输出**：账号 disabled / 数据按合同条款归档
- **子阶段**：
  - 8.1 · 续约判定（SE 拉数据 + 客户决策 + 续约 = 进入新一轮"商机识别"OR 退出 = 数据归档）
  - 8.2 · 数据处理（合规导出 + 平台归档 / 删除）
- **角色**：SE（续约）+ DE（数据处理）

## 2 · 全生命周期总览

```
1. 商机识别 (1-4 周)
   ↓ SE
2. 客户意愿确认 (2-8 周) - 含 KGP 内容包 review
   ↓ SE+KGP
3. 签单 (1-3 周) - 服务周期 ≤ 1 年, 项目周期 ≤ 1 年
   ↓ SE+DE
4. 开营组织 (2-4 周) - 含 Pilot 子阶段（可选）
   ↓ DE+HR
5. 学习期 - 项目周期主体（默认 6 个月，可延期）- Ora 看板（不主动推送）+ 变更管理子流程
   ↓ Neo+Ora+HR+DE+SE
6. 结营 - 4 类报告 + 汇报会
   ↓ Ora+HR+SE
7. 服务期延续 - 项目结束 → 服务周期到期，学员正常使用平台
   ↓ Neo+Ora
8. 服务截至 - 续约 / 退出 / 归档
   ↓ SE+DE
```
