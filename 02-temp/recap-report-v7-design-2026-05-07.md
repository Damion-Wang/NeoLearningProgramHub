# Recap 报告 v7 设计要点（基于 v6 4 项反馈 + 20 题决议）

## v6 → v7 关键调整摘要

DM 对 v6 mock 的 4 项反馈：
1. 学习前数据可能取不到 → 需要兜底方案
2. 学习表现要描述变化 → 需要基线 + 变化叙事
3. 行动建议太作业化 → 一句话一个代办
4. 知识点关联太浅且不应是流线结构 → 用文字表述

20 题决议落地 5 个核心设计变更：

---

## 变更 1：低数据兜底机制（A1-A4）

**A1-A3 综合答案：不纠结判定阈值，但要有兜底**

设计实现：
- Neo 内部**多维度组合判定**（学习时长 + 互动量 + 完成度）是否进入"低数据模式"
- 入兜底时报告**坦诚一次**（综合评估章节降级为"本课程印象"），但**不强行追问学员**
- 综合评估有兜底文案：标题改、范围缩，但仍写一段话（不留空白、不写"数据不足"）

**A4：6 维画像不会缺失**
- 6 维数据来源已锁定，不需要兜底分支
- 6 维永远显示完整轮廓

---

## 变更 2：变化对比的"Activity 集合"模型（B1-B4）

**B1：对比基准 = "关联 Activity 起点 vs 终点"**（不是开营、不是上次报告）
- 一份 recap 报告挂在一个 Activity 集合上
- 基线 = 这个 Activity 集合启动时学员的状态
- 当前 = 这个 Activity 集合完成时学员的状态

**B2：第一次报告也有变化章节**
- 即使是首报，也是"关联 Activity 开始时"vs"所有 Activity 结束时"的对比
- 不存在"无基线"情况

**B3：持平 → 写"稳定保持"+稳定点**
- 不是没成长，是"稳定在 X 水平"
- 稳定点写明在哪里（哪个能力、哪种场景）

**B4：变化表达 = 文字描述 + 关键对话片段举证**
- **不量化**（删除 v6 的 "+15%" 等数字）
- 主体是文字叙事："你在 X 任务上从 Y 状态变到了 Z 状态"
- 举证是关键对话片段："比如你在 lecture 第 3 节说的'…'"

**B5：退步 → 软化为"近期波动"**
- 不直接说"退步了"
- 用"近期波动 / 状态浮动 / 暂时回落"等词
- 但仍要给具体证据，不能逃避

---

## 变更 3：行动建议的"便签"形态（C1-C4）

**C1：时间表达 → 模糊三档**
- 不再用"24h / 本周 / 月内"
- 改为 **立即 / 近期 / 中期** 三档表达

**C2：删除"推荐学习内容卡片"**
- v6 mock 里每条建议带"推荐看 X 课"等推卡片 → 全删
- 行动建议聚焦"行动"本身，不夹带学习推送

**C3：Neo 同步推送任务卡片，学员选择是否接受**
- 报告里写文字便签
- 同时 Neo 在场域里推送一个"任务卡片"
- 学员可点"接受 / 不接受"决定是否进入待办系统
- **报告 ≠ 系统强行加待办**，但有路径接入

**C4：强制最多 3 条，Neo 内部排序取优**
- 不展示折叠后的"还有 5 条"
- Neo 必须做取舍，最多 3 条出来

---

## 变更 4：知识点关联的"if-then 应用关联"模型（D1-D4）

**【重要】D4 颠覆了 v6 的"知识点路图"假设：**
> "不要这种抽象的关联关系，主要是用 if-then 逻辑说明在不同的场景里怎么用的关联关系"

**关联不是位置关系，是应用关联**：
- ❌ v6 错误版：「PIN 法 → 推拉沟通法 → POWER 法」（流线箭头）
- ✅ v7 正确版：「**在向上汇报场景里用 PIN 法**；**在跨部门博弈场景里用推拉沟通法**；**在长期项目跟进里用 POWER 法**」

**D1：60-100 字一段话**
- 一段文字讲清楚"在哪个场景用哪个工具"
- 不展示树状/流线图

**D2：不展示推理链，只说结论**
- Neo 不写"我之所以这么判断…"
- 直接给"在 X 场景用 Y 法"的应用结论

**D3：不展示未学内容**
- 不预告未来课程
- 严格贴现阶段已学内容

---

## 变更 5：Neo 语气的双层设计（E1-E3）

**【重要】E3 澄清：报告口吻 ≠ Neo 场域口吻**

**报告层（recap）= 客观正式**：
- 用 "数据显示 / 观察到"（E2）
- 报告是正式产物，需要严肃、可举证
- 不用"我"作为主语

**场域层（lecture / practice）= 1v1 老师**：
- 用 "我"（教练人称）
- 像人、有立场、敢指出错位
- 严肃教练 vs 朋友的取舍 = "像人 > 懂你"原则保留

**E1：每章都引用跨课程记忆**
- 不是只在重要节点引（v6 设计）
- 综合评估、内容锚点、学习成长、学习表现、行动建议 5 章都要带跨课程引用
- 形式：「数据显示，自《角色认知》《横向协作》以来，你在 X 维度持续 Y …」

## v7 报告 5 章结构（更新版）

| 章节 | v6 旧设计 | v7 新设计 |
|------|-----------|-----------|
| 第 1 章 综合评估 | 1 段 Neo 严肃教练总结（用"我"）| 1 段 Neo 严肃教练总结（用"数据显示/观察到"）+ 跨课程记忆引用 / **低数据时降级为"本课程印象"** |
| 第 2 章 内容锚点 | 知识点 + **流线关联图**（错）| 知识点 + **if-then 应用关联文字段**（60-100 字 / 跨课程引用）|
| 第 3 章 学习成长 | 高光卡片 + 3 特质 + 3 转变 | 保留 + **基于 Activity 集合起点 vs 终点** + 跨课程引用 |
| 第 4 章 学习表现 | 4 数据卡 + 6 维雷达 + Neo 解读 | 保留 + **变化叙事**（文字 + 对话片段举证 / 持平写"稳定保持" / 退步写"近期波动"）+ 跨课程引用 |
| 第 5 章 行动建议 | 1-3 条 + 时间窗口 + 推卡片 | 1-3 条 + **立即/近期/中期** + **删推学习卡片** + **Neo 任务卡片同步推送（学员选择接受）** + 跨课程引用 |
