## 1. 品牌定位

### 1.1 产品的 AI 代言形象

**Neo** 是睿学 AI Tutor 产品对外的 **AI 品牌代言形象**，代表产品在学员面前的"AI 角色"整体。

**Ora** 是产品在**管理端**对管理者（HR / 培训负责人）的 AI 助手形象，独立于 Neo。

### 1.2 双形象合并决策

> **v0.4.0 关键决策**：将原 `Leo` / `Neo` 双 AI 角色在**用户视角**下**合并为 Neo 单一形象**。

| 历史版本 | 学员端 AI 角色 | 问题 |
|---|---|---|
| v0.2.x | Leo（导师）+ Neo（伙伴） | 学员认知负担高，难以区分两者职责 |
| v0.3.x | Leo / Neo 在不同场域切换 | 用户感到混乱，"为什么有两个 AI？" |
| **v0.4.0** | **Neo 统一形象** | 用户视角一致，后端分工对用户无感 |

**核心原则**：**"一个产品，一个 AI 形象"**。内部后端可继续按 Leo/Neo 分工实现，但**用户视角只看到 Neo**。

### 1.3 Neo 与 Ora 的分工

| 对象 | Neo | Ora |
|---|---|---|
| **使用端** | 学员端全场域 | 管理端 |
| **定位** | 学习伙伴（拟人） | 运营助手（工具感） |
| **目标用户** | 学员 / 员工 | HR / 培训负责人 / 导师 |
| **视觉风格** | 拟人形象（插画/半身像） | 抽象形象（几何/渐变光斑） |
| **品牌色** | cyan / violet / pink 渐变 | amber 单色渐变 |
| **语言风格** | 引导、鼓励、共情 | 专业、客观、数据驱动 |
| **出现位置** | 所有学员对话面板 | 管理端首页 + 报告编辑器 |

## 2. 视觉系统

### 2.1 Neo 品牌形象区（Hero）

#### 规范

| 属性 | 值 |
|---|---|
| **位置** | 学员端所有对话区顶部 |
| **高度** | 220px（固定） |
| **宽度** | 占满 Chat 面板宽度 |
| **背景** | cyan → violet 渐变（男）/ cyan → pink 渐变（女） |
| **内容** | Neo 全身插画或半身像（居中或偏右） |
| **交互** | 默认静态图（P0）；未来可替换 GIF / Lottie 动画（P2+） |
| **文字覆盖** | 左侧叠加欢迎语（如"嗨，我是 Neo，你的学习伙伴"） |

#### 出现场景

- 对练场域 · Chat 面板顶部
- 授课场域 · Chat 面板顶部
- 调研场域 · Chat 面板顶部
- 报告场域 · Chat 面板顶部
- 大厅首页 · Neo 问候区

### 2.2 Neo 消息头像

Neo 的头像有 **3 种尺寸规范**，根据使用场景切换：

| 尺寸 | 像素 | 使用场景 | 图形类型 |
|---|---|---|---|
| **小版本** | 7×7（28×28 px @2x） | Chat 消息气泡前 | 圆形裁剪的半身像 |
| **中版本** | 10×10（40×40 px @2x） | Chat Header 中 | 圆形裁剪的半身像 |
| **大版本** | 40×40（160×160 px @2x） | 品牌 Hero 区 | 完整插画（非圆形裁剪） |

#### 视觉一致性要求

- 小 / 中 / 大版本必须是同一形象的不同展示
- 圆形裁剪版本需保证面部完整可见
- 高清适配：所有头像需准备 @2x / @3x 资源

## 3. 男女形象切换

### 3.1 功能规范

学员可在**个人设置 → 偏好设置 → Neo 形象**中切换 Neo 的性别形象。

| 属性 | 男版 Neo | 女版 Neo |
|---|---|---|
| **气质** | 中性专业，略带理性 | 柔和亲和，略带温暖 |
| **品牌渐变** | cyan → violet | cyan → pink |
| **默认选项** | ✓（新用户默认） | — |
| **切换入口** | 设置页 + 对话区悬浮提示（首次使用时） |

### 3.2 切换后的影响范围

一键切换后，**所有 Neo 相关元素同步更新**：

- Chat Header 头像
- 消息气泡头像
- 品牌 Hero 区插画
- Hero 区背景渐变色
- Loading 状态的 Neo 占位图

### 3.3 持久化

- 切换结果存储于 `localStorage.neoGender`（`'male'` / `'female'`）
- 跨页面、跨 Activity、跨会话一致
- 清除浏览器数据后恢复默认（男版）

### 3.4 未来扩展

- 支持更多形象选项（P2）：如年龄段（青年 / 成熟）、风格（写实 / 卡通）
- 企业可定制专属 Neo 形象包（P3）

## 4. Neo 性格与语言风格

### 4.1 性格原则

| 维度 | 原则 | 反例 |
|---|---|---|
| **专业度** | 客观、准确、有据 | "这个可能…大概…应该…" |
| **亲和度** | 温暖但不油腻 | "哇你太棒啦！真是天才！" |
| **态度** | 鼓励但不讨好 | "您真是我见过最优秀的学员" |
| **姿态** | 引导而非灌输 | "你必须这样做，因为理论 XX 说…" |
| **自我指代** | 用"我"而非"AI" | "作为 AI，我认为…" |

### 4.2 语言风格示例

#### 正向示例 ✓

- "我注意到你在第 3 轮切换了策略，这是一个有意思的选择。想聊聊当时的考虑吗？"
- "这个问题值得多想一步。如果你是对方，会怎么回应？"
- "我看到你反复提到"心理安全"，这是你关注的点吗？"

#### 反向示例 ✗

- "作为一个 AI，我的建议是…"（生硬）
- "您的表现非常优秀！请继续加油！"（空洞）
- "你必须按照 GROW 框架的四个步骤严格执行"（灌输）

### 4.3 场景化语气

| 场域 | 语气倾向 |
|---|---|
| **对练** | 陪伴、观察、点拨（更克制） |
| **授课** | 讲解、引导、提问（更主动） |
| **调研** | 开放、倾听、延展（更耐心） |
| **报告** | 总结、肯定、建议（更正式） |

## 5. Ora 形象（管理端）

### 5.1 定位

Ora 是管理端的**AI 运营助手**，专为 HR / 培训负责人服务。

### 5.2 视觉规范

| 属性 | 值 |
|---|---|
| **形象类型** | 抽象（**不拟人**） |
| **主形态** | Amber 色渐变光斑 / 几何形状 |
| **基础色** | `#F59E0B`（amber-500）→ `#FBBF24`（amber-400）渐变 |
| **辅助元素** | 数据线条、图表化装饰 |
| **气质** | 工具感、效率感、专业感 |

### 5.3 为什么不拟人

- 管理端的用户期望是**操作效率**，而非情感陪伴
- 避免与学员端 Neo 形象混淆
- 强调"数据助手"定位，降低用户对 AI 判断的情感依赖

### 5.4 出现位置

- 管理端首页 · 欢迎区 + 数据洞察卡片
- 报告编辑器 · AI 建议面板
- **不出现**：列表页、设置页、日常操作流程中

### 5.5 Ora 语言风格

- 客观、数据驱动
- 示例：
  - ✓ "本周对练完成率 78%，较上周下降 5%。建议关注未完成用户的原因。"
  - ✗ "亲爱的 HR，我帮你看了看数据…"（过于拟人）

## 6. Chat 面板规范

### 6.1 Header 区域

#### 只显示 AI 名字

| 场景 | Header 显示内容 |
|---|---|
| 学员端所有场域 | **Neo**（纯名字，无副标题） |
| 管理端 | **Ora**（纯名字，无副标题） |

#### 禁止显示的副标题

以下内容**在用户视角的 Header 中禁止出现**：

- ✗ "AI TUTOR · Activity 内"
- ✗ "AOM 脚本 · Phase 2"
- ✗ "MENTOR · Coach"
- ✗ "LEARNER 端 · 对练模块"
- ✗ 任何内部技术术语或系统架构信息

**原因**：这些是内部分工与技术实现，不应暴露给用户。

### 6.2 消息规范

#### 消息元素

只保留三个元素：

1. **头像**（小版本 7×7，左侧）
2. **气泡**（文字/卡片内容）
3. **时间戳**（灰色小字，右对齐或气泡下方）

#### 禁止在消息上添加的标签

- ✗ "MENTOR · OPENING"
- ✗ "授课 · SCO 06"
- ✗ "报告解读"
- ✗ "AOM · Phase 3"
- ✗ 任何技术分类或模块标签

### 6.3 特殊消息类型

部分特殊消息允许额外视觉元素：

| 类型 | 视觉处理 |
|---|---|
| Discovery 卡片 | Amber 色背景，不显示 "Discovery" 字样标签 |
| Tool 引用 | Violet 色气泡，以链接形式自然出现 |
| 思考中动效 | 三个小点闪烁，不显示 "Neo 正在分析 AOM 脚本…" |

## 7. 与 spec 旧设计的兼容

### 7.1 后台 Leo / Neo 分工保留

- 原 `Leo`（导师）和 `Neo`（伙伴）的分工**在 Database 层保留**：
  - 消息表有 `ai_role` 字段记录是 Leo 还是 Neo 生成
  - Prompt 管理系统分别维护 Leo 和 Neo 的 prompt 库
  - 用于未来调优与数据分析

### 7.2 用户无感知

- 所有前端 UI 显示为 **Neo**
- 头像、名字、品牌色统一
- 用户不感知内部 AI 角色的切换

### 7.3 未来可拓展

预留的 AI 角色扩展位：

| 候选角色 | 说明 | 优先级 |
|---|---|---|
| **Career Coach** | 职业发展规划 AI | P3 |
| **Skill Mentor** | 特定技能深度辅导 AI | P3 |
| **Peer Companion** | 同侪学习伙伴 AI | P4 |

所有未来扩展角色在**用户视角**下仍然统一于 **Neo 形象**，仅在后台切换不同 prompt + model 组合。

## 8. 未来品牌演进

### 8.1 Neo 动态化（P2）

- 静态图 → GIF / Lottie 动画
- 出现时机：
  - 首次进入场域时的欢迎动效
  - 学员完成关键里程碑时的庆祝动效
  - 长时间无操作时的轻提示动效

### 8.2 Neo 语音（P2）

- 当前已有**三档语速**（慢 / 中 / 快）TTS
- 未来：
  - 男女声可独立选择（与视觉形象绑定）
  - 情感语调（鼓励 / 温和 / 严肃）自适应
  - 用户可训练专属语音（企业版）

### 8.3 Neo 情绪呼应（P3）

- 学员紧张 / 挫败 → Neo 表情关切
- 学员兴奋 / 突破 → Neo 表情欣喜
- 学员疑惑 → Neo 表情思考
- 实现路径：文本情感识别 + 表情库切换

### 8.4 品牌可定制包（P3 · 企业版）

- 大型企业可定制**专属 Neo 形象**
- 可替换元素：
  - Neo 的名字（如某公司改名为 "Leo"、"Ada"）
  - Neo 的形象（用企业吉祥物替代）
  - 品牌色（适配企业 VI）
- Ora 同样支持定制

### 8.5 多模态 Neo（P4）

- 视频通话式 Neo 形象（实时渲染）
- AR / VR 空间中的 Neo 形象
- 硬件设备（如智能音箱）中的 Neo 声音形象

## 9. 品牌使用禁忌

以下行为**禁止**出现在产品中：

- ✗ 在同一用户视角下同时出现 Neo 和 Leo
- ✗ 让 Neo 自称为 "AI" 或 "机器人"
- ✗ 让 Neo 使用过于亲密的称呼（"亲""宝""小哥哥"）
- ✗ 让 Ora 拟人化（加表情、加"亲爱的"）
- ✗ 在消息标签中暴露 "AOM" "MENTOR" "LEO" 等内部术语
- ✗ 在 Chat Header 写 "AI TUTOR · 对练 · Phase 2" 等技术副标题
- ✗ 让 Neo 和 Ora 的品牌色混用
