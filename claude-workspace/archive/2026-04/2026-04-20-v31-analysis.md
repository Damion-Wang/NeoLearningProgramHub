## 四、v3.1 中值得参考但不直接进 spec 的内容

### 核心设计哲学（v3.1 §1.5）
1. AI 不做"判官"——所有学员输入都有价值
2. 内容驱动行为——平台不硬编码教学逻辑
3. 看板区与对话区联动——双向联动
4. LUI 优先于 GUI——新手引导用对话式交互
5. 教学态与教练态流畅切换

### AOM 五层数据模型（v3.1 §16.2）
- Project → Course → Activity → SCO → Segment
- SCO 和 Segment 一对一对应，Segment 用作版本管理
- 自适应切换的是 Segment（版本），不是 SCO

### 双轨编排模型（v3.1 §16.6）
- 刚性轨（必修）：组织统一能力要求，不可跳过
- 弹性轨（自适应）：AI 教练基于学员 Profile 自主编排

### Profile 驱动自适应（v3.1 §18.4）
- 三大 Profile 维度：基本信息、认知状态与熟练水平、急缓需求
- 适配供给：学什么（内容）+ 怎么学（手段）

### 报告 BI 图表（v3.1 §13.5）
- 雷达图、热力图、饼图、折线图、柱状图
- 通用图表组件库，图表可交互联动 Agent

### 组织洞察维度（v3.1 §13.3）
- 超越个人报告：个体/群体画像、人岗映射、群体共性发现

## 五、综合建议

v3.1 是一份非常完整的产品定义文档（22章），定位是"产品需求文档"。我们的 spec/design 定位是"产品规格说明书"（面向实现的详细设计）。两者是上下游关系：

- **v3.1** = WHAT（做什么、为什么做）
- **spec/design** = HOW（怎么做、交互细节、功能规格）

建议 spec/design 以 v3.1 作为上游输入，但不重复 v3.1 的内容，而是在 v3.1 基础上细化实现规格。
