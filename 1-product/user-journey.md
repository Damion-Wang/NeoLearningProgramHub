# AI老师 — 用户旅程地图

> **阶段**: Phase 6 — User Journey Mapping  
> **日期**: 2026-04-09  
> **产品**: AI老师 — AI原生企业培训平台  
> **数据来源**: Intake Brief、目标受众研究报告、客户之声研究  
> **标签说明**: [Data] = 已验证数据, [Estimate] = 推算值, [Assumption] = 待验证假设, [Opinion] = 主观判断  

---

## 旅程一：HR/培训经理 — 从发现到内部倡导

> **核心目标**: 找到一个能帮她"用数据说话"的工具，解决政治生存问题  
> **Aha Moment**: 第一次看到AI自动生成的行为举证报告 — 这份报告她过去需要花数天手工拼凑，而且还拿不出行为层数据  
> **关键Drop-off**: 漫长的采购流程（大企业2-6个月）、组织内部对AI的怀疑态度

### 阶段总览

```
发现 → 评估 → 采购 → 项目启动 → 监控 → 报告审阅 → 内部倡导
```

---

### 阶段1：发现（Discovery）

| 维度 | 描述 |
|------|------|
| **用户行为** | 在智享会活动、HR微信社群、三茅网看到AI老师的案例分享或白皮书；或被绚星销售主动触达（cross-sell） |
| **系统响应** | 无直接系统交互。触达渠道为内容营销、行业活动、销售拜访 |
| **情绪** | 好奇但警惕 — "又一个AI噱头？"；同时焦虑 — 预算又被砍了，需要新故事 |
| **Drop-off风险** | **高** — 如果内容像传统SaaS营销（功能罗列），HR会直接划过。64%的企业因"效果不可视"砍预算 [Data]，她对"效果"二字已经脱敏 |
| **缓解策略** | 用同行业标杆案例+具体行为数据截图作为触达物料，不讲功能讲结果。核心话术："这是XX公司培训后的行为改变数据报告" [Opinion] |

### 阶段2：评估（Evaluation）

| 维度 | 描述 |
|------|------|
| **用户行为** | 申请Demo或POC；拉IT部门评估数据安全；向同行打听口碑；内部汇报给HRD |
| **系统响应** | 提供Demo环境、POC方案模板、数据安全白皮书、ROI测算工具 |
| **情绪** | 兴奋与担忧交织 — "如果能行就太好了，但万一翻车我要背锅" |
| **Drop-off风险** | **极高** — 评估链条长（HR→HRD→IT→业务代表→总裁）[Data]，任何一环说"不"就终止。65.46%担心数据安全 [Data] |
| **缓解策略** | (1) 主动提供POC — "选型过程中最具决定性的环节" [Data]；(2) 准备好IT安全问答文档；(3) 给HR提供内部汇报PPT模板，降低她的汇报成本 [Opinion] |

### 阶段3：采购（Purchase）

| 维度 | 描述 |
|------|------|
| **用户行为** | 走内部审批流程；预算确认；合同签署 |
| **系统响应** | 提供灵活定价方案（800元/账号入门价可由培训经理自行决定 [Estimate]）；提供分期或试用转正方案 |
| **情绪** | 焦虑 — "预算能批下来吗？"；压力 — "选错了供应商怎么办？" |
| **Drop-off风险** | **高** — 大企业采购周期2-6个月 [Data]，期间可能遇到预算冻结、领导换人、优先级调整 |
| **缓解策略** | (1) 800元入门价作为"轻决策"切入 [Estimate]；(2) 提供Neo Course微项目（约3万元）作为低风险起步方案；(3) 保持与HR的定期沟通，帮她推进内部流程 [Opinion] |

### 阶段4：项目启动（Project Setup）

| 维度 | 描述 |
|------|------|
| **用户行为** | 与运营团队协作配置培训项目；确定学员名单、课程安排、评估标准；通知业务部门 |
| **系统响应** | 项目创建向导；智能推荐课程配置（基于行业/岗位）；自动生成项目计划；企业微信集成配置 |
| **情绪** | 期待但紧张 — "第一个项目必须成功"；轻微挫败 — 如果配置流程复杂 |
| **Drop-off风险** | **中** — 配置复杂度超预期导致拖延；业务部门不配合释放员工时间 |
| **缓解策略** | (1) 提供"开箱即用"的标准项目模板（如基管V1横向协同）；(2) 客户成功团队1v1带跑首个项目；(3) 项目启动checklist自动化 [Opinion] |

### 阶段5：监控（Monitoring）

| 维度 | 描述 |
|------|------|
| **用户行为** | 定期查看项目进度看板；关注完成率、参与度、异常数据；处理学员反馈 |
| **系统响应** | 实时数据看板；异常预警（如：某学员连续3天未登录）；AI运营自动催学；进度周报自动推送 |
| **情绪** | 逐渐放心 — 看到学员真的在互动而非挂机；惊喜 — AI自动处理了催学工作 |
| **Drop-off风险** | **低** — 项目已启动，沉没成本驱动继续。但如果数据看板不直观或数据延迟，会降低信任 |
| **缓解策略** | (1) 看板设计以"行为改变指标"为核心，而非传统的"完成率/学时" [Opinion]；(2) 关键节点自动推送摘要到企业微信 |

### 阶段6：报告审阅（Report Review） — Aha Moment

| 维度 | 描述 |
|------|------|
| **用户行为** | 收到AI自动生成的培训成果报告；审阅行为改变数据、能力提升曲线、错误模式分析；准备向管理层汇报 |
| **系统响应** | **AI报告引擎自动生成**：(1) 学员个体能力画像；(2) 群体行为改变数据（Kirkpatrick L3）；(3) 对比基线的能力提升量化；(4) 可直接用于汇报的可视化图表 |
| **情绪** | **Aha Moment** — "这就是我一直想要的！" 她第一次拿到了行为层数据，而不是只有满意度和考试分数。这份报告她过去要花好几天手工拼凑，而且根本拿不出行为改变的证据 [Opinion] |
| **Drop-off风险** | **极低** — 这是价值交付的高峰。但如果报告内容空洞或数据不可信，信任会崩塌 |
| **缓解策略** | (1) 报告数据必须可追溯到具体学员行为记录；(2) 提供"报告解读指南"帮助HR理解数据含义；(3) 报告格式可自定义以匹配企业汇报模板 [Opinion] |

### 阶段7：内部倡导（Internal Advocacy）

| 维度 | 描述 |
|------|------|
| **用户行为** | 向管理层汇报培训成果（用AI报告）；在内部推广AI老师到更多部门/项目；在HR社群分享成功经验 |
| **系统响应** | 提供续约/扩展方案；提供跨部门推广工具包；邀请参与标杆案例共创 |
| **情绪** | 自豪 — "这次终于能用数据说话了"；有底气 — 向业务部门展示价值不再心虚；社会认同 — 在HR圈子获得专业认可 |
| **Drop-off风险** | **中** — 如果第一个项目效果未达预期，不会续约也不会推荐；如果汇报后管理层反应冷淡，动力消失 |
| **缓解策略** | (1) 首个项目选择效果最容易量化的场景（如销售话术训练）；(2) 提供"管理层汇报话术"帮助HR包装成果；(3) 建立客户成功社群，互相激励 [Opinion] |

## 旅程二：学员/员工 — 从被分配到工作应用

> **核心目标**: 用最少时间完成培训要求；如果内容真的有用，愿意多投入  
> **Aha Moment**: AI根据自己的实际工作场景调整教学内容 — "千人千面"不再是口号  
> **关键Drop-off**: 把AI老师当作又一个可以"刷课"的系统来对待

### 现状对比（被替代的旧流程）

| 阶段 | 旧流程 | 痛点 |
|------|--------|------|
| 通知 | 收到企业微信群通知"请在X天内完成培训" | 被动、强制 |
| 学习 | 打开视频课，挂机播放（47%培训中玩手机 [Data]） | 无互动、无个性化、可刷课 |
| 催学 | 运营人员在微信群反复@、逐一私信 | 烦躁、抵触 |
| 考核 | 做选择题，通过率80%但实际应用不足30% [Data] | 虚假繁荣 |
| 线下 | 周末/工作日抽2天参加集训 | 占用时间、与工作冲突 |
| 结果 | 领个证书，回到工作中什么都没变 | 走过场 |

### 阶段总览

```
任务分配 → 首次登录 → AI授课 → AI练习 → AI测评 → 查看结果 → 应用到工作
```

---

### 阶段1：任务分配（Assignment）

| 维度 | 描述 |
|------|------|
| **用户行为** | 在企业微信收到培训任务通知；查看培训要求和截止时间 |
| **系统响应** | 企业微信内推送个性化通知（非群发催学）；展示预估学习时长、与自己岗位的关联说明 |
| **情绪** | 不耐烦 — "又来了"；51%对公司培训失去信心 [Data]；71.7%视培训为任务而非机会 [Data] |
| **Drop-off风险** | **高** — 历史培训体验太差，员工第一反应是找"刷课软件"。如果通知看起来和以前一样，直接忽略 |
| **缓解策略** | (1) 通知话术突出"这次不一样" — 强调AI交互而非看视频；(2) 展示预估时长（降低时间焦虑）；(3) 不要在通知中提"考试/考核"，用"AI教练陪你练" [Opinion] |

### 阶段2：首次登录（First Login）

| 维度 | 描述 |
|------|------|
| **用户行为** | 点击企业微信入口进入AI老师；完成简单的背景调研（岗位、经验、当前挑战） |
| **系统响应** | 企业微信内嵌打开（无需跳转App）[Data — 准入门槛]；AI以对话方式了解学员背景；根据背景信息初始化个性化学习路径 |
| **情绪** | 好奇 — "这个跟以前不一样？"；轻微防备 — "AI会不会监控我？" |
| **Drop-off风险** | **中高** — 如果首次体验像传统课程（点击→视频播放），学员会立即切换到"刷课心态"。背景调研如果太长也会流失 |
| **缓解策略** | (1) 30秒内让学员感受到"这是对话不是视频" — 首屏即AI对话；(2) 背景调研控制在3个问题以内；(3) 明确告知数据用途："帮你匹配最相关的内容" [Opinion] |

### 阶段3：AI授课（AI Teaching）

| 维度 | 描述 |
|------|------|
| **用户行为** | 与AI老师进行交互式学习；提问、回答、讨论；按自己节奏推进 |
| **系统响应** | AI根据学员背景自适应调整内容深度和案例（记忆工程）；穿插quiz检测理解度；用学员所在行业/岗位的案例讲解概念 |
| **情绪** | **Aha Moment开始酝酿** — "这个案例跟我工作中遇到的问题很像"；参与感上升 — 不再是被动听讲 |
| **Drop-off风险** | **中** — 如果AI回答质量低、内容与工作脱节（63%员工的核心抱怨 [Data]），学员会快速失去耐心 |
| **缓解策略** | (1) KGP内容工厂确保内容质量和行业相关性；(2) AI实时感知学员注意力（回答速度、质量）动态调整；(3) 每10分钟设置一个"micro-win"（小成就反馈）[Opinion] |

### 阶段4：AI练习（AI Practice） — Aha Moment

| 维度 | 描述 |
|------|------|
| **用户行为** | 进入AI对练/情景模拟环节；在模拟场景中练习刚学到的技能（如：与下属的绩效面谈、客户异议处理） |
| **系统响应** | AI扮演场景中的角色（下属/客户/同事）；根据学员的实际岗位和汇报的挑战定制场景；实时反馈表现并指出改进点 |
| **情绪** | **Aha Moment** — "这个AI居然知道我团队的情况，给的场景跟我下周要处理的事一模一样！"（千人千面）。紧张但兴奋 — 第一次在安全环境里练习高难度对话 [Opinion] |
| **Drop-off风险** | **低** — 如果场景足够真实，学员会自发投入。但如果AI角色表演生硬、场景不真实，学员会觉得"又是走过场" |
| **缓解策略** | (1) 场景必须基于学员前期输入的真实工作背景；(2) AI角色反应要足够"刁钻"——太容易反而没有练习价值；(3) 提供"重来一次"选项，降低挫败感 [Opinion] |

### 阶段5：AI测评（AI Assessment）

| 维度 | 描述 |
|------|------|
| **用户行为** | 完成AI驱动的能力评估；不是传统选择题，而是开放式场景应答和行为表现评估 |
| **系统响应** | AI基于整个学习过程的行为数据（对话质量、练习表现、错误模式）生成能力评估；对比学习前后的能力变化 |
| **情绪** | 紧张 — 但比传统考试压力小（过程性评估而非一次性考试）；好奇 — "AI怎么评价我？" |
| **Drop-off风险** | **低** — 已经投入了学习和练习时间，完成测评是自然延续。但如果评估结果不合理（与自我认知严重不符），信任会受损 |
| **缓解策略** | (1) 评估结果配合具体行为证据（"在模拟场景3中，你的处理方式从X改进到了Y"）；(2) 定位为"成长记录"而非"打分排名"；(3) 不与绩效直接挂钩 [Opinion] |

### 阶段6：查看结果（Results）

| 维度 | 描述 |
|------|------|
| **用户行为** | 查看个人学习报告；了解自己的能力图谱、进步轨迹、待加强领域 |
| **系统响应** | 个性化学习报告：能力雷达图、进步曲线、具体行为改变示例、后续学习建议 |
| **情绪** | 成就感 — 看到自己的进步被量化；认同感 — "这个评估挺准的"；动力 — 想继续提升 |
| **Drop-off风险** | **低** — 但如果报告内容空洞（只有分数没有洞察），学员不会再次使用 |
| **缓解策略** | (1) 报告中包含"可直接用于工作的行动建议"；(2) 提供"分享给上级"选项（学员主动展示成长意愿）；(3) 推荐下一步学习路径 [Opinion] |

### 阶段7：应用到工作（Apply to Work）

| 维度 | 描述 |
|------|------|
| **用户行为** | 在实际工作中运用所学技能；遇到问题时回来向AI老师求助 |
| **系统响应** | AI答疑功能持续可用（学习不止于课程结束）；定期推送与学员岗位相关的"微练习"；跟踪学习后的应用反馈 |
| **情绪** | 信任建立 — "这次培训确实有用"；习惯形成 — 遇到问题先问AI老师 |
| **Drop-off风险** | **中** — 课程结束后如果没有持续触点，学员会遗忘。传统培训迁移率仅10%-20% [Data] |
| **缓解策略** | (1) 课程结束后2周、1个月推送"实践复盘"微对话；(2) AI答疑作为长期可用的工具（不只是课程期间）；(3) 收集工作应用反馈，反哺报告数据（Kirkpatrick L3→L4）[Opinion] |

## 旅程三：培训运营人员 — 从项目创建到结项

> **核心目标**: 高效管理多个并行培训项目，减少重复劳动  
> **Aha Moment**: AI自动生成了一份她过去需要花好几天手工拼凑的项目报告  
> **关键Drop-off**: 对AI工具不熟悉，习惯了微信群+Excel的工作方式

### 现状对比（被替代的旧流程）

| 阶段 | 旧流程 | 痛点 |
|------|--------|------|
| 项目创建 | 在LMS手动创建项目、上传课件、配置规则 | 多平台切换、配置繁琐 |
| 学员分配 | Excel导入名单、手动分班分组 | 易出错、费时 |
| 催学 | **微信群逐一@未完成学员、私信催促** | 占一半工作时间 [Assumption] |
| 数据收集 | **从平台导出Excel→手动汇总→制作PPT** | 纯手工、耗时数天 [Data] |
| 报告 | 手动撰写项目总结报告 | 数据颗粒度粗、缺行为数据 |
| 跨项目管理 | "每个项目一个微信群"，十几个群同时跑 | 混乱、易遗漏 |

### 阶段总览

```
项目创建 → 配置 → 学员分配 → 过程监控 → 干预 → 报告生成 → 结项
```

---

### 阶段1：项目创建（Project Creation）

| 维度 | 描述 |
|------|------|
| **用户行为** | 根据培训经理的要求在系统中创建新项目；选择课程模板、设置时间节点 |
| **系统响应** | 项目创建向导；AI推荐匹配的课程模板（基于项目类型和目标人群）；一键复制历史项目配置 |
| **情绪** | 平静 — 日常工作流程；轻微期待 — 如果新系统比旧LMS更快 |
| **Drop-off风险** | **中** — 如果创建流程比旧系统更复杂（学习曲线），会产生抵触。运营人员习惯了旧工具 |
| **缓解策略** | (1) 提供"3步创建项目"的极简流程；(2) 支持从旧LMS导入项目配置；(3) 首次使用时AI引导式教学 [Opinion] |

### 阶段2：配置（Configuration）

| 维度 | 描述 |
|------|------|
| **用户行为** | 配置AI教学参数（难度、场景、评估标准）；设置自动化规则（催学时机、预警阈值） |
| **系统响应** | 智能默认值（基于项目类型自动推荐配置）；可视化配置界面；配置预览（"学员将看到这样的体验"） |
| **情绪** | 困惑 — AI配置是新概念，不确定参数含义；焦虑 — "配错了怎么办？" |
| **Drop-off风险** | **高** — AI配置参数对运营人员来说是全新概念。如果界面不够直观，会退回到"找技术同事帮忙" |
| **缓解策略** | (1) 提供"推荐配置"一键应用，高级配置折叠隐藏；(2) 每个参数配hover说明和效果预览；(3) 提供"配置检查"功能，AI自动检测不合理配置 [Opinion] |

### 阶段3：学员分配（Assignment）

| 维度 | 描述 |
|------|------|
| **用户行为** | 导入学员名单；按部门/岗位/层级分组；发送培训通知 |
| **系统响应** | 批量导入+智能匹配（自动识别部门、岗位）；AI建议分组方案；一键通过企业微信下发通知 |
| **情绪** | 满意 — 如果比Excel分班更快；担心 — 通知是否能触达所有学员 |
| **Drop-off风险** | **低** — 这是标准化操作，只要比旧流程不差就行 |
| **缓解策略** | (1) 支持企业微信通讯录直接导入；(2) 通知发送后显示"已读/未读"状态；(3) 未读学员自动二次提醒 [Opinion] |

### 阶段4：过程监控（Monitoring）

| 维度 | 描述 |
|------|------|
| **用户行为** | 查看多项目并行看板；关注学员进度、参与度、异常情况 |
| **系统响应** | 跨项目统一看板（不再需要切换十几个微信群）；**AI自动催学**替代手动微信@；异常学员自动标记并推送预警 |
| **情绪** | 惊喜 — "不用再一个个群里催了？"；轻松 — 终于能从催学中解放出来 |
| **Drop-off风险** | **低** — 这是运营人员最直接的效率提升感知点。但如果AI催学效果不如手动催学（如学员不理AI），信任会下降 |
| **缓解策略** | (1) AI催学效果数据透明（对比历史手动催学的完成率）；(2) 保留"人工介入"选项，AI处理不了的自动升级给运营人员；(3) 催学话术可自定义 [Opinion] |

### 阶段5：干预（Intervention）

| 维度 | 描述 |
|------|------|
| **用户行为** | 对AI标记的异常学员进行人工干预（如：多次未登录、测评得分异常低）；调整项目进度或配置 |
| **系统响应** | AI提供干预建议（"建议延长该学员截止时间"/"该学员可能需要降低难度"）；一键执行干预操作 |
| **情绪** | 从"被动救火"变为"主动管理" — 运营人员的角色从"催学客服"升级为"运营策略师" [Assumption] |
| **Drop-off风险** | **低** — 有明确的操作指引。但如果AI建议不准确（误报太多），运营人员会忽略预警 |
| **缓解策略** | (1) 预警精准度优先于覆盖率（宁可漏报不要误报）；(2) 干预操作要极简（一键执行）；(3) 干预效果可追踪（"干预后该学员完成了学习"）[Opinion] |

### 阶段6：报告生成（Report Generation） — Aha Moment

| 维度 | 描述 |
|------|------|
| **用户行为** | 项目接近尾声，需要生成项目报告提交给培训经理 |
| **系统响应** | **AI一键生成项目报告**：包含整体完成率、学员能力变化分析、行为改变数据、异常情况说明、项目运营效率指标。格式可导出为PPT/PDF |
| **情绪** | **Aha Moment** — "这份报告我以前要花三四天从各个Excel里拼出来，现在AI一键搞定！而且数据比我手工统计的详细多了！" [Opinion] |
| **Drop-off风险** | **极低** — 这是价值交付的高峰。但如果报告数据有明显错误，信任会严重受损 |
| **缓解策略** | (1) 报告生成后提供"数据校验"环节，运营人员可标记疑问数据；(2) 支持自定义报告模板（匹配企业汇报规范）；(3) 报告中的每个数据点可钻取到原始记录 [Opinion] |

### 阶段7：结项（Project Closure）

| 维度 | 描述 |
|------|------|
| **用户行为** | 归档项目；整理学员反馈；为下一期项目积累经验 |
| **系统响应** | 项目自动归档；AI生成"运营复盘建议"（哪些配置效果好、哪些需优化）；历史数据沉淀为知识库 |
| **情绪** | 成就感 — 项目顺利结项；期待 — "下个项目可以更快启动了" |
| **Drop-off风险** | **低** — 但如果结项流程繁琐（需要手动填写大量表单），会留下负面印象 |
| **缓解策略** | (1) 一键结项（AI自动填充结项表单）；(2) "复制为模板"功能，下次创建同类项目直接复用；(3) 运营效率对比数据（"本次项目比上次节省了X小时"）[Opinion] |

## Flags

### Red Flags

- **学员"刷课心态"是最大的产品风险** — 51%员工对培训失去信心 [Data]，47%培训中玩手机 [Data]。如果AI老师的首次体验不能在30秒内打破"又一个视频课"的预期，学员会直接进入应付模式。这不是功能问题，是认知惯性问题 [Opinion]
- **运营人员的AI工具学习曲线被低估** — 当前运营人员日常工具是微信群+Excel [Data]，AI配置参数对她们是全新概念。如果没有足够的上手引导，运营人员会成为推广瓶颈而非推动者 [Opinion]
- **三条旅程的Aha Moment都依赖"AI报告"质量** — HR要行为举证报告、运营要项目报告、学员要个人报告。如果AI报告引擎质量不达标，三条旅程同时崩塌 [Opinion]

### Yellow Flags

- **HR采购周期与产品迭代节奏可能不匹配** — 大企业采购2-6个月 [Data]，但产品MVP计划6月才上线。如果天使客户5月内测、6月MVP、7月才有第一个完整项目数据，Q3-Q4的销售转化窗口可能来不及 [Assumption]
- **"千人千面"承诺的兑现难度** — 学员旅程的Aha Moment建立在"AI了解我的工作场景"之上，但这需要记忆工程和KGP内容工厂的成熟度支撑。MVP阶段可能无法完全实现 [Opinion]
- **运营人员的"替代焦虑"** — 21.13%的培训从业者担心AI替代岗位 [Data]。AI自动催学、自动生成报告虽然是效率提升，但也精准替代了运营人员的核心日常工作。产品叙事需要格外小心 [Data]
- **员工隐私顾虑未充分验证** — AI对话、行为数据、能力评估都涉及员工个人数据。"AI会不会监控我？"是学员旅程中的潜在地雷，目前缺少终端用户对此的态度数据 [Data Gap]
