## 3 学习核心层 · 3 个学习场域

> **作用**：lecture / practice / recap 是学员端的"学习核心"——Neo 在 3 个场域以不同形态陪伴学员完成 AOM 编排的 Activity 序列。
>
> **结构**：§ 3.0 总览（三场域统一框架）→ [05-learner/03-lecture § 1] / [05-learner/04-practice § 1] / [05-learner/05-recap § 1] 逐场域展开 → § 3.4 跨场域协同与统一约束（2026-05-02 起合并原 9.3.4 + 原 9.3.5 章节）。

### 3.0 学习核心层总览

#### 3.0.1 三场域统一框架

lecture / practice / recap 都是 **AOM 编排的 Activity 容器**——学员进入场域即进入 Activity；完成 Activity 后由学员选择 **回到大厅 / 自由复习 / 继续下一 Activity**（详见 [05-learner/03-lecture § 1.7.2] 完成 Modal）。Neo 在不同场域调用不同的 skill 组合。

#### 3.0.2 三场域差异化（对应 [02-foundation/02-methodology § 1.5] 7 动作矩阵）

| 场域 | 学员主任务 | 主导动作 |
|------|-----------|---------|
| **lecture** | 概念吸收 + 信号检测 + 闭环采集 | 讲授 + 解惑 + 追问 |
| **practice** | 拟真行为养成 | 练习 + 旁观介入 |
| **recap** | 反思迁移 + 应用计划 | 反思 + 追问 + 讨论 |

#### 3.0.3 三场域共享设计

- 阶段/SCO 锁定（详见 § 3.4.2）
- 闭环反馈（详见 § 3.4.2）
- 不剧透 / 不给答案（详见 § 3.4.2 + [02-foundation/02-methodology § 1.7]）
- Neo 跨场域记忆（详见 § 4 + § 3.4）

#### 3.4.1 三场域协同机制

##### 3.4.1.1 典型 Course Module 编排

`N 个 lecture（概念吸收）→ 1 个 practice（行为养成）→ 1 个 recap（反思迁移 + 行动转向）`。三场域 Activity 在 AOM 中混合编排，KGP 可调整。详见 [02-foundation/04-data-model § 1.3] + § 4.2。

##### 3.4.1.2 Neo 跨场域连续性

跨场域同一 Neo（身份 / voice / 方法论 / 记忆），不同场域调用不同主导动作组合（讲授为主 / 旁观介入 / 引导反思）。详见 § 4.1。

##### 3.4.1.3 数据连续性（场域间数据流）

| 数据流 | 详见 |
|------|------|
| lecture → practice（FEEDBACK_COLLECT → 剧本 hint）| § 4.2 / [05-learner/03-lecture § 1.7.4] / [05-learner/04-practice § 1.8.9] |
| lecture / practice → recap（高光 / 卡点 / 6 维画像变化 / 互动日志 → 报告）| [05-learner/05-recap § 1.2] 报告 5 块数据来源 |
| 大厅 → 各场域（断点续播 / 跨场域记忆素材）| § 4.1 |
| 各场域 → 大厅（完成状态 / 高光卡片入大厅高光区 / recap 应用计划入大厅待办）| [05-learner/02-hub § 1.2] / [05-learner/05-recap § 1.7.5] |

##### 3.4.1.4 Neo Chat 上下文跨场域规则

| 维度 | 规则 |
|------|------|
| **场域内 Activity 内** | Chat 历史不被清空，学员可翻看 |
| **跨 Activity / 跨场域** | Chat 上下文以**短期记忆**形式跨场域延续 —— Neo 在新场域可访问学员近期对话语境（如学员刚在大厅聊过的话题），保证体感连续 |
| **长期记忆 / 跨场域记忆** | 由 Neo 记忆系统承担 —— 含学员说过的话、做过的事、6 维画像变化点等，跨场域调用 |

具体短期窗口长度 / 长期记忆持久化 / 召回机制详见 § 4。

##### 3.4.1.5 高光与待办的跨场域闭环

- **高光卡片**：各场域产出 → 大厅高光区统一汇集；recap ④ 块按关联范围筛子集（详见 [05-learner/05-recap § 1.2.5]）
- **待办任务**：Activity 完成事件触发后产出 → 大厅待办清单；学员点掉 / Neo 跟进 / 允许点掉 在大厅闭环（详见 [05-learner/05-recap § 1.7.5] + [05-learner/05-recap § 1.9.7]）

#### 3.4.3 recap 例外清单

recap 因其"反思镜场域"定位，与 lecture / practice 在以下维度有差异：

| 维度 | lecture / practice | recap 例外 |
|------|-------------------|-----------|
| **入口锁定** | 严格顺序锁定 | 随时进，不锁前序完成度 |
| **完成判定** | 走完 SCO / 阶段 | Activity 完成事件概念（仅完整态首次触发）|
| **数据来源** | Activity 内 | KGP 配的关联 Activity 内 + Neo 跨场域综合 |
| **报告形态** | Activity 内一次性产物 | Course 级反思整合 / 完成事件层 + 自由复习层两层结构 |
| **报告更新** | 一次性 | Activity 完成事件前增量覆盖 / 触发后完成事件层永久 + 自由复习层覆盖式更新 |
| **承担信息边界** | 学习 + 练习内容 | 仅内容反思（不承担项目进度，由大厅承担）|
| **应用计划产出** | 不产出 | Neo 首聊态产出 1-3 条入大厅待办 |

详见 [05-learner/05-recap § 1] 各小节。

## 4 跨场域机制 · 系统级支持

> **作用**：把零散的"中枢 + 学习核心"串成一个连贯的学习体验——Neo 跨场域陪伴 + 学员视角的 AOM 演绎 + 数据持久化与剩余服务期。

### 4.1 Neo 跨场域陪伴与召回

- Neo 是同一个 AI 角色 跨大厅 + 3 场域统一身份
- 跨场域召回的产品体现（如学员在大厅说过的内容 → lecture / practice 主动调用）
- 反幻觉机制（详见 § 3.4.1）
- 与管理端 Ora 的边界（详见 § 3.1.3 视图 B）

### 4.2 学员视角的 AOM 演绎

- AOM 内容包对学员而言"不可见"——学员只看到 Course / Activity / SCO 的呈现
- 跨 Activity 的连贯性（前一个 Activity 的 FEEDBACK_COLLECT → 后续 FEEDBACK_REVIEW，通过 memory_id 关联）
- AOM 编排对学员体验的影响（典型 Course（Module）= N 个 lecture + 1 个 practice + 1 个 recap）
- 与 § 5 内容模型的对应关系

### 4.3 数据持久化与剩余服务期

- 学习记录的持久化范围（学员产出物 / Neo 对话 / 笔记 / 反思）
- 项目结营 → 剩余服务期切换 学员侧的体验变化（详见 [03-lifecycle/03-completion § 2]）
- 剩余服务期内学员可继续做的（复习 / 重练 / 与 Neo 对话）
- 剩余服务期与项目周期内学习记录的标记区分

### 4.4 Neo 卡片产出（5 类 × 3 场域 + 大厅）

> **本节是 5 卡片的产品形态 Truth Source** —— 落地形态 / 落地区 / 学员可见动作 / 跨场域矩阵。方法论级（5 卡片是什么 + 触发逻辑）见 § 2.6。
>
> **作用**：Neo 在大厅 + 3 学习场域均可主动产出 5 类结构化卡片，统一形态 + 统一落地区，让学员有稳定的认知预期（如"我的高光"区永远沉淀高光卡片，无论从哪个场域产生）。

#### 4.4.1 5 类卡片产品形态

| 卡片类型 | 产品形态 | 学员可见动作 | 落地区 / 沉淀位置 |
|---------|---------|------------|------------------|
| **askUserQuestion 卡片** | 1 个问题 + 2-4 选项按钮 + "其他"自填项 | 点击选项 / 自填"其他" / 忽略继续对话 | 对话流内即时呈现 / 不沉淀 |
| **课程卡片** | Course 名 + 简介 + "进入"按钮 | 点击"进入" → 跳转该 Activity；忽略继续对话 | 对话流内呈现 / 不沉淀（推荐历史在 Neo 记忆中）|
| **高光卡片** | 高光描述 + "收藏到我的高光"按钮 | 收藏 / 忽略（不可手动添加）| **大厅 → 看板 → 学习成果区 → "我的高光"**（详见 [05-learner/02-hub § 1.2.3]）|
| **任务卡片** | 任务描述 + "加入待办" / "暂不"按钮 | 加入待办 / 暂不（无强制）| **大厅 → 看板 → 项目总览区 → 待办清单**（详见 [05-learner/02-hub § 1.2.1]）|
| **知识卡片** | 工具/概念名 + 简介 + "查看"按钮 | 点击 → 跳转 4 库 - 工具库 / 忽略 | 对话流内呈现 / 跳转沉淀在 4 库；本期仅推**已解锁**工具 |

#### 4.4.2 5 类 × 3 场域 + 大厅 卡片矩阵

| 场域 \ 卡片 | askUserQuestion | 课程卡片 | 高光卡片 | 任务卡片 | 知识卡片 |
|-----------|:--:|:--:|:--:|:--:|:--:|
| **大厅 Hall** | ★ | ★ | ✓ | ★ | ★ |
| **lecture 授课** | — | — | ★ | ✓ | — |
| **practice 对练** | ✓ | — | ★ | — | ✓ |
| **recap 小结** | ✓ | — | ✓ | ★ | ✓ |

★ = 主场推送 · ✓ = 辅助推送 · — = 不推送

> **未来学习场域**（inquiry / assignment / sandbox（沙盘）等）：本期不实现 / 未列入矩阵。

**矩阵备注**：
- **课程卡片仅大厅推** —— 课程卡片是大厅入口形态 / 学员已在 Activity 内不再推切换；学员切换 Activity 走完成 Modal 流程（详见 [05-learner/03-lecture § 1.7.2]）
- **askUserQuestion**：大厅 ★ / practice 导入+复盘 ✓ / recap 全程 ✓ / lecture — 不推（不打断学习流推选项）
- **高光卡片** 3 学习场域差异化：lecture / practice 是"学员做"的高密度场域 → 高光主场 ★ / recap 是反思场域不产**新**高光 → ✓ 辅助（recap ④ 关键证据块**引用**关联范围已有高光 / 自由复习态识别学员新顿悟可推）/ 大厅 ✓ 辅助（学员主动分享时偶有顿悟）
- **任务卡片** 双主场 + 差异化：大厅 ★（日常行动建议入待办）/ recap ★（Activity 完成事件触发产 1-3 条入大厅待办）/ lecture ✓（学员说"我下周打算 X"即推）/ practice — 不推（行动承诺以报告"提升建议"段呈现 / 不单独推卡）
- **知识卡片** 大厅 ★ / practice 复盘+报告 ✓ / recap 完整+自由复习 ✓ / lecture — 不推（不打断学习流推工具卡）

**如何读本矩阵**：先看你的场域行 → 该场域 ★ 卡片是 Neo 主动产出的高频形态、✓ 卡片是辅助形态（按需触发）、— 表示该场域不出此卡。

#### 4.4.3 跨场域统一约束

- **形态统一**：5 类卡片在所有场域呈现形态一致（按 § 4.4.1 表格）
- **落地区统一**：高光 → 大厅"我的高光"；任务 → 大厅待办清单；3 类一致汇集到大厅，无论从哪个场域产出
- **不重复推送**：Neo 内部记忆同一识别条件已推过的卡，不在短时间内重复推（具体阈值由研发实现）
- **不强迫动作**：所有卡片可忽略，忽略不影响对话流 / 不影响完成判定
- **学员手动产出**：笔记由学员手动通过笔记悬浮球添加（详见 [05-learner/02-hub § 1.3]）—— **笔记不是 Neo 卡片产出**，是学员侧产物
