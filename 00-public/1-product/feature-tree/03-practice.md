## 一句话定位

Actor 角色扮演 + Neo 三任务 + 后台导演。核心目的是能力评估和行为训练，不做知识传递。

---

## 功能结构

```
A. 三角色架构 ★v0.4.0 核心新架构
   A1 Actor（演员）— 前台角色扮演对象（人设 · 情绪 · 防剧透 · 表情切换 P1）
   A2 Neo（Tutor）— 三任务：铺垫引导（B1）· 答疑与辅导（B2）· 复盘追问（B3）
   A3 导演（后台，不暴露学员）— 分析学员发言 · 决定 Actor 走向 · 提供 Neo 辅导上下文

B. 三阶段流程
   B1 阶段一：情景导入（信息卡片三组 · Neo 铺垫不透底线 · 开始对练按钮）
   B2 阶段二：角色扮演 ★左右双栏对话
       B2.1 左栏 Actor 对话 / 右栏 Neo 辅导（双栏分离）
       B2.2 Actor 直接开场（无旁白）
       B2.3 10-15-20 轮对话
       B2.4 Neo 答疑与辅导（给方向不给答案）
       B2.5 情绪响应（缓和 / 升级，受天花板）
       B2.6 对练结束（Actor 结束语 + 过渡提示，非突然静默）
   B3 阶段三：反思复盘（Actor 退出 · 正向反馈先行 · 苏格拉底追问 · 对话回看面板 · ★必须给总结不留开放）

C. 剧本控制元素（映射 AOM）
   C1 characterProfile（心理画像，待 KGP 字段定义）
   C2 behaviorChain（递进反应链）
   C3 constraints（防剧透指令）
   C4 successCriteria（分阶段通关条件）
   C5 emotionCeiling（情绪天花板）
   C6 rubric 复用 competencyGoals.rubric（L1-L5 = 能力熟练度 5 级）

D. 对练多版本
   D1 Segment 版本管理（销售版 / 产品版 / 管理版 等）
   D2 情景导入阶段内置打标（因标签限 Activity 内，无法读取授课打标）
   D3 降级策略（打标失败 → defaultSegment）

E. 重练 + 断点续播
   E1 断点续播（轮次信息 + 导演状态恢复，确保 Actor 连贯）
   E2 重练仅重练 B2 角色扮演阶段（保留 B1 情景导入信息）
   E3 数据独立（新旧分开存储，不覆盖不追加）
   E4 报告取值规则（默认取最近一次完成）

F. 角色形象展示
   F1 文生图头像（左栏展示）
   F2 未来表情切换（P1 预留接口）

G. 能力评估碎片产出
   G1 对练完成后生成能力评估碎片
   G2 写入统一 Database
   （碎片 schema 由 09-A 定义，本模块产出）

H. 对练场域布局
   H1 可折叠导航 + 左栏看板（按阶段变化）+ 右栏对话
   H2 演绎阶段左右双栏对话（区别于其他场域，见 07.K6）

附录 Z. 已知限制（本期不做）
   Z1 恶意测试 AI 底线 — 不做主动防御，依赖情绪天花板兜底
   Z2 实时教练介入 — 引导职能放反思阶段
   Z3 多人模拟 — 本期仅 1v1
   Z4 费曼 / 角色互换 / 辩论 — 未来通过不同剧本实现
```

## 术语速查

- **导演** = A3 后台 AI 角色，分析学员发言并指挥 Actor（不暴露给学员）
- **rubric** = 能力评分标准表（L1-L5 = 5 级熟练度）
- **Segment / defaultSegment** = 对练剧本的多版本 / 打标失败时的兜底版本
- **ASSESSMENT_TAG** = 打标 SCO（见 `02.C7` / `09-A.A`）
- **情绪天花板 emotionCeiling** = 剧本定义 Actor 情绪升级的上限（避免学员被无限激怒）
