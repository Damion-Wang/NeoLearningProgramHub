# 通用模块总览

## 定位

通用模块是跨端口、跨页面的基础功能集合。不属于学员端场域，也不属于管理端看板，而是所有页面共享的通用能力。全部纯 GUI，不涉及 Agent。

## 顶部通用栏（Topbar）设计

### 布局

```
┌──────────────────────────────────────────────────────────┐
│ [Logo] [当前端口名称]                     [🔔铃铛] [👤头像] │
└──────────────────────────────────────────────────────────┘
```

- **左侧**：产品 Logo（可被管理端「平台个性化」配置替换为客户企业 LOGO）+ 当前端口标识（"学员端" / "管理端"）
- **右侧**：两个可交互元素

### 铃铛🔔 — 消息提醒入口

- 有未读消息时显示红点
- 点击后下拉消息面板（消息列表 + 右上角"全部已读"按钮）
- 本期消息只读，不支持回复
- 三类消息：平台消息 / 系统消息（自动催学）/ 运营消息（群发或定点）
- 详见 [02-notification.md](02-notification.md)

### 头像👤 — 通用功能入口

点击后展开下拉菜单：

| 菜单项 | 交互 | 说明 |
|--------|------|------|
| **账号管理** | 弹窗 | 个人信息查看编辑 + 密码修改 |
| **个人设置** | 弹窗 | AI语速（本期）+ 页面风格（本期尽量上线）+ 语种/个人偏好（待上线） |
| **帮助** | 新窗口 | 产品使用手册（文档形式） |
| **端口切换** | 展开子下拉列表 | 列出当前账号可用的端口（学员端/管理端），选择后页面切换 |
| **退出登录** | 弹窗确认 | "确定退出登录？" → 确认后回到登录页 |

账号管理、个人设置、帮助三个页面详见 [01-general.md](01-general.md)

### 端口切换细节

- 点击"端口切换"后展开子下拉列表
- 列出当前账号拥有的所有角色对应的端口
- 当前端口高亮标识，其余可点击切换
- 只有学员角色的用户看不到"管理端"选项
- 不在学员名单中的管理员看不到"学员端"选项（无法进入学员端）
- 同时在学员名单中的管理员切换到学员端时，按正常学员身份学习（不是体验模式）
- 切换后页面刷新为目标端口的默认页面

### 两端差异

通用栏 UI 在学员端和管理端完全一致，差异仅在于：
- **端口切换的可选项**：根据账号角色动态显示（学员只看到学员端，管理员看到两个端口）
- **消息内容**：学员收学习提醒，管理员收催学和进度相关消息

## 反向补充：场域层交互范式统一

学员端 4 个场域（授课/对练/调研/报告）统一使用如下布局范式：

```
┌─────────────────────────────────────┐
│  Topbar (h-14, 固定)                 │
├─────────────────────────────────────┤
│  Sub-bar (h-11, 固定)                 │
│  [☰] 大厅 > 课程 > Activity  | 状态  │
├───────┬─────────────────────────────┤
│  [抽屉] │                             │
│  覆盖式  │   中间全幅内容区              │
│  Z-50   │                             │
│  点外部 │                             │
│  关闭   │                             │
│         │        [AI Chat 固定右侧]    │
└───────┴─────────────────────────────┘
```

### 组件规范

| 组件 | 规则 |
|------|-----|
| Topbar | h-14 固定；含 Logo + 端口 chip + 铃铛 + 头像 |
| Sub-bar | h-11 固定；左侧面包屑+toggle，右侧状态 |
| 左侧抽屉 | top:56px（不覆盖 Topbar）；w-280px；z-index:51；背景半透明+blur |
| 主内容区 | 全幅（抽屉收起时）；根据场域固定或滚动 |
| AI Chat | 右侧固定 w-420px；含 Neo 品牌 Hero（h-220px）+ 消息流 + 输入框 |

### Sub-bar 面包屑统一格式

```
[☰ toggle]  大厅 > 课程 > Activity  ...间距...  右侧 状态/进度
```

- "大厅"永远可点击，链到 hall-daily
- 其他部分是静态文字
- 右侧状态每个场域不同：
  - Lecture: `SCO N/12 · PPT/VIDEO/QUIZ`
  - Practice: `进度 N/5 轮 · 进行 MM:SS`
  - Report: 空（报告无进度概念）
  - Inquiry: `阶段 N/3 · 说明/核心调研/确认`

## 反向补充：AI Chat Header 简化规范

Chat Header 只显示：
- AI 头像（10×10 圆角）
- AI 名字（Neo / Ora）

**不显示**：
- 副标题（如"AI TUTOR · Activity 内 · AOM 脚本"——内部术语）
- 状态 chip（如"授课中""辅导中"）

**Chat 消息**：不加业务标签（如 MENTOR·OPENING / 授课·SCO 06），只保留头像+气泡+时间戳。

原因：内部术语（AOM、SCO-ID 等）对用户无意义，增加认知负担。
