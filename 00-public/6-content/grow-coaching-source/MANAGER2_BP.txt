# Activity: Activity 02: 标准化流程引导：催化主动型员工的产出

## Block 1: [Type: Hook] - 摆脱“保姆式”管理的困境
* **Composition**: Pure PPT
* **Block Discription**: 本模块通过对比“管理者疲于奔命”与“员工有心无力”的痛点场景，揭示辅导高意愿员工时的常见误区，引发管理者对“如何正确引导主动型员工”的强烈求知欲。
* **Block Intent**: 作为课程的开篇（Hook），旨在通过强烈的视觉对比和痛点共鸣，打破管理者“只要员工愿意干，我就不用管”或“员工搞不定，我就直接上手”的惯性思维，建立学习“标准化引导流程”的必要性连接。

### Segment 1.1: [Type: PPT] - 你的“好心”是否正在扼杀员工的主动性？

**[If PPT]**
* **Layout Strategy**: Overview + Contrast
* **Context Prompt (内容上下文)**: 
    *   左侧展示“保姆式管理者”的典型画像：面对员工的困难，管理者直接给出指令和方案，背上背着全是员工的“猴子”（责任），虽然问题解决了，但管理者精疲力竭。
    *   右侧展示“主动型员工”的困境：员工本身具备高意愿（想解决问题），但因为缺乏方法或资源陷入瓶颈。面对管理者的直接干预，员工从“我要做”退化为“要我做”，眼神中失去了光彩。
    *   中间通过裂痕或问号连接，引出核心设问：“面对想干事却干不好的员工，你是要做‘救火队长’，还是‘催化剂’？”
* **Design Intent（设置意图）**: 利用强烈的反差视觉，直击管理者痛点，指出过度干预对主动型员工的隐性伤害，为引入GROW标准化流程做铺垫。
* **Visual Intent (视觉与图例)**: 
    *   屏幕被物理中轴线一分为二。
    *   左侧区域色调偏冷灰，画面中央是一位眉头紧锁的管理者，正对着电脑疯狂敲击，身后堆满了标有“To-Do”的文件山，压得他喘不过气。
    *   右侧区域色调偏暖但被压暗，一位年轻员工站在一旁，手中拿着未完成的方案，表情从期待转为无奈，身体姿态呈现后退状。
    *   画面中央是一道明亮的闪电状分割线，分割线上方浮现一行醒目的红色大字：“你的‘帮忙’，可能是一种‘掠夺’。”
    *   底部横幅渐入，文案为：“如何释放高意愿员工的潜能？”

## Block 2: [Type: Explain] - 识别你的“催化”对象
* **Composition**: Pure PPT
* **Block Discription**: 本模块详细定义了“主动型辅导场景”的准入标准，即如何通过“责任意愿”和“问题意识”两个维度，精准筛选出适合使用本套GROW流程的员工。
* **Block Intent**: 在进入具体流程前，先建立“条件型知识”的认知。确保管理者明白这套高强度的追问流程不是万能药，仅适用于特定状态的员工，避免因对象错配导致的辅导失败。

### Segment 2.1: [Type: PPT] - 主动型辅导的“双高”门槛

**[If PPT]**
* **Layout Strategy**: Overview + Quadrant
* **Context Prompt (内容上下文)**: 
    *   介绍“主动型辅导场景诊断”模型。
    *   定义X轴为“责任意愿”（从推诿责任到主动承担），Y轴为“问题意识”（从无感到敏锐觉察）。
    *   重点讲解第一象限（右上角，双高）：这是我们需要“催化”的对象。他们既想解决问题（High Will），又能清晰看到问题（High Awareness）。
    *   对比其他象限：如“有心无力”（需技能培训）、“有力无心”（需意愿激励），强调本课程仅聚焦于第一象限。
* **Design Intent（设置意图）**: 帮助管理者建立清晰的筛选标准，明确本课程所授方法的适用边界，防止在低意愿员工身上滥用“追问”导致抵触。
* **Visual Intent (视觉与图例)**: 
    *   屏幕展示一个标准的四象限坐标系。
    *   X轴从左至右延伸，标签为“责任意愿（Want to）”；Y轴从下至上延伸，标签为“问题意识（See it）”。
    *   右上角的第一象限（Q1）被高亮显示，背景呈现充满活力的橙色光晕，内部图标为一个正在奔跑且头顶有灯泡亮起的小人，标签为“主动型员工（催化对象）”。
    *   其余三个象限（Q2, Q3, Q4）处于半透明的压暗状态，仅保留淡淡的轮廓和灰色标签。
    *   Q1区域旁浮现注释框：“高意愿 + 高觉察 = 启动GROW引导的信号”。

### Segment 2.2: [Type: PPT] - 信号捕捉：听其言，观其行

**[If PPT]**
* **Layout Strategy**: Focus + Universal
* **Context Prompt (内容上下文)**: 
    *   具体描述“高责任意愿”和“高问题意识”的行为信号。
    *   高责任意愿表现为：主动发起沟通、使用“我要解决”而非“要我解决”的语言、不推诿。
    *   高问题意识表现为：能准确描述现状痛点、预判对目标的负面影响、不需管理者唤醒即可感知危机。
* **Design Intent（设置意图）**: 将抽象的象限概念转化为可观察的具体行为指标，提升管理者在实际工作中的识别能力。
* **Visual Intent (视觉与图例)**: 
    *   左侧50%区域展示一张特写图片：一位员工正拿着笔记本眼神坚定地看着镜头，仿佛在汇报工作。
    *   右侧50%为文本区域，分为上下两组关键信号列表。
    *   上组标题“责任信号”，下列出“主动敲门”、“使用‘我’为主语”等关键词，字体加粗。
    *   下组标题“意识信号”，下列出“痛点清晰”、“预判风险”等关键词。
    *   图片中的员工嘴边浮现气泡框，文字为：“经理，关于C计划的延期，我发现了一个关键卡点，想和您讨论下。”（对应案例中的张主动）。

## Block 3: [Type: Observe] - 诊断实战演练
* **Composition**: Pure Quiz
* **Block Discription**: 通过一个具体的员工行为描述场景，检验管理者是否能准确判断该员工是否符合“主动型辅导”的标准。
* **Block Intent**: 这是一个Type A（引发思考/轻量验证）的观察环节，位于理论讲解之后、流程讲解之前，用于巩固“诊断”这一前置条件，确保地基稳固。

### Segment 3.1: [Type: Quiz] - 这个员工适合“被催化”吗？

**[If Quiz]**
* **Type**: Type A
* **Question**: 你的下属小李跑来找你：“老板，客户那边一直在变需求，导致我们现在的开发进度完全乱了（痛点清晰）。虽然很难，但我不想就这么烂尾，我想试着去把进度追回来（意愿强烈），但现在有点不知道从哪下手。” 请问，针对小李当下的状态，你是否应该启动标准化的GROW引导流程？
* **Explaination**: 是的，应该启动。小李清晰地描述了“需求变更导致进度混乱”的现状（高问题意识），并明确表达了“不想烂尾、想追回进度”的决心（高责任意愿），完全符合“主动型员工”的双高特征。此时他缺乏的是方法，而非意愿，正是管理者从旁引导、催化方案的最佳时机。
* **Design Intent**: 确认学员能够识别“双高”信号，区分“抱怨”与“求助”，为后续学习辅导流程确立正确的应用对象。

## Block 4: [Type: Explain] - 流程启动：同理与对齐
* **Composition**: Pure PPT
* **Block Discription**: 本模块讲解GROW流程的前两个阶段（Goal & Reality）。重点在于如何通过同理倾听平复员工情绪，并引导其将模糊的愿望转化为SMART目标，同时盘点已付出的努力。
* **Block Intent**: 讲授标准动作。对于主动型员工，管理者往往容易忽略“倾听”和“现状盘点”，直接跳到方案。本模块旨在纠正这一急躁心理，强调“慢即是快”，通过对齐目标和现状为后续方案爆发蓄力。

### Segment 4.1: [Type: PPT] - 倾听是理性的“破冰船”

**[If PPT]**
* **Layout Strategy**: Overview + Flow
* **Context Prompt (内容上下文)**: 
    *   讲解“同理倾听与反思引导”的步骤。
    *   第一步：镜像反馈。像镜子一样复述员工的焦虑和逻辑（如“所以你担心的是...”），让员工感到被理解，降低防御。
    *   第二步：深度反思。在情绪平复后，引导员工从抱怨外部环境（如“配合度低”）转向内部归因（如“沟通机制缺失”）。
    *   强调：不评判，只映射。
* **Design Intent（设置意图）**: 教会管理者如何处理员工初期的焦虑情绪，通过镜像技术建立信任场域，为理性对话铺路。
* **Visual Intent (视觉与图例)**: 
    *   屏幕展示从左至右流动的两个核心步骤卡片。
    *   左侧卡片标题“镜像反馈”，图标为一面镜子反射出波浪线（代表情绪），下方文字说明“复述确认，降噪情绪”。
    *   右侧卡片标题“深度反思”，图标为一个钻头向下钻探，下方文字说明“从表象症状钻取根本原因”。
    *   两张卡片之间由一个平滑的箭头连接，箭头上方标注“情绪转理性”。
    *   背景隐约可见声波纹路，暗示“倾听”的氛围。

### Segment 4.2: [Type: PPT] - 目标与现状的“双向校准”

**[If PPT]**
* **Layout Strategy**: Overview + Contrast
* **Context Prompt (内容上下文)**: 
    *   讲解Goal与Reality的对齐技术。
    *   Goal端：从“模糊愿景”（如“我想控制进度”）转化为“SMART目标”（如“下个里程碑延期控制在1周内”）。强调由员工口述目标。
    *   Reality端：进行“已付努力清单盘点”。询问“你已经做了什么？”，确认事实，排除无效路径。
    *   对比：左边是“模糊的焦虑”，右边是“清晰的战场”。
* **Design Intent（设置意图）**: 明确GROW中G和R的标准动作，强调“员工自主陈述”的重要性，确立对话的事实基础。
* **Visual Intent (视觉与图例)**: 
    *   屏幕中央有一道垂直分割线。
    *   左侧区域标题“Before：模糊的焦虑”，画面中漂浮着散乱的云团，文字标签为“想解决问题”、“大概也许”、“尽力而为”。
    *   右侧区域标题“After：清晰的战场”，画面中是整齐的积木结构。
    *   积木上方是“SMART目标”（具体数字、时间），下方是“现状基石”（已做事项清单）。
    *   一个箭头从左侧云团指向右侧积木，标注文字“自主陈述与盘点”。

## Block 5: [Type: Case] - 案例：张主动的“迷雾突围”
* **Composition**: Pure Video
* **Block Discription**: 演绎案例Case 3。展示项目负责人张主动在焦虑状态下找到王经理，王经理如何通过倾听和提问，帮助他理清思路，定下SMART目标。
* **Block Intent**: 通过正向案例（Video），将前序Block 4的理论知识（倾听、SMART目标转化）具象化。让学员看到“同理倾听”如何瞬间降低员工焦虑，以及“目标对齐”如何让问题变得可控。

### Segment 5.1: [Type: Video] - 从“我该怎么办”到“我要达成它”

**[If Video]**
* **Script Prompt (分镜剧本)**:
    *   **场景1**: 嘈杂的办公区角落，张主动满头大汗地找到王经理，语速极快且焦虑：“王总，C计划彻底乱了，研发那边根本推不动，需求变来变去，我真不知道该怎么控制进度了！”
    *   **剧情描述**: 王经理没有打断，而是身体前倾，眼神专注。等张主动发泄完，王经理平静地运用镜像反馈：“先别急，我听到你现在最头疼的是需求变更导致的研发配合问题，这让你对进度失控感到很焦虑，是这样吗？”张主动愣了一下，深吸一口气，肩膀放松下来：“对，就是这种无力感。”
    *   **场景2**: 会议室白板前。王经理递给张主动一支笔：“既然现状如此，那你觉得在现有资源下，我们能争取的最好结果是什么？能给我一个具体的数字吗？”
    *   **剧情描述**: 张主动拿着笔沉思片刻，在白板上写下“延期1周”。王经理追问：“具体是哪个里程碑？1周是底线还是目标？”张主动眼神逐渐聚焦，擦掉重写：“保证下个月15号的里程碑交付，延期严格控制在5个工作日内。”
    *   **场景3**: 两人看着白板上的目标。张主动眼神从迷茫转为坚定：“有了这个目标，我就知道接下来该去盘点哪些资源了。”
* **Design Intent（设置意图）**: 真实还原“情绪平复”到“理智回归”的过程，展示管理者如何通过不给建议、只给提问，让员工自己找回掌控感。

## Block 6: [Type: Explain] - 核心算法：方案的极限压榨
* **Composition**: Pure PPT
* **Block Discription**: 本模块深入GROW流程最核心的Options（方案）阶段。详细拆解“还有什么”追问循环、零批判场域构建，以及最重要的“询问许可”机制。
* **Block Intent**: 这是本课程的“核武器”。针对主动型员工，最大的价值在于激发其潜能。本模块旨在教会管理者如何通过“逼问”榨干员工思维库存，以及如何在必须介入时，通过“询问许可”保护员工的责任感（这是避免保姆式管理的关键）。

### Segment 6.1: [Type: PPT] - “还有什么”：思维的挤牙膏战术

**[If PPT]**
* **Layout Strategy**: Focus + Canvas
* **Context Prompt (内容上下文)**: 
    *   讲解“方案发散递归机制”。
    *   核心隐喻：好的方案就像挤牙膏，最有价值的那一段，往往在最后一次用力时才会出来。
    *   操作动作：机械式重复“还有吗？”（What else?），直到员工明确表示“没有了”。
    *   目的：穷尽思维库存，逼出潜意识里的创新解法，打破常规路径依赖。
* **Design Intent（设置意图）**: 利用生动的隐喻（挤牙膏），让管理者记住“追问”的必要性，克服“见好就收”的习惯，追求方案的数量和广度。
* **Visual Intent (视觉与图例)**: 
    *   背景是一张极具张力的大图：一只手正在用力挤压一支几乎干瘪的牙膏管。
    *   牙膏管口挤出了一段闪着金光的牙膏，形状像一个灯泡（代表创新方案）。
    *   牙膏管身上印着“常规思维”的字样，而那段金色的牙膏旁标注“创新突破”。
    *   画面右侧展示巨大的金句文本：“最有价值的方案，往往在最后一次‘还有什么？’之后出现。”

### Segment 6.2: [Type: PPT] - 零批判场域：让子弹飞一会儿

**[If PPT]**
* **Layout Strategy**: Overview + Contrast
* **Context Prompt (内容上下文)**: 
    *   讲解“零批判头脑风暴场域构建”。
    *   左侧“禁区”：评价性语言（“这行不通”、“这主意不错”）。任何评价都会导致思维坍缩。
    *   右侧“绿区”：只记录，不评判。鼓励数量，鼓励荒谬。
    *   金句强化：在“方案”阶段，“正确”是“可能性”的敌人。
* **Design Intent（设置意图）**: 强调管理者在发散阶段的“闭嘴”纪律，确保员工敢于提出非传统方案，维护心理安全感。
* **Visual Intent (视觉与图例)**: 
    *   屏幕中央显示一个大大的“VS”图标。
    *   左侧区域色调灰暗，画着一个红色的禁止符号，符号内是“过早评判”四个字，周围散落着破碎的灯泡（代表被扼杀的想法）。
    *   右侧区域明亮开阔，画着一个绿色的护盾，护盾内是“纯粹记录”四个字，周围漂浮着各种颜色、大小不一的完整灯泡。
    *   底部横幅展示金句：“在方案阶段，‘正确’是‘可能性’的敌人。”

### Segment 6.3: [Type: PPT] - 询问许可：敲门再进入

**[If PPT]**
* **Layout Strategy**: Overview + Flow
* **Context Prompt (内容上下文)**: 
    *   讲解“建议授权交互协议”。这是防止责任转移的关键。
    *   Step 1: 确认枯竭。确认员工已经说出“我想不到了”。
    *   Step 2: 询问许可（敲门）。话术：“我有一个补充视角，可以分享吗？”
    *   Step 3: 分享建议。使用协商式语气，非命令式。
    *   Step 4: 归还决策。话术：“你觉得这个建议如何？你会采纳吗？”
* **Design Intent（设置意图）**: 详细拆解管理者介入建议的标准动作。通过“敲门”和“归还”两个动作，确保即便管理者给了建议，责任依然在员工肩上。
* **Visual Intent (视觉与图例)**: 
    *   屏幕展示四步阶梯状的流程图，从左下向右上延伸。
    *   第一步卡片：“确认枯竭”，图标为倒空的杯子。
    *   第二步卡片（高亮强调）：“询问许可”，图标为一只手正在敲门，门上写着“员工领地”。
    *   第三步卡片：“提供资源”，图标为双手捧着一个拼图块（而非指令）。
    *   第四步卡片：“归还决策”，图标为将印章交还给员工。
    *   连接线流畅，第二步卡片周围有光圈脉冲，强调这是最易被忽略的环节。

## Block 7: [Type: Observe] - 关键时刻的抉择
* **Composition**: Pure Quiz
* **Block Discription**: 设置一个管理者忍不住想给建议的场景，检验学员是否掌握了“询问许可”的触发机制和话术。
* **Block Intent**: 这是一个Type B（知识验证）的观察环节。重点考核“建议授权交互协议”的应用，防止管理者学了流程后依然习惯性“抢答”。

### Segment 7.1: [Type: Quiz] - 你能忍住不“抢答”吗？

**[If Quiz]**
* **Type**: Type B
* **Question**: 在辅导小李时，你发现他提出的方案都局限在技术层面，完全忽略了跨部门沟通这个关键点。你心里很急，这时候你应该怎么做？
* **Explaination**: 正确的做法是：继续追问“还有吗？”，尝试引导他转换视角（如“如果从人的角度看呢？”）。如果他确实想不到了（思维枯竭），再执行“询问许可”：“关于沟通方面，我有个经验，想听听吗？”。严禁直接打断说：“你应该去搞搞关系”。只有先敲门，建议才能作为资源被接纳，而不是作为指令被执行。
* **Design Intent**: 验证学员对“建议介入时机”和“询问许可话术”的掌握程度，强化“责任归属”的意识。

## Block 8: [Type: Explain] - 落地闭环：从想法到算法
* **Composition**: Pure PPT
* **Block Discription**: 讲解GROW流程的最后一步Way Forward（行动）。重点在于锁定“第一步行动”和量化“执行把握度”。
* **Block Intent**: 确保方案不悬浮。通过定义最小启动动作和时间节点，将心理契约转化为物理行动，完成辅导闭环。

### Segment 8.1: [Type: PPT] - 锁定“第一公里”与信心指数

**[If PPT]**
* **Layout Strategy**: Focus + Universal
* **Context Prompt (内容上下文)**: 
    *   讲解“行动落地闭环锁定”的三个要素。
    *   1. 首步行动（First Step）：离开会议室后的第一个原子动作（如“发邮件”），打破启动摩擦力。
    *   2. 时间锚点：具体的截止时间（Deadline）。
    *   3. 把握度量化：询问“0-10分，你有几分把握？”。低于7分必须回炉重造，挖掘顾虑。
* **Design Intent（设置意图）**: 提供一套将方案转化为执行计划的标准模板，特别是“把握度量化”这一工具，能有效预警执行风险。
* **Visual Intent (视觉与图例)**: 
    *   左侧展示一个放大的“起跑器”图片，起跑器上写着“First Step”。
    *   右侧列出三个关键检查项：
    *   1. “原子动作”：图标为微小的齿轮，文字说明“离开即执行的最小单位”。
    *   2. “时间戳”：图标为日历打钩，文字说明“本周五下午4点”。
    *   3. “信心仪表盘”：图标为一个类似汽车转速表的仪表盘，指针指向“8”，红色区域在0-6，绿色区域在7-10。
    *   仪表盘下方高亮文字：“低于7分 = 方案存疑，需重新辅导”。

### Segment 8.2: [Type: PPT] - GROW全景流程图

**[If PPT]**
* **Layout Strategy**: Overview + Flow
* **Context Prompt (内容上下文)**: 
    *   总结串联整个GROW流程，形成一张完整的作战地图。
    *   G/R：同理倾听 -> 目标对齐 -> 现状盘点。
    *   O：循环追问 -> 询问许可 -> 方案优选。
    *   W：首步行动 -> 信心确认 -> 闭环锁定。
    *   强调这是一套针对“主动型员工”的标准化闭环。
* **Design Intent（设置意图）**: 帮助学员在脑海中建立完整的流程视图，将碎片化的知识点（倾听、追问、授权、锁定）串联成线，便于记忆和提取。
* **Visual Intent (视觉与图例)**: 
    *   屏幕展示一条横向贯穿的流程路径，分为四个阶段颜色块。
    *   第一阶段（蓝色）“G/R 对齐”：包含“镜像反馈”、“SMART目标”节点。
    *   第二阶段（橙色，最长）“O 催化”：包含一个循环图标代表“What else”，以及一个门形图标代表“询问许可”。
    *   第三阶段（绿色）“W 落地”：包含“首步行动”、“信心打分”节点。
    *   流程的终点是一个“火箭升空”的图标，象征员工产出被激活。

## Block 9: [Type: Observe] - 综合应用挑战 (Transfer)
* **Composition**: Pure Quiz
* **Block Discription**: 设置一个全流程的综合应用场景，要求学员在面对一个既定情境时，能够按顺序识别出正确的辅导步骤或纠正错误的辅导行为。
* **Block Intent**: 这是一个Type B（知识验证）但功能上作为Transfer（迁移应用）的环节。位于课程末尾，旨在检验学员是否能将整套GROW流程在脑海中融会贯通，并应用到模拟的业务难题中。

### Segment 9.1: [Type: Quiz] - 你的辅导闭环了吗？

**[If Quiz]**
* **Type**: Type B
* **Question**: 辅导结束前，员工小张兴致勃勃地列出了三个大方案。你看了看表，时间到了，于是你说：“很好，就按这三个方案去办吧，下周汇报结果。” 请问，作为一次针对主动型员工的标准化辅导，你的这个结尾遗漏了哪个最致命的环节？这会导致什么后果？
* **Explaination**: 遗漏了“行动落地闭环锁定（Way Forward）”。虽然有了方案，但缺乏“首步行动”的定义和“执行把握度”的量化确认。这会导致方案停留在口头兴奋阶段，员工离开后可能因为第一步太难而拖延，或者因为潜意识信心不足而执行打折。正确的做法是追问：“为了实现这三个方案，你出门后要做的第一件事是什么？你有几分把握？”
* **Design Intent**: 检验学员对流程完整性的掌握，特别是对“虎头蛇尾”现象的警惕，确保辅导成果能真正穿透会议室的墙壁落地。

## Block 10: [Type: Review] - 成为思维的催化剂
* **Composition**: Pure PPT
* **Block Discription**: 课程总结与升华。回顾核心知识点，并以金句收尾，强化管理者从“指令下达者”转型为“思维催化剂”的角色认知。
* **Block Intent**: 收束全课，提炼精华，通过角色身份的重新定义（Identity Shift），激发管理者在未来实践这套流程的内在动力。

### Segment 10.1: [Type: PPT] - 你的新角色：思维催化剂
