# AI 老师产品设计辩论 — 第二次辩论 R2 汇总

**日期：** 2026-04-13
**议题：** 5个核心问题深挖

---

## 一、各问题建议方案

### 问题1：分段式自适应——打标SCO独立化

**建议方案：显式打标SCO + 概率累积判定**

- 打标SCO作为AOM中独立的SCO类型（`scoType: ASSESSMENT_TAG`），KGP在Activity编排时显式放置
- 打标SCO内部汇总前序互动SCO的累积信号，用概率模型输出标签（不是自己做独立判定）
- 打标结果存储为**确定标签 + 置信度**：标签供报告和路径切换使用，置信度供系统调试
- 低置信度（<0.6）时降级为默认Segment版本，不硬猜
- 跨Activity传递：打标结果写入学员标签表，对练Activity可读取"最新有效标签"
- KGP不需要配概率权重——概率模型参数由技术团队按互动类型设定默认值

**可直接写入需求文档的结论**：
- AOM数据模型新增`ASSESSMENT_TAG`类型SCO
- 打标结果存储字段：`tag_dimension`、`assigned_tag`、`confidence`、`evidence_summary`、`effective_until`
- 对练Activity增加`segmentSelection.readTagFrom`字段，支持跨Activity读取标签

### 问题2：三栏布局联动设计

**建议方案：场景预设比例 + 通用看板组件库 + 双向联动**

- 为每种场景设定固定的默认三栏比例（不做自动弹性调整），用户可手动拖拽微调
- 建立通用看板组件库，系统根据SCO类型自动挂载默认组件（KGP可选覆盖）
- 双向联动必做：看板元素点击→触发Agent解读；Agent提到的实体→看板高亮/跳转

**建议的预设比例**：

| 场景 | 左:中:右 |
|------|---------|
| 大厅 | 15:50:35 |
| 授课-PPT | 10:55:35 |
| 授课-视频 | 10:65:25 |
| 授课-Quiz | 10:40:50 |
| 对练-导入 | 10:55:35 |
| 对练-演绎 | 10:35:55 |
| 对练-复盘 | 10:50:40 |
| 测评-对话 | 10:40:50 |
| 测评-量表 | 10:60:30 |
| 报告 | 10:55:35 |
| HR/运营端 | 15:50:35 |

**看板默认组件**：
- Quiz对话 → 关联知识卡片 + 进度指示
- 对练演绎 → 角色形象 + 场景信息
- 对练复盘 → **对话回看面板**（新组件，可点击某轮触发Agent分析）
- 纯讲述 → PPT课件

**可直接写入需求文档的结论**：
- 三栏默认比例定义（上表）
- 新增"对话回看"看板组件，用于反思复盘场景
- 看板→对话、对话→看板的双向联动API定义

### 问题3：闭环反馈的完整数据链路

**建议方案：结构化采集 + 必选/可选引用槽 + 独立SCO设计**

- 课前采集数据用结构化标签格式：`structured_tags`（topic_dimension, scenario_type, scenario_keywords, emotional_state, specific_challenge等）+ `raw_dialogue`双层存储
- 课中引用用"必选+可选"混合模式：KGP在关键知识点设1-2个必选引用槽（AI必须在此引用学员场景），其余位置AI自主判断
- 闭环反馈拆成两个独立SCO（不做"双端SCO"复杂设计）：
  - `FEEDBACK_COLLECT` SCO：Activity开头，课前采集
  - `FEEDBACK_REVIEW` SCO：Activity末尾，课后复盘
  - 两者通过memory_id关联
- 辅导教练引用规则：
  - 回到大厅时必然触发1次（1-2句话）
  - 话题相关时AI自主判断引用
  - 去掉"主动回顾"——不打扰发呆的学员

**可直接写入需求文档的结论**：
- AOM新增`FEEDBACK_COLLECT`和`FEEDBACK_REVIEW`两种SCO类型
- 授课脚本新增`referenceSlots`字段（含`required`标志）
- 课前采集记忆的存储模板定义
- 辅导教练回到大厅时必然引用一次最近学习

### 问题4：报告排名设计

**建议方案：学员端成长优先 + HR端排名透明**

**学员端报告**：
- 主信息：个人成长时间线（折线图展示各维度跨Course的进步趋势）
- 辅助信息：匿名分位区间（前25%/中间50%/后25%），用柔和灰色标尺展示
- 不展示具体排名名次和分组标签
- 文案用成长型语言

**HR端报告**：
- 完整的成长幅度排名表（可排序）
- 各能力维度全员均分和分布图（柱状图/热力图）
- 聚合统计：平均成长幅度、最大提升维度、需关注学员Top 3
- HR可见分组（高成长/稳健/潜力），学员不可见

**成长幅度计算**：
- 优先：课前BEI基线 vs 课后评分
- 备选：前1/3阶段均值 vs 后1/3阶段均值
- 引入标准化，消除初始分数偏置

**跨Course统一维度**：Project级定义3-5个通用能力维度，各Course细分维度映射到通用维度

**可直接写入需求文档的结论**：
- 学员端报告展示个人成长时间线+匿名分位区间（不展示名次）
- HR端报告展示完整成长幅度排名+聚合统计
- AOM Report模块新增`rankingDimensions`和`groupingStrategy`配置
- 需要在Project级定义通用能力维度映射体系

### 问题5：记忆工程触发时机和规则

**建议方案：分层触发 + 被动信号辅助 + 跨Course打通**

**辅导引用触发**：
- 必然触发：完成Activity回大厅时，教练主动提及（1-2句话）
- 条件触发：话题相关时AI自主判断（System Prompt指令）
- 防冲突：学员回大厅后立刻说话，优先响应学员话题，跳过必然触发

**纯讲述SCO数据**：
- 不强制互动
- 采集被动信号：停留时长、是否中途离开、是否回翻、是否点击工具卡片
- 报告数据优先级：Quiz互动 > 对练行为 > 测评 > 辅导对话 > 被动信号

**碎片生产优化**：
- 学习特质碎片：从"每SCO一个"优化为"每Activity一个汇总碎片"（减少AI调用）
- 认知转化碎片：每Quiz闭环后生成（不变）
- 能力评估碎片：每对练完成后生成（不变）

**跨Course记忆：本期打通**
- 引用优先级：当前Course > 相关Course > 远期Course
- 简单时间排序，不做复杂权重衰减

**可直接写入需求文档的结论**：
- 辅导教练回到大厅时必然引用机制
- 半结构化记录格式扩展（新增passive_signals字段）
- 碎片生产规则定义（Activity级汇总 / Quiz级 / 对练级）
- 跨Course记忆引用优先级规则

## 二、需要创始人决策的7个分歧点

| # | 分歧点 | 方案选项 | 辩论中的倾向 |
|---|--------|---------|------------|
| 1 | **打标SCO对学员是否完全透明** | A: 纯后台行为，学员无感知 / B: 有轻量前台交互（如一道快速选择题） | 倾向A（纯后台），但如果是嵌入Quiz中的打标则学员有感知 |
| 2 | **跨Activity标签传递规则** | A: "影响范围只到下一个打标SCO"仅限Activity内，跨Activity读"最新有效标签" / B: 严格限制为Activity内，跨Activity重新打标 | 倾向A，否则对练无法获取授课的打标结果 |
| 3 | **Quiz对话时中间看板展示什么** | A: 关联知识卡片+进度指示 / B: 保持空白或极简提示 / C: 创始人另有想法？ | 倾向A，但需确认创始人意见 |
| 4 | **课后复盘在教室内还是大厅** | A: 教室内，AOM脚本控制（FEEDBACK_REVIEW SCO） / B: 大厅，辅导教练发起 | 倾向A（保持内容驱动），学员张三倾向B（更自然） |
| 5 | **HR端是否展示绝对排名** | A: 仅成长幅度排名 / B: 同时展示绝对能力排名和成长幅度排名 | HR倾向B，但魔鬼代言人警告绝对排名的小样本统计问题 |
| 6 | **纯讲述SCO是否追加互动引导** | A: 每个讲述SCO结尾追加一句互动 / B: 不强制，用被动信号采集 | 倾向B，学员张三明确反对强制互动 |
| 7 | **跨Course记忆本期是否打通** | A: 打通，辅导教练可引用全部Course记忆 / B: 不打通，先验证单Course闭环 | 倾向A，符合"教练了解你的一切"承诺 |

## 三、可直接写入需求文档v3.2的结论

### AOM数据模型扩展

1. **新增SCO类型**：
   - `ASSESSMENT_TAG`：打标SCO，含`tagConfig`（标签维度、可能标签值、数据来源范围、降级策略）
   - `FEEDBACK_COLLECT`：闭环反馈课前端，采集学员真实场景
   - `FEEDBACK_REVIEW`：闭环反馈课后端，引导学员复盘

2. **新增字段**：
   - SCO层：`referenceSlots`（引用槽，支持`required`标志）
   - Activity层：`segmentSelection.readTagFrom`（跨Activity读取标签）
   - Project层：`universalCapabilityDimensions`（通用能力维度映射）
   - Report层：`rankingDimensions`、`groupingStrategy`

### 三栏布局规范

3. **默认比例定义**：11种场景的左:中:右比例值（见上表）
4. **新增看板组件**："对话回看"面板（用于反思复盘场景）
5. **联动机制**：看板→对话（点击触发解读）、对话→看板（实体高亮/跳转）

### 记忆工程规范

6. **课前采集记忆存储模板**：`structured_tags` + `raw_dialogue`双层格式
7. **辅导教练引用规则**：回大厅必然触发+话题条件触发
8. **碎片生产规则**：Activity级学习特质碎片、Quiz级认知转化碎片、对练级能力评估碎片
9. **半结构化记录格式扩展**：新增`passive_signals`字段（停留时长、是否离开、是否回翻、是否点击）
10. **跨Course记忆引用优先级**：当前Course > 相关Course > 远期Course

### 报告排名规范

11. **学员端**：个人成长时间线+匿名分位区间（不展示名次）
12. **HR端**：完整成长幅度排名+聚合统计+能力维度分布图
13. **成长幅度计算规则**：BEI基线优先；无基线时用前1/3 vs 后1/3均值

## 四、可转化为用户故事的发现

### 新增用户故事

1. **作为学员**，我希望AI在打标判定时如果不确定我的类型，给我默认版本而不是猜错的版本，以便我的学习路径不会因为误判而偏移

2. **作为学员**，我希望在对练复盘时，中间看板展示刚才的对话记录，我可以点击某一轮让AI帮我分析，以便我能具体回顾自己的表现

3. **作为学员**，我希望看板上的数据点可以点击后让右侧Agent自动解读，以便我不需要手动问"这个数据什么意思"

4. **作为HR**，我希望报告中每个学员的成长幅度排名可以排序和筛选，以便我快速找到进步最大和需要关注的学员

5. **作为HR**，我希望项目报告中展示各能力维度的全员均分和分布，以便我向CEO汇报"整体能力提升了多少"

6. **作为学员**，我希望辅导教练在我回到大厅时简短提及刚学完的内容（1-2句话），而不是长篇总结，以便我快速获得反馈而不觉得啰嗦

7. **作为学员**，我希望AI在PPT讲解时不强制要求我互动——我可以安静听完，我的参与度通过其他方式（如停留时长）被记录，以便我按自己的节奏学习

8. **作为学员**，我希望辅导教练记得我在所有课程中分享的内容（不只是当前课程），以便我不需要重复讲述之前分享过的工作问题

9. **作为KGP内容设计师**，我希望在关键知识点处设置"必选引用槽"，确保AI一定在此处引用学员的真实场景，以便闭环反馈不会被遗漏

10. **作为KGP内容设计师**，我希望打标SCO只需定义标签维度和可能的标签值，不需要配置概率权重，以便我专注于教学设计而非技术参数

## 五、R2遗留问题（建议R3或后续讨论）

1. **打标SCO的概率模型参数**：技术团队需要定义不同互动类型的默认权重值，需要技术评审
2. **通用能力维度映射体系**：8个Course的细分维度如何映射到Project级通用维度，需要KGP团队设计
3. **看板组件库的技术选型**：是否采用ShadCN等开源UI框架，需要前端技术评审
4. **碎片生产的AI调用成本估算**：Activity级碎片 vs SCO级碎片的成本差异，需要技术团队评估
5. **成长幅度标准化算法**：消除初始分数偏置的具体计算方法，需要数据团队设计
