# 管理端看板页

## 模块定位

看板页是 HR/运营共用的数据监控中心——查看学习进度、管理催学、配置项目、与 Agent 对话查询一切项目信息。本期 HR 端和运营端完全一致。

---

## 功能区域

| 区域 | 内容 |
|------|------|
| **左侧：目录区**（可折叠） | 总览、学员管理、进度看板、数据展示、催学管理、报告工厂、体验课程入口、系统设置 |
| **中间：管理看板**（主交互区） | 根据左侧选择展示对应功能：项目配置 / 学员列表 / 进度数据 / 催学规则 / 报告管理 |
| **右侧：Agent 对话** | HR/运营 Agent（本期两端一致），自然语言查询 + 报告生产 + 数据解读 |

## 用户故事：运营小李的一个典型工作日

### 场景背景
小李是绚星团队的培训运营，负责某消费品企业"基层管理者培养项目"。项目已进行 6 周，32 名学员在学。今天是周一上午。

### 完整旅程

**09:00 — 登录查看总览**

小李登录运营端，进入看板页。中间看板默认展示「总览」：
- 项目进度：已进行 42 天 / 剩余 138 天 / 整体完成率 28%
- 学员活跃度：上周平均活跃 3.2 天/人，比前一周下降 0.4 天
- 需关注学员 Top 3（进度最低）：王芳 8%、赵刚 12%、刘洋 15%
- 上周催学消息效果：发送 15 条 → 查看 12 条 → 登录 8 人 → 完成活动 5 人

**09:10 — 向 Agent 查询详情**

小李在右侧对话区问："上周进度最快的 3 个人是谁？"

Agent 回答：
> "上周进度提升最快的三位学员：
> 1. 张磊：完成率从 22% 提升到 35%（+13%），完成了横向协作全部授课和对练
> 2. 陈思：完成率从 18% 提升到 28%（+10%），完成了角色认知课程报告
> 3. 周明：完成率从 30% 提升到 38%（+8%），完成了辅导反馈授课模块
>
> 数据截至 2026-05-25 08:00"

**09:15 — 查看学员个人详情**

小李点击左侧「学员管理」，中间看板展示学员列表。点击"王芳"的行，展开个人详情面板：
- 完成进度：8%（仅完成 Course 1 的第一个 Activity）
- 最后登录：5 天前
- 学习时长：累计 1.2 小时
- 催学消息：已发送 3 条，查看 2 条，无后续登录

**09:20 — 检查催学规则**

小李点击左侧「催学管理」，中间看板展示催学规则列表：
- 规则 1：连续 3 天未登录 → 自动发送产品内消息 + Email | 状态：运行中
- 规则 2：进度落后整体均值 20% 以上 → 看板高亮 + 通知运营 | 状态：运行中
- 规则 3：连续 7 天未登录 → 升级为人工确认 | 状态：运行中

小李发现王芳已触发规则 3（7 天未登录），需要人工确认。她编辑一条个性化消息：
> "王芳你好，上次你学的角色认知课反馈很好，接下来的横向协作课程和你的日常工作关联更大——里面有个很实用的 PIN 法可以帮你处理跨部门沟通。有空的话来看看吧。"

消息遵循"价值导向"原则——告诉学员来了有什么好处。

**09:30 — 数据展示看板**

小李点击左侧「数据展示」，中间看板展示详细数据：
- 各 Course 完成率柱状图：Course 1（58%）> Course 2（32%）> Course 3（12%）
- 各 Activity 完成率明细表（Activity 级粒度）
- 活跃度趋势折线图（按周）
- 被动信号聚合数据：平均停留时长 18 分钟/次、中途离开率 22%、回翻率 35%

小李点击 Course 2 的柱状条 → 右侧 Agent 自动解读：
> "Course 2 横向协作的完成率 32%，低于 Course 1（58%）。可能原因：Course 2 包含对练模块，部分学员在对练阶段停滞——有 8 名学员完成了授课但未开始对练。"

**09:40 — 催学效果追踪**

小李查看催学漏斗：规则 1 上周执行情况 → 发送 15 条 → 查看 12 条（80%）→ 登录 8 人（53%）→ 完成活动 5 人（33%）。

转化率尚可，但从"登录"到"完成活动"的转化率偏低。小李问 Agent：
> "登录了但没完成活动的那 3 个人，卡在什么地方？"

Agent 回答："3 人中 2 人停在了对练导入阶段（看了场景卡片但未点击'开始对练'），1 人停在了授课中的 Quiz 环节。"

**10:00 — 结束上午巡检**

小李的上午巡检完成。所有发现已通过 Agent 对话记录，后续可追溯。

## 核心交互清单

### 1. 项目总览

| 项目 | 内容 |
|------|------|
| **触发** | HR/运营登录或点击「总览」 |
| **响应** | 中间看板展示项目整体进度、活跃度、需关注学员 Top 3、催学效果概览 |
| **联动** | 点击任何数据元素 → 右侧 Agent 自动解读 |

### 2. 学员管理

| 项目 | 内容 |
|------|------|
| **触发** | 点击左侧「学员管理」 |
| **响应** | 中间看板展示学员列表（含进度百分比、最后登录时间、活跃状态） |
| **联动** | 点击某学员 → 展开个人详情面板（Activity 级进度、学习时长、催学记录） |

### 3. Agent 自然语言查询

| 项目 | 内容 |
|------|------|
| **触发** | HR/运营在右侧对话区输入任何问题 |
| **响应** | Agent 基于统一 Database 回答，标注数据更新时间 |
| **联动** | Agent 无法回答时坦诚告知并建议联系相关团队 |

### 4. 催学规则配置

| 项目 | 内容 |
|------|------|
| **触发** | 点击「催学管理」→「新建规则」 |
| **响应** | 配置界面：触发条件（未登录天数/进度落后比例等）+ 消息模板（支持变量插入）+ 执行策略（频率/上限/升级规则） |
| **联动** | 普通催学全自动执行；升级情况需人工确认 |

### 5. 催学效果追踪

| 项目 | 内容 |
|------|------|
| **触发** | 查看催学规则的执行效果 |
| **响应** | 展示完整漏斗：发送 → 查看 → 登录 → 完成活动 |
| **联动** | 可按规则/时间段/学员维度筛选 |

### 6. 数据展示看板

| 项目 | 内容 |
|------|------|
| **触发** | 点击「数据展示」 |
| **响应** | 展示学习进度、活跃度、完课率、被动信号聚合数据 |
| **联动** | 所有图表元素可点击触发 Agent 解读 |

### 7. 报告工厂

| 项目 | 内容 |
|------|------|
| **触发** | 点击「报告工厂」 |
| **响应** | 选择报告类型/范围/模板 → 批量生成 → 审核队列 → 导出 PDF |
| **联动** | 审核过程中标注"数据不对"→ Agent 立即给出数据来源和计算逻辑 |

### 8. 体验课程

| 项目 | 内容 |
|------|------|
| **触发** | HR 点击「体验课程」入口 |
| **响应** | 切换为学员身份体验课程 |
| **联动** | HR 账号不在学员列表中时，体验数据单独记录，不污染学员统计 |

## 已确认决策

- 运营端/HR 端本期界面与功能完全一致，两个独立端口
- 本期 Agent 两端一致，不做独立运营 Agent
- 管理端数据粒度为 Activity 级，不做 SCO 级
- 数据呈现为主，本期不做智能预警评价
- 催学规则可配置（条件/模板/策略），HR 端和运营端都可配置
- HR 体验课程的数据隔离：不在学员列表中则单独记录
- 去掉"销售部门"等分组概念——30-40 人暂无学员组划分

---

## 本期不做的功能

- 独立运营 Agent——本期与 HR Agent 一致
- 智能预警评价——先呈现数据，运营/HR 自行判断
- 临时任务下发——运营可通过企业微信等渠道完成
- 学员分组分析——暂无分组需求
- CEO 汇报 PPT 生成——先做好报告格式
