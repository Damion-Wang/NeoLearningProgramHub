### 6.5 阶段 4 · 开营组织 ★ 重点

**业务定义**：admin 在 program config 完成配置 → 批量导入人员名单创建账号（默认拥有学员角色）→ 通过**角色分配**勾选部分人**兼任管理员角色**（角色可多选）→ 确认开营 → 全员账号开放 + 系统消息通知。

**时间范围**：开营前 **1-2 周** 集中操作。

**涉及角色**：admin（持有根账号，做关键决策与配置）/ 销售（协助）/ 产品运营（监控）。

**program config 完整配置流程**（含必填 / 选填 + 默认值）：

| 配置模块 | 字段 | 必填？ | 默认值 / 备注 |
|---------|------|--------|------|
| **项目信息** | 项目名 | 选填 | 默认自动生成（如"{客户名}-培训项目"），admin 可改 |
| | **服务周期** | 选填 | **默认 1 年**——开营前 admin 可通过特殊安全措施修改，**开营后锁定** |
| | 项目周期 | 选填 | **默认 = 服务周期 - 1 周**（admin 可调，需严格确认）|
| | 项目描述 | 选填 | 空 |
| | 客户名 | 选填 | **本期不打通合同**，仅供运营人员标识不同项目用 |
| | 公司画像 context | 选填 | 空（可后续补） |
| **人员名单与角色** ★ | 批量导入人员名单（CSV / Excel + 手动添加）| 必填 | 全部默认拥有**学员角色** |
| | **角色分配**（学员角色 + 管理员角色 · **可多选**）| 选填 | 默认仅学员角色；可勾选某人**兼任管理员角色**（同一人可同时是学员 + 管理员）|
| **催学规则** | **本期仅站内消息** + 触发节奏 + 文案模板（邮件 / IM / SMS 未来扩展，详见 § 6.6.5 / § 7.4） | 选填 | 默认仅站内消息 + 基础文案（可空配） |
| **平台个性化** | Logo / 品牌色 / 欢迎语 | 选填 | 默认睿学品牌 |
| **内容预览** | 浏览本项目绑定的 Course / Activity 内容 | 选填（仅查看）| - |
| **Token 用量统计** | （后台默认开启，无开关）| - | **本期仅作统计，不做开关 / 不做额度限制**（不设单学员上限 / 项目总额度）|

**关键设计 ★**：

- **不存在"管理员账号创建"独立步骤**——所有账号统一通过批量导入创建（默认仅有学员角色），再通过**角色分配**勾选部分人**兼任管理员角色**（一份名单一次完成；角色可多选——同一人可同时是学员 + 管理员）
- **不做学员分组与标签功能**——本期不实现学员分组 / 类型标签 / 入组等高级功能
- **预览模式本期不做**——内容预览通过"内容预览"接口直接进入体验即可，不做"以学员身份模拟登录"的预览模式

**"确认开营"按钮**——明确按钮触发以下连锁动作：

1. **所有账号**（学员 + 管理员）统一开放登录
2. **统一发送系统消息**通知所有学员 + 管理员（含登录方式 / 项目介绍）
3. program config 进入**只读 + 变更管理**状态

**关于"启动会"**：

- **本期平台不支持启动会场景**——启动会是客户线下行为，平台不提供启动会会议室 / 议程模板 / 实时互动等功能
- **平台上的开营是轻量化的**：核心动作 = 账号开放 + 系统消息通知 + program config 状态切换
- 客户如有启动会需求，由 HR 在线下 / 借助第三方会议工具自行安排

**program config 功能映射**：

- 完整配置 UI（多 tab 形态，必填字段标注）
- 配置完整性校验（必填字段未填时阻塞确认开营）
- 人员名单批量导入（CSV / Excel）+ 默认学员角色创建
- **角色分配**（多选 checkbox：学员角色 / 管理员角色）
- "确认开营"按钮 + 连锁动作（账号开放 + 系统消息 + 状态切换）
