# AI老师 — 风险分析矩阵 (Risk Analysis)

**v2 - 基于 requirements v0.3.3 更新于 2026-04-13**

**阶段:** Phase 8 — Validation  
**项目:** ai-teacher  
**日期:** 2026-04-13  
**目的:** 识别、评估、排序所有关键风险，为每个高优先级风险制定缓解策略和早期预警信号  
**标签说明:** [Data] = 已验证数据, [Estimate] = 推算值, [Assumption] = 待验证假设, [Opinion] = 主观判断  
**变更说明:** 基于 v0.3.3 完整产品形态，产品风险细化到模块级别，技术风险增加前端复杂度和数据性能维度

## 1. 风险矩阵总览

### 评估标准

**可能性 (Likelihood):**  
- 5 = 几乎确定（>80%）  
- 4 = 很可能（60-80%）  
- 3 = 可能（40-60%）  
- 2 = 不太可能（20-40%）  
- 1 = 罕见（<20%）

**影响 (Impact):**  
- 5 = 致命（项目终止/商业模式不成立）  
- 4 = 严重（核心假设失败，需根本性调整）  
- 3 = 中等（需要显著调整，但可恢复）  
- 2 = 轻微（需要小幅调整）  
- 1 = 可忽略

**风险等级 = 可能性 x 影响**  
- 15-25: 极高风险（红色）  
- 10-14: 高风险（橙色）  
- 5-9: 中等风险（黄色）  
- 1-4: 低风险（绿色）

### 2.2 产品风险（v0.3.3 细化到模块级别）

|----|---------|--------|------|--------|------|------|
| P1 | **[辅导模块] 教练质量风险：辅导教练的回答质量无法满足"懂你的教练"的承诺** — 辅导教练需要同时具备：跨Course记忆引用（引用优先级三级）、回大厅必然引用一句+话题条件触发、文件层级知识检索（ADR-006）、长期陪伴人格一致性。任何一个环节断裂，学员就会感到"AI不懂我"，辅导的核心价值崩塌 | 3 | 4 | **12** | 高 | [Assumption — ADR-003将辅导确立为主入口，辅导质量决定产品第一印象] |

| P3 | **[记忆工程] 三级记忆管道链路断裂风险：闭环反馈从采集到复盘到辅导引用到报告呈现的完整链路中任一环节断裂** — v0.3.3设计了完美的闭环：FEEDBACK_COLLECT（课前采集，structured_tags+raw_dialogue双层存储）→referenceSlots课中引用→FEEDBACK_REVIEW（课后复盘，memory_id关联）→辅导引用→报告证据链。链路越长越复杂，断裂概率越高。而内测已证明"情感错付"是致命痛点 | 3 | 4 | **12** | 高 | [Data+Assumption — 内测验证了痛点(Data)，解决方案的链路复杂度带来新的断裂风险(Assumption)] |
| P4 | **[打标SCO] 自适应准确度风险：ASSESSMENT_TAG的概率模型打标结果不准确，导致学员被分配到错误的Segment版本** — 打标SCO汇总前序互动SCO的累积信号，用概率模型输出标签。如果置信度系统性偏低（大量<0.6降级为默认版本），自适应形同虚设；如果误判概率高，学员体验更差（如防御拖延型学员被匹配到利益博弈型内容） | 3 | 3 | **9** | 中等 | [Assumption — ADR-008确认架构，概率模型参数由技术团队设定默认值，未经实际数据校准] |
| P5 | **[对练多版本] 打标匹配+剧本质量的组合风险：对练导学阶段打标不准确+对应版本剧本质量不够，双重失败叠加** — 对练需要在导学阶段内置ASSESSMENT_TAG SCO进行打标（因为标签不跨Activity），打标后匹配对练剧本版本。如果打标不准+剧本质量不高，学员将体验到"不相关+不自然"的对练 | 3 | 3 | **9** | 中等 | [Assumption — 两个独立风险的组合，概率叠加] |
| P6 | **[授课模块] PPT自主发挥讲解的质量不可控风险：同一页PPT面对不同学员讲解方式不同，质量无法统一保证** — v0.3.3明确"AI不是照着念PPT，而是根据提示理解教学意图，结合学员状态自主组织讲解"。这意味着每个学员听到的版本不同，AI可能讲偏/讲错/讲浅。虽然有blueprint和rule控制方向，但"怎么讲"的自由度风险高 | 3 | 2 | **6** | 中等 | [Assumption — v0.3.3已决定本期不建立AI讲解质量监控（Round-02 R1确认），依赖System Prompt中的知识点断言清单做基础预防] |
| P7 | **[整体] v0.3.3 P0范围过大导致质量不达标或延期** — 六月版本P0包含：三端三栏布局（11种场景预设+双向联动）、授课（7种SCO类型+引用槽+打标）、对练（导演-演员分离+多版本+对话回看）、测评（对话式+量表式）、报告（BI图表+讨论）、记忆工程三级全做、举证机制、AI质量控制基础版、自适应L1+L2、R3补充功能（20+项）。范围极度庞大 | 4 | 3 | **12** | 高 | [Opinion — v0.3.3的P0范围比v1时期的"6新建+2产品化"更加具体且更加庞大] |

## 4. 极高风险详细分析与缓解

### M1: 战略悖论风险（培训预算缩减 vs ROI可证明性）— 风险值16

**v0.3.3 更新：产品对此风险的解药更加清晰了。**

> v0.3.3设计了从课前采集到课后复盘到报告呈现的完整举证链路。如果这条链路能成功运转，AI老师将是市场上第一个能产出"可举证的行为改变报告"的产品。但链路越复杂，断裂风险越高。 [Data + Opinion]

**缓解策略：**
1. **举证链路优先开发**：v0.3.3资源分配建议中"举证+报告"占40%是正确的。确保FEEDBACK_COLLECT→referenceSlots→FEEDBACK_REVIEW→报告证据链的完整性
2. **报告质量是唯一检验标准**：最终HR拿到的报告中，每个结论是否有具体行为证据支撑，直接决定核心价值主张能否兑现
3. **销售叙事**：不卖"AI培训"，卖"培训ROI可视化解决方案"——帮HR保住培训预算
4. **天使客户标杆案例**：尽快积累3-5个标杆案例，用客户自己的ROI数据说服后续客户

**早期预警信号：**
- [ ] E1访谈中>50%客户提到"培训预算已冻结或削减"
- [ ] 天使客户中>30%表示"今年没有新增AI培训预算"
- [ ] 绚星母公司Q2季报显示客户数继续下降

### M2: 客户将AI培训视为免费升级 — 风险值15

**与v1分析一致，缓解策略不变。**

**缓解策略：**
1. **产品差异化**：v0.3.3定义的完整产品体验（辅导教练+授课+对练+测评+报告闭环）远超"LMS+AI功能"，但需要在销售Demo中充分展示
2. **价值锚定**：对标"5万/次线下专家"而非"800元/人视频课账号"
3. **独立品牌**：NeoLearning作为独立品牌推出
4. **效果承诺**：提供试用后付费机制

**早期预警信号：**
- [ ] E1访谈中>40%客户说"这应该是绚星平台标配功能"
- [ ] Van Westendorp测试中OPP<500元/账号
- [ ] 天使客户试用后不愿转为付费

### P2: 对练剧本质量风险 — 风险值16（v0.3.3新增极高风险）

**这是v0.3.3引入的新极高风险。**

> v0.3.3将对练行为完全委托给剧本控制——导演-演员分离架构意味着AI角色的表演质量100%取决于剧本设计质量。剧本需要定义：角色心理画像（显性目标/隐性目标/触发点/软化点/底线）、行为模式（递进式反应链）、防剧透指令、成功标准（分阶段通关条件）、技能阶梯（L1-L5量化标准）、情绪天花板。这不是简单的"写个剧本"，而是需要深度理解教学设计+心理学+戏剧创作的跨学科能力。 [Opinion]

**缓解策略：**
1. **先制作1个标杆剧本**：以横向协作Course的对练为试点，由团队中最资深的培训专家亲自制作，验证剧本规格是否可执行
2. **建立剧本制作模板和标准**：从标杆剧本中提取可复制的模板，降低后续8个Course的制作门槛
3. **剧本质量评审流程**：每个对练剧本上线前必须经过3轮内部测试（内容专家+产品+学员代表）
4. **降级方案**：如果复杂剧本（多版本+情绪天花板）无法在开营前完成，先上线单版本简化剧本，后续迭代增加复杂度

**早期预警信号：**
- [ ] 首个对练剧本（横向协作）制作周期超过2周
- [ ] 内部测试中AI角色3轮对话内脱离角色设定
- [ ] 情绪天花板机制无法按预期控制AI角色行为
- [ ] KGP无法理解导演-演员分离架构的设计意图

### T3: KGP内容制作能力瓶颈 — 风险值16（v0.3.3从中等升级为极高）

**这是v0.3.3最大的隐性风险升级。**

> v0.3.3的"内容驱动行为"设计哲学（ADR-004）意味着：产品体验的天花板完全由KGP的内容质量决定。平台只是"钢琴"，KGP制作的AOM数据包才是"乐谱"。但v0.3.3对KGP的要求已经从"制作PPT+视频+考试题"大幅提升到"制作AI行为控制脚本+概率模型配置+多版本剧本+闭环反馈设计"。KGP是否具备这种能力，是否有相应的制作工具，这些都是未知数。 [Opinion]

**缓解策略：**
1. **立即评估KGP现有能力**：列出v0.3.3要求KGP制作的所有新型内容（tagConfig/referenceSlots/对练剧本/闭环反馈脚本/学员拒绝处理策略），逐项评估KGP是否有能力和工具完成
2. **优先开发KGP制作工具**：如果现有工具不支持新型内容的制作，需要提前开发或提供模板
3. **内容制作进度监控**：建立周报机制，追踪8个Course中每个Course的内容完成度
4. **范围降级预案**：如果KGP无法按时完成全部新型内容，优先保证横向协作Course的完整内容包，其他Course降级为基础版本（仅PPT+Quiz，不含打标/多版本/闭环反馈）

**早期预警信号：**
- [ ] KGP团队表示不理解ASSESSMENT_TAG的tagConfig如何编写
- [ ] 首个Course的闭环反馈脚本（FEEDBACK_COLLECT+FEEDBACK_REVIEW）制作周期超过1周
- [ ] KGP缺少制作referenceSlots（引用槽）的工具或模板
- [ ] 5月初仅横向协作1个Course的内容包完成

### P9: AI测评无法产出可信L3行为证据 — 风险值15

**与v1分析一致（原P1），核心逻辑不变。**

> 如果AI测评的产出质量不足以让HR信服，核心假设将直接失败。v0.3.3中测评模块已设计了对话式和GUI量表两种类型，BEI追问策略也有详细定义——但这些都是"测"的设计。关键在于"评"的质量：AI基于采集数据产出的评估结论是否有说服力。 [Opinion]

**缓解策略：**
1. **E3实验优先执行**：在MVP全面开发前完成AI测评原型验证
2. **降级方案**：如果纯AI评估不够，设计Human-in-the-loop通道
3. **评估维度聚焦**：先做好1-2个核心维度的深度评估（如"横向协作中的冲突协调能力"），做深做透
4. **与得助智能对标**：参考其14维度评估体系

**早期预警信号：**
- [ ] E3盲测中专家无法区分AI评估报告与通用完课报告
- [ ] 5位HR对样本报告的可信度评分<3.0/5.0
- [ ] AI评分与人工专家评分的相关性<0.6

## 6. 风险相关性分析

### 主风险链条（与v1一致）

```
M1（预算缩减）
    ↓ 加剧
M2（客户不愿付费）
    ↓ 导致
B1（定价假设失败）
    ↓ 导致
F1（收入目标不可达）
    ↓ 触发
B4（客户基础进一步萎缩）
    ↓ 回到
M1（更多预算缩减）= 恶性循环
```

**打断循环的唯一方式：** P9（AI测评产出可信L3证据）→ M1（证明ROI，逆转预算缩减趋势）

### v0.3.3 新增风险链条

```
T3（KGP内容制作能力不足）
    ↓ 导致
P2（对练剧本质量差）
    ↓ 同时导致
P4（打标SCO配置不准确）→ P5（对练版本匹配失败）
    ↓ 同时导致
P3（闭环反馈脚本质量差）→ 记忆闭环链路断裂
    ↓ 最终导致
P9（报告中无有效行为证据）
    ↓ 触发
M1（无法证明ROI，预算继续缩减）
```

**关键洞察：v0.3.3引入了一条新的风险传导路径——KGP内容制作能力（T3）已成为与AI测评可信度（P9）并列的第二条生死线。** 

v0.3.3的"内容驱动行为"设计哲学意味着：即使平台技术完美实现，如果KGP无法制作高质量的内容包（打标配置、对练剧本、闭环反馈脚本、引用槽定义），产品体验仍然会崩塌。这是v1风险分析中被低估的维度。 [Opinion]

## Flags

**Red Flags:**
- **极高风险从4个增加到5个**（新增P2对练剧本质量、T3 KGP从中等升极高）。v0.3.3让产品方向更清晰了，但同时暴露了更多结构性风险。 [Opinion]
- **KGP内容制作能力（T3）从中等风险升级为极高风险**，这是v0.3.3最大的风险变化。"内容驱动行为"的设计哲学是正确的，但它将产品质量的天花板从"技术能力"转移到了"内容制作能力"——而KGP的能力尚未经过验证。 [Opinion]
- **技术风险维度大幅扩展**——三栏布局前端复杂度（TK1）是v0.3.3引入的新高风险，11种场景预设比例+双向联动+拖拽调整+场景切换动画的组合，前端工程量可能超出预期。 [Opinion]

**Yellow Flags:**
- 风险评估中的可能性和影响打分仍基于推断而非数据——这本身就是一个风险。需要通过E1/E2/E3尽快获取一手数据来校准风险评估。 [Assumption]
- v0.3.3的风险是"高质量完整交付"的风险，而非"方向错误"的风险。辩论确认的10条ADR设计决策说明方向日趋收敛。问题不在于"该不该做"，而在于"做这么多能不能做好"。 [Opinion]
