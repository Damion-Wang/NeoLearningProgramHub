# AI 老师 — 产品架构（白板图确认记录）

**来源：** 创始人白板图 + 确认访谈
**日期：** 2026-04-10
**范围：** 6月培训项目的产品规划

---

## 一、产品一句话定义

> AI 老师是一个可编排的 AI 原生培训项目交付引擎，通过自适应的 AI 动作序列（授课/辅导/测评/对练/萃取），将企业培训从"不可验证的课程完成"升级为"可举证的行为改变"。

---

## 二、角色与端口

### 4 个角色（泳道图角色）

| 角色 | 归属 | 职责 |
|------|------|------|
| **学员** | 客户企业 | 学习者 |
| **HR** | 客户企业 | 项目发起人和验收者，关注成果和汇报 |
| **运营** | 供应商侧（我们团队派出） | 全程辅助项目交付，远期目标产品自运营 |
| **内容制作（KGP）** | 供应商侧 | 课程内容生产，不是软件端口 |

### 3 个产品端口（三端合一）

学员端、HR 端、运营端是**同一个 Web 应用**，通过权限控制不同角色看到的入口和功能。HR 也可以切换为学员身份体验课程。

## 三、产品模块架构

### 学员端（独立 Web 应用）

**布局**：左侧看板 | 右侧 Chat UI（辅导对话）

#### 辅导模块（学员端主界面/入口）
- **定位**：模拟长期陪伴的 1v1 辅导教练
- **行为**：基于课程主题相关问题，陪伴式地基于课程内容进行回答和指导
- **附加能力**：推荐项目内的相关课程或任务进行学习
- **右侧**：Chat UI 对话界面
- **左侧看板**内容：
  - 学习进度（个人完成情况）
  - 课程结构（项目整体结构导航）
  - 个人笔记
  - 可带走的内容（学习卡片、工具表格、工具文本）
    - 来源：部分 KGP 预制 + 部分 AI 根据学员情况实时生成
  - 待办任务/作业

#### 三种教学模块（全屏切换，可随时返回辅导）

每种类型下有多种玩法，三种模块之间有**三重隐性关联**：
- **数据关联**：授课中的 Quiz 结果会影响对练的场景选择，测评结果会改变授课路径
- **内容关联**：三种模块围绕同一个知识点/主题运转，共享内容上下文
- **序列关联**：它们之间有固定或推荐的先后顺序（如先授课再对练最后测评）

**类型一：授课模块（课程演绎模块）**
- AI 老师执行一系列多版本 SCO（自适应切换版本），版本是半预制的
- 包含的 SCO 形式：
  - 讲解带有 PPT 的知识点
  - 播放案例视频
  - 简单的小测评（基于当前课程内容，用于自适应或增加代入感）
  - 简单的小解答（基于学员的真实问题和课程中发生过的记忆）
  - Quiz 小测验（类型多样：引发思考、带入个人场景、检验阶段性理解、场景化迁移）
- **本期 Quiz 交互类型**：问答题、选择题、判断题、连线题

**类型二：对练模块**
- **本质**：基于某种内容的 Roleplay，AI 模拟一个角色和学员对话
- 玩法：真实场景演练、角色互换（换位思考）、费曼学习法（AI扮学生）、辩论
- **本版本**：仅支持单人
- **未来迭代**：多人模拟

**类型三：测评模块**
- **本质行为模型**：AI 遵照后台指令，通过对话获取学员的真实信息，并做整理和评判
- 可能的类型：BEI（行为事件访谈）、学员画像、学员困惑收集、学员高光经验萃取
- BEI 时机由内容设计决定，不固定

#### 统一数据层（Database）
- 辅导模块 + 三种教学模块的**所有对话内容**和**生产的总结性内容**统一存入
- **本期单向**流入管理员端，**未来双向**回流影响学员端体验

### HR 端

- HR 通过同一 Web 应用登录，权限控制入口
- 核心能力：
  - 查看学习进度
  - 查看报告
  - 通过 Agent 了解过程中发生了什么（**学习期和结项期都可以随时使用**）
  - 生产汇报素材/文件（按需定制）
- HR 也可切换为学员身份体验课程

### 运营端

- 运营通过同一 Web 应用登录，权限控制入口
- **布局**：标准管理后台 + 右侧运营 Agent
- 运营 Agent 与学员端辅导模块**完全独立**——不同的 Agent，不同的能力，不同的数据源

**运营核心能力（本期）**：
- 导入学员名单、配置项目参数、任务下发
- 查看学习进度、监控异常
- 跟逼催（AI 有默认规则自动执行 + 运营可手动干预和调整）
- 使用报告智能体生产报告、整理成果、输出文件包

**运营行为（本期不做，未来迭代）**：
- 定期布置作业（原在产品外）
- 阶段性评比（原在产品外）
- 奖项/学分规则设置
- 开营/结营活动组织

### 报告智能体（HR 端 + 运营端共享）
- **不仅仅是解读报告**，而是解读所有记录下来的行为数据
- 能力：
  - 解读行为数据
  - 根据需求修改报告
  - 生产其他内容
  - 输出 **PDF/PPT 文件包**
- **贯穿学习期和结项期**，不只是结项时才用

## 四、数据模型

```
Project → Course → Activity → SCO（多版本 + Tag）
                                   ↑ 每种内容形式 = 独立 SCO
                                   ↑ 版本半预制，自适应切换
```

### SCO 粒度
- 一个 PPT 讲解 = 一个 SCO
- 一个 Quiz = 一个 SCO
- 一个案例视频 = 一个 SCO
- 一个小测评 = 一个 SCO
- 一个小解答 = 一个 SCO
- 每个 SCO 支持多版本（半预制，自适应切换）

---

## 五、自适应体系

### 本期
- **L1**：LLM 实时响应（每次交互自适应）
- **L2**：SCO 多版本 Tag 切换（判定点在 Project 层和 Activity 层）

### 未来迭代
- **L3**：Activity 顺序调整
- **更高层面**：实时获取学员信息，临时新增/跳过/略讲某个模块，或生产临时文件

## 六、交互设计决策

| 决策 | 结论 |
|------|------|
| 学员端入口 | 独立 Web 应用 |
| 辅导模块交互 | 左侧看板 + 右侧 Chat UI |
| 进入教学模块 | AI 推荐 + 学员自主选择（两者结合） |
| 教学模块界面 | 全屏切换（PPT/对练/测评各有专属 UI） |
| 返回辅导 | 随时可返回，断点续做 |
| 三端关系 | 同一 Web 应用，权限控制入口 |
| HR 端布局 | 待确认 |
| 运营端布局 | 标准管理后台 + 右侧运营 Agent |

## 七、培训项目生命周期与产品覆盖

### 传统培训项目全流程
```
线下售前 → POC → 签单 → 开营 → 线上学习 → 线下集训 
→ HR成果整理 → 结项 → 供应商结项报告
（开营→结营全程有运营陪伴）
```

### 本次产品覆盖范围
```
开营选人 → 任务下发 → 线上学习 → 实时线上辅导 
→ HR成果整理和报告 → 运营端写成果 → 全流程运营
```

### 不覆盖的部分（线下/人工）
- 线下售前、POC、签单
- 定期线下老师集中授课和辅导
- 结项颁奖/领导汇报（线下活动）

## 八、核心用户体验流程（泳道图）

### 4 泳道 × 4 阶段

| 阶段 | 内容制作(KGP) | 运营 | HR | 学员 |
|------|--------------|------|-----|------|
| **开营前** | 制作全部课程内容（SCO+多版本+Tag+对练脚本+测评框架） | — | — | — |
| **开营** | — | 导入名单、配置项目、任务下发、通知学员 | 提供参训名单（线下） | 收到通知、进入系统 |
| **学习期** | — | 看进度、催学（AI自动+手动）、监控异常 | 看进度、随时通过Agent了解过程发生了什么、随时生产汇报素材 | 辅导→授课/对练/测评循环学习；过程中可带走学习卡片、工具表格、工具文本等内容 |
| **结项** | — | 报告智能体生产报告、整理成果、输出文件包 | 看报告、Agent解答、生产汇报素材 | 查看个人学习报告 |

### 关键数据流
```
学员交互 → 统一Database → 报告智能体 → 运营/HR查看+定制 → PDF/PPT文件包
          (本期单向)
```

## 九、迭代方向（白板黑色部分，非本期）

| 维度 | 含义 | 示例 |
|------|------|------|
| **自适应（高级）** | 实时新增/跳过/略讲模块，生产临时文件 | 学员某知识点已掌握 → 跳过 |
| **模块质量** | 每个模块内部的智能性和交互形态丰富度 | Quiz 未来更多交互形态 |
| **内容覆盖** | 同一软件系统支持不同内容主题 | 销售、项目管理、AI协同等 |
| **增加模块类型** | 授课/对练/测评之外的新类型 | 沙盘、研讨、播客 |
| **感知** | 感知学员学习状态、情绪、心理变化 | 交互埋点 |
| **定制** | 企业的定制需求 | — |
