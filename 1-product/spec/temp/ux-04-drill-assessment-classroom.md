## 1. 阶段一: 情景导入

### 三栏内容定义
- **左侧目录区**: Project目录（无额外Tab）
- **中间看板区**: 三组信息卡片依次展示
- **右侧对话区**: 简要引导提示
- **默认比例**: 10:55:35

### 信息卡片序列

```
第1屏: 背景介绍卡                    第2屏: 目标介绍卡                    第3屏: 技能介绍卡

│ 你的角色:           │    │ 主目标:             │    │ PIN法              │
│ 项目经理张明        │    │ 说服李红参与合作     │    │ 立场-利益-需求分析   │
```

## 2. 阶段二: Roleplay演绎

### 三栏内容定义
- **左侧目录区**: Project目录
- **中间看板区**: AI角色形象 + 场景概览折叠面板
- **右侧对话区**: Roleplay多轮对话 + 学员主动结束按钮
- **默认比例**: 10:35:55（过渡300ms, ease-in-out）

### 看板区布局

```
┌───────────────────────────────┐
│                               │
│      [AI角色形象: 李红]         │
│      (文生图生成的角色图)        │
│                               │
│      市场部负责人               │
│      "对合作持消极态度"         │
│                               │
├───────────────────────────────┤
│ ▶ 场景概览 (折叠)              │
│   背景: ...                   │
│   目标: ...                   │
│   技能: ...                   │
└───────────────────────────────┘
```

### 对话区布局

```
┌──────────────────────────┐
│ [结束对练]  ← 右上角按钮   │
├──────────────────────────┤
│                          │
│ [李红]: 张明啊，你找我什么  │
│ 事？我现在真的忙得很。     │
│                          │
│         [张明]: 李红，我知 │
│         道你最近很忙...    │
│                          │
│ [李红]: 嗯，什么项目？     │
│ 快说吧，我一会还有会。     │
│                          │
├──────────────────────────┤
│ [在这里输入回应...]  [发送] │
└──────────────────────────┘
```

### 关键交互

| 交互 | 触发 | 响应 | 联动 |
|------|------|------|------|
| 对话交替 | 学员发送消息 | 内容安全过滤→导演分析→演员生成回应(3秒内) | 无 |
| 场景回看 | 点击"场景概览"折叠面板 | 展开显示背景/目标/技能信息 | 无 |
| 主动结束 | 点击"结束对练" | 确认弹窗→确定后导演生成结束语→进入复盘 | 三栏比例过渡 |
| 内容安全拦截 | 学员发送不当内容 | 提示"请围绕当前场景对话"，不传递给AI | 记录安全日志 |
| 催促 | 学员>3分钟未回应 | AI角色催促（角色语气） | 无 |

### 内容安全审核流

```
学员输入 → 内容安全过滤层 → 通过? → 是 → 导演分析 → 演员回应
                              │
                              └─ 否 → 对话区提示"请围绕场景对话"
                                      + 记录安全事件日志
```

### 状态变化

| AI角色情绪 | 触发条件 | 视觉表现 |
|-----------|---------|---------|
| 冷淡 | 初始状态 | 短句、不耐烦语气 |
| 缓和 | 学员使用有效沟通策略 | 语气变软，开始展露需求 |
| 升级 | 学员施压/忽视 | 翻旧账，情绪对抗（不超天花板） |
| 合作 | 学员持续有效沟通 | 积极配合，讨论方案 |

## 3. 阶段三: 反思复盘

### 三栏内容定义
- **左侧目录区**: Project目录
- **中间看板区**: 对话回看面板（按轮次展示完整对话记录）
- **右侧对话区**: 教练身份的反思追问
- **默认比例**: 10:50:40（过渡300ms, ease-in-out）

### 看板区: 对话回看面板

```
┌──────────────────────────────────┐
│ 对话回看         [1][2][3]...[15] │ ← 轮次导航
├──────────────────────────────────┤
│                                  │
│ 第1轮 ─────────────────────      │
│ [李红]: 张明啊，你找我什么事？     │
│ [张明]: 李红，我知道你最近很忙... │
│                                  │
│ 第2轮 ─────────────────────      │
│ [李红]: 嗯，什么项目？            │
│ [张明]: 是关于市场推广的合作...   │
│                                  │
│ 第3轮 ━━━━━━━━━━━━━━━━━━━  🔵    │ ← 当前分析高亮（蓝色边框+浅蓝底色）
│ [李红]: 又是CEO安排的？上次...    │
│ [张明]: 这次不一样...            │
│                                  │
└──────────────────────────────────┘
```

### 对话区: 教练反思

```
角色切换动效: 李红头像 → 0.5秒渐变 → 教练图标
气泡颜色: 角色色 → 渐变 → 教练色

[教练]: 刚才的对话我都看了。你做得好的
地方是——第3轮你主动询问了李红的具体
困难，这是很好的共情策略。

[教练]: 在第5轮你提到了CEO的安排，
李红翻了旧账。你觉得为什么？

                     学员: 可能她觉得
                     我在用权力施压...

[教练]: 对的。那用PIN法分析，你跳过了
什么步骤？
```

### 关键交互

| 交互 | 触发 | 响应 | 联动 |
|------|------|------|------|
| 点击轮次 | 在回看面板点击某轮 | 该轮高亮 + 教练自动分析该轮 | 对话区新增分析消息 |
| 轮次导航 | 点击顶部数字按钮 | 面板快速滚动到该轮 | 无 |
| 教练分析联动 | 教练提到某轮 | 回看面板自动滚动并高亮该轮 | 同步联动 |
| 当前高亮 | 教练分析进行中 | 被分析轮次持续蓝色高亮 | 切换分析时高亮随之移动 |

## 5. 对话式测评 (BEI/画像/困惑/高光)

### 三栏内容定义
- **左侧目录区**: Project目录
- **中间看板区**: 背景信息展示（测评主题+说明）
- **右侧对话区**: AI测评对话全程
- **默认比例**: 10:40:50

### BEI测评对话

```
AI: 接下来我会跟你聊大约10-15分钟，了解你的
一些工作经历。能分享一个你最近处理团队冲突的
真实案例吗？

                           学员: 上个月我们组有
                           两个人因为项目分工吵架了

AI: 当时是什么背景？涉及哪些人？(追问S)

                           学员: 我们做一个新项目...

AI: 你具体做了哪些动作？(追问A)

                           学员: 嗯...就一般处理了
                           (第1次模糊)

AI: 能具体说说"一般处理"包括哪些动作吗？

                           学员: 没什么特别的
                           (第2次模糊→触发切换)

AI: 没关系，我们换个话题。能说说最近一次
你成功说服同事的经历吗？
```

**时间预告:** AI开场必须包含"大约10-15分钟"的时间预告
**虚假案例标注:** 后台记录细节一致性评分，不向学员展示

## 6. GUI量表测评

### 三栏内容定义
- **左侧目录区**: Tab1 题目目录(已答/未答) / Tab2 Project目录
- **中间看板区**: 标准化量表GUI界面
- **右侧对话区**: 简要引导（测阶段）→ AI解读对话（评阶段）
- **默认比例**: 10:60:30

### 量表界面

```
看板区                                        对话区
┌───────────────────────────────┐    ┌──────────────────┐
│ 第 7 题 / 共 30 题             │    │ AI: 请根据你的    │
│                               │    │ 第一反应选择，    │
│ 以下描述中，哪个最符合你在团队  │    │ 没有对错之分。    │
│ 中的表现？                     │    │                  │
│                               │    │                  │
│ ○ A. 我倾向于独立决策          │    │                  │
│ ● B. 我喜欢征求团队意见        │    │                  │
│ ○ C. 我通常委托他人决策        │    │                  │
│ ○ D. 我根据情况灵活调整        │    │                  │
│                               │    │                  │
│      [← 上一题]  [下一题 →]   │    │                  │
└───────────────────────────────┘    └──────────────────┘
```

### 断点续答

```
┌─────────────────────────────────┐
│ 上次你在管理风格测评中离开       │
│ 已完成: 18/30 题                │
│                                 │
│ [继续答题]  [重新开始]           │
│                                 │
│ 未完成的测评下次可继续           │
└─────────────────────────────────┘
```

### 评阶段视图切换

量表提交后:
- 看板区: 可展示结果图表（雷达图/柱状图）
- 对话区: 从简要引导切换为AI解读对话

```
AI: 你的量表结果显示，在"团队建设"维度得分
最高(85分)，在"冲突处理"维度相对较低(62分)。
你对这个结果怎么看？

                           学员: 冲突处理确实
                           是我的短板...

AI: 从你的回答来看，你倾向于回避冲突而非
直面处理。这在某些场景下是有效的策略，
但频繁使用可能导致问题积累...
```

## 7. 组件需求

| 组件 | 规格 |
|------|------|
| 信息卡片 | 背景/目标/技能三种卡片，淡入淡出切换，底部按钮 |
| 角色形象展示 | 文生图角色图，居中展示，形象下方角色名+描述 |
| 场景概览折叠面板 | 默认折叠，点击展开显示导入信息，在演绎阶段可随时回看 |
| 对话回看面板 | 按轮次展示，轮次导航条，高亮支持（蓝色边框+浅蓝底色） |
| 轮次导航 | 数字按钮横排，可快速跳转 |
| 内容安全过滤提示 | 系统级提示条，与对话气泡视觉区分 |
| "结束对练"按钮 | 对话区右上角，次要样式，点击弹确认 |
| 量表GUI | 单选/多选/Likert/滑块，支持题目导航，进度指示 |
| 题目目录 | 已答(绿)/未答(灰)/当前(蓝)三态 |
| 角色切换渐变 | 0.5秒头像+颜色渐变动画 |
