# 授课场域模块

## 模块定位

授课场域是 Neo（Tutor）按 AOM 脚本执行结构化教学的空间——Neo 不是照着 PPT 念，而是根据教学提示和学员状态自主演绎，全程可互动。

---

## Neo（Tutor）— 授课场域中的角色定位

### Neo 是谁

**前台名称**：Neo
**角色类型**：Tutor（老师），不是 Coach（教练）
**知识范围**：Activity 内相关知识

### Neo 在授课场域的核心价值

1. **结构化知识传递**——按 AOM 脚本演绎知识，确保教学质量和一致性，同时根据学员状态自主调整讲解方式
2. **即时教学反馈**——通过 Quiz、答疑口播等方式实时检测学员理解程度，发现①缺概念/②遗忘/③误用/④选错工具/⑥错误心智模型等教学信号并即时回应
3. **高光捕捉（P3）**——在教学过程中捕捉学员顿悟瞬间和高光应用，生成推荐笔记卡片（InsightCaptureSkill 轻量版）

### Neo 的核心能力（授课场域）

| 能力 | 说明 |
|------|------|
| **结构化教学** | 按 AOM 脚本演绎 PPT/视频/Quiz/闭环反馈，根据 blueprint 和 rule 自主发挥讲解 |
| **ContentSkill** | 授课口播（主动讲解）+ 答疑口播（被动回应学员提问），知识范围限 Activity 内 |
| **EvaluationSkill** | Quiz 评估反馈、学习信号检测（5种教学信号） |
| **InsightCaptureSkill（轻量，P3）** | 捕捉课堂高光瞬间，生成推荐笔记卡片，学员可选择收藏入笔记库 |

## 功能区域

| 区域 | 内容 |
|------|------|
| **左侧：目录区** | Tab1: SCO 列表（类似 PPT 页面缩略图，已学完的可点击回看，未播放的锁定状态）；Tab2: 当前 Course 的 Activity 列表（显示各 Activity 完成状态，点击已解锁 Activity 可跳转，弹窗提醒确认） |
| **中间：看板区**（课件展示） | 每个 SCO 都有对应内容：PPT → HTML/图片渲染；Video → 视频播放器；Quiz → 题干 + 答题区（选择题/填空题/判断题在此交互；问答题仅展示题干，学员通过对话区回答） |
| **右侧：Neo（Tutor）对话** | 授课口播（Neo 主动讲解，除 Video 外所有 SCO 触发）+ 答疑口播（学员插嘴提问，所有 SCO 均可），两种口播共用对话区 |

> **笔记悬浮球**：屏幕边缘常驻笔记悬浮球，学员可随时展开记录学习要点，不中断授课流程。详见 [06-notes.md](06-notes.md)。

## 授课内容模型

### 看板区内容规则

AOM 序列中每一个 SCO 都在看板区展示对应内容：

| SCO 类型 | 看板区展示 | 授课口播 | 说明 |
|---------|-----------|---------|------|
| **PPT** | PPT 图片/HTML 渲染 | ✅ 有 | Neo 主动讲解 PPT 内容，是授课的主要载体。授课口播支持暂停 |
| **Video** | 视频播放器（支持暂停/快进/后退/进度条拖拽） | ❌ 无 | 视频自身承载内容，Neo 不做讲解。如需总结/复盘，由紧接的 PPT SCO 承担 |
| **Quiz** | 题干 + 答题区（支持图片/表格辅助，点击放大） | ✅ 有 | Neo 对题目做简要引导或解释，学员作答后给反馈。**交互区分**：选择题/填空题/判断题在看板区作答；问答题通过对话区语音或打字输入 |
| **FEEDBACK_COLLECT** | Quiz 形式的场景采集题（问答题） | ✅ 有 | 看板区展示题干，学员通过对话区语音或打字回答 |
| **FEEDBACK_REVIEW** | Quiz 形式的复盘题（题干基于之前采集数据生成，问答题） | ✅ 有 | 看板区展示题干，学员通过对话区语音或打字回答 |

### 两种口播

| 类型 | 触发方式 | 适用范围 | 说明 |
|------|---------|---------|------|
| **授课口播**（主动） | Neo 按 AOM 序列驱动，主动讲解 | 除 Video 外所有 SCO 类型 | PPT 讲解、Quiz 题目引导、采集/复盘引导 |
| **答疑口播**（被动） | 学员插嘴提问触发 | 所有 SCO 类型（含 Video） | 学员随时可以提问，Neo 回应后恢复授课口播 |

两种口播共用右侧对话区，交替出现。授课口播进行中学员插嘴时，Neo 暂停授课口播，切换为答疑口播回应，回答完毕后恢复授课口播。

> 答疑口播的知识范围：当前 Activity 内的教学知识 + 已完成 Activity 的知识 + 学员画像数据（记忆工程）。不含未解锁 Activity 的内容（防剧透）。

### 对话区输入方式

学员可通过两种方式在对话区输入：

| 方式 | 交互 |
|------|------|
| **打字** | 直接在输入框打字发送 |
| **语音** | 点击语音按钮开始说话 → 语音转文字 → 点击发送 |

### 多版本内容与选择判定

AOM 支持同一教学节点提供多个**内容变体**。多版本不是路径分支——授课逻辑/序列完全相同，只是同一位置的内容有不同的呈现风格或场景适配。

**变体类型举例**：

| 变体维度 | 示例 |
|---------|------|
| **呈现风格** | 同一 PPT 的两个版本：一页只展示标题（适合不喜欢高密度阅读的学员） vs 展示详细解释说明 |
| **行业场景** | 同一剧情走向的两个视频案例：销售场景版 vs 建筑行业场景版，由前序身份判定决定 |

**核心概念**：

| 概念 | 说明 |
|------|------|
| **separateTag** | 学员类型标签，标记学员特征（如"简洁型"/"详细型"/"销售行业"/"建筑行业"） |
| **groupKey** | 版本槽位，将同一教学节点的多个变体 Segment 编为一组，系统从中选择一个展示 |
| **tutorMode** | SCO 级开关，`tutorMode=1` 告诉播放引擎"这个 SCO 需要根据学员标签选择变体" |
| **判定环节** | 在 groupKey 位置前由 AOM 脚本设置的评估步骤，用于打标或切换 separateTag |

**流程**：

```
AOM 序列正常推进
        ↓
遇到判定环节（AOM 脚本控制）→ 执行（Quiz/诊断对话/passConditions 规则）→ 打标 separateTag
        ↓
后续遇到 groupKey 分组的 SCO → 根据 separateTag 选择匹配变体
        ↓
Neo 按选中变体的 AOM 脚本做授课口播（授课逻辑不变，内容呈现不同）
```

**判定环节由 AOM 脚本控制**——脚本定义在哪里判定、用什么方式判定、打什么标签。Neo 不自主决定判定时机。

**举例**：

课前 FEEDBACK_COLLECT 采集到张磊是销售岗位 → passConditions 规则打标 separateTag="销售行业"。后续 AOM 序列中：
- groupKey="5.1" 的案例导入 PPT 有 3 个变体 → 选中"销售行业"版本，展示销售场景的案例
- groupKey="5.2" 的视频案例有 3 个变体 → 选中"销售行业"版本，播放销售场景视频

如果是建筑行业的学员 → 同样的教学逻辑，但看到的案例全部是建筑行业场景。

> **tag 作用域限定在 Activity 内，不跨 Activity。**

## 用户故事：PIN 法那节课的完整 25 分钟旅程

### 场景背景
学员张磊进入 Course 2「横向协作」的第一个授课 Activity「PIN 法三步走」。这是一节约 25 分钟的互动课。

### 完整旅程

**00:00 — 课前场景采集（FEEDBACK_COLLECT SCO）**

看板区展示 Quiz 形式的场景采集题：

> 题干：「你在横向协作中有没有遇到过特别棘手的情况？比如需要别的部门配合，但对方不太配合的场景？请描述一个具体的例子。」

Neo（Tutor）在对话区做授课口播：
> "正式开始之前，先了解一下你的实际情况。看看左边的问题，说说你遇到过的一个真实场景——打字或者语音都行。"

张磊在对话区打字回答："上周市场部的张总找我要紧急调配 3 个销售支援他们的活动，但我这边 Q2 指标压力很大，根本抽不出人。"

Neo 在对话区追问 1-2 轮，提取关键信息：
- 场景类型：跨部门资源协调
- 关键词：资源争夺、优先级冲突
- 情绪状态：焦虑、被动
- 具体挑战：市场部紧急需求 vs 自身指标压力

数据以 `structured_tags` + `raw_dialogue` 双层格式写入记忆，分配 `memory_id`。

Neo 过渡（授课口播）：
> "明白了。这种情况非常典型——双方都有自己的压力和利益。接下来要学的 PIN 法，就是专门用来处理这类场景的。学完之后，我们再回过头来看看你和张总的这个情况。"

**02:30 — 负面案例视频（VIDEO SCO）**

左侧 SCO 列表中，视频 SCO 高亮为当前项。看板区切换为视频播放器。

视频展示一个负面案例：某主管遇到跨部门资源请求，直接回应"我这边也很忙，你去找别人吧"，结果关系恶化、项目延期。

视频播放期间 Neo 不做授课口播，学员可随时在对话区提问（答疑口播）。

**03:30 — 视频复盘（PPT SCO）**

视频播放完毕后，看板区自动切换为复盘 PPT——列出视频中的关键节点和双方反应。

Neo 做授课口播：
> "这个案例里，李经理只看到了对方表面的 Pressure——'给我人'，完全没探索背后的 Interest 和 Need。结果双方都觉得不被理解。你觉得他哪一步最致命？"

张磊回答后，Neo 引导过渡到概念讲解。

**04:30 — P 概念讲解（PPT SCO）**

看板区展示 PPT 图片——冰山模型示意图，上方是 Pressure，下方是 Interest 和 Need。

Neo 不照念 PPT，而是自主发挥：
> "PIN 法的第一层是 Pressure——也就是对方表面上说的那个诉求。就像刚才视频里那位李经理听到的'给我调 3 个人'。Pressure 通常是最直接的、最有情绪的——因为它来自上层压力。但如果你只回应 Pressure，就像两个冰山在水面上互相碰撞。"

张磊突然插嘴："等一下，P 和普通的需求有什么区别？"

Neo 立即响应（不打断课程节奏）：
> "好问题。Pressure 不是需求，是压力的外显表达。比如张总说'紧急调人'——这是 Pressure。他的真实需求（Need）可能是'保证活动不翻车，不在老板面前丢脸'。Pressure 是表面，Need 是深层。等下讲到 N 的时候你会更清楚。"

讲完后 Neo 自动回到当前进度。

**07:00 — I 概念讲解（PPT SCO）**

PPT 展示 Interest（兴趣/利益点）层。Neo 结合张磊之前的场景自主讲解（此处是必选引用槽 `referenceSlots[required: true]`）：
> "Interest 是比 Pressure 深一层的东西。拿你之前说的那个情况——张总表面在要人，但他的 Interest 可能是什么？也许是'这个活动效果好，他在老板面前能加分'，也许是'他的团队能力不足需要外部支撑'。如果你能探到他的 Interest，谈判空间就大了。"

**09:00 — N 概念讲解（PPT SCO）**

PPT 展示 Need（深层需求）层。Neo 继续自主讲解。

**11:00 — 概念验证 Quiz（QUIZ SCO）**

看板区切换为 Quiz 模式，展示题干和辅助图片。

Neo 做授课口播，简要引导：
> "刚才讲了 P、I、N 三层概念，现在来检验一下。看看左边这道连线题——把每条表述归到正确的层级。"

题目类型：连线题。给出一个场景描述，要求张磊判断哪些表述属于 P、哪些属于 I、哪些属于 N。

张磊将"给我调 3 个人"连到了 I（错误）。

Neo 反馈：
> "这条其实是 Pressure，不是 Interest。'给我调人'是一个直接的压力表达——它很紧迫、很有情绪。Interest 会更深一层，比如'通过这次活动在公司获得更多话语权'。你再看看其他几条？"

张磊修正后全部连线正确，通过。

**13:00 — 正面案例视频（VIDEO SCO）**

看板区播放正面案例视频：另一位主管面对同样的跨部门请求，先追问对方的 Interest，发现对方真正需要的不是"3 个人"而是"活动现场有经验丰富的人坐镇"，于是调了 1 个资深员工 + 提供远程支持方案，双方满意。

视频播放期间 Neo 不做授课口播。

**14:00 — 视频复盘（PPT SCO）**

看板区切换为复盘 PPT——对比正反两个案例的关键行为差异。

Neo 做授课口播，总结关键点并引导张磊结合自身经验思考。

**15:00 — 案例分析 Quiz（QUIZ SCO，问答题）**

看板区展示问答题题干：「在刚才的正面案例中，那位主管的哪个动作是关键转折点？他是怎么从 Pressure 探到 Interest 的？」

Neo 做授课口播，简要解释题意：
> "这道题考的是从案例中提炼关键行为。想想那位主管做了什么，才让对话从'要人'变成了'找方案'。打字或者语音说都行。"

张磊在对话区回答后，Neo 用苏格拉底式引导追问 2 轮，帮他深入理解。

**17:00 — 四象限方法 PPT 讲解（4 个 PPT SCO）**

四页 PPT，分别讲解横向协作中四种典型人物画像和应对策略。Neo 逐页做授课口播，在讲到"资源争夺型"象限时，Neo 自主判断此处偏抽象理论，在对话区插入一个互动问题（SCO 间互动自主决定，不新增 SCO）：
> "你之前提到的张总的情况——你觉得她属于四象限中的哪一种？"

张磊在对话区回答，Neo 做简要点评后继续下一页讲解。

**21:00 — 综合升华（PPT SCO）**

Neo 做整体总结，将 PIN 法和四象限方法关联起来。

**23:00 — 课后复盘（FEEDBACK_REVIEW SCO）**

看板区展示 Quiz 形式的复盘题——题干基于课前采集数据（通过 `memory_id` 关联）自动生成：

> 题干：「还记得你开始时分享的情况吗？市场部张总要你紧急调 3 个人支援活动。现在学完 PIN 法，请分析：张总的 P（表面压力）、I（利益点）、N（深层需求）分别是什么？如果重来一次，你会怎么和他谈？」

Neo 做授课口播引导：
> "这道题没有标准答案——用你刚学的 PIN 法，重新拆解你自己的真实场景。说说你的分析。"

张磊在对话区回答分析。Neo 逐层点评，引导他深入思考 Interest 层，给出具体建议。

**25:00 — Activity 完成**

场域最后一屏显示：
- "下一个任务：PIN 法跨部门协商对练 | 约 20 分钟"
- [开始] 按钮（主要动作）
- "回到辅导" 链接（次要动作）

## 核心交互清单

### 1. SCO 锁定机制

| 项目 | 内容 |
|------|------|
| **触发** | 学员在左侧 SCO 列表中点击未播放到的 SCO |
| **响应** | 该 SCO 显示为锁定状态（灰色+锁图标），不可点击。学员只能回顾已学完的 SCO |
| **联动** | 回顾已学完 SCO 的流程见交互 #4（主动回顾跳转） |

### 2. 防挂机机制

| 项目 | 内容 |
|------|------|
| **触发** | 学员连续 5 分钟无任何操作（无键盘输入、无鼠标点击、无滚动） |
| **响应** | 系统弹出提醒或暂停课程，需学员确认后继续 |
| **联动** | 暂停期间不计入学习时长 |

### 3. 学员插嘴提问

| 项目 | 内容 |
|------|------|
| **触发** | 学员在 Neo 讲解过程中输入任何问题 |
| **响应** | Neo 立即暂停当前讲解，回答学员问题。如问题涉及后续内容，告知"后面会讲到"并将问题写入记忆 |
| **联动** | 回答完毕后自动恢复到当前讲解进度，不重复中间内容 |

### 4. 主动回顾跳转

| 项目 | 内容 |
|------|------|
| **触发** | 学员点击左侧目录中已学完的某一页 SCO |
| **响应** | Neo 简要讲解该页核心内容（不完整重播） |
| **联动** | 讲解完毕后 Neo 询问"要回到刚才学到的地方吗？还是继续看看其他内容？"学员确认后再跳回原进度位置。学员也可以继续点击其他已学完 SCO 继续回顾 |

### 5. Quiz 答题反馈

| 项目 | 内容 |
|------|------|
| **触发** | 学员完成 Quiz 答题（选择题/填空题/判断题在看板区提交；问答题在对话区回答） |
| **响应** | Neo 根据 AOM 脚本中定义的反馈规则进行演绎——包括是否允许透露答案、是否允许提示、反馈深度等 |
| **联动** | 若 AOM 规则允许提示，学员说"不知道"或"给我提示"时 Neo 给引导性提示；若 AOM 规则不允许透露答案，Neo 不提供提示，引导学员自行思考 |

### 6. 视频播放与复盘

| 项目 | 内容 |
|------|------|
| **触发** | Video SCO 开始播放 |
| **响应** | 视频在看板区播放，Neo 不做授课口播（视频自身承载内容）。学员可随时在对话区提问（答疑口播） |
| **联动** | 如需总结/复盘，由 AOM 序列中紧接的 PPT SCO 承担——看板区切换为复盘 PPT，Neo 对该 PPT 做授课口播 |

### 7. SCO 间互动自主决定

| 项目 | 内容 |
|------|------|
| **触发** | Neo 讲完一个完整知识点模块后自主判断 |
| **响应** | 如果内容偏抽象理论，在对话区插入一个结合学员工作场景的互动问题；如果内容已足够互动，直接继续 |
| **联动** | 互动问题必须与当前内容教学相关，不能泛化（如"你觉得呢？"不行）。不新增 SCO，不影响 AOM 数据结构。学员不被强制互动，不回应时通过被动信号采集参与度 |

### 8. 学员主动要求出题

| 项目 | 内容 |
|------|------|
| **触发** | 学员在对话区说"再出一道题"/"我想练习一下"等 |
| **响应** | Neo 基于大模型能力在对话区即时生成一道与当前知识点相关的练习题，不新增 SCO |
| **联动** | 学员回答后 Neo 在对话区给出反馈，然后继续正常授课流程 |

### 9. Activity 完成判定

| 项目 | 内容 |
|------|------|
| **触发** | 学员到达 AOM 序列最后一个 SCO |
| **响应** | 所有 SCO 完成即 Activity 完成。Quiz SCO 的完成标准是满足 passConditions（不要求答对，但要完成判定流程） |
| **联动** | 大厅课程目录中该 Activity 状态变为 ✅，场域最后一屏展示下一个任务卡片 |

### 10. Activity 间跳转

| 项目 | 内容 |
|------|------|
| **触发** | 学员点击左侧 Tab2 中的某个 Activity |
| **响应** | 若该 Activity 已解锁，弹窗提醒"确定要离开当前课程吗？当前进度已保存。"确认后跳转 |
| **联动** | 当前 Activity 触发断点续播保存。未解锁 Activity 不可点击 |

### 11. 断点续播

| 项目 | 内容 |
|------|------|
| **触发** | 学员中途退出授课场域（关闭浏览器、点击返回大厅、网络断开等） |
| **响应** | 系统记录当前 SCO 位置和对话上下文。学员下次进入同一 Activity 时，自动恢复到上次离开的 SCO，Neo 简要回顾上次进度后继续 |
| **联动** | 大厅课程目录中该 Activity 显示为 ▶ 学习中状态，Neo 登录消息中提示"你上次学到了[X]，要继续吗？"（该提示发生在大厅，不在场域内） |

## 已确认决策

- [ADR-004] 授课行为由 AOM 脚本控制，PPT 讲解根据提示和学员状态自主发挥
- SCO 锁定：未播放到的 SCO 锁定，学员只能回顾已学完内容
- 防挂机：每 5 分钟无操作时提醒/暂停
- SCO 间互动由 Neo 根据内容逻辑自主决定，在对话区进行，不新增 SCO
- 学员要求出题时 Neo 在对话区即时出题，不新增 SCO
- Quiz 交互区分：选择题/填空题/判断题在看板区作答，问答题通过对话区语音或打字输入
- Quiz 答题反馈遵循 AOM 脚本规则（是否允许提示、是否透露答案等）
- 主动回顾跳转：讲解完毕后确认再拉回，学员可继续回顾其他内容
- 闭环反馈拆分为 FEEDBACK_COLLECT + FEEDBACK_REVIEW 两个独立 SCO，通过 memory_id 关联，均以 Quiz 形式在看板区展示
- 授课口播 vs 答疑口播：除 Video 外所有 SCO 类型均有授课口播；Video 仅支持答疑口播
- 视频后如需总结，由紧接的 PPT SCO 承担授课口播，Video SCO 本身不配讲解动作
- 授课脚本 referenceSlots 支持必选/可选引用槽
- 答疑口播知识范围覆盖当前 Activity 内教学知识和聊天记录
- 所有对话半结构化记录，写入统一 Database，各模块按需读取
- 学员拒绝分享时的处理由 AOM 脚本定义
- 视频支持常规播放控制（暂停/快进/后退/进度条），PPT 授课口播支持暂停
- Activity 完成条件：所有 SCO 完成，Quiz SCO 完成标准是满足 passConditions
- 左侧 Tab2 点击已解锁 Activity 可跳转，弹窗确认，当前进度自动保存
- 对话区支持打字和语音输入（点击语音按钮 → 说话 → 语音转文字 → 点击发送）
- 多版本是同一教学逻辑的内容变体（呈现风格/行业场景不同），不是路径分支
- 伪视频格式：本期视频数据结构与标准 MP4 不同，但对学员体验无影响
- 自由复习：学员可从 hall 重新进入已完成的 Activity，和 Neo 答疑讨论+翻看之前页面（SCO 全部已解锁状态）

## 反向补充：Activity 完成弹窗（精细化）

原设计：Activity 完成后在看板区显示完成卡片+任务链接。

demo 演化为：**Modal 弹窗形态**，更强引导+不被其他内容干扰。

### 弹窗结构

```
┌─────────────────────────────┐
│                        [X]  │
│         ✅                    │
│    Activity 完成！             │
│  你已完成「A3 重构被动型员工」  │
│                              │
│  [开始下一个对练] [回到辅导]    │
└─────────────────────────────┘
```

### 按钮行为

| 按钮 | 视觉 | 目标 |
|------|-----|------|
| 开始下一个对练 | 主 CTA，cyan→violet 渐变 | 跳到 AOM 下一节点（对练/报告/下一 Activity） |
| 回到辅导 | 次要边框 | 跳到大厅（Neo 会接收该 Activity 完成状态） |

### 触发与关闭

- 触发：最后一个 SCO 完成后自动弹出（可手动触发预览）
- 关闭：按钮点击跳转、X 关闭、Esc 关闭、遮罩点击关闭
- 遮罩：半透明黑色 + blur，背景页面 blurred

### 记忆
- 弹窗关闭后，AOM 记录该 Activity 为已完成
- 下次进入该 Activity 自动解锁所有 SCO（自由复习模式）
