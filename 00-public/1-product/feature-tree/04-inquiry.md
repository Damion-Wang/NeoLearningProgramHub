# 功能树 04：Inquiry（调研场域）— **P1 可选**

**Owner：** 胡璐、打野
**日期：** 2026-04-21
**版本：** v0.4.0 对齐版（L1+L2）
**对应 Spec：** `spec/design/learner/04-inquiry-classroom.md`

## 一句话定位

Neo（Tutor）通过对话或 GUI 量表采集学员真实信息并进行评估。不是考试，是深度访谈或标准化量表。

**优先级**：P1 可选 —— 视开发资源决定是否纳入六月版本。

---

## 功能结构

```
A. Neo 在调研场域
   A1 Neo 身份（Tutor；Activity 内 + 已完成 Activity + 学员画像）
   A2 结构化访谈 / 量表执行（按调研剧本）
   A3 温和专业访谈（被倾听而非被审讯）
   A4 信息确认与修正能力

B. 三阶段流程 ★v0.4.0 从两段改三段
   B1 阶段一：调研说明（双栏）
       B1.1 说明卡片（主题 / 时长 / 形式 / "无标准答案"）
       B1.2 Neo 温和开场
   B2 阶段二：核心调研（单栏沉浸）
       B2.1 对话式 → Neo 按剧本引导
       B2.2 量表式 → GUI 量表界面
   B3 阶段三：信息确认复盘（双栏）★不是苏格拉底，是信息确认
       B3.1 Neo 提炼 3-5 个核心信息点
       B3.2 学员逐一确认 / 修正
       B3.3 追问是否有补充

C. 两种调研类型
   C1 对话式 — interviewType 参数区分
       C1.1 BEI 行为事件访谈（STAR 追问 + 完整性检查）
       C1.2 学员画像采集（profile）
       C1.3 困惑收集（confusion）
       C1.4 高光萃取（highlight）
   C2 GUI 量表测评
       C2.1 标准化量表界面（点选 / 滑块）
       C2.2 自动计分（按剧本计分逻辑）
       C2.3 结果写入统一 Database

D. 剧本控制追问策略
   D1 访谈剧本定义追问路径 / 深度 [ADR-004]
   D2 能力维度由 KGP 定义
   D3 拒绝分享处理（脚本控制，不硬编码统一规则）
   D4 回避行为阈值由剧本定义（如 2 次模糊后切换新案例）

E. 设计哲学
   E1 AI 不做判官（编造案例也是学习信息）
   E2 不完整记录本身就是评估信息

F. 采集产出
   F1 BEI 案例证据 → 统一 Database
   F2 画像 / 困惑 / 高光碎片 → 统一 Database
   F3 量表结果 → 统一 Database
   → BEI 评分由 Evaluation 处理

G. 布局（三阶段栏数不同）
   G1 说明阶段：双栏（说明卡片 + Neo 对话）
   G2 核心阶段：单栏沉浸
   G3 确认阶段：双栏（信息点 + Neo 对话）

H. ★断点恢复 [补漏]
   H1 阶段级断点（恢复到"上次在说明 / 核心 / 确认"哪个阶段）
   H2 核心阶段轮次恢复（对话式：恢复到上次对话的最后一轮 + Neo 状态；量表式：恢复到已答题目位置）
   H3 确认阶段信息点保留（已确认 / 已修正的信息点不重复询问）
   H4 超时清理（超过 N 天未完成的调研作废，下次进入重新开始）
   → 规范沿用 Infra M 统一断点恢复机制

附录 Y. 未来扩展
   Y1 萃取（经验提炼）
   Y2 考试
   Y3 其他信息采集类行为
```

## 跨模块依赖

## 与 v0.3.3 差异

- ★ **两段 → 三段**（说明 / 核心 / 信息确认复盘）
- ★ **复盘 ≠ 苏格拉底**：v0.4.0 明确复盘 = 信息确认，不再是对话式评估
- 原 G2 "评（分析评估）"阶段移除，合并到 B3 信息确认
- 原 I3.2 "量表-测阶段右侧"需按新三阶段重新定义
- 从"测评（Assessment）"正式改名 **Inquiry（调研）**
- 整个模块标注 **P1 可选**
