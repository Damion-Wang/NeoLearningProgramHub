## 0 · debate 配置（基于 10 题确认）

### 主题
基于当前 spec v0.4.0 + 50 题决策清单，深化产品设计的三个维度：
1. 项目全生命周期（重写：商机 → 服务截至）
2. GUI spec（6 模块逐一深化）
3. Agent BDD（5 场域 × Agent 能力 × BDD 用户故事）

### 三轮调度（一轮一维度）
| 轮 | 主题 | SDD skill 借鉴 | 主输入 | 主输出 |
|----|------|--------------|--------|--------|
| 1 | 项目生命周期 | define-feature | requirements § 3 + 0424 决策 S-11/S-12 + 早期 a3 v0.1 + 50 题决策 | 重写的 7 阶段全生命周期文档 |
| 2 | GUI spec | (PM 自由讨论 + define-feature) | 6 模块 spec + 原型 v1.1 + 50 题决策 GUI 相关 | 6 模块 GUI 深化清单（每模块关键交互）|
| 3 | Agent BDD | write-story + write-behavior + group-stories + define-feature | Neo Soul final + Memory 设计 + 7 动作理论家庭 + 50 题决策 | 5 场域 × Agent 能力 BDD 用户故事 |

### 7 角色配置
| 角色 | 简称 | 立场 |
|------|------|------|
| **主持人** | Host | 引导讨论、识别冲突、裁定共识、控制节奏 |
| **挑战者** | Challenger | 反对一切、挖矛盾、问"如果不…会怎样" |
| **产品经理** | PM | 用户体验闭环、商业可行性 |
| **交互设计师** | UX | 视觉一致性、交互流畅、可用性 |
| **学员代表** | Student | 新手友好、学习负担、心理安全 |
| **企业 HR 代表** | HR | 管理效率、数据可见性、客户成功 |
| **业务专家** | Expert | 基层管理学内容专家 + 1v1 教练实践经验 |

### 流程机制
- **共识判定**：主持人裁定（听 7 角色 + 挑战者反驳后拍板）
- **网络搜索**：仅轮间环节调用（轮内不打断辩论）
- **DM 互动**：不锁题量，主持人按需拍板
- **每轮多个来回**：自由讨论直到主持人判共识达成
- **过程记录**：一字不拉记录所有发言

## 2 · 三轮议题展开

### 轮 1 · 项目全生命周期（重写）

**输入**：
- requirements-v0.4.0 § 3（粗，1 页 7 阶段）
- 早期 a3-project-lifecycle-sop-v0.1.md（七阶段骨架，因偏 SOP 暂搁置）
- 0424 决策 S-11（项目周期默认 6 月 + 账号成本分摊）/ S-12（结项后只读）/ N-01（项目周期 vs 账号有效期分离）
- 50 题决策中"研发后天使客户试用调优"

**核心议题**（候选，主持人开场展开）：
1. 7 阶段完整性：商机识别 / 客户意愿确认 / 签单（合同+计费）/ 开营组织 / 学习期 / 结营 / 服务截至 — 漏了哪些子阶段？
2. **N-01 周期分离**：项目周期 vs 账号有效期 是否产品层面感知 / 商业层面拆分？
3. **S-12 结项后状态**：只读 vs 延续期 vs 立即关闭 — 学员体验 / HR 期望 / 商业模式 哪个为先？
4. **S-11 计费维度**：账号成本分摊 vs 项目打包 — 影响哪些产品决策？
5. 试用调优期与正式开营的关系：试用算"商机"还是"开营"？
6. KGP 内容预制 + 客户定制 的边界：本期 100% 预制菜，定制是 V2？

**预期输出**：
- 一份重写的"项目全生命周期"文档（替代 requirements § 3 草稿）
- 涵盖每阶段：触发 / 输入 / 输出 / 角色 / 关键决策点
- 与 0424 决策的协调结论

**轮间环节**（轮 1 → 轮 2）：
- 网络搜索：企业培训项目交付的工业实践（如已有但凭印象，需求证）
- DM 互动：主持人判定的 N 题（按需）

## 3 · debate 节奏建议

### 单轮节奏（参考）
1. 主持人开场 + 列议题
2. 7 角色依次发言（轮 1 每议题）
3. 主持人识别冲突点
4. 挑战者反驳关键发言
5. 冲突方互相回应
6. 自由讨论（多个来回直到趋同）
7. 主持人判共识达成 / 列待决议题
8. 阶段性结论总结

### 全部三轮工作量
- 一字不拉记录意味着每轮可能 5K-10K 字
- 三轮合计 15K-30K 字 + 调研报告 + DM 互动 + 总结
- 因不锁时间，**研发能力决定，本会话 + 后续会话连续推进**
