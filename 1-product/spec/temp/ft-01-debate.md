# 功能树 01 Debate 记录: 三栏布局框架 + 账户与权限

**日期:** 2026-04-13
**状态:** Debate 完成

---

## R1: 功能树走查

### 学员张三（一线使用者视角）

1. **缺失 — 折叠后的恢复操作**：A1.1.1 只写了折叠，没写怎么展开回来。我折叠了目录区，找不到展开按钮怎么办？需要明确展开触发方式。
2. **缺失 — 输入框能力不足**：A1.3.3 只写了"文字输入+发送"，需求文档提到辅导教练要引用我的具体情境，我能不能发图片/文件？至少本期要明确"仅文字"。
3. **断点恢复体验模糊**：B5.2.2 弹窗提示"是否继续"——如果我选"否"呢？去哪里？功能树没写"否"的分支。
4. **缺失 — 首次进入引导**：需求文档 5.2 节明确要求"首次进入辅导老师自我介绍（LUI 思维）"，功能树 B1.2 只写了"通过通知链接首次进入"，没有体现首次登录后的引导流。

### HR 李姐（管理者视角）

1. **角色切换缺失返回入口位置**：B3.1/B3.2 写了 HR↔学员切换，但没写切换按钮在界面哪里。三栏布局里这个按钮放在哪？需要在 A 部分有体现。
2. **缺失 — HR 端目录区内容**：A1.1.2 写"根据场景自动加载目录内容"，但没有定义 HR/运营端的目录区具体包含什么。需求文档 4.3 明确了 HR 目录区有：项目总览、报告、数据。
3. **体验课程的入口缺失**：B2.3.2 提到 HR 可切换学员身份体验课程，但功能树没在布局部分定义"体验课程"入口的位置。
4. **B2.4.2 表述不清**：运营 Agent "本期与 HR Agent 一致"，这个信息放在权限树里不合适，它是功能定义不是权限定义。

### 课程设计师（内容与教学视角）

1. **A1.1.3 Tab 切换描述不完整**：只写了授课教室有 Tab，需求文档 4.4 明确测评教室量表类也有题目目录 Tab。功能树遗漏了。
2. **A2.3 场景比例与需求文档一致**，确认无误。
3. **A3.2 对话→看板联动粒度不够**：A3.2.1 "Agent 提到某个学员/课程/数据"——触发条件太模糊。是靠 NER 实体识别？还是结构化 metadata？需要明确技术触发机制，否则开发无法实现。
4. **缺失 — 看板区内容分端定义**：需求文档 4.2 明确三端看板内容不同（学员看板 vs HR 看板 vs 运营看板），功能树 A1.2 完全没区分。

### 交互设计师（交互与体验视角）

1. **A1.3.2 拖拽只写了右侧对话区**：需求文档说"对话区宽度可调"，但实际三栏的左右边界都应该有拖拽能力。目前只有右侧有拖拽，左侧折叠/展开是二态开关，逻辑不统一但可接受——需要明确写"左栏仅折叠/展开，不支持拖拽"。
2. **A2.1.1 过渡动画 0.3 秒**：与需求文档一致，OK。但 A4.3 加载态只写了"过渡动画/骨架屏"，没有区分场景切换动画和数据加载骨架屏，两者体验不同。
3. **缺失 — 键盘快捷键/无障碍**：本期可标注为"不做"，但应该在功能树中显式排除。
4. **A4.2 移动端提示太简单**：需要指定在什么屏幕宽度下触发提示，以及提示后是否完全阻断使用。

### 魔鬼代言人（找漏洞视角）

1. **B1.3 "记住登录"缺安全边界**：多久过期？浏览器清 cookie 后呢？多设备同时登录允许吗？功能树没有任何安全策略。
2. **B5.2.5 "后台打标 SCO 自动跳过"属于授课模块细节**：放在"账户与权限→断点恢复"层级不对。断点恢复应该只管"恢复到哪"，具体跳过逻辑属于授课教室。
3. **B4 项目上下文 — 本期单项目为何还需要 B4.2 项目隔离？** 既然本期不考虑多项目（B4.1 已注明），B4.2 是未来功能，不应出现在本期功能树。
4. **A2.1.3 教室间直跳描述有误**："通过隐藏目录中的 Project 目录"——什么叫"隐藏目录"？是左侧目录区折叠后的某个入口？触发方式描述不清。
5. **B5.3 重新学习放在断点恢复下不合适**：重新学习是用户主动选择，不是断点恢复场景。应该独立或归入教室功能。

1. 目录区展开操作的触发方式
2. 输入框能力边界（本期仅文字）
